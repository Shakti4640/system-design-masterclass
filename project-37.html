🎯 HYPOTHETICAL PROBLEM

You are the payments architect at an e-commerce platform

→ 50 million customers
→ 5 million transactions per day
→ $500 million in daily transaction volume
→ Operates in 30 countries with 15 currencies
→ Peak: Black Friday — 50,000 transactions per MINUTE

A customer in Berlin clicks "Place Order":
  → Cart: 3 items totaling €147.50
  → Payment method: Visa credit card
  → The system must:
      1. Verify the items are still in stock
      2. Reserve the inventory (so nobody else gets them)
      3. Authorize the payment (check card is valid, funds available)
      4. Capture the payment (actually charge the card)
      5. Confirm the order
      6. Trigger shipping
  → ALL within 3 seconds
  → If ANY step fails → undo EVERYTHING cleanly

NOW the hard parts:

  SCENARIO A: Payment Gateway Timeout
    → We send charge request to Stripe/Adyen
    → No response for 30 seconds
    → Did they charge the card or not?
    → We DON'T KNOW
    → If we retry → might double-charge
    → If we don't retry → might lose the sale
    → WHAT DO WE DO?

  SCENARIO B: Partial Failure in Multi-Step Transaction
    → Inventory reserved ✅
    → Payment authorized ✅
    → Payment captured ✅
    → Order confirmation service is DOWN ❌
    → We charged the customer but can't confirm the order
    → Must we refund? Or retry the confirmation?

  SCENARIO C: Reconciliation Mismatch
    → Our records say: 5,000,000 transactions today = $500M
    → Stripe's records say: 5,000,003 transactions = $500,000,089.97
    → 3 transactions exist at Stripe that we don't know about
    → $89.97 discrepancy
    → WHERE did those 3 phantom charges come from?
    → HOW do we find and fix them?

  SCENARIO D: Currency Conversion
    → Customer pays in EUR
    → Merchant account is in USD
    → Exchange rate at authorization: 1 EUR = 1.08 USD
    → Exchange rate at settlement (3 days later): 1 EUR = 1.09 USD
    → Who absorbs the $1.47 difference?

HOW do you build a payment system that:
  → Never charges a customer twice
  → Never loses a charge that succeeded
  → Handles partial failures gracefully
  → Maintains a perfect audit trail
  → Reconciles with external payment gateways
  → Processes 50K transactions per minute at peak

🔍 HOW TO APPROACH IT

DECOMPOSE INTO SUB-SYSTEMS:

Sub-System 1: PAYMENT API
  → Accept payment requests from order service
  → Validate input, check idempotency
  → Route to correct payment processor

Sub-System 2: PAYMENT STATE MACHINE
  → Track payment through its lifecycle
  → States: CREATED → AUTHORIZED → CAPTURED → SETTLED → (or REFUNDED)
  → Enforce valid transitions, prevent invalid ones

Sub-System 3: SAGA ORCHESTRATOR
  → Coordinate multi-step transactions
  → Execute steps in sequence
  → If any step fails → run compensations in reverse
  → Ensure consistency without distributed transactions

Sub-System 4: LEDGER SERVICE
  → Double-entry bookkeeping
  → Every debit has a matching credit
  → Immutable append-only log
  → Source of truth for all money movement

Sub-System 5: GATEWAY ADAPTER
  → Abstract away specific payment providers
  → Handle idempotency keys per gateway
  → Manage timeouts and retries safely
  → Support multiple providers (Stripe, Adyen, PayPal)

Sub-System 6: RECONCILIATION ENGINE
  → Compare our ledger with gateway records
  → Detect discrepancies
  → Generate exception reports
  → From Project 36: scheduled daily reconciliation job

🛠️ HOW CAN WE SOLVE IT

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PART A: PAYMENT STATE MACHINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Every payment follows a strict state machine:

  ┌─────────┐   authorize   ┌────────────┐   capture   ┌──────────┐
  │ CREATED │──────────────→│ AUTHORIZED │────────────→│ CAPTURED │
  └────┬────┘               └─────┬──────┘             └────┬─────┘
       │                          │                         │
       │ fail                     │ void                    │ settle
       ▼                          ▼                         ▼
  ┌─────────┐              ┌───────────┐            ┌──────────┐
  │ FAILED  │              │  VOIDED   │            │ SETTLED  │
  └─────────┘              └───────────┘            └────┬─────┘
                                                         │
                                                         │ refund
                                                         ▼
                                                   ┌──────────┐
                                                   │ REFUNDED │
                                                   └──────────┘

  VALID TRANSITIONS ONLY:
    CREATED → AUTHORIZED (card is valid, funds reserved)
    CREATED → FAILED (invalid card, insufficient funds)
    AUTHORIZED → CAPTURED (funds actually charged)
    AUTHORIZED → VOIDED (release hold, no charge)
    CAPTURED → SETTLED (funds transferred to merchant)
    SETTLED → REFUNDED (money returned to customer)
    CAPTURED → REFUNDED (refund before settlement)

  INVALID TRANSITIONS (rejected by state machine):
    FAILED → AUTHORIZED ❌ (can't authorize a failed payment)
    CAPTURED → AUTHORIZED ❌ (can't go backward)
    REFUNDED → CAPTURED ❌ (can't re-capture refunded money)

  WHY A STATE MACHINE:
    → Prevents impossible states (double capture, refund before charge)
    → Every transition logged → perfect audit trail
    → Concurrent operations checked against valid transitions
    → External system failures don't corrupt internal state

PART B: SAGA PATTERN FOR MULTI-STEP TRANSACTIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ORDER SAGA — the complete checkout flow:

  Step 1: RESERVE INVENTORY
    Action:       Reserve 3 items in warehouse
    Compensation: Unreserve 3 items

  Step 2: AUTHORIZE PAYMENT
    Action:       Auth €147.50 on customer's Visa
    Compensation: Void authorization

  Step 3: CAPTURE PAYMENT
    Action:       Charge €147.50 to customer's Visa
    Compensation: Refund €147.50

  Step 4: CONFIRM ORDER
    Action:       Create order record, assign order number
    Compensation: Cancel order

  Step 5: TRIGGER SHIPPING
    Action:       Send shipping request to fulfillment
    Compensation: Cancel shipment

  EXECUTION FLOW:

  SUCCESS PATH:
    Reserve ✅ → Authorize ✅ → Capture ✅ → Confirm ✅ → Ship ✅
    DONE ✅

  FAILURE AT STEP 3 (Capture fails):
    Reserve ✅ → Authorize ✅ → Capture ❌
    COMPENSATE:
      → Void authorization (undo step 2)
      → Unreserve inventory (undo step 1)
    Customer sees: "Payment failed, please try again"

  FAILURE AT STEP 5 (Shipping fails):
    Reserve ✅ → Authorize ✅ → Capture ✅ → Confirm ✅ → Ship ❌
    COMPENSATE:
      → Cancel order (undo step 4)
      → Refund payment (undo step 3)
      → Void auth (already captured, so skip)
      → Unreserve inventory (undo step 1)
    Customer sees: "Order could not be fulfilled, refund issued"

  TWO SAGA APPROACHES:

  Choreography (event-driven):
    → Each service emits events → next service reacts
    → No central coordinator
    → Hard to track overall progress
    → Hard to implement compensation

  Orchestration (our approach):
    → Central Saga Orchestrator drives the flow
    → Knows all steps and their compensations
    → On failure: runs compensations in reverse
    → Tracks saga state persistently
    → From Project 29: event-driven + orchestration hybrid

PART C: DOUBLE-ENTRY LEDGER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  FUNDAMENTAL RULE: Every money movement creates TWO entries
    → One DEBIT (money leaving an account)
    → One CREDIT (money entering an account)
    → Sum of all debits ALWAYS equals sum of all credits

  Example: Customer pays €147.50

  ┌────────────────────────────────────────────────────────────┐
  │ LEDGER ENTRIES                                             │
  │                                                            │
  │ Entry 1: DEBIT  Customer Account    €147.50                │
  │ Entry 2: CREDIT Merchant Revenue    €142.50                │
  │ Entry 3: CREDIT Platform Fee        €5.00                  │
  │                                                            │
  │ Total Debits:  €147.50                                     │
  │ Total Credits: €147.50 (€142.50 + €5.00)                  │
  │ Balance: €0.00 ✅ (debits = credits)                       │
  └────────────────────────────────────────────────────────────┘

  Refund of same payment:

  ┌────────────────────────────────────────────────────────────┐
  │ Entry 4: CREDIT Customer Account    €147.50  (money back)  │
  │ Entry 5: DEBIT  Merchant Revenue    €142.50  (revenue back)│
  │ Entry 6: DEBIT  Platform Fee        €5.00    (fee back)    │
  │                                                            │
  │ Running Balance: €0.00 ✅                                   │
  └────────────────────────────────────────────────────────────┘

  WHY DOUBLE-ENTRY:
    → Self-balancing: if debits ≠ credits → something is WRONG
    → Immutable: entries are NEVER modified, only new entries added
    → Auditable: complete trail of every cent moved
    → Detectable: any discrepancy is immediately visible

PART D: IDEMPOTENCY AT THE GATEWAY LEVEL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  THE CRITICAL PROBLEM:
    We send: POST /charges {amount: 14750, idempotency_key: "pay_abc123"}
    Gateway timeout → we DON'T KNOW if it charged
    We retry with SAME idempotency_key
    Gateway sees: "I already processed pay_abc123 → here's the result"
    No double charge ✅

  HOW IT WORKS:

    Client → PayFlow → Gateway
    
    Request 1: {amount: 14750, key: "pay_abc123"}
      → Gateway: first time seeing "pay_abc123"
      → Process charge → store result keyed by "pay_abc123"
      → Return: {status: "success", charge_id: "ch_xyz"}
    
    Request 2 (retry): {amount: 14750, key: "pay_abc123"}
      → Gateway: already have result for "pay_abc123"
      → Return SAME result: {status: "success", charge_id: "ch_xyz"}
      → NO new charge created ✅

  IDEMPOTENCY KEY GENERATION:
    → Must be deterministic for the same logical operation
    → key = hash(order_id + payment_attempt + amount)
    → Same order, same attempt, same amount → same key
    → Different attempt → different key (allows genuine retries)

  OUR INTERNAL IDEMPOTENCY:
    → Every payment request has an idempotency_key
    → Before processing: check if key already processed
    → If yes → return cached result
    → If no → process and store result
    → From Project 36: idempotency tracking in executor

📌 WHAT COULD BE DONE

SAGA IMPLEMENTATION OPTIONS:

Option 1: Synchronous Saga (Sequential)
  → Execute steps one by one in a single request
  → Simple, easy to reason about
  → Slow: total latency = sum of all steps
  → If caller times out → saga state unclear

Option 2: Async Saga with Orchestrator (Our Approach)
  → Saga orchestrator persists state after each step
  → Each step is async → emit event on completion
  → Orchestrator advances saga based on events
  → Survives crashes: state is in database
  → Can be retried from last successful step

Option 3: Choreography-Based Saga
  → No central coordinator
  → Each service listens for events and acts
  → Harder to track, harder to compensate
  → Better for loosely coupled systems

WE IMPLEMENT: Option 2 (Orchestrated Saga)
  → Clear control flow
  → Easy compensation logic
  → Persistent state survives crashes
  → Full visibility into saga progress

PAYMENT GATEWAY OPTIONS:

Option 1: Single Gateway
  → All payments through Stripe
  → Simple, one integration
  → Single point of failure

Option 2: Multi-Gateway with Failover (Our Approach)
  → Primary: Stripe
  → Fallback: Adyen
  → If Stripe fails → route to Adyen
  → Circuit breaker (Project 21) per gateway
  → Different gateways for different regions/currencies

Option 3: Smart Routing
  → Route based on: card type, currency, amount, region
  → Optimize for: success rate, fees, speed
  → ML model selects best gateway per transaction
  → Production systems (like Shopify) use this

✅ WHAT SHOULD DO

1. ALWAYS USE IDEMPOTENCY KEYS FOR GATEWAY CALLS
   → Every charge, refund, void must have a unique idempotency key
   → Key = deterministic hash of (payment_id + operation + attempt)
   → Gateway deduplicates based on this key
   → Safe to retry indefinitely without double-charging

2. ALWAYS PERSIST SAGA STATE BEFORE EACH STEP
   → Before calling gateway: save "AUTHORIZING" state
   → After gateway response: save "AUTHORIZED" or "FAILED" state
   → If crash between: recovery knows we were in "AUTHORIZING"
   → Can query gateway with idempotency key to learn outcome

3. ALWAYS USE DOUBLE-ENTRY LEDGER
   → Every money movement = debit + credit
   → Ledger entries are IMMUTABLE (never update, only append)
   → Balance check: sum(debits) must equal sum(credits)
   → Run balance check daily as part of reconciliation

4. ALWAYS STORE AMOUNTS IN SMALLEST CURRENCY UNIT (CENTS)
   → Never use floating point for money → rounding errors
   → €147.50 stored as 14750 (integer cents)
   → $99.99 stored as 9999 (integer cents)
   → Currency code stored alongside: {amount: 14750, currency: "EUR"}

5. ALWAYS LOG GATEWAY RESPONSES COMPLETELY
   → Full request and response bodies
   → Timestamps, latency, HTTP status
   → Gateway transaction ID (for reconciliation)
   → This is your audit trail and debugging lifeline

6. ALWAYS SET TIMEOUTS ON GATEWAY CALLS
   → Authorization: 10 second timeout
   → Capture: 15 second timeout
   → On timeout: DO NOT assume success or failure
   → Query gateway for status before retrying
   → From Project 21: circuit breaker on gateway

7. ALWAYS SEPARATE AUTHORIZE FROM CAPTURE
   → Authorize: "Can this card pay €147.50?" (hold, don't charge)
   → Capture: "Actually charge the €147.50"
   → Two-step gives you flexibility:
     → Authorize at checkout → capture at shipment
     → If out of stock → void auth → customer never charged
     → Industry standard for e-commerce

❌ WHAT SHOULD NOT DO

1. NEVER USE FLOATING POINT FOR MONEY
   → 0.1 + 0.2 = 0.30000000000000004 in most languages
   → Over millions of transactions → significant discrepancy
   → ALWAYS use integer cents + currency code
   → 147.50 EUR → {amount_cents: 14750, currency: "EUR"}

2. NEVER RETRY A CHARGE WITHOUT IDEMPOTENCY KEY
   → Retry without key → gateway treats as NEW charge
   → Customer charged twice → refund nightmare → trust lost
   → EVERY gateway call MUST have an idempotency key

3. NEVER STORE RAW CARD NUMBERS
   → PCI-DSS compliance: never store full card numbers
   → Use tokenization: card → token via gateway
   → Store only: last 4 digits, expiry, card brand
   → Let the gateway handle card data

4. NEVER ASSUME GATEWAY TIMEOUT MEANS FAILURE
   → Timeout = "I don't know what happened"
   → Gateway might have charged successfully
   → MUST query gateway for status before deciding
   → Never auto-refund on timeout (might refund a successful charge
     that hasn't been recorded yet)

5. NEVER EXECUTE COMPENSATION WITHOUT CHECKING STATE
   → Before refunding: verify payment was actually captured
   → Before voiding auth: verify auth actually succeeded
   → State machine prevents: "refund a payment that was never charged"
   → Check current state before every compensation action

6. NEVER MODIFY LEDGER ENTRIES
   → Ledger is append-only, IMMUTABLE
   → Made a mistake? Add a CORRECTION entry, don't edit
   → This is fundamental to financial auditing
   → Every change is traceable

7. NEVER PROCESS PAYMENTS IN THE API REQUEST THREAD
   → Payment processing takes 2-10 seconds
   → Holding HTTP connection open → timeouts, retries, confusion
   → Accept request → return payment_id → process async
   → Client polls for status or receives webhook
   → From Project 11: decouple via message queue

8. NEVER IGNORE CURRENCY IN AMOUNT COMPARISONS
   → 14750 EUR ≠ 14750 USD
   → Always compare (amount, currency) as a pair
   → Never add amounts in different currencies

🚫 WHEN NOT TO USE

1. DO NOT USE SAGA PATTERN WHEN:
   → Single database transaction suffices
   → All operations are on the same service
   → Complexity of saga outweighs benefit
   → Use simple DB transaction with rollback instead

2. DO NOT USE TWO-STEP (AUTH + CAPTURE) WHEN:
   → Immediate delivery (digital goods)
   → Subscription recurring charges
   → Low-value transactions (< $1)
   → Direct capture is simpler and faster

3. DO NOT BUILD YOUR OWN PAYMENT GATEWAY WHEN:
   → Stripe, Adyen, Braintree exist
   → PCI compliance is enormously expensive
   → Build the orchestration layer, not the card processing

4. DO NOT USE SYNCHRONOUS SAGA FOR:
   → Transactions involving external APIs with high latency
   → Steps that might take minutes (manual review)
   → Use async saga with persistent state instead

    1. Unreserve inventory: inventory → unreserve(rsv_001)

  Payment state: FAILED

STEP 4: GATEWAY TIMEOUT HANDLING

  We call: gateway.capture(auth_xyz, 14750)
  → 10 seconds pass → TIMEOUT

  DO NOT:
    ❌ Assume failure and compensate (might have charged)
    ❌ Retry immediately without checking (might double-charge)

  DO:
    1. Mark payment as: CAPTURE_UNCERTAIN
    2. Wait 5 seconds
    3. Query gateway: GET /charges?idempotency_key=cap_pay_001_v1

       a. "charge found, status: succeeded" → mark as CAPTURED ✅
       b. "charge found, status: failed" → mark as FAILED, compensate
       c. "no charge found" → safe to retry with same idempotency key

       → Exponential backoff: 30s, 2m, 10m, 1h
       → After 4 retries → manual review queue

      → Not found in our records? ⚠️ Unknown charge (phantom)

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: DOUBLE CHARGE
  → Most feared failure in payment systems
  → Causes: retry without idempotency, race condition on capture
  → FIX: Idempotency keys on EVERY gateway call
         State machine prevents double-capture
         Distributed lock on payment_id during processing (Project 27)

PITFALL 2: ORPHANED AUTHORIZATION
  → Payment authorized but never captured (saga failed after auth)
  → Customer's card shows "pending charge" that never resolves
  → Authorization hold expires after 7-30 days (card issuer dependent)
  → FIX: Compensation voids the authorization immediately
         Daily cleanup job finds orphaned auths > 24 hours → void them
         TaskMaster (Project 36) schedules the cleanup

PITFALL 3: COMPENSATION FAILURE
  → Saga fails at step 3 → compensate step 2 (void auth)
  → But void ALSO fails (gateway down)
  → Now we have a charged customer and no compensation
  → FIX: Compensation has its own retry policy
         Persist compensation as a task in TaskMaster
         Exponential backoff: will eventually succeed
         If all retries fail → manual review queue
         Never give up on compensations

PITFALL 4: LEDGER IMBALANCE
  → Bug creates a debit without matching credit
  → Sum of debits ≠ sum of credits
  → Money appears to be "lost" or "created from nothing"
  → FIX: Every ledger write is a TRANSACTION (debit + credit atomic)
         Balance check runs hourly
         Alert immediately on any imbalance
         Immutable ledger → can trace exactly which entry caused it

PITFALL 5: CURRENCY CONVERSION ROUNDING
  → €147.50 at 1.08 = $159.30
  → But 14750 cents × 1.08 = 15930.0 cents ✅
  → What about 14751 × 1.08 = 15930.78 → round to 15931? or 15930?
  → Different rounding across systems → penny discrepancies
  → FIX: Define rounding rule (HALF_UP) and apply consistently
         Store both original and converted amounts
         Reconcile in original currency

PITFALL 6: RACE CONDITION ON REFUND
  → Customer clicks "Refund" twice quickly
  → Two refund requests arrive simultaneously
  → Both check: "is payment captured?" → YES
  → Both issue refund → customer gets double refund
  → FIX: Distributed lock on payment_id (Project 27)
         State machine: CAPTURED → REFUNDING → REFUNDED
         Only one transition to REFUNDING succeeds

PITFALL 7: SAGA STATE CORRUPTION
  → Saga orchestrator crashes between step execution and state save
  → On recovery: doesn't know if step 3 executed or not
  → FIX: Persist state BEFORE calling external service
         On recovery: check external service status via idempotency key
         Idempotency makes it safe to re-execute any step

PITFALL 8: SETTLEMENT TIMING
  → We mark payment as CAPTURED immediately
  → But bank settles funds 2-3 business days later
  → Merchant sees "revenue" they can't withdraw yet
  → FIX: Separate CAPTURED from SETTLED states
         Track settlement dates per payment
         Dashboard shows: captured vs settled vs available

@dataclass
class Money:
    """Represents a monetary amount — ALWAYS in smallest unit (cents)"""
    amount_cents: int
    currency: str

    def __str__(self):
        major = self.amount_cents // 100
        minor = self.amount_cents % 100
        symbols = {"USD": "$", "EUR": "€", "GBP": "£", "JPY": "¥"}
        sym = symbols.get(self.currency, self.currency + " ")
        if self.currency == "JPY":  # JPY has no minor unit
            return f"{sym}{self.amount_cents}"
        return f"{sym}{major}.{minor:02d}"

    def __eq__(self, other):
        return (self.amount_cents == other.amount_cents and
                self.currency == other.currency)

    def __add__(self, other):
        if self.currency != other.currency:
            raise ValueError(
                f"Cannot add {self.currency} and {other.currency}"
            )
        return Money(self.amount_cents + other.amount_cents, self.currency)

class PaymentStateMachine:
    """
    Enforces valid payment state transitions
    Every transition is logged for audit trail
    """

    def __init__(self):
        self.lock = threading.Lock()
        self.invalid_transitions = 0

    def transition(self, payment: Payment,
                   new_status: PaymentStatus,
                   reason: str = "") -> bool:
        """
        Attempt a state transition
        Returns True if valid, False if invalid
        """
        with self.lock:
            current = payment.status
            valid_next = VALID_TRANSITIONS.get(current, set())

            if new_status not in valid_next:
                self.invalid_transitions += 1
                payment.transitions.append({
                    "from": current.value,
                    "to": new_status.value,
                    "status": "REJECTED",
                    "reason": f"Invalid transition: "
                              f"{current.value} → {new_status.value}",
                    "timestamp": time.time()
                })
                return False

            # Valid transition — apply it
            old_status = payment.status
            payment.status = new_status

            # Update timestamps
            if new_status == PaymentStatus.AUTHORIZED:
                payment.authorized_at = time.time()
            elif new_status == PaymentStatus.CAPTURED:
                payment.captured_at = time.time()
            elif new_status == PaymentStatus.SETTLED:
                payment.settled_at = time.time()
            elif new_status == PaymentStatus.REFUNDED:
                payment.refunded_at = time.time()

            payment.transitions.append({
                "from": old_status.value,
                "to": new_status.value,
                "status": "APPLIED",
                "reason": reason,
                "timestamp": time.time()
            })

            return True

class Ledger:
    """
    Immutable double-entry bookkeeping ledger
    Every money movement = debit + credit
    Sum of debits MUST equal sum of credits
    """

    def __init__(self):
        self.entries: List[LedgerEntry] = []
        self.lock = threading.Lock()
        self.entry_counter = 0

    def record_transaction(self, payment_id: str,
                           debit_account: str,
                           credit_account: str,
                           amount: Money,
                           description: str) -> Tuple[str, str]:
        """
        Record a double-entry transaction
        Returns (debit_entry_id, credit_entry_id)
        """
        with self.lock:
            reference = f"txn_{uuid.uuid4().hex[:8]}"
            timestamp = time.time()

            self.entry_counter += 1
            debit_id = f"le_{self.entry_counter:06d}"
            debit_entry = LedgerEntry(
                entry_id=debit_id,
                payment_id=payment_id,
                account=debit_account,
                entry_type=LedgerEntryType.DEBIT,
                amount=amount,
                description=f"DEBIT: {description}",
                timestamp=timestamp,
                reference=reference
            )

            self.entry_counter += 1
            credit_id = f"le_{self.entry_counter:06d}"
            credit_entry = LedgerEntry(
                entry_id=credit_id,
                payment_id=payment_id,
                account=credit_account,
                entry_type=LedgerEntryType.CREDIT,
                amount=amount,
                description=f"CREDIT: {description}",
                timestamp=timestamp,
                reference=reference
            )

            self.entries.append(debit_entry)
            self.entries.append(credit_entry)

            return debit_id, credit_id

    def record_split_transaction(self, payment_id: str,
                                  debit_account: str,
                                  credits: List[Tuple[str, Money]],
                                  description: str) -> List[str]:
        """Record transaction with one debit split across multiple credits"""
        with self.lock:
            reference = f"txn_{uuid.uuid4().hex[:8]}"
            timestamp = time.time()
            entry_ids = []

            total_debit = sum(amt.amount_cents for _, amt in credits)

            self.entry_counter += 1
            debit_id = f"le_{self.entry_counter:06d}"
            currency = credits[0][1].currency
            self.entries.append(LedgerEntry(
                entry_id=debit_id,
                payment_id=payment_id,
                account=debit_account,
                entry_type=LedgerEntryType.DEBIT,
                amount=Money(total_debit, currency),
                description=f"DEBIT: {description}",
                timestamp=timestamp,
                reference=reference
            ))
            entry_ids.append(debit_id)

            for credit_account, credit_amount in credits:
                self.entry_counter += 1
                credit_id = f"le_{self.entry_counter:06d}"
                self.entries.append(LedgerEntry(
                    entry_id=credit_id,
                    payment_id=payment_id,
                    account=credit_account,
                    entry_type=LedgerEntryType.CREDIT,
                    amount=credit_amount,
                    description=f"CREDIT: {description}",
                    timestamp=timestamp,
                    reference=reference
                ))
                entry_ids.append(credit_id)

            return entry_ids

    def get_balance(self, account: str) -> int:
        """Get net balance for an account (credits - debits) in cents"""
        balance = 0
        for entry in self.entries:
            if entry.account == account:
                if entry.entry_type == LedgerEntryType.CREDIT:
                    balance += entry.amount.amount_cents
                else:
                    balance -= entry.amount.amount_cents
        return balance

    def check_balance(self) -> dict:
        """Verify debits equal credits (fundamental invariant)"""
        total_debits = sum(
            e.amount.amount_cents for e in self.entries
            if e.entry_type == LedgerEntryType.DEBIT
        )
        total_credits = sum(
            e.amount.amount_cents for e in self.entries
            if e.entry_type == LedgerEntryType.CREDIT
        )
        balanced = total_debits == total_credits
        return {
            "balanced": balanced,
            "total_debits_cents": total_debits,
            "total_credits_cents": total_credits,
            "discrepancy_cents": abs(total_debits - total_credits),
            "total_entries": len(self.entries)
        }

    def get_entries_for_payment(self, payment_id: str) -> List[LedgerEntry]:
        return [e for e in self.entries if e.payment_id == payment_id]

    def get_stats(self) -> dict:
        balance = self.check_balance()
        accounts = set(e.account for e in self.entries)
        return {
            **balance,
            "unique_accounts": len(accounts)
        }

class PaymentGateway:
    """
    Simulates an external payment gateway (Stripe, Adyen, etc.)
    Supports idempotency keys
    Has configurable failure rates for testing
    """

    def __init__(self, name: str, failure_rate: float = 0.05,
                 timeout_rate: float = 0.02,
                 latency_ms_range: Tuple[float, float] = (100, 500)):
        self.name = name
        self.failure_rate = failure_rate
        self.timeout_rate = timeout_rate
        self.latency_range = latency_ms_range
        # Idempotency store
        self.processed: Dict[str, GatewayResponse] = {}
        self.charges: Dict[str, dict] = {}  # charge_id → details
        self.lock = threading.Lock()
        # Metrics
        self.total_calls = 0
        self.successes = 0
        self.failures = 0
        self.timeouts = 0
        self.idempotent_hits = 0

    def authorize(self, amount_cents: int, currency: str,
                  token: str, idempotency_key: str) -> GatewayResponse:
        """Authorize a payment (check card, hold funds)"""
        return self._process_request(
            "authorize", amount_cents, currency, token, idempotency_key
        )

    def capture(self, auth_id: str, amount_cents: int,
                idempotency_key: str) -> GatewayResponse:
        """Capture a previously authorized payment"""
        return self._process_request(
            "capture", amount_cents, "", "", idempotency_key,
            reference_id=auth_id
        )

    def void(self, auth_id: str,
             idempotency_key: str) -> GatewayResponse:
        """Void an authorization (release hold)"""
        return self._process_request(
            "void", 0, "", "", idempotency_key,
            reference_id=auth_id
        )

    def refund(self, charge_id: str, amount_cents: int,
               idempotency_key: str) -> GatewayResponse:
        """Refund a captured payment"""
        return self._process_request(
            "refund", amount_cents, "", "", idempotency_key,
            reference_id=charge_id
        )

    def query_status(self, idempotency_key: str) -> Optional[GatewayResponse]:
        """Query the result of a previous request by idempotency key"""
        with self.lock:
            return self.processed.get(idempotency_key)

    def _process_request(self, operation: str, amount_cents: int,
                         currency: str, token: str,
                         idempotency_key: str,
                         reference_id: str = "") -> GatewayResponse:
        """Core processing logic with idempotency"""
        self.total_calls += 1

        # Simulate network latency
        latency = random.uniform(*self.latency_range)
        time.sleep(latency / 1000)

        with self.lock:
            # Check idempotency
            if idempotency_key in self.processed:
                self.idempotent_hits += 1
                return self.processed[idempotency_key]

            # Simulate timeout
            if random.random() < self.timeout_rate:
                self.timeouts += 1
                # Note: the operation MAY have succeeded at gateway
                # We simulate 50% success on timeout
                if random.random() < 0.5:
                    # Actually succeeded but we don't know
                    response = self._create_success_response(
                        operation, amount_cents
                    )
                    self.processed[idempotency_key] = response
                raise TimeoutError(
                    f"Gateway {self.name} timeout for {operation}"
                )

            # Simulate failure
            if random.random() < self.failure_rate:
                self.failures += 1
                response = GatewayResponse(
                    success=False,
                    error_code="card_declined",
                    error_message="Insufficient funds",
                    raw_response={"status": "failed"}
                )
                self.processed[idempotency_key] = response
                return response

            # Success
            response = self._create_success_response(
                operation, amount_cents
            )
            self.processed[idempotency_key] = response
            self.successes += 1
            return response

    def _create_success_response(self, operation: str,
                                  amount_cents: int) -> GatewayResponse:
        txn_id = f"{self.name}_{operation}_{uuid.uuid4().hex[:8]}"
        auth_code = f"auth_{uuid.uuid4().hex[:6]}"

        if operation in ("capture", "authorize"):
            self.charges[txn_id] = {
                "transaction_id": txn_id,
                "amount_cents": amount_cents,
                "operation": operation,
                "timestamp": time.time(),
                "status": "succeeded"
            }

        return GatewayResponse(
            success=True,
            transaction_id=txn_id,
            auth_code=auth_code,
            raw_response={
                "status": "succeeded",
                "transaction_id": txn_id,
                "amount": amount_cents
            }
        )

    def get_all_charges(self) -> Dict[str, dict]:
        """For reconciliation — return all charges"""
        return dict(self.charges)

    def get_stats(self) -> dict:
        total = self.total_calls
        return {
            "name": self.name,
            "total_calls": total,
            "successes": self.successes,
            "failures": self.failures,
            "timeouts": self.timeouts,
            "idempotent_hits": self.idempotent_hits,
            "success_rate_pct": round(
                self.successes / max(total, 1) * 100, 1
            ),
            "total_charges": len(self.charges)
        }


class GatewayAdapter:
    """
    Abstracts multiple gateways with failover
    Circuit breaker pattern (from Project 21)
    """

    def __init__(self):
        self.gateways: Dict[str, PaymentGateway] = {}
        self.primary_gateway: Optional[str] = None
        # Simple circuit breaker state per gateway
        self.failures_count: Dict[str, int] = defaultdict(int)
        self.circuit_open: Dict[str, bool] = defaultdict(lambda: False)
        self.FAILURE_THRESHOLD = 5

    def add_gateway(self, gateway: PaymentGateway,
                    is_primary: bool = False):
        self.gateways[gateway.name] = gateway
        if is_primary:
            self.primary_gateway = gateway.name

    def get_gateway(self) -> PaymentGateway:
        """Get best available gateway (with failover)"""
        # Try primary first
        if (self.primary_gateway and
                not self.circuit_open[self.primary_gateway]):
            return self.gateways[self.primary_gateway]

        # Fallback to any open gateway
        for name, gw in self.gateways.items():
            if not self.circuit_open[name]:
                return gw

        # All circuits open — try primary anyway (half-open)
        if self.primary_gateway:
            self.circuit_open[self.primary_gateway] = False
            return self.gateways[self.primary_gateway]

        raise Exception("No payment gateways available")

    def record_success(self, gateway_name: str):
        self.failures_count[gateway_name] = 0
        self.circuit_open[gateway_name] = False

    def record_failure(self, gateway_name: str):
        self.failures_count[gateway_name] += 1
        if self.failures_count[gateway_name] >= self.FAILURE_THRESHOLD:
            self.circuit_open[gateway_name] = True

class IdempotencyService:
    """
    Tracks processed idempotency keys to prevent duplicate processing
    From Project 29/36: idempotency guard
    """

    def __init__(self):
        self.processed: Dict[str, dict] = {}
        self.lock = threading.Lock()
        self.hits = 0
        self.misses = 0

    def check(self, key: str) -> Optional[dict]:
        """Check if this key was already processed"""
        with self.lock:
            if key in self.processed:
                self.hits += 1
                return self.processed[key]
            self.misses += 1
            return None

    def store(self, key: str, result: dict):
        """Store result for an idempotency key"""
        with self.lock:
            self.processed[key] = {
                **result,
                "cached_at": time.time()
            }

    def get_stats(self) -> dict:
        total = self.hits + self.misses
        return {
            "total_checks": total,
            "hits": self.hits,
            "misses": self.misses,
            "hit_rate_pct": round(
                self.hits / max(total, 1) * 100, 1
            ),
            "keys_stored": len(self.processed)
        }

class InventoryService:
    """Simulates inventory reservation for saga integration"""

    def __init__(self):
        self.stock: Dict[str, int] = {}
        self.reservations: Dict[str, dict] = {}
        self.lock = threading.Lock()

    def set_stock(self, sku: str, quantity: int):
        self.stock[sku] = quantity

    def reserve(self, items: List[dict]) -> dict:
        """Reserve inventory for an order"""
        with self.lock:
            # Check availability
            for item in items:
                sku = item["sku"]
                qty = item["qty"]
                available = self.stock.get(sku, 0)
                if available < qty:
                    return {
                        "success": False,
                        "error": f"Insufficient stock for {sku}: "
                                 f"need {qty}, have {available}"
                    }

            # Reserve
            reservation_id = f"rsv_{uuid.uuid4().hex[:8]}"
            for item in items:
                self.stock[item["sku"]] -= item["qty"]

            self.reservations[reservation_id] = {
                "items": items,
                "created_at": time.time(),
                "status": "active"
            }

            return {
                "success": True,
                "reservation_id": reservation_id
            }

    def unreserve(self, reservation_id: str) -> dict:
        """Release a reservation (compensation)"""
        with self.lock:
            reservation = self.reservations.get(reservation_id)
            if not reservation or reservation["status"] != "active":
                return {"success": False, "error": "Reservation not found"}

            for item in reservation["items"]:
                self.stock[item["sku"]] = (
                    self.stock.get(item["sku"], 0) + item["qty"]
                )

            reservation["status"] = "released"
            return {"success": True, "reservation_id": reservation_id}

class SagaOrchestrator:
    """
    Coordinates multi-step transactions with compensation
    Persists state after each step for crash recovery
    Runs compensations in reverse order on failure
    """

    def __init__(self):
        self.sagas: Dict[str, Saga] = {}
        self.lock = threading.Lock()
        self.total_sagas = 0
        self.completed_sagas = 0
        self.compensated_sagas = 0
        self.failed_sagas = 0

    def create_saga(self, payment_id: str,
                    steps: List[SagaStep]) -> Saga:
        """Create a new saga instance"""
        saga_id = f"saga_{uuid.uuid4().hex[:8]}"
        saga = Saga(
            saga_id=saga_id,
            payment_id=payment_id,
            status=SagaStatus.STARTED,
            steps=steps,
            current_step=0,
            created_at=time.time()
        )
        saga.history.append({
            "event": "saga_created",
            "steps": [s.step_name for s in steps],
            "timestamp": time.time()
        })

        with self.lock:
            self.sagas[saga_id] = saga
            self.total_sagas += 1

        return saga

    def execute_saga(self, saga_id: str) -> dict:
        """
        Execute all saga steps sequentially
        On failure: compensate completed steps in reverse
        """
        saga = self.sagas.get(saga_id)
        if not saga:
            return {"error": "Saga not found"}

        saga.status = SagaStatus.IN_PROGRESS

        # Execute steps forward
        for i, step in enumerate(saga.steps):
            saga.current_step = i
            step.status = "executing"

            saga.history.append({
                "event": "step_executing",
                "step": step.step_name,
                "step_index": i,
                "timestamp": time.time()
            })

            try:
                # Execute the action
                result = step.action()
                step.result = result

                # Check if action reported failure
                if isinstance(result, dict) and not result.get("success", True):
                    raise Exception(
                        result.get("error", "Step returned failure")
                    )

                step.status = "completed"
                saga.history.append({
                    "event": "step_completed",
                    "step": step.step_name,
                    "result": str(result)[:100],
                    "timestamp": time.time()
                })

            except Exception as e:
                step.status = "failed"
                step.error = str(e)

                saga.history.append({
                    "event": "step_failed",
                    "step": step.step_name,
                    "error": str(e),
                    "timestamp": time.time()
                })

                # Run compensations in reverse
                compensation_result = self._compensate(saga, i)
                return {
                    "saga_id": saga_id,
                    "status": "compensated",
                    "failed_step": step.step_name,
                    "error": str(e),
                    "compensations": compensation_result
                }

        # All steps completed successfully
        saga.status = SagaStatus.COMPLETED
        saga.completed_at = time.time()
        self.completed_sagas += 1

        saga.history.append({
            "event": "saga_completed",
            "duration_ms": round(
                (saga.completed_at - saga.created_at) * 1000, 1
            ),
            "timestamp": time.time()
        })

        return {
            "saga_id": saga_id,
            "status": "completed",
            "steps_completed": len(saga.steps),
            "duration_ms": round(
                (saga.completed_at - saga.created_at) * 1000, 1
            )
        }

    def _compensate(self, saga: Saga, failed_step_index: int) -> List[dict]:
        """Run compensations in reverse order for completed steps"""
        saga.status = SagaStatus.COMPENSATING
        compensations = []

        saga.history.append({
            "event": "compensation_started",
            "from_step": failed_step_index,
            "timestamp": time.time()
        })

        # Compensate from the step BEFORE the failed one, going backward
        for i in range(failed_step_index - 1, -1, -1):
            step = saga.steps[i]

            if step.status != "completed":
                continue  # Only compensate completed steps

            if step.compensation is None:
                compensations.append({
                    "step": step.step_name,
                    "status": "skipped",
                    "reason": "No compensation defined"
                })
                continue

            try:
                comp_result = step.compensation()
                step.status = "compensated"
                step.compensation_result = comp_result

                compensations.append({
                    "step": step.step_name,
                    "status": "compensated",
                    "result": str(comp_result)[:100]
                })

                saga.history.append({
                    "event": "step_compensated",
                    "step": step.step_name,
                    "timestamp": time.time()
                })

            except Exception as comp_error:
                compensations.append({
                    "step": step.step_name,
                    "status": "compensation_failed",
                    "error": str(comp_error)
                })

                saga.history.append({
                    "event": "compensation_failed",
                    "step": step.step_name,
                    "error": str(comp_error),
                    "timestamp": time.time()
                })

        saga.status = SagaStatus.COMPENSATED
        self.compensated_sagas += 1

        saga.history.append({
            "event": "compensation_completed",
            "compensations_run": len(compensations),
            "timestamp": time.time()
        })

        return compensations

    def get_saga_details(self, saga_id: str) -> dict:
        saga = self.sagas.get(saga_id)
        if not saga:
            return {"error": "Saga not found"}

        return {
            "saga_id": saga.saga_id,
            "payment_id": saga.payment_id,
            "status": saga.status.value,
            "current_step": saga.current_step,
            "steps": [
                {
                    "name": s.step_name,
                    "status": s.status,
                    "error": s.error
                }
                for s in saga.steps
            ],
            "history_count": len(saga.history),
            "history": saga.history[-5:]
        }

    def get_stats(self) -> dict:
        return {
            "total_sagas": self.total_sagas,
            "completed": self.completed_sagas,
            "compensated": self.compensated_sagas,
            "failed": self.failed_sagas,
            "active": sum(
                1 for s in self.sagas.values()
                if s.status == SagaStatus.IN_PROGRESS
            )
        }

class ReconciliationEngine:
    """
    Compares our payment records with gateway records
    Detects discrepancies for investigation
    Scheduled daily via TaskMaster (Project 36)
    """

    def __init__(self):
        self.reports: List[ReconciliationReport] = []

    def reconcile(self, our_payments: Dict[str, Payment],
                  gateway: PaymentGateway,
                  report_date: str = None) -> ReconciliationReport:
        """
        Compare our records with gateway records
        Returns detailed reconciliation report
        """
        if report_date is None:
            report_date = time.strftime("%Y-%m-%d")

        report = ReconciliationReport(report_date=report_date)

        # Get gateway charges
        gateway_charges = gateway.get_all_charges()

        # Build lookup maps
        our_by_gateway_id: Dict[str, Payment] = {}
        for payment in our_payments.values():
            if payment.gateway_charge_id:
                our_by_gateway_id[payment.gateway_charge_id] = payment
            if payment.gateway_auth_id:
                our_by_gateway_id[payment.gateway_auth_id] = payment

        # Count our captured/settled payments
        our_captured = {
            pid: p for pid, p in our_payments.items()
            if p.status in (
                PaymentStatus.CAPTURED, PaymentStatus.SETTLED,
                PaymentStatus.REFUNDED
            )
        }
        report.our_count = len(our_captured)
        report.our_total_cents = sum(
            p.amount.amount_cents for p in our_captured.values()
        )

        # Count gateway charges
        report.gateway_count = len(gateway_charges)
        report.gateway_total_cents = sum(
            c["amount_cents"] for c in gateway_charges.values()
        )

        # Match records
        matched_gateway_ids = set()

        for pid, payment in our_captured.items():
            gw_id = payment.gateway_charge_id or payment.gateway_auth_id
            if gw_id and gw_id in gateway_charges:
                gw_charge = gateway_charges[gw_id]
                matched_gateway_ids.add(gw_id)

                # Check amount match
                if gw_charge["amount_cents"] != payment.amount.amount_cents:
                    report.amount_mismatches += 1
                    report.exceptions.append({
                        "type": "amount_mismatch",
                        "payment_id": pid,
                        "gateway_id": gw_id,
                        "our_amount": payment.amount.amount_cents,
                        "gateway_amount": gw_charge["amount_cents"]
                    })
                else:
                    report.matched += 1
            else:
                report.missing_at_gateway += 1
                report.exceptions.append({
                    "type": "missing_at_gateway",
                    "payment_id": pid,
                    "gateway_id": gw_id
                })

        # Find unknown gateway charges
        for gw_id in gateway_charges:
            if gw_id not in matched_gateway_ids:
                report.unknown_at_gateway += 1
                report.exceptions.append({
                    "type": "unknown_at_gateway",
                    "gateway_id": gw_id,
                    "amount_cents": gateway_charges[gw_id]["amount_cents"]
                })

        report.discrepancy_cents = abs(
            report.our_total_cents - report.gateway_total_cents
        )

        self.reports.append(report)
        return report

class PaymentPlatform:
    """Main payment platform — ties all components together"""

    PLATFORM_FEE_BPS = 340  # 3.4% in basis points (hundredths of percent)

    def __init__(self):
        # Core components
        self.state_machine = PaymentStateMachine()
        self.ledger = Ledger()
        self.saga_orchestrator = SagaOrchestrator()
        self.idempotency = IdempotencyService()
        self.reconciliation = ReconciliationEngine()
        self.inventory = InventoryService()

        # Gateways
        self.gateway_adapter = GatewayAdapter()
        self.primary_gateway = PaymentGateway(
            "stripe", failure_rate=0.05, timeout_rate=0.02
        )
        self.fallback_gateway = PaymentGateway(
            "adyen", failure_rate=0.08, timeout_rate=0.03
        )
        self.gateway_adapter.add_gateway(self.primary_gateway, is_primary=True)
        self.gateway_adapter.add_gateway(self.fallback_gateway)

        # Payment storage
        self.payments: Dict[str, Payment] = {}
        self.lock = threading.Lock()

        # Metrics
        self.total_payments = 0
        self.successful_payments = 0
        self.failed_payments = 0
        self.total_refunds = 0

    # ============================================================
    # PAYMENT PROCESSING
    # ============================================================

    def process_payment(self, order_id: str, customer_id: str,
                        amount_cents: int, currency: str,
                        payment_token: str,
                        items: List[dict] = None,
                        idempotency_key: str = None) -> dict:
        """
        Full payment processing with saga
        1. Check idempotency
        2. Create payment record
        3. Execute saga (reserve → authorize → capture → confirm)
        4. Return result
        """
        # Generate idempotency key if not provided
        if not idempotency_key:
            idempotency_key = f"pay_{order_id}_v1"

        # Step 1: Idempotency check
        cached = self.idempotency.check(idempotency_key)
        if cached:
            return cached

        # Step 2: Create payment
        payment_id = f"pay_{uuid.uuid4().hex[:10]}"
        amount = Money(amount_cents, currency)

        payment = Payment(
            payment_id=payment_id,
            order_id=order_id,
            customer_id=customer_id,
            amount=amount,
            payment_method_token=payment_token,
            status=PaymentStatus.CREATED,
            idempotency_key=idempotency_key,
            created_at=time.time()
        )

        with self.lock:
            self.payments[payment_id] = payment
            self.total_payments += 1

        # Step 3: Build and execute saga
        items = items or []

        # Shared state for saga steps (closures capture this)
        saga_context = {
            "reservation_id": None,
            "auth_id": None,
            "charge_id": None,
            "gateway_name": None
        }

        def step_reserve():
            if not items:
                return {"success": True, "reservation_id": "none"}
            result = self.inventory.reserve(items)
            if result["success"]:
                saga_context["reservation_id"] = result["reservation_id"]
            return result

        def compensate_reserve():
            rid = saga_context.get("reservation_id")
            if rid and rid != "none":
                return self.inventory.unreserve(rid)
            return {"success": True, "skipped": True}

        def step_authorize():
            self.state_machine.transition(
                payment, PaymentStatus.AUTHORIZING, "Saga authorize step"
            )
            gateway = self.gateway_adapter.get_gateway()
            saga_context["gateway_name"] = gateway.name
            payment.gateway_name = gateway.name

            idem_key = f"auth_{payment_id}_v1"
            try:
                response = gateway.authorize(
                    amount_cents, currency, payment_token, idem_key
                )
                if response.success:
                    self.gateway_adapter.record_success(gateway.name)
                    saga_context["auth_id"] = response.transaction_id
                    payment.gateway_auth_id = response.transaction_id
                    self.state_machine.transition(
                        payment, PaymentStatus.AUTHORIZED, "Auth succeeded"
                    )
                    return {"success": True, "auth_id": response.transaction_id}
                else:
                    self.gateway_adapter.record_failure(gateway.name)
                    self.state_machine.transition(
                        payment, PaymentStatus.FAILED,
                        response.error_message or "Auth failed"
                    )
                    payment.error_message = response.error_message
                    return {
                        "success": False,
                        "error": response.error_message
                    }
            except TimeoutError as e:
                self.gateway_adapter.record_failure(gateway.name)
                self.state_machine.transition(
                    payment, PaymentStatus.UNCERTAIN, "Auth timeout"
                )
                # Query gateway to check if it actually succeeded
                status = gateway.query_status(idem_key)
                if status and status.success:
                    saga_context["auth_id"] = status.transaction_id
                    payment.gateway_auth_id = status.transaction_id
                    self.state_machine.transition(
                        payment, PaymentStatus.AUTHORIZED,
                        "Auth confirmed after timeout"
                    )
                    return {
                        "success": True,
                        "auth_id": status.transaction_id
                    }
                return {"success": False, "error": str(e)}

        def compensate_authorize():
            auth_id = saga_context.get("auth_id")
            if auth_id:
                gateway = self.gateway_adapter.gateways.get(
                    saga_context.get("gateway_name", "")
                )
                if gateway:
                    idem_key = f"void_{payment_id}_v1"
                    response = gateway.void(auth_id, idem_key)
                    if response.success:
                        self.state_machine.transition(
                            payment, PaymentStatus.VOIDED,
                            "Voided during compensation"
                        )
                    return {"success": response.success}
            return {"success": True, "skipped": True}

        def step_capture():
            auth_id = saga_context.get("auth_id")
            if not auth_id:
                return {"success": False, "error": "No auth_id"}

            self.state_machine.transition(
                payment, PaymentStatus.CAPTURING, "Saga capture step"
            )

            gateway = self.gateway_adapter.gateways.get(
                saga_context.get("gateway_name", "")
            )
            if not gateway:
                gateway = self.gateway_adapter.get_gateway()

            idem_key = f"cap_{payment_id}_v1"
            try:
                response = gateway.capture(auth_id, amount_cents, idem_key)
                if response.success:
                    self.gateway_adapter.record_success(gateway.name)
                    saga_context["charge_id"] = response.transaction_id
                    payment.gateway_charge_id = response.transaction_id
                    self.state_machine.transition(
                        payment, PaymentStatus.CAPTURED, "Capture succeeded"
                    )

                    # Record in ledger
                    fee_cents = int(amount_cents * self.PLATFORM_FEE_BPS / 10000)
                    merchant_cents = amount_cents - fee_cents

                    self.ledger.record_split_transaction(
                        payment_id=payment_id,
                        debit_account=f"customer:{customer_id}",
                        credits=[
                            (
                                f"merchant:{order_id}",
                                Money(merchant_cents, currency)
                            ),
                            (
                                "platform:fees",
                                Money(fee_cents, currency)
                            )
                        ],
                        description=(
                            f"Payment {payment_id} for order {order_id}"
                        )
                    )

                    return {
                        "success": True,
                        "charge_id": response.transaction_id
                    }
                else:
                    self.gateway_adapter.record_failure(gateway.name)
                    self.state_machine.transition(
                        payment, PaymentStatus.FAILED,
                        response.error_message or "Capture failed"
                    )
                    return {
                        "success": False,
                        "error": response.error_message
                    }
            except TimeoutError as e:
                self.gateway_adapter.record_failure(gateway.name)
                self.state_machine.transition(
                    payment, PaymentStatus.UNCERTAIN, "Capture timeout"
                )
                # Query status
                status = gateway.query_status(idem_key)
                if status and status.success:
                    saga_context["charge_id"] = status.transaction_id
                    payment.gateway_charge_id = status.transaction_id
                    self.state_machine.transition(
                        payment, PaymentStatus.CAPTURED,
                        "Capture confirmed after timeout"
                    )
                    return {
                        "success": True,
                        "charge_id": status.transaction_id
                    }
                return {"success": False, "error": str(e)}

        def compensate_capture():
            charge_id = saga_context.get("charge_id")
            if charge_id:
                gateway = self.gateway_adapter.gateways.get(
                    saga_context.get("gateway_name", "")
                )
                if gateway:
                    idem_key = f"refund_{payment_id}_compensation_v1"
                    response = gateway.refund(
                        charge_id, amount_cents, idem_key
                    )
                    if response.success:
                        # Reverse ledger entries
                        fee_cents = int(
                            amount_cents * self.PLATFORM_FEE_BPS / 10000
                        )
                        merchant_cents = amount_cents - fee_cents

                        self.ledger.record_split_transaction(
                            payment_id=payment_id,
                            debit_account="platform:refunds",
                            credits=[
                                (
                                    f"customer:{customer_id}",
                                    Money(amount_cents, currency)
                                )
                            ],
                            description=(
                                f"Compensation refund for {payment_id}"
                            )
                        )
                    return {"success": response.success}
            return {"success": True, "skipped": True}

        def step_confirm():
            return {
                "success": True,
                "order_id": order_id,
                "confirmed": True
            }

        def compensate_confirm():
            return {
                "success": True,
                "order_id": order_id,
                "cancelled": True
            }

        # Build saga
        saga_steps = [
            SagaStep("reserve_inventory", step_reserve, compensate_reserve),
            SagaStep("authorize_payment", step_authorize, compensate_authorize),
            SagaStep("capture_payment", step_capture, compensate_capture),
            SagaStep("confirm_order", step_confirm, compensate_confirm),
        ]

        saga = self.saga_orchestrator.create_saga(payment_id, saga_steps)
        payment.saga_id = saga.saga_id

        # Execute saga
        saga_result = self.saga_orchestrator.execute_saga(saga.saga_id)

        # Build response
        if saga_result.get("status") == "completed":
            self.successful_payments += 1
            result = {
                "payment_id": payment_id,
                "order_id": order_id,
                "status": "success",
                "amount": str(amount),
                "gateway": payment.gateway_name,
                "charge_id": payment.gateway_charge_id,
                "saga_id": saga.saga_id,
                "saga_duration_ms": saga_result.get("duration_ms", 0)
            }
        else:
            self.failed_payments += 1
            result = {
                "payment_id": payment_id,
                "order_id": order_id,
                "status": "failed",
                "amount": str(amount),
                "error": saga_result.get("error", "Unknown error"),
                "failed_step": saga_result.get("failed_step"),
                "compensations": saga_result.get("compensations", []),
                "saga_id": saga.saga_id
            }

        # Store in idempotency cache
        self.idempotency.store(idempotency_key, result)

        return result

    # ============================================================
    # REFUND PROCESSING
    # ============================================================

    def process_refund(self, payment_id: str,
                       reason: str = "customer_request") -> dict:
        """Process a refund for a captured/settled payment"""
        payment = self.payments.get(payment_id)
        if not payment:
            return {"error": "Payment not found"}

        # State machine check
        if not self.state_machine.transition(
            payment, PaymentStatus.REFUNDING, f"Refund: {reason}"
        ):
            return {
                "error": f"Cannot refund payment in state "
                         f"{payment.status.value}"
            }

        # Call gateway
        gateway = self.gateway_adapter.gateways.get(payment.gateway_name)
        if not gateway:
            gateway = self.gateway_adapter.get_gateway()

        charge_id = payment.gateway_charge_id
        idem_key = f"refund_{payment_id}_v1"

        try:
            response = gateway.refund(
                charge_id, payment.amount.amount_cents, idem_key
            )

            if response.success:
                self.state_machine.transition(
                    payment, PaymentStatus.REFUNDED, "Refund succeeded"
                )

                # Reverse ledger entries
                self.ledger.record_transaction(
                    payment_id=payment_id,
                    debit_account="platform:refunds",
                    credit_account=f"customer:{payment.customer_id}",
                    amount=payment.amount,
                    description=f"Refund for {payment_id}: {reason}"
                )

                self.total_refunds += 1

                return {
                    "payment_id": payment_id,
                    "status": "refunded",
                    "amount": str(payment.amount),
                    "refund_id": response.transaction_id,
                    "reason": reason
                }
            else:
                self.state_machine.transition(
                    payment, PaymentStatus.FAILED,
                    response.error_message or "Refund failed"
                )
                return {
                    "payment_id": payment_id,
                    "status": "refund_failed",
                    "error": response.error_message
                }

        except TimeoutError:
            return {
                "payment_id": payment_id,
                "status": "refund_uncertain",
                "error": "Gateway timeout during refund"
            }

    # ============================================================
    # RECONCILIATION
    # ============================================================

    def run_reconciliation(self) -> ReconciliationReport:
        """Run reconciliation against primary gateway"""
        return self.reconciliation.reconcile(
            self.payments, self.primary_gateway
        )

    def inspect_payment(self, payment_id: str) -> dict:
        payment = self.payments.get(payment_id)
        if not payment:
            return {"error": "Payment not found"}

        ledger_entries = self.ledger.get_entries_for_payment(payment_id)

        return {
            "payment_id": payment.payment_id,
            "order_id": payment.order_id,
            "customer_id": payment.customer_id,
            "amount": str(payment.amount),
            "status": payment.status.value,
            "gateway": payment.gateway_name,
            "auth_id": payment.gateway_auth_id,
            "charge_id": payment.gateway_charge_id,
            "created_at": payment.created_at,
            "saga_id": payment.saga_id,
            "retry_count": payment.retry_count,
            "error": payment.error_message,
            "transitions": payment.transitions[-5:],
            "ledger_entries": [
                {
                    "id": e.entry_id,
                    "type": e.entry_type.value,
                    "account": e.account,
                    "amount": str(e.amount),
                    "description": e.description
                }
                for e in ledger_entries
            ]
        }

    def get_platform_metrics(self) -> dict:
        return {

            }

        lines = []
        lines.append("")
        lines.append("=" * 65)

        if result.get("status") == "success":
            lines.append(f"  ✅ PAYMENT SUCCESSFUL")
            lines.append("=" * 65)
            lines.append(f"  Payment ID:  {result['payment_id']}")

            lines.append(f"  Amount:      {result['amount']}")

            lines.append(f"  Charge ID:   {result['charge_id']}")
            lines.append(f"  Saga:        {result['saga_id']} "

        elif result.get("status") == "failed":
            lines.append(f"  ❌ PAYMENT FAILED")
            lines.append("=" * 65)
            lines.append(f"  Payment ID:  {result['payment_id']}")
            lines.append(f"  Order ID:    {result['order_id']}")
            lines.append(f"  Amount:      {result['amount']}")

            if comps:
                lines.append(f"  Compensations:")
                for c in comps:
                    status_icon = "✅" if c["status"] == "compensated" else "❌"
                    lines.append(f"    {status_icon} {c['step']}: {c['status']}")
        else:
            lines.append(f"  ⚠️ {result}")

        lines.append("=" * 65)
        return "\n".join(lines)

        lines = []

        lines.append(f"  Our total:       "
                    f"{report.our_total_cents / 100:.2f}")
        lines.append(f"  Gateway total:   "
                    f"{report.gateway_total_cents / 100:.2f}")
        lines.append(f"  Discrepancy:     "
                    f"{report.discrepancy_cents / 100:.2f}")

        status = "✅ CLEAN" if report.discrepancy_cents == 0 else "⚠️ EXCEPTIONS"
        lines.append(f"\n  Status: {status}")

            for exc in report.exceptions[:5]:
                lines.append(f"    ⚠️ {exc['type']}: {exc}")

        lines.append("=" * 65)

        lines.append("")
        lines.append("=" * 65)

                    if isinstance(v, dict):
                        lines.append(f"  │  {k}:")
                        for k2, v2 in v.items():

                    else:
                        lines.append(f"  │  {k}: {v}")
                lines.append(f"  └{'─' * 50}")

        lines.append("")
        return "\n".join(lines)

✅ Payment state machine prevents impossible operations
   → CREATED → AUTHORIZED → CAPTURED → SETTLED
   → Can't capture without authorization
   → Can't refund without capture
   → Every transition logged for audit

✅ Saga pattern handles multi-step transactions without distributed TX
   → Each step has a compensating action (undo)
   → On failure: run compensations in reverse order
   → State persisted after each step → crash-recoverable

✅ Double-entry ledger ensures financial integrity
   → Every debit has a matching credit
   → Sum of debits ALWAYS equals sum of credits
   → Immutable append-only → perfect audit trail
   → Any imbalance immediately detectable

✅ Idempotency prevents double-charging
   → API level: same request key → return cached result
   → Gateway level: same idempotency key → gateway deduplicates
   → Safe to retry ANY operation without side effects

✅ Gateway timeout ≠ failure
   → Timeout means "unknown" → must query before retrying
   → Idempotency key makes retry safe regardless
   → Never assume failure on timeout

✅ Money must be integer cents with currency
   → Floating point causes rounding errors over millions of transactions
   → Different currencies must never be mixed
   → Currency is part of the type, not just metadata

THE CONNECTION:

PayFlow's saga calls InventoryService.reserve() as step 1
→ But our inventory service was SIMPLIFIED
→ Real inventory under high concurrency has HARD problems:

SCENARIO: Flash Sale — 1000 units, 50,000 buyers click "Buy" simultaneously
  → 50,000 concurrent reserve requests for the same SKU
  → Naive approach: read stock (1000) → check (1000 ≥ 1) → decrement
  → Race condition: 5000 requests all read "1000" before any decrement
  → Result: 5000 reservations for 1000 units → overselling → disaster

WHAT PROJECT 38 SOLVES:
  → Reservation pattern with TTL (reserve expires if not confirmed)
  → Optimistic locking (version-based concurrency control)
  → Atomic stock decrement (compare-and-swap or DB-level locking)
  → Order state machine (placed → confirmed → shipped → delivered)
  → Consistency between payment state and order state
  → From Project 27: distributed locks for inventory operations
  → From Project 37: saga integration for the full checkout flow
