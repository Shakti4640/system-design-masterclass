  Server 1 Memory:                Server 3 Memory:
  ┌──────────────────────┐       ┌──────────────────────┐

  │   "sess_abc": Alice, │       │                      │

  Each server has its OWN memory
  Sessions stored in ONE server's memory
  Other servers know NOTHING about those sessions
  Load balancer sends users to RANDOM servers
  → User authenticated on Server 1 is a STRANGER on Server 3

THE HOTEL KEY CARD ANALOGY:

  STATEFUL (Bad — like a hotel with no central system):
  → You check in at Front Desk A → clerk A writes your name in their notebook
  → You come back later → Front Desk B is open → clerk B has no record of you
  → "Sorry, who are you? I don't have you in MY notebook"
  → Each clerk has their OWN notebook → no sharing

  STATELESS (Good — like a modern hotel):
  → You check in → hotel gives you a KEY CARD
  → Key card contains: your room number, dates, access level
  → Any door reader can verify the key card
  → No clerk needs to remember you
  → You can use ANY entrance, ANY elevator, ANY door
  → Lost key card? Get a new one. Hotel doesn't need to "remember"

  JWT Token = The Key Card
  → Contains all needed information INSIDE the token itself
  → Any server can verify it without checking any database
  → Truly stateless — servers remember NOTHING

🛠️ HOW CAN WE SOLVE IT

THREE STRATEGIES FOR SESSION MANAGEMENT:

═══════════════════════════════════════════════════════
STRATEGY 1: STICKY SESSIONS (Don't Do This)
═══════════════════════════════════════════════════════

  Load Balancer always sends same user to same server
  (IP Hash from Project 6)

  ┌────────┐     ┌────┐     ┌──────────┐
  │ Alice  │────→│ LB │────→│ Server 1 │  ALWAYS Server 1
  └────────┘     │    │     └──────────┘
  ┌────────┐     │    │     ┌──────────┐
  │  Bob   │────→│    │────→│ Server 2 │  ALWAYS Server 2
  └────────┘     └────┘     └──────────┘

  ❌ Server 1 dies → Alice loses session → must re-login
  ❌ Server 1 overloaded → can't move Alice to Server 2
  ❌ Defeats the purpose of load balancing
  ❌ Scaling up = complex rebalancing
  → AVOID unless absolutely necessary (legacy systems)

═══════════════════════════════════════════════════════
STRATEGY 2: EXTERNAL SESSION STORE (Server-Side)
═══════════════════════════════════════════════════════

  Sessions stored in shared external store (Redis/DB)
  ALL servers read/write to the SAME store

  ┌────────┐     ┌────┐     ┌──────────┐
  │ Alice  │────→│ LB │────→│ Server 1 │──┐
  └────────┘     │    │     └──────────┘  │
                 │    │     ┌──────────┐  │   ┌───────────┐
                 │    │────→│ Server 2 │──┼──→│   REDIS   │
                 │    │     └──────────┘  │   │ (shared)  │
                 │    │     ┌──────────┐  │   │           │
                 │    │────→│ Server 3 │──┘   │ sess_abc: │
                 └────┘     └──────────┘      │  {Alice}  │
                                              └───────────┘

  Flow:
  1. Alice logs in → Server 1 creates session in Redis
  2. Server 1 returns cookie: session_id=sess_abc
  3. Alice's next request → goes to Server 3
  4. Server 3 reads cookie → looks up sess_abc in Redis
  5. Redis returns Alice's data → Server 3 knows who Alice is
  6. Works regardless of which server handles the request!

  ✅ Any server can handle any request
  ✅ Session survives server crashes
  ✅ Can store complex session data
  ❌ Extra network hop to Redis on every request
  ❌ Redis becomes a dependency (must be highly available)
  ❌ Session data lives on server side (storage grows with users)

═══════════════════════════════════════════════════════
STRATEGY 3: JWT TOKENS (Client-Side — Stateless)
═══════════════════════════════════════════════════════

  All session data encoded IN the token itself
  Server doesn't store ANYTHING — truly stateless

  ┌────────┐     ┌────┐     ┌──────────┐
  │ Alice  │────→│ LB │────→│ Server 1 │  (no session store)
  │        │     │    │     └──────────┘
  │ Token: │     │    │     ┌──────────┐
  │ eyJhb..│     │    │────→│ Server 2 │  (no session store)
  │        │     │    │     └──────────┘
  └────────┘     └────┘     ┌──────────┐
                            │ Server 3 │  (no session store)
                            └──────────┘

  JWT Structure:
  ┌──────────────────────────────────────────────────────┐
  │  HEADER.PAYLOAD.SIGNATURE                             │
  │                                                       │
  │  Header:  {"alg": "HS256", "typ": "JWT"}             │
  │  Payload: {"user_id": 42, "name": "Alice",           │
  │            "role": "customer", "exp": 1704153600}     │
  │  Signature: HMAC-SHA256(header + payload, SECRET_KEY) │
  │                                                       │
  │  Encoded: eyJhbGciOiJIUzI1NiJ9.eyJ1c2VyX2lkIjo0Mn0  │
  │           .SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw │
  └──────────────────────────────────────────────────────┘

  Flow:
  1. Alice logs in → Server 1 creates JWT with Alice's data
  2. JWT is SIGNED with server's SECRET_KEY
  3. JWT returned to client (in response body or cookie)
  4. Alice's next request → includes JWT in Authorization header
  5. ANY server receives it → verifies signature → reads payload
  6. No database lookup, no Redis lookup, no external call
  7. Token is self-contained — server is truly STATELESS

  ✅ Truly stateless — servers store NOTHING
  ✅ No external dependency (no Redis needed for sessions)
  ✅ Scales infinitely — add servers with zero session concerns
  ✅ Works across different services (microservices)
  ❌ Can't revoke individual tokens easily (until expiry)
  ❌ Token size grows with data (sent on every request)
  ❌ Sensitive data in token (must not store passwords, etc.)

═══════════════════════════════════════════════════════
STRATEGY 4: HYBRID (JWT + Refresh Token) — BEST PRACTICE
═══════════════════════════════════════════════════════

  → Short-lived JWT (15 min) for authentication
  → Long-lived refresh token (7 days) stored in DB/Redis
  → JWT expires → use refresh token to get new JWT
  → Revocation: delete refresh token from DB

  ┌─────────────────────────────────────────────────────┐
  │  Login:                                              │
  │    → Verify credentials                              │
  │    → Create JWT (expires in 15 min)                  │
  │    → Create refresh_token (stored in DB, 7 days)     │
  │    → Return both to client                           │
  │                                                      │
  │  Normal Request:                                     │
  │    → Client sends JWT in header                      │
  │    → Server verifies JWT signature + expiry           │
  │    → No DB lookup needed (stateless!)                │
  │                                                      │
  │  JWT Expired:                                        │
  │    → Client sends refresh_token                      │
  │    → Server checks refresh_token in DB               │
  │    → If valid → issue new JWT                        │
  │    → If invalid → user must re-login                 │
  │                                                      │
  │  Logout / Revoke:                                    │
  │    → Delete refresh_token from DB                    │
  │    → Current JWT works until it expires (15 min max) │
  │    → Acceptable tradeoff for most applications       │
  └─────────────────────────────────────────────────────┘

📌 WHAT COULD BE DONE

Option A: Sticky Sessions Only
  → No external state needed
  → Simple but fragile
  → When: Legacy systems that can't be refactored

Option B: External Session Store Only (Redis)
  → Server-side sessions in shared cache
  → Full control over session lifecycle
  → When: Complex session data, need instant revocation

Option C: JWT Only
  → Completely stateless
  → No revocation until expiry
  → When: Short-lived tokens, microservices, APIs

Option D: JWT + Refresh Token (Hybrid)
  → Best of both worlds
  → Stateless for 99% of requests
  → DB check only for refresh (rare)
  → When: Production systems, mobile apps
  → THIS IS WHAT WE BUILD

Option E: OAuth2 / OpenID Connect
  → Delegated authentication (Google, GitHub login)
  → Industry standard for third-party auth
  → When: "Login with Google", enterprise SSO
  → Built on top of JWT concepts

✅ WHAT SHOULD DO

1. ALWAYS hash passwords before storing
   → NEVER store plain text passwords
   → Use bcrypt, scrypt, or argon2 (slow by design)
   ❌ password = "secret123" → stored as "secret123"
   ❌ password =[REDACTED:PASSWORD]6("secret123") → rainbow table vulnerable
   ✅ password = bcrypt("secret123", salt) → secure

2. ALWAYS use HTTPS for authentication
   → JWT in HTTP = anyone on network can steal it
   → HTTPS encrypts everything in transit
   → Set Secure flag on cookies (only sent over HTTPS)

3. ALWAYS set token expiration
   → JWT without expiration = permanent access if stolen
   → Access token: 15-60 minutes
   → Refresh token: 7-30 days
   → Shorter = more secure, more re-authentication

4. ALWAYS validate tokens on EVERY request
   → Don't skip validation for "internal" endpoints
   → Don't trust token just because it looks right
   → Verify: signature, expiration, issuer, audience

5. ALWAYS use strong secret keys
   → JWT signing key must be long and random
   ❌ SECRET_KEY = "password"
   ❌ SECRET_KEY = "my-secret"
   ✅ SECRET_KEY = "[REDACTED:AWS_SECRET_KEY]..." (256-bit random)
   → Key compromise = anyone can forge tokens

6. ALWAYS separate authentication from authorization
   → Authentication: WHO are you? (login, token verification)
   → Authorization: WHAT can you do? (role checks, permissions)
   → User might be authenticated but NOT authorized for an action

7. ALWAYS handle token refresh gracefully
   → Don't force re-login for expired access tokens
   → Auto-refresh using refresh token
   → Only force re-login when refresh token expires

❌ WHAT SHOULD NOT DO

1. NEVER store sensitive data in JWT payload
   ❌ {"password": "[REDACTED:PASSWORD]", "ssn": "[REDACTED:SSN]"}
   → JWT payload is BASE64 encoded, NOT encrypted
   → Anyone can decode and read the payload
   → Only store: user_id, role, name, expiration
   → For sensitive data: encrypt the JWT (JWE) or use server-side sessions

2. NEVER use JWT without expiration
   ❌ {"user_id": 42, "role": "admin"}  ← no "exp" field
   → Token valid FOREVER if stolen
   → Always include "exp" (expiration timestamp)

3. NEVER store JWT in localStorage
   ❌ localStorage.setItem("token", jwt)
   → Vulnerable to XSS attacks (any script can read it)
   ✅ HttpOnly cookie (JavaScript cannot access it)
   ✅ Or: memory only (lost on page refresh — use refresh token)

4. NEVER implement your own crypto
   ❌ Writing custom JWT signing algorithm
   ❌ Writing custom password hashing
   → Use well-tested libraries (PyJWT, bcrypt, passlib)
   → Crypto bugs = catastrophic security failures

5. NEVER use "none" algorithm in JWT
   ❌ {"alg": "none"} → no signature verification
   → Attacker changes payload → server accepts it
   → Always enforce specific algorithm (HS256, RS256)
   → Reject tokens with "alg": "none"

6. NEVER log tokens or passwords
   ❌ print(f"User logged in with token: {jwt_token}")
   ❌ logger.info(f"Password attempt: {password}")
   → Logs are often stored, shared, leaked
   → Log: user_id, action, timestamp — NOT credentials

7. NEVER share the JWT secret key across different services
   ❌ Same SECRET_KEY for auth service, order service, payment service
   → One compromised service = all services compromised
   → Use asymmetric keys (RS256): private key signs, public key verifies
   → Or: separate secrets per service

🚫 WHEN NOT TO USE

1. JWT is NOT suitable when:
   → Need instant revocation (user banned → token still works until expiry)
     → Workaround: short expiry + token blacklist (adds state)
   → Session data is very large (token size grows)
     → JWT in every request header = bandwidth cost
   → Need server-controlled session (shopping cart stored server-side)

2. Server-side sessions are NOT suitable when:
   → Need to scale to millions of concurrent users
     → Session store becomes bottleneck
   → Microservices need to verify identity independently
     → Each service needs access to session store
   → Mobile/SPA clients that don't support cookies well

3. Sticky sessions are NOT suitable when:
   → You need true high availability (server death = session loss)
   → You want even load distribution
   → You plan to scale horizontally (new servers get no sessions)

PART 1: HOW JWT WORKS — BYTE BY BYTE
═══════════════════════════════════════════════════════

  JWT has THREE parts separated by dots:
  
  HEADER.PAYLOAD.SIGNATURE

  Step 1: CREATE HEADER
    Raw: {"alg": "HS256", "typ": "JWT"}
    Base64URL encode: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9

  Step 2: CREATE PAYLOAD (Claims)
    Raw: {
      "user_id": 42,
      "name": "Alice",
      "role": "customer",
      "iat": 1704067200,     ← issued at (Unix timestamp)
      "exp": 1704068100,     ← expires at (15 min later)
      "iss": "restaurant-api" ← issuer
    }
    Base64URL encode: eyJ1c2VyX2lkIjo0MiwibmFtZSI6IkFsaWNlIn0

  Step 3: CREATE SIGNATURE
    input = base64(header) + "." + base64(payload)
    signature = HMAC-SHA256(input, SECRET_KEY)
    Base64URL encode signature

  Step 4: COMBINE
    JWT = header + "." + payload + "." + signature
    
    eyJhbGciOiJIUzI1NiJ9.eyJ1c2VyX2lkIjo0Mn0.SflKxwRJSMeKKF2QT

  VERIFICATION PROCESS:
  
  Server receives JWT:
  1. Split by "." → header, payload, signature
  2. Decode header → check algorithm is allowed
  3. Recompute: expected_sig = HMAC-SHA256(header.payload, SECRET_KEY)
  4. Compare expected_sig with received signature
  5. If match → token is authentic (not tampered)
  6. Decode payload → check "exp" > current time
  7. If not expired → extract user data from payload
  8. DONE — no database query needed!

  WHY THIS IS SECURE:
  → Attacker modifies payload: {"role": "admin"}
  → Signature no longer matches modified payload
  → Server rejects the token
  → Attacker can't compute new signature without SECRET_KEY

═══════════════════════════════════════════════════════
PART 2: HOW PASSWORD HASHING WORKS
═══════════════════════════════════════════════════════

  WRONG WAY (plain text):
    DB stores: password = [REDACTED:PASSWORD]3"
    DB leaked → attacker has all passwords instantly

  WRONG WAY (simple hash):
    DB stores: password =[REDACTED:PASSWORD]6("secret123") = "a1b2c3..."
    Attacker: precomputes sha256 for millions of common passwords
    → "Rainbow table" → finds match instantly

  RIGHT WAY (bcrypt with salt):
    salt = random_bytes(16)  → unique per user
    hash = bcrypt("secret123", salt, cost=12)
    DB stores: "$2b$12$salt...hash..."

    Verification:
    1. User submits: "secret123"
    2. Server reads stored hash (includes salt)
    3. Recompute: bcrypt("secret123", stored_salt, cost=12)
    4. Compare with stored hash
    5. Match → password correct

    WHY bcrypt is secure:
    → Salt: same password → different hash per user
    → Cost factor: deliberately SLOW (100ms per hash)
    → Attacker must spend 100ms per guess × millions of guesses
    → Brute force becomes impractical

PART 3: HOW COOKIES WORK (Transport Mechanism)
═══════════════════════════════════════════════════════

  SETTING A COOKIE:
  Server response:
    HTTP/1.1 200 OK
    Set-Cookie: session_id=abc123; HttpOnly; Secure; 
                SameSite=Strict; Path=/; Max-Age=3600

  Flags explained:
  ┌──────────────┬──────────────────────────────────────────┐
  │ HttpOnly     │ JavaScript cannot read this cookie       │
  │              │ Prevents XSS from stealing sessions      │
  ├──────────────┼──────────────────────────────────────────┤
  │ Secure       │ Only sent over HTTPS connections         │
  │              │ Prevents man-in-the-middle theft         │
  ├──────────────┼──────────────────────────────────────────┤
  │ SameSite     │ Strict: only sent for same-site requests │
  │              │ Prevents CSRF attacks                    │
  ├──────────────┼──────────────────────────────────────────┤
  │ Path=/       │ Sent for all paths on this domain        │
  ├──────────────┼──────────────────────────────────────────┤
  │ Max-Age=3600 │ Expires in 1 hour                        │
  │              │ Omit = session cookie (deleted on close) │
  └──────────────┴──────────────────────────────────────────┘

  SENDING A COOKIE (automatic):
  Browser request:
    GET /api/v1/orders HTTP/1.1
    Cookie: session_id=abc123    ← browser sends automatically

  Server reads Cookie header → extracts session_id → looks up session

PART 4: HOW AUTHORIZATION HEADER WORKS (JWT Transport)
═══════════════════════════════════════════════════════

  Client request:
    GET /api/v1/orders HTTP/1.1
    Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.eyJ1c2VyX2...

  Format: "Bearer " + JWT_TOKEN

  Server:
  1. Read Authorization header
  2. Strip "Bearer " prefix
  3. Verify JWT signature
  4. Extract user from payload
  5. Process request with user context

  WHY "Bearer":
  → HTTP standard (RFC 6750)
  → "Bearer" = "whoever bears (carries) this token has access"
  → Other schemes: "Basic" (base64 user:pass), "Digest", "API-Key"

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: Storing Sessions in Server Memory
  → Works for one server, breaks with multiple
  → Already explained in problem statement
  → FIX: External store (Redis) or JWT

PITFALL 2: JWT Token Too Large
  → Storing entire user profile in JWT payload
  → Token = 2KB+ → sent on EVERY request
  → Mobile networks: adds significant overhead
  → FIX: Store only essentials: user_id, role, exp
  → Fetch full profile from DB when needed

PITFALL 3: No Token Expiration
  → Token stolen → attacker has permanent access
  → FIX: Always set exp claim (15-60 minutes)
  → Use refresh token for longer sessions

PITFALL 4: Secret Key in Source Code
  ❌ SECRET_KEY = "my-super-secret" in config.py
  → Code pushed to GitHub → key exposed → tokens forgeable
  → FIX: Environment variables, secret managers (AWS Secrets Manager)

PITFALL 5: Not Validating Token on Every Request
  → "Token was valid at login, so it's always valid"
  → Token could be expired, revoked, or tampered
  → FIX: Validate signature + expiration on EVERY request

PITFALL 6: Refresh Token Reuse
  → Refresh token used once → should be rotated
  → If attacker steals refresh token → can keep refreshing
  → FIX: Rotate refresh token on each use
  → Detect reuse: if old refresh token used → revoke ALL tokens

PITFALL 7: CORS and Cookie Issues
  → API on api.restaurant.com, frontend on www.restaurant.com
  → Cookies not sent cross-domain by default
  → FIX: Configure CORS + SameSite=None + Secure
  → Or: Use Authorization header instead of cookies

PITFALL 8: Timing Attacks on Token Comparison
  → String comparison "==" returns false at first mismatch
  → Attacker can measure response time to guess token byte-by-byte
  → FIX: Use constant-time comparison (hmac.compare_digest)

class PasswordHasher:
    """
    Secure password hashing using PBKDF2 with SHA-256.
    In production: use bcrypt or argon2 library.
    PBKDF2 is built-in and demonstrates the same concepts.
    """
    
    ITERATIONS = 100_000  # Deliberately slow
    
    @staticmethod
    def hash_password(password):
        """Hash a password with random salt."""
        salt = os.urandom(32)
        key = hashlib.pbkdf2_hmac(
            'sha256',
            password.encode('utf-8'),
            salt,
            PasswordHasher.ITERATIONS
        )
        # Store salt + hash together
        return salt.hex() + ':' + key.hex()
    
    @staticmethod
    def verify_password(password, stored_hash):
        """Verify password against stored hash."""
        try:
            salt_hex, key_hex = stored_hash.split(':')
            salt = bytes.fromhex(salt_hex)
            stored_key = bytes.fromhex(key_hex)
            
            new_key = hashlib.pbkdf2_hmac(
                'sha256',
                password.encode('utf-8'),
                salt,
                PasswordHasher.ITERATIONS
            )
            
            # Constant-time comparison (prevents timing attacks)
            return hmac.compare_digest(new_key, stored_key)
        except Exception:
            return False

class JWTEngine:
    """
    JSON Web Token implementation from scratch.
    Supports HS256 (HMAC-SHA256) signing.
    """
    
    def __init__(self, secret_key):
        if len(secret_key) < 32:
            raise ValueError("Secret key must be at least 32 characters")
        self.secret_key = secret_key.encode('utf-8')
    
    @staticmethod
    def _base64url_encode(data):
        """Base64URL encoding (URL-safe, no padding)."""
        if isinstance(data, str):
            data = data.encode('utf-8')
        return base64.urlsafe_b64encode(data).rstrip(b'=').decode('utf-8')
    
    @staticmethod
    def _base64url_decode(data):
        """Base64URL decoding (add padding back)."""
        padding = 4 - len(data) % 4
        if padding != 4:
            data += '=' * padding
        return base64.urlsafe_b64decode(data.encode('utf-8'))
    
    def create_token(self, payload, expires_in_seconds=900):
        """
        Create a signed JWT token.
        
        payload: dict with user data (user_id, role, etc.)
        expires_in_seconds: token lifetime (default 15 minutes)
        """
        # Header
        header = {"alg": "HS256", "typ": "JWT"}
        
        # Add standard claims to payload
        now = time.time()
        full_payload = dict(payload)
        full_payload["iat"] = int(now)              # Issued At
        full_payload["exp"] = int(now + expires_in_seconds)  # Expiration
        full_payload["iss"] = "restaurant-api"       # Issuer
        
        # Encode
        header_encoded = self._base64url_encode(json.dumps(header))
        payload_encoded = self._base64url_encode(json.dumps(full_payload))
        
        # Sign
        message = f"{header_encoded}.{payload_encoded}".encode('utf-8')
        signature = hmac.new(
            self.secret_key, message, hashlib.sha256
        ).digest()
        signature_encoded = self._base64url_encode(signature)
        
        # Combine
        token = f"{header_encoded}.{payload_encoded}.{signature_encoded}"
        
        return {
            "token": token,
            "expires_at": full_payload["exp"],
            "expires_in": expires_in_seconds
        }
    
    def verify_token(self, token):
        """
        Verify and decode a JWT token.
        
        Returns: (payload_dict, error_string)
        On success: (payload, None)
        On failure: (None, error_message)
        """
        try:
            parts = token.split('.')
            if len(parts) != 3:
                return None, "Invalid token format"
            
            header_encoded, payload_encoded, signature_encoded = parts
            
            # Verify header
            header_json = self._base64url_decode(header_encoded)
            header = json.loads(header_json)
            
            if header.get("alg") != "HS256":
                return None, f"Unsupported algorithm: {header.get('alg')}"
            
            if header.get("alg") == "none":
                return None, "Algorithm 'none' is not allowed"
            
            # Verify signature
            message = f"{header_encoded}.{payload_encoded}".encode('utf-8')
            expected_signature = hmac.new(
                self.secret_key, message, hashlib.sha256
            ).digest()
            
            received_signature = self._base64url_decode(signature_encoded)
            
            if not hmac.compare_digest(expected_signature, received_signature):
                return None, "Invalid signature"
            
            # Decode payload
            payload_json = self._base64url_decode(payload_encoded)
            payload = json.loads(payload_json)
            
            # Check expiration
            if "exp" in payload:
                if time.time() > payload["exp"]:
                    return None, "Token expired"
            
            # Check issuer
            if payload.get("iss") != "restaurant-api":
                return None, "Invalid issuer"
            
            return payload, None
        
        except json.JSONDecodeError:
            return None, "Invalid token encoding"
        except Exception as e:
            return None, f"Token verification failed: {str(e)}"
    
    def decode_without_verify(self, token):
        """Decode payload WITHOUT verification (for debugging only)."""
        try:
            parts = token.split('.')
            if len(parts) != 3:
                return None
            payload_json = self._base64url_decode(parts[1])
            return json.loads(payload_json)
        except Exception:
            return None

class UserDatabase:
    """
    User storage with password hashing.
    Uses SQLite (from Project 3 concepts).
    """
    
    def __init__(self, db_path=":memory:"):
        self.conn = sqlite3.connect(db_path, check_same_thread=False)
        self.conn.row_factory = sqlite3.Row
        self.lock = threading.Lock()
        self._create_tables()
        self._seed_users()
    
    def _create_tables(self):
        with self.lock:
            self.conn.executescript("""
                CREATE TABLE IF NOT EXISTS users (
                    id INTEGER PRIMARY KEY AUTOINCREMENT,
                    email TEXT UNIQUE NOT NULL,
                    password_hash TEXT NOT NULL,
                    name TEXT NOT NULL,
                    role TEXT NOT NULL DEFAULT 'customer',
                    created_at TEXT DEFAULT (datetime('now')),
                    last_login TEXT
                );
                
                CREATE TABLE IF NOT EXISTS refresh_tokens (
                    id INTEGER PRIMARY KEY AUTOINCREMENT,
                    token TEXT UNIQUE NOT NULL,
                    user_id INTEGER NOT NULL,
                    expires_at REAL NOT NULL,
                    revoked INTEGER DEFAULT 0,
                    created_at TEXT DEFAULT (datetime('now')),
                    FOREIGN KEY (user_id) REFERENCES users(id)
                );
                
                CREATE INDEX IF NOT EXISTS idx_refresh_token
                    ON refresh_tokens(token);
                CREATE INDEX IF NOT EXISTS idx_refresh_user
                    ON refresh_tokens(user_id);
            """)
    
    def _seed_users(self):
        seed_data = [
            ("alice@restaurant.com", "password123", "Alice Johnson", "customer"),
            ("bob@restaurant.com", "password456", "Bob Smith", "customer"),
            ("carol@restaurant.com", "admin789", "Carol White", "admin"),
            ("dave@restaurant.com", "manager321", "Dave Brown", "manager"),
        ]
        
        for email, password, name, role in seed_data:
            self.create_user(email, password, name, role)
    
    def create_user(self, email, password, name, role="customer"):
        """Register a new user."""
        password_hash = PasswordHasher.hash_password(password)
        
        with self.lock:
            try:
                cursor = self.conn.execute(
                    """INSERT INTO users (email, password_hash, name, role)
                       VALUES (?, ?, ?, ?)""",
                    (email.lower(), password_hash, name, role)
                )
                self.conn.commit()
                return {
                    "id": cursor.lastrowid,
                    "email": email.lower(),
                    "name": name,
                    "role": role
                }
            except sqlite3.IntegrityError:
                return None  # Email already exists
    
    def authenticate(self, email, password):
        """Verify email/password combination."""
        with self.lock:
            cursor = self.conn.execute(
                "SELECT * FROM users WHERE email = ?",
                (email.lower(),)
            )
            user = cursor.fetchone()
        
        if not user:
            # Simulate hash time to prevent timing-based user enumeration
            PasswordHasher.hash_password("dummy")
            return None
        
        if PasswordHasher.verify_password(password, user["password_hash"]):
            # Update last login
            with self.lock:
                self.conn.execute(
                    "UPDATE users SET last_login = datetime('now') WHERE id = ?",
                    (user["id"],)
                )
                self.conn.commit()
            
            return {
                "id": user["id"],
                "email": user["email"],
                "name": user["name"],
                "role": user["role"]
            }
        
        return None
    
    def get_user(self, user_id):
        with self.lock:
            cursor = self.conn.execute(
                "SELECT id, email, name, role, created_at, last_login "
                "FROM users WHERE id = ?",
                (user_id,)
            )
            row = cursor.fetchone()
            return dict(row) if row else None
    
    def list_users(self):
        with self.lock:
            cursor = self.conn.execute(
                "SELECT id, email, name, role, created_at, last_login "
                "FROM users ORDER BY id"
            )
            return [dict(row) for row in cursor.fetchall()]
    
    # ---- Refresh Token Management ----
    
    def store_refresh_token(self, user_id, token, expires_in=604800):
        """Store refresh token (default: 7 days)."""
        expires_at = time.time() + expires_in
        with self.lock:
            self.conn.execute(
                """INSERT INTO refresh_tokens (token, user_id, expires_at)
                   VALUES (?, ?, ?)""",
                (token, user_id, expires_at)
            )
            self.conn.commit()
    
    def verify_refresh_token(self, token):
        """Verify and return user_id for valid refresh token."""
        with self.lock:
            cursor = self.conn.execute(
                """SELECT user_id, expires_at, revoked 
                   FROM refresh_tokens WHERE token = ?""",
                (token,)
            )
            row = cursor.fetchone()
        
        if not row:
            return None, "Refresh token not found"
        
        if row["revoked"]:
            return None, "Refresh token has been revoked"
        
        if time.time() > row["expires_at"]:
            return None, "Refresh token expired"
        
        return row["user_id"], None
    
    def revoke_refresh_token(self, token):
        """Revoke a specific refresh token."""
        with self.lock:
            self.conn.execute(
                "UPDATE refresh_tokens SET revoked = 1 WHERE token = ?",
                (token,)
            )
            self.conn.commit()
    
    def revoke_all_user_tokens(self, user_id):
        """Revoke ALL refresh tokens for a user (password change, etc.)."""
        with self.lock:
            cursor = self.conn.execute(
                "UPDATE refresh_tokens SET revoked = 1 WHERE user_id = ?",
                (user_id,)
            )
            self.conn.commit()
            return cursor.rowcount

class AuthService:
    """
    Centralized authentication and authorization service.
    Handles: registration, login, token management, permissions.
    """
    
    def __init__(self, db, jwt_engine):
        self.db = db
        self.jwt = jwt_engine
        self.access_token_ttl = 900       # 15 minutes
        self.refresh_token_ttl = 604800   # 7 days
        
        # Audit log
        self.audit_log = []
        self.lock = threading.Lock()
    
    def register(self, email, password, name):
        """Register a new user."""
        # Validate input
        errors = []
        if not email or '@' not in email:
            errors.append({"field": "email", "issue": "invalid email"})
        if not password or len(password) < 8:
            errors.append({
                "field": "password",
                "issue": "must be at least 8 characters"
            })
        if not name or len(name) < 2:
            errors.append({
                "field": "name",
                "issue": "must be at least 2 characters"
            })
        
        if errors:
            return {"success": False, "errors": errors}
        
        user = self.db.create_user(email, password, name)
        if not user:
            return {
                "success": False,
                "errors": [{"field": "email", "issue": "already registered"}]
            }
        
        self._audit("REGISTER", user["id"], f"New user: {email}")
        
        return {"success": True, "user": user}
    
    def login(self, email, password, client_ip="unknown"):
        """Authenticate and issue tokens."""
        user = self.db.authenticate(email, password)
        
        if not user:
            self._audit("LOGIN_FAILED", None, f"Failed login: {email} from {client_ip}")
            return {
                "success": False,
                "error": "Invalid email or password"
            }
        
        # Create access token (JWT)
        access = self.jwt.create_token(
            payload={
                "user_id": user["id"],
                "email": user["email"],
                "name": user["name"],
                "role": user["role"]
            },
            expires_in_seconds=self.access_token_ttl
        )
        
        # Create refresh token (opaque, stored in DB)
        refresh_token = secrets.token_urlsafe(48)
        self.db.store_refresh_token(
            user["id"], refresh_token, self.refresh_token_ttl
        )
        
        self._audit("LOGIN", user["id"], f"Login from {client_ip}")
        
        return {
            "success": True,
            "access_token": access["token"],
            "refresh_token": refresh_token,
            "token_type": "Bearer",
            "expires_in": access["expires_in"],
            "user": {
                "id": user["id"],
                "name": user["name"],
                "email": user["email"],
                "role": user["role"]
            }
        }
    
    def verify_access(self, token):
        """Verify access token and return user context."""
        payload, error = self.jwt.verify_token(token)
        
        if error:
            return None, error
        
        return {
            "user_id": payload["user_id"],
            "email": payload["email"],
            "name": payload["name"],
            "role": payload["role"],
            "token_exp": payload["exp"]
        }, None
    
    def refresh_access(self, refresh_token):
        """Use refresh token to get new access token."""
        user_id, error = self.db.verify_refresh_token(refresh_token)
        
        if error:
            return {"success": False, "error": error}
        
        # Revoke old refresh token (rotation)
        self.db.revoke_refresh_token(refresh_token)
        
        # Get user data
        user = self.db.get_user(user_id)
        if not user:
            return {"success": False, "error": "User not found"}
        
        # Issue new access token
        access = self.jwt.create_token(
            payload={
                "user_id": user["id"],
                "email": user["email"],
                "name": user["name"],
                "role": user["role"]
            },
            expires_in_seconds=self.access_token_ttl
        )
        
        # Issue new refresh token (rotation)
        new_refresh = secrets.token_urlsafe(48)
        self.db.store_refresh_token(user_id, new_refresh, self.refresh_token_ttl)
        
        self._audit("TOKEN_REFRESH", user_id, "Access token refreshed")
        
        return {
            "success": True,
            "access_token": access["token"],
            "refresh_token": new_refresh,
            "expires_in": access["expires_in"]
        }
    
    def logout(self, refresh_token, user_id=None):
        """Revoke refresh token (logout)."""
        self.db.revoke_refresh_token(refresh_token)
        
        if user_id:
            self._audit("LOGOUT", user_id, "User logged out")
        
        return {"success": True, "message": "Logged out successfully"}
    
    def logout_all_devices(self, user_id):
        """Revoke ALL refresh tokens for user."""
        count = self.db.revoke_all_user_tokens(user_id)
        self._audit("LOGOUT_ALL", user_id, f"Revoked {count} refresh tokens")
        
        return {
            "success": True,
            "message": f"Logged out from {count} sessions"
        }
    
    def check_permission(self, user_context, required_role):
        """Check if user has required role."""
        role_hierarchy = {
            "admin": 3,
            "manager": 2,
            "customer": 1
        }
        
        user_level = role_hierarchy.get(user_context["role"], 0)
        required_level = role_hierarchy.get(required_role, 0)
        
        return user_level >= required_level
    
    def _audit(self, action, user_id, detail):
        with self.lock:
            self.audit_log.append({
                "timestamp": time.time(),
                "action": action,
                "user_id": user_id,
                "detail": detail
            })
    
    def get_audit_log(self, limit=20):
        with self.lock:
            return self.audit_log[-limit:]

        self.mode = mode

        self.requests_handled = 0
    
    def handle_login(self, email, password, client_ip="unknown"):
        """Handle login request."""
        self.requests_handled += 1
        
        if self.mode == "stateful":
            # WRONG: Store session in local memory
            user = self.auth_service.db.authenticate(email, password)
            if not user:
                return {"success": False, "error": "Invalid credentials",
                        "server": self.server_id}

                "user_id": user["id"],
                "name": user["name"],
                "role": user["role"]
            }

                "success": True,
                "session_id": session_id,
                "user": user,
                "server": self.server_id,
                "mode": "stateful"
            }

            # RIGHT: Use JWT (stateless)
            result = self.auth_service.login(email, password, client_ip)
            result["server"] = self.server_id
            result["mode"] = "stateless"
            return result

        """Handle a request that requires authentication."""
        self.requests_handled += 1
        
        if self.mode == "stateful":
            # WRONG: Look up in local session store
            if not session_id or session_id not in self.local_sessions:
                return {
                    "success": False,
                    "error": "Session not found on this server",
                    "server": self.server_id,
                    "mode": "stateful",
                    "local_sessions": len(self.local_sessions)
                }

            return {
                "success": True,
                "user": session,
                "server": self.server_id,
                "mode": "stateful"
            }

            # RIGHT: Verify JWT (no server state needed)
            if not auth_header or not auth_header.startswith("Bearer "):

                    "success": False,
                    "error": "Missing or invalid Authorization header",
                    "server": self.server_id,
                    "mode": "stateless"
                }
            
            token = auth_header.replace("Bearer ", "")
            user_context, error = self.auth_service.verify_access(token)
            
            if error:
                return {
                    "success": False,
                    "error": error,
                    "server": self.server_id,
                    "mode": "stateless"
                }
            
            return {
                "success": True,
                "user": user_context,
                "server": self.server_id,
                "mode": "stateless"
            }
