🎯 HYPOTHETICAL PROBLEM

You are building a video conferencing platform for a global enterprise
→ 10 million daily active users across 50 countries
→ Meetings range from 1:1 calls to 500-person webinars
→ Users have wildly different network conditions:
    → Executive in NYC: 1 Gbps fiber
    → Sales rep in rural India: 2 Mbps mobile
    → Remote worker in Brazil: 50 Mbps with 5% packet loss

REAL-TIME REQUIREMENTS:
→ End-to-end latency: <150ms (mouth to ear)
    → >150ms: noticeable delay
    → >300ms: conversation becomes difficult
    → >500ms: unusable (people talk over each other)

→ Video: 30 frames/sec at 720p minimum
    → Each uncompressed 720p frame: ~2.7 MB
    → 30 fps × 2.7 MB = 81 MB/sec = 648 Mbps per stream
    → IMPOSSIBLE without compression
    → H.264/VP8 compressed: 1-4 Mbps per stream
    → Still: 10 participants × 4 Mbps = 40 Mbps download

→ Audio: continuous, low-latency
    → Opus codec: 32-128 kbps
    → Must NEVER drop audio (humans tolerate video glitches,
      NOT audio glitches)
    → Audio jitter buffer: 20-60ms

THE CORE CHALLENGES:
  Challenge 1: MEDIA ROUTING
    → 10 participants in a call
    → Peer-to-peer: each sends to 9 others = 9 upload streams
    → On 10 Mbps upload → 1.1 Mbps per stream → potato quality
    → Need a server to relay/mix media

  Challenge 2: ADAPTIVE QUALITY
    → Participant A has 100 Mbps → wants 1080p
    → Participant B has 2 Mbps → can only receive 360p
    → Server must send DIFFERENT quality to DIFFERENT participants
    → In real-time, with no buffering

  Challenge 3: GLOBAL LATENCY
    → User in Tokyo ↔ User in London
    → Network RTT: ~250ms
    → Leaves ZERO room for processing
    → Need media servers in BOTH regions
    → Route media through nearest server

  Challenge 4: SCALE
    → 100,000 concurrent meetings
    → Average 5 participants each = 500,000 media streams
    → Each stream: 1-4 Mbps
    → Total bandwidth: 500 Tbps - 2 Pbps
    → Must be distributed across edge servers worldwide

REAL NUMBERS:

     ┌───┐     │     │     ┌───┐     MCU ENCODES and sends ONE stream

     └───┘     └─────┘     └───┘

     Download per client: 1 × bitrate (single mixed stream)

    → Used by: Zoom, Google Meet, Jitsi, Twilio

📌 WHAT COULD BE DONE

QUALITY ADAPTATION OPTIONS:

  Option A: Simulcast
    → Sender encodes video at MULTIPLE resolutions simultaneously
    → Example: 1080p + 720p + 360p (three separate streams)
    → SFU picks which quality to forward to each receiver
    → Receiver with good bandwidth → gets 1080p
    → Receiver with poor bandwidth → gets 360p
    → Pro: Simple SFU (just picks a stream to forward)
    → Con: Sender uses 3× encoding CPU and 2-3× upload bandwidth
    → Used by: Google Meet, Zoom

  Option B: SVC (Scalable Video Coding)
    → Sender encodes ONE stream with multiple LAYERS
    → Base layer: low quality (always decodable)
    → Enhancement layers: add resolution/framerate
    → SFU can DROP layers per receiver
    → Pro: More bandwidth-efficient than simulcast
    → Con: Codec support is limited, more complex
    → Used by: Some enterprise systems

  Option C: Server-side Transcoding
    → SFU receives one quality, transcodes for each receiver
    → Pro: Sender only sends once
    → Con: MASSIVE server CPU cost, adds latency
    → Only for: specific cases (recording, dial-in)

  FOR THIS PROJECT → Option A (Simulcast)
    → Industry standard, well-supported in WebRTC
    → Simplest SFU implementation
    → Best latency characteristics

BANDWIDTH ESTIMATION OPTIONS:

  Option A: REMB (Receiver Estimated Maximum Bitrate)
    → Receiver estimates available bandwidth from packet loss/delay
    → Sends REMB message back to sender
    → Sender adjusts bitrate accordingly
    → Simple but reactive (reacts AFTER congestion)

  Option B: Transport-CC (Transport-wide Congestion Control)
    → Receiver sends detailed per-packet arrival times
    → Sender runs congestion control algorithm (GCC)
    → More proactive, better quality
    → Used by: modern WebRTC implementations

  Option C: BBR-like (Bottleneck Bandwidth and RTT)
    → Probe for bandwidth by gradually increasing rate
    → Model the bottleneck link capacity
    → Most sophisticated, best performance
    → Still experimental for real-time media

  FOR THIS PROJECT → Option B (Transport-CC with GCC)
    → Modern standard for WebRTC

✅ WHAT SHOULD DO

1. ALWAYS PRIORITIZE AUDIO OVER VIDEO
   → Humans tolerate frozen video for seconds
   → Humans notice audio gaps of >20ms
   → When bandwidth drops → reduce video first, protect audio
   → Audio: 32-128 kbps (tiny compared to video)
   → Video: 500-4000 kbps (can be reduced aggressively)
   → Separate audio and video into different streams

2. USE SIMULCAST FOR GROUPS OF 3+
   → 1:1 calls: single quality stream is fine (both have similar bandwidth)
   → 3+ participants: network conditions vary wildly
   → Simulcast lets SFU adapt per-receiver
   → Typical layers: high (720p/1080p), medium (360p), low (180p)

3. IMPLEMENT SPEAKER DETECTION (DOMINANT SPEAKER)
   → In a 20-person meeting, only 1-2 speak at a time
   → Don't forward video of ALL 20 participants
   → Forward: active speaker + last 3 speakers + pinned participants
   → Others: send only audio (or very low-res thumbnail)
   → Massive bandwidth savings

4. USE REGIONAL SFU CASCADING
   → Meeting: 3 users in US, 2 in EU, 1 in Asia
   → Don't route ALL media through one SFU
   → US users → US SFU, EU users → EU SFU
   → SFUs exchange ONE copy of each stream between regions
   → Each user has low latency to their regional SFU
   → Cross-region latency only between SFUs (not per-user)

5. IMPLEMENT JITTER BUFFER
   → UDP packets arrive with variable delay (jitter)
   → Buffer packets briefly to smooth out delivery
   → Audio jitter buffer: 20-60ms
   → Video jitter buffer: 50-200ms
   → Adaptive: grow buffer when jitter is high, shrink when low
   → Trade latency for smoothness

6. ENCRYPT ALL MEDIA
   → DTLS for key exchange (like TLS but for UDP)
   → SRTP for media encryption (AES-128)
   → End-to-end encryption option (SFU can't decrypt)
   → This is non-negotiable for enterprise use

❌ WHAT SHOULD NOT DO

1. NEVER USE TCP FOR MEDIA TRANSPORT
   → TCP retransmits lost packets → adds latency
   → A retransmitted video frame arrives too late to display
   → Head-of-line blocking: one lost packet delays all following
   → UDP + application-level recovery (FEC, NACK) is correct
   → Only exception: TURN relay as last resort (TCP fallback)

2. NEVER SEND ALL STREAMS AT FULL QUALITY TO ALL PARTICIPANTS
   → 20 participants × 4 Mbps × 19 receivers = 1.5 Gbps per meeting
   → Completely impractical
   → Use: speaker detection, simulcast, quality adaptation
   → Active speaker: high quality, others: low/none

3. NEVER PUT ALL PARTICIPANTS ON ONE SFU SERVER
   → One server failure = entire meeting drops
   → One overloaded server = everyone suffers
   → Distribute across servers, cascade between them
   → Redundancy: if SFU fails, participants reconnect to backup

4. NEVER IGNORE PACKET LOSS
   → 1% packet loss → noticeable video artifacts
   → 5% packet loss → unusable without mitigation
   → Implement: FEC (Forward Error Correction) for recovery
   → Implement: NACK (request retransmission of specific packets)
   → FEC for audio (critical), NACK for video keyframes

5. NEVER BUFFER MORE THAN 200ms OF VIDEO
   → Buffering improves smoothness but kills interactivity
   → >200ms buffer + network latency → >350ms end-to-end
   → Users start talking over each other
   → Better to show a glitch than to add delay

6. NEVER SKIP ONICE CANDIDATE GATHERING
   → Many users are behind NAT/firewalls
   → Without STUN/TURN: cannot establish UDP connections
   → STUN: discovers public IP (works for most NATs)
   → TURN: relays media through server (last resort, works everywhere)
   → ~15% of connections need TURN relay

═══════════════════════════════════════════════════════════
MECHANIC 1: NAT TRAVERSAL (ICE/STUN/TURN)
═══════════════════════════════════════════════════════════

  PROBLEM: Most devices are behind NAT
    → Your laptop's IP: 192.168.1.5 (private, not routable)
    → Router's public IP: 73.42.108.22
    → Remote participant can't send UDP to 192.168.1.5 directly

  SOLUTION: ICE (Interactive Connectivity Establishment)

  Step 1: Gather candidates (possible connection paths)
    
    a) Host candidate: local IP:port
       → 192.168.1.5:54321 (works if on same network)
    
    b) Server-reflexive candidate: public IP via STUN
       → Client sends STUN request to STUN server
       → STUN server responds: "You appear as 73.42.108.22:54321"
       → This is your public address (works for most NATs)
    
    c) Relay candidate: via TURN server
       → Client establishes connection to TURN server
       → TURN allocates a relay address: 52.15.200.10:49152
       → All media routes THROUGH the TURN server
       → Last resort (adds latency) but works through any firewall

  Step 2: Exchange candidates via signaling server
    → Client A sends its candidates to signaling server
    → Signaling server forwards to Client B
    → Client B sends its candidates back

  Step 3: Connectivity checks
    → Try ALL candidate pairs (A's candidates × B's candidates)
    → Send STUN binding request on each pair
    → First pair that works → selected!
    
    Priority: host > server-reflexive > relay
    (prefer direct connection over relayed)

  Typical results:
    → Same network: host candidate works (~0.5ms)
    → Different networks: server-reflexive works (~10-50ms)
    → Behind strict firewall: relay needed (~20-100ms extra)
    → ~85% connections: direct (STUN), ~15%: relayed (TURN)


═══════════════════════════════════════════════════════════
MECHANIC 2: SFU FORWARDING LOGIC
═══════════════════════════════════════════════════════════

  SFU receives streams from N participants
  SFU must decide: which quality to forward to each receiver

  INCOMING (from each sender):
    Simulcast layers:
      Layer 0 (high):   1080p, 30fps, ~3000 kbps
      Layer 1 (medium): 720p,  30fps, ~1500 kbps
      Layer 2 (low):    360p,  15fps, ~500 kbps

  FORWARDING DECISION per receiver:

    For Receiver X watching Sender A:
      1. Is Sender A the active speaker?
         YES → forward highest quality X can receive
         NO  → forward low quality (or nothing)
      
      2. What is X's estimated bandwidth?
         available = X's bandwidth - audio_overhead - other_streams
         
      3. Which layer fits?
         if available > 3000 kbps → Layer 0 (high)
         if available > 1500 kbps → Layer 1 (medium)
         if available > 500 kbps  → Layer 2 (low)
         else → pause video, audio only

    Example: 6-person meeting
      Active speaker: Participant C
      
      Receiver A (50 Mbps):
        → C: high (1080p)     ← active speaker
        → B: medium (720p)    ← recent speaker
        → D,E,F: low (360p)   ← others
        Total: 3000 + 1500 + 500×3 = 6000 kbps ✅
      
      Receiver B (3 Mbps):
        → C: medium (720p)    ← active speaker (reduced)
        → A: low (360p)       ← recent speaker (reduced)
        → D,E,F: audio only   ← others (video paused)
        Total: 1500 + 500 + 128×3 = 2384 kbps ✅

  LAYER SWITCHING:
    When active speaker changes:
      → SFU immediately switches which sender gets high quality
      → Request keyframe from new speaker's high layer
      → Forward keyframe to all receivers
      → Keyframe latency: ~100-200ms (one-time switch cost)


═══════════════════════════════════════════════════════════
MECHANIC 3: BANDWIDTH ESTIMATION (GCC Algorithm)
═══════════════════════════════════════════════════════════

  Goal: Estimate how much bandwidth is available to each receiver

  Transport-CC flow:
    Sender → packets with sequence numbers → Receiver
    Receiver → feedback report (arrival times) → Sender/SFU

  Feedback report:
    Packet #1001: sent at T1, arrived at T1 + 50ms
    Packet #1002: sent at T1 + 5ms, arrived at T1 + 56ms
    Packet #1003: sent at T1 + 10ms, arrived at T1 + 63ms
    
    Inter-arrival deltas:
      #1001→#1002: sent_delta=5ms, recv_delta=6ms → +1ms delay
      #1002→#1003: sent_delta=5ms, recv_delta=7ms → +2ms delay
      
    Delay is INCREASING → congestion building!

  GCC State Machine:
    ┌──────────┐   delay increasing    ┌──────────┐
    │          │ ────────────────────→ │          │
    │ INCREASE │                       │ DECREASE │
    │ (probe)  │ ←──────────────────── │ (backoff)│
    │          │   delay stable        │          │
    └────┬─────┘                       └──────────┘
         │
         │ delay stable for extended period
         ▼
    ┌──────────┐
    │   HOLD   │  ← Optimal point found
    └──────────┘

  Bitrate adjustment:
    INCREASE state: bitrate *= 1.08 (probe 8% more)
    HOLD state:     bitrate unchanged
    DECREASE state: bitrate *= 0.85 (back off 15%)
    
    React quickly to congestion (decrease fast)
    Probe slowly for more bandwidth (increase slow)

═══════════════════════════════════════════════════════════
MECHANIC 6: PACKET LOSS RECOVERY
═══════════════════════════════════════════════════════════

  Two strategies: FEC and NACK

  FEC (Forward Error Correction):
    → Sender adds redundant packets alongside real packets
    → If original packet is lost, receiver reconstructs from FEC
    → Example: send 10 data packets + 2 FEC packets
    → Can recover from any 2 losses in the group
    → Pro: No round-trip delay for recovery
    → Con: Uses extra bandwidth (10-20% overhead)
    → Best for: audio (critical, small packets)

  NACK (Negative Acknowledgment):
    → Receiver detects gap in sequence numbers
    → Sends NACK: "Please retransmit packet #1047"
    → Sender retransmits
    → Pro: No bandwidth overhead when no loss
    → Con: Adds 1 RTT delay for recovery
    → Best for: video keyframes (large, important)

  Combined strategy:
    Audio: FEC always (20% overhead, no delay)
    Video keyframes: NACK (retransmit, worth the wait)
    Video delta frames: skip (next keyframe will fix it)

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: ECHO AND FEEDBACK LOOPS
  Scenario:
    → User A speaks → audio goes to User B
    → User B's speakers play audio → B's mic picks it up
    → B's mic audio goes back to A → A hears their own voice
    → Loop amplifies → HORRIBLE SCREECH
  
  Fix:
    → Acoustic Echo Cancellation (AEC) on every client
    → AEC learns the speaker-to-mic transfer function
    → Subtracts the echo from mic input
    → Built into WebRTC audio processing pipeline

PITFALL 2: KEYFRAME REQUEST STORM
  Scenario:
    → New participant joins meeting
    → Needs keyframe from every sender to start decoding
    → SFU requests keyframe from all 20 senders simultaneously
    → 20 keyframes × 50-100KB each = 1-2MB burst
    → Network spike → packet loss → more keyframe requests → cascade
  
  Fix:
    → Stagger keyframe requests (not all at once)
    → Cache recent keyframes on SFU
    → Rate-limit keyframe requests per sender (max 1 per second)

PITFALL 3: SFU OVERLOAD
  Scenario:
    → SFU handles 100 meetings, 500 streams
    → CPU spikes: bandwidth estimation, packet routing
    → Packets queued → latency increases → quality degrades
    → All 100 meetings suffer simultaneously
  
  Fix:
    → Isolate meetings on different SFU instances
    → Auto-scale: spin up new SFUs when load > 70%
    → Circuit breaker: reject new meetings when overloaded
    → Monitor: packets-per-second, CPU, jitter buffer levels

PITFALL 4: PARTICIPANT BEHIND SYMMETRIC NAT
  Scenario:
    → Corporate firewall blocks ALL incoming UDP
    → STUN fails (firewall drops unsolicited UDP)
    → No direct connection possible
    → If no TURN server → participant CANNOT join
  
  Fix:
    → ALWAYS have TURN servers available
    → TURN over TCP/443 (looks like HTTPS → passes most firewalls)
    → Distribute TURN servers globally (latency matters)
    → Budget: ~10-20% of users will need TURN

PITFALL 5: MOBILE NETWORK HANDOFF
  Scenario:
    → User on mobile switches from WiFi to cellular
    → IP address changes
    → All UDP connections break
    → Audio/video freezes for 2-5 seconds during reconnect
  
  Fix:
    → ICE restart: re-gather candidates, re-establish connection
    → Keep signaling WebSocket alive (TCP survives handoff better)
    → Buffer last audio frames, play comfort noise during gap
    → Aggressive reconnect: start ICE restart within 500ms of detection

PITFALL 6: SIMULCAST LAYER MISMATCH
  Scenario:
    → SFU switches from low to high layer
    → High layer's keyframe hasn't arrived yet
    → Receiver gets delta frames for high layer → can't decode
    → Video freezes or shows artifacts
  
  Fix:
    → SFU requests keyframe BEFORE switching layers
    → Wait for keyframe arrival → then switch
    → Or: use SVC where layers are independently decodable

class BandwidthEstimator:
    """
    Estimates available bandwidth for each participant.
    
    Simplified GCC (Google Congestion Control):
      → Monitors inter-packet delay variations
      → Detects congestion before packet loss occurs
      → Adjusts estimated bandwidth up/down
    
    In production: runs on actual packet timing data
    Here: simulates based on network conditions + noise
    """

    class State(Enum):
        INCREASE = "increase"
        HOLD = "hold"
        DECREASE = "decrease"

    def __init__(self, initial_bandwidth_kbps: int):
        self.estimated_bw_kbps = initial_bandwidth_kbps
        self.state = self.State.INCREASE
        self.min_bw_kbps = 100
        self.max_bw_kbps = initial_bandwidth_kbps
        self.history: List[Tuple[float, int]] = []
        self.delay_gradient_ms: float = 0.0

    def update(self, network: NetworkCondition, actual_send_kbps: int) -> int:
        """
        Update bandwidth estimate based on current network conditions.
        Returns new estimated bandwidth.
        """
        # Simulate delay gradient measurement
        base_delay_gradient = 0.0

        # If sending close to capacity → delay increases
        utilization = actual_send_kbps / max(network.effective_bandwidth(), 1)
        if utilization > 0.9:
            base_delay_gradient = random.uniform(1.0, 5.0)  # Congestion building
        elif utilization > 0.7:
            base_delay_gradient = random.uniform(-0.5, 1.5)  # Borderline
        else:
            base_delay_gradient = random.uniform(-2.0, 0.5)  # Under-utilizing

        # Add noise from jitter
        noise = random.gauss(0, network.jitter_ms * 0.1)
        self.delay_gradient_ms = base_delay_gradient + noise

        # Packet loss signal
        if random.random() * 100 < network.packet_loss_pct:
            self.delay_gradient_ms += 5.0  # Loss indicates severe congestion

        # State transitions
        if self.delay_gradient_ms > 2.0:
            self.state = self.State.DECREASE
        elif self.delay_gradient_ms < -1.0:
            if self.state == self.State.HOLD:
                self.state = self.State.INCREASE
        else:
            self.state = self.State.HOLD

        # Adjust bandwidth
        if self.state == self.State.INCREASE:
            self.estimated_bw_kbps = int(self.estimated_bw_kbps * 1.05)
        elif self.state == self.State.DECREASE:
            self.estimated_bw_kbps = int(self.estimated_bw_kbps * 0.85)

        # Clamp
        self.estimated_bw_kbps = max(
            self.min_bw_kbps,
            min(self.max_bw_kbps, self.estimated_bw_kbps)
        )

        self.history.append((time.time(), self.estimated_bw_kbps))

        return self.estimated_bw_kbps

class DominantSpeakerDetector:
    """
    Detects which participant is the current dominant (active) speaker.
    
    Uses audio energy levels with smoothing and hysteresis
    to prevent rapid switching between speakers.
    """

    def __init__(self):
        self.current_dominant: Optional[str] = None
        self.energy_history: Dict[str, deque] = defaultdict(lambda: deque(maxlen=50))
        self.dominant_since: float = 0
        self.min_switch_interval_sec: float = 0.5
        self.speaker_order: List[str] = []
        self._lock = threading.Lock()

    def update_audio_energy(self, participant_id: str, energy: float):
        with self._lock:
            self.energy_history[participant_id].append(energy)

    def detect(self) -> Tuple[Optional[str], List[str]]:
        """
        Returns (dominant_speaker_id, ordered_list_of_recent_speakers)
        """
        with self._lock:
            if not self.energy_history:
                return None, []

            # Calculate smoothed energy per participant
            scores = {}
            for pid, history in self.energy_history.items():
                if not history:
                    scores[pid] = 0.0
                    continue

                recent = list(history)[-20:]
                avg_energy = sum(recent) / len(recent)

                # Voice Activity Detection (simplified)
                speaking_frames = sum(1 for e in recent if e > 0.1)
                vad_ratio = speaking_frames / len(recent)

                # Score = energy × VAD ratio × recency
                recency_weight = 1.0
                scores[pid] = avg_energy * vad_ratio * recency_weight

            if not scores:
                return self.current_dominant, list(self.speaker_order)

            # Sort by score
            ranked = sorted(scores.items(), key=lambda x: x[1], reverse=True)

            top_speaker = ranked[0][0] if ranked[0][1] > 0.05 else None

            # Hysteresis: don't switch too quickly
            now = time.time()
            if (top_speaker and top_speaker != self.current_dominant
                    and now - self.dominant_since > self.min_switch_interval_sec):
                self.current_dominant = top_speaker
                self.dominant_since = now

                # Update speaker order
                if top_speaker in self.speaker_order:
                    self.speaker_order.remove(top_speaker)
                self.speaker_order.insert(0, top_speaker)
                self.speaker_order = self.speaker_order[:10]

            return self.current_dominant, list(self.speaker_order)

class SFUServer:
    """
    The core media routing server.
    
    Receives media streams from all participants.
    Decides which quality of which stream to forward to each receiver.
    Does NOT decode or re-encode — just smart packet forwarding.
    """

    def __init__(self, sfu_id: str, region: str, capacity_streams: int = 500):
        self.sfu_id = sfu_id
        self.region = region
        self.capacity_streams = capacity_streams

        # Meeting state
        self.meetings: Dict[str, dict] = {}
        self.participants: Dict[str, Participant] = {}
        self.participant_to_meeting: Dict[str, str] = {}

        # Media state
        self.incoming_streams: Dict[str, MediaStream] = {}
        self.forwarding_table: Dict[str, Dict[str, VideoQuality]] = defaultdict(dict)

        # Bandwidth estimators per participant
        self.bw_estimators: Dict[str, BandwidthEstimator] = {}

        # Speaker detection per meeting
        self.speaker_detectors: Dict[str, DominantSpeakerDetector] = {}

        # Stats
        self.total_streams = 0
        self.total_bytes_forwarded = 0
        self.stream_switches = 0

        # Cascading
        self.cascade_peers: Dict[str, 'SFUServer'] = {}
        self.cascade_streams: Dict[str, Set[str]] = defaultdict(set)

        self._lock = threading.Lock()

    def create_meeting(self, meeting_id: str):
        with self._lock:
            self.meetings[meeting_id] = {
                "meeting_id": meeting_id,
                "participants": [],
                "created_at": time.time(),
                "active": True
            }
            self.speaker_detectors[meeting_id] = DominantSpeakerDetector()

    def add_participant(self, meeting_id: str, participant: Participant) -> dict:
        with self._lock:
            if meeting_id not in self.meetings:
                return {"success": False, "error": "Meeting not found"}

            self.participants[participant.participant_id] = participant
            self.participant_to_meeting[participant.participant_id] = meeting_id
            self.meetings[meeting_id]["participants"].append(participant.participant_id)

            # Create bandwidth estimator
            self.bw_estimators[participant.participant_id] = BandwidthEstimator(
                participant.network.effective_bandwidth()
            )

            # Create incoming stream entries
            if participant.video_enabled:
                stream_id = f"video_{participant.participant_id}"
                self.incoming_streams[stream_id] = MediaStream(
                    stream_id=stream_id,
                    sender_id=participant.participant_id,
                    stream_type="video",
                    simulcast_layers=dict(SIMULCAST_LAYERS)
                )
                self.total_streams += 1

            if participant.audio_enabled:
                stream_id = f"audio_{participant.participant_id}"
                self.incoming_streams[stream_id] = MediaStream(
                    stream_id=stream_id,
                    sender_id=participant.participant_id,
                    stream_type="audio"
                )
                self.total_streams += 1

            # Update forwarding table
            self._update_forwarding_table(meeting_id)

            return {
                "success": True,
                "sfu": self.sfu_id,
                "region": self.region,
                "participant_count": len(self.meetings[meeting_id]["participants"])
            }

    def remove_participant(self, participant_id: str):
        with self._lock:
            meeting_id = self.participant_to_meeting.get(participant_id)
            if meeting_id and meeting_id in self.meetings:
                self.meetings[meeting_id]["participants"].remove(participant_id)

            # Clean up streams
            for stream_id in list(self.incoming_streams.keys()):
                if participant_id in stream_id:
                    del self.incoming_streams[stream_id]
                    self.total_streams -= 1

            self.participants.pop(participant_id, None)
            self.participant_to_meeting.pop(participant_id, None)
            self.bw_estimators.pop(participant_id, None)
            self.forwarding_table.pop(participant_id, None)

            if meeting_id:
                self._update_forwarding_table(meeting_id)

    def _update_forwarding_table(self, meeting_id: str):
        """
        Core SFU logic: decide which quality to forward to each receiver.
        
        Factors:
          1. Is the sender the active speaker? → higher quality
          2. Receiver's available bandwidth
          3. Number of streams to receive
          4. Priority: audio > active speaker video > others
        """
        if meeting_id not in self.meetings:
            return

        participants_in_meeting = self.meetings[meeting_id]["participants"]
        if len(participants_in_meeting) < 2:
            return

        # Get dominant speaker
        detector = self.speaker_detectors.get(meeting_id)
        dominant_speaker, recent_speakers = None, []
        if detector:
            dominant_speaker, recent_speakers = detector.detect()

        for receiver_id in participants_in_meeting:
            receiver = self.participants.get(receiver_id)
            if not receiver:
                continue

            # Estimate available bandwidth for this receiver
            estimator = self.bw_estimators.get(receiver_id)
            available_bw = receiver.network.effective_bandwidth()
            if estimator:
                available_bw = estimator.estimated_bw_kbps

            # Reserve bandwidth for audio (always full quality)
            other_count = len(participants_in_meeting) - 1
            audio_reserved = other_count * AUDIO_BITRATE_KBPS
            video_budget = max(0, available_bw - audio_reserved)

            # Allocate video quality per sender
            for sender_id in participants_in_meeting:
                if sender_id == receiver_id:
                    continue

                sender = self.participants.get(sender_id)
                if not sender or not sender.video_enabled:
                    self.forwarding_table[receiver_id][sender_id] = VideoQuality.NONE
                    continue

                # Determine quality based on speaker status and budget
                if sender_id == dominant_speaker:
                    # Active speaker gets highest quality receiver can handle
                    if video_budget > 3000:
                        quality = VideoQuality.HIGH
                        video_budget -= 3000
                    elif video_budget > 1500:
                        quality = VideoQuality.MEDIUM
                        video_budget -= 1500
                    elif video_budget > 500:
                        quality = VideoQuality.LOW
                        video_budget -= 500
                    else:
                        quality = VideoQuality.THUMBNAIL
                        video_budget -= 150

                elif sender_id in recent_speakers[:3]:
                    # Recent speakers get medium quality
                    if video_budget > 1500:
                        quality = VideoQuality.MEDIUM
                        video_budget -= 1500
                    elif video_budget > 500:
                        quality = VideoQuality.LOW
                        video_budget -= 500
                    else:
                        quality = VideoQuality.THUMBNAIL
                        video_budget -= 150
                else:
                    # Others get low/thumbnail
                    if video_budget > 500:
                        quality = VideoQuality.LOW
                        video_budget -= 500
                    elif video_budget > 150:
                        quality = VideoQuality.THUMBNAIL
                        video_budget -= 150
                    else:
                        quality = VideoQuality.NONE

                old_quality = self.forwarding_table[receiver_id].get(sender_id)
                if old_quality != quality:
                    self.stream_switches += 1

                self.forwarding_table[receiver_id][sender_id] = quality

    def simulate_media_tick(self, tick_duration_ms: int = 100):
        """
        Simulate one tick of media forwarding.
        Calculate bytes forwarded, update bandwidth estimates.
        """
        with self._lock:
            for meeting_id, meeting in self.meetings.items():
                participants = meeting["participants"]
                detector = self.speaker_detectors.get(meeting_id)

                for pid in participants:
                    participant = self.participants.get(pid)
                    if not participant:
                        continue

                    # Simulate audio energy (speech patterns)
                    if participant.audio_enabled:
                        if random.random() < 0.15:
                            # Start/stop speaking randomly
                            participant.is_speaking = not participant.is_speaking

                        if participant.is_speaking:
                            participant.audio_energy = random.uniform(0.3, 1.0)
                            participant.speaking_duration_ms += tick_duration_ms
                            participant.last_spoke_at = time.time()
                        else:
                            participant.audio_energy = random.uniform(0.0, 0.05)
                            participant.speaking_duration_ms = 0

                        if detector:
                            detector.update_audio_energy(pid, participant.audio_energy)

                # Update speaker detection and forwarding table
                self._update_forwarding_table(meeting_id)

                # Calculate bytes forwarded this tick
                for receiver_id in participants:
                    receiver = self.participants.get(receiver_id)
                    if not receiver:
                        continue

                    bytes_this_tick = 0

                    for sender_id in participants:
                        if sender_id == receiver_id:
                            continue

                        sender = self.participants.get(sender_id)
                        if not sender:
                            continue

                        # Audio always forwarded
                        if sender.audio_enabled:
                            audio_bytes = int(AUDIO_BITRATE_KBPS * tick_duration_ms / 8)
                            bytes_this_tick += audio_bytes

                        # Video based on forwarding table
                        quality = self.forwarding_table.get(receiver_id, {}).get(
                            sender_id, VideoQuality.NONE
                        )
                        if quality != VideoQuality.NONE and sender.video_enabled:
                            layer = SIMULCAST_LAYERS.get(quality)
                            if layer:
                                video_bytes = int(layer.bitrate_kbps * tick_duration_ms / 8)

                                # Simulate packet loss
                                if random.random() * 100 < receiver.network.packet_loss_pct:
                                    video_bytes = int(video_bytes * 0.95)

                                bytes_this_tick += video_bytes

                    receiver.total_bytes_received += bytes_this_tick
                    self.total_bytes_forwarded += bytes_this_tick

                    # Update bandwidth estimator
                    estimator = self.bw_estimators.get(receiver_id)
                    if estimator:
                        actual_receive_kbps = int(bytes_this_tick * 8 / tick_duration_ms)
                        estimator.update(receiver.network, actual_receive_kbps)

                # Track send bytes
                for sender_id in participants:
                    sender = self.participants.get(sender_id)
                    if not sender:
                        continue
                    if sender.video_enabled:
                        send_quality = sender.send_quality
                        layer = SIMULCAST_LAYERS.get(send_quality)
                        if layer:
                            # Simulcast: sending all layers
                            total_send = sum(
                                l.bitrate_kbps for l in SIMULCAST_LAYERS.values()
                                if l.quality.value <= send_quality.value
                            )
                            send_bytes = int(total_send * tick_duration_ms / 8)
                            sender.total_bytes_sent += send_bytes
                    if sender.audio_enabled:
                        sender.total_bytes_sent += int(AUDIO_BITRATE_KBPS * tick_duration_ms / 8)

    def add_cascade_peer(self, peer_sfu: 'SFUServer'):
        """Connect to another SFU for cross-region cascading"""
        self.cascade_peers[peer_sfu.sfu_id] = peer_sfu

    def get_meeting_stats(self, meeting_id: str) -> dict:
        with self._lock:
            if meeting_id not in self.meetings:
                return {"error": "Meeting not found"}

            meeting = self.meetings[meeting_id]
            participants = meeting["participants"]
            detector = self.speaker_detectors.get(meeting_id)
            dominant, recent = (None, [])
            if detector:
                dominant, recent = detector.detect()

            participant_stats = []
            for pid in participants:
                p = self.participants.get(pid)
                if not p:
                    continue

                # What this participant receives
                receives = {}
                for sender_id, quality in self.forwarding_table.get(pid, {}).items():
                    sender = self.participants.get(sender_id)
                    sender_name = sender.display_name if sender else "unknown"
                    receives[sender_name] = quality.value

                estimator = self.bw_estimators.get(pid)
                est_bw = estimator.estimated_bw_kbps if estimator else 0
                est_state = estimator.state.value if estimator else "unknown"

                participant_stats.append({
                    "id": pid,
                    "name": p.display_name,
                    "region": p.region,
                    "bandwidth_kbps": p.network.bandwidth_kbps,
                    "estimated_bw_kbps": est_bw,
                    "bw_state": est_state,
                    "rtt_ms": p.network.rtt_ms,
                    "packet_loss_pct": p.network.packet_loss_pct,
                    "is_speaking": p.is_speaking,
                    "is_dominant": pid == dominant,
                    "connection": p.connection_type.value,
                    "receives": receives,
                    "sent_kb": round(p.total_bytes_sent / 1024, 1),
                    "received_kb": round(p.total_bytes_received / 1024, 1)
                })

            return {
                "meeting_id": meeting_id,
                "participant_count": len(participants),
                "dominant_speaker": dominant,
                "recent_speakers": recent[:5],
                "participants": participant_stats,
                "sfu": self.sfu_id,
                "region": self.region
            }

    def get_sfu_stats(self) -> dict:
        with self._lock:
            return {
                "sfu_id": self.sfu_id,
                "region": self.region,
                "meetings": len(self.meetings),
                "participants": len(self.participants),
                "total_streams": self.total_streams,
                "capacity_streams": self.capacity_streams,
                "utilization_pct": round(
                    self.total_streams / max(self.capacity_streams, 1) * 100, 1
                ),
                "total_bytes_forwarded": self.total_bytes_forwarded,
                "total_mb_forwarded": round(self.total_bytes_forwarded / (1024 * 1024), 2),
                "stream_quality_switches": self.stream_switches,
                "cascade_peers": list(self.cascade_peers.keys())
            }

class SignalingServer:
    """
    Manages room state and participant signaling.
    
    In production: WebSocket server handling SDP offer/answer
    and ICE candidate exchange.
    Here: simulated signaling for room management.
    """

    def __init__(self):
        self.rooms: Dict[str, dict] = {}
        self.room_counter = 0
        self._lock = threading.Lock()

    def create_room(self, host_name: str) -> dict:
        with self._lock:
            self.room_counter += 1
            room_id = f"room_{self.room_counter:06d}"
            self.rooms[room_id] = {
                "room_id": room_id,
                "host": host_name,
                "participants": [],
                "created_at": time.time(),
                "state": "active",
                "assigned_sfus": {}
            }
            return {"room_id": room_id, "success": True}

    def join_room(self, room_id: str, participant_id: str, region: str) -> dict:
        with self._lock:
            if room_id not in self.rooms:
                return {"success": False, "error": "Room not found"}

            room = self.rooms[room_id]
            room["participants"].append({
                "participant_id": participant_id,
                "region": region,
                "joined_at": time.time()
            })

            # SDP negotiation simulation
            sdp_offer = {
                "type": "offer",
                "sdp": f"v=0\nm=audio 9 UDP/TLS/RTP/SAVPF 111\n"
                       f"m=video 9 UDP/TLS/RTP/SAVPF 96 97 98\n"
                       f"a=simulcast:send h;m;l\n"
                       f"a=rid:h send\na=rid:m send\na=rid:l send"
            }
            sdp_answer = {
                "type": "answer",
                "sdp": f"v=0\nm=audio 9 UDP/TLS/RTP/SAVPF 111\n"
                       f"m=video 9 UDP/TLS/RTP/SAVPF 96 97 98\n"
                       f"a=simulcast:recv h;m;l"
            }

            # ICE candidate simulation
            ice_candidates = [
                {"type": "host", "ip": f"192.168.1.{random.randint(2,254)}",
                 "port": random.randint(10000, 65535)},
                {"type": "srflx", "ip": f"{random.randint(1,223)}.{random.randint(0,255)}."
                                        f"{random.randint(0,255)}.{random.randint(1,254)}",
                 "port": random.randint(10000, 65535)},
            ]

            return {
                "success": True,
                "room_id": room_id,
                "participant_count": len(room["participants"]),
                "sdp_offer": sdp_offer,
                "sdp_answer": sdp_answer,
                "ice_candidates": ice_candidates,
                "signaling_latency_ms": random.uniform(10, 50)
            }

    def leave_room(self, room_id: str, participant_id: str):
        with self._lock:
            if room_id in self.rooms:
                self.rooms[room_id]["participants"] = [
                    p for p in self.rooms[room_id]["participants"]
                    if p["participant_id"] != participant_id
                ]

    def get_room_info(self, room_id: str) -> dict:
        with self._lock:
            return dict(self.rooms.get(room_id, {}))

class QualityMetrics:
    """
    Calculates call quality metrics similar to real-world monitoring.
    
    MOS (Mean Opinion Score): 1.0-5.0 scale
      5.0: Excellent
      4.0: Good
      3.0: Fair
      2.0: Poor
      1.0: Bad
    """

    @staticmethod
    def calculate_mos(rtt_ms: float, packet_loss_pct: float,
                      jitter_ms: float, codec: str = "opus") -> float:
        """
        Simplified E-model (ITU-T G.107) for MOS estimation.
        """
        # Base R-factor for Opus codec
        r_factor = 93.2

        # Delay impairment
        effective_delay = rtt_ms / 2 + jitter_ms + 10  # 10ms processing
        if effective_delay < 160:
            delay_factor = 0
        else:
            delay_factor = 0.024 * effective_delay + 0.11 * (effective_delay - 160)
        delay_factor = min(delay_factor, 30)

        # Packet loss impairment
        loss_factor = 0
        if packet_loss_pct > 0:
            loss_factor = 7.0 + 20.0 * math.log(1 + packet_loss_pct * 10)
        loss_factor = min(loss_factor, 50)

        # Jitter impairment
        jitter_factor = min(jitter_ms * 0.1, 10)

        r_factor -= (delay_factor + loss_factor + jitter_factor)
        r_factor = max(0, min(100, r_factor))

        # Convert R-factor to MOS
        if r_factor < 0:
            mos = 1.0
        elif r_factor > 100:
            mos = 4.5
        else:
            mos = 1.0 + 0.035 * r_factor + r_factor * (r_factor - 60) * (100 - r_factor) * 7e-6

        return round(max(1.0, min(5.0, mos)), 2)

    @staticmethod
    def quality_label(mos: float) -> str:
        if mos >= 4.3:
            return "Excellent ✅"
        elif mos >= 3.5:
            return "Good ✅"
        elif mos >= 2.5:
            return "Fair ⚠️"
        elif mos >= 1.5:
            return "Poor ❌"
        else:
            return "Bad ❌"

    @staticmethod
    def video_quality_score(fps: int, resolution_height: int,
                            target_fps: int = 30, target_height: int = 720) -> float:
        fps_score = min(fps / target_fps, 1.0)
        res_score = min(resolution_height / target_height, 1.0)
        return round((fps_score * 0.4 + res_score * 0.6) * 5.0, 2)

class MeetingOrchestrator:
    """
    High-level orchestrator that manages the complete meeting lifecycle.
    Assigns SFUs, handles cascading, monitors quality.
    """

    def __init__(self):
        self.signaling = SignalingServer()
        self.sfus: Dict[str, SFUServer] = {}
        self.region_to_sfu: Dict[str, str] = {}

        # Cross-region latencies for cascading
        self.region_latencies: Dict[Tuple[str, str], float] = {}

        # Global stats
        self.total_meetings_created = 0
        self.total_participants_joined = 0

    def add_sfu(self, sfu_id: str, region: str, capacity: int = 500):
        sfu = SFUServer(sfu_id, region, capacity)
        self.sfus[sfu_id] = sfu
        self.region_to_sfu[region] = sfu_id

    def set_region_latency(self, region_a: str, region_b: str, latency_ms: float):
        self.region_latencies[(region_a, region_b)] = latency_ms
        self.region_latencies[(region_b, region_a)] = latency_ms

    def setup_cascading(self):
        """Connect SFUs in different regions for cascading"""
        sfu_list = list(self.sfus.values())
        for i, sfu_a in enumerate(sfu_list):
            for sfu_b in sfu_list[i + 1:]:
                if sfu_a.region != sfu_b.region:
                    sfu_a.add_cascade_peer(sfu_b)
                    sfu_b.add_cascade_peer(sfu_a)

    def get_nearest_sfu(self, region: str) -> Optional[SFUServer]:
        """Find the SFU nearest to the participant's region"""
        if region in self.region_to_sfu:
            sfu_id = self.region_to_sfu[region]
            return self.sfus.get(sfu_id)

        # Find nearest by latency
        best_sfu = None
        best_latency = float('inf')
        for sfu in self.sfus.values():
            latency = self.region_latencies.get(
                (region, sfu.region), 999
            )
            if latency < best_latency:
                best_latency = latency
                best_sfu = sfu
        return best_sfu

    def create_meeting(self, host_name: str) -> str:
        result = self.signaling.create_room(host_name)
        room_id = result["room_id"]
        self.total_meetings_created += 1

        # Create meeting on all SFUs (for cascading readiness)
        for sfu in self.sfus.values():
            sfu.create_meeting(room_id)

        return room_id

    def join_meeting(self, meeting_id: str, participant: Participant) -> dict:
        self.total_participants_joined += 1

        # Signaling
        signal_result = self.signaling.join_room(
            meeting_id, participant.participant_id, participant.region
        )

        if not signal_result["success"]:
            return signal_result

        # Determine connection type (NAT traversal simulation)
        nat_roll = random.random()
        if nat_roll < 0.70:
            participant.connection_type = ConnectionType.DIRECT
        else:
            participant.connection_type = ConnectionType.RELAY
            # TURN relay adds latency
            participant.network.rtt_ms += random.uniform(10, 30)

        # Find nearest SFU
        sfu = self.get_nearest_sfu(participant.region)
        if not sfu:
            return {"success": False, "error": "No SFU available"}

        # Add to SFU
        sfu_result = sfu.add_participant(meeting_id, participant)

        return {
            "success": True,
            "meeting_id": meeting_id,
            "sfu": sfu.sfu_id,
            "sfu_region": sfu.region,
            "connection_type": participant.connection_type.value,
            "signaling_latency_ms": signal_result["signaling_latency_ms"],
            "ice_candidates": len(signal_result["ice_candidates"]),
            "participant_count": sfu_result.get("participant_count", 0)
        }

    def simulate_meeting_tick(self, tick_ms: int = 100):
        """Run one simulation tick for all SFUs"""
        for sfu in self.sfus.values():
            sfu.simulate_media_tick(tick_ms)

    def get_meeting_quality_report(self, meeting_id: str) -> dict:
        """Comprehensive quality report for a meeting"""
        all_participant_stats = []

        for sfu in self.sfus.values():
            stats = sfu.get_meeting_stats(meeting_id)
            if "participants" in stats:
                all_participant_stats.extend(stats["participants"])

        if not all_participant_stats:
            return {"error": "Meeting not found or no participants"}

        # Calculate quality metrics
        quality_report = []
        for p in all_participant_stats:
            mos = QualityMetrics.calculate_mos(
                p["rtt_ms"], p["packet_loss_pct"],
                p.get("jitter_ms", 10)
            )

            # Determine received video quality
            receives = p.get("receives", {})
            max_quality = "none"
            for sender, quality in receives.items():
                if quality != "none":
                    max_quality = quality

            quality_report.append({
                "name": p["name"],
                "region": p["region"],
                "mos": mos,
                "mos_label": QualityMetrics.quality_label(mos),
                "bandwidth_kbps": p["bandwidth_kbps"],
                "estimated_bw_kbps": p["estimated_bw_kbps"],
                "bw_state": p["bw_state"],
                "rtt_ms": p["rtt_ms"],
                "packet_loss_pct": p["packet_loss_pct"],
                "connection": p["connection"],
                "is_dominant": p["is_dominant"],
                "max_receive_quality": max_quality,
                "sent_kb": p["sent_kb"],
                "received_kb": p["received_kb"],
                "receives_from": p["receives"]
            })

        avg_mos = statistics.mean([q["mos"] for q in quality_report])

        return {
            "meeting_id": meeting_id,
            "participant_count": len(quality_report),
            "avg_mos": round(avg_mos, 2),
            "avg_mos_label": QualityMetrics.quality_label(avg_mos),
            "participants": quality_report
        }

    def get_global_stats(self) -> dict:
        sfu_stats = []
        total_streams = 0
        total_mb = 0

        for sfu in self.sfus.values():
            stats = sfu.get_sfu_stats()
            sfu_stats.append(stats)
            total_streams += stats["total_streams"]
            total_mb += stats["total_mb_forwarded"]

        return {
            "total_meetings": self.total_meetings_created,
            "total_participants": self.total_participants_joined,
            "total_streams": total_streams,
            "total_mb_forwarded": round(total_mb, 2),
            "sfus": sfu_stats
        }
