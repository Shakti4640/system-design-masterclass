🎯 HYPOTHETICAL PROBLEM

You are building a collaborative document editor like Google Docs

The situation:
→ 15 users have the same document open simultaneously
→ User A in New York types "Hello" at the beginning of line 3
→ User B in Tokyo deletes a word in line 3 at the EXACT same time
→ User C in London pastes a paragraph between line 2 and line 3
→ All three operations happen within the same 50 milliseconds
→ Network latency: NY→Server = 30ms, Tokyo→Server = 120ms, London→Server = 60ms

WHAT GOES WRONG without proper design:

  Scenario A — LOST UPDATES:
    → Document state: "The quick brown fox"
    → User A changes "quick" → "slow"     (sees "The slow brown fox")
    → User B changes "brown" → "red"      (sees "The quick red fox")
    → Last write wins
    → Final: "The quick red fox" — User A's edit is LOST
    → User A sees their change disappear → rage quits

  Scenario B — DIVERGENT DOCUMENTS:
    → Document state: "ABCD"
    → User A inserts "X" at position 2 → sees "AXBCD"
    → User B inserts "Y" at position 3 → sees "ABCYD" (on their screen)
    → Server receives A's op first, applies it: "AXBCD"
    → Server receives B's op: insert "Y" at position 3
    → Server applies: "AXYBCD" ← position 3 is now different!
    → User B expected "Y" after "C", but it ended up after "X"
    → Documents DIVERGE — User A sees "AXYBCD", User B sees "ABCYD"
    → They never converge → document is corrupted

  Scenario C — INTENTION VIOLATION:
    → Document: "Hello World"
    → User A selects "World" and types "Earth"
    → User B puts cursor after "Hello" and types " Beautiful"
    → User A INTENDED: "Hello Earth"
    → User B INTENDED: "Hello Beautiful World"
    → Correct merged result: "Hello Beautiful Earth"
    → But naive merging could produce: "Hello BeautEarthiful"
    → The INTENTION of each edit must be preserved

  Scenario D — ORDERING CHAOS:
    → User A sends operations: Op1, Op2, Op3
    → Network delivers them: Op1, Op3, Op2 (out of order)
    → Applying Op3 before Op2 produces wrong state
    → Every client must see operations in a CONSISTENT order
    → Even if network delivers them differently

THE REAL CHALLENGE:
→ How do you let 15 users type simultaneously
→ Without any locks (locking = unusable latency)
→ Without losing anyone's edits
→ Without documents diverging across clients
→ While network latency varies from 30ms to 200ms
→ And operations can arrive out of order
→ And all clients CONVERGE to the exact same document state

🔍 HOW TO APPROACH IT

Decompose into 6 DISTINCT sub-problems:

SUB-PROBLEM 1: OPERATION REPRESENTATION
  → How to describe "what the user did" as a data structure
  → Insert(position, text), Delete(position, length), Retain(count)
  → Operations must be small, composable, and transformable

SUB-PROBLEM 2: CONCURRENT OPERATION MERGING
  → When two operations happen simultaneously on the same state
  → How to transform them so both can be applied correctly
  → This is the CORE algorithm: Operational Transform (OT)

SUB-PROBLEM 3: CONVERGENCE GUARANTEE
  → ALL clients must end up with the EXACT same document
  → Regardless of operation arrival order
  → Mathematical property: eventual consistency with convergence

SUB-PROBLEM 4: REAL-TIME DELIVERY
  → Operations must be broadcast to all clients in near-real-time
  → WebSocket-based push (already solved in Project 31)
  → But now with operation ordering guarantees

SUB-PROBLEM 5: CONFLICT-FREE DATA STRUCTURES (CRDT)
  → Alternative to OT: data structures that merge automatically
  → No central server needed for conflict resolution
  → Trade-off: more memory, simpler algorithm

SUB-PROBLEM 6: PERSISTENCE & HISTORY
  → Save document state periodically
  → Support undo/redo across multiple users
  → Version history ("who changed what when")

TWO MAJOR APPROACHES:

  APPROACH 1: OPERATIONAL TRANSFORM (OT)
    → Used by Google Docs
    → Central server transforms and orders operations
    → Complex algorithm, but efficient on wire
    → Requires server as coordination point

  APPROACH 2: CONFLICT-FREE REPLICATED DATA TYPES (CRDT)
    → Used by Figma, Apple Notes
    → Each character has a unique ID
    → Operations commute naturally (order doesn't matter)
    → No central server needed (peer-to-peer possible)
    → More memory overhead, simpler correctness guarantees

  WE WILL BUILD BOTH to understand the trade-offs

═══════════════════════════════════════════════════════
APPROACH 1: OPERATIONAL TRANSFORM (OT) — Server-Centric
═══════════════════════════════════════════════════════

CORE IDEA:
  → Every edit is an "operation" (insert, delete)
  → When two operations are concurrent (neither saw the other)
  → TRANSFORM one against the other before applying
  → The transformation adjusts positions to account for the other's effect

EXAMPLE — THE CLASSIC OT SCENARIO:

  Document: "ABC"    (all clients agree on this state)

  User A: Insert("X", position=1)  →  "AXBC"
  User B: Insert("Y", position=2)  →  "ABYC"

  Both happen concurrently (neither saw the other's operation)

  Server receives A's operation first:
    Apply A: "ABC" → "AXBC"

  Now server must apply B's operation on top of A's result:
    B says: Insert("Y", position=2)
    But A inserted "X" at position 1, which SHIFTED everything right
    Position 2 in original = position 3 in new state
    TRANSFORM B against A: Insert("Y", position=3)
    Apply transformed B: "AXBC" → "AXBYC"

  Meanwhile, User A must apply B's operation on their state "AXBC":
    Same transform: B's position 2 → position 3
    "AXBC" → "AXBYC" ✅

  User B must apply A's operation on their state "ABYC":
    A says: Insert("X", position=1)
    B already inserted "Y" at position 2, which is AFTER position 1
    So A's position doesn't need adjustment
    TRANSFORM A against B: Insert("X", position=1) (unchanged)
    "ABYC" → "AXBYC" ✅

  ALL THREE (server, A, B) converge to "AXBYC" ✅

THE TRANSFORM FUNCTION:
  transform(opA, opB) → (opA', opB')
  
  Where:
    apply(apply(doc, opA), opB') = apply(apply(doc, opB), opA')
  
  This is the "diamond property" — both paths lead to same state:
  
           doc
          /   \
        opA   opB
        /       \
     docA      docB
        \       /
       opB'   opA'
          \   /
          docFinal  ← MUST be identical via both paths

APPROACH 2: CRDT — Decentralized
═══════════════════════════════════════════════════════

CORE IDEA:
  → Every character in the document has a GLOBALLY UNIQUE ID
  → IDs define a total ordering that ALL clients agree on
  → Insert: create new ID between two existing IDs
  → Delete: mark ID as "tombstone" (logically deleted)
  → No transforms needed — operations commute naturally

EXAMPLE — CRDT WITH UNIQUE IDs:

  Document: "ABC"
  Internal representation:
    {id: (0,S), char: 'A'} → {id: (1,S), char: 'B'} → {id: (2,S), char: 'C'}

  User A inserts "X" between A and B:
    New ID: (0.5, A)  ← between (0,S) and (1,S), tagged with user A
    {id:(0,S),'A'} → {id:(0.5,A),'X'} → {id:(1,S),'B'} → {id:(2,S),'C'}

  User B inserts "Y" between B and C:
    New ID: (1.5, B)  ← between (1,S) and (2,S), tagged with user B
    {id:(0,S),'A'} → {id:(1,S),'B'} → {id:(1.5,B),'Y'} → {id:(2,S),'C'}

  When A receives B's operation:
    Just insert (1.5,B),'Y' in sorted position
    Result: A X B Y C ✅

  When B receives A's operation:
    Just insert (0.5,A),'X' in sorted position
    Result: A X B Y C ✅

  CONVERGENCE IS AUTOMATIC — no transforms needed!
  Order of receiving operations DOESN'T MATTER

📌 WHAT COULD BE DONE

OPTION A: NAIVE LAST-WRITE-WINS
  → Simply overwrite with latest version
  → Trivial to implement
  → LOSES user edits
  → Unacceptable for collaborative editing
  → Only viable for single-user with auto-save

OPTION B: LOCKING (Pessimistic Concurrency)
  → Lock the paragraph/section being edited
  → Only one user can edit at a time
  → NO data loss, but terrible UX
  → "Section locked by User B" → frustrating
  → Early collaborative editors used this (SharePoint)

OPTION C: OPERATIONAL TRANSFORM (OT)
  → Our primary approach
  → Server-centric: server is the "truth"
  → Each operation transformed against concurrent ones
  → Used by: Google Docs, Etherpad
  → Complex algorithm, proven at scale
  → Requires reliable server

OPTION D: CRDT (Conflict-Free Replicated Data Types)
  → Our secondary approach
  → Decentralized: no single point of truth needed
  → Each character has unique ID → operations commute
  → Used by: Figma, Apple Notes, Ink & Switch
  → Simpler correctness proof, more memory overhead
  → Works offline natively

OPTION E: DIFF-PATCH-MERGE
  → Periodically diff client state against server
  → Merge diffs using three-way merge (like Git)
  → Works for slow collaboration (minutes between edits)
  → Too slow for real-time keystroke-level collaboration

OUR DESIGN: OPTION C (OT) as primary + OPTION D (CRDT) as comparison
  → OT for the server-mediated real-time system
  → CRDT to demonstrate the alternative mental model
  → Both produce the same result: convergence

✅ WHAT SHOULD DO

1. TREAT EVERY KEYSTROKE AS AN OPERATION
   → Don't wait for "save" — stream operations in real-time
   → Small operations = easier to transform
   → Batch nearby keystrokes (typing "hello" = one insert of "hello")

2. USE A CENTRAL SERVER AS ORDERING AUTHORITY (for OT)
   → Server assigns a global sequence number to each operation
   → Clients must acknowledge: "I've seen up to operation #N"
   → This creates a total order that everyone agrees on

3. VERSION VECTOR ON EVERY OPERATION
   → Each operation carries: "I was created when document was at version X"
   → Server knows which operations the client has already seen
   → Only transforms against operations the client HASN'T seen

4. BUFFER CLIENT OPERATIONS WHILE AWAITING ACK
   → Client sends Op1 to server
   → Before server ACKs, client types more → Op2 created locally
   → Op2 is buffered (not sent yet)
   → When Op1 is ACK'd, Op2 is transformed if needed, then sent
   → This prevents operation explosion

5. APPLY LOCALLY FIRST, THEN SEND
   → User types → immediately see their own change (no lag)
   → Send operation to server in background
   → If server transforms it differently → adjust local state
   → This gives the feeling of instant response

6. SNAPSHOT PERIODICALLY
   → Don't replay 10,000 operations from the beginning
   → Save full document state every N operations
   → New clients load latest snapshot + recent ops
   → Event sourcing with compaction (Project 29 concept)

7. TOMBSTONE DELETES (for CRDT)
   → Never physically remove characters
   → Mark them as deleted (tombstone)
   → Needed so late-arriving operations can reference them
   → Garbage collect tombstones after all clients have seen them

8. CURSOR AND SELECTION AWARENESS
   → Broadcast each user's cursor position
   → Transform cursor positions along with operations
   → Show colored cursors with user names
   → This is a PRESENCE feature (Project 31 concept)

❌ WHAT SHOULD NOT DO

1. NEVER USE POSITION-ONLY WITHOUT TRANSFORM
   → "Insert X at position 5" is meaningless after concurrent edits
   → Position 5 might now be position 7 or position 3
   → ALWAYS transform positions against concurrent operations

2. NEVER APPLY REMOTE OPERATIONS WITHOUT VERSION CHECK
   → Client at version 10 receives op created at version 8
   → Must transform against operations 8, 9 before applying
   → Skipping this = divergent documents

3. NEVER BLOCK THE UI WHILE WAITING FOR SERVER
   → User types → show immediately
   → Server round-trip: 30-200ms
   → If UI freezes for 200ms per keystroke → unusable
   → ALWAYS: local-first, sync in background

4. NEVER SEND FULL DOCUMENT ON EVERY CHANGE
   → Document is 50KB of text
   → User types one character
   → Sending 50KB vs sending 20 bytes (the operation)
   → Operations are 1000x more efficient on the wire

5. NEVER ASSUME OPERATIONS ARRIVE IN ORDER
   → Network can reorder, duplicate, or delay
   → Use sequence numbers to detect gaps
   → Buffer out-of-order operations until gap is filled

6. NEVER SKIP CONFLICT RESOLUTION ON DELETE-INSERT CLASH
   → User A deletes text at position 5-10
   → User B inserts text at position 7 (inside A's deletion range)
   → What happens to B's insertion?
   → Decision: preserve B's insert at the deletion boundary
   → This is a POLICY choice — document it clearly

7. NEVER FORGET UNDO IN COLLABORATIVE CONTEXT
   → User A types "Hello", User B types "World"
   → User A presses Ctrl+Z
   → Should undo "Hello" but NOT "World"
   → Undo must be PER-USER, not global
   → Undo of A's operation = inverse operation, transformed against B's

8. NEVER GROW CRDT METADATA UNBOUNDEDLY
   → Every character has a unique ID → metadata overhead
   → Deleted characters become tombstones → memory leak
   → MUST garbage collect tombstones periodically
   → Only safe when ALL clients have acknowledged past that point

🚫 WHEN NOT TO USE

1. OT IS NOT SUITABLE WHEN:
   → No reliable central server (edge computing, P2P)
   → Offline editing is a primary use case
   → More than ~50 concurrent editors on same document
     (server becomes bottleneck transforming N² operations)
   → Non-linear data structures (graphs, trees)
     OT works best on linear sequences (text)

2. CRDT IS NOT SUITABLE WHEN:
   → Memory is extremely constrained (IoT devices)
   → Document is huge and metadata overhead matters
   → You need strong consistency (not eventual)
   → Tombstone garbage collection is impractical
     (clients can go offline for weeks)

3. COLLABORATIVE EDITING IS NOT SUITABLE WHEN:
   → Data has strict schema constraints (database records)
     → Use distributed transactions instead (Project 37)
   → Edits are mutually exclusive (inventory — Project 38)
     → Use reservation pattern, not merge
   → Content is append-only (chat messages — Project 31)
     → Much simpler — just append, no transforms needed

4. REAL-TIME SYNC IS NOT SUITABLE WHEN:
   → Users edit once per day (wiki, documentation)
     → Simple version control with manual merge is sufficient
   → Document is binary (images, video)
     → Use operational patches specific to the format

PART A: OPERATIONAL TRANSFORM — DEEP MECHANICS
═══════════════════════════════════════════════════════════════

OPERATIONS ARE REPRESENTED AS SEQUENCES OF COMPONENTS:

  Operation = list of components, processed left-to-right

  Components:
    Retain(n)        → Skip n characters (leave them unchanged)
    Insert(text)     → Insert text at current position
    Delete(n)        → Delete n characters at current position

  EXAMPLE:
    Document: "Hello World" (length 11)
    User wants to change "World" to "Earth":
    
    Operation: [Retain(6), Delete(5), Insert("Earth")]
    
    Execution:
      Retain(6)        → cursor moves past "Hello " (unchanged)
      Delete(5)        → removes "World"
      Insert("Earth")  → inserts "Earth" at cursor
    
    Result: "Hello Earth"

  EVERY OPERATION MUST SPAN THE ENTIRE DOCUMENT:
    Sum of (Retain lengths + Delete lengths) = document length
    This ensures operations are "complete" — no ambiguity

THE TRANSFORM FUNCTION — HEART OF OT
═══════════════════════════════════════════════════════════════

  transform(opA, opB) → (opA', opB')

  Both opA and opB were created against the SAME document state.
  
  opA' = opA adjusted to apply AFTER opB has been applied
  opB' = opB adjusted to apply AFTER opA has been applied

  RULES FOR TRANSFORM (Insert vs Insert):
  ─────────────────────────────────────────
    opA = Insert("X", pos=3)
    opB = Insert("Y", pos=5)

    A before B:
      opA' = Insert("X", pos=3)    ← B is after, no change needed
      opB' = Insert("Y", pos=6)    ← A shifted B right by 1

    A after B:
      opA' = Insert("X", pos=4)    ← B shifted A right by 1
      opB' = Insert("Y", pos=5)    ← A is after, no change needed

    A same position (tie-breaking):
      Use user_id to break ties deterministically
      Lower user_id goes first (arbitrary but consistent)

  RULES FOR TRANSFORM (Insert vs Delete):
  ─────────────────────────────────────────
    opA = Insert("X", pos=3)
    opB = Delete(pos=5, len=2)

    Insert before delete region:
      opA' = Insert("X", pos=3)       ← delete is after, no change
      opB' = Delete(pos=6, len=2)     ← insert shifted delete right by 1

    Insert inside delete region:
      opA' = Insert("X", pos=5)       ← insert at start of deleted region
      opB' = Delete(pos=5, len=2)     ← split around insert? 
      → POLICY DECISION: preserve the insert, adjust delete

    Insert after delete region:
      opA' = Insert("X", pos=1)       ← delete shortened doc, adjust left
      opB' = Delete(pos=5, len=2)     ← insert is after, no change

  RULES FOR TRANSFORM (Delete vs Delete):
  ─────────────────────────────────────────
    opA = Delete(pos=3, len=4)   → deletes chars 3,4,5,6
    opB = Delete(pos=5, len=3)   → deletes chars 5,6,7

    Overlapping deletes:
      Overlap: chars 5,6 are in both
      opA' = Delete(pos=3, len=2)  ← only delete chars 3,4 (5,6 already gone)
      opB' = Delete(pos=3, len=1)  ← only delete char 7 (5,6 already gone)
      Position adjusted because A's earlier deletion shifted things


═══════════════════════════════════════════════════════════════
CLIENT STATE MACHINE (OT Protocol)
═══════════════════════════════════════════════════════════════

  Each client is in one of THREE states:

  STATE 1: SYNCHRONIZED
    → Client's document matches server's document
    → Client has no pending operations
    → When user types → apply locally, send to server, go to State 2

  STATE 2: AWAITING CONFIRMATION
    → Client sent an operation, waiting for server ACK
    → If user types more → buffer it (don't send yet)
    → If server ACKs our operation → 
        If buffer empty → go to State 1
        If buffer has ops → send buffer, go to State 2
    → If server sends SOMEONE ELSE's operation →
        Transform our pending op against it
        Transform our buffer against it
        Apply the (transformed) remote op locally

  STATE 3: AWAITING WITH BUFFER
    → Client has sent op (awaiting ACK) AND has buffered new ops
    → If server ACKs → send buffer, go to State 2
    → If server sends remote op → transform both pending AND buffer

  DIAGRAM:
    ┌──────────────┐
    │ SYNCHRONIZED │ ←────────────────────────────────┐
    └──────┬───────┘                                  │
           │ user types                               │
           │ (apply local, send to server)            │
           ▼                                          │
    ┌──────────────────┐                              │
    │ AWAITING CONFIRM │ ──── server ACK + no buffer ─┘
    └──────┬───────────┘
           │ user types more (buffer it)
           │
           ▼
    ┌──────────────────────┐
    │ AWAITING WITH BUFFER │
    └──────┬───────────────┘
           │ server ACK → send buffer → back to AWAITING
           │ server ACK + no more buffer → SYNCHRONIZED


═══════════════════════════════════════════════════════════════
SERVER PROTOCOL
═══════════════════════════════════════════════════════════════

  Server maintains:
    → document: current text state
    → revision: integer, increments with each applied operation
    → history: list of all applied operations (for transforms)

  When server receives operation from Client X:
    1. Client says: "This op was created at revision R"
    2. Server's current revision is S (where S >= R)
    3. Server must transform the op against all ops from R to S
       (these are ops the client hadn't seen when they created theirs)
    4. Apply transformed operation to document
    5. Increment revision to S+1
    6. Broadcast transformed op to ALL clients (including sender)
    7. Sender knows their op was accepted
    8. Others know to transform and apply

  TRANSFORM CHAIN:
    Client op created at revision 5
    Server is at revision 8
    Operations 5→6, 6→7, 7→8 are in history

    client_op' = transform(client_op, history[5])
    client_op'' = transform(client_op', history[6])
    client_op''' = transform(client_op'', history[7])
    
    Apply client_op''' to document → revision 9

PART B: CRDT — DEEP MECHANICS
═══════════════════════════════════════════════════════════════

  TYPE: Sequence CRDT (for text documents)
  VARIANT: LSEQ / RGA / Logoot / YATA

  CORE PRINCIPLE:
    Every character gets a unique, immutable position identifier
    These identifiers define a TOTAL ORDER
    Any client can generate new identifiers between existing ones
    The ordering is consistent across all clients WITHOUT coordination

  POSITION IDENTIFIER STRUCTURE:
    (fractional_position, site_id, logical_clock)

    fractional_position: a number between two neighbors
    site_id: unique per client (breaks ties)
    logical_clock: monotonically increasing per client

  EXAMPLE:
    Initial: [(0.0, S, 0, 'H'), (0.25, S, 0, 'i')]
    
    Client A inserts 'X' between H and i:
      New position: midpoint = (0.0 + 0.25) / 2 = 0.125
      ID: (0.125, A, 1)
      
    Client B inserts 'Y' between H and i (concurrently):
      New position: midpoint = 0.125 (same calculation!)
      ID: (0.125, B, 1)
      
    TIE-BREAKING:
      Both have position 0.125
      Compare site_id: A < B (alphabetical)
      So X comes before Y
      
    Result on ALL clients: H, X, Y, i ✅

  DELETION IN CRDT:
    → Don't remove the entry
    → Set a "tombstone" flag: visible = False
    → Entry stays in the data structure forever (until GC)
    → Why? Because a concurrent insert might reference it as a neighbor
    → If we remove it, the insert wouldn't know where to go

  GARBAGE COLLECTION:
    → When ALL connected clients have seen operation N
    → All tombstones from before N can be physically removed
    → Requires tracking "causal stability" across clients
    → Complex in practice with unreliable networks

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: OT TRANSFORM FUNCTION BUGS
  → The transform function has ~20 edge cases
  → Missing even ONE case → documents diverge
  → Example: Insert at exactly the boundary of a delete range
  → FIX: Exhaustive property-based testing
    Generate random operations, apply in both orders,
    verify convergence. Run millions of tests.

PITFALL 2: OPERATION EXPLOSION DURING NETWORK PARTITION
  → Client goes offline for 5 minutes, types 500 characters
  → Comes back with 500 operations to transform
  → Server has 300 other operations since then
  → Transform 500 × 300 = 150,000 transform calls
  → FIX: Compose client operations into fewer, larger operations
  → FIX: Send document snapshot instead of transforming if gap too large

PITFALL 3: CRDT METADATA BLOAT
  → 10-page document = ~50,000 characters
  → Each character: ~50 bytes metadata (position ID, tombstone flag)
  → Document text: ~50KB, CRDT metadata: ~2.5MB (50x overhead!)
  → With tombstones from editing: could be 10MB+
  → FIX: Periodic compaction / snapshotting
  → FIX: Run-length encoding for consecutive characters by same user

PITFALL 4: CURSOR POSITION INVALIDATION
  → User A has cursor at position 10
  → User B deletes characters 5-8 (4 characters)
  → User A's cursor should now be at position 6 (shifted left by 4)
  → If not transformed → cursor jumps to wrong location
  → FIX: Transform cursor positions using the same OT engine

PITFALL 5: UNDO ACROSS CONCURRENT EDITS
  → User A: types "Hello" (Op1), types " World" (Op2)
  → User B: types "!" at end (Op3, between A's Op1 and Op2)
  → User A presses Undo → should undo Op2 (" World")
  → But Op3 was applied between Op1 and Op2 on server
  → Undo of Op2 must be TRANSFORMED against Op3
  → FIX: Undo = generate inverse operation, transform it, apply it
  → NOT: simply revert to previous state

PITFALL 6: LATE JOINER SYNCHRONIZATION
  → Document has 10,000 operations in history
  → New user opens the document
  → Don't replay all 10,000 operations!
  → FIX: Send latest snapshot + only operations AFTER that snapshot
  → Snapshot every 100-500 operations

PITFALL 7: SPLIT-BRAIN IN SERVER CLUSTER
  → Project 26 (Leader Election) is relevant here
  → If two servers both think they're the OT authority
  → Two different transform sequences → document diverges
  → FIX: Single leader for OT processing per document
  → Use leader election (Project 26) per document partition

PITFALL 8: CHARACTER ENCODING ISSUES
  → Emoji "👋" is 2 UTF-16 code units but 1 visible character
  → Position 5 in UTF-8 ≠ position 5 in UTF-16
  → Client (JavaScript) uses UTF-16, server (Python) uses UTF-8
  → Operations reference different positions for same character!
  → FIX: Standardize on one encoding for positions
  → Or use character indices, not byte offsets

# SECTION 1: OPERATION REPRESENTATION
# Operations describe document mutations as composable units
# ================================================================

class OpType(Enum):
    RETAIN = "retain"
    INSERT = "insert"
    DELETE = "delete"


@dataclass
class OpComponent:
    op_type: OpType
    count: int = 0       # For RETAIN and DELETE
    text: str = ""       # For INSERT

    def __repr__(self):
        if self.op_type == OpType.RETAIN:
            return f"Retain({self.count})"
        elif self.op_type == OpType.INSERT:
            return f"Insert('{self.text}')"
        elif self.op_type == OpType.DELETE:
            return f"Delete({self.count})"


class Operation:
    """
    An operation is a sequence of components that transforms a document.
    
    The operation must "consume" the entire input document:
      sum(retain counts) + sum(delete counts) = input document length
    
    The output length:
      sum(retain counts) + sum(insert lengths) = output document length
    """

    def __init__(self, components: List[OpComponent] = None):
        self.components = components or []
        self.metadata = {}

    def retain(self, count: int):
        if count > 0:
            self.components.append(OpComponent(OpType.RETAIN, count=count))
        return self

    def insert(self, text: str):
        if text:
            self.components.append(OpComponent(OpType.INSERT, text=text))
        return self

    def delete(self, count: int):
        if count > 0:
            self.components.append(OpComponent(OpType.DELETE, count=count))
        return self

    def input_length(self) -> int:
        """Length of document this operation expects as input"""
        total = 0
        for c in self.components:
            if c.op_type == OpType.RETAIN:
                total += c.count
            elif c.op_type == OpType.DELETE:
                total += c.count
        return total

    def output_length(self) -> int:
        """Length of document after this operation is applied"""
        total = 0
        for c in self.components:
            if c.op_type == OpType.RETAIN:
                total += c.count
            elif c.op_type == OpType.INSERT:
                total += len(c.text)
        return total

    def apply(self, document: str) -> str:
        """Apply this operation to a document string"""
        if len(document) != self.input_length():
            raise ValueError(
                f"Operation expects document length {self.input_length()}, "
                f"got {len(document)}"
            )

        result = []
        doc_index = 0

        for comp in self.components:
            if comp.op_type == OpType.RETAIN:
                result.append(document[doc_index:doc_index + comp.count])
                doc_index += comp.count
            elif comp.op_type == OpType.INSERT:
                result.append(comp.text)
            elif comp.op_type == OpType.DELETE:
                doc_index += comp.count  # Skip deleted characters

        return "".join(result)

    def __repr__(self):
        return f"Op({self.components})"

# SECTION 2: OT TRANSFORM ENGINE
# The mathematical heart — transforms concurrent operations
# ================================================================

class OTEngine:
    """
    Operational Transform engine.
    
    transform(opA, opB) → (opA', opB')
    
    Property: apply(apply(doc, opA), opB') == apply(apply(doc, opB), opA')
    
    Both opA and opB were created against the same document state.
    opA' is opA adjusted to apply after opB.
    opB' is opB adjusted to apply after opA.
    """

    @staticmethod
    def transform(
        op_a: Operation, op_b: Operation
    ) -> Tuple[Operation, Operation]:
        """
        Transform two concurrent operations.
        Returns (op_a_prime, op_b_prime).
        """
        if op_a.input_length() != op_b.input_length():
            raise ValueError(
                f"Operations must have same input length: "
                f"{op_a.input_length()} vs {op_b.input_length()}"
            )

        # Iterators over components
        a_comps = list(op_a.components)
        b_comps = list(op_b.components)
        a_idx, b_idx = 0, 0
        a_offset, b_offset = 0, 0  # Partial consumption within a component

        a_prime = Operation()
        b_prime = Operation()

        while a_idx < len(a_comps) or b_idx < len(b_comps):
            a_comp = a_comps[a_idx] if a_idx < len(a_comps) else None
            b_comp = b_comps[b_idx] if b_idx < len(b_comps) else None

            # Case 1: A inserts (inserts go first — they don't consume input)
            if a_comp and a_comp.op_type == OpType.INSERT:
                insert_len = len(a_comp.text) - a_offset
                a_prime.insert(a_comp.text[a_offset:])
                b_prime.retain(insert_len)
                a_idx += 1
                a_offset = 0
                continue

            # Case 2: B inserts
            if b_comp and b_comp.op_type == OpType.INSERT:
                insert_len = len(b_comp.text) - b_offset
                b_prime.insert(b_comp.text[b_offset:])
                a_prime.retain(insert_len)
                b_idx += 1
                b_offset = 0
                continue

            # Both must be retain or delete from here
            if a_comp is None or b_comp is None:
                break

            a_remaining = (a_comp.count - a_offset) if a_comp.op_type != OpType.INSERT else 0
            b_remaining = (b_comp.count - b_offset) if b_comp.op_type != OpType.INSERT else 0
            min_len = min(a_remaining, b_remaining)

            if min_len == 0:
                break

            # Case 3: Both retain
            if a_comp.op_type == OpType.RETAIN and b_comp.op_type == OpType.RETAIN:
                a_prime.retain(min_len)
                b_prime.retain(min_len)

            # Case 4: A deletes, B retains
            elif a_comp.op_type == OpType.DELETE and b_comp.op_type == OpType.RETAIN:
                a_prime.delete(min_len)
                # B's retain is consumed by A's delete — B skips those chars

            # Case 5: A retains, B deletes
            elif a_comp.op_type == OpType.RETAIN and b_comp.op_type == OpType.DELETE:
                b_prime.delete(min_len)
                # A's retain is consumed by B's delete — A skips those chars

            # Case 6: Both delete same region
            elif a_comp.op_type == OpType.DELETE and b_comp.op_type == OpType.DELETE:
                # Both delete the same characters — neither needs to in the prime
                pass

            # Advance offsets
            a_offset += min_len
            b_offset += min_len

            if a_offset >= a_comp.count if a_comp.op_type != OpType.INSERT else a_offset >= len(a_comp.text):
                a_idx += 1
                a_offset = 0
            if b_offset >= b_comp.count if b_comp.op_type != OpType.INSERT else b_offset >= len(b_comp.text):
                b_idx += 1
                b_offset = 0

        return a_prime, b_prime

    @staticmethod
    def transform_cursor(cursor_pos: int, operation: Operation) -> int:
        """Transform a cursor position through an operation"""
        new_pos = cursor_pos
        current_pos = 0

        for comp in operation.components:
            if comp.op_type == OpType.RETAIN:
                current_pos += comp.count
            elif comp.op_type == OpType.INSERT:
                if current_pos <= cursor_pos:
                    new_pos += len(comp.text)
                current_pos += 0  # Insert doesn't consume input
            elif comp.op_type == OpType.DELETE:
                if current_pos < cursor_pos:
                    deleted_before = min(comp.count, cursor_pos - current_pos)
                    new_pos -= deleted_before
                current_pos += comp.count

        return max(0, new_pos)

# SECTION 3: OT SERVER (Collaboration Hub)
# Central authority for operation ordering and transformation

class OTServer:
    """
    Server maintains the canonical document state.
    All operations are transformed and applied through the server.
    Server assigns revision numbers (total ordering).
    """

    def __init__(self, document: str = ""):
        self.document = document
        self.revision = 0
        self.history = []           # List of applied operations
        self.lock = threading.Lock()
        self.clients = {}           # client_id → callback function
        self.snapshots = {}         # revision → document snapshot

        # Take initial snapshot
        self.snapshots[0] = document

    def register_client(self, client_id: str, callback):
        """Register a client to receive operation broadcasts"""
        with self.lock:
            self.clients[client_id] = callback

    def unregister_client(self, client_id: str):
        with self.lock:
            self.clients.pop(client_id, None)

    def receive_operation(
        self, client_id: str, client_revision: int, operation: Operation
    ) -> dict:
        """
        Receive an operation from a client.
        
        client_revision: the revision the client was at when they created this op.
        We must transform against all operations from client_revision to current.
        """
        with self.lock:
            # Transform against all operations the client hasn't seen
            transformed_op = operation
            for i in range(client_revision, self.revision):
                historical_op = self.history[i]
                transformed_op, _ = OTEngine.transform(transformed_op, historical_op)

            # Apply to server document
            try:
                self.document = transformed_op.apply(self.document)
            except ValueError as e:
                return {"success": False, "error": str(e)}

            # Record in history
            self.history.append(transformed_op)
            self.revision += 1

            # Snapshot every 100 revisions
            if self.revision % 100 == 0:
                self.snapshots[self.revision] = self.document

            # Broadcast to ALL clients
            broadcast_data = {
                "type": "operation",
                "client_id": client_id,
                "revision": self.revision,
                "operation": transformed_op,
            }

            for cid, callback in self.clients.items():
                try:
                    callback(broadcast_data)
                except Exception:
                    pass

            return {
                "success": True,
                "revision": self.revision,
                "transformed_op": transformed_op,
            }

    def get_document(self) -> str:
        with self.lock:
            return self.document

    def get_revision(self) -> int:
        with self.lock:
            return self.revision

    def get_snapshot_for_new_client(self) -> dict:
        """New client joining gets latest snapshot + recent ops"""
        with self.lock:
            latest_snapshot_rev = max(self.snapshots.keys())
            recent_ops = self.history[latest_snapshot_rev:]
            return {
                "snapshot": self.snapshots[latest_snapshot_rev],
                "snapshot_revision": latest_snapshot_rev,
                "recent_operations": recent_ops,
                "current_revision": self.revision,
            }

    def get_document_hash(self) -> str:
        with self.lock:
            return hashlib.md5(self.document.encode()).hexdigest()

# SECTION 4: OT CLIENT (Editor Instance)
# Implements the three-state client protocol

class ClientState(Enum):
    SYNCHRONIZED = "SYNCHRONIZED"
    AWAITING_CONFIRM = "AWAITING_CONFIRM"
    AWAITING_WITH_BUFFER = "AWAITING_WITH_BUFFER"


class OTClient:
    """
    Client-side OT implementation.
    
    Three states:
    1. SYNCHRONIZED — no pending ops, matches server
    2. AWAITING_CONFIRM — sent an op, waiting for ACK
    3. AWAITING_WITH_BUFFER — awaiting ACK + has buffered new ops
    """

    def __init__(self, client_id: str, server: OTServer, document: str = ""):
        self.client_id = client_id
        self.server = server
        self.document = document
        self.revision = 0  # Last known server revision
        self.state = ClientState.SYNCHRONIZED
        self.pending_op = None   # Operation sent to server, awaiting ACK
        self.buffer_op = None    # Buffered local operations not yet sent
        self.cursor_pos = 0
        self.edit_log = []       # Local history for debugging

        # Register with server for broadcasts
        self.server.register_client(client_id, self._on_server_broadcast)

    def local_insert(self, position: int, text: str):
        """User types text at position — apply locally, then sync"""
        # Build operation
        op = Operation()
        doc_len = len(self.document)
        if position > 0:
            op.retain(position)
        op.insert(text)
        if position < doc_len:
            op.retain(doc_len - position)

        # Apply locally (instant feedback)
        self.document = op.apply(self.document)
        self.cursor_pos = position + len(text)

        self.edit_log.append({
            "action": "insert",
            "position": position,
            "text": text,
            "state_before": self.state.value,
        })

        # Send or buffer based on current state
        self._handle_local_operation(op)

    def local_delete(self, position: int, length: int):
        """User deletes text — apply locally, then sync"""
        op = Operation()
        doc_len = len(self.document)
        if position > 0:
            op.retain(position)
        op.delete(length)
        remaining = doc_len - position - length
        if remaining > 0:
            op.retain(remaining)

        self.document = op.apply(self.document)
        self.cursor_pos = position

        self.edit_log.append({
            "action": "delete",
            "position": position,
            "length": length,
            "state_before": self.state.value,
        })

        self._handle_local_operation(op)

    def _handle_local_operation(self, op: Operation):
        """Route operation based on client state machine"""
        if self.state == ClientState.SYNCHRONIZED:
            # Send immediately
            self.pending_op = op
            self.state = ClientState.AWAITING_CONFIRM
            self._send_to_server(op)

        elif self.state == ClientState.AWAITING_CONFIRM:
            # Buffer it — can't send until previous is ACK'd
            self.buffer_op = self._compose_ops(self.buffer_op, op)
            self.state = ClientState.AWAITING_WITH_BUFFER

        elif self.state == ClientState.AWAITING_WITH_BUFFER:
            # Add to existing buffer
            self.buffer_op = self._compose_ops(self.buffer_op, op)

    def _send_to_server(self, op: Operation):
        """Send operation to server with current revision"""
        self.server.receive_operation(self.client_id, self.revision, op)

    def _on_server_broadcast(self, data: dict):
        """Called when server broadcasts an operation"""
        remote_op = data["operation"]
        source_client = data["client_id"]
        server_revision = data["revision"]

        if source_client == self.client_id:
            # This is our own operation being ACK'd
            self._handle_own_ack(server_revision)
        else:
            # This is someone else's operation
            self._handle_remote_operation(remote_op, server_revision)

    def _handle_own_ack(self, server_revision: int):
        """Server acknowledged our operation"""
        self.revision = server_revision

        if self.state == ClientState.AWAITING_CONFIRM:
            # No buffer — we're synchronized
            self.pending_op = None
            self.state = ClientState.SYNCHRONIZED

        elif self.state == ClientState.AWAITING_WITH_BUFFER:
            # Send the buffer now
            self.pending_op = self.buffer_op
            self.buffer_op = None
            self.state = ClientState.AWAITING_CONFIRM
            self._send_to_server(self.pending_op)

    def _handle_remote_operation(self, remote_op: Operation, server_revision: int):
        """Apply someone else's operation to our local state"""
        self.revision = server_revision

        if self.state == ClientState.SYNCHRONIZED:
            # Simple: just apply it
            self.document = remote_op.apply(self.document)
            self.cursor_pos = OTEngine.transform_cursor(self.cursor_pos, remote_op)

        elif self.state == ClientState.AWAITING_CONFIRM:
            # Transform our pending op against the remote op
            pending_prime, remote_prime = OTEngine.transform(
                self.pending_op, remote_op
            )
            self.pending_op = pending_prime
            self.document = remote_prime.apply(self.document)
            self.cursor_pos = OTEngine.transform_cursor(self.cursor_pos, remote_prime)

        elif self.state == ClientState.AWAITING_WITH_BUFFER:
            # Transform pending against remote
            pending_prime, remote_prime = OTEngine.transform(
                self.pending_op, remote_op
            )
            # Then transform buffer against the already-transformed remote
            buffer_prime, remote_double_prime = OTEngine.transform(
                self.buffer_op, remote_prime
            )
            self.pending_op = pending_prime
            self.buffer_op = buffer_prime
            self.document = remote_double_prime.apply(self.document)
            self.cursor_pos = OTEngine.transform_cursor(
                self.cursor_pos, remote_double_prime
            )

    def _compose_ops(self, existing_op: Optional[Operation], new_op: Operation) -> Operation:
        """
        Compose two sequential operations into one.
        For simplicity, we store them as a list and apply sequentially.
        In production, you'd implement proper operation composition.
        """
        if existing_op is None:
            return new_op
        # Simplified: create a wrapper that applies both
        composed = ComposedOperation(existing_op, new_op)
        return composed

    def get_document_hash(self) -> str:
        return hashlib.md5(self.document.encode()).hexdigest()


class ComposedOperation(Operation):
    """Two operations composed into one (simplified for demo)"""

    def __init__(self, op1: Operation, op2: Operation):
        super().__init__()
        self.op1 = op1
        self.op2 = op2
        # Build actual composed components
        self._compose()

    def _compose(self):
        """Apply op1, then op2 — merge into single operation sequence"""
        # For simplicity in demo, we flatten by tracking position changes
        # In production: proper compose algorithm
        self.components = self.op1.components.copy()
        # Adjust op2's components based on op1's output
        self._inner_op2 = self.op2

    def apply(self, document: str) -> str:
        intermediate = self.op1.apply(document)
        return self._inner_op2.apply(intermediate)

    def input_length(self) -> int:
        return self.op1.input_length()

    def output_length(self) -> int:
        return self._inner_op2.output_length()

# Alternative approach: no central transform authority needed

@dataclass
class CRDTChar:
    """A character in the CRDT sequence with unique identifier"""
    position_id: tuple    # (fractional_pos, site_id, clock)
    char: str             # The actual character
    visible: bool = True  # False = tombstone (deleted)
    author: str = ""
    timestamp: float = 0.0

    def __lt__(self, other):
        """Total ordering: by position, then site_id, then clock"""
        if self.position_id[0] != other.position_id[0]:
            return self.position_id[0] < other.position_id[0]
        if self.position_id[1] != other.position_id[1]:
            return self.position_id[1] < other.position_id[1]
        return self.position_id[2] < other.position_id[2]


class SequenceCRDT:
    """
    A CRDT-based collaborative text document.
    
    Key properties:
    → Every character has a unique, immutable position ID
    → Insert: generate ID between two neighbors
    → Delete: mark as tombstone (don't remove)
    → Merge: union of all characters, sorted by ID
    → Convergence: guaranteed by total ordering of IDs
    """

    def __init__(self, site_id: str):
        self.site_id = site_id
        self.clock = 0
        self.chars: List[CRDTChar] = []
        self.pending_ops = []
        self.applied_ops = []
        self.tombstone_count = 0

        # Boundary sentinels (position 0.0 and 1.0 are boundaries)
        self.MIN_POS = 0.0
        self.MAX_POS = 1.0

    def _next_clock(self) -> int:
        self.clock += 1
        return self.clock

    def _generate_position_between(
        self, pos_before: float, pos_after: float
    ) -> float:
        """
        Generate a fractional position between two existing positions.
        
        In production CRDT (LSEQ, Logoot):
        → Use variable-length identifiers to avoid precision loss
        → Use allocation strategies (random vs boundary+)
        
        For this demo: simple midpoint with jitter
        """
        # Midpoint with small random offset to reduce collision
        mid = (pos_before + pos_after) / 2.0

        # If positions are too close, add tiny jitter
        if abs(pos_after - pos_before) < 1e-10:
            mid = pos_before + random.uniform(1e-12, 1e-11)

        return mid

    def _visible_index_to_internal(self, visible_idx: int) -> int:
        """
        Convert a visible character index to internal array index.
        Must skip tombstones.
        """
        visible_count = 0
        for i, char in enumerate(self.chars):
            if char.visible:
                if visible_count == visible_idx:
                    return i
                visible_count += 1
        # Past the end — return length
        return len(self.chars)

    def _find_insert_position(self, internal_idx: int) -> int:
        """Find correct sorted position for insertion"""
        # Binary search would be better in production
        # Linear scan for correctness in demo
        return internal_idx

    def insert(self, visible_pos: int, text: str) -> list:
        """
        Insert text at visible position.
        Returns list of operations for broadcasting.
        """
        ops = []

        for i, char in enumerate(text):
            actual_insert_pos = visible_pos + i
            internal_idx = self._visible_index_to_internal(actual_insert_pos)

            # Find neighbor positions for ID generation
            if internal_idx == 0:
                pos_before = self.MIN_POS
            else:
                pos_before = self.chars[internal_idx - 1].position_id[0]

            if internal_idx >= len(self.chars):
                pos_after = self.MAX_POS
            else:
                pos_after = self.chars[internal_idx].position_id[0]

            # Generate unique position ID
            new_pos = self._generate_position_between(pos_before, pos_after)
            clock = self._next_clock()
            position_id = (new_pos, self.site_id, clock)

            # Create character
            new_char = CRDTChar(
                position_id=position_id,
                char=char,
                visible=True,
                author=self.site_id,
                timestamp=time.time(),
            )

            # Insert at correct sorted position
            inserted = False
            for j in range(len(self.chars)):
                if new_char < self.chars[j]:
                    self.chars.insert(j, new_char)
                    inserted = True
                    break
            if not inserted:
                self.chars.append(new_char)

            op = {
                "type": "insert",
                "position_id": position_id,
                "char": char,
                "author": self.site_id,
                "clock": clock,
            }
            ops.append(op)
            self.applied_ops.append(op)

        return ops

    def delete(self, visible_pos: int, length: int = 1) -> list:
        """
        Delete characters starting at visible position.
        Marks as tombstone — does NOT physically remove.
        Returns list of operations for broadcasting.
        """
        ops = []
        deleted = 0
        visible_count = 0

        for i, char in enumerate(self.chars):
            if not char.visible:
                continue

            if visible_count >= visible_pos and deleted < length:
                # Mark as tombstone
                char.visible = False
                self.tombstone_count += 1

                op = {
                    "type": "delete",
                    "position_id": char.position_id,
                    "author": self.site_id,
                    "clock": self._next_clock(),
                }
                ops.append(op)
                self.applied_ops.append(op)
                deleted += 1

            visible_count += 1

            if deleted >= length:
                break

        return ops

    def apply_remote_op(self, op: dict):
        """
        Apply an operation received from another site.
        
        For INSERT: create character and insert in sorted position.
        For DELETE: find character by position_id and tombstone it.
        
        ORDER DOESN'T MATTER — this is the CRDT guarantee!
        """
        if op["type"] == "insert":
            position_id = tuple(op["position_id"]) if isinstance(op["position_id"], list) else op["position_id"]

            # Check if already applied (idempotency)
            for char in self.chars:
                if char.position_id == position_id:
                    return  # Already have this character

            new_char = CRDTChar(
                position_id=position_id,
                char=op["char"],
                visible=True,
                author=op["author"],
                timestamp=time.time(),
            )

            # Insert in sorted position
            inserted = False
            for j in range(len(self.chars)):
                if new_char < self.chars[j]:
                    self.chars.insert(j, new_char)
                    inserted = True
                    break
            if not inserted:
                self.chars.append(new_char)

        elif op["type"] == "delete":
            position_id = tuple(op["position_id"]) if isinstance(op["position_id"], list) else op["position_id"]

            for char in self.chars:
                if char.position_id == position_id and char.visible:
                    char.visible = False
                    self.tombstone_count += 1
                    break

    def get_text(self) -> str:
        """Get visible document text (skip tombstones)"""
        return "".join(c.char for c in self.chars if c.visible)

    def get_text_hash(self) -> str:
        return hashlib.md5(self.get_text().encode()).hexdigest()

    def get_stats(self) -> dict:
        total = len(self.chars)
        visible = sum(1 for c in self.chars if c.visible)
        return {
            "total_entries": total,
            "visible_chars": visible,
            "tombstones": self.tombstone_count,
            "memory_overhead_ratio": f"{total / max(visible, 1):.2f}x",
            "ops_applied": len(self.applied_ops),
        }

    def garbage_collect(self, min_clock_across_all_sites: int):
        """
        Remove tombstones that ALL sites have seen.
        Only safe when we KNOW every site has processed past this point.
        """
        before_count = len(self.chars)
        self.chars = [
            c for c in self.chars
            if c.visible or c.position_id[2] > min_clock_across_all_sites
        ]
        removed = before_count - len(self.chars)
        self.tombstone_count -= removed
        return removed

# SECTION 6: CRDT COLLABORATION HUB
# Relays operations between CRDT clients (no transforms needed!)
# ================================================================

class CRDTCollaborationHub:
    """
    Unlike OT Server, this hub does NOT transform operations.
    It simply RELAYS them to all other clients.
    Each client merges independently — convergence is guaranteed by CRDT.
    """

    def __init__(self):
        self.clients = {}
        self.operation_log = []
        self.lock = threading.Lock()

    def register_client(self, site_id: str, crdt: SequenceCRDT):
        with self.lock:
            self.clients[site_id] = crdt

    def broadcast_ops(self, source_site: str, ops: list):
        """Relay operations from one client to all others"""
        with self.lock:
            self.operation_log.extend(ops)

            for site_id, crdt in self.clients.items():
                if site_id != source_site:
                    for op in ops:
                        crdt.apply_remote_op(op)

    def verify_convergence(self) -> dict:
        """Check that all clients have the same document"""
        with self.lock:
            texts = {}
            hashes = {}
            for site_id, crdt in self.clients.items():
                texts[site_id] = crdt.get_text()
                hashes[site_id] = crdt.get_text_hash()

            all_same = len(set(hashes.values())) <= 1
            return {
                "converged": all_same,
                "texts": texts,
                "hashes": hashes,
                "total_ops": len(self.operation_log),
            }

class OTConvergenceProof:
    """
    Property-based testing for OT correctness.
    
    For any two concurrent operations A and B on document D:
      apply(apply(D, A), transform(B, A).B') 
        == 
      apply(apply(D, B), transform(A, B).A')
    
    This is tested with random operations on random documents.
    """

    @staticmethod
    def generate_random_document(min_len: int = 5, max_len: int = 20) -> str:
        length = random.randint(min_len, max_len)
        chars = [chr(random.randint(65, 90)) for _ in range(length)]
        return "".join(chars)

    @staticmethod
    def generate_random_operation(doc_len: int) -> Operation:
        """Generate a random valid operation for a document of given length"""
        op = Operation()
        remaining = doc_len
        inserted_something = False

        while remaining > 0:
            action = random.choice(["retain", "delete", "insert"])

            if action == "retain":
                count = random.randint(1, remaining)
                op.retain(count)
                remaining -= count

            elif action == "delete":
                count = random.randint(1, min(3, remaining))
                op.delete(count)
                remaining -= count

            elif action == "insert":
                text = "".join(
                    chr(random.randint(97, 122))
                    for _ in range(random.randint(1, 3))
                )
                op.insert(text)
                inserted_something = True

        # Maybe insert at the end too
        if random.random() < 0.3:
            text = chr(random.randint(97, 122))
            op.insert(text)

        return op

    @staticmethod
    def run_convergence_test(num_tests: int = 500) -> dict:
        """Run randomized convergence tests"""
        passed = 0
        failed = 0
        errors = []

        for i in range(num_tests):
            try:
                doc = OTConvergenceProof.generate_random_document(3, 15)
                doc_len = len(doc)

                if doc_len == 0:
                    passed += 1
                    continue

                op_a = OTConvergenceProof.generate_random_operation(doc_len)
                op_b = OTConvergenceProof.generate_random_operation(doc_len)

                # Verify input lengths match
                if op_a.input_length() != doc_len or op_b.input_length() != doc_len:
                    continue

                # Transform
                op_a_prime, op_b_prime = OTEngine.transform(op_a, op_b)

                # Path 1: Apply A then B'
                doc_after_a = op_a.apply(doc)
                doc_path1 = op_b_prime.apply(doc_after_a)

                # Path 2: Apply B then A'
                doc_after_b = op_b.apply(doc)
                doc_path2 = op_a_prime.apply(doc_after_b)

                if doc_path1 == doc_path2:
                    passed += 1
                else:
                    failed += 1
                    if len(errors) < 5:
                        errors.append({
                            "test": i,
                            "document": doc,
                            "op_a": str(op_a),
                            "op_b": str(op_b),
                            "path1_result": doc_path1,
                            "path2_result": doc_path2,
                        })

            except Exception as e:
                # Some random operations may be invalid — skip
                continue

        return {
            "total_tests": num_tests,
            "passed": passed,
            "failed": failed,
            "pass_rate": f"{(passed / max(passed + failed, 1)) * 100:.1f}%",
            "sample_errors": errors,
        }

class CRDTConvergenceProof:

    Operations can be applied in ANY ORDER and all replicas converge.
    """

    @staticmethod
    def run_order_independence_test(num_tests: int = 100) -> dict:
        """
        Generate operations, apply in different orders,
        verify all replicas end up with same document.
        """
        passed = 0
        failed = 0
        errors = []

        for test_num in range(num_tests):
            try:
                # Create 3 independent replicas
                crdt_a = SequenceCRDT("site-A")
                crdt_b = SequenceCRDT("site-B")
                crdt_c = SequenceCRDT("site-C")

                # Start with same initial text
                initial_text = "Hello"
                ops_init = crdt_a.insert(0, initial_text)
                for op in ops_init:
                    crdt_b.apply_remote_op(op)
                    crdt_c.apply_remote_op(op)

                # Each site makes concurrent edits
                ops_from_a = crdt_a.insert(5, " World")
                ops_from_b = crdt_b.insert(0, "Say: ")
                ops_from_c = crdt_c.insert(5, "!")

                # Apply in DIFFERENT orders to each site

                # Site A receives: B's ops, then C's ops
                for op in ops_from_b:
                    crdt_a.apply_remote_op(op)
                for op in ops_from_c:
                    crdt_a.apply_remote_op(op)

                # Site B receives: C's ops, then A's ops (reversed!)
                for op in ops_from_c:
                    crdt_b.apply_remote_op(op)
                for op in ops_from_a:
                    crdt_b.apply_remote_op(op)

                # Site C receives: A's ops, then B's ops (another order!)
                for op in ops_from_a:
                    crdt_c.apply_remote_op(op)
                for op in ops_from_b:
                    crdt_c.apply_remote_op(op)

                # ALL THREE must have the SAME text
                text_a = crdt_a.get_text()
                text_b = crdt_b.get_text()
                text_c = crdt_c.get_text()

                if text_a == text_b == text_c:
                    passed += 1
                else:
                    failed += 1
                    if len(errors) < 3:
                        errors.append({
                            "test": test_num,
                            "site_a": text_a,
                            "site_b": text_b,
                            "site_c": text_c,
                        })

            except Exception as e:
                failed += 1
                if len(errors) < 3:
                    errors.append({"test": test_num, "error": str(e)})

        return {
            "total_tests": num_tests,
            "passed": passed,
            "failed": failed,
            "pass_rate": f"{(passed / max(passed + failed, 1)) * 100:.1f}%",
            "sample_errors": errors,
        }

class DocumentHistoryManager:
    """
    Manages document snapshots and operation replay.
    New clients don't replay entire history — they load
    latest snapshot + recent operations.
    """

    def __init__(self, snapshot_interval: int = 5):
        self.snapshots = {}           # revision → document text
        self.operations = []          # full operation log
        self.snapshot_interval = snapshot_interval
        self.current_revision = 0
        self.document = ""

    def initialize(self, text: str):
        self.document = text
        self.snapshots[0] = text

    def apply_operation(self, op: Operation, author: str) -> int:
        """Apply operation and manage snapshots"""
        self.document = op.apply(self.document)
        self.current_revision += 1

        self.operations.append({
            "revision": self.current_revision,
            "operation": op,
            "author": author,
            "timestamp": time.time(),
        })

        # Snapshot periodically
        if self.current_revision % self.snapshot_interval == 0:
            self.snapshots[self.current_revision] = self.document

        return self.current_revision

    def get_state_for_new_client(self) -> dict:
        """Efficient state transfer for late joiners"""
        latest_snapshot_rev = max(self.snapshots.keys())
        recent_ops = [
            op for op in self.operations
            if op["revision"] > latest_snapshot_rev
        ]

        return {
            "snapshot_text": self.snapshots[latest_snapshot_rev],
            "snapshot_revision": latest_snapshot_rev,
            "recent_ops_count": len(recent_ops),
            "current_revision": self.current_revision,
            "total_ops_in_history": len(self.operations),
            "total_snapshots": len(self.snapshots),
        }

    def time_travel(self, target_revision: int) -> Optional[str]:
        """
        Reconstruct document at any past revision.
        Find nearest snapshot before target, replay ops.
        """
        # Find nearest snapshot at or before target
        valid_snapshots = [
            rev for rev in self.snapshots.keys()
            if rev <= target_revision
        ]

        if not valid_snapshots:
            return None

        start_rev = max(valid_snapshots)
        doc = self.snapshots[start_rev]

        # Replay operations from snapshot to target
        for entry in self.operations:
            if start_rev < entry["revision"] <= target_revision:
                doc = entry["operation"].apply(doc)

        return doc

    @staticmethod
    def run_demo():
        print("\n" + "=" * 70)
        print("  ⏰ DOCUMENT HISTORY & TIME TRAVEL DEMO")
        print("  Event Sourcing applied to collaborative editing (Project 29)")
        print("=" * 70)

        manager = DocumentHistoryManager(snapshot_interval=3)
        manager.initialize("Hello")

        # Simulate a series of edits
        edits = [
            ("Alice", lambda d: Operation().retain(5).insert(" World"),
             "Added ' World'"),
            ("Bob", lambda d: Operation().retain(len(d)).insert("!"),
             "Added '!'"),
            ("Alice", lambda d: Operation().retain(len(d)).insert(" How are you"),
             "Added ' How are you'"),
            ("Charlie", lambda d: Operation().insert("Say: ").retain(len(d)),
             "Added 'Say: ' at start"),
            ("Bob", lambda d: Operation().retain(len(d)).insert("?"),
             "Added '?'"),
            ("Alice", lambda d: Operation().retain(len(d)).insert(" Today"),
             "Added ' Today'"),
        ]

        print(f"\n  Applying edits with snapshots every 3 operations:")
        print(f"    Rev 0: \"{manager.document}\"")

        for author, op_fn, description in edits:
            op = op_fn(manager.document)
            rev = manager.apply_operation(op, author)
            snapshot_marker = " 📸 SNAPSHOT" if rev in manager.snapshots else ""
            print(f"    Rev {rev}: \"{manager.document}\" ({author}: {description}){snapshot_marker}")

        # Show state for new client
        print(f"\n  State for new client joining now:")
        state = manager.get_state_for_new_client()
        for key, value in state.items():
            print(f"    {key}: {value}")

        # Time travel
        print(f"\n  Time Travel — reconstructing past states:")
        for rev in range(0, manager.current_revision + 1):
            doc = manager.time_travel(rev)
            snapshot_marker = " 📸" if rev in manager.snapshots else ""
            print(f"    Rev {rev}: \"{doc}\"{snapshot_marker}")

        print(f"\n  💡 INSIGHT:")
        print(f"     Without snapshots: must replay ALL {len(manager.operations)} operations from empty doc")
        print(f"     With snapshots: load snapshot + replay only {state['recent_ops_count']} recent ops")
        print(f"     At scale (10,000 ops): replay 10,000 vs replay ~{manager.snapshot_interval}")
        print(f"{'=' * 70}")
