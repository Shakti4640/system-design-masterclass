PROJECT 21: THE CIRCUIT BREAKER — RESILIENCE IN DISTRIBUTED CALLS

📋 PHASE 3 (Medium) — Professional Techniques

🎯 HYPOTHETICAL PROBLEM

You are the lead engineer at a food delivery platform
Everything was working perfectly after Project 20

Your microservice architecture:
  → Order Service calls Payment Service (discovered via registry)
  → Payment Service calls Bank API (external)
  → Order Service calls Restaurant Service
  → Order Service calls Notification Service

TUESDAY 2:47 AM — DISASTER STRIKES:

  The Bank API that Payment Service depends on goes DOWN
  
  What happens next is a CHAIN REACTION:

  SECOND 0:
    → Bank API stops responding
    → Payment Service calls Bank API... waits... waits... TIMEOUT (30s)
    
  SECOND 5:
    → 50 more payment requests arrive
    → Each one opens a connection to Bank API
    → Each one WAITS for 30 seconds
    → Payment Service now has 50 threads blocked
    
  SECOND 30:
    → First batch of requests timeout
    → 200 more requests have piled up
    → Payment Service thread pool is EXHAUSTED (all 200 threads waiting)
    → Payment Service can't accept ANY new connections
    → Payment Service appears DEAD
    
  SECOND 35:
    → Order Service tries to call Payment Service
    → Payment Service doesn't respond (all threads busy)
    → Order Service waits... waits... TIMEOUT
    → Order Service threads start filling up too
    
  SECOND 60:
    → Order Service thread pool is 80% exhausted
    → Restaurant Service calls start timing out too
    → Notification Service calls start timing out
    → Order Service is effectively DEAD
    
  SECOND 90:
    → The API Gateway (Project 19) tries to route to Order Service
    → Order Service doesn't respond
    → Customer-facing app shows "Something went wrong"
    → ALL customers affected
    → Your entire platform is DOWN
    
  THE CAUSE:
    → One external API (Bank) went down
    → But it took down Payment → Order → EVERYTHING
    
  THIS IS CALLED: CASCADING FAILURE
    → One domino falls → knocks down the next → knocks down ALL

  THE QUESTION:
    → How do you STOP the cascade?
    → How do you PROTECT healthy services from sick ones?
    → How do you FAIL FAST instead of waiting and dying slowly?

  THE ANSWER: Circuit Breaker Pattern

🔍 HOW TO APPROACH IT

Think of it like ELECTRICAL CIRCUIT BREAKERS in your house:

WITHOUT circuit breaker:
  → Power surge on one circuit
  → Surge travels through all wires
  → Every appliance in the house fries
  → Total destruction

WITH circuit breaker:
  → Power surge on one circuit
  → Breaker DETECTS overcurrent
  → Breaker TRIPS (opens the circuit)
  → Surge is ISOLATED to that one circuit
  → Rest of the house is FINE
  → You reset the breaker when the problem is fixed

MAP TO MICROSERVICES:

  Power Surge         = Service failure / slow response
  Electrical Circuit   = Network connection between services
  Circuit Breaker      = Software component that monitors calls
  Tripping             = Stopping calls to the failing service
  Rest of the House    = Other healthy services
  Resetting            = Testing if failed service recovered

THE THREE STATES:

  │                                                         │
  │   ┌──────────┐    failures    ┌──────────┐             │
  │   │          │  exceed limit  │          │             │
  │   │  CLOSED  │───────────────→│   OPEN   │             │
  │   │ (normal) │                │ (blocked)│             │
  │   │          │←───────────────│          │             │
  │   └──────────┘   success in   └────┬─────┘             │
  │        ↑         half-open         │                   │
  │        │                           │ after timeout     │
  │        │                           ↓                   │
  │        │         ┌────────────────────┐                │
  │        │         │                    │                │
  │        └─────────│    HALF-OPEN       │                │
  │         success  │  (testing)         │                │

  │                                           again)       │
  └─────────────────────────────────────────────────────────┘

  CLOSED:    Everything normal, requests pass through
  OPEN:      Service is broken, ALL requests blocked immediately
  HALF-OPEN: Cautiously testing if service recovered

                      │   FALLBACK      │

    → Without breaker: Fail in 30 seconds (timeout)

  Order Service has:

    │ Breaker: payment    │ → monitors calls to Payment Service
    ├─────────────────────┤

📌 WHAT COULD BE DONE

OPTION A: Simple Consecutive Failure Counter
  → Count consecutive failures
  → After N consecutive failures → OPEN the breaker
  → One success resets the counter
  → PROBLEM: One lucky success keeps breaker closed
  → USE: Simple cases with clear fail/success

OPTION B: Failure Rate Window (Time-Based)
  → Track success/failure in a sliding time window
  → If failure rate > threshold (e.g., 50%) → OPEN
  → Minimum call volume required before evaluating
  → USE: Production systems — THIS is the standard

OPTION C: Slow Call Rate Detection
  → Track not just failures but SLOW responses
  → If >60% of calls take >5 seconds → OPEN
  → Catches degraded services before they fully fail
  → USE: Combined with Option B for comprehensive protection

OPTION D: Per-Instance Breakers
  → Instead of one breaker per service NAME
  → One breaker per service INSTANCE
  → payment-1 fails → breaker for payment-1 opens
  → payment-2 still reachable through its breaker
  → Combines with Project 20 (Discovery) for smart routing
  → USE: Large-scale systems with many instances

FOR THIS PROJECT → We build Option B + Option C + Option D
  Complete production-grade circuit breaker

✅ WHAT SHOULD DO

1. CONFIGURE thresholds carefully
   → failure_rate_threshold: 50% is common default
   → slow_call_rate_threshold: 80% calls exceeding duration
   → slow_call_duration: depends on service SLA (e.g., 3 seconds)
   → minimum_calls: don't evaluate with < 10 calls (statistical noise)
   → window_size: 10-60 seconds of data

2. ALWAYS provide a fallback
   → Breaker opens → what does the caller get?
   → Option 1: Cached last-known-good response
   → Option 2: Sensible default value
   → Option 3: Queue the request for later (Project 11)
   → Option 4: Degraded response ("payment pending, will confirm later")
   → NEVER just throw an error with no recovery path

3. IMPLEMENT half-open correctly
   → After timeout → allow LIMITED test requests
   → Not ALL requests — just 1-3 probe requests
   → If probes succeed → close breaker
   → If probes fail → re-open breaker, extend timeout
   → Exponential backoff on repeated failures

4. EMIT events and metrics
   → Breaker state changes → emit event
   → "payment-breaker: CLOSED → OPEN at 14:32:05"
   → Track: open count, half-open transitions, recovery time
   → Alert when breaker opens (something is wrong)
   → Alert when breaker stays open > 5 minutes (manual intervention)

5. COMBINE with retry logic
   → Retry BEFORE the breaker trips (transient errors)
   → But limit retries (3 max with exponential backoff)
   → Retries that fail → count toward breaker failure rate
   → After breaker opens → NO retries (instant fail)

6. USE per-instance breakers with discovery
   → Discover 3 instances (Project 20)
   → Instance A breaker open → skip A → try Instance B
   → All breakers open → return fallback
   → Maximizes availability

❌ WHAT SHOULD NOT DO

1. NEVER set breaker thresholds too sensitive
   → 1 failure = OPEN? Too aggressive
   → Normal network has occasional failures
   → Minimum 5-10 calls before evaluating
   → 50% failure rate is reasonable, not 10%
   → Too sensitive = breaker flapping constantly

2. NEVER set breaker thresholds too lenient
   → 95% failure rate before opening? Too late
   → By then, thread pool is already exhausted
   → The damage is already done
   → Find the balance: 50-70% failure rate

3. NEVER forget the half-open timeout
   → Breaker opens → stays open FOREVER?
   → Service might have recovered hours ago
   → Always implement recovery path
   → Typical: 30-60 seconds open, then test

4. NEVER make the fallback ALSO call a failing service
   → Breaker opens for Payment Service
   → Fallback: "Let me try the BACKUP Payment Service"
   → Backup is also down (same root cause)
   → Now fallback also fails → no protection
   → Fallback must be LOCAL (cache, default, queue)

5. NEVER ignore slow calls
   → Service responds... but takes 29 seconds
   → Technically "successful" (200 status)
   → But consuming threads for 29 seconds each
   → Slow calls are WORSE than fast failures
   → Monitor and circuit-break on slow calls too

6. NEVER share breaker state across unrelated services
   → Payment breaker is for Payment calls ONLY
   → Don't let Payment failures affect Restaurant breaker
   → Each dependency gets its OWN breaker
   → Isolation is the entire point

7. NEVER retry after breaker is OPEN
   → Breaker is open = "stop calling, it's broken"
   → Retrying defeats the purpose
   → Retries belong INSIDE the breaker (before it trips)
   → Once tripped → instant fallback, zero retries

🚫 WHEN NOT TO USE

1. Calling local functions (same process)
   → No network involved
   → Function either works or throws exception
   → Use try/catch, not circuit breaker
   → Breaker is for REMOTE calls

2. Fire-and-forget async messages
   → You put message on a queue (Project 11)
   → Queue is durable, message will be processed eventually
   → No synchronous wait → no thread exhaustion risk
   → Queue itself might need a breaker though

3. One-time batch jobs
   → Run once, process all items, exit
   → If external service is down, just wait and retry
   → No continuous traffic to protect
   → Simple retry loop is sufficient

4. Services with guaranteed SLAs and proven reliability
   → Internal service with 99.999% uptime
   → Overhead of breaker not justified
   → HOWEVER: still good practice (things change)
   → "It's always worked" is famous last words

5. When the caller IS the fallback
   → Service A is the primary AND the fallback
   → Breaker opens → fallback calls same service → loops
   → Restructure: separate the fallback logic

COMPONENT 1: SLIDING WINDOW — TRACKING CALL OUTCOMES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Two types of sliding windows:

A) COUNT-BASED WINDOW:
   → Track the last N calls (e.g., last 20 calls)
   → Ring buffer of outcomes: [S, S, F, S, F, F, S, S, F, S, ...]
   → Calculate failure rate from this buffer
   → New call → overwrite oldest entry

   Example (window_size=10):
   [S, S, F, S, F, F, S, S, F, S]
    ↑                           ↑
   oldest                     newest
   
   Failures: 4/10 = 40% → below 50% threshold → CLOSED
   
   Next call fails:
   [S, F, S, F, F, S, S, F, S, F]  → 5/10 = 50% → STILL CLOSED
   
   Next call fails:
   [F, S, F, F, S, S, F, S, F, F]  → 6/10 = 60% → OPEN!

B) TIME-BASED WINDOW:
   → Track all calls in the last N seconds (e.g., last 30s)
   → Divide into buckets (e.g., 10 buckets of 3 seconds each)
   → Each bucket counts successes, failures, slow calls
   → Window slides: oldest bucket drops off, new bucket added

   ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
   │ B1  │ B2  │ B3  │ B4  │ B5  │ B6  │ B7  │ B8  │ B9  │ B10 │
   │ 3s  │ 3s  │ 3s  │ 3s  │ 3s  │ 3s  │ 3s  │ 3s  │ 3s  │ 3s  │
   │S:5  │S:3  │S:4  │S:2  │S:1  │S:0  │S:1  │S:0  │S:0  │S:1  │
   │F:0  │F:1  │F:0  │F:2  │F:4  │F:5  │F:3  │F:6  │F:5  │F:4  │
   └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
   ← 30 seconds ago                              NOW →
   
   Total: Success=17, Failure=30
   Failure Rate: 30/47 = 63.8% → exceeds 50% → OPEN!

COMPONENT 2: STATE MACHINE — THE BRAIN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  STATE: CLOSED (normal operation)
  ┌──────────────────────────────────────────────┐
  │ Every call passes through to the target      │
  │                                              │
  │ On SUCCESS:                                  │
  │   → Record success in sliding window         │
  │   → Return result to caller                  │
  │                                              │
  │ On FAILURE:                                  │
  │   → Record failure in sliding window         │
  │   → Return error to caller                   │
  │   → Check: failure_rate > threshold?          │
  │     → YES and min_calls met → transition OPEN│
  │     → NO → stay CLOSED                       │
  │                                              │
  │ On SLOW CALL (success but > duration limit): │
  │   → Record as slow in sliding window         │
  │   → Return result to caller (it worked)      │
  │   → Check: slow_rate > slow_threshold?        │
  │     → YES → transition OPEN                  │
  └──────────────────────────────────────────────┘
              │
              │ failure_rate > threshold
              ▼
  STATE: OPEN (blocking calls)
  ┌──────────────────────────────────────────────┐
  │ NO calls pass through                        │
  │ All requests rejected IMMEDIATELY             │
  │                                              │
  │ On ANY call:                                 │
  │   → DON'T call the target service            │
  │   → Return fallback immediately              │
  │   → Record as "not_permitted"                │
  │   → Increment rejected counter               │
  │                                              │
  │ Timer running:                               │
  │   → Wait for open_timeout (e.g., 30 seconds) │
  │   → When timeout expires → transition HALF_OPEN│
  │                                              │
  │ WHY this matters:                            │
  │   → 1000 requests in 30 seconds              │
  │   → All fail in <1ms (instant rejection)     │
  │   → vs 30 seconds each without breaker       │
  │   → Saves: 1000 × 30s = 30,000 seconds       │
  │     of thread time                            │
  └──────────────────────────────────────────────┘
              │
              │ open_timeout expires
              ▼
  STATE: HALF-OPEN (cautious testing)
  ┌──────────────────────────────────────────────┐
  │ Allow LIMITED test requests through           │
  │ (e.g., max 3 concurrent probe requests)      │
  │                                              │
  │ On PROBE SUCCESS:                            │
  │   → Increment success counter                │
  │   → If success_count >= required_probes      │
  │     → Service recovered!                     │
  │     → Transition to CLOSED                   │
  │     → Reset sliding window (fresh start)     │
  │                                              │
  │ On PROBE FAILURE:                            │
  │   → Service still broken                     │
  │   → Transition back to OPEN                  │
  │   → Increase open_timeout (exponential)      │
  │     → First: 30s, then 60s, then 120s...     │
  │     → Prevents hammering a sick service      │
  │                                              │
  │ EXCESS calls (beyond probe limit):           │
  │   → Still rejected with fallback             │
  │   → Only probe requests get through          │
  └──────────────────────────────────────────────┘

COMPONENT 3: FALLBACK CHAIN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  When breaker is OPEN, what do we return?

  STRATEGY 1: Cached Response
    → Last successful response for this request type
    → "Here's the payment status from 5 minutes ago"
    → Best for READ operations

  STRATEGY 2: Default Value
    → Predefined safe default
    → "Payment status: PENDING (unable to confirm)"
    → Best for non-critical information

  STRATEGY 3: Queue for Later (Project 11)
    → Put the request on a message queue
    → Process when service recovers
    → "Payment will be processed shortly"
    → Best for WRITE operations

  STRATEGY 4: Alternative Service
    → Call a DIFFERENT service that can handle it
    → Backup payment processor
    → Must be truly independent (not same failure domain)

  STRATEGY 5: Graceful Degradation
    → Skip this step entirely
    → "Order placed, payment will be confirmed via email"
    → Continue the flow without the broken piece

  FALLBACK CHAIN (try in order):
    Cache → Default → Queue → Degrade → Error

COMPONENT 4: BULKHEAD PATTERN (Companion to Circuit Breaker)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Problem: Even BEFORE breaker trips, slow calls consume threads

  WITHOUT bulkhead:
    ┌──────────────────────────────────┐
    │ Thread Pool (200 threads total)  │
    │                                  │
    │ Payment calls:     150 threads   │ ← All waiting on slow Payment
    │ Restaurant calls:    0 threads   │ ← Can't get any threads!
    │ Notification calls:  0 threads   │ ← Can't get any threads!
    │ Available:          50 threads   │
    └──────────────────────────────────┘
    → Payment slowness blocks EVERYTHING

  WITH bulkhead (separate pools per dependency):
    ┌──────────────────────────────────┐
    │ Payment Pool:      50 threads    │ ← Isolated
    │   → 50 waiting on Payment        │
    │   → Can't take more than 50      │
    │                                  │
    │ Restaurant Pool:   50 threads    │ ← Unaffected!
    │   → 5 active, 45 available       │
    │                                  │
    │ Notification Pool: 30 threads    │ ← Unaffected!
    │   → 2 active, 28 available       │
    │                                  │
    │ General Pool:      70 threads    │ ← Unaffected!
    └──────────────────────────────────┘
    → Payment issues contained to its own pool

  Named after SHIP BULKHEADS:
    → Ship has watertight compartments
    → Leak in one compartment → seal it off
    → Rest of the ship stays dry
    → Same principle: isolate failure domains

COMPONENT 5: RETRY WITH CIRCUIT BREAKER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ORDER OF OPERATIONS:

  Request arrives
      │
      ▼
  ┌─ CIRCUIT BREAKER CHECK ─┐
  │ Is breaker OPEN?         │
  │   YES → return fallback  │
  │   NO  → proceed          │
  └─────────┬────────────────┘
            │
            ▼
  ┌─ BULKHEAD CHECK ─────────┐
  │ Thread pool full?         │
  │   YES → reject (503)     │
  │   NO  → acquire thread   │
  └─────────┬────────────────┘
            │
            ▼
  ┌─ TIMEOUT WRAPPER ────────┐
  │ Set timeout (e.g., 5s)   │
  └─────────┬────────────────┘

MEMORY FOOTPRINT:
  → Count-based window (20 calls): ~20 entries × 100 bytes = 2 KB
  → Time-based window (30s, 10 buckets): ~10 × 200 bytes = 2 KB
  → Per breaker overhead: ~5 KB including metadata
  → 50 breakers (50 dependencies): 250 KB
  → Negligible memory impact

CPU OVERHEAD:
  → Breaker check: O(1) state check
  → Window update: O(1) ring buffer write
  → Failure rate calculation: O(n) where n = window size
  → With 20-element window: trivial computation
  → No measurable latency added

BREAKER PER INSTANCE VS PER SERVICE:
  → Per service: 1 breaker for "payment-service"
    → Simple, but one bad instance trips it for all
  → Per instance: 1 breaker per payment-1, payment-2, payment-3
    → More breakers, but surgical isolation
    → payment-1 broken → skip it → use payment-2

  → Per endpoint: 1 breaker per service + endpoint combination
    → Most granular, most breakers
    → payment-service:/process vs payment-service:/status

DISTRIBUTED BREAKER STATE:
  → Breaker state is LOCAL to each caller instance

  → They may disagree (Order-1 sees failures, Order-2 doesn't)
  → THIS IS FINE — each instance has its own view

AT MASSIVE SCALE (10,000+ services):

  → Proxy sidecar implements all resilience patterns

# SLIDING WINDOW — Tracks call outcomes
# ============================================================

class SlidingWindow:
    """
    Count-based sliding window using a ring buffer.
    Tracks the last N call outcomes and computes failure/slow rates.
    """

    def __init__(self, size=20):
        self.size = size
        self.outcomes = deque(maxlen=size)
        self.lock = threading.Lock()

    def record(self, outcome, duration_ms=0):
        with self.lock:
            self.outcomes.append({
                "outcome": outcome,
                "duration_ms": duration_ms,
                "timestamp": time.time()
            })

    def get_metrics(self, slow_threshold_ms=3000):
        with self.lock:
            if not self.outcomes:
                return {
                    "total": 0,
                    "success": 0,
                    "failure": 0,
                    "slow": 0,
                    "failure_rate": 0.0,
                    "slow_rate": 0.0
                }

            total = len(self.outcomes)
            success = sum(
                1 for o in self.outcomes
                if o["outcome"] in (
                    CallOutcome.SUCCESS,
                    CallOutcome.SLOW_SUCCESS
                )
            )
            failure = sum(
                1 for o in self.outcomes
                if o["outcome"] in (
                    CallOutcome.FAILURE,
                    CallOutcome.TIMEOUT
                )
            )
            slow = sum(
                1 for o in self.outcomes
                if o["duration_ms"] > slow_threshold_ms
                and o["outcome"] != CallOutcome.REJECTED
            )

            return {
                "total": total,
                "success": success,
                "failure": failure,
                "slow": slow,
                "failure_rate": failure / total if total > 0 else 0.0,
                "slow_rate": slow / total if total > 0 else 0.0
            }

    def reset(self):
        with self.lock:
            self.outcomes.clear()


# ============================================================
# BULKHEAD — Limits concurrent calls to a dependency
# ============================================================

class Bulkhead:
    """
    Limits the number of concurrent calls to a dependency.
    Prevents one slow service from consuming ALL threads.
    Named after ship compartments that contain flooding.
    """

    def __init__(self, max_concurrent=10, max_wait_ms=500):
        self.semaphore = threading.Semaphore(max_concurrent)
        self.max_concurrent = max_concurrent
        self.max_wait_seconds = max_wait_ms / 1000.0
        self.active_count = 0
        self.lock = threading.Lock()

    def try_acquire(self):
        acquired = self.semaphore.acquire(
            timeout=self.max_wait_seconds
        )
        if acquired:
            with self.lock:
                self.active_count += 1
        return acquired

    def release(self):
        with self.lock:
            self.active_count = max(0, self.active_count - 1)
        self.semaphore.release()

    def get_active(self):
        with self.lock:
            return self.active_count

# CIRCUIT BREAKER — The main component
# ============================================================

class CircuitBreaker:
    """
    Production-grade circuit breaker with:
    - Configurable failure/slow thresholds
    - Sliding window for outcome tracking
    - Three states with automatic transitions
    - Bulkhead for concurrency isolation
    - Exponential backoff on repeated failures
    - Fallback chain support
    - Full metrics and event emission
    """

    def __init__(
        self,
        name,
        failure_rate_threshold=0.5,       # 50% failure rate → OPEN
        slow_call_rate_threshold=0.8,     # 80% slow calls → OPEN
        slow_call_duration_ms=3000,       # What counts as "slow"
        minimum_calls=5,                   # Min calls before evaluating
        window_size=20,                    # Sliding window size
        open_timeout_seconds=30,          # Time in OPEN before testing
        max_open_timeout_seconds=300,     # Maximum backoff timeout
        half_open_max_calls=3,            # Probe calls in HALF-OPEN
        bulkhead_max_concurrent=10,       # Max concurrent calls
        bulkhead_max_wait_ms=500,         # Max wait for bulkhead slot
        fallback_fn=None,                 # Fallback function
        on_state_change=None              # Callback for state changes
    ):
        self.name = name

        # Thresholds
        self.failure_rate_threshold = failure_rate_threshold
        self.slow_call_rate_threshold = slow_call_rate_threshold
        self.slow_call_duration_ms = slow_call_duration_ms
        self.minimum_calls = minimum_calls

        # State
        self.state = BreakerState.CLOSED
        self.state_lock = threading.Lock()

        # Sliding window
        self.window = SlidingWindow(size=window_size)

        # Open state management
        self.base_open_timeout = open_timeout_seconds
        self.current_open_timeout = open_timeout_seconds
        self.max_open_timeout = max_open_timeout_seconds
        self.opened_at = None
        self.consecutive_open_cycles = 0

        # Half-open management
        self.half_open_max_calls = half_open_max_calls
        self.half_open_call_count = 0
        self.half_open_success_count = 0

        # Bulkhead
        self.bulkhead = Bulkhead(
            max_concurrent=bulkhead_max_concurrent,
            max_wait_ms=bulkhead_max_wait_ms
        )

        # Fallback
        self.fallback_fn = fallback_fn

        # Callbacks
        self.on_state_change = on_state_change

        # Metrics
        self.metrics = {
            "total_calls": 0,
            "successful_calls": 0,
            "failed_calls": 0,
            "rejected_calls": 0,
            "slow_calls": 0,
            "bulkhead_rejections": 0,
            "fallback_calls": 0,
            "state_transitions": [],
            "created_at": datetime.utcnow().isoformat()
        }
        self.metrics_lock = threading.Lock()

    # ============================================================
    # MAIN ENTRY POINT — Execute a call through the breaker
    # ============================================================
    def call(self, fn, *args, **kwargs):
        """
        Execute function fn through the circuit breaker.
        Returns (result, call_outcome) tuple.
        """
        with self.metrics_lock:
            self.metrics["total_calls"] += 1

        with self.state_lock:
            current_state = self.state

        # ---- STATE: OPEN → Reject immediately ----
        if current_state == BreakerState.OPEN:
            if self._should_transition_to_half_open():
                self._transition_to(BreakerState.HALF_OPEN)
            else:
                return self._handle_rejection("Breaker is OPEN")

        # ---- STATE: HALF-OPEN → Allow limited probes ----
        if current_state == BreakerState.HALF_OPEN:
            with self.state_lock:
                if self.half_open_call_count >= self.half_open_max_calls:
                    return self._handle_rejection(
                        "HALF-OPEN probe limit reached"
                    )
                self.half_open_call_count += 1

        # ---- BULKHEAD CHECK ----
        if not self.bulkhead.try_acquire():
            with self.metrics_lock:
                self.metrics["bulkhead_rejections"] += 1
            self.window.record(CallOutcome.BULKHEAD_FULL)
            return self._handle_rejection("Bulkhead full")

        # ---- EXECUTE THE CALL ----
        start_time = time.time()
        try:
            result = fn(*args, **kwargs)
            duration_ms = (time.time() - start_time) * 1000

            # Determine if it was slow
            if duration_ms > self.slow_call_duration_ms:
                outcome = CallOutcome.SLOW_SUCCESS
                with self.metrics_lock:
                    self.metrics["slow_calls"] += 1
            else:
                outcome = CallOutcome.SUCCESS

            with self.metrics_lock:
                self.metrics["successful_calls"] += 1

            self.window.record(outcome, duration_ms)
            self._on_success()
            return result, outcome

        except Exception as e:
            duration_ms = (time.time() - start_time) * 1000

            # Determine failure type
            if "timeout" in str(e).lower():
                outcome = CallOutcome.TIMEOUT
            else:
                outcome = CallOutcome.FAILURE

            with self.metrics_lock:
                self.metrics["failed_calls"] += 1

            self.window.record(outcome, duration_ms)
            self._on_failure()

            # Return fallback if available
            if self.fallback_fn:
                with self.metrics_lock:
                    self.metrics["fallback_calls"] += 1
                fallback_result = self.fallback_fn(e)
                return fallback_result, outcome

            raise  # Re-raise if no fallback

        finally:
            self.bulkhead.release()

    # ============================================================
    # STATE TRANSITIONS
    # ============================================================
    def _on_success(self):
        with self.state_lock:
            if self.state == BreakerState.HALF_OPEN:
                self.half_open_success_count += 1
                if self.half_open_success_count >= self.half_open_max_calls:
                    self._transition_to(BreakerState.CLOSED)
            elif self.state == BreakerState.CLOSED:
                self._evaluate_thresholds()

    def _on_failure(self):
        with self.state_lock:
            if self.state == BreakerState.HALF_OPEN:
                # One failure in half-open → back to OPEN
                self._transition_to(BreakerState.OPEN)
            elif self.state == BreakerState.CLOSED:
                self._evaluate_thresholds()

    def _evaluate_thresholds(self):
        metrics = self.window.get_metrics(self.slow_call_duration_ms)

        # Need minimum calls before evaluating
        if metrics["total"] < self.minimum_calls:
            return

        # Check failure rate
        if metrics["failure_rate"] >= self.failure_rate_threshold:
            self._transition_to(BreakerState.OPEN)
            return

        # Check slow call rate
        if metrics["slow_rate"] >= self.slow_call_rate_threshold:
            self._transition_to(BreakerState.OPEN)

    def _should_transition_to_half_open(self):
        if self.opened_at is None:
            return False
        elapsed = time.time() - self.opened_at
        return elapsed >= self.current_open_timeout

    def _transition_to(self, new_state):
        old_state = self.state

        if old_state == new_state:
            return

        self.state = new_state
        timestamp = datetime.utcnow().isoformat()

        # State-specific setup
        if new_state == BreakerState.OPEN:
            self.opened_at = time.time()
            if old_state == BreakerState.HALF_OPEN:
                # Exponential backoff
                self.consecutive_open_cycles += 1
                self.current_open_timeout = min(
                    self.base_open_timeout * (2 ** self.consecutive_open_cycles),
                    self.max_open_timeout
                )
            else:
                self.consecutive_open_cycles = 0
                self.current_open_timeout = self.base_open_timeout

        elif new_state == BreakerState.HALF_OPEN:
            self.half_open_call_count = 0
            self.half_open_success_count = 0

        elif new_state == BreakerState.CLOSED:
            self.window.reset()
            self.consecutive_open_cycles = 0
            self.current_open_timeout = self.base_open_timeout
            self.opened_at = None

        # Record transition
        transition = {
            "from": old_state.value,
            "to": new_state.value,
            "timestamp": timestamp,
            "open_timeout": self.current_open_timeout
        }
        with self.metrics_lock:
            self.metrics["state_transitions"].append(transition)
            # Keep last 100 transitions
            if len(self.metrics["state_transitions"]) > 100:
                self.metrics["state_transitions"] = (
                    self.metrics["state_transitions"][-100:]
                )

        print(f"  ⚡ [{self.name}] {old_state.value} → {new_state.value} "
              f"(timeout={self.current_open_timeout}s)")

        # Callback
        if self.on_state_change:
            self.on_state_change(self.name, old_state, new_state)

    # ============================================================
    # REJECTION HANDLING
    # ============================================================
    def _handle_rejection(self, reason):
        with self.metrics_lock:
            self.metrics["rejected_calls"] += 1

        if self.fallback_fn:
            with self.metrics_lock:
                self.metrics["fallback_calls"] += 1
            fallback_result = self.fallback_fn(
                Exception(f"Circuit breaker [{self.name}]: {reason}")
            )
            return fallback_result, CallOutcome.REJECTED

        raise CircuitBreakerOpenError(
            f"Circuit breaker [{self.name}] is OPEN: {reason}"
        )

    # ============================================================
    # STATUS AND METRICS
    # ============================================================
    def get_status(self):
        window_metrics = self.window.get_metrics(
            self.slow_call_duration_ms
        )
        with self.metrics_lock:
            return {
                "name": self.name,
                "state": self.state.value,
                "window_metrics": window_metrics,
                "bulkhead_active": self.bulkhead.get_active(),
                "current_open_timeout": self.current_open_timeout,
                "consecutive_failures": self.consecutive_open_cycles,
                "lifetime_metrics": dict(self.metrics)
            }

    def force_open(self):
        with self.state_lock:
            self._transition_to(BreakerState.OPEN)

    def force_close(self):
        with self.state_lock:
            self._transition_to(BreakerState.CLOSED)

class ResilientServiceCaller:
    """
    The COMPLETE resilience stack:
    1. Discover instances (Project 20)
    2. Try each instance through its circuit breaker
    3. Retry on failure (with backoff)
    4. Fallback when all options exhausted
    
    This is what production code actually looks like.
    """

    def __init__(self, breaker_registry, discovery_client=None):
        self.breaker_registry = breaker_registry
        self.discovery_client = discovery_client
        # Retry configuration
        self.max_retries = 3
        self.retry_base_delay_ms = 100
        self.retry_max_delay_ms = 2000

    def call(self, service_name, call_fn, fallback_fn=None,
             instance_id=None):
        """
        Make a resilient call to a service.
        
        call_fn: function(instance) → result
                 Takes an instance dict, returns result
        fallback_fn: function(error) → fallback_result
        instance_id: specific instance (or None for discovery)
        """

        # Step 1: Get list of instances to try
        if instance_id:
            instances = [{"instance_id": instance_id}]
        elif self.discovery_client:
            instances = self.discovery_client.discover(service_name)
        else:
            instances = [{"instance_id": service_name}]

        if not instances:
            if fallback_fn:
                return fallback_fn(
                    Exception(f"No instances for {service_name}")
                )
            raise Exception(f"No instances available for {service_name}")

        # Step 2: Try each instance with its breaker
        last_error = None
        for instance in instances:
            inst_id = instance.get("instance_id", service_name)
            breaker_name = f"{service_name}/{inst_id}"

            breaker = self.breaker_registry.get_or_create(
                breaker_name,
                fallback_fn=None  # Handle fallback at caller level
            )

            # Skip if breaker is OPEN and not ready for half-open
            if (breaker.state == BreakerState.OPEN
                    and not breaker._should_transition_to_half_open()):
                continue

            # Step 3: Retry loop with this instance
            for attempt in range(self.max_retries):
                try:
                    result, outcome = breaker.call(call_fn, instance)

                    if outcome in (
                        CallOutcome.SUCCESS,
                        CallOutcome.SLOW_SUCCESS
                    ):
                        return result

                except CircuitBreakerOpenError:
                    break  # This instance's breaker is open, try next

                except Exception as e:
                    last_error = e
                    if attempt < self.max_retries - 1:
                        delay = min(
                            self.retry_base_delay_ms * (2 ** attempt),
                            self.retry_max_delay_ms
                        ) / 1000.0
                        # Add jitter
                        delay *= (0.5 + random.random())
                        time.sleep(delay)

        # Step 4: All instances exhausted → fallback
        if fallback_fn:
            return fallback_fn(last_error or Exception("All instances failed"))

        raise last_error or Exception(
            f"All instances of {service_name} failed"
        )

        self.notification_svc = notification_svc

                "minimum_calls": 5,
                "window_size": 10,
                "open_timeout_seconds": 10,
                "half_open_max_calls": 3,
                "bulkhead_max_concurrent": 5
            }

        )
        self.restaurant_breaker = self.breaker_registry.get_or_create(

            fallback_fn=self._restaurant_fallback
        )
        self.notification_breaker = self.breaker_registry.get_or_create(

            fallback_fn=self._notification_fallback
        )

        # Order tracking
        self.orders = {}
        self.order_counter = 0
        self.lock = threading.Lock()

    def place_order(self, user_id, restaurant_id, items, amount):
        """
        Full order flow:
        1. Get restaurant info (via breaker)
        2. Process payment (via breaker)
        3. Send notification (via breaker)
        Each step is independently protected.
        """
        with self.lock:
            self.order_counter += 1
            order_id = f"ORD-{self.order_counter:04d}"

        order = {
            "order_id": order_id,
            "user_id": user_id,
            "restaurant_id": restaurant_id,
            "items": items,
            "amount": amount,
            "status": "processing",
            "steps": {}
        }

        # ---- Step 1: Get Restaurant Info ----
        try:
            result, outcome = self.restaurant_breaker.call(
                self.restaurant_svc.get_restaurant,
                restaurant_id
            )
            order["steps"]["restaurant"] = {
                "status": "success",
                "outcome": outcome.value,
                "data": result
            }
        except CircuitBreakerOpenError as e:
            order["steps"]["restaurant"] = {
                "status": "breaker_open",
                "data": self._restaurant_fallback(e)
            }

        # ---- Step 2: Process Payment ----
        try:
            result, outcome = self.payment_breaker.call(
                self.payment_svc.process_payment,
                amount
            )
            order["steps"]["payment"] = {
                "status": "success",
                "outcome": outcome.value,
                "data": result
            }
            order["status"] = "confirmed"
        except CircuitBreakerOpenError as e:
            order["steps"]["payment"] = {
                "status": "breaker_open",
                "data": self._payment_fallback(e)
            }
            order["status"] = "payment_pending"
        except Exception as e:
            order["steps"]["payment"] = {
                "status": "failed",
                "error": str(e),
                "data": self._payment_fallback(e)
            }
            order["status"] = "payment_pending"

        # ---- Step 3: Send Notification ----
        try:
            message = (f"Order {order_id}: {order['status']}")
            result, outcome = self.notification_breaker.call(
                self.notification_svc.send_notification,
                user_id,
                message
            )

                "status": "success",
                "outcome": outcome.value,
                "data": result
            }
        except CircuitBreakerOpenError as e:
            order["steps"]["notification"] = {
                "status": "breaker_open",
                "data": self._notification_fallback(e)
            }
        except Exception:
            order["steps"]["notification"] = {
                "status": "skipped",
                "reason": "non-critical, will retry later"
            }

        self.orders[order_id] = order
        return order

    # ---- Fallback Functions ----

            "status": "payment_queued",
            "message": "Payment will be processed when service recovers",
            "fallback": True,
            "original_error": str(error)
        }

            "status": "cached",

        }

    def _notification_fallback(self, error):
        return {
            "status": "queued",
            "message": "Notification queued for later delivery",

        }

class DiscoveryAwareResilientCaller:
    """
    Combines Project 20 (Service Discovery) with Project 21 (Circuit Breaker)
    
    Flow:
    1. Discover all instances of target service
    2. Filter out instances whose breakers are OPEN
    3. Try remaining instances with per-instance breakers
    4. If all breakers open → fallback
    
    This is the pattern used by Netflix Hystrix, Resilience4j,
    and service meshes like Istio.
    """

    def __init__(self, breaker_registry, discovery_client=None):
        self.breaker_registry = breaker_registry
        self.discovery_client = discovery_client

    def call_service(self, service_name, call_fn,
                     fallback_fn=None, strategy="try_all"):
        """
        call_fn: function(host, port) → result
        fallback_fn: function(error) → fallback_result
        strategy:
          "try_all"   → try each instance until one succeeds
          "failfast"  → try one, fail immediately if it fails
        """

        # Step 1: Discover instances
        if self.discovery_client:
            instances = self.discovery_client.discover(service_name)
        else:
            instances = [{"instance_id": "default", "host": "localhost",
                          "port": 8080}]

        if not instances:
            if fallback_fn:
                return fallback_fn(
                    Exception(f"No instances for {service_name}")
                )
            raise Exception(f"No instances for {service_name}")

        # Step 2: Sort instances — prefer those with CLOSED breakers
        def breaker_priority(inst):
            breaker_name = f"{service_name}/{inst['instance_id']}"
            breaker = self.breaker_registry.breakers.get(breaker_name)
            if breaker is None:
                return 0  # New breaker (not yet created) → highest priority
            if breaker.state == BreakerState.CLOSED:
                return 1
            if breaker.state == BreakerState.HALF_OPEN:
                return 2
            return 3  # OPEN → lowest priority

        sorted_instances = sorted(instances, key=breaker_priority)

        # Step 3: Try each instance through its breaker
        last_error = None
        attempted = 0

        for instance in sorted_instances:
            inst_id = instance["instance_id"]
            host = instance.get("host", "localhost")
            port = instance.get("port", 8080)
            breaker_name = f"{service_name}/{inst_id}"

            breaker = self.breaker_registry.get_or_create(
                breaker_name,
                fallback_fn=None  # Handle at this level
            )

            # Skip fully OPEN breakers (not ready for half-open yet)
            if (breaker.state == BreakerState.OPEN
                    and not breaker._should_transition_to_half_open()):
                print(f"    ⏭️  Skipping {inst_id} (breaker OPEN)")
                continue

            attempted += 1
            try:
                result, outcome = breaker.call(call_fn, host, port)
                if outcome in (CallOutcome.SUCCESS,
                               CallOutcome.SLOW_SUCCESS):
                    print(f"    ✅ {inst_id} succeeded "
                          f"({outcome.value})")
                    return result
            except CircuitBreakerOpenError:
                print(f"    🔴 {inst_id} breaker just opened")
                continue
            except Exception as e:
                last_error = e
                print(f"    ❌ {inst_id} failed: {e}")
                if strategy == "failfast":
                    break
                continue

        # Step 4: All instances exhausted
        if attempted == 0:
            msg = (f"All {len(sorted_instances)} instances of "
                   f"{service_name} have OPEN breakers")
        else:
            msg = (f"Tried {attempted} instances of "
                   f"{service_name}, all failed")

        print(f"    🛑 {msg}")

        if fallback_fn:
            return fallback_fn(last_error or Exception(msg))

        raise last_error or Exception(msg)

    def get_service_health_view(self, service_name):
        """
        Combined view: discovery + breaker state per instance
        Shows which instances are healthy from BOTH perspectives
        """
        if self.discovery_client:
            instances = self.discovery_client.discover(
                service_name, use_cache=False
            )
        else:
            instances = []

        view = []
        for inst in instances:
            inst_id = inst["instance_id"]
            breaker_name = f"{service_name}/{inst_id}"
            breaker = self.breaker_registry.breakers.get(breaker_name)

            if breaker:
                status = breaker.get_status()
                breaker_state = status["state"]
                failure_rate = status["window_metrics"]["failure_rate"]
            else:
                breaker_state = "NO_BREAKER"
                failure_rate = 0.0

            registry_status = inst.get("status", "UNKNOWN")

            # Combined health assessment
            if registry_status == "UP" and breaker_state == "CLOSED":
                health = "🟢 HEALTHY"
            elif registry_status == "UP" and breaker_state == "HALF_OPEN":
                health = "🟡 TESTING"
            elif registry_status == "UP" and breaker_state == "OPEN":
                health = "🔴 FAILING"
            elif registry_status != "UP":
                health = "⚫ DOWN"
            else:
                health = "⚪ UNKNOWN"

            view.append({
                "instance_id": inst_id,
                "host": inst.get("host"),
                "port": inst.get("port"),
                "registry_status": registry_status,
                "breaker_state": breaker_state,
                "failure_rate": f"{failure_rate:.1%}",
                "health": health
            })

        return view

    def print_service_health(self, service_name):
        view = self.get_service_health_view(service_name)
        print(f"\n  Service Health: {service_name}")
        print(f"  {'Instance':<15} {'Registry':<10} {'Breaker':<12} "
              f"{'Fail%':<8} {'Health'}")
        print(f"  {'─' * 65}")
        for v in view:
            print(f"  {v['instance_id']:<15} "
                  f"{v['registry_status']:<10} "
                  f"{v['breaker_state']:<12} "
                  f"{v['failure_rate']:<8} "
                  f"{v['health']}")
        print()
