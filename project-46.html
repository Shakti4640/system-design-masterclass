KEY INSIGHT: MULTI-RAFT
  → Data is split into RANGES (like shards from Project 18)
  → Each range has its OWN Raft group (3 or 5 replicas)
  → Replicas of a range are SPREAD across regions
  → The LEADER of each range can be in ANY region
  → Leader is placed NEAR the data's primary users
  → This gives local-latency writes for most operations


DATA FLOW FOR A WRITE:

  User in Tokyo: "Transfer $100 from account Q123 to Q456"
  
  1. Request hits Tokyo SQL Gateway
  2. Gateway determines: account Q123 → Range [Q-Z]
  3. Range [Q-Z] leader is in Tokyo (local!) ✅
  4. Tokyo leader appends to Raft log
  5. Sends AppendEntries to Frankfurt + Virginia replicas
  6. Frankfurt acknowledges (120ms RTT)
  7. Majority achieved: Tokyo + Frankfurt = 2 of 3 → COMMIT
  8. Response to user: ~125ms total
     (mostly cross-region replication wait)

  Compare to single-region:
    User in Tokyo → Virginia leader → 150ms × 2 = 300ms
    With Multi-Raft + Tokyo leader: 125ms → 2.4× faster!


DATA FLOW FOR A LOCAL READ:

  Option A: Linearizable Read (strongest)
    → Must go through leader → leader in Tokyo → LOCAL → ~2ms ✅
    → But if leader is in Virginia → 150ms RTT ❌

  Option B: Follower Read with Lease
    → Leader grants read lease to local follower
    → Follower serves reads during lease period
    → Slightly stale (by lease duration) but LOCAL → ~2ms ✅

  Option C: Stale Read (explicit)
    → Read from any replica, accept staleness
    → Application specifies: "I accept data up to 5s old"
    → Always local → ~1ms ✅


CROSS-REGION TRANSACTION:

  User: "Transfer $100 from account A123 (US) to Q456 (Asia)"
  
  A123 → Range [A-H] → Leader in US-East
  Q456 → Range [Q-Z] → Leader in Tokyo
  
  This is a DISTRIBUTED TRANSACTION across two Raft groups!
  
  Solution: Two-Phase Commit over Raft
  
  Phase 1 (Prepare):
    → US-East leader: PREPARE "debit A123 by $100"
      → Raft-commit the prepare → durable
    → Tokyo leader: PREPARE "credit Q456 by $100"
      → Raft-commit the prepare → durable
  
  Phase 2 (Commit):
    → If both prepared successfully:
      → US-East leader: COMMIT the debit
      → Tokyo leader: COMMIT the credit
    → If either failed:
      → Both: ABORT
  
  Latency: ~max(US-East commit, Tokyo commit) + coordination
         = ~150ms (cross-region RTT dominates)

📌 WHAT COULD BE DONE

REPLICATION TOPOLOGY OPTIONS:

  Option A: Single-Region Raft + Async Cross-Region
    → Raft group entirely in one region (3 nodes, same DC)
    → Async replication to other regions (followers, not voters)
    → Pro: Fast local commits (~5ms)
    → Con: Cross-region followers can be stale
    → Con: Region failure → data loss for uncommitted async entries
    → Used by: MySQL Group Replication + async replicas

  Option B: Multi-Region Raft (Synchronous)
    → Raft group spans 3 regions (1 replica per region)
    → Commit requires 2 of 3 regions to acknowledge
    → Pro: Survives entire region failure
    → Con: Every commit pays cross-region latency
    → Used by: CockroachDB, Spanner (default)

  Option C: Witness Replica (Quorum Optimization)
    → 2 full replicas + 1 lightweight witness
    → Witness only votes, doesn't store full data
    → Place witness in a third region for tie-breaking
    → Pro: Lower storage cost, same availability
    → Con: Recovery requires full replica, not witness
    → Used by: CockroachDB (witness replicas)

  Option D: Multi-Master with Conflict Resolution
    → Multiple regions can accept writes independently
    → Conflicts resolved after the fact (LWW, merge functions)
    → Pro: Lowest write latency (always local)
    → Con: Conflicts! Not suitable for financial data
    → Used by: DynamoDB Global Tables, Cassandra

  FOR THIS PROJECT → Option B as primary (strong consistency)
                   → Show Option A for read-heavy workloads
                   → Option D for comparison (eventual consistency)

CLOCK SYNCHRONIZATION OPTIONS:

  Option A: TrueTime (Google Spanner)
    → GPS receivers + atomic clocks in every datacenter
    → API: TrueTime.now() returns [earliest, latest] interval
    → If intervals don't overlap → definite ordering
    → Pro: Real physical time ordering
    → Con: Requires expensive hardware (GPS + atomic clocks)

  Option B: Hybrid Logical Clocks (HLC)
    → Combines physical time + logical counter
    → HLC = (physical_time, logical_counter, node_id)
    → Always moves forward (never backwards)
    → Captures causality without expensive hardware
    → Pro: Works with regular NTP-synced clocks
    → Con: Cannot order truly concurrent events (but Raft handles that)
    → Used by: CockroachDB, YugabyteDB

  Option C: Lamport Timestamps
    → Pure logical clocks (Project 45 term numbers are a form of this)
    → Pro: Simple, no physical clock dependency
    → Con: No connection to wall-clock time
    → Not suitable for: "give me all events from last 5 minutes"

  FOR THIS PROJECT → Option B (Hybrid Logical Clocks)
    → Most practical for real systems without GPS hardware

✅ WHAT SHOULD DO

1. PLACE RAFT LEADERS NEAR DATA'S PRIMARY USERS
   → If account data is mostly accessed from Europe → leader in Europe
   → This makes most writes LOCAL (only cross-region for replication)
   → Use access pattern analysis to determine optimal placement
   → Periodically re-evaluate and migrate leaders

2. USE MULTI-RAFT WITH RANGE-BASED SHARDING
   → Don't put ALL data in one Raft group
   → Split data into ranges (e.g., by key prefix, by user region)
   → Each range = independent Raft group
   → Different ranges can have leaders in different regions
   → Thousands of ranges → fine-grained placement

3. IMPLEMENT HYBRID LOGICAL CLOCKS
   → Every node maintains HLC
   → Every message carries sender's HLC
   → Receiver updates: max(local_HLC, received_HLC) + 1
   → This gives causal ordering across regions
   → Transactions use HLC for MVCC timestamps

4. USE QUORUM LEASES FOR LOCAL READS
   → Leader grants read lease to followers
   → During lease: follower can serve reads locally
   → Lease duration < election timeout (safety)
   → Reduces read latency from cross-region to local

5. BATCH CROSS-REGION REPLICATION
   → Don't send one entry at a time across regions
   → Batch multiple entries per AppendEntries RPC
   → Amortize cross-region RTT across many operations
   → Dramatically improves throughput

6. IMPLEMENT REGION-AWARE ROUTING
   → Client connects to nearest SQL gateway
   → Gateway knows which region leads each range
   → Local-leader requests: serve directly
   → Remote-leader requests: proxy to correct region
   → Return routing hints so client can optimize future requests

❌ WHAT SHOULD NOT DO

1. NEVER PUT ALL RAFT REPLICAS IN ONE REGION
   → Defeats the purpose of multi-region
   → Region failure = total data loss for those ranges
   → Minimum: 2 regions for 3-replica Raft
   → Recommended: 3 regions for 3-replica Raft

2. NEVER ASSUME CLOCKS ARE SYNCHRONIZED
   → NTP gives ~1-10ms accuracy between datacenters
   → That's an ETERNITY for distributed transactions
   → Never use System.currentTimeMillis() for ordering
   → Use HLC or TrueTime → ALWAYS

3. NEVER DO SYNCHRONOUS CROSS-REGION CALLS IN THE READ PATH
   → Read from local replica whenever possible
   → Linearizable reads from leader (if leader is local → great)
   → If leader is remote → offer "bounded staleness" option
   → Users will accept 1-second-old data over 200ms latency

4. NEVER IGNORE NETWORK PARTITION BETWEEN REGIONS
   → Cross-region links fail more often than same-region
   → Submarine cable cuts, BGP misconfigurations, etc.
   → Design for: "What if US can't talk to Europe for 30 minutes?"
   → Raft handles this (minority side stops committing)
   → But your APPLICATION must handle "leader unavailable" gracefully

5. NEVER USE TWO-PHASE COMMIT WITHOUT RAFT UNDERNEATH
   → 2PC coordinator is a single point of failure
   → If coordinator crashes during commit → participants stuck
   → Raft-replicate the coordinator state → crash-safe 2PC
   → This is how Spanner and CockroachDB do it

6. NEVER SHARD BY REGION ALONE
   → "All EU users → EU shard" sounds clean
   → But cross-region transactions become common
     (EU user buys from US merchant)
   → Shard by data access patterns, not just geography
   → Co-locate related data in same range when possible

MECHANIC 2: HYBRID LOGICAL CLOCKS (HLC)
═══════════════════════════════════════════════════════════

  Problem: Need to ORDER events across regions
  
  Physical clocks drift:
    US clock:  14:00:00.000
    EU clock:  14:00:00.003  (3ms ahead)
    Asia clock: 13:59:59.998  (2ms behind)
  
  If US writes at 14:00:00.000 and Asia writes at 13:59:59.998:
    → Asia's write happened LATER in real time
    → But Asia's clock says EARLIER
    → Physical time ordering is WRONG
  
  HLC Structure:
    HLC = (physical_time, logical_counter, node_id)
  
  Rules:
    1. On local event or send:
       hlc.physical = max(hlc.physical, system_clock())
       hlc.logical += 1
    
    2. On receive message with sender's HLC:
       if sender.physical > hlc.physical:
         hlc.physical = sender.physical
         hlc.logical = sender.logical + 1
       elif sender.physical == hlc.physical:
         hlc.logical = max(hlc.logical, sender.logical) + 1
       else:
         hlc.physical = max(hlc.physical, system_clock())
         hlc.logical += 1
  
  Properties:
    → HLC always moves forward (never backwards)
    → If event A causes event B → HLC(A) < HLC(B) (causality)
    → HLC stays close to physical time (bounded drift)
    → Can be used as MVCC timestamp for transactions
  
  Example:
    US writes key "x" at HLC(14:00:00.000, 1, US)
    EU reads key "x" at HLC(14:00:00.003, 1, EU)
    EU's HLC > US's HLC → EU sees US's write ✅

MECHANIC 3: DISTRIBUTED TRANSACTIONS (2PC OVER RAFT)
═══════════════════════════════════════════════════════════

  Transaction: "Transfer $100 from account A (US) to account Q (Asia)"
  
  Account A → Range [A-D] → Leader in US
  Account Q → Range [Q-T] → Leader in Asia
  
  Coordinator: The SQL gateway that received the request
  (Coordinator state is ALSO Raft-replicated for crash safety)
  
  PHASE 1: PREPARE

  │ Coordinator │                    │              │
  │ (US Gateway)│                    │              │
  └──────┬──────┘                    │              │
         │                           │              │
         │ PREPARE "debit A by $100" │              │
         │──────────────────────────→│ US Range     │
         │                           │ [A-D] Leader │
         │                           │              │
         │ Raft-commit PREPARE       │              │
         │ (lock row A, check bal)   │              │
         │←─── PREPARED ─────────────│              │
         │                           └──────────────┘
         │
         │ PREPARE "credit Q by $100"
         │──────────────────────────→ ┌──────────────┐
         │                            │ Asia Range   │
         │                            │ [Q-T] Leader │
         │ Raft-commit PREPARE        │              │
         │ (lock row Q)               │              │
         │←─── PREPARED ──────────────│              │
         │                            └──────────────┘
  
  PHASE 2: COMMIT
         │
         │ Both prepared? YES
         │
         │ COMMIT
         │──────────────────────────→ US Range: apply debit
         │──────────────────────────→ Asia Range: apply credit
         │
         │←─── COMMITTED (from both)
         │
         │ Respond to client: "Transfer complete"
  
  WHY RAFT MAKES 2PC SAFE:
    → Traditional 2PC: coordinator crash = participants stuck
    → Raft-backed 2PC: coordinator state is replicated
    → If coordinator node crashes, new Raft leader has the state
    → Can resume commit/abort from where it left off
    → NO stuck participants ✅
  
  LATENCY:
    Prepare phase: max(US commit, Asia commit)
      US commit: ~5ms (leader is local)
      Asia commit: ~150ms RTT (US coordinator → Asia leader)
    
    Commit phase: same pattern
    
    Total: ~300-350ms for cross-region transaction
    (This is why you minimize cross-region transactions!)

MECHANIC 4: LEADER PLACEMENT & TRANSFER
═══════════════════════════════════════════════════════════

  Goal: Leader should be NEAR the data's primary users
  
  Detection:
    → Track write origins per range per region
    → Range [Q-T]: 80% writes from Asia, 15% EU, 5% US
    → Optimal leader: Asia ✅ (already there)
    
    → Range [A-D]: 60% writes from US, 30% EU, 10% Asia
    → Current leader: EU ← suboptimal!
    → Optimal leader: US → TRANSFER LEADER
  
  Leader Transfer Protocol (Raft extension):
    1. Current leader (EU) sends TimeoutNow to target (US)
    2. US node immediately starts election (skips timeout)
    3. US node has all committed entries (guaranteed by Raft)
    4. US node wins election (it's up-to-date + starts first)
    5. US node becomes new leader for Range [A-D]
    6. Future writes from US users → LOCAL leader → fast!
  
  When to transfer:
    → Traffic pattern changes significantly (>30% shift)
    → Region becomes overloaded (load balancing)
    → Region planned maintenance (drain leaders out)
    → New region added (redistribute leaders)

MECHANIC 5: QUORUM CONFIGURATION
═══════════════════════════════════════════════════════════

  3-Region Setup: US, EU, Asia
  3-Replica Raft: one replica per region
  
  Commit requires: 2 of 3 replicas (majority)
  
  Scenario 1: Leader in US
    → US commits: wait for US(local) + EU(80ms) = ~80ms
    → Don't need to wait for Asia (already have majority)
    → Commit latency = RTT to NEAREST non-local replica
  
  Scenario 2: Leader in Asia
    → Asia commits: wait for Asia(local) + EU(120ms) = ~120ms
    → EU is closer than US to Asia
    → Smart quorum: always wait for the NEAREST replica
  
  QUORUM OPTIMIZATION: Flexible Quorum
    → Instead of majority of ALL replicas:
    → Write quorum (Qw) + Read quorum (Qr) > N
    → Example: N=5, Qw=3, Qr=3 (standard)
    → Or: N=5, Qw=2, Qr=4 (fast writes, slower reads)
    → Or: N=5, Qw=4, Qr=2 (slow writes, fast reads)
    
  For read-heavy workloads:
    → Qw=4, Qr=2: reads only need 2 replicas (often local)
    → Writes slower but reads are blazing fast
    
  For write-heavy workloads:

class HybridLogicalClock:
    """
    Combines physical time with logical counter for causal ordering.
    
    HLC = (physical_time_ms, logical_counter, node_id)
    
    Properties:
      → Always monotonically increasing
      → Captures causality: if A→B then HLC(A) < HLC(B)
      → Stays close to physical time (bounded drift)
    
    Used for: MVCC timestamps, transaction ordering
    """

    def __init__(self, node_id: str, clock_offset_ms: float = 0):
        self.node_id = node_id
        self.physical = 0
        self.logical = 0
        self.clock_offset_ms = clock_offset_ms
        self._lock = threading.Lock()

    def _wall_clock(self) -> int:
        return int(time.time() * 1000) + int(self.clock_offset_ms)

    def now(self) -> Tuple[int, int, str]:
        with self._lock:
            wall = self._wall_clock()
            if wall > self.physical:
                self.physical = wall
                self.logical = 0
            else:
                self.logical += 1
            return (self.physical, self.logical, self.node_id)

    def update(self, remote_physical: int, remote_logical: int):
        with self._lock:
            wall = self._wall_clock()
            if wall > self.physical and wall > remote_physical:
                self.physical = wall
                self.logical = 0
            elif remote_physical > self.physical:
                self.physical = remote_physical
                self.logical = remote_logical + 1
            elif self.physical == remote_physical:
                self.logical = max(self.logical, remote_logical) + 1
            else:
                self.logical += 1
            return (self.physical, self.logical, self.node_id)

    def timestamp(self) -> str:
        ts = self.now()
        return f"{ts[0]}.{ts[1]:04d}.{ts[2]}"

    @staticmethod
    def compare(ts1: Tuple[int, int, str], ts2: Tuple[int, int, str]) -> int:
        if ts1[0] != ts2[0]:
            return -1 if ts1[0] < ts2[0] else 1
        if ts1[1] != ts2[1]:
            return -1 if ts1[1] < ts2[1] else 1
        if ts1[2] != ts2[2]:
            return -1 if ts1[2] < ts2[2] else 1
        return 0

class RangeMap:
    """
    Maps keys to their owning DataRange.
    Maintains sorted list of ranges for efficient lookup.
    Equivalent to the meta-range in CockroachDB.
    """

    def __init__(self):
        self.ranges: List[DataRange] = []
        self._lock = threading.Lock()

    def add_range(self, data_range: DataRange):
        with self._lock:
            self.ranges.append(data_range)
            self.ranges.sort(key=lambda r: r.start_key)

    def find_range(self, key: str) -> Optional[DataRange]:
        with self._lock:
            for r in self.ranges:
                if r.contains_key(key):
                    return r
            return None

    def find_ranges_for_keys(self, keys: List[str]) -> Dict[str, DataRange]:
        result = {}
        for key in keys:
            r = self.find_range(key)
            if r:
                result[key] = r
        return result

    def get_all_ranges(self) -> List[DataRange]:
        with self._lock:
            return list(self.ranges)

    def transfer_leader(self, range_id: str, new_leader_region: str):
        with self._lock:
            for r in self.ranges:
                if r.range_id == range_id:
                    r.leader_region = new_leader_region
                    r.version += 1
                    return True
            return False

class RegionNode:
    """
    Represents one database node in a specific region.
    Participates in multiple Raft groups (one per range it holds).
    
    Each node stores:
      → Key-value data for ranges it replicates
      → Raft log entries per range
      → HLC for timestamp generation
    """

    def __init__(self, node_id: str, region: str, clock_offset_ms: float = 0):
        self.node_id = node_id
        self.region = region
        self.hlc = HybridLogicalClock(node_id, clock_offset_ms)

        # Data storage per range
        self.data: Dict[str, Dict[str, Any]] = defaultdict(dict)

        # Raft log per range (simplified)
        self.raft_logs: Dict[str, List[dict]] = defaultdict(list)
        self.commit_indices: Dict[str, int] = defaultdict(int)

        # Write tracking for placement optimization
        self.write_origins: Dict[str, Dict[str, int]] = defaultdict(lambda: defaultdict(int))

        # Locks per range (for transactions)
        self.row_locks: Dict[str, Dict[str, str]] = defaultdict(dict)

        self.is_alive = True
        self._lock = threading.Lock()

    def write(self, range_id: str, key: str, value: Any,
              origin_region: str, hlc_ts: Tuple) -> dict:
        with self._lock:
            if not self.is_alive:
                return {"success": False, "error": "NODE_DOWN"}

            # Update HLC
            self.hlc.update(hlc_ts[0], hlc_ts[1])
            write_ts = self.hlc.now()

            # Store data
            self.data[range_id][key] = {
                "value": value,
                "timestamp": write_ts,
                "version": self.data[range_id].get(key, {}).get("version", 0) + 1
            }

            # Log the write
            entry = {
                "type": "WRITE",
                "key": key,
                "value": value,
                "timestamp": write_ts,
                "origin": origin_region
            }
            self.raft_logs[range_id].append(entry)
            self.commit_indices[range_id] = len(self.raft_logs[range_id])

            # Track write origins
            self.write_origins[range_id][origin_region] += 1

            return {
                "success": True,
                "timestamp": write_ts,
                "version": self.data[range_id][key]["version"]
            }

    def read(self, range_id: str, key: str) -> dict:
        with self._lock:
            if not self.is_alive:
                return {"success": False, "error": "NODE_DOWN"}

            if range_id in self.data and key in self.data[range_id]:
                entry = self.data[range_id][key]
                return {
                    "success": True,
                    "key": key,
                    "value": entry["value"],
                    "timestamp": entry["timestamp"],
                    "version": entry["version"]
                }
            return {
                "success": True,
                "key": key,
                "value": None,
                "version": 0
            }

    def prepare_transaction(self, range_id: str, txn_id: str,
                            operations: List[dict]) -> dict:
        with self._lock:
            if not self.is_alive:
                return {"success": False, "error": "NODE_DOWN"}

            # Check for lock conflicts
            for op in operations:
                key = op["key"]
                if key in self.row_locks[range_id]:
                    existing_txn = self.row_locks[range_id][key]
                    if existing_txn != txn_id:
                        return {
                            "success": False,
                            "error": f"LOCK_CONFLICT on {key} (held by {existing_txn})"
                        }

            # Acquire locks
            for op in operations:
                self.row_locks[range_id][op["key"]] = txn_id

            # Log prepare
            self.raft_logs[range_id].append({
                "type": "PREPARE",
                "txn_id": txn_id,
                "operations": operations
            })

            return {"success": True, "status": "PREPARED"}

    def commit_transaction(self, range_id: str, txn_id: str,
                           operations: List[dict],
                           origin_region: str, hlc_ts: Tuple) -> dict:
        with self._lock:
            if not self.is_alive:
                return {"success": False, "error": "NODE_DOWN"}

            self.hlc.update(hlc_ts[0], hlc_ts[1])
            commit_ts = self.hlc.now()

            for op in operations:
                key = op["key"]
                if op["type"] == "SET":
                    self.data[range_id][key] = {
                        "value": op["value"],
                        "timestamp": commit_ts,
                        "version": self.data[range_id].get(key, {}).get("version", 0) + 1
                    }
                elif op["type"] == "DELETE":
                    self.data[range_id].pop(key, None)

                # Release lock
                self.row_locks[range_id].pop(key, None)

            self.raft_logs[range_id].append({
                "type": "COMMIT",
                "txn_id": txn_id,
                "timestamp": commit_ts
            })
            self.commit_indices[range_id] = len(self.raft_logs[range_id])
            self.write_origins[range_id][origin_region] += 1

            return {"success": True, "timestamp": commit_ts}

    def abort_transaction(self, range_id: str, txn_id: str,
                          operations: List[dict]) -> dict:
        with self._lock:
            for op in operations:
                key = op["key"]
                if self.row_locks[range_id].get(key) == txn_id:
                    del self.row_locks[range_id][key]

            self.raft_logs[range_id].append({
                "type": "ABORT",
                "txn_id": txn_id
            })
            return {"success": True, "status": "ABORTED"}

    def replicate_from(self, range_id: str, entries: List[dict]):
        with self._lock:
            for entry in entries:
                if entry["type"] == "WRITE":
                    key = entry["key"]
                    self.data[range_id][key] = {
                        "value": entry["value"],
                        "timestamp": entry["timestamp"],
                        "version": self.data[range_id].get(key, {}).get("version", 0) + 1
                    }
                elif entry["type"] == "COMMIT":
                    pass  # Apply committed operations
                self.raft_logs[range_id].append(entry)
            self.commit_indices[range_id] = len(self.raft_logs[range_id])

    def get_write_origin_stats(self, range_id: str) -> Dict[str, int]:
        with self._lock:
            return dict(self.write_origins.get(range_id, {}))

    def get_data_snapshot(self, range_id: str) -> Dict[str, Any]:
        with self._lock:
            return {k: v["value"] for k, v in self.data.get(range_id, {}).items()}

    def crash(self):
        self.is_alive = False

    def recover(self):
        self.is_alive = True

class CrossRegionNetwork:
    """
    Simulates network latency between regions.
    Each region-pair has a configured RTT.
    """

    def __init__(self):
        self.latencies: Dict[Tuple[str, str], float] = {}
        self.partitioned_pairs: Set[Tuple[str, str]] = set()
        self.total_cross_region_calls = 0
        self.total_latency_ms = 0.0
        self._lock = threading.Lock()

    def set_latency(self, region_a: str, region_b: str, rtt_ms: float):
        self.latencies[(region_a, region_b)] = rtt_ms
        self.latencies[(region_b, region_a)] = rtt_ms

    def get_latency(self, from_region: str, to_region: str) -> float:
        if from_region == to_region:
            return random.uniform(0.5, 2.0)  # Same region: <2ms
        return self.latencies.get(
            (from_region, to_region),
            random.uniform(50, 200)  # Default cross-region
        )

    def simulate_rpc(self, from_region: str, to_region: str) -> float:
        if (from_region, to_region) in self.partitioned_pairs:
            return -1  # Partition: request fails

        latency = self.get_latency(from_region, to_region)
        jitter = latency * random.uniform(-0.1, 0.1)
        actual_latency = max(0.1, latency + jitter)

        with self._lock:
            self.total_cross_region_calls += 1
            self.total_latency_ms += actual_latency

        # Simulate the delay (scaled down for simulation speed)
        time.sleep(actual_latency / 10000.0)

        return actual_latency

    def add_partition(self, region_a: str, region_b: str):
        self.partitioned_pairs.add((region_a, region_b))
        self.partitioned_pairs.add((region_b, region_a))

    def remove_partition(self, region_a: str, region_b: str):
        self.partitioned_pairs.discard((region_a, region_b))
        self.partitioned_pairs.discard((region_b, region_a))

    def heal_all(self):
        self.partitioned_pairs.clear()

    def get_stats(self) -> dict:
        with self._lock:
            avg = (self.total_latency_ms / self.total_cross_region_calls
                   if self.total_cross_region_calls > 0 else 0)
            return {
                "total_cross_region_calls": self.total_cross_region_calls,
                "total_latency_ms": round(self.total_latency_ms, 1),
                "avg_latency_ms": round(avg, 1)
            }

class TransactionCoordinator:
    """
    Coordinates distributed transactions across multiple ranges/regions.
    
    For single-range transactions: direct Raft commit (fast path)
    For multi-range transactions: Two-Phase Commit (2PC)
    
    Coordinator state is durable (would be Raft-replicated in production).
    """

    def __init__(self, network: CrossRegionNetwork):
        self.network = network
        self.txn_counter = 0
        self.txn_log: List[dict] = []
        self._lock = threading.Lock()

    def _new_txn_id(self) -> str:
        with self._lock:
            self.txn_counter += 1
            return f"TXN_{self.txn_counter:06d}"

    def execute_single_range_txn(self, region: Region, range_obj: DataRange,
                                 operations: List[dict],
                                 origin_region: str) -> dict:
        """Fast path: all operations in one range → direct Raft commit"""
        txn_id = self._new_txn_id()
        start_time = time.time()

        node = region.get_primary()
        if not node or not node.is_alive:
            return {"success": False, "error": "REGION_UNAVAILABLE", "txn_id": txn_id}

        # If leader is in a different region, add cross-region latency
        leader_region = range_obj.leader_region
        rpc_latency = 0.0
        if origin_region != leader_region:
            rpc_latency = self.network.simulate_rpc(origin_region, leader_region)
            if rpc_latency < 0:
                return {"success": False, "error": "NETWORK_PARTITION", "txn_id": txn_id}

        hlc_ts = node.hlc.now()

        # Apply all operations
        for op in operations:
            if op["type"] == "SET":
                result = node.write(range_obj.range_id, op["key"], op["value"],
                                    origin_region, hlc_ts)
                if not result["success"]:
                    return {"success": False, "error": result["error"], "txn_id": txn_id}

            elif op["type"] == "DELETE":
                node.data[range_obj.range_id].pop(op["key"], None)

        # Simulate Raft replication to other regions
        replication_latencies = []
        for replica_region in range_obj.replicas:
            if replica_region != leader_region:
                rep_latency = self.network.simulate_rpc(leader_region, replica_region)
                if rep_latency > 0:
                    replication_latencies.append(rep_latency)

        # Raft majority: need (N/2) non-leader acknowledgments
        total_replicas = len(range_obj.replicas)
        needed = (total_replicas // 2)  # Excluding leader who already has it
        replication_latencies.sort()

        commit_latency = 0.0
        if replication_latencies and needed > 0:
            # Wait for fastest 'needed' replicas (Raft majority)
            commit_latency = replication_latencies[min(needed - 1, len(replication_latencies) - 1)]

        total_latency = rpc_latency + commit_latency
        elapsed_real = (time.time() - start_time) * 1000

        txn_record = {
            "txn_id": txn_id,
            "type": "single_range",
            "range": range_obj.range_id,
            "operations": len(operations),
            "origin_region": origin_region,
            "leader_region": leader_region,
            "simulated_latency_ms": round(total_latency, 2),
            "rpc_latency_ms": round(rpc_latency, 2),
            "commit_latency_ms": round(commit_latency, 2),
            "success": True
        }
        self.txn_log.append(txn_record)

        return {
            "success": True,
            "txn_id": txn_id,
            "latency_ms": round(total_latency, 2),
            "leader_region": leader_region,
            "cross_region": origin_region != leader_region
        }

    def execute_multi_range_txn(self, ranges_and_ops: List[Tuple[DataRange, Region, List[dict]]],
                                origin_region: str,
                                regions: Dict[str, Region]) -> dict:
        """
        Slow path: operations span multiple ranges → Two-Phase Commit.
        
        Phase 1 (Prepare): Each range's leader prepares and locks rows
        Phase 2 (Commit):  If all prepared → commit all; else abort all
        """
        txn_id = self._new_txn_id()
        start_time = time.time()

        # ── PHASE 1: PREPARE ──
        prepare_results = []
        prepare_latencies = []
        participants = []

        for range_obj, region, operations in ranges_and_ops:
            leader_region = range_obj.leader_region
            leader_node = regions[leader_region].get_primary()

            if not leader_node or not leader_node.is_alive:
                # Abort all previously prepared
                self._abort_all(participants, txn_id)
                return {
                    "success": False,
                    "error": f"LEADER_UNAVAILABLE for range {range_obj.range_id}",
                    "txn_id": txn_id
                }

            # Cross-region latency to reach this range's leader
            prep_latency = 0.0
            if origin_region != leader_region:
                prep_latency = self.network.simulate_rpc(origin_region, leader_region)
                if prep_latency < 0:
                    self._abort_all(participants, txn_id)
                    return {
                        "success": False,
                        "error": f"PARTITION to {leader_region}",
                        "txn_id": txn_id
                    }

            # Prepare on leader (includes Raft replication of PREPARE record)
            prep_result = leader_node.prepare_transaction(
                range_obj.range_id, txn_id, operations
            )

            if not prep_result["success"]:
                self._abort_all(participants, txn_id)
                return {
                    "success": False,
                    "error": prep_result["error"],
                    "txn_id": txn_id,
                    "phase": "PREPARE"
                }

            # Simulate Raft replication of prepare
            rep_latency = self._simulate_raft_commit(range_obj, leader_region)
            total_prep = prep_latency + rep_latency
            prepare_latencies.append(total_prep)

            participants.append({
                "range": range_obj,
                "region": regions[leader_region],
                "node": leader_node,
                "operations": operations
            })

        # All prepared successfully
        prepare_total = max(prepare_latencies) if prepare_latencies else 0

        # ── PHASE 2: COMMIT ──
        commit_latencies = []
        hlc_ts = regions[origin_region].get_primary().hlc.now()

        for participant in participants:
            leader_region = participant["range"].leader_region
            node = participant["node"]

            commit_result = node.commit_transaction(
                participant["range"].range_id,
                txn_id,
                participant["operations"],
                origin_region,
                hlc_ts
            )

            if not commit_result["success"]:
                # In production: this is a critical error
                # Raft ensures this won't happen if prepare succeeded
                pass

            # Cross-region latency for commit acknowledgment
            commit_lat = 0.0
            if origin_region != leader_region:
                commit_lat = self.network.simulate_rpc(origin_region, leader_region)

            rep_lat = self._simulate_raft_commit(participant["range"], leader_region)
            commit_latencies.append(commit_lat + rep_lat)

        commit_total = max(commit_latencies) if commit_latencies else 0
        total_latency = prepare_total + commit_total
        elapsed_real = (time.time() - start_time) * 1000

        txn_record = {
            "txn_id": txn_id,
            "type": "multi_range_2pc",
            "ranges": [r.range_id for r, _, _ in ranges_and_ops],
            "origin_region": origin_region,
            "prepare_latency_ms": round(prepare_total, 2),
            "commit_latency_ms": round(commit_total, 2),
            "total_latency_ms": round(total_latency, 2),
            "participants": len(participants),
            "success": True
        }
        self.txn_log.append(txn_record)

        return {
            "success": True,
            "txn_id": txn_id,
            "type": "2PC",
            "latency_ms": round(total_latency, 2),
            "prepare_ms": round(prepare_total, 2),
            "commit_ms": round(commit_total, 2),
            "ranges_involved": len(participants)
        }

    def _simulate_raft_commit(self, range_obj: DataRange, leader_region: str) -> float:
        """Simulate Raft replication latency for a range's commit"""
        rep_latencies = []
        for replica_region in range_obj.replicas:
            if replica_region != leader_region:
                lat = self.network.get_latency(leader_region, replica_region)
                rep_latencies.append(lat)

        if not rep_latencies:
            return 0.0

        rep_latencies.sort()
        needed = len(range_obj.replicas) // 2
        if needed > 0 and rep_latencies:
            return rep_latencies[min(needed - 1, len(rep_latencies) - 1)]
        return 0.0

    def _abort_all(self, participants: List[dict], txn_id: str):
        """Abort all previously prepared participants"""
        for p in participants:
            try:
                p["node"].abort_transaction(
                    p["range"].range_id, txn_id, p["operations"]
                )
            except Exception:
                pass

    def get_txn_stats(self) -> dict:
        if not self.txn_log:
            return {"total": 0}

        single = [t for t in self.txn_log if t["type"] == "single_range"]
        multi = [t for t in self.txn_log if t["type"] == "multi_range_2pc"]

        single_latencies = [t.get("simulated_latency_ms", 0) for t in single if t["success"]]
        multi_latencies = [t.get("total_latency_ms", 0) for t in multi if t["success"]]

        stats = {
            "total_transactions": len(self.txn_log),
            "single_range_txns": len(single),
            "multi_range_txns": len(multi),
            "success_rate": sum(1 for t in self.txn_log if t["success"]) / max(len(self.txn_log), 1) * 100
        }

        if single_latencies:
            stats["single_range_avg_ms"] = round(statistics.mean(single_latencies), 2)
            stats["single_range_p50_ms"] = round(
                sorted(single_latencies)[len(single_latencies) // 2], 2)
            stats["single_range_p99_ms"] = round(
                sorted(single_latencies)[int(len(single_latencies) * 0.99)], 2)

        if multi_latencies:
            stats["multi_range_avg_ms"] = round(statistics.mean(multi_latencies), 2)
            stats["multi_range_p50_ms"] = round(
                sorted(multi_latencies)[len(multi_latencies) // 2], 2)
            stats["multi_range_p99_ms"] = round(
                sorted(multi_latencies)[int(len(multi_latencies) * 0.99)], 2)

        return stats

class PlacementManager:
    """
    Optimizes leader placement across regions based on write patterns.
    
    Goal: Place each range's leader in the region that generates
    the most writes for that range → minimizes cross-region commits.
    
    Runs periodically, analyzes write origins, recommends transfers.
    """

    def __init__(self, range_map: RangeMap, regions: Dict[str, Region],
                 network: CrossRegionNetwork):
        self.range_map = range_map
        self.regions = regions
        self.network = network
        self.transfer_history: List[dict] = []

    def analyze_placement(self) -> List[dict]:
        """Analyze current placement and suggest improvements"""
        recommendations = []

        for range_obj in self.range_map.get_all_ranges():
            # Gather write origins from the leader node
            leader_region = range_obj.leader_region
            if leader_region not in self.regions:
                continue

            leader_node = self.regions[leader_region].get_primary()
            if not leader_node:
                continue

            write_stats = leader_node.get_write_origin_stats(range_obj.range_id)
            total_writes = sum(write_stats.values())

            if total_writes < 10:
                continue  # Not enough data to decide

            # Find region with most writes
            best_region = max(write_stats, key=write_stats.get) if write_stats else leader_region
            best_pct = (write_stats.get(best_region, 0) / total_writes * 100) if total_writes > 0 else 0
            current_pct = (write_stats.get(leader_region, 0) / total_writes * 100) if total_writes > 0 else 0

            # Recommend transfer if another region has >20% more writes
            if best_region != leader_region and best_pct - current_pct > 20:
                recommendations.append({
                    "range_id": range_obj.range_id,
                    "current_leader": leader_region,
                    "recommended_leader": best_region,
                    "current_pct": round(current_pct, 1),
                    "recommended_pct": round(best_pct, 1),
                    "total_writes": total_writes,
                    "write_distribution": {k: round(v / total_writes * 100, 1)
                                           for k, v in write_stats.items()}
                })

        return recommendations

    def execute_transfers(self, recommendations: List[dict]) -> List[dict]:
        """Execute recommended leader transfers"""
        results = []

        for rec in recommendations:
            range_id = rec["range_id"]
            new_leader = rec["recommended_leader"]

            success = self.range_map.transfer_leader(range_id, new_leader)

            transfer_record = {
                "range_id": range_id,
                "from": rec["current_leader"],
                "to": new_leader,
                "success": success,
                "reason": f"Write affinity: {rec['recommended_pct']:.0f}% from {new_leader}"
            }
            self.transfer_history.append(transfer_record)
            results.append(transfer_record)

        return results

    def get_placement_summary(self) -> dict:
        """Summary of leader distribution across regions"""
        leaders_per_region = defaultdict(int)
        for range_obj in self.range_map.get_all_ranges():
            leaders_per_region[range_obj.leader_region] += 1

        return {
            "leaders_per_region": dict(leaders_per_region),
            "total_ranges": len(self.range_map.get_all_ranges()),
            "total_transfers": len(self.transfer_history)
        }

class QueryRouter:
    """
    Routes client queries to the correct range and region.
    
    Sits in each region as a "SQL Gateway" (from architecture diagram).
    Knows the range map and can route locally or cross-region.
    """

    def __init__(self, local_region: str, range_map: RangeMap,
                 regions: Dict[str, Region], network: CrossRegionNetwork,
                 txn_coordinator: TransactionCoordinator):
        self.local_region = local_region
        self.range_map = range_map
        self.regions = regions
        self.network = network
        self.txn_coordinator = txn_coordinator

        self.query_count = 0
        self.local_hits = 0
        self.remote_hits = 0
        self._lock = threading.Lock()

    def write(self, key: str, value: Any) -> dict:
        """Route a single-key write to the correct leader"""
        with self._lock:
            self.query_count += 1

        range_obj = self.range_map.find_range(key)
        if not range_obj:
            return {"success": False, "error": f"No range found for key '{key}'"}

        leader_region = range_obj.leader_region
        is_local = (leader_region == self.local_region)

        with self._lock:
            if is_local:
                self.local_hits += 1
            else:
                self.remote_hits += 1

        region = self.regions.get(leader_region)
        if not region or not region.is_available():
            return {"success": False, "error": f"Leader region {leader_region} unavailable"}

        operations = [{"type": "SET", "key": key, "value": value}]
        result = self.txn_coordinator.execute_single_range_txn(
            region, range_obj, operations, self.local_region
        )

        return result

    def read(self, key: str, consistency: str = "leader") -> dict:
        """
        Route a read request.
        
        Consistency levels:
          "leader":  Read from range leader (linearizable)
          "local":   Read from local replica (possibly stale)
          "bounded": Read from local if recent, else leader
        """
        with self._lock:
            self.query_count += 1

        range_obj = self.range_map.find_range(key)
        if not range_obj:
            return {"success": False, "error": f"No range found for key '{key}'"}

        if consistency == "local" or consistency == "bounded":
            # Try local replica first
            local_region = self.regions.get(self.local_region)
            if local_region and local_region.is_available():
                node = local_region.get_primary()
                if node:
                    result = node.read(range_obj.range_id, key)
                    if result["success"]:
                        with self._lock:
                            self.local_hits += 1
                        result["served_from"] = self.local_region
                        result["consistency"] = consistency
                        result["latency_ms"] = round(random.uniform(0.5, 2.0), 2)
                        return result

        # Leader read (linearizable)
        leader_region = range_obj.leader_region
        region = self.regions.get(leader_region)

        if not region or not region.is_available():
            return {"success": False, "error": f"Leader region unavailable"}

        node = region.get_primary()
        if not node:
            return {"success": False, "error": "No primary node"}

        latency = 0.0
        if leader_region != self.local_region:
            latency = self.network.simulate_rpc(self.local_region, leader_region)
            with self._lock:
                self.remote_hits += 1
        else:
            latency = random.uniform(0.5, 2.0)
            with self._lock:
                self.local_hits += 1

        result = node.read(range_obj.range_id, key)
        result["served_from"] = leader_region
        result["consistency"] = "linearizable"
        result["latency_ms"] = round(latency, 2)
        return result

    def multi_key_write(self, operations: List[dict]) -> dict:
        """
        Write to multiple keys that may span different ranges.
        Uses single-range fast path if all in one range.
        Uses 2PC if spanning multiple ranges.
        """
        with self._lock:
            self.query_count += 1

        # Group operations by range
        range_groups: Dict[str, Tuple[DataRange, List[dict]]] = {}

        for op in operations:
            range_obj = self.range_map.find_range(op["key"])
            if not range_obj:
                return {"success": False, "error": f"No range for key '{op['key']}'"}

            if range_obj.range_id not in range_groups:
                range_groups[range_obj.range_id] = (range_obj, [])
            range_groups[range_obj.range_id][1].append(op)

        if len(range_groups) == 1:
            # Single range → fast path
            range_obj, ops = list(range_groups.values())[0]
            region = self.regions.get(range_obj.leader_region)
            if not region:
                return {"success": False, "error": "Leader region unavailable"}

            return self.txn_coordinator.execute_single_range_txn(
                region, range_obj, ops, self.local_region
            )
        else:
            # Multi-range → 2PC
            ranges_and_ops = []
            for range_obj, ops in range_groups.values():
                region = self.regions.get(range_obj.leader_region)
                if not region:
                    return {"success": False, "error": f"Region unavailable for {range_obj.range_id}"}
                ranges_and_ops.append((range_obj, region, ops))

            return self.txn_coordinator.execute_multi_range_txn(
                ranges_and_ops, self.local_region, self.regions
            )

    def get_router_stats(self) -> dict:
        with self._lock:
            total = max(self.query_count, 1)
            return {
                "region": self.local_region,
                "total_queries": self.query_count,
                "local_hits": self.local_hits,
                "remote_hits": self.remote_hits,
                "local_hit_rate_pct": round(self.local_hits / total * 100, 1)
            }

class GlobeDB:
    """
    The complete multi-region distributed database.
    Ties together all components.
    """

    def __init__(self):
        # Network
        self.network = CrossRegionNetwork()

        # Regions
        self.regions: Dict[str, Region] = {}

        # Range map
        self.range_map = RangeMap()

        # Transaction coordinator
        self.txn_coordinator = TransactionCoordinator(self.network)

        # Query routers (one per region)
        self.routers: Dict[str, QueryRouter] = {}

        # Placement manager
        self.placement_manager: Optional[PlacementManager] = None

    def add_region(self, region_id: str, display_name: str,
                   clock_offset_ms: float = 0):
        region = Region(region_id, display_name, clock_offset_ms)
        self.regions[region_id] = region

    def configure_network(self, latencies: Dict[Tuple[str, str], float]):
        for (a, b), latency in latencies.items():
            self.network.set_latency(a, b, latency)

    def create_ranges(self, range_configs: List[dict]):
        """Create data ranges with specified leader and replica placement"""
        for config in range_configs:
            data_range = DataRange(
                range_id=config["range_id"],
                start_key=config["start_key"],
                end_key=config["end_key"],
                leader_region=config["leader"],
                replicas=config["replicas"]
            )
            self.range_map.add_range(data_range)

    def initialize_routers(self):
        """Create a query router for each region"""
        for region_id in self.regions:
            self.routers[region_id] = QueryRouter(
                local_region=region_id,
                range_map=self.range_map,
                regions=self.regions,
                network=self.network,
                txn_coordinator=self.txn_coordinator
            )
        self.placement_manager = PlacementManager(
            self.range_map, self.regions, self.network
        )

    def get_router(self, region_id: str) -> Optional[QueryRouter]:
        return self.routers.get(region_id)

    def print_topology(self):
        print(f"\n  🌍 GLOBEDB TOPOLOGY")
        print(f"  {'─' * 65}")

        print(f"\n  Regions:")
        for rid, region in self.regions.items():
            alive = sum(1 for n in region.nodes.values() if n.is_alive)
            total = len(region.nodes)
            status = "✅" if region.is_available() else "❌"
            print(f"    {status} {rid:<12} ({region.display_name}) "
                  f"— {alive}/{total} nodes alive")

        print(f"\n  Network Latencies (RTT):")
        region_ids = list(self.regions.keys())
        header = f"    {'':>12}"
        for rid in region_ids:
            header += f" {rid:>12}"
        print(header)

        for r1 in region_ids:
            row = f"    {r1:>12}"
            for r2 in region_ids:
                lat = self.network.get_latency(r1, r2)
                if r1 == r2:
                    row += f" {'<2ms':>12}"
                else:
                    row += f" {f'{lat:.0f}ms':>12}"
            print(row)

        print(f"\n  Data Ranges:")
        print(f"    {'Range':<12} {'Keys':<14} {'Leader':<12} {'Replicas'}")
        print(f"    {'─' * 60}")
        for r in self.range_map.get_all_ranges():
            print(f"    {r.range_id:<12} [{r.start_key}-{r.end_key})"
                  f"{'':>3} 👑{r.leader_region:<10} {r.replicas}")

    def print_data_consistency(self):
        """Compare data across all regions for consistency check"""
        print(f"\n  🔍 DATA CONSISTENCY CHECK")
        print(f"  {'─' * 65}")

        for range_obj in self.range_map.get_all_ranges():
            snapshots = {}
            for region_id in range_obj.replicas:
                if region_id in self.regions:
                    node = self.regions[region_id].get_primary()
                    if node and node.is_alive:
                        snapshots[region_id] = node.get_data_snapshot(range_obj.range_id)

            if len(snapshots) < 2:
                continue

            values = list(snapshots.values())
            all_match = all(v == values[0] for v in values)
            status = "✅" if all_match else "⚠️"
            key_count = len(values[0]) if values else 0

            print(f"    {status} {range_obj.range_id}: {key_count} keys — "
                  f"{'CONSISTENT' if all_match else 'DIVERGENT'} "
                  f"across {list(snapshots.keys())}")

            if not all_match:
                for region_id, data in snapshots.items():
                    print(f"       {region_id}: {dict(list(data.items())[:5])}")
