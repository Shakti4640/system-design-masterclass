🎯 HYPOTHETICAL PROBLEM

Your pizza shop is now a NATIONAL chain with a beautiful website
The website serves:
  → HTML pages (50KB each)
  → CSS stylesheets (150KB)
  → JavaScript bundles (400KB)
  → Pizza images (high-res, 500KB each × 30 pizzas = 15MB total)
  → Font files (300KB)
  → Video of pizza making (50MB)
  → Menu PDF downloads (2MB)

Your single origin server is in New York

SCENARIO A: The Geography Problem
  → User in Tokyo visits your website
  → Browser requests: logo.png (50KB)
  → Request travels: Tokyo → Pacific Ocean → San Francisco → New York
  → Round trip time (RTT): ~200ms just for the NETWORK
  → Page has 40 assets (images, CSS, JS, fonts)
  → Even with HTTP/2 multiplexing: 200ms × several round trips = 1-3 seconds
  → Just for static files that are IDENTICAL for every visitor

  → User in London: RTT = ~80ms to New York
  → User in Sydney: RTT = ~250ms to New York
  → User in São Paulo: RTT = ~150ms to New York
  → EVERY user pays the latency tax to reach New York

SCENARIO B: The Bandwidth Problem
  → Your site gets 1,000,000 page views/day
  → Each page loads ~3MB of static assets
  → Total bandwidth: 1M × 3MB = 3TB per DAY
  → Your origin server's network: 1 Gbps
  → 3TB / 86,400 seconds = ~280 Mbps average
  → During peak hours (3x average): ~840 Mbps
  → You're nearly MAXING OUT your server's network
  → And 95% of that bandwidth is serving THE SAME FILES repeatedly

SCENARIO C: The Flash Sale Crash
  → You announce a 50% off Super Bowl special
  → 500,000 people visit your site in 10 minutes
  → Each downloads 3MB of static assets
  → That's 1.5TB in 10 minutes = 20 Gbps needed
  → Your 1 Gbps server: completely overwhelmed
  → Website goes DOWN during your biggest promotion
  → Estimated revenue loss: $200,000

SCENARIO D: The Redundancy Problem
  → Your New York server goes down for maintenance
  → ENTIRE website is unavailable globally
  → Even though all these files are STATIC and could be served from anywhere
  → Single point of failure for content that never changes

THE ANALOGY:
  → You have ONE warehouse in New York
  → Every customer in every city orders from New York
  → Customer in LA waits 3 days for shipping
  → Customer in Tokyo waits 7 days

  → SOLUTION: Open WAREHOUSES in every major city
  → Stock them with your most popular products
  → Customer in LA orders → LA warehouse ships → 1 day
  → Customer in Tokyo → Tokyo warehouse → 1 day
  → New York warehouse is only used when local warehouse doesn't have the item

  → The WAREHOUSES are CDN EDGE SERVERS
  → The PRODUCTS are your STATIC FILES
  → The NEW YORK warehouse is your ORIGIN SERVER
  → "Stocking the warehouse" is CDN CACHING
  → "Item not in local warehouse" is a CACHE MISS

🔍 HOW TO APPROACH IT

Step 1: Identify WHAT is static vs dynamic
         → Static: images, CSS, JS, fonts, videos, PDFs
           → Same content for every user
           → Changes rarely (only on deployment)
         → Dynamic: user profiles, order status, cart contents
           → Different per user
           → Changes frequently
         → CDN is primarily for STATIC content
           (dynamic caching is advanced — covered briefly)

Step 2: Understand WHERE edge servers are
         → CDN providers have servers in 50-300+ cities worldwide
         → Called "Points of Presence" (PoPs)
         → User is routed to NEAREST PoP automatically
         → Routing happens via DNS (Project 7 concepts)

Step 3: Understand HOW content gets to edge servers
         → PULL model: Edge fetches from origin on first request (cache miss)
         → PUSH model: You upload to edge servers proactively
         → Most CDNs use PULL (simpler, on-demand)

Step 4: Understand WHEN to serve cached vs fresh
         → Cache-Control headers tell CDN how long to cache
         → TTL (Time To Live) from Project 5 caching concepts
         → Cache invalidation when content changes

Step 5: Understand the ROUTING mechanism
         → DNS resolves your domain to NEAREST edge IP
         → Anycast: same IP advertised from multiple locations
         → User connects to geographically closest server automatically

Step 6: Design CACHE KEY strategy
         → URL is the primary cache key
         → Query params, headers may vary the cache
         → Versioning: style.css?v=2.0 forces new cache

THE CDN ARCHITECTURE:

  WITHOUT CDN:
  ════════════
  
  User (Tokyo)  ──────────200ms──────────→  Origin (New York)
  User (London) ──────────80ms───────────→  Origin (New York)
  User (Sydney) ──────────250ms──────────→  Origin (New York)
  User (NYC)    ──────────5ms────────────→  Origin (New York)

  ALL requests hit origin, regardless of location


  WITH CDN:
  ═════════
  
  User (Tokyo)  ───5ms───→  Edge (Tokyo)  ──cache miss──→ Origin (NYC)
  User (London) ───5ms───→  Edge (London) ──cache hit!──→ (no origin call)
  User (Sydney) ───5ms───→  Edge (Sydney) ──cache hit!──→ (no origin call)
  User (NYC)    ───5ms───→  Edge (NYC)    ──cache hit!──→ (no origin call)

  First request per region: cache miss → fetch from origin → cache at edge
  All subsequent requests: served from edge → 5ms instead of 200ms
  Origin sees only ~1% of traffic (cache misses only)

TWO CDN MODELS:

MODEL 1: PULL CDN (Origin Pull)
════════════════════════════════
  → Edge server has NO content initially
  → User requests file → edge checks local cache
  → Cache MISS → edge fetches from origin → caches → serves user
  → Cache HIT → serves directly (no origin contact)
  → Content cached based on TTL (Cache-Control headers)

  Flow:
  User → Edge: "Give me logo.png"
  Edge: "I don't have it" (cache miss)
  Edge → Origin: "Give me logo.png"
  Origin → Edge: [logo.png data] + "Cache for 86400 seconds"
  Edge: stores logo.png, sets TTL=86400s
  Edge → User: [logo.png data]
  
  Next user → same Edge: "Give me logo.png"
  Edge: "I have it! TTL still valid" (cache hit)
  Edge → User: [logo.png data]  (origin never contacted)

  PROS: Simple, no upfront work, on-demand
  CONS: First user per region has slow load (cold cache)


MODEL 2: PUSH CDN
══════════════════
  → YOU upload content to CDN edge servers proactively
  → Before any user requests it
  → All edges have content from the start
  → No cold cache problem

  Flow:
  You → CDN API: "Here's logo.png, push to all edges"
  CDN: distributes to 100+ edge servers worldwide
  User → Edge: "Give me logo.png"
  Edge: "I have it!" (always cache hit)
  Edge → User: [logo.png data]

  PROS: No cold cache, predictable performance
  CONS: You manage uploads, wastes space if content not requested
  USE: Large files (videos), predictable content sets


CACHE INVALIDATION STRATEGIES:
═════════════════════════════

  STRATEGY 1: TTL-Based (Time Expiry)
  → Set Cache-Control: max-age=86400 (24 hours)
  → After 24 hours, edge fetches fresh copy
  → Simple, but content is stale for up to 24 hours
  → Good for: images, fonts, CSS/JS with versioned URLs

  STRATEGY 2: Versioned URLs (Cache Busting)
  → style.css?v=1.0 → style.css?v=2.0
  → Or: style.a1b2c3.css → style.d4e5f6.css (content hash)
  → Different URL = different cache entry
  → Old version still served until TTL expires (that's fine)
  → New version fetched immediately (new URL = cache miss)
  → BEST PRACTICE for CSS/JS

  STRATEGY 3: Purge/Invalidate API
  → Call CDN API: "Purge /images/logo.png from all edges"
  → CDN removes cached copy from all edge servers
  → Next request triggers fresh fetch from origin
  → Use sparingly: purging is expensive operation
  → Good for: emergency content removal, critical updates

  STRATEGY 4: Stale-While-Revalidate
  → Cache-Control: max-age=3600, stale-while-revalidate=86400
  → For first 3600s: serve cached (fresh)
  → After 3600s: serve cached (stale) BUT revalidate in background
  → User gets fast response, edge fetches fresh copy async
  → Best user experience: never waits for revalidation

📌 WHAT COULD BE DONE

Option A: No CDN — Single Origin Server
  → All traffic hits origin
  → Simplest setup
  → Fails at scale (Scenario B and C)
  → High latency for distant users (Scenario A)

Option B: Pull CDN (Most Common)
  → Configure CDN to pull from your origin
  → Set Cache-Control headers on responses
  → CDN handles caching, routing, edge distribution
  → Minimal code changes needed
  → THIS IS WHAT WE BUILD

Option C: Push CDN + Object Storage
  → Upload assets to S3/GCS (Project 25 preview)
  → CDN pulls from object storage (not your server)
  → Origin server never serves static files
  → Best for large media files

Option D: Multi-Tier CDN
  → Edge (Tier 1) → Regional (Tier 2) → Origin
  → Cache miss at edge → check regional first
  → Regional might have it (another edge in same region cached it)
  → Reduces origin load even further

Option E: Full Site CDN (Edge Computing)
  → Run application logic AT the edge
  → Not just static files, but dynamic responses too
  → Cloudflare Workers, AWS Lambda@Edge
  → Most advanced, most complex

✅ WHAT SHOULD DO

1. ALWAYS set Cache-Control headers on static assets
   → Images: Cache-Control: public, max-age=31536000 (1 year)
   → CSS/JS: Cache-Control: public, max-age=31536000 (versioned URLs)
   → HTML: Cache-Control: public, max-age=300 (5 minutes, changes more)
   → API responses: Cache-Control: private, no-cache (user-specific)

2. ALWAYS use versioned URLs for CSS/JS
   → style.css?v=abc123 or style.abc123.css
   → When you deploy new CSS → new version in URL → new cache entry
   → Never worry about stale CSS/JS again
   → Build tools (Webpack, Vite) do this automatically

3. ALWAYS set proper Content-Type headers
   → CDN serves files based on Content-Type
   → Wrong type → browser can't render
   → image/png, text/css, application/javascript, etc.

4. ALWAYS use HTTPS between CDN and origin
   → CDN edge ↔ origin communication must be encrypted
   → Prevents man-in-the-middle attacks
   → CDN provides free SSL certificates for edge ↔ user

5. ALWAYS configure custom error pages
   → If origin is down → CDN should serve a cached error page
   → Not a generic "502 Bad Gateway"
   → "We're experiencing issues, please try again shortly"

6. ALWAYS monitor cache hit ratio
   → Target: >90% cache hit ratio for static assets
   → Below 80% → check TTL settings, URL patterns
   → Low hit ratio = CDN isn't helping, you're paying for nothing

7. ALWAYS compress before caching
   → Enable gzip/brotli compression on origin
   → CDN caches the compressed version
   → 50-80% smaller files → faster delivery
   → Most CDNs can compress at edge too

8. ALWAYS separate static and dynamic domains
   → static.pizzashop.com → CDN (all static files)
   → api.pizzashop.com → direct to origin (dynamic API)
   → Clear separation of what's cached vs not

❌ WHAT SHOULD NOT DO

1. NEVER cache user-specific or sensitive content on CDN
   → User profiles, order history, account pages
   → Cache-Control: private (browser can cache, CDN must NOT)
   → CDN caching user A's data → user B might see it
   → SECURITY DISASTER

2. NEVER set infinite TTL without versioned URLs
   → max-age=31536000 on style.css (no version)
   → You update style.css → users still see OLD version for a year
   → FIX: style.css?v=2 or style.abc123.css

3. NEVER ignore cache invalidation in deployment pipeline
   → Deploy new code → new CSS/JS/images
   → Old versions still cached at edges worldwide
   → FIX: Versioned URLs (automatic invalidation)
   → FIX: CDN purge API in deployment script

4. NEVER cache API responses by default
   → POST /orders should NEVER be cached
   → GET /user/profile should NEVER be cached on shared CDN
   → Only cache: truly public, identical-for-everyone responses
   → Exception: GET /menu (public, same for everyone) → can cache briefly

5. NEVER use query strings for cache busting in some CDNs
   → Some CDNs ignore query strings by default
   → style.css?v=1 and style.css?v=2 might serve same cached file
   → FIX: Use filename-based versioning: style.v1.css, style.v2.css
   → FIX: Configure CDN to include query strings in cache key

6. NEVER forget about cache stampede
   → Popular file TTL expires at edge server
   → 1000 simultaneous requests arrive → all are cache misses
   → 1000 requests hit origin simultaneously
   → FIX: "Request coalescing" — CDN makes ONE origin request,
     holds others, serves all from single fetch
   → Most CDNs handle this automatically

7. NEVER serve large files without range request support
   → User downloads 50MB video, connection drops at 40MB
   → Without range requests: restart from 0
   → With range requests: resume from 40MB
   → CDN should support Range headers

🚫 WHEN NOT TO USE

1. DO NOT use CDN for purely dynamic, personalized content
   → "Welcome back, Alice! Your cart has 3 items"
   → This is different for every user
   → Caching would show Alice's data to Bob
   → Serve from origin with Cache-Control: private

2. DO NOT use CDN when content changes every second
   → Live stock ticker, real-time sports scores
   → Even 1-second TTL means stale data
   → Use WebSockets (Project 31) for real-time data

3. DO NOT use CDN for internal/admin tools
   → Admin dashboard used by 5 people in one office
   → CDN adds cost with zero benefit
   → Direct origin access is fine for small user bases

4. DO NOT use CDN when your users are all in one location
   → Company intranet used only from headquarters
   → All users are 5ms from origin already
   → CDN would add a hop, making it slower

5. DO NOT use CDN for POST/PUT/DELETE requests
   → These MODIFY data — caching makes no sense
   → CDN should pass these through to origin immediately
   → Only GET (and sometimes HEAD) are cacheable

6. DO NOT use CDN as a substitute for application optimization
   → Slow API taking 5 seconds → CDN caching for 10 seconds
   → First user still waits 5 seconds
   → Fix the API (indexes — Project 10, queues — Project 11)
   → CDN complements optimization, doesn't replace it

⚙️ HOW IT ONLY WORKS (INTERNAL MECHANICS)

THE COMPLETE CDN REQUEST FLOW:
═════════════════════════════

Step 1: DNS RESOLUTION (Project 7 concepts)
  → User types www.pizzashop.com
  → DNS query: "What's the IP for www.pizzashop.com?"
  → WITHOUT CDN: Returns origin IP (e.g., 45.33.21.10 — New York)
  → WITH CDN: Returns edge IP nearest to user
  
  How CDN DNS works:
  → pizzashop.com CNAME → pizzashop.cdn-provider.net
  → cdn-provider's DNS uses GEOLOCATION
  → User in Tokyo → resolves to 103.21.44.55 (Tokyo edge)
  → User in London → resolves to 104.18.22.33 (London edge)
  → Same domain, different IPs based on location
  
  Alternative: ANYCAST
  → ALL edge servers advertise the SAME IP address
  → Internet routing (BGP) automatically routes to nearest one
  → No special DNS needed
  → Cloudflare uses this approach


Step 2: TLS HANDSHAKE (at edge, not origin)
  → User connects to edge server via HTTPS
  → TLS handshake happens with EDGE (1 round trip = ~5ms)
  → WITHOUT CDN: TLS with origin (1 round trip = 200ms for Tokyo→NYC)
  → CDN terminates TLS at edge = massive latency saving
  
  Without CDN (Tokyo → NYC):
    DNS:  50ms
    TCP:  200ms (SYN → SYN-ACK → ACK)
    TLS:  400ms (2 round trips)
    HTTP: 200ms (request → response)
    TOTAL: 850ms for first byte
  
  With CDN (Tokyo → Tokyo edge):
    DNS:  50ms (cached after first lookup)
    TCP:  5ms
    TLS:  10ms
    HTTP: 5ms (cache hit) or 5ms + 200ms (cache miss, edge→origin)
    TOTAL: 70ms (cache hit) or 270ms (cache miss)


Step 3: CACHE LOOKUP AT EDGE
  → Edge server receives: GET /images/pizza-margherita.jpg
  → Cache key: combination of URL + relevant headers
  
  Cache Key Construction:
    Default: URL path
    → /images/pizza-margherita.jpg
    
    With Vary header: URL + specified headers
    → /images/pizza-margherita.jpg + Accept-Encoding:gzip
    → /images/pizza-margherita.jpg + Accept-Encoding:br
    (Different cached version for gzip vs brotli clients)
    
    With query params: URL + query string
    → /style.css?v=1.0
    → /style.css?v=2.0
    (Different cache entries for different versions)

  Cache Lookup:
  ┌────────────────────────────────────────────────────┐
  │ Edge Cache (hash map / disk storage)               │
  │                                                    │
  │ Key: /images/pizza.jpg                             │
  │   → Content: [binary data, 450KB]                  │
  │   → Content-Type: image/jpeg                       │
  │   → Cached-At: 2025-01-15 10:00:00                │
  │   → TTL: 86400 seconds                            │
  │   → Expires-At: 2025-01-16 10:00:00               │
  │   → ETag: "abc123def456"                           │
  │   → Content-Encoding: gzip                         │
  │   → Size-On-Disk: 320KB (compressed)               │
  │                                                    │
  │ RESULT: HIT ✅ (not expired, serve immediately)     │
  └────────────────────────────────────────────────────┘


Step 4a: CACHE HIT — Serve from Edge
  → Content found in edge cache AND TTL not expired
  → Edge serves directly to user
  → Origin is NEVER contacted
  → Response headers include:
    X-Cache: HIT
    Age: 3600 (seconds since cached)
    CF-Cache-Status: HIT (Cloudflare example)


Step 4b: CACHE MISS — Fetch from Origin
  → Content NOT in edge cache OR TTL expired
  
  Sub-flow:
  1. Edge → Origin: GET /images/pizza.jpg
     If-None-Match: "abc123def456" (ETag from last cache)
  
  2a. Origin: "Content hasn't changed"
      → 304 Not Modified (no body sent!)
      → Edge refreshes TTL, serves cached copy
      → Saves bandwidth (no data transfer)
  
  2b. Origin: "Here's the new content"
      → 200 OK + full file content
      → Edge caches new version
      → Edge serves to user
      → Response: X-Cache: MISS
  
  3. REQUEST COALESCING (thundering herd protection):
     → 100 users request same uncached file simultaneously
     → Edge sends only ONE request to origin
     → Other 99 requests wait
     → Origin responds once → edge serves all 100
     → Origin sees 1 request, not 100


Step 5: CACHE EVICTION (when edge runs out of space)
  → Edge servers have limited disk/memory (typically 1-10 TB)
  → When full, must evict old content
  → LRU (Least Recently Used) — same as Project 5 caching
  → Popular content stays, unpopular content evicted
  → Evicted content fetched from origin on next request


CACHE-CONTROL HEADER DEEP DIVE:
═══════════════════════════════

  Cache-Control: public, max-age=31536000, immutable
  │              │       │                  │
  │              │       │                  └─ Never revalidate (even on refresh)
  │              │       └─ Cache for 1 year (in seconds)
  │              └─ CDN CAN cache this (shared cache)
  └─ The header name

  Common directives:
  ┌────────────────────────┬────────────────────────────────────────┐
  │ Directive              │ Meaning                                │
  ├────────────────────────┼────────────────────────────────────────┤
  │ public                 │ Any cache can store (CDN, browser)     │
  │ private                │ Only browser can cache (NOT CDN)       │
  │ no-cache               │ Must revalidate before using cached    │
  │ no-store               │ Don't cache at all (sensitive data)    │
  │ max-age=N              │ Fresh for N seconds                    │
  │ s-maxage=N             │ CDN-specific max-age (overrides)       │
  │ stale-while-revalidate │ Serve stale while fetching fresh       │
  │ immutable              │ Content will NEVER change at this URL  │
  │ must-revalidate        │ After expiry, MUST check with origin   │
  └────────────────────────┴────────────────────────────────────────┘

  RECOMMENDED SETTINGS:
  → Static assets (versioned): public, max-age=31536000, immutable
  → HTML pages: public, max-age=0, must-revalidate
  → API (public): public, s-maxage=60, stale-while-revalidate=300
  → API (private): private, no-cache
  → Sensitive data: no-store


CONDITIONAL REQUESTS (ETag / Last-Modified):
════════════════════════════════════════════

  First request:
    Client → Edge: GET /logo.png
    Edge → Client: 200 OK
      ETag: "abc123"
      Last-Modified: Mon, 01 Jan 2025 00:00:00 GMT
      Content-Length: 50000
      [50KB of image data]

  Subsequent request (cache expired):
    Client → Edge: GET /logo.png
      If-None-Match: "abc123"
      If-Modified-Since: Mon, 01 Jan 2025 00:00:00 GMT
    
    Edge → Origin: (same conditional headers)
    
    Origin checks: has logo.png changed?
    
    If NOT changed:
      Origin → Edge: 304 Not Modified (NO BODY — saves bandwidth)
      Edge: refreshes TTL, serves cached copy
      Edge → Client: 304 Not Modified
    
    If changed:
      Origin → Edge: 200 OK + new data + new ETag
      Edge: caches new version
      Edge → Client: 200 OK + new data

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: Caching Dynamic/Private Content

# ORIGIN SERVER — Source of truth for all content

class OriginServer:
    """
    The original server where content lives
    CDN edge servers fetch from here on cache miss
    """
    
    def __init__(self, server_location: str = "New York"):
        self.location = server_location
        self.content = {}
        self.request_count = 0
        self.bandwidth_served_bytes = 0
        self.lock = threading.Lock()
        
        # Simulated network latency from origin to different regions
        self.latency_to_region = {
            "New York": 5,
            "Los Angeles": 40,
            "London": 80,
            "Frankfurt": 90,
            "Tokyo": 200,
            "Sydney": 250,
            "São Paulo": 150,
            "Singapore": 220,
            "Mumbai": 180,
        }
    
    def add_content(self, path: str, body: bytes, content_type: str,
                    cache_control: str = "public, max-age=86400"):
        """Add or update content on origin"""
        etag = hashlib.md5(body).hexdigest()[:16]
        self.content[path] = {
            "body": body,
            "content_type": content_type,
            "etag": etag,
            "size_bytes": len(body),
            "cache_control": cache_control,
            "last_modified": time.time()
        }
    
    def handle_request(self, path: str, if_none_match: str = None,
                       requesting_location: str = "Unknown") -> dict:
        """
        Handle a request from an edge server
        Supports conditional requests (304 Not Modified)
        """
        with self.lock:
            self.request_count += 1
        
        # Simulate network latency
        latency_ms = self.latency_to_region.get(requesting_location, 100)
        time.sleep(latency_ms / 1000)
        
        if path not in self.content:
            return {
                "status": 404,
                "body": b"Not Found",
                "headers": {},
                "latency_ms": latency_ms
            }
        
        content = self.content[path]
        
        # Conditional request: check if content changed
        if if_none_match and if_none_match == content["etag"]:
            return {
                "status": 304,
                "body": None,  # No body sent — saves bandwidth
                "headers": {
                    "ETag": content["etag"],
                    "Cache-Control": content["cache_control"]
                },
                "latency_ms": latency_ms,
                "bandwidth_saved_bytes": content["size_bytes"]
            }
        
        # Full response
        with self.lock:
            self.bandwidth_served_bytes += content["size_bytes"]
        
        return {
            "status": 200,
            "body": content["body"],
            "headers": {
                "Content-Type": content["content_type"],
                "Content-Length": str(content["size_bytes"]),
                "ETag": content["etag"],
                "Cache-Control": content["cache_control"],
                "Last-Modified": str(content["last_modified"])
            },
            "latency_ms": latency_ms
        }
    
    def get_stats(self) -> dict:
        return {
            "location": self.location,
            "total_requests": self.request_count,
            "bandwidth_served_mb": round(
                self.bandwidth_served_bytes / (1024 * 1024), 2),
            "content_items": len(self.content)
        }

# EDGE SERVER — CDN Point of Presence
# ============================================================

class EdgeServer:
    """
    One CDN edge server (Point of Presence)
    Caches content locally, fetches from origin on miss
    """
    
    def __init__(self, location: str, origin: OriginServer,
                 cache_size_mb: int = 100):
        self.location = location
        self.origin = origin
        self.cache_size_mb = cache_size_mb
        self.cache = {}  # path → cached entry
        self.cache_used_bytes = 0
        self.lock = threading.Lock()
        
        # In-flight requests for request coalescing
        self.inflight = {}
        self.inflight_lock = threading.Lock()
        
        # Metrics
        self.metrics = {
            "cache_hits": 0,
            "cache_misses": 0,
            "cache_revalidations": 0,
            "cache_304s": 0,
            "total_requests": 0,
            "bandwidth_served_bytes": 0,
            "bandwidth_from_origin_bytes": 0,
            "total_latency_ms": 0,
        }
    
    def _parse_max_age(self, cache_control: str) -> int:
        """Extract max-age value from Cache-Control header"""
        if not cache_control:
            return 0
        for directive in cache_control.split(","):
            directive = directive.strip()
            if directive.startswith("max-age="):
                try:
                    return int(directive.split("=")[1])
                except ValueError:
                    return 0
        return 0
    
    def _is_cacheable(self, cache_control: str) -> bool:
        """Determine if response should be cached"""
        if not cache_control:
            return False
        lower = cache_control.lower()
        if "no-store" in lower or "private" in lower:
            return False
        if "public" in lower or "max-age" in lower or "s-maxage" in lower:
            return True
        return False
    
    def _is_fresh(self, entry: dict) -> bool:
        """Check if cached entry is still within TTL"""
        age = time.time() - entry["cached_at"]
        return age < entry["max_age"]
    
    def _evict_if_needed(self, needed_bytes: int):
        """LRU eviction when cache is full (Project 5 concept)"""
        max_bytes = self.cache_size_mb * 1024 * 1024
        
        while self.cache_used_bytes + needed_bytes > max_bytes and self.cache:
            # Find least recently used entry
            oldest_path = None
            oldest_time = float('inf')
            
            for path, entry in self.cache.items():
                if entry["last_accessed"] < oldest_time:
                    oldest_time = entry["last_accessed"]
                    oldest_path = path
            
            if oldest_path:
                evicted = self.cache.pop(oldest_path)
                self.cache_used_bytes -= evicted["size_bytes"]
    
    def handle_request(self, path: str) -> dict:
        """
        Handle a client request:
        1. Check local cache
        2. If hit and fresh → serve
        3. If hit but stale → revalidate with origin
        4. If miss → fetch from origin → cache → serve
        """
        request_start = time.time()
        
        with self.lock:
            self.metrics["total_requests"] += 1
        
        # ---- Step 1: Check local cache ----
        with self.lock:
            cached = self.cache.get(path)
        
        if cached:
            if self._is_fresh(cached):
                # CACHE HIT — serve directly
                with self.lock:
                    cached["last_accessed"] = time.time()
                    cached["hit_count"] += 1
                    self.metrics["cache_hits"] += 1
                    self.metrics["bandwidth_served_bytes"] += cached["size_bytes"]
                
                latency = 2  # Local cache lookup: ~2ms
                with self.lock:
                    self.metrics["total_latency_ms"] += latency
                
                return {
                    "status": 200,
                    "body": cached["body"],
                    "headers": {
                        **cached["headers"],
                        "X-Cache": "HIT",
                        "X-Cache-Edge": self.location,
                        "Age": str(int(time.time() - cached["cached_at"]))
                    },
                    "latency_ms": latency,
                    "cache_status": "HIT"
                }
            else:
                # STALE — revalidate with origin
                with self.lock:
                    self.metrics["cache_revalidations"] += 1
                
                origin_resp = self.origin.handle_request(
                    path,
                    if_none_match=cached.get("etag"),
                    requesting_location=self.location
                )
                
                if origin_resp["status"] == 304:
                    # Not modified — refresh TTL, serve cached
                    with self.lock:
                        cached["cached_at"] = time.time()
                        cached["last_accessed"] = time.time()
                        self.metrics["cache_304s"] += 1
                        self.metrics["cache_hits"] += 1
                        self.metrics["bandwidth_served_bytes"] += cached["size_bytes"]
                        self.metrics["total_latency_ms"] += origin_resp["latency_ms"]
                    
                    return {
                        "status": 200,
                        "body": cached["body"],
                        "headers": {
                            **cached["headers"],
                            "X-Cache": "REVALIDATED",
                            "X-Cache-Edge": self.location,
                            "Age": "0"
                        },
                        "latency_ms": origin_resp["latency_ms"],
                        "cache_status": "REVALIDATED"
                    }
                
                # Content changed — cache new version
                self._store_in_cache(path, origin_resp)
                
                with self.lock:
                    self.metrics["cache_misses"] += 1
                    self.metrics["total_latency_ms"] += origin_resp["latency_ms"]
                
                return {
                    "status": origin_resp["status"],
                    "body": origin_resp["body"],
                    "headers": {
                        **origin_resp["headers"],
                        "X-Cache": "UPDATED",
                        "X-Cache-Edge": self.location,
                    },
                    "latency_ms": origin_resp["latency_ms"],
                    "cache_status": "UPDATED"
                }
        
        # ---- Step 2: Cache MISS — fetch from origin ----
        
        # Request coalescing: if another request for same path is in-flight,
        # wait for it instead of making another origin request
        with self.inflight_lock:
            if path in self.inflight:
                event = self.inflight[path]
        
        # Fetch from origin
        with self.lock:
            self.metrics["cache_misses"] += 1
        
        origin_resp = self.origin.handle_request(
            path, requesting_location=self.location
        )
        
        if origin_resp["status"] == 200 and origin_resp["body"]:
            self._store_in_cache(path, origin_resp)
            
            with self.lock:
                self.metrics["bandwidth_from_origin_bytes"] += len(origin_resp["body"])
                self.metrics["bandwidth_served_bytes"] += len(origin_resp["body"])
                self.metrics["total_latency_ms"] += origin_resp["latency_ms"]
        
        return {
            "status": origin_resp["status"],
            "body": origin_resp["body"],
            "headers": {
                **origin_resp.get("headers", {}),
                "X-Cache": "MISS",
                "X-Cache-Edge": self.location,
            },
            "latency_ms": origin_resp["latency_ms"],
            "cache_status": "MISS"
        }
    
    def _store_in_cache(self, path: str, origin_resp: dict):
        """Store origin response in local cache"""
        cache_control = origin_resp["headers"].get("Cache-Control", "")
        
        if not self._is_cacheable(cache_control):
            return
        
        max_age = self._parse_max_age(cache_control)
        if max_age <= 0:
            return
        
        body = origin_resp["body"]
        size = len(body) if body else 0
        
        with self.lock:
            self._evict_if_needed(size)
            
            self.cache[path] = {
                "body": body,
                "headers": origin_resp["headers"],
                "etag": origin_resp["headers"].get("ETag"),
                "max_age": max_age,
                "cached_at": time.time(),
                "last_accessed": time.time(),
                "size_bytes": size,
                "hit_count": 0
            }
            self.cache_used_bytes += size
    
    def purge(self, path: str) -> bool:
        """Remove specific path from cache (invalidation)"""
        with self.lock:
            if path in self.cache:
                self.cache_used_bytes -= self.cache[path]["size_bytes"]
                del self.cache[path]
                return True
            return False
    
    def purge_all(self):
        """Clear entire cache"""
        with self.lock:
            self.cache.clear()
            self.cache_used_bytes = 0
    
    def get_metrics(self) -> dict:
        with self.lock:
            total = self.metrics["total_requests"]
            hits = self.metrics["cache_hits"]
            hit_ratio = (hits / total * 100) if total > 0 else 0
            avg_latency = (self.metrics["total_latency_ms"] / total) if total > 0 else 0
            
            return {
                "location": self.location,
                "total_requests": total,
                "cache_hits": hits,
                "cache_misses": self.metrics["cache_misses"],
                "cache_revalidations": self.metrics["cache_revalidations"],
                "cache_304s": self.metrics["cache_304s"],
                "hit_ratio_pct": round(hit_ratio, 1),
                "avg_latency_ms": round(avg_latency, 1),
                "cache_entries": len(self.cache),
                "cache_size_mb": round(self.cache_used_bytes / (1024*1024), 2),
                "bandwidth_served_mb": round(
                    self.metrics["bandwidth_served_bytes"] / (1024*1024), 2),
                "bandwidth_from_origin_mb": round(
                    self.metrics["bandwidth_from_origin_bytes"] / (1024*1024), 2),
            }

class CDN:
    """
    The CDN network: DNS routing + edge servers + origin
    Routes users to nearest edge server based on location
    """
    
    def __init__(self, origin: OriginServer):
        self.origin = origin
        self.edges = {}
        self.user_routing_log = []
    
    def add_edge(self, location: str, cache_size_mb: int = 100):
        """Deploy an edge server in a location"""
        edge = EdgeServer(location, self.origin, cache_size_mb)
        self.edges[location] = edge
        return edge
    
    def route_request(self, user_location: str, path: str) -> dict:
        """
        DNS-based routing: find nearest edge server
        This is what happens when user's browser resolves the CDN domain
        """
        # Find nearest edge (simplified geo-routing)
        nearest_edge = self._find_nearest_edge(user_location)
        
        if nearest_edge is None:
            # No edge servers — go directly to origin
            resp = self.origin.handle_request(
                path, requesting_location=user_location)
            resp["routed_to"] = "ORIGIN (no edge)"
            return resp
        
        # Route to nearest edge
        resp = nearest_edge.handle_request(path)
        resp["routed_to"] = f"Edge: {nearest_edge.location}"
        resp["user_location"] = user_location
        
        return resp
    
    def _find_nearest_edge(self, user_location: str) -> Optional[EdgeServer]:
        """Simple nearest-edge lookup"""
        if user_location in self.edges:
            return self.edges[user_location]
        
        # If no exact match, find closest by origin latency as proxy
        if not self.edges:
            return None
        
        # Return first available edge as fallback
        return list(self.edges.values())[0]
    
    def purge_path(self, path: str):
        """Purge a path from ALL edge servers"""
        purged = 0
        for edge in self.edges.values():
            if edge.purge(path):
                purged += 1
        return purged
    
    def purge_all(self):
        """Purge all content from all edges"""
        for edge in self.edges.values():
            edge.purge_all()
    
    def get_network_stats(self) -> dict:
        edge_stats = {}
        for location, edge in self.edges.items():
            edge_stats[location] = edge.get_metrics()
        
        return {
            "origin": self.origin.get_stats(),
            "edges": edge_stats,
            "total_edge_servers": len(self.edges)
        }

import time
import random


class CacheControlSimulator:
    """
    Simulates how different Cache-Control headers affect
    CDN edge server behavior over time
    """
    
    def __init__(self):
        self.cache = {}
        self.origin_requests = 0
        self.cache_hits = 0
        self.stale_serves = 0
    
    def reset(self):
        self.cache = {}
        self.origin_requests = 0
        self.cache_hits = 0
        self.stale_serves = 0
    
    def parse_cache_control(self, header: str) -> dict:
        """Parse Cache-Control header into directives"""
        directives = {}
        for part in header.split(","):
            part = part.strip().lower()
            if "=" in part:
                key, value = part.split("=", 1)
                directives[key.strip()] = value.strip()
            else:
                directives[part] = True
        return directives
    
    def simulate_request(self, path: str, cache_control: str,
                         current_time: float) -> dict:
        """Simulate a single request with given Cache-Control"""
        directives = self.parse_cache_control(cache_control)
        
        # no-store: never cache
        if directives.get("no-store"):
            self.origin_requests += 1
            return {
                "action": "ORIGIN_FETCH",
                "reason": "no-store directive — caching prohibited",
                "cached": False
            }
        
        # private: CDN must not cache (browser can)
        if directives.get("private"):
            self.origin_requests += 1
            return {
                "action": "ORIGIN_FETCH",
                "reason": "private directive — CDN cannot cache",
                "cached": False
            }
        
        # Check if we have a cached version
        cached = self.cache.get(path)
        
        if cached:
            age = current_time - cached["stored_at"]
            max_age = cached.get("max_age", 0)
            s_maxage = cached.get("s_maxage")
            
            # s-maxage overrides max-age for shared caches (CDN)
            effective_max_age = s_maxage if s_maxage is not None else max_age
            
            if age <= effective_max_age:
                # FRESH — serve from cache
                self.cache_hits += 1
                return {
                    "action": "CACHE_HIT",
                    "reason": f"Fresh (age={age:.0f}s, max-age={effective_max_age}s)",
                    "age": int(age),
                    "cached": True
                }
            
            # STALE — check stale-while-revalidate
            swr = cached.get("stale_while_revalidate", 0)
            if swr and age <= effective_max_age + swr:
                # Serve stale, revalidate in background
                self.stale_serves += 1
                self.origin_requests += 1  # Background revalidation
                
                # Update cache with fresh content (simulated)
                cached["stored_at"] = current_time
                
                return {
                    "action": "STALE_WHILE_REVALIDATE",
                    "reason": (f"Stale but within SWR window "
                              f"(age={age:.0f}s, max-age={effective_max_age}s, "
                              f"swr={swr}s)"),
                    "age": int(age),
                    "cached": True,
                    "background_revalidation": True
                }
            
            # Check must-revalidate
            if directives.get("must-revalidate") or cached.get("must_revalidate"):
                self.origin_requests += 1
                # Simulate 304 (content unchanged 80% of time)
                if random.random() < 0.8:
                    cached["stored_at"] = current_time
                    self.cache_hits += 1
                    return {
                        "action": "REVALIDATED_304",
                        "reason": "must-revalidate: origin confirmed unchanged",
                        "cached": True
                    }
                else:
                    cached["stored_at"] = current_time
                    return {
                        "action": "REVALIDATED_200",
                        "reason": "must-revalidate: origin sent new content",
                        "cached": True
                    }
            
            # Default: stale, fetch new
            self.origin_requests += 1
            cached["stored_at"] = current_time
            return {
                "action": "STALE_REFETCH",
                "reason": f"TTL expired (age={age:.0f}s > max-age={effective_max_age}s)",
                "cached": True
            }
        
        # no-cache: always revalidate (but can cache)
        if directives.get("no-cache"):
            self.origin_requests += 1
            
            # Still cache it for conditional requests
            max_age = int(directives.get("max-age", 0))
            self.cache[path] = {
                "stored_at": current_time,
                "max_age": 0,  # Always revalidate
                "must_revalidate": True
            }
            
            return {
                "action": "ORIGIN_FETCH",
                "reason": "no-cache: must revalidate every time",
                "cached": True
            }
        
        # CACHE MISS — fetch and cache
        self.origin_requests += 1
        
        max_age = int(directives.get("max-age", 0))
        s_maxage = directives.get("s-maxage")
        if s_maxage:
            s_maxage = int(s_maxage)
        swr = directives.get("stale-while-revalidate")
        if swr:
            swr = int(swr)
        
        if max_age > 0 or (s_maxage and s_maxage > 0):
            self.cache[path] = {
                "stored_at": current_time,
                "max_age": max_age,
                "s_maxage": s_maxage,
                "stale_while_revalidate": swr or 0,
                "must_revalidate": bool(directives.get("must-revalidate")),
                "immutable": bool(directives.get("immutable"))
            }
        
        return {
            "action": "CACHE_MISS",
            "reason": "Content not in cache — fetched from origin",
            "cached": max_age > 0 or (s_maxage and s_maxage > 0)
        }

class DeploymentSimulator:
    """
    Simulates deploying new versions of a website
    with and without URL versioning
    """
    
    def __init__(self):
        self.cdn_cache = {}  # Simulated CDN edge cache
        self.origin_content = {}  # Current content on origin

            "updated_at": time.time()
        }
    
    def request(self, path: str, current_time: float) -> dict:
        """Simulate a CDN request"""
        cached = self.cdn_cache.get(path)
        
        if cached and (current_time - cached["cached_at"]) < self.cache_ttl:
            return {
                "source": "CDN_CACHE",
                "content": cached["body"],
                "content_hash": cached["hash"],
                "age_seconds": int(current_time - cached["cached_at"])
            }
        
        # Cache miss — fetch from origin
        origin = self.origin_content.get(path)
        if origin is None:
            return {"source": "404", "content": None}
        
        # Cache at edge
        self.cdn_cache[path] = {
            "body": origin["body"],
            "hash": origin["hash"],
            "cached_at": current_time
        }
        
        return {
            "source": "ORIGIN",
            "content": origin["body"],
            "content_hash": origin["hash"],
            "age_seconds": 0
        }
    
    def purge(self, path: str):
        """Purge path from CDN cache"""
        if path in self.cdn_cache:
            del self.cdn_cache[path]
