🎯 HYPOTHETICAL PROBLEM

You are the CTO of a growing food delivery platform

6 months ago:
  → 1 monolith application
  → Everything on ONE server at IP 10.0.0.5
  → Every service knew where to find everything
  → Life was simple

TODAY — you have microservices:
  → Order Service ......... runs on 3 instances
  → Payment Service ....... runs on 2 instances
  → Restaurant Service .... runs on 4 instances
  → Delivery Service ...... runs on 3 instances
  → Notification Service .. runs on 2 instances
  → User Service .......... runs on 2 instances

TOTAL: 16 service instances across multiple machines

THE NIGHTMARE BEGINS:
  → Order Service needs to call Payment Service
  → But Payment Service runs on 10.0.0.12 AND 10.0.0.13
  → Which one? Is 10.0.0.12 even alive right now?
  
  → You deploy a NEW instance of Restaurant Service on 10.0.0.20
  → How does ANYONE know this new instance exists?
  
  → 10.0.0.13 (Payment) crashes at 3 AM
  → Order Service keeps sending requests to a dead machine
  → Orders start failing, customers leave
  
  → You scale Delivery Service from 3 to 7 instances for weekend rush
  → Monday you scale back to 3
  → Who updates all the addresses? Manually? Every time?

THE REAL QUESTION:
  → In a world where services are BORN, DIE, MOVE, and MULTIPLY constantly
  → How does ANY service find ANY other service... RELIABLY?

THIS is the Service Discovery problem

🔍 HOW TO APPROACH IT

Think of it like a HOSPITAL DIRECTORY:

WITHOUT a directory:
  → "I need a cardiologist"
  → Walk to Room 201... empty
  → Walk to Room 305... that's dermatology now
  → Walk to Room 412... cardiologist moved to Building B
  → You collapse before finding one

WITH a directory (reception desk):
  → "I need a cardiologist"
  → Reception checks the LIVE directory
  → "Dr. Smith is available in Room 201, Dr. Patel in Room 508"
  → You go directly to an available one

THE DIRECTORY must handle:
  1. REGISTRATION — new doctor joins → added to directory
  2. DEREGISTRATION — doctor leaves → removed from directory
  3. HEALTH TRACKING — doctor on break → marked unavailable
  4. LOOKUP — patient asks → get list of available doctors
  5. CHANGE DETECTION — room changed → directory updates automatically

Map to microservices:
  Doctor          = Service Instance
  Room Number     = IP:Port
  Specialty       = Service Name
  Reception Desk  = Service Registry
  Patient         = Calling Service
  Health Check    = Heartbeat (Project 15)

    2. LB asks registry: "Where are Payment instances?"

    1. Client resolves "payment-service.internal" via DNS
    2. DNS returns one or more IP addresses
    3. Client connects to returned IP

📌 WHAT COULD BE DONE

OPTION A: Static Configuration File
  → Hardcode all service addresses in a config file
  → Every service reads this file at startup
  → PROBLEM: Manual updates, no dynamic scaling
  → USE: Only for tiny, stable environments

OPTION B: DNS-Based (Approach 3)
  → Use internal DNS with short TTL
  → Services register DNS records
  → PROBLEM: DNS caching causes stale entries
  → TTL propagation delay (Project 7 explained this)
  → USE: Simple setups, Kubernetes internal DNS

OPTION C: Dedicated Service Registry (Approaches 1 & 2)
  → Consul, Eureka, etcd, ZooKeeper
  → Services self-register on startup
  → Registry health-checks services (Project 15)
  → Dead services automatically removed
  → USE: Production microservices — THIS is the standard

OPTION D: Service Mesh (Sidecar Pattern)
  → Every service gets a "sidecar" proxy (Envoy, Istio)
  → Sidecar handles discovery, routing, retries
  → Application code doesn't even know about discovery
  → USE: Large-scale production (beyond this project's scope)

FOR THIS PROJECT → We build Option C (Dedicated Registry)
  with both Client-Side and Server-Side discovery patterns

✅ WHAT SHOULD DO

1. SELF-REGISTRATION on startup
   → Service starts → immediately registers with registry
   → Sends: service name, instance ID, host, port, health URL
   → No manual intervention required

2. HEARTBEAT continuously
   → Project 15 pattern: periodic "I'm alive" signals
   → If heartbeat stops → registry marks instance as DOWN
   → Configurable interval (every 10-30 seconds)

3. GRACEFUL DEREGISTRATION on shutdown
   → Service shutting down → tells registry "I'm leaving"
   → Immediate removal, no waiting for heartbeat timeout
   → Clean exit vs crash exit handling

4. CACHE the registry locally on clients
   → Don't query registry for EVERY request
   → Cache the service list with short TTL
   → Reduces registry load dramatically

5. HEALTH-AWARE routing
   → Only return HEALTHY instances to callers
   → Combine with health checks from Project 15
   → Don't send traffic to degraded instances

6. SUPPORT metadata
   → Register not just IP:Port but also:
   → Version, region, environment, weight, tags
   → Enables canary deployments, A/B routing

❌ WHAT SHOULD NOT DO

1. NEVER hardcode service addresses
   → This is the ENTIRE problem we're solving
   → Even "just for now" leads to production incidents
   → Always go through discovery, even in development

2. NEVER make the registry a single point of failure
   → Registry itself must be clustered/replicated
   → If registry dies → nobody can find anything
   → Run 3-5 registry nodes minimum in production

3. NEVER skip health checks on registered services
   → A registered but DEAD service is worse than no service
   → Clients will route to it, get errors, waste time
   → Always combine registration with health monitoring

4. NEVER use long TTLs for service cache
   → Services can die between cache refreshes
   → 10-30 seconds max for service list cache
   → Contrast with DNS TTL issues from Project 7

5. NEVER expose the registry to external networks
   → Registry is INTERNAL infrastructure
   → Contains all your service topology
   → Expose through API Gateway (Project 19) only

6. NEVER let zombie registrations accumulate
   → Service crashed without deregistering
   → Heartbeat timeout must REMOVE it automatically
   → Implement reaping/eviction for dead instances

🚫 WHEN NOT TO USE

1. Monolith application
   → All components in one process
   → No network calls between services
   → Service discovery adds unnecessary complexity

2. Serverless / FaaS (Lambda, Cloud Functions)
   → Platform handles routing entirely
   → You call by function name, platform resolves
   → Discovery is abstracted away by the cloud provider

3. Static infrastructure that NEVER changes
   → Same 3 servers for 5 years
   → Load balancer with fixed backend list
   → Registry overhead not justified

4. Extremely latency-sensitive hot paths
   → Registry lookup adds milliseconds
   → For sub-millisecond requirements
   → Use pre-resolved, cached connections
   → Or compile-time service addresses

5. Tight-coupled services that ALWAYS deploy together
   → If Service A and Service B always live on same host
   → Use localhost communication
   → Discovery overhead unnecessary

  ┌──────────────┐                       ┌──────────────┐

  │     up       │                       │              │

  │              │  {name, host, port,   │    instance  │

  │              │   metadata}           │              │

  │  4. Start    │                       │              │

  │              │                       │              │
  └──────────────┘                       └──────────────┘

  LEASE MECHANISM:
    → Registry gives instance a "lease" (e.g., 30 seconds)
    → Instance must RENEW before lease expires
    → If lease expires → instance is EVICTED
    → This handles crash scenarios (no graceful shutdown)
    → Similar to how DHCP leases work for IP addresses

  LOAD BALANCING STRATEGIES (Client picks):
    → Round Robin: cycle through instances in order
    → Random: pick randomly (surprisingly effective)
    → Least Connections: pick least busy (needs tracking)
    → Weighted: prefer instances with higher weight metadata
    → Same logic as Project 6, but done by the CLIENT

COMPONENT 4: EVICTION / REAPING FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Background Thread in Registry (runs every N seconds):

  for each service_name in registry:
      for each instance in service_name:
          time_since_heartbeat = now - instance.last_heartbeat
          
          if time_since_heartbeat > lease_duration:
              mark instance as DOWN
          
          if time_since_heartbeat > (lease_duration * 3):
              REMOVE instance entirely (evict)
              
              notify_watchers(service_name, "instance_removed", instance)

  WHY two thresholds?
    → First threshold (1x lease): Mark DOWN
      → Stops routing traffic to it
      → But keeps the record (might come back)
    
    → Second threshold (3x lease): Full EVICTION
      → Remove from registry entirely
      → Instance is considered permanently gone
      → Prevents zombie entries accumulating

COMPONENT 5: CHANGE NOTIFICATION (WATCH/SUBSCRIBE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Instead of polling the registry repeatedly:

  ┌──────────┐     SUBSCRIBE("payment")     ┌──────────┐
  │  Order   │─────────────────────────────→ │ Registry │
  │ Service  │                               │          │
  │          │←── EVENT: instance added ─────│          │
  │          │←── EVENT: instance removed ───│          │
  │          │←── EVENT: instance DOWN ──────│          │
  └──────────┘                               └──────────┘

  Push-based updates:
    → Client subscribes to changes for a service name
    → Registry pushes events when instances change
    → Client updates local cache immediately
    → No polling delay, instant awareness
    → Uses long-polling or WebSocket connections

COMPONENT 6: REGISTRY HIGH AVAILABILITY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Single registry = single point of failure
  
  Solution: Cluster of registry nodes

  ┌────────────┐   ┌────────────┐   ┌────────────┐
  │ Registry-1 │←─→│ Registry-2 │←─→│ Registry-3 │
  │  (Leader)  │   │ (Follower) │   │ (Follower) │
  └────────────┘   └────────────┘   └────────────┘
       ↑                ↑                ↑
       └────── Replicate state ──────────┘

  Two strategies:
  
  A) Peer-to-Peer Replication (Eureka style)
     → All nodes are equal
     → Writes go to any node, replicate to others
     → Eventually consistent (AP in CAP theorem)
     → If nodes disagree → resolved on next sync
  
  B) Leader-Follower (Consul/etcd/ZooKeeper style)
     → One leader accepts writes
     → Followers replicate from leader
     → Strongly consistent (CP in CAP theorem)
     → Uses consensus protocol (Raft — Project 45)

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: Registry Becomes the Bottleneck
  → Every service queries registry for every call
  → Registry gets thousands of queries per second
  → Registry slows down → ALL services slow down
  → FIX: Client-side caching with short TTL (10-30s)
  → FIX: Use watch/subscribe instead of polling

PITFALL 2: Stale Cache After Instance Death
  → Instance dies at T=0
  → Client cache TTL expires at T=15
  → For 15 seconds, client routes to dead instance
  → FIX: Implement circuit breaker (Project 21)
  → FIX: On connection failure → evict from local cache → re-query registry
  → FIX: Use watch/subscribe for instant invalidation

PITFALL 3: Split Brain in Registry Cluster
  → Network partition separates registry nodes
  → Each partition thinks it's the source of truth
  → Services register with different partitions
  → FIX: Quorum-based writes (majority must agree)
  → FIX: Self-preservation mode (Eureka approach)
  → Eureka: If too many instances disappear suddenly
  →         assume it's a NETWORK issue, not mass death
  →         KEEP existing registrations instead of evicting

PITFALL 4: Thundering Herd on Registry Restart
  → Registry restarts → all instances re-register simultaneously
  → Thousands of registration requests at once
  → FIX: Jittered retry (random delay before re-registration)
  → FIX: Registry persists state to disk, recovers on restart

PITFALL 5: Forgetting to Deregister
  → Service shuts down but doesn't deregister
  → Instance stays in registry until lease expires
  → Traffic still routes there during that window
  → FIX: Shutdown hooks that call deregister
  → FIX: Short lease durations (30s, not 5 minutes)

PITFALL 6: Health Check Misconfiguration
  → Health endpoint returns 200 but service is actually broken
  → (e.g., /health returns OK but database connection is dead)
  → Registry thinks instance is healthy → routes traffic → failures
  → FIX: Deep health checks (Project 15 — liveness vs readiness)
  → FIX: Include dependency checks in health response

PITFALL 7: Version Mismatch Routing
  → Payment Service v1 and v2 both registered
  → Client doesn't specify version
  → Request meant for v2 goes to v1 → incompatible response
  → FIX: Register version as metadata
  → FIX: Filter by version during discovery

📈 SCALABILITY CONSIDERATIONS

REGISTRY ITSELF:
  → Single registry node: handles ~10,000 instances easily
  → Consul benchmarks: 5,000+ registrations/second
  → etcd benchmarks: 10,000+ reads/second
  → For most companies, 3-node cluster is sufficient
  → Netflix (Eureka): handles 100,000+ instances

CLIENT CACHING MATH:
  → 50 services × 10 instances each = 500 entries
  → Each entry ~500 bytes = 250 KB total cache
  → Trivial memory footprint
  → Cache refresh every 15s = 4 queries/minute per client
  → 100 clients × 4 queries/min = 400 queries/min to registry
  → Very manageable

# PROJECT 20A: SERVICE REGISTRY — THE CENTRAL BRAIN
# A fully functional service registry with:
#   → Registration / Deregistration
#   → Heartbeat tracking & lease-based eviction
#   → Health-aware discovery
#   → Watch/Subscribe for change notifications
#   → Metadata support (version, region, tags)
# ============================================================

import threading
import time
import json
import uuid
from http.server import HTTPServer, BaseHTTPRequestHandler
from collections import defaultdict
from datetime import datetime, timedelta


class ServiceRegistry:
    """
    The central registry that knows about ALL service instances.
    Think of it as the hospital reception desk.
    """

    def __init__(self, default_lease_seconds=30, eviction_multiplier=3):
        # Two-level map: service_name → instance_id → instance_info
        self.registry = defaultdict(dict)
        # Lock for thread-safe access (multiple services register concurrently)
        self.lock = threading.RLock()
        # Default lease: how long before an instance is considered dead
        self.default_lease_seconds = default_lease_seconds
        # After lease * multiplier with no heartbeat → full eviction
        self.eviction_multiplier = eviction_multiplier
        # Watchers: service_name → list of callback queues
        self.watchers = defaultdict(list)
        # Event log for interpretability
        self.event_log = []
        # Statistics
        self.stats = {
            "total_registrations": 0,
            "total_deregistrations": 0,
            "total_heartbeats": 0,
            "total_queries": 0,
            "total_evictions": 0
        }

    # ---- REGISTER a new service instance ----
    def register(self, service_name, instance_id, host, port,
                 health_url="/health", metadata=None,
                 lease_seconds=None):
        
        lease = lease_seconds or self.default_lease_seconds
        now = datetime.utcnow()

        instance_info = {
            "instance_id": instance_id,
            "service_name": service_name,
            "host": host,
            "port": port,
            "health_url": health_url,
            "status": "UP",
            "metadata": metadata or {},
            "registered_at": now.isoformat(),
            "last_heartbeat": now.isoformat(),
            "lease_duration_sec": lease,
            "lease_expiry": (now + timedelta(seconds=lease)).isoformat()
        }

        with self.lock:
            is_new = instance_id not in self.registry[service_name]
            self.registry[service_name][instance_id] = instance_info
            self.stats["total_registrations"] += 1

        event_type = "REGISTERED" if is_new else "RE-REGISTERED"
        self._log_event(event_type, service_name, instance_id,
                        f"{host}:{port}")
        self._notify_watchers(service_name, event_type, instance_info)

        return {
            "status": "registered",
            "instance_id": instance_id,
            "lease_duration_sec": lease,
            "message": f"Instance {instance_id} registered under {service_name}"
        }

    # ---- DEREGISTER a service instance (graceful shutdown) ----
    def deregister(self, service_name, instance_id):
        with self.lock:
            if service_name in self.registry:
                instance = self.registry[service_name].pop(instance_id, None)
                if instance:
                    # Clean up empty service entries
                    if not self.registry[service_name]:
                        del self.registry[service_name]
                    self.stats["total_deregistrations"] += 1
                    self._log_event("DEREGISTERED", service_name,
                                    instance_id, "graceful shutdown")
                    self._notify_watchers(service_name, "DEREGISTERED",
                                          instance)
                    return {"status": "deregistered", "instance_id": instance_id}

        return {"status": "not_found", "instance_id": instance_id}

    # ---- HEARTBEAT — renew the lease ----
    def heartbeat(self, service_name, instance_id):
        now = datetime.utcnow()

        with self.lock:
            if (service_name in self.registry and
                    instance_id in self.registry[service_name]):
                instance = self.registry[service_name][instance_id]
                lease = instance["lease_duration_sec"]
                instance["last_heartbeat"] = now.isoformat()
                instance["lease_expiry"] = (
                    now + timedelta(seconds=lease)
                ).isoformat()
                # If it was marked DOWN, bring it back UP
                if instance["status"] == "DOWN":
                    instance["status"] = "UP"
                    self._log_event("REVIVED", service_name, instance_id,
                                    "heartbeat received after DOWN status")
                    self._notify_watchers(service_name, "STATUS_CHANGED",
                                          instance)
                self.stats["total_heartbeats"] += 1
                return {"status": "renewed", "lease_expiry": instance["lease_expiry"]}

        return {"status": "not_found",
                "message": "Instance not registered. Register first."}

    # ---- DISCOVER — find instances of a service ----
    def discover(self, service_name, only_healthy=True,
                 version_filter=None, region_filter=None):
        self.stats["total_queries"] += 1

        with self.lock:
            if service_name not in self.registry:
                return {"service": service_name, "instances": [],
                        "count": 0}

            instances = list(self.registry[service_name].values())

        # Filter by health status
        if only_healthy:
            instances = [i for i in instances if i["status"] == "UP"]

        # Filter by version metadata
        if version_filter:
            instances = [
                i for i in instances
                if i.get("metadata", {}).get("version") == version_filter
            ]

        # Filter by region metadata
        if region_filter:
            instances = [
                i for i in instances
                if i.get("metadata", {}).get("region") == region_filter
            ]

        # Return clean discovery response
        result = []
        for inst in instances:
            result.append({
                "instance_id": inst["instance_id"],
                "host": inst["host"],
                "port": inst["port"],
                "status": inst["status"],
                "metadata": inst["metadata"]
            })

        return {
            "service": service_name,
            "instances": result,
            "count": len(result)
        }

    # ---- LIST all registered services ----
    def list_services(self):
        with self.lock:
            summary = {}
            for svc_name, instances in self.registry.items():
                up_count = sum(
                    1 for i in instances.values() if i["status"] == "UP"
                )
                down_count = len(instances) - up_count
                summary[svc_name] = {
                    "total_instances": len(instances),
                    "healthy": up_count,
                    "unhealthy": down_count
                }
        return summary

    # ---- WATCH — subscribe to changes for a service ----
    def add_watcher(self, service_name, callback):
        with self.lock:
            self.watchers[service_name].append(callback)

    # ---- EVICTION — background thread to reap dead instances ----
    def run_evictor(self, check_interval=10):
        def _evict_loop():
            while True:
                time.sleep(check_interval)
                self._check_and_evict()
        
        thread = threading.Thread(target=_evict_loop, daemon=True)
        thread.start()

    def _check_and_evict(self):
        now = datetime.utcnow()
        to_mark_down = []
        to_evict = []

        with self.lock:
            for svc_name, instances in self.registry.items():
                for inst_id, inst in instances.items():
                    last_hb = datetime.fromisoformat(inst["last_heartbeat"])
                    elapsed = (now - last_hb).total_seconds()
                    lease = inst["lease_duration_sec"]

                    # First threshold: mark as DOWN
                    if (elapsed > lease and inst["status"] == "UP"):
                        to_mark_down.append((svc_name, inst_id))

                    # Second threshold: full eviction
                    if elapsed > (lease * self.eviction_multiplier):
                        to_evict.append((svc_name, inst_id))

        # Mark instances as DOWN
        for svc_name, inst_id in to_mark_down:
            with self.lock:
                if (svc_name in self.registry and
                        inst_id in self.registry[svc_name]):
                    self.registry[svc_name][inst_id]["status"] = "DOWN"
                    inst = self.registry[svc_name][inst_id]
                    self._log_event("MARKED_DOWN", svc_name, inst_id,
                                    "heartbeat lease expired")
                    self._notify_watchers(svc_name, "INSTANCE_DOWN", inst)

        # Fully evict instances
        for svc_name, inst_id in to_evict:
            with self.lock:
                if (svc_name in self.registry and
                        inst_id in self.registry[svc_name]):
                    inst = self.registry[svc_name].pop(inst_id)
                    if not self.registry[svc_name]:
                        del self.registry[svc_name]
                    self.stats["total_evictions"] += 1
                    self._log_event("EVICTED", svc_name, inst_id,
                                    "exceeded eviction threshold")
                    self._notify_watchers(svc_name, "INSTANCE_EVICTED", inst)

    # ---- Internal: Log events for interpretability ----
    def _log_event(self, event_type, service_name, instance_id, details=""):
        event = {
            "timestamp": datetime.utcnow().isoformat(),
            "event": event_type,
            "service": service_name,
            "instance": instance_id,
            "details": details
        }
        self.event_log.append(event)
        # Keep only last 1000 events
        if len(self.event_log) > 1000:
            self.event_log = self.event_log[-1000:]
        print(f"  [{event['timestamp']}] {event_type:15s} | "
              f"{service_name:20s} | {instance_id:15s} | {details}")

    # ---- Internal: Notify watchers of changes ----
    def _notify_watchers(self, service_name, event_type, instance_info):
        with self.lock:
            watchers = list(self.watchers.get(service_name, []))
        for callback in watchers:
            try:
                callback(event_type, service_name, instance_info)
            except Exception as e:
                print(f"  [WATCHER ERROR] {e}")

    def get_event_log(self, last_n=50):
        return self.event_log[-last_n:]

    def get_stats(self):
        return dict(self.stats)

class RegistryHTTPHandler(BaseHTTPRequestHandler):

        self.send_header("Content-Length", str(len(body)))

        self.wfile.write(body.encode("utf-8"))

    def read_body(self):
        length = int(self.headers.get("Content-Length", 0))
        if length:
            return json.loads(self.rfile.read(length).decode("utf-8"))
        return {}

    # ---- GET endpoints ----

        # GET /services — list all services

                "services": REGISTRY.list_services(),

            parts = path.split("/")
            service_name = parts[2]
            self.send_json(200, REGISTRY.discover(service_name))

        # GET /events — view recent events
        elif path == "/events":
            self.send_json(200, {
                "events": REGISTRY.get_event_log(50)

        # GET /health — registry's own health
        elif path == "/health":
            self.send_json(200, {
                "status": "healthy",
                "registry": "UP",
                "stats": REGISTRY.get_stats()
            })

        else:
            self.send_json(404, {"error": f"Unknown endpoint: {path}"})

    # ---- POST endpoints ----
    def do_POST(self):
        path = self.path
        body = self.read_body()

        # POST /register — register an instance
        if path == "/register":
            required = ["service_name", "instance_id", "host", "port"]
            missing = [f for f in required if f not in body]
            if missing:
                self.send_json(400, {
                    "error": f"Missing fields: {missing}"
                })
                return

            result = REGISTRY.register(
                service_name=body["service_name"],
                instance_id=body["instance_id"],
                host=body["host"],
                port=body["port"],
                health_url=body.get("health_url", "/health"),
                metadata=body.get("metadata", {}),
                lease_seconds=body.get("lease_seconds")
            )
            self.send_json(201, result)

        # POST /heartbeat — renew lease
        elif path == "/heartbeat":
            if "service_name" not in body or "instance_id" not in body:
                self.send_json(400, {
                    "error": "Missing service_name or instance_id"
                })
                return

            result = REGISTRY.heartbeat(
                body["service_name"],
                body["instance_id"]
            )
            status = 200 if result["status"] == "renewed" else 404
            self.send_json(status, result)

        else:
            self.send_json(404, {"error": f"Unknown endpoint: {path}"})

import urllib.request
import json
import threading
import time
import random
import signal
import sys
import atexit


class ServiceRegistryClient:
    """
    SDK that any microservice uses to interact with the registry.
    Handles registration, heartbeat, discovery, and load balancing.
    """

    def __init__(self, registry_url="http://127.0.0.1:8500"):
        self.registry_url = registry_url
        # Local cache: service_name → {instances: [...], fetched_at: time}
        self.cache = {}
        self.cache_ttl_seconds = 15  # Short TTL as recommended
        self.cache_lock = threading.Lock()
        # Heartbeat thread reference
        self._heartbeat_thread = None
        self._heartbeat_running = False
        # This instance's registration info
        self._my_service = None
        self._my_instance = None
        # Round-robin index tracker per service
        self._rr_index = {}

    # ---- REGISTER this service instance ----
    def register(self, service_name, instance_id, host, port,
                 health_url="/health", metadata=None,
                 lease_seconds=30, heartbeat_interval=10):

        self._my_service = service_name
        self._my_instance = instance_id

        payload = {
            "service_name": service_name,
            "instance_id": instance_id,
            "host": host,
            "port": port,
            "health_url": health_url,
            "metadata": metadata or {},
            "lease_seconds": lease_seconds
        }

        result = self._http_post("/register", payload)

        if result and result.get("status") == "registered":
            print(f"  [SDK] ✅ Registered as {service_name}/{instance_id} "
                  f"at {host}:{port}")
            # Start automatic heartbeat
            self._start_heartbeat(service_name, instance_id,
                                  heartbeat_interval)
            # Register shutdown hook for graceful deregistration
            self._register_shutdown_hook(service_name, instance_id)
            return True
        else:
            print(f"  [SDK] ❌ Registration failed: {result}")
            return False

    # ---- DISCOVER instances of a service (with caching) ----
    def discover(self, service_name, use_cache=True):
        # Check local cache first
        if use_cache:
            with self.cache_lock:
                if service_name in self.cache:
                    entry = self.cache[service_name]
                    age = time.time() - entry["fetched_at"]
                    if age < self.cache_ttl_seconds:
                        return entry["instances"]

        # Cache miss or expired → query registry
        result = self._http_get(
            f"/services/{service_name}/instances"
        )

        if result and "instances" in result:
            instances = result["instances"]
            # Update cache
            with self.cache_lock:
                self.cache[service_name] = {
                    "instances": instances,
                    "fetched_at": time.time()
                }
            return instances

        # If registry is unreachable, return stale cache if available
        with self.cache_lock:
            if service_name in self.cache:
                print(f"  [SDK] ⚠️ Registry unreachable. "
                      f"Using stale cache for {service_name}")
                return self.cache[service_name]["instances"]

        return []

    # ---- GET one instance using load balancing strategy ----
    def get_instance(self, service_name, strategy="round_robin"):
        instances = self.discover(service_name)

        if not instances:
            print(f"  [SDK] ❌ No instances found for {service_name}")
            return None

        if strategy == "round_robin":
            return self._round_robin(service_name, instances)
        elif strategy == "random":
            return random.choice(instances)
        elif strategy == "first":
            return instances[0]
        else:
            return random.choice(instances)

    # ---- BUILD a callable URL for a service ----
    def get_service_url(self, service_name, path="/",
                        strategy="round_robin"):
        instance = self.get_instance(service_name, strategy)
        if not instance:
            return None
        return f"http://{instance['host']}:{instance['port']}{path}"

    # ---- CALL another service (full discovery + routing) ----
    def call_service(self, service_name, path="/", method="GET",
                     body=None, strategy="round_robin"):
        url = self.get_service_url(service_name, path, strategy)
        if not url:
            return {"error": f"No instances available for {service_name}"}

        try:
            if method == "GET":
                return self._http_get_raw(url)
            elif method == "POST":
                return self._http_post_raw(url, body)
        except Exception as e:
            # On failure → invalidate cache for this service
            # So next call gets fresh data from registry
            with self.cache_lock:
                self.cache.pop(service_name, None)
            print(f"  [SDK] ❌ Call to {url} failed: {e}")
            print(f"  [SDK] Cache invalidated for {service_name}")
            return {"error": str(e)}

    # ---- INVALIDATE cache for a specific service ----
    def invalidate_cache(self, service_name=None):
        with self.cache_lock:
            if service_name:
                self.cache.pop(service_name, None)
            else:
                self.cache.clear()

    # ============================================================
    # INTERNAL: Heartbeat mechanism
    # ============================================================
    def _start_heartbeat(self, service_name, instance_id, interval):
        self._heartbeat_running = True

        def _heartbeat_loop():
            while self._heartbeat_running:
                time.sleep(interval)
                if not self._heartbeat_running:
                    break
                result = self._http_post("/heartbeat", {
                    "service_name": service_name,
                    "instance_id": instance_id
                })
                if result and result.get("status") == "renewed":
                    pass  # Silent success (too noisy to print every beat)
                else:
                    print(f"  [SDK] ⚠️ Heartbeat failed: {result}")

        self._heartbeat_thread = threading.Thread(
            target=_heartbeat_loop, daemon=True
        )
        self._heartbeat_thread.start()

    # ============================================================
    # INTERNAL: Graceful shutdown hook
    # ============================================================
    def _register_shutdown_hook(self, service_name, instance_id):
        def _shutdown():
            self._heartbeat_running = False
            print(f"\n  [SDK] Deregistering {service_name}/{instance_id}...")
            try:
                url = (f"{self.registry_url}/services/"
                       f"{service_name}/instances/{instance_id}")
                req = urllib.request.Request(url, method="DELETE")
                with urllib.request.urlopen(req, timeout=5) as resp:
                    pass
                print(f"  [SDK] ✅ Deregistered successfully")
            except Exception as e:
                print(f"  [SDK] ⚠️ Deregistration failed: {e}")

        atexit.register(_shutdown)

    # ============================================================
    # INTERNAL: Round-robin load balancing
    # ============================================================
    def _round_robin(self, service_name, instances):
        if service_name not in self._rr_index:
            self._rr_index[service_name] = 0
        idx = self._rr_index[service_name] % len(instances)
        self._rr_index[service_name] = idx + 1
        return instances[idx]

    # ============================================================
    # INTERNAL: HTTP helpers
    # ============================================================
    def _http_post(self, path, payload):
        try:
            url = f"{self.registry_url}{path}"
            data = json.dumps(payload).encode("utf-8")
            req = urllib.request.Request(
                url, data=data, method="POST",
                headers={"Content-Type": "application/json"}
            )
            with urllib.request.urlopen(req, timeout=5) as resp:
                return json.loads(resp.read().decode("utf-8"))
        except Exception as e:
            return {"error": str(e)}

    def _http_get(self, path):
        try:
            url = f"{self.registry_url}{path}"
            req = urllib.request.Request(url, method="GET")
            with urllib.request.urlopen(req, timeout=5) as resp:
                return json.loads(resp.read().decode("utf-8"))
        except Exception as e:
            return {"error": str(e)}

    def _http_get_raw(self, url):
        req = urllib.request.Request(url, method="GET")
        with urllib.request.urlopen(req, timeout=5) as resp:
            return json.loads(resp.read().decode("utf-8"))

    def _http_post_raw(self, url, payload):
        data = json.dumps(payload).encode("utf-8") if payload else None
        req = urllib.request.Request(
            url, data=data, method="POST",
            headers={"Content-Type": "application/json"}
        )
        with urllib.request.urlopen(req, timeout=5) as resp:

class MiniServiceHandler(BaseHTTPRequestHandler):
    """Generic handler for any simulated microservice."""

    service_name = "unknown"
    instance_id = "unknown"

    def do_GET(self):
        if self.path == "/health":
            self.send_json(200, {
                "status": "healthy",
                "service": self.service_name,
                "instance": self.instance_id
            })
        elif self.path == "/process":
            # Simulate some work
            time.sleep(random.uniform(0.01, 0.05))
            self.send_json(200, {
                "service": self.service_name,
                "instance": self.instance_id,
                "result": f"Processed by {self.instance_id}",
                "timestamp": time.time()
            })
        else:
            self.send_json(404, {"error": "not found"})

    def send_json(self, status, data):
        self.send_response(status)
        self.send_header("Content-Type", "application/json")
        body = json.dumps(data)
        self.send_header("Content-Length", str(len(body)))
        self.end_headers()
        self.wfile.write(body.encode("utf-8"))

    def log_message(self, fmt, *args):
        pass


def create_service_handler(svc_name, inst_id):
    """Factory to create handler class with specific service info."""

    class Handler(MiniServiceHandler):

    return Handler


def start_mini_service(service_name, instance_id, port,
                       registry_url, metadata=None):
    """Start a microservice that auto-registers with the registry."""

    handler_class = create_service_handler(service_name, instance_id)
    server = HTTPServer(("127.0.0.1", port), handler_class)

    # Start HTTP server in background
    thread = threading.Thread(target=server.serve_forever, daemon=True)
    thread.start()

    # Register with service registry
    sdk = ServiceRegistryClient(registry_url)
    sdk.register(
        service_name=service_name,
        instance_id=instance_id,
        host="127.0.0.1",
        port=port,
        metadata=metadata or {},
        lease_seconds=15,
        heartbeat_interval=5
    )

    return server, sdk

class DiscoveryAwareGateway:
    """
    API Gateway that uses Service Discovery instead of
    hardcoded backend addresses.
    
    Project 19 had:
      routes = {"/api/payments": "http://10.0.0.12:8080"}  ← HARDCODED
    
    NOW we have:
      routes = {"/api/payments": "payment-service"}  ← SERVICE NAME
      Gateway resolves via registry at request time
    """

    def __init__(self, registry_url="http://127.0.0.1:8500"):
        self.sdk = ServiceRegistryClient(registry_url)
        # Route map: path prefix → service name (NOT IP:Port)
        self.routes = {}
        # Rate limiter state (from Project 9)
        self.request_counts = {}
        self.rate_limit = 100  # requests per minute per client

    def add_route(self, path_prefix, service_name):
        self.routes[path_prefix] = service_name
        print(f"  [GATEWAY] Route added: {path_prefix} → {service_name}")

    def route_request(self, method, path, client_id,
                      body=None, headers=None):
        """
        Main gateway logic:
        1. Match path to service name
        2. Discover healthy instances
        3. Load balance across them
        4. Forward request
        5. Return response
        """

        # Step 1: Find which service handles this path
        target_service = None
        matched_prefix = None
        for prefix, service_name in self.routes.items():
            if path.startswith(prefix):
                target_service = service_name
                matched_prefix = prefix
                break

        if not target_service:
            return {
                "status": 404,
                "error": f"No route found for path: {path}"
            }

        # Step 2: Rate limiting check (from Project 9 concepts)
        if not self._check_rate_limit(client_id):
            return {
                "status": 429,
                "error": "Rate limit exceeded"
            }

        # Step 3: Discover instances via registry
        instances = self.sdk.discover(target_service)

        if not instances:
            return {
                "status": 503,
                "error": f"No healthy instances for {target_service}"
            }

        # Step 4: Pick an instance (client-side load balancing)
        instance = self.sdk.get_instance(
            target_service, strategy="round_robin"
        )

        # Step 5: Build the backend URL
        # Strip the gateway prefix, forward the rest
        backend_path = path[len(matched_prefix):]
        if not backend_path.startswith("/"):
            backend_path = "/" + backend_path
        backend_url = (f"http://{instance['host']}:"
                       f"{instance['port']}{backend_path}")

        # Step 6: Forward the request
        try:
            if method == "GET":
                req = urllib.request.Request(backend_url, method="GET")
            else:
                data = json.dumps(body).encode("utf-8") if body else None
                req = urllib.request.Request(
                    backend_url, data=data, method=method,
                    headers={"Content-Type": "application/json"}
                )

            with urllib.request.urlopen(req, timeout=5) as resp:
                result = json.loads(resp.read().decode("utf-8"))

            return {
                "status": 200,
                "data": result,
                "_gateway_metadata": {
                    "routed_to": instance["instance_id"],
                    "backend_url": backend_url,
                    "service": target_service
                }
            }

        except Exception as e:
            # On failure → invalidate cache, try another instance
            self.sdk.invalidate_cache(target_service)
            return {
                "status": 502,
                "error": f"Backend call failed: {str(e)}",
                "failed_instance": instance["instance_id"],
                "hint": "Will try different instance on next request"
            }

    def _check_rate_limit(self, client_id):
        now = time.time()
        minute_key = f"{client_id}:{int(now / 60)}"
        self.request_counts[minute_key] = (
            self.request_counts.get(minute_key, 0) + 1
        )
        return self.request_counts[minute_key] <= self.rate_limit

    def show_route_table(self):
        print(f"\n  Gateway Route Table:")
        print(f"  {'Path Prefix':<25} {'Service Name':<25} {'Instances'}")
        print(f"  {'-' * 75}")
        for prefix, service_name in self.routes.items():
            instances = self.sdk.discover(service_name)
            instance_ids = [i["instance_id"] for i in instances]
            print(f"  {prefix:<25} {service_name:<25} {instance_ids}")

THE PROBLEM DISCOVERY DOESN'T SOLVE:

  → Discovery tells you WHERE a service is
  → Discovery tells you if a service is REGISTERED and sending heartbeats
  → Discovery does NOT tell you if the service will SUCCEED when you call it

SCENARIO:
  → Payment Service instance payment-2 is "UP" in the registry
  → It's sending heartbeats (it's running)
  → BUT its database connection is broken
  → Every actual payment request FAILS with 500 error
  → Discovery keeps routing traffic to it
  → 50% of all payment calls fail

  → Even worse: Payment Service is overloaded
  → Response time went from 50ms to 30 seconds
  → Your Order Service waits 30 seconds per call
  → Order Service threads fill up waiting
  → Order Service itself becomes unresponsive
  → Now TWO services are down
  → This is called CASCADING FAILURE

WHAT PROJECT 21 SOLVES:
  → Circuit Breaker Pattern
  → "If a service keeps failing, STOP calling it temporarily"
  → Three states: CLOSED (normal) → OPEN (blocked) → HALF-OPEN (testing)
  → Like an electrical circuit breaker:
    → Too much current? → Trip the breaker → Protect the system
  → Combined with Discovery: Route AROUND broken instances
  → Combined with Fallback: Return cached/default data instead of failing

  The FLOW becomes:
    1. Discover instances (Project 20)
    2. Call instance through circuit breaker (Project 21)
    3. If breaker TRIPS → skip that instance → try another
    4. If ALL breakers trip → return fallback response
    5. Periodically test if failed instance recovered
