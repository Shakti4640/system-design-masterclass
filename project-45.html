🎯 HYPOTHETICAL PROBLEM

You run a banking system with 5 servers across 5 data centers
→ A customer sends: "Transfer $500 from Account A to Account B"
→ This request hits Server 1

NOW THE NIGHTMARE:
→ Server 1 processes the transfer and records it
→ Server 1 crashes BEFORE telling Servers 2-5
→ Customer calls bank: "Where is my $500?"
→ Servers 2-5 say: "What transfer? We have no record"
→ Server 1 comes back online: "Transfer was done!"
→ WHO IS RIGHT?

WORSE SCENARIO:
→ Network splits: Servers {1,2} can talk to each other
                   Servers {3,4,5} can talk to each other
                   But {1,2} CANNOT talk to {3,4,5}
→ Customer hits Server 1: "Transfer $500 A→B" → APPROVED
→ Same customer hits Server 3: "Transfer $500 A→B" → APPROVED AGAIN
→ Network heals → $1000 transferred instead of $500
→ THIS IS THE SPLIT-BRAIN PROBLEM

THE FUNDAMENTAL QUESTION:
→ How do N machines AGREE on a sequence of operations
→ Even when some machines crash?
→ Even when the network splits?
→ Even when messages arrive out of order or are duplicated?
→ Without EVER producing conflicting results?

THIS IS THE CONSENSUS PROBLEM
→ The HARDEST problem in distributed computing
→ Proven impossible in async systems with failures (FLP impossibility)
→ But PRACTICALLY solvable with Raft (with timing assumptions)

REAL SYSTEMS BUILT ON CONSENSUS:

RAFT DECOMPOSITION INTO 3 SUB-PROBLEMS:

  Sub-Problem 1: LEADER ELECTION
    → At any time, exactly ONE leader exists (or none during election)
    → If leader dies, a new one is elected within milliseconds
    → Elections use randomized timeouts to avoid conflicts
    → Term numbers prevent stale leaders from acting

  Sub-Problem 2: LOG REPLICATION
    → Leader receives client commands
    → Leader appends to its log, sends to followers
    → Once MAJORITY acknowledge → command is COMMITTED
    → Committed commands are NEVER lost (even if leader crashes)
    → All nodes apply committed commands in SAME ORDER

  Sub-Problem 3: SAFETY
    → Only a node with ALL committed entries can become leader
    → Committed entries are never overwritten
    → If two nodes disagree, the one with higher term wins
    → No split-brain: majority requirement prevents two leaders

  TOGETHER: These three guarantees mean
    → Every committed operation is permanent
    → Every node eventually sees the same operations in the same order
    → The system survives any minority of failures
    → 5 nodes → survives 2 failures
    → 3 nodes → survives 1 failure

🛠️ HOW CAN WE SOLVE IT

RAFT STATE MACHINE:

  Every node is in ONE of three states:

  ┌────────────┐    election timeout     ┌─────────────┐
  │            │ ─────────────────────→  │             │
  │  FOLLOWER  │                         │  CANDIDATE  │
  │            │ ←─────────────────────  │             │
  └────────────┘    discovers leader     └──────┬──────┘
       ▲              or higher term            │
       │                                  wins election
       │                                        │
       │            ┌──────────────┐            │
       │            │              │ ←──────────┘
       └─────────── │    LEADER    │
        discovers   │              │
        higher term └──────────────┘


NODE STATE:
  ┌──────────────────────────────────────────────────┐
  │ Persistent State (survives crashes):             │
  │   currentTerm: 0    ← latest term seen           │
  │   votedFor: null     ← who I voted for this term │
  │   log: []            ← log entries               │
  │                                                  │
  │ Volatile State (all servers):                    │
  │   commitIndex: 0     ← highest committed entry   │
  │   lastApplied: 0     ← highest applied to state  │
  │                                                  │
  │ Volatile State (leaders only):                   │
  │   nextIndex[]:  {}   ← next entry to send each   │
  │   matchIndex[]: {}   ← highest replicated entry   │
  └──────────────────────────────────────────────────┘


LOG STRUCTURE:
  Index:  1       2       3       4       5       6
  Term:   1       1       1       2       2       3
  Cmd:    SET     SET     DEL     SET     SET     SET
          x=1     y=2     x       z=3     x=5     y=7
                          ↑
                    commitIndex=4
                    (entries 1-4 are committed)
                    (entries 5-6 are uncommitted)


LEADER ELECTION FLOW:

  Time ──────────────────────────────────────────────→

  Node A (Follower):  waiting... waiting... TIMEOUT!
    → Increment currentTerm (1→2)
    → Vote for self
    → Send RequestVote to all nodes
    → State = CANDIDATE

  Node B (Follower):  receives RequestVote(term=2)
    → My term is 1, candidate term is 2 → valid
    → I haven't voted in term 2 → grant vote
    → Send VoteGranted to Node A

  Node C (Follower):  receives RequestVote(term=2)
    → Same logic → grant vote → Send VoteGranted

  Node A:  received 2 votes + self = 3 out of 5 → MAJORITY
    → State = LEADER
    → Immediately send heartbeats to all nodes
    → All nodes recognize Node A as leader for term 2


LOG REPLICATION FLOW:

  Client → Leader:  "SET x = 42"

  Leader (Node A):
    1. Append to own log: {term:2, index:7, cmd:"SET x=42"}
    2. Send AppendEntries(entries=[{term:2, cmd:"SET x=42"}])
       to Nodes B, C, D, E
    3. Wait for majority to respond

  Node B: receives AppendEntries
    → Check: leader's term ≥ my term? YES
    → Check: my log matches at prevLogIndex? YES
    → Append entry to my log
    → Respond: SUCCESS

  Node C: same as B → SUCCESS

  Leader counts: A(self) + B + C = 3 out of 5 → MAJORITY
    → Advance commitIndex to 7
    → Apply "SET x=42" to state machine
    → Respond to client: "OK, x=42 committed"
    → Next heartbeat tells followers: commitIndex=7
    → Followers apply entry 7 to their state machines

  Node D (slow): responds SUCCESS later
    → Leader updates matchIndex[D]
    → No effect on commit (already committed)

  Node E (crashed): no response
    → Leader will retry indefinitely
    → When E recovers, leader sends missing entries
    → E catches up to the same state

✅ WHAT SHOULD DO

1. ALWAYS PERSIST BEFORE RESPONDING
   → currentTerm, votedFor, log entries
   → Must survive crash and restart
   → If you respond to AppendEntries before persisting
     → Crash → data lost → safety violated
   → This is the #1 implementation bug

2. ALWAYS CHECK TERM NUMBERS
   → Every RPC includes sender's term
   → If you see a higher term → immediately step down
   → If you see a lower term → reject the RPC
   → Term is the "logical clock" of Raft

3. USE RANDOMIZED ELECTION TIMEOUTS
   → If all nodes timeout simultaneously → all become candidates
   → All vote for themselves → nobody gets majority → repeat forever
   → Random timeout (150-300ms) ensures one node times out first
   → That node almost always wins the election

4. LEADER MUST SEND HEARTBEATS
   → Even when there are no new entries
   → Heartbeat = empty AppendEntries
   → Prevents followers from starting unnecessary elections
   → Heartbeat interval << election timeout (e.g., 50ms vs 150-300ms)

5. COMMIT ONLY ENTRIES FROM CURRENT TERM
   → Leader cannot commit entries from previous terms directly
   → Must commit an entry from ITS OWN term first
   → Previous entries get committed indirectly
   → This prevents a subtle safety bug (Raft paper Figure 8)

6. IMPLEMENT PROPER LOG MATCHING
   → AppendEntries includes prevLogIndex and prevLogTerm
   → Follower rejects if its log doesn't match at that position
   → Leader decrements nextIndex and retries
   → This ensures logs are identical up to the commit point

❌ WHAT SHOULD NOT DO

1. NEVER ALLOW TWO LEADERS IN THE SAME TERM
   → Each node votes AT MOST ONCE per term
   → Majority required to win → at most one winner per term
   → If you see a leader with a higher term → YOUR leader is stale

2. NEVER COMMIT BY COUNTING ENTRIES FROM OLD TERMS
   → The Figure 8 scenario from the Raft paper
   → An entry from term 2 replicated to majority
   → But leader from term 3 hasn't committed anything yet
   → If term 3 leader commits a term 3 entry, term 2 entries
     get committed INDIRECTLY (by the Log Matching Property)
   → Never directly advance commitIndex based on old-term entries

3. NEVER TRUNCATE COMMITTED ENTRIES
   → Once commitIndex advances past an entry → it's permanent
   → Even if a new leader has a different log
   → Safety property: committed entries survive any leader change
   → Implementation: only truncate UNCOMMITTED suffix of log

4. NEVER USE WALL CLOCK TIME FOR ORDERING
   → Clocks drift, clocks can go backwards (NTP adjustments)
   → Use term numbers and log indices for ordering
   → Wall clock only for timeouts (election timeout, heartbeat)
   → This is a fundamental distributed systems principle

5. NEVER PROCESS CLIENT REQUESTS AS FOLLOWER
   → Followers redirect clients to the leader
   → Only leader can propose new log entries
   → Reading from follower can return stale data
   → For linearizable reads: must go through leader

6. NEVER SKIP LOG ENTRIES
   → Entries must be applied in order: 1, 2, 3, 4...
   → If entry 3 is missing, cannot apply entry 4
   → Leader sends entries in order, fills gaps

🚫 WHEN NOT TO USE

1. DO NOT USE RAFT WHEN:
   → You need eventual consistency (much simpler, faster)
     → Use gossip, CRDTs, last-writer-wins
   
   → You have > 9 nodes in a single group
     → Raft doesn't scale well beyond 7-9 nodes
     → Use Multi-Raft: many small groups, each managing a shard
   
   → You need sub-millisecond commits
     → Raft requires at least 1 network round trip for commit
     → In same datacenter: ~1-5ms per commit
     → Cross-datacenter: 50-200ms per commit
   
   → Your data is idempotent and order doesn't matter
     → CRDTs or simple replication is sufficient
     → Raft's total ordering is overkill

2. DO NOT IMPLEMENT YOUR OWN RAFT FOR PRODUCTION:
   → Use etcd, Consul, or a proven Raft library
   → Raft has MANY subtle edge cases
   → Off-by-one errors in log indexing can corrupt data
   → Timing bugs can cause split-brain
   → Our implementation is for LEARNING

3. DO NOT USE CONSENSUS FOR EVERY DECISION:
   → Consensus is expensive (network round trips)
   → Use it for: leader election, config changes, critical state
   → Don't use it for: every read, caching, metrics
   → Reads can be served from followers (with staleness trade-off)

MECHANIC 1: LEADER ELECTION — COMPLETE FLOW
═══════════════════════════════════════════════════════════

TIMELINE (5-node cluster: A, B, C, D, E):

  t=0:    All nodes start as FOLLOWERS, term=0
          Each sets a random election timeout:
            A: 250ms, B: 180ms, C: 300ms, D: 220ms, E: 280ms

  t=180ms: Node B's timer fires FIRST
           B → CANDIDATE, term=1, votes_received={B}
           B sends RequestVote(term=1, lastLogIndex=0, lastLogTerm=0)
           to A, C, D, E

  t=182ms: A receives RequestVote from B
           A's term is 0, B's term is 1 → valid
           A hasn't voted in term 1 → grants vote
           A resets its election timer
           A sends VoteGranted(term=1) to B

  t=183ms: D receives RequestVote → grants vote → sends to B
  
  t=184ms: B has votes: {B, A, D} = 3 out of 5 → MAJORITY
           B becomes LEADER for term 1
           B immediately sends heartbeat (empty AppendEntries)
           to A, C, D, E

  t=185ms: C receives heartbeat from B (term=1)
           C's term is 0 → update to 1
           C recognizes B as leader → resets election timer

  t=186ms: E receives heartbeat → same as C

  ELECTION COMPLETE ✅ Leader = B, Term = 1


WHAT IF TWO NODES TIMEOUT SIMULTANEOUSLY?

  t=180ms: B and D both become CANDIDATE, both term=1
  
  t=182ms: B gets votes from A and C → {B, A, C} = 3 → B WINS
           D gets vote from E → {D, E} = 2 → NOT ENOUGH
  
  OR:
  t=182ms: B gets A's vote, D gets E's vote
           C hasn't voted yet → grants to whoever asks first
           If B reaches C first → B wins with {B, A, C}
           If D reaches C first → D wins with {D, E, C}
  
  OR (worst case):
  t=182ms: B gets {B, A} = 2, D gets {D, E} = 2, C crashes
           Neither gets majority → election fails
           Both wait for NEW random timeout → retry with term=2
           Randomization ensures one usually wins quickly

  KEY INSIGHT: Random timeouts + majority requirement
  → Almost always elect a leader in 1-2 rounds
  → Mathematically: expected 1.6 rounds with good timeout spread

MECHANIC 2: LOG REPLICATION — COMPLETE FLOW
═══════════════════════════════════════════════════════════

  Client → Leader B: "SET x = 42"

  Leader B's log BEFORE:
    Index: 1       2       3
    Term:  1       1       1
    Cmd:   SET a=1 SET b=2 DEL a

  STEP 1: Leader appends to own log
    Index: 1       2       3       4
    Term:  1       1       1       1
    Cmd:   SET a=1 SET b=2 DEL a   SET x=42

  STEP 2: Leader sends AppendEntries to all followers:
    AppendEntries {
      term: 1,
      leaderId: B,
      prevLogIndex: 3,       ← "your log should match at index 3"
      prevLogTerm: 1,        ← "with term 1"
      entries: [{term:1, cmd:"SET x=42"}],
      leaderCommit: 3        ← "I've committed up to index 3"
    }

  STEP 3: Follower A receives:
    → Check: my term ≤ leader's term? YES (1 ≤ 1)
    → Check: do I have entry at index 3 with term 1? YES
    → Append new entry at index 4
    → Update commitIndex to min(leaderCommit, lastNewIndex) = 3
    → Respond: SUCCESS, matchIndex=4

  STEP 4: Follower C receives: same process → SUCCESS

  STEP 5: Leader B counts acknowledgments:
    → B (self) + A + C = 3 out of 5 → MAJORITY
    → Advance commitIndex from 3 to 4
    → Apply "SET x=42" to state machine
    → Respond to client: "OK"

  STEP 6: Next heartbeat includes leaderCommit=4
    → All followers advance their commitIndex to 4
    → All followers apply "SET x=42" to their state machines
    → ALL NODES NOW HAVE IDENTICAL STATE


WHAT IF FOLLOWER'S LOG IS BEHIND?

  Leader B's log:    [1:a=1] [1:b=2] [1:del_a] [1:x=42]
  Follower D's log:  [1:a=1] [1:b=2]   ← missed entries 3,4

  Leader sends AppendEntries with prevLogIndex=3, prevLogTerm=1
  Follower D: "I don't have entry at index 3!" → REJECT

  Leader decrements nextIndex[D] from 4 to 3
  Leader resends with prevLogIndex=2, prevLogTerm=1
  Follower D: "I have entry at index 2 with term 1!" → MATCH
  Follower D appends entries 3 and 4
  Follower D's log: [1:a=1] [1:b=2] [1:del_a] [1:x=42] ✅

  This "backtracking" ensures PERFECT log consistency


WHAT IF FOLLOWER HAS CONFLICTING ENTRIES?

  Leader B (term 3):  [1:a=1] [1:b=2] [2:c=3] [3:d=4]
  Follower E (stale): [1:a=1] [1:b=2] [2:c=3] [2:e=5]
                                                 ↑ CONFLICT
                                                 (different entry at index 4)

  Leader sends prevLogIndex=3, prevLogTerm=2
  Follower E: "I have index 3 with term 2" → MATCH
  Leader sends entry 4: {term:3, cmd:"d=4"}
  Follower E: TRUNCATES index 4 (removes [2:e=5])
              APPENDS {term:3, cmd:"d=4"}
  
  Follower E's log: [1:a=1] [1:b=2] [2:c=3] [3:d=4] ✅
  
  KEY: Conflicting UNCOMMITTED entries are overwritten
       Conflicting COMMITTED entries NEVER exist (safety guarantee)

MECHANIC 3: SAFETY — WHY COMMITTED ENTRIES SURVIVE
═══════════════════════════════════════════════════════════

  ELECTION RESTRICTION:
    A candidate's RequestVote includes:
      lastLogIndex: index of its last log entry
      lastLogTerm: term of its last log entry

    Voter grants vote ONLY IF candidate's log is
    "at least as up-to-date" as voter's log:
      → candidate's lastLogTerm > voter's lastLogTerm
      → OR same term AND candidate's lastLogIndex ≥ voter's

  WHY THIS WORKS:
    If entry E is committed → it's on MAJORITY of nodes
    Any new leader needs MAJORITY of votes
    Majority who has E ∩ Majority who votes = at least 1 node
    That node will REJECT any candidate that doesn't have E
    Therefore: new leader MUST have all committed entries

  PROOF BY EXAMPLE:
    5 nodes: {A, B, C, D, E}
    Entry X committed on {A, B, C} (majority)
    
    New election: candidate must get 3 votes (majority)
    
    Candidate F (doesn't have X) asks for votes:
      → A has X, F doesn't → A rejects F
      → B has X, F doesn't → B rejects F
      → C has X, F doesn't → C rejects F
      → D and E might vote for F → only 2 votes → NOT MAJORITY
    
    F CANNOT become leader ✅
    Only candidates with entry X can become leader ✅

  Rules:
    1. Term only increases, never decreases
    2. Each term has AT MOST one leader
    3. If node sees higher term → update own term, step down
    4. If node sees lower term in RPC → reject the RPC
    5. Term acts as a "logical clock" to detect stale leaders

  Scenario: Network partition heals
    → Old leader A (term=2) sends heartbeat to C
    → C is leader with term=4
    → A sees term 4 > its term 2 → A steps down to follower
    → C continues as leader
    → NO SPLIT-BRAIN ✅

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: NOT PERSISTING STATE BEFORE RESPONDING
  Scenario:
    → Follower receives AppendEntries
    → Responds SUCCESS immediately
    → Crashes before writing to disk
    → Restarts with old log (missing the entry)
    → Leader thinks entry is replicated, commits it
    → But follower doesn't have it → DATA LOSS
  
  Fix:
    → ALWAYS fsync to disk before responding to any RPC
    → This is why Raft commits are not "free"
    → Disk write = 1-10ms, but NECESSARY for safety

PITFALL 2: ELECTION TIMEOUT TOO SHORT
  Scenario:
    → Election timeout: 50ms
    → Network latency: 40ms
    → Heartbeat takes 40ms to arrive
    → Follower times out before heartbeat arrives
    → Starts unnecessary election
    → Disrupts the current leader
    → Cluster spends all time electing, not serving
  
  Fix:
    → Election timeout >> heartbeat interval >> network RTT
    → Rule of thumb: election timeout = 10× heartbeat interval
    → Heartbeat: 50ms, Election: 150-300ms (random)

PITFALL 3: THE FIGURE 8 BUG (Most Famous Raft Bug)
  Scenario (simplified):
    → Leader in term 2 replicates entry to majority but crashes
    → New leader in term 3 doesn't have that entry
    → New leader overwrites it with a different entry
    → The entry from term 2 was on a majority but NOT committed
  
  Fix:
    → Leader only commits entries from its CURRENT term
    → Previous-term entries get committed indirectly
    → When a current-term entry is committed, all prior entries
      are implicitly committed (Log Matching Property)

PITFALL 4: LIVELOCK IN ELECTIONS
  Scenario:
    → 3 nodes: A, B, C
    → A times out, becomes candidate, gets own vote
    → B times out simultaneously, becomes candidate, gets own vote
    → C votes for whoever asks first (say A)
    → A gets {A, C} = 2 out of 3 → wins? NO, B also asked C
    → If B reached C first → B wins, A loses
    → If neither gets majority → both retry
    → If timeouts are identical → infinite loop
  
  Fix:
    → RANDOMIZE election timeout (150-300ms uniform random)
    → After failed election, pick NEW random timeout
    → Probability of repeated collision drops exponentially

PITFALL 5: SPLIT-BRAIN DURING PARTITION
  Scenario:
    → 5 nodes, Leader = A
    → Partition: {A, B} isolated from {C, D, E}
    → A still thinks it's leader but can't reach majority
    → C times out, becomes candidate, wins with {C, D, E}
    → TWO "leaders": A (stale) and C (legitimate)
  
  This is ACTUALLY SAFE in Raft because:
    → A cannot commit anything (needs 3 out of 5)
    → Clients talking to A get no response (or timeout)
    → C can commit (has 3 out of 5)
    → When partition heals, A sees C's higher term → steps down

PITFALL 6: LOG GROWING UNBOUNDED
  Scenario:
    → System runs for months
    → Log has millions of entries
    → Node restarts → must replay entire log
    → Takes hours to recover
  
  Fix:
    → Log compaction / snapshotting
    → Periodically snapshot the state machine
    → Discard all log entries before the snapshot
    → New/recovering nodes receive snapshot + recent log
    → (Not implemented in our code, but critical for production)

📈 SCALABILITY CONSIDERATIONS

THROUGHPUT:
  Raft throughput = leader's ability to process + replicate
  
  Single Raft group:
    → Same datacenter: 10,000-50,000 commits/sec
    → Cross datacenter: 100-1,000 commits/sec (limited by RTT)
  
  Multi-Raft (shard per group):
    → 1000 groups × 10,000 commits/sec = 10M commits/sec
    → This is how CockroachDB and TiKV scale

LATENCY:
  Commit latency = time for leader to:
    1. Append to own log (local disk write): 1-5ms
    2. Send to followers: network RTT/2
    3. Wait for majority to respond: network RTT/2 + their disk write
  
  Same datacenter: 2-10ms per commit
  Cross datacenter (US East ↔ US West): 60-80ms per commit
  Cross continent (US ↔ Europe): 100-200ms per commit

SCALING STRATEGIES:
  ┌───────────────────────┬──────────────────────────────────┐
  │ Bottleneck            │ Solution                         │
  ├───────────────────────┼──────────────────────────────────┤
  │ Write throughput      │ Multi-Raft (shard data)          │
  │ Read throughput       │ Follower reads (with staleness)  │
  │ Cross-DC latency      │ Witness replicas (non-voting)    │
  │ Leader bottleneck     │ Batch multiple entries per RPC   │
  │ Log size              │ Snapshotting + compaction        │
  │ Recovery time         │ Snapshot + incremental replay    │
  │ Election storms       │ Pre-vote mechanism (Raft §9.6)   │
  └───────────────────────┴──────────────────────────────────┘

BATCHING FOR THROUGHPUT:
  Without batching:
    Client 1: SET x=1 → replicate → commit → respond (5ms)
    Client 2: SET y=2 → replicate → commit → respond (5ms)
    = 10ms total, 2 RPCs to each follower

  With batching:
    Client 1: SET x=1 → buffer
    Client 2: SET y=2 → buffer
    → Replicate both in ONE AppendEntries RPC → commit → respond
    = 5ms total, 1 RPC to each follower
    → 2× throughput improvement (more with higher load)

class StateMachine:
    """
    The application-level state machine.
    Raft guarantees all nodes apply the SAME commands in the SAME order.
    Therefore all state machines converge to IDENTICAL state.
    """

    def __init__(self, node_id: str):
        self.node_id = node_id
        self.data: Dict[str, str] = {}
        self.applied_count = 0
        self._lock = threading.Lock()

    def apply(self, entry: LogEntry) -> Optional[str]:
        with self._lock:
            self.applied_count += 1
            cmd_parts = entry.command.upper().split()

            if not cmd_parts:
                return None

            op = cmd_parts[0]

            if op == "SET" and entry.key and entry.value:
                self.data[entry.key] = entry.value
                return f"OK: {entry.key}={entry.value}"

            elif op == "DELETE" and entry.key:
                if entry.key in self.data:
                    old_val = self.data.pop(entry.key)
                    return f"DELETED: {entry.key} (was {old_val})"
                return f"NOT_FOUND: {entry.key}"

            elif op == "GET" and entry.key:
                val = self.data.get(entry.key, "NOT_FOUND")
                return f"VALUE: {entry.key}={val}"

            elif op == "NOOP":
                return "NOOP"

            return f"UNKNOWN_COMMAND: {entry.command}"

    def get(self, key: str) -> Optional[str]:
        with self._lock:
            return self.data.get(key)

    def get_all(self) -> Dict[str, str]:
        with self._lock:
            return dict(self.data)

    def snapshot(self) -> dict:
        with self._lock:
            return {
                "data": dict(self.data),
                "applied_count": self.applied_count
            }

class SimulatedNetwork:
    """
    Simulates network communication between Raft nodes.
    Supports: message delay, message loss, network partitions.
    
    In production: real TCP/gRPC connections.
    Here: in-memory message queues with configurable unreliability.
    """

    def __init__(self):
        self.nodes: Dict[str, Any] = {}
        self.message_queues: Dict[str, deque] = defaultdict(deque)
        self.partitions: Set[Tuple[str, str]] = set()
        self.message_delay_ms: Tuple[float, float] = (1.0, 5.0)
        self.drop_probability: float = 0.0
        self.total_messages_sent = 0
        self.total_messages_dropped = 0
        self.total_messages_delivered = 0
        self._lock = threading.Lock()
        self._running = True

    def register_node(self, node_id: str, node: Any):
        self.nodes[node_id] = node

    def add_partition(self, node_a: str, node_b: str):
        with self._lock:
            self.partitions.add((node_a, node_b))
            self.partitions.add((node_b, node_a))

    def remove_partition(self, node_a: str, node_b: str):
        with self._lock:
            self.partitions.discard((node_a, node_b))
            self.partitions.discard((node_b, node_a))

    def remove_all_partitions(self):
        with self._lock:
            self.partitions.clear()

    def is_partitioned(self, from_node: str, to_node: str) -> bool:
        with self._lock:
            return (from_node, to_node) in self.partitions

    def send(self, from_node: str, to_node: str, message_type: str, message: Any):
        with self._lock:
            self.total_messages_sent += 1

        if self.is_partitioned(from_node, to_node):
            with self._lock:
                self.total_messages_dropped += 1
            return

        if random.random() < self.drop_probability:
            with self._lock:
                self.total_messages_dropped += 1
            return

        delay = random.uniform(*self.message_delay_ms) / 1000.0

        def deliver():
            time.sleep(delay)
            if not self._running:
                return
            if self.is_partitioned(from_node, to_node):
                return
            if to_node in self.nodes and self.nodes[to_node].is_alive:
                self.nodes[to_node].receive_message(message_type, message, from_node)
                with self._lock:
                    self.total_messages_delivered += 1

        t = threading.Thread(target=deliver, daemon=True)
        t.start()

    def stats(self) -> dict:
        with self._lock:
            return {
                "total_sent": self.total_messages_sent,
                "total_dropped": self.total_messages_dropped,
                "total_delivered": self.total_messages_delivered,
                "active_partitions": len(self.partitions) // 2,
                "drop_probability": self.drop_probability
            }

    def shutdown(self):
        self._running = False

class RaftNode:
    """
    Complete Raft consensus node implementation.
    
    Implements all three sub-protocols:
      1. Leader Election (RequestVote RPC)
      2. Log Replication (AppendEntries RPC)
      3. Safety (election restriction, commit rules)
    """

    def __init__(self, node_id: str, peers: List[str], network: SimulatedNetwork):
        self.node_id = node_id
        self.peers = peers
        self.network = network

        # ── Persistent State (must survive crashes) ──
        self.current_term = 0
        self.voted_for: Optional[str] = None
        self.log: List[LogEntry] = []

        # ── Volatile State (all servers) ──
        self.commit_index = 0
        self.last_applied = 0
        self.state = NodeState.FOLLOWER
        self.leader_id: Optional[str] = None

        # ── Volatile State (leaders only) ──
        self.next_index: Dict[str, int] = {}
        self.match_index: Dict[str, int] = {}

        # ── Election State ──
        self.votes_received: Set[str] = set()
        self.election_timeout_ms: float = 0
        self.last_heartbeat_time: float = time.time()
        self._reset_election_timeout()

        # ── State Machine ──
        self.state_machine = StateMachine(node_id)

        # ── Client Request Tracking ──
        self.pending_requests: Dict[int, dict] = {}

        # ── Lifecycle ──
        self.is_alive = True
        self._lock = threading.Lock()

        # ── Observability ──
        self.election_count = 0
        self.terms_seen: List[int] = [0]
        self.event_log: List[dict] = []

    # ────────────────────────────────────
    # TIMER MANAGEMENT
    # ────────────────────────────────────

    def _reset_election_timeout(self):
        self.election_timeout_ms = random.uniform(300, 600)
        self.last_heartbeat_time = time.time()

    def _election_timeout_elapsed(self) -> bool:
        elapsed = (time.time() - self.last_heartbeat_time) * 1000
        return elapsed >= self.election_timeout_ms

    # ────────────────────────────────────
    # LOG HELPERS
    # ────────────────────────────────────

    def _last_log_index(self) -> int:
        return len(self.log)

    def _last_log_term(self) -> int:
        if not self.log:
            return 0
        return self.log[-1].term

    def _log_term_at(self, index: int) -> int:
        if index <= 0 or index > len(self.log):
            return 0
        return self.log[index - 1].term

    def _log_entry_at(self, index: int) -> Optional[LogEntry]:
        if index <= 0 or index > len(self.log):
            return None
        return self.log[index - 1]

    # ────────────────────────────────────
    # STATE TRANSITIONS
    # ────────────────────────────────────

    def _become_follower(self, term: int, reason: str = ""):
        old_state = self.state
        self.state = NodeState.FOLLOWER
        if term > self.current_term:
            self.current_term = term
            self.voted_for = None
            self.terms_seen.append(term)
        self._reset_election_timeout()
        self._log_event("STATE_CHANGE", {
            "from": old_state.value,
            "to": "FOLLOWER",
            "term": self.current_term,
            "reason": reason
        })

    def _become_candidate(self):
        self.state = NodeState.CANDIDATE
        self.current_term += 1
        self.voted_for = self.node_id
        self.votes_received = {self.node_id}
        self.leader_id = None
        self.election_count += 1
        self.terms_seen.append(self.current_term)
        self._reset_election_timeout()
        self._log_event("ELECTION_START", {
            "term": self.current_term,
            "election_number": self.election_count
        })

    def _become_leader(self):
        self.state = NodeState.LEADER
        self.leader_id = self.node_id

        for peer in self.peers:
            self.next_index[peer] = self._last_log_index() + 1
            self.match_index[peer] = 0

        self._log_event("LEADER_ELECTED", {
            "term": self.current_term,
            "log_length": self._last_log_index()
        })

        noop_entry = LogEntry(
            term=self.current_term,
            index=self._last_log_index() + 1,
            command="NOOP",
            key="",
            value=""
        )
        self.log.append(noop_entry)

        self._send_heartbeats()

    # ────────────────────────────────────
    # LEADER ELECTION (RequestVote RPC)
    # ────────────────────────────────────

    def _start_election(self):
        with self._lock:
            self._become_candidate()
            args = RequestVoteArgs(
                term=self.current_term,
                candidate_id=self.node_id,
                last_log_index=self._last_log_index(),
                last_log_term=self._last_log_term()
            )
            current_term = self.current_term

        for peer in self.peers:
            self.network.send(
                self.node_id, peer,
                "RequestVote", args
            )

    def _handle_request_vote(self, args: RequestVoteArgs, from_node: str):
        with self._lock:
            reply = RequestVoteReply(
                term=self.current_term,
                vote_granted=False,
                voter_id=self.node_id
            )

            if args.term > self.current_term:
                self._become_follower(args.term, f"higher term from candidate {from_node}")

            if args.term < self.current_term:
                self.network.send(
                    self.node_id, from_node,
                    "RequestVoteReply", reply
                )
                return

            can_vote = (self.voted_for is None or self.voted_for == args.candidate_id)

            candidate_up_to_date = False
            if args.last_log_term > self._last_log_term():
                candidate_up_to_date = True
            elif (args.last_log_term == self._last_log_term()
                  and args.last_log_index >= self._last_log_index()):
                candidate_up_to_date = True

            if can_vote and candidate_up_to_date:
                self.voted_for = args.candidate_id
                reply.vote_granted = True
                self._reset_election_timeout()
                reply.term = self.current_term
                self._log_event("VOTE_GRANTED", {
                    "to": args.candidate_id,
                    "term": self.current_term
                })
            else:
                reply.term = self.current_term

        self.network.send(
            self.node_id, from_node,
            "RequestVoteReply", reply
        )

    def _handle_request_vote_reply(self, reply: RequestVoteReply):
        with self._lock:
            if self.state != NodeState.CANDIDATE:
                return

            if reply.term > self.current_term:
                self._become_follower(reply.term, f"higher term in vote reply from {reply.voter_id}")
                return

            if reply.term < self.current_term:
                return

            if reply.vote_granted:
                self.votes_received.add(reply.voter_id)
                total_nodes = len(self.peers) + 1
                majority = (total_nodes // 2) + 1

                self._log_event("VOTE_RECEIVED", {
                    "from": reply.voter_id,
                    "votes": len(self.votes_received),
                    "needed": majority,
                    "term": self.current_term
                })

                if len(self.votes_received) >= majority:
                    self._become_leader()

    # ────────────────────────────────────
    # LOG REPLICATION (AppendEntries RPC)
    # ────────────────────────────────────

    def _send_heartbeats(self):
        with self._lock:
            if self.state != NodeState.LEADER:
                return

            for peer in self.peers:
                self._send_append_entries(peer)

    def _send_append_entries(self, peer: str):
        next_idx = self.next_index.get(peer, 1)
        prev_log_index = next_idx - 1
        prev_log_term = self._log_term_at(prev_log_index)

        entries = []
        if next_idx <= self._last_log_index():
            entries = self.log[next_idx - 1:]

        args = AppendEntriesArgs(
            term=self.current_term,
            leader_id=self.node_id,
            prev_log_index=prev_log_index,
            prev_log_term=prev_log_term,
            entries=list(entries),
            leader_commit=self.commit_index
        )

        self.network.send(
            self.node_id, peer,
            "AppendEntries", args
        )

    def _handle_append_entries(self, args: AppendEntriesArgs, from_node: str):
        with self._lock:
            reply = AppendEntriesReply(
                term=self.current_term,
                success=False,
                follower_id=self.node_id,
                match_index=0
            )

            if args.term > self.current_term:
                self._become_follower(args.term, f"leader {from_node} has higher term")

            if args.term < self.current_term:
                reply.term = self.current_term
                self.network.send(
                    self.node_id, from_node,
                    "AppendEntriesReply", reply
                )
                return

            self.state = NodeState.FOLLOWER
            self.leader_id = args.leader_id
            self._reset_election_timeout()

            if args.prev_log_index > 0:
                if args.prev_log_index > self._last_log_index():
                    reply.conflict_index = self._last_log_index() + 1
                    reply.conflict_term = -1
                    reply.term = self.current_term
                    self.network.send(
                        self.node_id, from_node,
                        "AppendEntriesReply", reply
                    )
                    return

                if self._log_term_at(args.prev_log_index) != args.prev_log_term:
                    conflict_term = self._log_term_at(args.prev_log_index)
                    conflict_index = args.prev_log_index
                    while (conflict_index > 1
                           and self._log_term_at(conflict_index - 1) == conflict_term):
                        conflict_index -= 1
                    reply.conflict_index = conflict_index
                    reply.conflict_term = conflict_term
                    reply.term = self.current_term
                    self.network.send(
                        self.node_id, from_node,
                        "AppendEntriesReply", reply
                    )
                    return

            if args.entries:
                for entry in args.entries:
                    idx = entry.index
                    if idx <= self._last_log_index():
                        if self._log_term_at(idx) != entry.term:
                            self.log = self.log[:idx - 1]
                            self.log.append(entry)
                        # else: already have this entry, skip
                    else:
                        self.log.append(entry)

            if args.leader_commit > self.commit_index:
                new_commit = min(args.leader_commit, self._last_log_index())
                if new_commit > self.commit_index:
                    old_commit = self.commit_index
                    self.commit_index = new_commit
                    self._apply_committed_entries()

            reply.success = True
            reply.match_index = self._last_log_index()
            reply.term = self.current_term

        self.network.send(
            self.node_id, from_node,
            "AppendEntriesReply", reply
        )

    def _handle_append_entries_reply(self, reply: AppendEntriesReply):
        with self._lock:
            if self.state != NodeState.LEADER:
                return

            if reply.term > self.current_term:
                self._become_follower(reply.term,
                    f"higher term in AE reply from {reply.follower_id}")
                return

            peer = reply.follower_id

            if reply.success:
                self.next_index[peer] = reply.match_index + 1
                self.match_index[peer] = reply.match_index
                self._try_advance_commit_index()
            else:
                if reply.conflict_term >= 0:
                    found = -1
                    for i in range(self._last_log_index(), 0, -1):
                        if self._log_term_at(i) == reply.conflict_term:
                            found = i
                            break
                    if found >= 0:
                        self.next_index[peer] = found + 1
                    else:
                        self.next_index[peer] = reply.conflict_index
                else:
                    self.next_index[peer] = max(
                        reply.conflict_index, 1
                    )

                self._send_append_entries(peer)

    def _try_advance_commit_index(self):
        """
        Leader advances commitIndex when a majority has replicated
        an entry FROM THE CURRENT TERM.
        
        Safety: only commit entries from current term directly.
        Previous-term entries get committed indirectly.
        """
        for n in range(self.commit_index + 1, self._last_log_index() + 1):
            if self._log_term_at(n) != self.current_term:
                continue

            replication_count = 1  # count self
            for peer in self.peers:
                if self.match_index.get(peer, 0) >= n:
                    replication_count += 1

            total_nodes = len(self.peers) + 1
            majority = (total_nodes // 2) + 1

            if replication_count >= majority:
                self.commit_index = n
                self._apply_committed_entries()
                self._resolve_pending_requests(n)

    def _apply_committed_entries(self):
        while self.last_applied < self.commit_index:
            self.last_applied += 1
            entry = self._log_entry_at(self.last_applied)
            if entry:
                result = self.state_machine.apply(entry)

    def _resolve_pending_requests(self, up_to_index: int):
        for idx in list(self.pending_requests.keys()):
            if idx <= up_to_index:
                req = self.pending_requests.pop(idx)
                if req.get("callback"):
                    entry = self._log_entry_at(idx)
                    result = f"COMMITTED at index {idx}, term {entry.term}" if entry else "COMMITTED"
                    req["callback"](True, result)

    # ────────────────────────────────────
    # CLIENT REQUEST HANDLING
    # ────────────────────────────────────

    def client_request(self, command: str, key: str = "", value: str = "",
                       callback: Callable = None) -> dict:
        with self._lock:
            if self.state != NodeState.LEADER:
                return {
                    "success": False,
                    "error": "NOT_LEADER",
                    "leader_hint": self.leader_id
                }

            new_index = self._last_log_index() + 1
            entry = LogEntry(
                term=self.current_term,
                index=new_index,
                command=command,
                key=key,
                value=value
            )
            self.log.append(entry)

            if callback:
                self.pending_requests[new_index] = {
                    "callback": callback,
                    "submitted_at": time.time()
                }

            self._log_event("CLIENT_REQUEST", {
                "command": command,
                "key": key,
                "value": value,
                "log_index": new_index,
                "term": self.current_term
            })

            for peer in self.peers:
                self._send_append_entries(peer)

            return {
                "success": True,
                "log_index": new_index,
                "term": self.current_term
            }

    # ────────────────────────────────────
    # MESSAGE ROUTER
    # ────────────────────────────────────

    def receive_message(self, message_type: str, message: Any, from_node: str):
        if not self.is_alive:
            return

        if message_type == "RequestVote":
            self._handle_request_vote(message, from_node)
        elif message_type == "RequestVoteReply":
            self._handle_request_vote_reply(message)
        elif message_type == "AppendEntries":
            self._handle_append_entries(message, from_node)
        elif message_type == "AppendEntriesReply":
            self._handle_append_entries_reply(message)

    # ────────────────────────────────────
    # TICK — Called periodically by cluster
    # ────────────────────────────────────

    def tick(self):
        if not self.is_alive:
            return

        with self._lock:
            if self.state == NodeState.LEADER:
                self._send_heartbeats()
            elif self._election_timeout_elapsed():
                pass  # Will start election outside lock

        if self.state != NodeState.LEADER and self._election_timeout_elapsed():
            self._start_election()

    # ────────────────────────────────────
    # CRASH / RECOVER SIMULATION
    # ────────────────────────────────────

    def crash(self):
        with self._lock:
            self.is_alive = False
            self._log_event("NODE_CRASH", {"term": self.current_term})

    def recover(self):
        with self._lock:
            self.is_alive = True
            self.state = NodeState.FOLLOWER
            self.leader_id = None
            self.votes_received = set()
            self.next_index = {}
            self.match_index = {}
            self._reset_election_timeout()
            self._log_event("NODE_RECOVER", {
                "term": self.current_term,
                "log_length": self._last_log_index(),
                "commit_index": self.commit_index
            })

    # ────────────────────────────────────
    # OBSERVABILITY
    # ────────────────────────────────────

    def _log_event(self, event_type: str, details: dict):
        self.event_log.append({
            "time": time.time(),
            "node": self.node_id,
            "event": event_type,
            "details": details
        })
        # Keep only last 200 events to prevent memory growth
        if len(self.event_log) > 200:
            self.event_log = self.event_log[-200:]

    def get_status(self) -> dict:
        with self._lock:
            return {
                "node_id": self.node_id,
                "state": self.state.value,
                "alive": self.is_alive,
                "term": self.current_term,
                "voted_for": self.voted_for,
                "leader_id": self.leader_id,
                "log_length": self._last_log_index(),
                "commit_index": self.commit_index,
                "last_applied": self.last_applied,
                "elections_started": self.election_count,
                "state_machine": self.state_machine.snapshot(),
                "next_index": dict(self.next_index) if self.state == NodeState.LEADER else {},
                "match_index": dict(self.match_index) if self.state == NodeState.LEADER else {}
            }

    def get_log_entries(self, start: int = 1, end: int = None) -> List[dict]:
        with self._lock:
            if end is None:
                end = self._last_log_index()
            entries = []
            for i in range(start, min(end + 1, self._last_log_index() + 1)):
                entry = self._log_entry_at(i)
                if entry:
                    entries.append(entry.to_dict())
            return entries

class RaftCluster:
    """
    Manages a cluster of Raft nodes with a simulated network.
    Drives the tick loop that makes nodes check timeouts and send heartbeats.
    """

    def __init__(self, node_count: int = 5):
        self.network = SimulatedNetwork()
        self.node_ids = [f"node_{i}" for i in range(1, node_count + 1)]
        self.nodes: Dict[str, RaftNode] = {}

        # Create all nodes with knowledge of their peers
        for nid in self.node_ids:
            peers = [p for p in self.node_ids if p != nid]
            node = RaftNode(nid, peers, self.network)
            self.nodes[nid] = node
            self.network.register_node(nid, node)

        self._running = False
        self._tick_thread: Optional[threading.Thread] = None
        self.tick_interval_ms = 50  # 50ms tick

    def start(self):
        self._running = True
        self._tick_thread = threading.Thread(target=self._tick_loop, daemon=True)
        self._tick_thread.start()

    def stop(self):
        self._running = False
        self.network.shutdown()
        if self._tick_thread:
            self._tick_thread.join(timeout=2)

    def _tick_loop(self):
        while self._running:
            for node in self.nodes.values():
                try:
                    node.tick()
                except Exception as e:
                    pass  # Node might be crashed
            time.sleep(self.tick_interval_ms / 1000.0)

    # ────────────────────────────────────
    # CLUSTER OPERATIONS
    # ────────────────────────────────────

    def get_leader(self) -> Optional[RaftNode]:
        for node in self.nodes.values():
            if node.is_alive and node.state == NodeState.LEADER:
                return node
        return None

    def get_leader_id(self) -> Optional[str]:
        leader = self.get_leader()
        return leader.node_id if leader else None

    def wait_for_leader(self, timeout_sec: float = 5.0) -> Optional[str]:
        start = time.time()
        while time.time() - start < timeout_sec:
            leader = self.get_leader()
            if leader:
                return leader.node_id
            time.sleep(0.1)
        return None

    def submit_command(self, command: str, key: str = "", value: str = "",
                       timeout_sec: float = 3.0) -> dict:
        """
        Submit a command to the cluster.
        Finds the leader and sends the command.
        Waits for commit confirmation.
        """
        leader = self.get_leader()
        if not leader:
            return {"success": False, "error": "NO_LEADER"}

        result_container = {"done": False, "success": False, "result": ""}
        result_event = threading.Event()

        def on_commit(success, result):
            result_container["done"] = True
            result_container["success"] = success
            result_container["result"] = result
            result_event.set()

        submit_result = leader.client_request(
            command=command, key=key, value=value, callback=on_commit
        )

        if not submit_result["success"]:
            return submit_result

        result_event.wait(timeout=timeout_sec)

        if result_container["done"]:
            return {
                "success": result_container["success"],
                "result": result_container["result"],
                "log_index": submit_result["log_index"],
                "term": submit_result["term"],
                "leader": leader.node_id
            }
        else:
            return {
                "success": False,
                "error": "TIMEOUT",
                "log_index": submit_result["log_index"]
            }

    def read_from_leader(self, key: str) -> Optional[str]:
        leader = self.get_leader()
        if not leader:
            return None
        return leader.state_machine.get(key)

    # ────────────────────────────────────
    # FAULT INJECTION
    # ────────────────────────────────────

    def crash_node(self, node_id: str):
        if node_id in self.nodes:
            self.nodes[node_id].crash()

    def recover_node(self, node_id: str):
        if node_id in self.nodes:
            self.nodes[node_id].recover()

    def partition_nodes(self, group_a: List[str], group_b: List[str]):
        for a in group_a:
            for b in group_b:
                self.network.add_partition(a, b)

    def heal_partitions(self):
        self.network.remove_all_partitions()

    # ────────────────────────────────────
    # OBSERVABILITY
    # ────────────────────────────────────

    def get_cluster_status(self) -> dict:
        statuses = {}
        for nid, node in self.nodes.items():
            statuses[nid] = node.get_status()
        return statuses

    def print_cluster_status(self, header: str = ""):
        if header:
            print(f"\n  📊 {header}")
        print(f"  {'─' * 72}")
        print(f"  {'Node':<10} {'State':<12} {'Alive':<7} {'Term':<6} "
              f"{'Log':<6} {'Commit':<8} {'Applied':<9} {'Leader':<10}")
        print(f"  {'─' * 72}")

        for nid in self.node_ids:
            status = self.nodes[nid].get_status()
            state_icon = {
                "LEADER": "👑",
                "FOLLOWER": "📋",
                "CANDIDATE": "🗳️"
            }.get(status["state"], "❓")
            alive_icon = "✅" if status["alive"] else "💀"

            print(f"  {status['node_id']:<10} "
                  f"{state_icon} {status['state']:<9} "
                  f"{alive_icon:<7} "
                  f"{status['term']:<6} "
                  f"{status['log_length']:<6} "
                  f"{status['commit_index']:<8} "
                  f"{status['last_applied']:<9} "
                  f"{status['leader_id'] or 'none':<10}")

    def print_logs_comparison(self, max_entries: int = 10):
        print(f"\n  📜 LOG COMPARISON (last {max_entries} entries):")
        print(f"  {'─' * 60}")

        all_logs = {}
        max_len = 0
        for nid in self.node_ids:
            entries = self.nodes[nid].get_log_entries()
            all_logs[nid] = entries
            max_len = max(max_len, len(entries))

        start = max(0, max_len - max_entries)

        header = f"  {'Idx':<5}"
        for nid in self.node_ids:
            header += f" {nid:<14}"
        print(header)
        print(f"  {'─' * 60}")

        for i in range(start, max_len):
            row = f"  {i+1:<5}"
            for nid in self.node_ids:
                entries = all_logs[nid]
                if i < len(entries):
                    e = entries[i]
                    cmd = e.get('command', '?')[:3]
                    key = e.get('key', '')[:4]
                    val = e.get('value', '')[:3]
                    term = e.get('term', '?')
                    cell = f"T{term}:{cmd}"
                    if key:
                        cell += f" {key}"
                    if val:
                        cell += f"={val}"
                    row += f" {cell:<14}"
                else:
                    row += f" {'---':<14}"
            print(row)

    def print_state_machines(self):
        print(f"\n  🗄️  STATE MACHINE COMPARISON:")
        print(f"  {'─' * 60}")

        all_data = {}
        all_keys = set()

        for nid in self.node_ids:
            data = self.nodes[nid].state_machine.get_all()
            all_data[nid] = data
            all_keys.update(data.keys())

        if not all_keys:
            print(f"  (empty — no data committed yet)")
            return

        header = f"  {'Key':<10}"
        for nid in self.node_ids:
            header += f" {nid:<12}"
        print(header)
        print(f"  {'─' * 60}")

        for key in sorted(all_keys):
            row = f"  {key:<10}"
            for nid in self.node_ids:
                val = all_data[nid].get(key, "—")
                row += f" {val:<12}"
            print(row)

        # Check consistency
        alive_data = [
            all_data[nid] for nid in self.node_ids
            if self.nodes[nid].is_alive
        ]
        if alive_data:
            # Only compare nodes with same commit index
            leader = self.get_leader()
            if leader:
                leader_commit = leader.commit_index
                matching = []
                for nid in self.node_ids:
                    node = self.nodes[nid]
                    if node.is_alive and node.last_applied >= leader_commit:
                        matching.append(all_data[nid])

                if matching and all(d == matching[0] for d in matching):
                    print(f"\n  ✅ All caught-up nodes have IDENTICAL state")
                elif matching:
                    print(f"\n  ⚠️  Some nodes still catching up")

THE PROBLEM WHEN WE GO GLOBAL:
  → Your bank has users in New York, London, Tokyo
  → Raft cluster in US East → London user commits take 80ms RTT
  → Tokyo user commits take 150ms RTT
  → EVERY write goes through US leader → terrible experience for global users

  → What if US East datacenter goes down entirely?
  → Raft needs majority → if 3 of 5 nodes are in US East → cluster dead
  → Global users get ZERO service

  → What if you put nodes in 5 different continents?
  → Every commit waits for 3 nodes across continents
  → Commit latency = cross-continent RTT = 200-300ms
  → TOO SLOW for interactive applications

  → Quorum configuration for latency vs availability tradeoffs
  → Conflict-free replication for multi-master writes
  → Clock synchronization (TrueTime, hybrid logical clocks)
  → This is how CockroachDB, Spanner, YugabyteDB work
  → The FINAL BOSS of distributed data systems

    → Place Raft group near the data's primary users
    → Cross-region only when necessary
    → This is Multi-Raft + intelligent placement
