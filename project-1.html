PROJECT 1: THE PIZZA SHOP — UNDERSTANDING CLIENT-SERVER MODEL

📋 PHASE 1 (Very Easy) — Foundation Thinking

🎯 HYPOTHETICAL PROBLEM

You live in a town where there is ONE pizza shop
→ You are hungry
→ You walk to the shop
→ You tell the counter person what you want
→ The kitchen prepares it
→ The counter person hands it to you
→ You walk back home and eat

NOW imagine:
→ 500 people want pizza at the same time
→ Some want different toppings
→ Some want to know the menu first before ordering
→ Some want to cancel their order midway
→ Some give wrong addresses for delivery

How does the pizza shop HANDLE all of this
without losing orders, mixing up pizzas, or crashing?

THIS is the Client-Server model
→ YOU are the Client
→ The PIZZA SHOP is the Server
→ The ORDER you give is the Request
→ The PIZZA you receive is the Response
→ The COUNTER is the Port
→ The LANGUAGE you speak to the counter person is the Protocol

🔍 HOW TO APPROACH IT

Step 1: Identify WHO is asking
         → The client (your browser, your app, your phone)

Step 2: Identify WHO is answering
         → The server (a computer sitting somewhere, always listening)

Step 3: Identify WHAT is being asked
         → The request (give me this data, save this data, delete this)

Step 4: Identify HOW they communicate
         → The protocol (HTTP, TCP, UDP — the "language" they agree on)

Step 5: Identify WHERE exactly on the server
         → The port (one machine can run multiple services on different ports)

Step 6: Identify WHAT comes back
         → The response (data, confirmation, error message)

🛠️ HOW CAN WE SOLVE IT

Mental Model:

  CLIENT                         SERVER
  ┌──────────┐    REQUEST      ┌──────────────┐
  │          │ ──────────────→ │              │
  │  Browser │                 │  Application │
  │  or App  │                 │  running on  │
  │          │ ←────────────── │  a machine   │
  └──────────┘    RESPONSE     └──────────────┘
                                     │
       Connected via:                │
       → Protocol (HTTP)             │
       → Port (8080)                 │
       → IP Address (192.168.1.1)    │
                                     ▼
                              ┌──────────────┐
                              │   DATABASE   │
                              │  (Kitchen    │
                              │   Storage)   │
                              └──────────────┘

The SOLUTION has 3 parts:
  1. A server that LISTENS on a specific port
  2. A client that SENDS requests to that port
  3. A defined FORMAT for request and response

📌 WHAT COULD BE DONE

Option A: Single Threaded Server
  → One customer at a time
  → Simple to build
  → Fails when 100 people come at once

Option B: Multi-Threaded Server
  → One thread per customer
  → Can handle multiple simultaneously
  → Uses more memory per connection

Option C: Event-Driven Server (Non-Blocking)
  → One thread handles many connections
  → Uses an event loop
  → Node.js / Python asyncio style
  → Most efficient for I/O-heavy workloads

Option D: Pre-forked Server
  → Spawn N worker processes ahead of time
  → Each worker handles one request
  → Apache HTTP server style

For THIS project → we start with Option A
then show Option C to understand the difference

✅ WHAT SHOULD DO

1. ALWAYS define a clear port
   → Port is like a counter number at the pizza shop
   → Without it, the client doesn't know WHERE to go

2. ALWAYS send back a response
   → Even if the request is bad → send an error response
   → Never leave the client hanging (timeout = frustration)

3. ALWAYS validate what comes in
   → Client might send garbage data
   → Server must check before processing

4. ALWAYS use a defined protocol
   → HTTP for web
   → TCP for reliable data transfer
   → UDP for speed (don't need every packet)

5. ALWAYS handle connection cleanup
   → Close sockets when done
   → Release memory/resources

6. ALWAYS log what happened
   → Who connected? What did they ask? What did we respond?

❌ WHAT SHOULD NOT DO

1. NEVER trust the client blindly
   → Client can send malicious data
   → Client can lie about who they are
   → ALWAYS validate server-side

2. NEVER hardcode IP addresses in production
   → IPs change, machines die
   → Use DNS or service names instead (covered in Project 7)

3. NEVER block the main thread with heavy computation
   → If pizza chef is also the counter person
   → Nobody else can order while one pizza is being made

4. NEVER ignore errors
   → Connection refused? Log it
   → Invalid data? Respond with proper error code
   → Server crash? Have a restart mechanism

5. NEVER expose internal server details in responses
   → Don't send stack traces to clients
   → Don't reveal database structure
   → Security from Day 1

6. NEVER assume the client will behave correctly
   → They might disconnect mid-request
   → They might send incomplete data
   → They might send requests too fast

⏱️ WHEN TO USE WHAT

┌─────────────────────────┬──────────────────────────────────────┐
│ SITUATION               │ USE THIS                             │
├─────────────────────────┼──────────────────────────────────────┤
│ Web page loading        │ HTTP over TCP (port 80/443)          │
│ Real-time game data     │ UDP (fast, lossy is okay)            │
│ File download           │ TCP (reliable, ordered delivery)     │
│ Chat application        │ WebSocket over TCP (persistent conn) │
│ Internal microservice   │ gRPC over HTTP/2 (fast, typed)       │
│ Simple prototype/demo   │ HTTP on any custom port (8080, 3000) │
│ Secure communication    │ HTTPS (TLS encrypted HTTP)           │
└─────────────────────────┴──────────────────────────────────────┘

Protocol Selection Rule:
  → Need reliability?     → TCP
  → Need speed over all?  → UDP
  → Need web standard?    → HTTP
  → Need persistent?      → WebSocket
  → Need structured RPC?  → gRPC

🚫 WHEN NOT TO USE

1. Client-Server is NOT suitable when:
   → Both parties are equal (use Peer-to-Peer instead)
      Example: BitTorrent, blockchain nodes
   
   → You need zero latency between two local processes
      Example: Use shared memory or IPC instead

   → The "server" has no state to manage
      Example: Static HTML can just be served by CDN (Project 12)

   → You need offline-first capability
      Example: Local-first apps that sync later

2. HTTP is NOT suitable when:
   → You need continuous bidirectional streaming
      Example: Live video → use WebSocket or WebRTC
   
   → You need sub-millisecond latency
      Example: High-frequency trading → use raw TCP/UDP

⚙️ HOW IT ONLY WORKS (INTERNAL MECHANICS)

THE COMPLETE LIFECYCLE — EVERY SINGLE STEP:

STEP 1: SERVER STARTS
  → Server process starts on a machine
  → Calls socket() → creates a communication endpoint
  → Calls bind()   → attaches to a specific IP + Port
  → Calls listen() → starts waiting for connections
  → Server is now "LISTENING" (pizza shop is OPEN)

STEP 2: CLIENT WANTS TO CONNECT
  → Client calls socket() → creates its own endpoint
  → Client calls connect() → tries to reach server IP:Port
  
STEP 3: TCP THREE-WAY HANDSHAKE (the "greeting")
  ┌────────┐                    ┌────────┐
  │ CLIENT │                    │ SERVER │
  └───┬────┘                    └───┬────┘
      │                             │
      │ ──── SYN ────────────────→  │  "Hey, I want to talk"
      │                             │
      │ ←──── SYN-ACK ───────────  │  "Sure, I hear you"
      │                             │
      │ ──── ACK ────────────────→  │  "Great, let's go"
      │                             │
  CONNECTION ESTABLISHED ✅

STEP 4: DATA EXCHANGE
  → Client sends HTTP Request through this connection
  → Request travels as TCP packets
  → Packets may take different routes (IP layer)
  → Server receives, reassembles, reads the request
  → Server processes it (queries DB, computes, etc.)
  → Server sends HTTP Response back through same connection
  → Client receives, reassembles, renders

STEP 5: CONNECTION CLOSURE (TCP Four-Way Teardown)
  ┌────────┐                    ┌────────┐
  │ CLIENT │                    │ SERVER │
  └───┬────┘                    └───┬────┘
      │                             │
      │ ──── FIN ────────────────→  │  "I'm done"
      │                             │
      │ ←──── ACK ───────────────  │  "Got it"
      │                             │
      │ ←──── FIN ───────────────  │  "I'm done too"
      │                             │
      │ ──── ACK ────────────────→  │  "Bye"
      │                             │
  CONNECTION CLOSED ✅

WHAT HAPPENS INSIDE HTTP REQUEST:
  ┌─────────────────────────────────────┐
  │ GET /menu HTTP/1.1                  │  ← Method + Path + Version
  │ Host: pizzashop.com                 │  ← Which server
  │ Accept: application/json            │  ← What format I want
  │ User-Agent: Chrome/120              │  ← Who is asking
  │ Connection: keep-alive              │  ← Don't close yet
  │                                     │
  │ (empty body for GET requests)       │
  └─────────────────────────────────────┘

WHAT HAPPENS INSIDE HTTP RESPONSE:
  ┌─────────────────────────────────────┐
  │ HTTP/1.1 200 OK                     │  ← Version + Status Code
  │ Content-Type: application/json      │  ← Format of response
  │ Content-Length: 245                  │  ← Size of body
  │ Date: Mon, 01 Jan 2025 12:00:00    │  ← When
  │                                     │
  │ {"menu": ["Margherita", "Pepperoni"]}│ ← The actual data
  └─────────────────────────────────────┘

PORT MECHANICS:
  → A port is a 16-bit number (0–65535)
  → Ports 0–1023: Well-known (HTTP=80, HTTPS=443, SSH=22)
  → Ports 1024–49151: Registered (MySQL=3306, PostgreSQL=5432)
  → Ports 49152–65535: Dynamic/Ephemeral (OS assigns to clients)
  → Client gets a RANDOM ephemeral port
  → Server uses a FIXED known port
  → Combination of IP:Port uniquely identifies a connection

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: Port Already in Use
  → You start a server on port 8080
  → Another process already uses 8080
  → ERROR: Address already in use
  → FIX: Kill the other process or use a different port

PITFALL 2: Firewall Blocking
  → Server is running perfectly
  → Client can't connect
  → A firewall is blocking that port
  → FIX: Open the port in firewall rules

PITFALL 3: Connection Timeout
  → Client sends request, server takes too long
  → Client gives up after timeout period
  → FIX: Set appropriate timeouts on both sides

PITFALL 4: Zombie Connections
  → Client crashes without closing connection
  → Server still holds resources for that connection
  → Memory slowly leaks
  → FIX: Implement server-side timeouts and keepalive probes

PITFALL 5: Blocking Single-Thread Server
  → One slow request blocks ALL other requests
  → Everyone waits in line
  → FIX: Use async/threaded architecture (we show both below)

PITFALL 6: Forgetting to Handle Partial Reads
  → TCP is a STREAM protocol, not message-based
  → You might receive half a message in one read
  → FIX: Use Content-Length or delimiters to know when message is complete

PITFALL 7: Localhost Works, Remote Doesn't
  → Server binds to 127.0.0.1 (localhost only)
  → Remote clients can't reach it
  → FIX: Bind to 0.0.0.0 (all interfaces)

🔍 INTERPRETABILITY CONSIDERATIONS

HOW TO "SEE" WHAT'S HAPPENING:

1. Logging on Server
   → Print every request received
   → Print every response sent
   → Print timestamp, client address, method, path
   → This is your EYES into the system

2. Status Codes Tell the Story
   → 200: Everything worked ✅
   → 400: Client sent bad data ❌ (client's fault)
   → 404: Client asked for something that doesn't exist
   → 500: Server broke internally ❌ (server's fault)
   → Status code = first thing you check when debugging

3. Network Tools
   → curl: Send requests from command line
   → netstat: See what ports are in use
   → tcpdump: See actual packets on the wire
   → browser DevTools → Network tab: See every request/response

4. Request/Response Clarity
   → ALWAYS return structured responses (JSON)
   → ALWAYS include meaningful error messages
   → NEVER return raw exceptions to client
   → Client should ALWAYS know what happened and why

# ============================================================
# PROJECT 1A: RAW TCP SERVER — THE ABSOLUTE FOUNDATION
# This is what happens UNDERNEATH HTTP
# Understanding this = understanding everything above it
# ============================================================

import socket
import threading
import json
import datetime

# ============================================================
# THE PIZZA SHOP SERVER (TCP Level)
# ============================================================

class PizzaShopServer:
    
    def __init__(self, host='127.0.0.1', port=9999):
        # The shop's address and counter number
        self.host = host
        self.port = port
        # Create the socket (the communication endpoint)
        # AF_INET = IPv4 addressing
        # SOCK_STREAM = TCP (reliable, ordered delivery)
        self.server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        # Allow reuse of address (prevents "Address already in use" error)
        self.server_socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
        # Our pizza menu (this is our "data")
        self.menu = {
            "margherita": {"price": 10, "time_mins": 12},
            "pepperoni": {"price": 13, "time_mins": 15},
            "veggie": {"price": 11, "time_mins": 14},
            "bbq_chicken": {"price": 14, "time_mins": 18}
        }
        # Track orders (simple in-memory storage)
        self.orders = {}
        self.order_counter = 0
        # Thread lock for order_counter (safe concurrent access)
        self.lock = threading.Lock()
    
    def start(self):
        # BIND: Attach socket to specific address + port
        self.server_socket.bind((self.host, self.port))
        # LISTEN: Start accepting connections (backlog=5)
        # Backlog = how many pending connections can wait in queue
        self.server_socket.listen(5)
        print(f"[PIZZA SHOP OPEN] Listening on {self.host}:{self.port}")
        print(f"[INFO] Waiting for customers...\n")
        
        while True:
            # ACCEPT: Block until a client connects
            # Returns: new socket for THIS specific client + their address
            client_socket, client_address = self.server_socket.accept()
            print(f"[NEW CUSTOMER] Connected from {client_address}")
            # Handle each customer in a separate thread
            # So one slow order doesn't block everyone
            thread = threading.Thread(
                target=self.handle_customer,
                args=(client_socket, client_address)
            )
            thread.daemon = True  # Thread dies when main program exits
            thread.start()
    
    def handle_customer(self, client_socket, client_address):
        try:
            while True:
                # RECEIVE data from client (max 4096 bytes at a time)
                raw_data = client_socket.recv(4096)
                
                # If empty data → client disconnected
                if not raw_data:
                    print(f"[CUSTOMER LEFT] {client_address}")
                    break
                
                # Decode bytes to string, then parse JSON
                request_str = raw_data.decode('utf-8').strip()
                print(f"[REQUEST from {client_address}] {request_str}")
                
                try:
                    request = json.loads(request_str)
                except json.JSONDecodeError:
                    response = {
                        "status": 400,
                        "error": "Invalid request format. Send JSON."
                    }
                    client_socket.send(json.dumps(response).encode('utf-8'))
                    continue
                
                # PROCESS the request based on "action"
                response = self.process_request(request, client_address)
                
                # SEND response back
                response_str = json.dumps(response)
                client_socket.send(response_str.encode('utf-8'))
                print(f"[RESPONSE to {client_address}] {response_str}\n")
                
        except ConnectionResetError:
            print(f"[CONNECTION LOST] {client_address} disconnected abruptly")
        except Exception as e:
            print(f"[ERROR] Handling {client_address}: {e}")
        finally:
            # ALWAYS clean up the connection
            client_socket.close()
    
    def process_request(self, request, client_address):
        action = request.get("action", "").lower()
        timestamp = datetime.datetime.now().isoformat()
        
        # ACTION: Get the menu
        if action == "get_menu":
            return {
                "status": 200,
                "message": "Here is our menu",
                "data": self.menu,
                "timestamp": timestamp
            }
        
        # ACTION: Place an order
        elif action == "place_order":
            pizza_name = request.get("pizza", "").lower()
            
            if pizza_name not in self.menu:
                return {
                    "status": 404,
                    "error": f"Pizza '{pizza_name}' not found on menu",
                    "available": list(self.menu.keys()),
                    "timestamp": timestamp
                }
            
            # Thread-safe order ID generation
            with self.lock:
                self.order_counter += 1
                order_id = f"ORD-{self.order_counter:04d}"
            
            pizza_info = self.menu[pizza_name]
            self.orders[order_id] = {
                "pizza": pizza_name,
                "price": pizza_info["price"],
                "status": "preparing",
                "customer": str(client_address),
                "ordered_at": timestamp
            }
            
            return {
                "status": 201,
                "message": f"Order placed successfully!",
                "order_id": order_id,
                "pizza": pizza_name,
                "price": pizza_info["price"],
                "estimated_time_mins": pizza_info["time_mins"],
                "timestamp": timestamp
            }
        
        # ACTION: Check order status
        elif action == "check_order":
            order_id = request.get("order_id", "")
            
            if order_id not in self.orders:
                return {
                    "status": 404,
                    "error": f"Order '{order_id}' not found",
                    "timestamp": timestamp
                }
            
            return {
                "status": 200,
                "order": self.orders[order_id],
                "timestamp": timestamp
            }
        
        # ACTION: Cancel an order
        elif action == "cancel_order":
            order_id = request.get("order_id", "")
            
            if order_id not in self.orders:
                return {
                    "status": 404,
                    "error": f"Order '{order_id}' not found",
                    "timestamp": timestamp
                }
            
            self.orders[order_id]["status"] = "cancelled"
            return {
                "status": 200,
                "message": f"Order {order_id} cancelled",
                "timestamp": timestamp
            }
        
        # UNKNOWN action
        else:
            return {
                "status": 400,
                "error": f"Unknown action: '{action}'",
                "available_actions": [
                    "get_menu",
                    "place_order",
                    "check_order",
                    "cancel_order"
                ],
                "timestamp": timestamp
            }


# ============================================================
# THE CUSTOMER CLIENT (TCP Level)
# ============================================================

class PizzaCustomerClient:
    
    def __init__(self, host='127.0.0.1', port=9999):
        self.host = host
        self.port = port
        self.socket = None
    
    def connect(self):
        # Create socket and connect to server
        self.socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        self.socket.connect((self.host, self.port))
        print(f"[CONNECTED] to pizza shop at {self.host}:{self.port}\n")
    
    def send_request(self, request_dict):
        if not self.socket:
            print("[ERROR] Not connected. Call connect() first.")
            return None
        
        # Convert dict to JSON string, then to bytes
        request_str = json.dumps(request_dict)
        self.socket.send(request_str.encode('utf-8'))
        
        # Wait for response
        raw_response = self.socket.recv(4096)
        response = json.loads(raw_response.decode('utf-8'))
        return response
    
    def disconnect(self):
        if self.socket:
            self.socket.close()
            print("[DISCONNECTED] from pizza shop")

# ============================================================
# PROJECT 1B: HTTP SERVER — BUILDING ON TOP OF TCP
# Now we move from raw TCP to the HTTP protocol
# HTTP is just a STRUCTURED FORMAT on top of TCP
# ============================================================

from http.server import HTTPServer, BaseHTTPRequestHandler
import json
import datetime
import urllib.parse

# ============================================================
# IN-MEMORY DATA STORE (our simple "kitchen")
# ============================================================

MENU = {
    "margherita": {"price": 10, "time_mins": 12},
    "pepperoni": {"price": 13, "time_mins": 15},
    "veggie": {"price": 11, "time_mins": 14},
    "bbq_chicken": {"price": 14, "time_mins": 18}
}

ORDERS = {}
ORDER_COUNTER = [0]  # Mutable container for closure access


# ============================================================
# HTTP REQUEST HANDLER
# Every HTTP request maps to a method here
# ============================================================

class PizzaShopHTTPHandler(BaseHTTPRequestHandler):
    
    # ---- UTILITY: Send JSON Response ----
    def send_json_response(self, status_code, data):
        self.send_response(status_code)
        self.send_header('Content-Type', 'application/json')
        self.send_header('Access-Control-Allow-Origin', '*')
        response_body = json.dumps(data, indent=2)
        self.send_header('Content-Length', str(len(response_body)))
        self.end_headers()
        self.wfile.write(response_body.encode('utf-8'))
    
    # ---- UTILITY: Read Request Body ----
    def read_body(self):
        content_length = int(self.headers.get('Content-Length', 0))
        if content_length > 0:
            body = self.rfile.read(content_length)
            return json.loads(body.decode('utf-8'))
        return {}
    
    # ---- UTILITY: Structured Logging ----
    def log_request_info(self, method):
        timestamp = datetime.datetime.now().isoformat()
        print(f"[{timestamp}] {method} {self.path} from {self.client_address[0]}")
    
    # ============================================================
    # GET REQUESTS — Reading Data (like asking questions)
    # ============================================================
    def do_GET(self):
        self.log_request_info("GET")
        parsed_path = urllib.parse.urlparse(self.path)
        path = parsed_path.path
        
        # GET /health — Is the server alive?
        if path == '/health':
            self.send_json_response(200, {
                "status": "healthy",
                "uptime": "running",
                "timestamp": datetime.datetime.now().isoformat()
            })
        
        # GET /menu — What pizzas are available?
        elif path == '/menu':
            self.send_json_response(200, {
                "status": 200,
                "message": "Pizza menu",
                "data": MENU,
                "total_items": len(MENU)
            })
        
        # GET /orders — List all orders
        elif path == '/orders':
            self.send_json_response(200, {
                "status": 200,
                "total_orders": len(ORDERS),
                "orders": ORDERS
            })
        
        # GET /orders/ORD-0001 — Get specific order
        elif path.startswith('/orders/'):
            order_id = path.split('/orders/')[1]
            if order_id in ORDERS:
                self.send_json_response(200, {
                    "status": 200,
                    "order": ORDERS[order_id]
                })
            else:
                self.send_json_response(404, {
                    "status": 404,
                    "error": f"Order '{order_id}' not found"
                })
        
        # Unknown path
        else:
            self.send_json_response(404, {
                "status": 404,
                "error": f"Endpoint '{path}' not found",
                "available_endpoints": {
                    "GET": ["/health", "/menu", "/orders", "/orders/{id}"],
                    "POST": ["/orders"],
                    "DELETE": ["/orders/{id}"]
                }
            })
    
    # ============================================================
    # POST REQUESTS — Creating Data (like placing an order)
    # ============================================================
    def do_POST(self):
        self.log_request_info("POST")
        
        if self.path == '/orders':
            body = self.read_body()
            pizza_name = body.get("pizza", "").lower()
            quantity = body.get("quantity", 1)
            
            # Validation
            if not pizza_name:
                self.send_json_response(400, {
                    "status": 400,
                    "error": "Missing 'pizza' field in request body"
                })
                return
            
            if pizza_name not in MENU:
                self.send_json_response(404, {
                    "status": 404,
                    "error": f"Pizza '{pizza_name}' not on menu",
                    "available": list(MENU.keys())
                })
                return
            
            if not isinstance(quantity, int) or quantity < 1:
                self.send_json_response(400, {
                    "status": 400,
                    "error": "Quantity must be a positive integer"
                })
                return
            
            # Create order
            ORDER_COUNTER[0] += 1
            order_id = f"ORD-{ORDER_COUNTER[0]:04d}"
            pizza_info = MENU[pizza_name]
            
            ORDERS[order_id] = {
                "order_id": order_id,
                "pizza": pizza_name,
                "quantity": quantity,
                "unit_price": pizza_info["price"],
                "total_price": pizza_info["price"] * quantity,
                "status": "preparing",
                "estimated_time_mins": pizza_info["time_mins"],
                "ordered_at": datetime.datetime.now().isoformat()
            }
            
            # 201 Created — the correct status for resource creation
            self.send_json_response(201, {
                "status": 201,
                "message": "Order placed successfully",
                "order": ORDERS[order_id]
            })
        
        else:
            self.send_json_response(404, {
                "status": 404,
                "error": f"Cannot POST to '{self.path}'"
            })
    
    # ============================================================
    # DELETE REQUESTS — Removing Data (like cancelling)
    # ============================================================
    def do_DELETE(self):
        self.log_request_info("DELETE")
        
        if self.path.startswith('/orders/'):
            order_id = self.path.split('/orders/')[1]
            
            if order_id not in ORDERS:
                self.send_json_response(404, {
                    "status": 404,
                    "error": f"Order '{order_id}' not found"
                })
                return
            
            if ORDERS[order_id]["status"] == "cancelled":
                self.send_json_response(400, {
                    "status": 400,
                    "error": "Order already cancelled"
                })
                return
            
            ORDERS[order_id]["status"] = "cancelled"
            ORDERS[order_id]["cancelled_at"] = datetime.datetime.now().isoformat()
            
            self.send_json_response(200, {
                "status": 200,
                "message": f"Order {order_id} cancelled",
                "order": ORDERS[order_id]
            })
        
        else:
            self.send_json_response(404, {
                "status": 404,
                "error": f"Cannot DELETE '{self.path}'"
            })
    
    # Suppress default logging (we have our own)
    def log_message(self, format, *args):
        pass

import socket
import json
import threading
import time
import random

def simulate_customer(customer_id, host='127.0.0.1', port=9999):
    """Each customer is an independent client thread"""
    try:
        client = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        client.connect((host, port))
        
        # Random delay to simulate real-world timing
        time.sleep(random.uniform(0.1, 0.5))
        
        # Each customer orders a random pizza
        pizzas = ["margherita", "pepperoni", "veggie", "bbq_chicken"]
        chosen_pizza = random.choice(pizzas)
        
        # Step 1: Get menu
        request = json.dumps({"action": "get_menu"})
        client.send(request.encode('utf-8'))
        response = json.loads(client.recv(4096).decode('utf-8'))
        
        time.sleep(random.uniform(0.1, 0.3))
        
        # Step 2: Place order
        request = json.dumps({
            "action": "place_order",
            "pizza": chosen_pizza
        })
        client.send(request.encode('utf-8'))
        response = json.loads(client.recv(4096).decode('utf-8'))
        
        order_id = response.get('order_id', 'NONE')
        price = response.get('price', 0)
        
        print(f"  Customer #{customer_id:02d} | "
              f"Ordered: {chosen_pizza:<12} | "
              f"Order: {order_id} | "
              f"Price: ${price}")
        
        client.close()
        return True
        
    except Exception as e:
        print(f"  Customer #{customer_id:02d} | ERROR: {e}")
        return False


def run_concurrent_simulation(num_customers=20):
    """Launch many customers simultaneously"""
    print(f"\n{'=' * 60}")
    print(f"  SIMULATING {num_customers} CONCURRENT CUSTOMERS")
    print(f"  (Make sure Part A server is running on port 9999)")
    print(f"{'=' * 60}\n")
    
    start_time = time.time()
    threads = []
    
    # Launch all customers at roughly the same time
    for i in range(1, num_customers + 1):
        t = threading.Thread(target=simulate_customer, args=(i,))
        threads.append(t)
        t.start()
    
    # Wait for all customers to finish
    for t in threads:
        t.join()
    
    elapsed = time.time() - start_time
    
    print(f"\n{'=' * 60}")
    print(f"  ALL {num_customers} CUSTOMERS SERVED")
    print(f"  Total time: {elapsed:.2f} seconds")
    print(f"  Avg per customer: {elapsed/num_customers:.3f} seconds")
    print(f"{'=' * 60}")
    print(f"\n  💡 INSIGHT: With threading, the server handled")
    print(f"     {num_customers} customers concurrently.")
    print(f"     Without threading, they'd wait in line = {elapsed * num_customers:.1f}s total")
    print(f"\n  🔗 This is WHY we need:")
    print(f"     → Project 4: Scaling (what if 10,000 customers?)")
    print(f"     → Project 6: Load Balancer (split across servers)")


if __name__ == "__main__":
    run_concurrent_simulation(20)
