🎯 HYPOTHETICAL PROBLEM

You work at a travel booking company
→ Search bar on the homepage
→ User types "n"
→ Within 50 milliseconds, show:
    "new york", "new delhi", "nairobi", "nice", "nagoya"
→ User types "ne"
→ Within 50 milliseconds, update to:
    "new york", "new delhi", "newcastle", "new orleans", "nepal"
→ User types "new y"
→ Within 50 milliseconds:
    "new york", "new york city flights", "new year deals"

CONSTRAINTS:
  → 10 million searchable terms (cities, airports, deals, routes)
  → 500,000 concurrent users typing simultaneously
  → Each keystroke = 1 request
  → Average user types 5-8 characters before selecting
  → That's 2.5-4 million requests per second
  → Each must respond in < 50ms (user perceives instant)
  → Suggestions must be RANKED by popularity/relevance

WHY NAIVE APPROACHES FAIL:

  Approach 1: SQL LIKE query
    → SELECT * FROM terms WHERE term LIKE 'ne%' ORDER BY popularity
    → Full table scan even with index (LIKE with leading wildcard)
    → 100-500ms per query on 10M rows
    → Cannot handle 4M queries/second
    → FAILS on latency AND throughput

  Approach 2: In-memory linear scan
    → Loop through 10M strings, check prefix match
    → O(N) per query = ~50ms even in memory
    → Cannot handle concurrent load
    → FAILS on throughput

  Approach 3: Distributed lock per query (Project 27)
    → Absolutely wrong — read-only operation needs NO locks
    → Lock overhead would add 1-5ms per query
    → The lesson from Project 27: best lock = no lock needed

WHAT WE NEED:
  → O(prefix_length) lookup, NOT O(total_terms)
  → Precomputed ranked suggestions per prefix
  → Multi-layer caching (Project 5)
  → Shardable for horizontal scaling (Project 14, 18)
  → No locking for reads (Project 27 insight)

🔍 HOW TO APPROACH IT

Step 1: Identify the ACCESS PATTERN
         → Input: a prefix string ("ne")
         → Output: top-K ranked completions (["new york", "new delhi"...])
         → Frequency: extremely high (every keystroke)
         → Latency: ultra-low (< 50ms end-to-end)
         → Nature: READ-ONLY (suggestions don't change per request)

Step 2: Identify the RIGHT data structure
         → Need: fast prefix lookup
         → Trie (prefix tree) = O(L) where L = prefix length
         → NOT hash map (exact match only, not prefix)
         → NOT B-tree (Project 10 — optimized for range, not prefix fan-out)
         → NOT full-text index (overkill for prefix matching)

Step 3: Identify the RANKING strategy
         → Most popular completions first
         → Each node stores top-K results for its prefix
         → Precomputed at build time, not query time
         → Updated periodically (batch), not per-request

Step 4: Identify the CACHING strategy (Project 5)
         → Hot prefixes: "the", "new", "how" → cache aggressively
         → Cold prefixes: "xyloph" → cache briefly or not at all
         → Keystroke locality: "n" → "ne" → "new" → cache warming

Step 5: Identify the SCALING strategy
         → Shard by first character or prefix hash
         → Each shard handles a partition of the alphabet
         → Replicate each shard for read throughput (Project 13)

Step 6: Identify the UPDATE strategy
         → Searches happen millions of times per second
         → Popularity scores change slowly (hourly/daily)
         → Build new trie periodically, swap atomically
         → NEVER update trie in-place during serving

📌 WHAT COULD BE DONE

Option A: Simple Trie with DFS at Query Time
  → Walk to prefix node
  → DFS to find all completions
  → Sort by frequency
  → Return top-K
  → PROBLEM: DFS can be slow for short prefixes ("a" → millions of completions)

Option B: Trie with Precomputed Top-K (CHOSEN)
  → Each node stores its top-K completions already sorted
  → Walk to prefix node → return stored top-K
  → O(prefix_length) for lookup, O(1) for results
  → Build cost is higher, but amortized over millions of queries

Option C: Sorted Array with Binary Search
  → Sort all terms lexicographically
  → Binary search to find prefix range
  → O(log N) lookup
  → Simpler but slower than trie for prefix operations
  → Cannot store precomputed rankings easily

Option D: Inverted Index (Project 41 preview)
  → Map each prefix to list of completions
  → Like a hash map: prefix → [results]
  → O(1) lookup but O(L * N) storage (every prefix of every term)
  → Memory-intensive but fastest at query time

FOR THIS PROJECT:
  → Option B as primary (Trie with precomputed top-K)
  → Option D as optimization layer (prefix → result cache)
  → Combined: Trie for structure + Cache for hot paths

✅ WHAT SHOULD DO

1. PRECOMPUTE everything possible at build time
   → Top-K per prefix node
   → Frequency scores
   → Normalized/cleaned terms
   → Query time should do MINIMAL work

2. USE multi-layer caching (Project 5 concepts)
   → L1: Application-level in-process cache (dict)
   → L2: Distributed cache (Redis — Project 23)
   → L3: Trie lookup (in-memory, still fast)
   → Cache hit ratio for autocomplete: 80-95% (short prefixes repeat)

3. NORMALIZE input before lookup
   → Lowercase: "New York" → "new york"
   → Strip whitespace: " new  york " → "new york"
   → Remove special characters: "new-york!" → "new york"
   → Consistent normalization at BOTH index time AND query time

4. DEBOUNCE on client side
   → Don't send request for EVERY keystroke
   → Wait 100-200ms after last keystroke
   → Reduces server load by 50-70%
   → Still feels instant to user

5. RETURN early for very short prefixes
   → Prefix "a" matches millions of terms
   → Return generic popular terms for 1-char prefixes
   → Start personalized suggestions at 2-3 characters

6. BUILD new trie offline, SWAP atomically
   → Never modify serving trie in-place
   → Build new trie from updated data
   → Atomic pointer swap: old_trie → new_trie
   → Old trie garbage collected after in-flight requests complete

7. TRACK what users actually SELECT
   → User types "new" → sees "new york" → clicks it
   → Log: ("new", "new york", selected=true)
   → This feedback loop improves rankings over time
   → Builds the frequency data for next trie build

❌ WHAT SHOULD NOT DO

1. NEVER query a database per keystroke
   → 4 million queries/second on DB = instant death
   → DB is for BUILDING the trie, not SERVING queries
   → Serve from in-memory structures only

2. NEVER sort results at query time
   → Sorting 10,000 completions for every keystroke = too slow
   → Precompute sorted top-K at build time
   → Query just returns the precomputed list

3. NEVER use distributed locks for read-only autocomplete
   → Project 27 lesson: best lock is no lock
   → Autocomplete is 100% read during serving
   → Lock-free data structures or immutable trie + swap

4. NEVER return ALL completions
   → Prefix "a" might have 500,000 completions
   → User only sees 5-10 suggestions
   → Always limit to top-K (K = 5-10)

5. NEVER update trie in real-time per search
   → User searches "pizza near me" → don't immediately update trie
   → Batch: collect search logs → rebuild trie hourly/daily
   → Real-time updates cause race conditions and inconsistency

6. NEVER ignore offensive/inappropriate suggestions
   → Autocomplete amplifies visibility
   → Blocklist offensive terms at build time
   → Review suggestions before they go live

7. NEVER send suggestions for empty prefix
   → User hasn't typed anything → show trending/popular instead
   → Don't query trie with empty string (returns everything)

MECHANISM 1: TRIE (PREFIX TREE) DATA STRUCTURE
═══════════════════════════════════════════════════════

WHAT IS A TRIE:
  → Tree where each node represents ONE CHARACTER
  → Path from root to node = a prefix
  → Children of a node = next possible characters
  → Leaf/marked nodes = complete words/terms

EXAMPLE — Inserting: "cat", "car", "card", "care", "cab"

  Root (empty)
  └── c
      └── a
          ├── t ★ ("cat")
          ├── r ★ ("car")
          │   ├── d ★ ("card")
          │   └── e ★ ("care")
          └── b ★ ("cab")

  ★ = marks end of a complete term

LOOKUP "car":
  Root → 'c' (exists?) → YES, go to child 'c'
  'c'  → 'a' (exists?) → YES, go to child 'a'
  'a'  → 'r' (exists?) → YES, go to child 'r'
  'r' is marked as end ★ → "car" IS a complete term
  'r' has children 'd', 'e' → completions: "card", "care"
  
  Total steps: 3 (length of prefix)
  → O(L) where L = length of prefix
  → INDEPENDENT of how many terms are in the trie


═══════════════════════════════════════════════════════
MECHANISM 2: PRECOMPUTED TOP-K PER NODE
═══════════════════════════════════════════════════════

WITHOUT PRECOMPUTATION:
  Query "ca" → walk to node 'a' under 'c'
  → DFS from 'a' to find ALL completions:
     "cat"(5000), "car"(8000), "card"(3000), "care"(6000), "cab"(2000)
  → Sort by frequency: car, care, cat, card, cab
  → Return top-3: ["car", "care", "cat"]
  
  PROBLEM: DFS might traverse millions of nodes for short prefixes
  
WITH PRECOMPUTATION:
  At BUILD time, for EACH node, store top-K completions:
  
  Node 'c':    top-3 = ["car"(8000), "care"(6000), "cat"(5000)]
  Node 'ca':   top-3 = ["car"(8000), "care"(6000), "cat"(5000)]
  Node 'car':  top-3 = ["car"(8000), "care"(6000), "card"(3000)]
  Node 'cat':  top-3 = ["cat"(5000)]
  
  Query "ca" → walk to node 'ca' → return stored top-3
  → O(L) walk + O(1) result retrieval
  → NO DFS, NO sorting at query time

BUILD PROCESS (bottom-up):
  1. Insert all terms into trie with frequencies
  2. For each LEAF node: top-K = [itself]
  3. For each INTERNAL node (bottom-up):
     → Merge top-K lists from all children
     → Keep only top-K by frequency
     → Store at this node
  4. Build takes O(N * L * K) but happens ONCE
  5. Queries take O(L) and happen MILLIONS of times

MECHANISM 3: COMPRESSED TRIE (PATRICIA / RADIX TREE)
═══════════════════════════════════════════════════════

PROBLEM: Standard trie wastes memory on single-child chains

  Standard Trie for "new york", "new delhi":
  n → e → w → ' ' → y → o → r → k ★
                   → d → e → l → h → i ★
  
  12 nodes for just 2 terms — many single-child chains

COMPRESSED TRIE: Merge single-child chains into one node
  "new " → "york" ★
         → "delhi" ★
  
  3 nodes for 2 terms — much more memory efficient

MEMORY SAVINGS:
  → Standard trie: ~50 bytes per node × millions of nodes
  → Compressed trie: 3-10x fewer nodes
  → For 10M terms: ~2GB standard → ~500MB compressed

MECHANISM 5: TRIE SHARDING FOR HORIZONTAL SCALING
═══════════════════════════════════════════════════════

Single-node trie limit: ~10-50M terms (memory bound)
For 100M+ terms: SHARD the trie

SHARDING STRATEGY 1: By First Character
  → Shard A: prefixes starting with 'a'
  → Shard B: prefixes starting with 'b'
  → ...
  → Shard Z: prefixes starting with 'z'
  → PROBLEM: Uneven distribution ('s' has way more terms than 'x')

SHARDING STRATEGY 2: By Prefix Hash (Project 14, 18)
  → hash("ne") % num_shards → shard 3
  → hash("ca") % num_shards → shard 7
  → Even distribution
  → PROBLEM: Related prefixes on different shards
    ("new" on shard 3, "new " on shard 5)

SHARDING STRATEGY 3: By Prefix Range (BEST)
  → Shard 1: "a" - "d" 
  → Shard 2: "e" - "j"
  → Shard 3: "k" - "o"
  → Shard 4: "p" - "z"
  → Ranges adjusted based on data distribution
  → Related prefixes stay together on same shard

ROUTING:
  API Gateway (Project 19) routes based on first character:
    → Request "new york" → first char 'n' → Shard 3
    → Request "paris"    → first char 'p' → Shard 4

# TRIE NODE — Single character in the prefix tree

@dataclass
class TrieNode:
    """
    Each node represents one character position in the trie.
    
    Key fields:
    → children: map of char → child node (next characters)
    → is_terminal: does a complete term END at this node?
    → term: the complete term (only set if is_terminal)
    → frequency: how often this term is searched
    → top_k: PRECOMPUTED top-K suggestions for this prefix
    """
    children: Dict[str, 'TrieNode'] = field(default_factory=dict)
    is_terminal: bool = False
    term: str = ""
    frequency: int = 0
    top_k: List[Tuple[int, str]] = field(default_factory=list)
    # top_k format: [(frequency, term), ...] sorted descending
    depth: int = 0


# ================================================================
# TRIE — Complete Prefix Tree with Precomputed Rankings
# ================================================================

class AutocompleteTrie:
    """
    Trie optimized for autocomplete:
    
    BUILD phase (offline):
      1. Insert all terms with frequencies
      2. Bottom-up propagation of top-K suggestions
      3. Freeze for serving
    
    SERVE phase (online):
      1. Walk trie following prefix characters
      2. Return precomputed top-K at destination node
      3. O(prefix_length) total — no DFS, no sorting
    """
    
    def __init__(self, k: int = 10):
        self.root = TrieNode(depth=0)
        self.k = k  # How many suggestions per prefix
        self.total_terms = 0
        self.total_nodes = 1  # Root counts
        self.max_depth = 0
        self.build_time_ms = 0
        self._frozen = False
    
    # ============================================================
    # BUILD PHASE — Insert terms and precompute rankings
    # ============================================================
    
    def insert(self, term: str, frequency: int = 1):
        """
        Insert a term into the trie.
        
        For "new york" with frequency 50000:
          Root → 'n' → 'e' → 'w' → ' ' → 'y' → 'o' → 'r' → 'k'
          Mark 'k' as terminal with term="new york", frequency=50000
        
        Time: O(L) where L = length of term
        """
        if self._frozen:
            raise RuntimeError("Trie is frozen — cannot insert during serving")
        
        # Normalize input
        normalized = self._normalize(term)
        if not normalized:
            return
        
        node = self.root
        for i, char in enumerate(normalized):
            if char not in node.children:
                node.children[char] = TrieNode(depth=i + 1)
                self.total_nodes += 1
            node = node.children[char]
        
        # Mark terminal
        if not node.is_terminal:
            self.total_terms += 1
        node.is_terminal = True
        node.term = normalized
        node.frequency = max(node.frequency, frequency)  # Keep highest
        self.max_depth = max(self.max_depth, len(normalized))
    
    def bulk_insert(self, terms_with_freq: List[Tuple[str, int]]):
        """Insert many terms at once (used during build phase)"""
        start = time.time()
        for term, freq in terms_with_freq:
            self.insert(term, freq)
        elapsed = time.time() - start
        print(f"  Inserted {len(terms_with_freq)} terms in {elapsed:.3f}s")
    
    def build_top_k(self):
        """
        Precompute top-K suggestions for EVERY node in the trie.
        
        Algorithm (bottom-up):
        1. For each leaf/terminal node: top_k = [(freq, term)]
        2. For each internal node:
           → Collect top_k from ALL children
           → Merge and keep only top-K by frequency
           → If node is terminal, include itself too
        
        This is a POST-ORDER traversal (children first, then parent).
        
        After this: every node has precomputed suggestions.
        Query = walk to node + return node.top_k
        """
        start = time.time()
        self._build_top_k_recursive(self.root)
        self.build_time_ms = (time.time() - start) * 1000
        self._frozen = True
        print(f"  Built top-{self.k} for {self.total_nodes} nodes "
              f"in {self.build_time_ms:.1f}ms")
    
    def _build_top_k_recursive(self, node: TrieNode) -> List[Tuple[int, str]]:
        """
        Recursively build top-K for this node.
        Returns this node's top-K list (used by parent).
        """
        # Collect candidates from all children
        candidates = []
        
        for child in node.children.values():
            child_top_k = self._build_top_k_recursive(child)
            candidates.extend(child_top_k)
        
        # Include this node itself if it's a terminal
        if node.is_terminal:
            candidates.append((node.frequency, node.term))
        
        # Keep only top-K using a heap (efficient for large candidate lists)
        # nlargest is O(N * log K) — better than full sort O(N log N) when K << N
        top_k = heapq.nlargest(self.k, candidates, key=lambda x: x[0])
        
        node.top_k = top_k
        return top_k
    
    # ============================================================
    # SERVE PHASE — Fast prefix lookup
    # ============================================================
    
    def search(self, prefix: str) -> List[Tuple[int, str]]:
        """
        Return top-K suggestions for the given prefix.
        
        Algorithm:
        1. Normalize prefix
        2. Walk trie character by character
        3. If any character not found → return empty (no matches)
        4. Return precomputed top_k at destination node
        
        Time: O(L) where L = length of prefix
        """
        normalized = self._normalize(prefix)
        if not normalized:
            # Empty prefix → return root's top-K (most popular overall)
            return self.root.top_k
        
        # Walk the trie
        node = self.root
        for char in normalized:
            if char not in node.children:
                return []  # No matches for this prefix
            node = node.children[char]
        
        return node.top_k
    
    def search_formatted(self, prefix: str) -> List[Dict]:
        """Return suggestions in a structured format"""
        results = self.search(prefix)
        return [
            {
                "rank": i + 1,
                "term": term,
                "score": freq
            }
            for i, (freq, term) in enumerate(results)
        ]
    
    # ============================================================
    # DFS SEARCH — For comparison (shows why precomputation matters)
    # ============================================================
    
    def search_dfs(self, prefix: str, limit: int = 10) -> List[Tuple[int, str]]:
        """
        Search WITHOUT precomputation — uses DFS at query time.
        
        THIS IS THE SLOW WAY (for comparison):
        1. Walk to prefix node
        2. DFS to find ALL completions
        3. Sort by frequency
        4. Return top-K
        
        Time: O(L + M * log M) where M = number of completions
        For short prefixes, M can be MILLIONS → very slow
        """
        normalized = self._normalize(prefix)
        
        # Walk to prefix node
        node = self.root
        for char in normalized:
            if char not in node.children:
                return []
            node = node.children[char]
        
        # DFS to collect ALL completions
        completions = []
        self._collect_all(node, completions)
        
        # Sort by frequency (expensive!)
        completions.sort(key=lambda x: x[0], reverse=True)
        
        return completions[:limit]
    
    def _collect_all(self, node: TrieNode,
                     results: List[Tuple[int, str]]):
        """DFS to collect all terminal nodes under this node"""
        if node.is_terminal:
            results.append((node.frequency, node.term))
        for child in node.children.values():
            self._collect_all(child, results)
    
    # ============================================================
    # UTILITIES
    # ============================================================
    
    @staticmethod
    def _normalize(text: str) -> str:
        """
        Normalize input for consistent matching.
        CRITICAL: Same normalization at insert AND search time.
        """
        return text.lower().strip()
    
    def get_stats(self) -> Dict:
        return {
            "total_terms": self.total_terms,
            "total_nodes": self.total_nodes,
            "max_depth": self.max_depth,
            "k": self.k,
            "build_time_ms": round(self.build_time_ms, 1),
            "frozen": self._frozen
        }
    
    def visualize(self, max_depth: int = 4, max_children: int = 3):
        """Print a visual representation of the trie"""
        print(f"\n  TRIE VISUALIZATION (depth≤{max_depth}):")
        self._visualize_node(self.root, "", "", max_depth, max_children)
    
    def _visualize_node(self, node: TrieNode, prefix: str,
                        indent: str, max_depth: int, max_children: int):
        # Show top-K for this node
        if node.top_k and node.depth > 0:
            top_terms = [t[1][:20] for t in node.top_k[:3]]
            terminal = " ★" if node.is_terminal else ""
            print(f"{indent}'{prefix}'{terminal}  "
                  f"→ top: {top_terms}")
        
        if node.depth >= max_depth:
            if node.children:
                print(f"{indent}  └── ... ({len(node.children)} more)")
            return
        
        children = sorted(node.children.items())[:max_children]
        remaining = len(node.children) - max_children
        
        for i, (char, child) in enumerate(children):
            is_last = (i == len(children) - 1) and remaining <= 0
            connector = "└── " if is_last else "├── "
            new_indent = indent + ("    " if is_last else "│   ")
            self._visualize_node(
                child, char, indent + connector,
                max_depth, max_children
            )
        
        if remaining > 0:
            print(f"{indent}└── ... (+{remaining} more branches)")

class CompressedTrieNode:
    """
    Node in a compressed trie (Patricia/Radix tree).
    Instead of one character per node, stores a STRING segment.
    
    Standard:  n → e → w → ' ' → y → o → r → k  (8 nodes)
    Compressed: "new " → "york"                    (2 nodes)
    """
    
    def __init__(self, segment: str = ""):
        self.segment = segment
        self.children: Dict[str, 'CompressedTrieNode'] = {}
        self.is_terminal: bool = False
        self.term: str = ""
        self.frequency: int = 0
        self.top_k: List[Tuple[int, str]] = []


class CompressedTrie:
    """Memory-efficient trie that merges single-child chains"""
    
    def __init__(self, k: int = 10):
        self.root = CompressedTrieNode()
        self.k = k
        self.total_terms = 0
        self.total_nodes = 1
    
    def insert(self, term: str, frequency: int = 1):
        normalized = term.lower().strip()
        if not normalized:
            return
        self._insert_recursive(self.root, normalized, normalized, frequency)
        self.total_terms += 1
    
    def _insert_recursive(self, node: CompressedTrieNode,
                          remaining: str, full_term: str, freq: int):
        if not remaining:
            node.is_terminal = True
            node.term = full_term
            node.frequency = max(node.frequency, freq)
            return
        
        first_char = remaining[0]
        
        if first_char in node.children:
            child = node.children[first_char]
            # Find common prefix between remaining and child's segment
            common_len = 0
            while (common_len < len(child.segment) and
                   common_len < len(remaining) and
                   child.segment[common_len] == remaining[common_len]):
                common_len += 1
            
            if common_len == len(child.segment):
                # Full match with child's segment — continue deeper
                self._insert_recursive(
                    child, remaining[common_len:], full_term, freq
                )
            else:
                # Partial match — split the child node
                # Create new internal node with common prefix
                new_node = CompressedTrieNode(child.segment[:common_len])
                self.total_nodes += 1
                
                # Old child becomes child of new node (with remaining segment)
                child.segment = child.segment[common_len:]
                new_node.children[child.segment[0]] = child
                
                # Replace in parent
                node.children[first_char] = new_node
                
                # Insert remaining string
                if common_len < len(remaining):
                    new_leaf = CompressedTrieNode(remaining[common_len:])
                    new_leaf.is_terminal = True
                    new_leaf.term = full_term
                    new_leaf.frequency = freq
                    new_node.children[remaining[common_len]] = new_leaf
                    self.total_nodes += 1
                else:
                    new_node.is_terminal = True
                    new_node.term = full_term
                    new_node.frequency = freq
        else:
            # No matching child — create new leaf
            new_node = CompressedTrieNode(remaining)
            new_node.is_terminal = True
            new_node.term = full_term
            new_node.frequency = freq
            node.children[first_char] = new_node
            self.total_nodes += 1
    
    def search(self, prefix: str) -> List[Tuple[int, str]]:
        """Walk compressed trie and return precomputed top-K"""
        normalized = prefix.lower().strip()
        if not normalized:
            return self.root.top_k
        
        node = self.root
        remaining = normalized
        
        while remaining:
            first_char = remaining[0]
            if first_char not in node.children:
                return []
            
            child = node.children[first_char]
            
            if remaining.startswith(child.segment):
                # Full segment match — move deeper
                remaining = remaining[len(child.segment):]
                node = child
            elif child.segment.startswith(remaining):
                # Prefix is within this segment — this IS the node
                node = child
                remaining = ""
            else:
                # Mismatch
                return []
        
        return node.top_k
    
    def build_top_k(self):
        """Precompute top-K for all nodes (same logic as standard trie)"""
        self._build_recursive(self.root)
    
    def _build_recursive(self, node: CompressedTrieNode) -> List[Tuple[int, str]]:
        candidates = []
        for child in node.children.values():
            child_results = self._build_recursive(child)
            candidates.extend(child_results)
        if node.is_terminal:
            candidates.append((node.frequency, node.term))
        node.top_k = heapq.nlargest(self.k, candidates, key=lambda x: x[0])
        return node.top_k

# L1 CACHE: In-Process LRU (Reference Part B above)
# ================================================================

class LRUCache:
    def __init__(self, capacity: int = 1000, ttl_seconds: float = 60.0):
        self.capacity = capacity
        self.ttl_seconds = ttl_seconds
        self.cache: OrderedDict = OrderedDict()
        self.timestamps: Dict[str, float] = {}
        self._lock = threading.Lock()
        self.hits = 0
        self.misses = 0

    def get(self, key: str) -> Optional[List]:
        with self._lock:
            if key in self.cache:
                if time.time() - self.timestamps[key] > self.ttl_seconds:
                    del self.cache[key]
                    del self.timestamps[key]
                    self.misses += 1
                    return None
                self.cache.move_to_end(key)
                self.hits += 1
                return self.cache[key]
            self.misses += 1
            return None

    def put(self, key: str, value: List):
        with self._lock:
            if key in self.cache:
                self.cache.move_to_end(key)
                self.cache[key] = value
                self.timestamps[key] = time.time()
            else:
                if len(self.cache) >= self.capacity:
                    oldest_key, _ = self.cache.popitem(last=False)
                    del self.timestamps[oldest_key]
                self.cache[key] = value
                self.timestamps[key] = time.time()

    @property
    def hit_rate(self) -> float:
        total = self.hits + self.misses
        return self.hits / total if total > 0 else 0.0

    def get_stats(self) -> Dict:
        return {
            "size": len(self.cache), "capacity": self.capacity,
            "hits": self.hits, "misses": self.misses,
            "hit_rate": f"{self.hit_rate:.1%}"
        }

class DistributedCache:
    def __init__(self, default_ttl: float = 300.0):
        self.store: Dict[str, Dict] = {}
        self.default_ttl = default_ttl
        self._lock = threading.Lock()
        self.hits = 0

    def get(self, key: str) -> Optional[List]:
        with self._lock:
            if key in self.store:
                entry = self.store[key]
                if time.time() < entry["expires_at"]:
                    self.hits += 1
                    time.sleep(0.001)  # Simulate 1ms network latency
                    return entry["value"]
                else:
                    del self.store[key]
            self.misses += 1
            return None

    def put(self, key: str, value: List, ttl: float = None):
        ttl = ttl or self.default_ttl
        with self._lock:
            self.store[key] = {
                "value": value,
                "expires_at": time.time() + ttl,
                "created_at": time.time()
            }

    @property
    def hit_rate(self) -> float:
        total = self.hits + self.misses
        return self.hits / total if total > 0 else 0.0

    def get_stats(self) -> Dict:
        return {
            "size": len(self.store), "hits": self.hits,
            "misses": self.misses, "hit_rate": f"{self.hit_rate:.1%}"
        }

# TRIE WITH PRECOMPUTED TOP-K (Simplified from Part A)
# ================================================================

class TrieNode:
    def __init__(self):
        self.children: Dict[str, 'TrieNode'] = {}
        self.is_terminal: bool = False
        self.term: str = ""
        self.frequency: int = 0
        self.top_k: List[Tuple[int, str]] = []


class AutocompleteTrie:
    def __init__(self, k: int = 10):
        self.root = TrieNode()
        self.k = k
        self.total_terms = 0
        self.total_nodes = 1

    def insert(self, term: str, frequency: int = 1):
        normalized = term.lower().strip()
        if not normalized:
            return
        node = self.root
        for char in normalized:
            if char not in node.children:
                node.children[char] = TrieNode()
                self.total_nodes += 1
            node = node.children[char]
        if not node.is_terminal:
            self.total_terms += 1
        node.is_terminal = True
        node.term = normalized
        node.frequency = max(node.frequency, frequency)

    def build_top_k(self):
        self._build_recursive(self.root)

    def _build_recursive(self, node: TrieNode) -> List[Tuple[int, str]]:
        candidates = []
        for child in node.children.values():
            candidates.extend(self._build_recursive(child))
        if node.is_terminal:
            candidates.append((node.frequency, node.term))
        node.top_k = heapq.nlargest(self.k, candidates, key=lambda x: x[0])
        return node.top_k

    def search(self, prefix: str) -> List[Tuple[int, str]]:
        normalized = prefix.lower().strip()
        if not normalized:
            return self.root.top_k
        node = self.root
        for char in normalized:
            if char not in node.children:
                return []
            node = node.children[char]
        return node.top_k

# TRENDING TERMS LAYER — Real-time boost on top of static trie
# ================================================================

class TrendingLayer:
    """
    Captures real-time search trends that may not yet be
    in the precomputed trie (rebuilt only periodically).

    Maintains a sliding window of recent searches.
    Terms that spike above threshold get boosted in results.

    This solves the "breaking news" pitfall:
    → Earthquake happens → everyone searches "earthquake"
    → Static trie (built yesterday) has "earthquake" ranked low
    → Trending layer detects spike → boosts it to top

    Update: real-time (every search logged)
    Read: merged with trie results at query time
    """

    def __init__(self, window_seconds: float = 3600.0,
                 min_count_for_trend: int = 5):
        self.window_seconds = window_seconds
        self.min_count = min_count_for_trend
        self.search_log: List[Tuple[float, str]] = []
        self._lock = threading.Lock()
        self.trend_cache: Dict[str, int] = {}
        self.last_cache_build = 0
        self.cache_ttl = 10.0  # Rebuild trend cache every 10s

    def record_search(self, term: str):
        """Log a search event"""
        normalized = term.lower().strip()
        with self._lock:
            now = time.time()
            self.search_log.append((now, normalized))
            # Prune old entries periodically
            if len(self.search_log) > 100000:
                cutoff = now - self.window_seconds
                self.search_log = [
                    (t, s) for t, s in self.search_log if t > cutoff
                ]

    def get_trending_for_prefix(self, prefix: str,
                                limit: int = 5) -> List[Tuple[int, str]]:
        """Get trending terms matching this prefix"""
        self._maybe_rebuild_cache()

        normalized = prefix.lower().strip()
        results = []

        with self._lock:
            for term, count in self.trend_cache.items():
                if term.startswith(normalized) and count >= self.min_count:
                    results.append((count * 100, term))  # Boost score

        results.sort(key=lambda x: x[0], reverse=True)
        return results[:limit]

    def _maybe_rebuild_cache(self):
        """Rebuild frequency cache from search log"""
        now = time.time()
        if now - self.last_cache_build < self.cache_ttl:
            return

        with self._lock:
            cutoff = now - self.window_seconds
            counts: Dict[str, int] = {}
            for timestamp, term in self.search_log:
                if timestamp > cutoff:
                    counts[term] = counts.get(term, 0) + 1
            self.trend_cache = counts
            self.last_cache_build = now

    def get_stats(self) -> Dict:
        with self._lock:
            return {
                "log_size": len(self.search_log),
                "trending_terms": len(self.trend_cache),
                "top_trending": sorted(
                    self.trend_cache.items(),
                    key=lambda x: x[1], reverse=True
                )[:10]
            }

class QueryAnalytics:
    """
    Tracks the full lifecycle:
    1. User types prefix → suggestions shown
    2. User selects a suggestion (or types full query)
    3. This data feeds back into trie rebuilds

    Metrics:
    → Click-through rate per prefix
    → Position of selected suggestion
    → Prefixes that produce zero results
    → Average suggestions shown before selection
    """

    def __init__(self):
        self._lock = threading.Lock()
        self.queries: List[Dict] = []
        self.selections: List[Dict] = []
        self.zero_result_prefixes: Dict[str, int] = {}

    def log_query(self, prefix: str, num_results: int,
                  latency_ms: float, cache_layer: str):
        with self._lock:
            self.queries.append({
                "time": time.time(),
                "prefix": prefix,
                "num_results": num_results,
                "latency_ms": round(latency_ms, 2),
                "cache_layer": cache_layer
            })
            if num_results == 0:
                self.zero_result_prefixes[prefix] = (
                    self.zero_result_prefixes.get(prefix, 0) + 1
                )

    def log_selection(self, prefix: str, selected_term: str,
                      position: int):
        with self._lock:
            self.selections.append({
                "time": time.time(),
                "prefix": prefix,
                "selected": selected_term,
                "position": position
            })

    def get_report(self) -> Dict:
        with self._lock:
            total_queries = len(self.queries)
            total_selections = len(self.selections)

            if total_queries == 0:
                return {"total_queries": 0}

            avg_latency = sum(q["latency_ms"] for q in self.queries) / total_queries
            zero_results = sum(1 for q in self.queries if q["num_results"] == 0)

            # Cache layer distribution
            cache_dist = {}
            for q in self.queries:
                layer = q["cache_layer"]
                cache_dist[layer] = cache_dist.get(layer, 0) + 1

            # Average selection position
            avg_position = 0
            if total_selections > 0:
                avg_position = sum(
                    s["position"] for s in self.selections
                ) / total_selections

            return {
                "total_queries": total_queries,
                "total_selections": total_selections,
                "click_through_rate": (
                    f"{total_selections / total_queries:.1%}"
                    if total_queries > 0 else "N/A"
                ),
                "avg_latency_ms": round(avg_latency, 2),
                "zero_result_rate": (
                    f"{zero_results / total_queries:.1%}"
                ),
                "avg_selection_position": round(avg_position, 1),
                "cache_layer_distribution": cache_dist,
                "top_zero_result_prefixes": sorted(
                    self.zero_result_prefixes.items(),
                    key=lambda x: x[1], reverse=True
                )[:10]
            }

# ================================================================

class SuggestionBlocklist:
    """
    Prevents offensive or inappropriate terms from appearing.

    Applied at:
    1. Build time: blocked terms excluded from trie
    2. Query time: results filtered before returning
    3. Manual review: flagged terms queued for review
    """

    def __init__(self):
        self.blocked_terms: set = set()
        self.blocked_prefixes: set = set()

    def add_blocked_term(self, term: str):
        self.blocked_terms.add(term.lower().strip())

    def add_blocked_prefix(self, prefix: str):
        self.blocked_prefixes.add(prefix.lower().strip())

    def filter_results(self, results: List[Tuple[int, str]]) -> List[Tuple[int, str]]:
        """Remove blocked terms from suggestion list"""
        filtered = []
        for freq, term in results:
            if term in self.blocked_terms:
                continue
            if any(term.startswith(bp) for bp in self.blocked_prefixes):
                continue
            filtered.append((freq, term))
        return filtered

    def is_blocked(self, term: str) -> bool:
        normalized = term.lower().strip()
        if normalized in self.blocked_terms:
            return True
        return any(normalized.startswith(bp) for bp in self.blocked_prefixes)

class AutocompleteService:
    """
    Complete autocomplete system combining:
    → L1 Cache (in-process LRU) — Project 5
    → L2 Cache (distributed Redis sim) — Project 23
    → L3 Trie (precomputed top-K) — This project
    → Trending layer (real-time boost)
    → Blocklist (content safety)
    → Analytics (monitoring & feedback)

    Request flow:
    1. Normalize prefix
    2. Check L1 cache → HIT? Return immediately
    3. Check L2 cache → HIT? Populate L1, return
    4. Query trie → merge with trending → filter blocklist
    5. Populate L1 + L2 caches
    6. Return results
    7. Log analytics
    """

    def __init__(self, k: int = 10):
        self.k = k
        self.trie = AutocompleteTrie(k=k)
        self.l1_cache = LRUCache(capacity=5000, ttl_seconds=60)
        self.l2_cache = DistributedCache(default_ttl=300)
        self.trending = TrendingLayer(window_seconds=3600, min_count_for_trend=3)
        self.blocklist = SuggestionBlocklist()
        self.analytics = QueryAnalytics()
        self._serving = False
        self._trie_lock = threading.Lock()
        self._trie_version = 0

    # ============================================================
    # BUILD PHASE
    # ============================================================

    def build(self, terms_with_freq: List[Tuple[str, int]],
              blocked_terms: List[str] = None):
        """
        Build the trie from term data.
        Called during offline build phase.
        """
        print(f"\n  [BUILD] Starting trie build...")
        start = time.time()

        # Set up blocklist
        if blocked_terms:
            for term in blocked_terms:
                self.blocklist.add_blocked_term(term)
            print(f"  [BUILD] Blocklist: {len(blocked_terms)} terms")

        # Filter blocked terms before inserting
        clean_terms = [
            (term, freq) for term, freq in terms_with_freq
            if not self.blocklist.is_blocked(term)
        ]

        # Build new trie
        new_trie = AutocompleteTrie(k=self.k)
        for term, freq in clean_terms:
            new_trie.insert(term, freq)
        new_trie.build_top_k()

        # Atomic swap (Project 27 insight: no locks needed for reads
        # if we swap the pointer atomically)
        with self._trie_lock:
            self.trie = new_trie
            self._trie_version += 1

        elapsed = (time.time() - start) * 1000
        self._serving = True

        print(f"  [BUILD] Complete in {elapsed:.1f}ms")
        print(f"  [BUILD] Terms: {new_trie.total_terms} | "
              f"Nodes: {new_trie.total_nodes} | "
              f"Version: {self._trie_version}")

    # ============================================================
    # SERVE PHASE — The hot path
    # ============================================================

    def suggest(self, prefix: str) -> Dict:
        """
        Main query endpoint — returns suggestions for prefix.

        This is the HOT PATH — called millions of times per second.
        Every microsecond matters here.
        """
        if not self._serving:
            return {"prefix": prefix, "suggestions": [],
                    "error": "Service not ready"}

        start = time.time()
        normalized = prefix.lower().strip()
        cache_key = f"ac:{normalized}"
        cache_layer = "NONE"

        # ---- Layer 1: In-Process LRU Cache ----
        cached = self.l1_cache.get(cache_key)
        if cached is not None:
            cache_layer = "L1_MEMORY"
            latency = (time.time() - start) * 1000
            self.analytics.log_query(normalized, len(cached),
                                     latency, cache_layer)
            return self._format_response(normalized, cached,
                                         latency, cache_layer)

        # ---- Layer 2: Distributed Cache ----
        cached = self.l2_cache.get(cache_key)
        if cached is not None:
            cache_layer = "L2_DISTRIBUTED"
            # Populate L1
            self.l1_cache.put(cache_key, cached)
            latency = (time.time() - start) * 1000
            self.analytics.log_query(normalized, len(cached),
                                     latency, cache_layer)
            return self._format_response(normalized, cached,
                                         latency, cache_layer)

        # ---- Layer 3: Trie Lookup ----
        cache_layer = "L3_TRIE"
        trie_results = self.trie.search(normalized)

        # ---- Merge with trending ----
        trending_results = self.trending.get_trending_for_prefix(
            normalized, limit=3
        )

        merged = self._merge_results(trie_results, trending_results)

        # ---- Filter blocklist ----
        filtered = self.blocklist.filter_results(merged)

        # Limit to K results
        final = filtered[:self.k]

        # ---- Populate caches ----
        # Determine TTL based on prefix length
        # Short prefixes = long TTL (stable, high reuse)
        # Long prefixes = short TTL (specific, low reuse)
        ttl_l2 = self._calculate_ttl(normalized)
        self.l1_cache.put(cache_key, final)
        self.l2_cache.put(cache_key, final, ttl=ttl_l2)

        latency = (time.time() - start) * 1000
        self.analytics.log_query(normalized, len(final),
                                 latency, cache_layer)

        return self._format_response(normalized, final,
                                     latency, cache_layer)

    def record_selection(self, prefix: str, selected_term: str,
                         position: int):
        """User selected a suggestion — log for analytics and trending"""
        self.trending.record_search(selected_term)
        self.analytics.log_selection(prefix, selected_term, position)

    # ============================================================
    # INTERNAL HELPERS
    # ============================================================

    def _merge_results(self, trie_results: List[Tuple[int, str]],
                       trending_results: List[Tuple[int, str]]
                       ) -> List[Tuple[int, str]]:
        """
        Merge static trie results with trending results.
        Trending terms get a boost but don't completely override.

        Strategy:
        → Trending terms inserted at top positions
        → Deduplicate (term might be in both lists)
        → Maintain overall limit of K
        """
        seen = set()
        merged = []

        # Trending first (with boost)
        for freq, term in trending_results:
            if term not in seen:
                merged.append((freq, term))
                seen.add(term)

        # Then trie results
        for freq, term in trie_results:
            if term not in seen:
                merged.append((freq, term))
                seen.add(term)

        # Sort by frequency (trending boost ensures they stay high)
        merged.sort(key=lambda x: x[0], reverse=True)
        return merged

    def _calculate_ttl(self, prefix: str) -> float:
        """
        Adaptive TTL based on prefix characteristics.

        Short prefix → many users type it → long TTL → high reuse
        Long prefix → few users type it → short TTL → save memory
        """
        length = len(prefix)
        if length <= 1:
            return 3600.0    # 1 hour (only 26-36 possible 1-char prefixes)
        elif length <= 2:
            return 1800.0    # 30 minutes
        elif length <= 3:
            return 600.0     # 10 minutes
        elif length <= 5:
            return 120.0     # 2 minutes
        else:
            return 30.0      # 30 seconds

    def _format_response(self, prefix: str,
                         results: List[Tuple[int, str]],
                         latency_ms: float,
                         cache_layer: str) -> Dict:
        return {
            "prefix": prefix,
            "suggestions": [
                {"rank": i + 1, "term": term, "score": freq}
                for i, (freq, term) in enumerate(results)
            ],
            "count": len(results),
            "latency_ms": round(latency_ms, 3),
            "served_from": cache_layer
        }

    # ============================================================
    # MONITORING
    # ============================================================

    def get_system_stats(self) -> Dict:
        return {
            "trie": {
                "terms": self.trie.total_terms,
                "nodes": self.trie.total_nodes,
                "version": self._trie_version
            },
            "l1_cache": self.l1_cache.get_stats(),
            "l2_cache": self.l2_cache.get_stats(),
            "trending": self.trending.get_stats(),
            "analytics": self.analytics.get_report()
        }

    def print_dashboard(self):
        stats = self.get_system_stats()
        print(f"\n  {'=' * 65}")
        print(f"  AUTOCOMPLETE SERVICE DASHBOARD")
        print(f"  {'=' * 65}")
        print(f"  TRIE: {stats['trie']['terms']} terms | "
              f"{stats['trie']['nodes']} nodes | "
              f"v{stats['trie']['version']}")
        print(f"  L1 CACHE: {stats['l1_cache']['size']}/"
              f"{stats['l1_cache']['capacity']} entries | "
              f"Hit rate: {stats['l1_cache']['hit_rate']}")
        print(f"  L2 CACHE: {stats['l2_cache']['size']} entries | "
              f"Hit rate: {stats['l2_cache']['hit_rate']}")

        analytics = stats['analytics']
        if analytics['total_queries'] > 0:
            print(f"  QUERIES: {analytics['total_queries']} total | "
                  f"Avg latency: {analytics['avg_latency_ms']}ms")
            print(f"  SELECTIONS: {analytics['total_selections']} | "
                  f"CTR: {analytics['click_through_rate']}")
            print(f"  ZERO RESULTS: {analytics['zero_result_rate']}")
            if analytics.get('cache_layer_distribution'):
                print(f"  CACHE DIST: {analytics['cache_layer_distribution']}")
        print(f"  {'=' * 65}")

class SimulatedUser:
    """
    Simulates a real user typing in a search box.

    Behavior:
    1. User has a target query in mind ("new york")
    2. Types one character at a time with delays
    3. After each character, receives suggestions
    4. May select a suggestion before finishing typing
    5. Probability of selecting increases with each character
    """

    def __init__(self, user_id: str, target_query: str,
                 service: AutocompleteService):
        self.user_id = user_id
        self.target = target_query.lower().strip()
        self.service = service
        self.keystrokes = 0
        self.selected = False
        self.selected_term = ""
        self.selected_at_keystroke = 0
        self.responses: List[Dict] = []

    def simulate_typing(self, typing_delay: float = 0.05):
        """Simulate typing the query character by character"""
        for i in range(1, len(self.target) + 1):
            prefix = self.target[:i]
            self.keystrokes += 1

            # Query autocomplete
            response = self.service.suggest(prefix)
            self.responses.append(response)

            # Check if target appears in suggestions
            suggestions = [s["term"] for s in response["suggestions"]]

            if self.target in suggestions:
                # Probability of selecting increases with characters typed
                select_prob = min(0.3 + (i / len(self.target)) * 0.5, 0.9)
                if random.random() < select_prob:
                    position = suggestions.index(self.target) + 1
                    self.service.record_selection(
                        prefix, self.target, position
                    )
                    self.selected = True
                    self.selected_term = self.target
                    self.selected_at_keystroke = i
                    break

            time.sleep(typing_delay)

        # If never selected from suggestions, "submit" the full query
        if not self.selected:
            self.service.record_selection(
                self.target, self.target, 0  # Position 0 = typed manually
            )

    def get_summary(self) -> Dict:
        return {
            "user": self.user_id,
            "target": self.target,
            "keystrokes": self.keystrokes,
            "selected_from_suggestions": self.selected,
            "selected_at_keystroke": self.selected_at_keystroke,
            "total_target_length": len(self.target)
        }

    print("\n" + "=" * 70)
    print("  DEMO 4: CONCURRENT USERS — THROUGHPUT TEST")
    print("=" * 70)

    service = AutocompleteService(k=8)
    dataset = build_travel_dataset()
    service.build(dataset)

    # Target queries for simulated users
    user_queries = [
        "new york", "paris flights", "cheap hotels",
        "london", "tokyo flights", "barcelona",
        "bali hotels", "las vegas", "san francisco",
        "dubai", "bangkok", "hawaii flights",
        "rome", "amsterdam", "singapore",
        "istanbul", "miami beach", "cancun",
        "chicago", "hong kong"
    ]

    num_users = len(user_queries)
    threads = []
    user_summaries = []
    summary_lock = threading.Lock()

    def user_task(user_id, query):
        user = SimulatedUser(f"user-{user_id:02d}", query, service)
        user.simulate_typing(typing_delay=0.01)
        summary = user.get_summary()
        with summary_lock:
            user_summaries.append(summary)

    print(f"\n  Launching {num_users} concurrent users...")
    start = time.time()

    for i, query in enumerate(user_queries):
        t = threading.Thread(target=user_task, args=(i + 1, query))
        threads.append(t)
        t.start()

    for t in threads:
        t.join()

    elapsed = time.time() - start

    # Results
    total_keystrokes = sum(s["keystrokes"] for s in user_summaries)
    selected_count = sum(1 for s in user_summaries
                         if s["selected_from_suggestions"])
    avg_keystrokes_to_select = 0
    selected_users = [s for s in user_summaries
                      if s["selected_from_suggestions"]]
    if selected_users:
        avg_keystrokes_to_select = sum(
            s["selected_at_keystroke"] for s in selected_users
        ) / len(selected_users)

    print(f"\n  RESULTS:")
    print(f"  {'User':<12} {'Target':<22} {'Keys':<6} "
          f"{'Selected?':<11} {'At Key#'}")
    print(f"  {'-'*12} {'-'*22} {'-'*6} {'-'*11} {'-'*7}")
    for s in sorted(user_summaries, key=lambda x: x["user"]):
        sel = "✅ Yes" if s["selected_from_suggestions"] else "❌ No"
        at_key = str(s["selected_at_keystroke"]) if s["selected_from_suggestions"] else "-"
        print(f"  {s['user']:<12} {s['target']:<22} "
              f"{s['keystrokes']:<6} {sel:<11} {at_key}")

    print(f"\n  SUMMARY:")
    print(f"    Total users:              {num_users}")
    print(f"    Total keystrokes:         {total_keystrokes}")
    print(f"    Time elapsed:             {elapsed:.2f}s")
    print(f"    Throughput:               "
          f"{total_keystrokes / elapsed:.0f} queries/sec")
    print(f"    Selected from suggestions: "
          f"{selected_count}/{num_users} "
          f"({selected_count/num_users:.0%})")
    print(f"    Avg keystrokes to select: {avg_keystrokes_to_select:.1f}")
    print(f"    Avg keys saved per user:  "
          f"{sum(s['total_target_length'] for s in selected_users) / max(1,len(selected_users)) - avg_keystrokes_to_select:.1f}")

    service.print_dashboard()

THE PROBLEM THAT COMES NEXT:
  → User searches "new york" → log event created
  → User books a flight → booking event created
  → Admin adds new route → catalog event created
  → Price changes → pricing event created
  → Each of these events needs to trigger DIFFERENT downstream actions:
    - Search log → update trending scores
    - Booking → update popularity rankings
    - New route → trigger trie rebuild
    - Price change → update deal suggestions
  → These are DIFFERENT systems consuming the SAME events
  → We can't have tight coupling between producer and consumers
  → We can't lose events even if a consumer is temporarily down

WHAT PROJECT 29 SOLVES:
  → Event-Driven Architecture (EDA)
  → Event Bus: decouple producers from consumers
  → Event Sourcing: store events as source of truth
  → Eventual Consistency: systems converge over time
  → Idempotency: safe to process the same event twice
  → This is HOW modern microservices communicate
  → Message Queues (Project 11) were point-to-point
  → Events are broadcast to MANY consumers simultaneously

    - Notification service (Project 24 — confirm booking)
    - Inventory service (update seat availability)
  → Each consumer processes independently, at its own pace
  → This is the backbone of modern distributed systems
