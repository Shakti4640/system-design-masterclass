🎯 HYPOTHETICAL PROBLEM

You manage a hospital with 200 patients in ICU beds
→ Each patient has a heart monitor attached
→ The monitor beeps every second to confirm "patient is alive"
→ If the beep STOPS for 5 seconds → nurse is alerted
→ If the beep STOPS for 15 seconds → doctor is paged IMMEDIATELY
→ If the beep is IRREGULAR → something is WRONG but patient is alive

NOW translate this to your system:
→ You have 30 microservices running in production
→ Service A talks to Service B talks to Service C
→ At 3:00 AM, Service B silently DIES
→ Service A keeps sending requests to dead Service B
→ Requests pile up → timeout → Service A slows down
→ Service C gets no data → starts returning errors
→ By 6:00 AM, your ENTIRE platform is down
→ You wake up to 5,000 angry customer emails

NOBODY KNEW Service B died at 3:00 AM because:
→ No heartbeat monitoring
→ No health check endpoint
→ No alerting system
→ No automated recovery

The load balancer from Project 6 kept sending traffic
to a DEAD server because it had no way to know it was dead

The message queue from Project 11 kept pushing messages
to a consumer that was no longer processing them

THIS project solves all of that

🔍 HOW TO APPROACH IT

Step 1: Define what "healthy" means for EACH service
        → Can it respond to requests? (Liveness)
        → Can it actually DO useful work? (Readiness)
        → Can it reach its dependencies? (Dependency health)

Step 2: Build health check endpoints INTO every service
        → A lightweight endpoint that reports current state
        → Not just "I'm running" but "I'm running AND functional"

Step 3: Build a heartbeat mechanism
        → Services periodically ANNOUNCE they are alive
        → OR a monitor periodically ASKS if they are alive
        → Two models: Push vs Pull

Step 4: Define failure detection logic
        → How many missed heartbeats = "probably dead"?
        → How to distinguish "slow" from "dead"?
        → How to avoid false positives?

Step 5: Build alerting and automated response
        → Alert humans when something is degraded
        → Automatically remove dead nodes from rotation
        → Automatically restart failed services if possible

Step 6: Build a monitoring dashboard
        → Aggregate health data from all services
        → Show historical trends (was it healthy yesterday?)
        → Show current state at a glance

📌 WHAT COULD BE DONE

Option A: Simple Ping Check
  → Just check if port is open (TCP connect)
  → Knows: "process is running"
  → Doesn't know: "process is actually working"
  → Shallow but fast

Option B: HTTP Health Endpoint
  → Service exposes GET /health
  → Returns detailed status including dependency checks
  → Knows: "service is running AND functional"
  → Deeper but slightly slower

Option C: Heartbeat with Lease
  → Service sends heartbeat every N seconds
  → If no heartbeat for 3N seconds → considered dead
  → Good for: service discovery, cluster membership
  → Used by: ZooKeeper, etcd, Consul

Option D: Deep Health Check (Readiness Probe)
  → Check database connectivity
  → Check cache reachability
  → Check disk space
  → Check memory usage
  → Check dependent service availability
  → Expensive but thorough

Option E: Synthetic Transaction Monitoring
  → Actually PERFORM a real operation end-to-end
  → Place a fake order, verify it completes
  → Most accurate but most expensive
  → Used for critical business flows

FOR THIS PROJECT:
  → We implement Options A + B + C + D as a complete system
  → Option E is the philosophy behind integration testing

✅ WHAT SHOULD DO

1. SEPARATE liveness from readiness
   → Liveness: "Is the process alive?" (restart if no)
   → Readiness: "Can it serve traffic?" (remove from LB if no)
   → A service can be ALIVE but NOT READY
     (e.g., still loading data, warming cache)

2. MAKE health checks lightweight
   → Health endpoint should respond in < 100ms
   → Don't do heavy computation in health check
   → Cache dependency check results for short periods

3. USE timeouts on health checks themselves
   → If health check takes > 2 seconds → treat as unhealthy
   → A hung service that never responds is worse than a dead one

4. IMPLEMENT graduated severity levels
   → HEALTHY: Everything works perfectly
   → DEGRADED: Working but with issues (slow DB, high memory)
   → UNHEALTHY: Cannot serve traffic properly
   → DEAD: Not responding at all

5. TRACK health over time, not just current state
   → A service that flaps (healthy → unhealthy → healthy → unhealthy)
     is a PROBLEM even if it's currently healthy
   → Store health history for trend analysis

6. ALERT on transitions, not states
   → Don't send "unhealthy" alert every 5 seconds
   → Send ONE alert when state CHANGES from healthy to unhealthy
   → Send RECOVERY alert when it comes back

7. IMPLEMENT cascading dependency checks
   → If DB is down → all services using DB are degraded
   → Don't alert on 20 services when the ROOT CAUSE is one DB
   → Correlate failures (covered deeply in Project 22)

❌ WHAT SHOULD NOT DO

1. NEVER make health checks expensive
   → Don't run full queries
   → Don't process large datasets
   → Don't call external APIs synchronously
   → A health check that takes 10 seconds CAUSES the problem it detects

2. NEVER use only TCP ping as health check
   → Process can have port open but be completely deadlocked
   → Always check APPLICATION-level health

3. NEVER hardcode thresholds without tuning
   → "3 missed heartbeats = dead" might cause false positives
   → In a network with 200ms latency, 1-second heartbeat is too aggressive
   → Tune based on YOUR system's characteristics

4. NEVER ignore partial failures
   → "Database is slow but responding" is NOT healthy
   → "Memory at 95%" is NOT healthy
   → Degraded states need attention BEFORE they become failures

5. NEVER create circular health dependencies
   → Service A's health check calls Service B
   → Service B's health check calls Service A
   → If either goes down → both report unhealthy → chaos

6. NEVER alert on single failures immediately
   → Network blip causes one missed heartbeat
   → That's normal, not an emergency
   → Use consecutive failure threshold (e.g., 3 failures in a row)

7. NEVER put business logic in health endpoints
   → Health check should be SEPARATE from business routes
   → Should work even when the business logic is broken
   → It's the smoke detector, not the kitchen

🚫 WHEN NOT TO USE

1. DON'T use heartbeat for request-level monitoring
   → Heartbeat says "service is alive"
   → It does NOT say "this specific request failed"
   → For request-level: use distributed tracing (Project 22)

2. DON'T use health checks as performance monitoring
   → Health check says "can respond"
   → It does NOT say "p99 latency is 2 seconds"
   → For performance: use metrics collection (Prometheus-style)

3. DON'T use pull-model health checks for 10,000 services
   → Monitor polling 10,000 services every 5 seconds = 2000 req/sec
   → At scale: use push-model + aggregation
   → Or use gossip protocol (Project 33)

4. DON'T use deep health checks too frequently
   → Checking DB + Cache + Disk + Memory every 1 second
   → The health checks themselves become load
   → Deep checks: every 30-60 seconds
   → Shallow checks: every 5-10 seconds

5. DON'T use health checks as the ONLY failure detection
   → Health check passes but responses are all errors
   → Need error rate monitoring too
   → Need latency monitoring too
   → Health checks are ONE signal, not the ONLY signal

  MONITOR LOOP (runs every check_interval seconds):

  for each registered_service in service_list:
    │
    ├─→ Send HTTP GET to service_url/health
    │     with timeout = 2 seconds
    │
    ├─→ IF response received AND status == 200:
    │     ├─→ Parse response body for detailed status
    │     ├─→ Record: service=HEALTHY, timestamp=now
    │     ├─→ Reset consecutive_failure_count to 0
    │     └─→ IF was previously UNHEALTHY:
    │           └─→ Send RECOVERY alert
    │
    ├─→ IF response received BUT status != 200:
    │     ├─→ Record: service=DEGRADED, timestamp=now
    │     ├─→ Increment consecutive_failure_count
    │     └─→ IF count >= threshold:
    │           ├─→ Mark as UNHEALTHY
    │           ├─→ Send ALERT
    │           └─→ Remove from load balancer rotation
    │
    └─→ IF no response (timeout/connection refused):
          ├─→ Record: service=UNREACHABLE, timestamp=now
          ├─→ Increment consecutive_failure_count
          └─→ IF count >= threshold:
                ├─→ Mark as DEAD
                ├─→ Send CRITICAL ALERT
                ├─→ Remove from load balancer rotation
                └─→ Trigger auto-restart if configured

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  SERVICE SIDE (sender):
  
  while service_is_running:
    │
    ├─→ Collect local health metrics:
    │     cpu_usage, memory_usage, active_connections,
    │     error_rate, last_request_time
    │
    ├─→ Build heartbeat payload:
    │     {
    │       "service_id": "order-service-3",

    │       "status": "healthy",
    │       "metrics": { ... }
    │     }
    │
    ├─→ Send to central monitor via UDP or TCP
    │     (UDP preferred: fire-and-forget, low overhead)
    │
    └─→ Sleep(heartbeat_interval)


  MONITOR SIDE (receiver):
  
  for each registered_service:
    │
    ├─→ Track: last_heartbeat_received_at
    │
    ├─→ Every check_cycle:
    │     time_since_last = now - last_heartbeat_received_at
    │
    ├─→ IF time_since_last < heartbeat_interval * 2:
    │     └─→ Service is ALIVE
    │
    ├─→ IF time_since_last < heartbeat_interval * 4:
    │     └─→ Service is SUSPECT (might be slow or network issue)
    │
    └─→ IF time_since_last >= heartbeat_interval * 4:
          └─→ Service is DEAD → alert + remove from rotation

MECHANIC 3: LIVENESS vs READINESS (Kubernetes Model)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  LIVENESS PROBE:
    Question: "Is the process fundamentally alive?"
    Check:    Can the HTTP server respond at all?
    Failure:  Process is deadlocked or crashed
    Action:   RESTART the container/process
    
    Example:  GET /healthz → 200 means alive
              Timeout or error → restart
    
    When it matters:
      → Process entered infinite loop
      → Thread deadlock
      → Memory corruption causing hang
      → The process EXISTS but is BRAIN DEAD

  READINESS PROBE:
    Question: "Can this process handle traffic RIGHT NOW?"
    Check:    Are all dependencies available?
    Failure:  Service is alive but not ready to work
    Action:   STOP sending traffic (but don't restart)
    
    Example:  GET /ready → checks DB, cache, config loaded
              200 = ready → send traffic
              503 = not ready → remove from load balancer
    
    When it matters:
      → Service just started, still loading data into cache
      → Database connection pool exhausted
      → Dependent service is down
      → Graceful shutdown in progress

  STARTUP PROBE:
    Question: "Has the process finished initializing?"
    Check:    One-time check during boot
    Failure:  Service needs more time to start
    Action:   WAIT (don't restart yet, don't send traffic)
    
    When it matters:
      → Service loads 10GB model into memory on startup
      → Service runs database migrations on boot
      → Service pre-warms cache before serving

MECHANIC 4: FAILURE DETECTION ACCURACY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  THE FUNDAMENTAL TRADEOFF:
  
  Fast detection (short timeout)
    → More false positives (network blip = "service dead")
    → Unnecessary restarts and alerts
    → Alert fatigue → humans start ignoring alerts
  
  Slow detection (long timeout)
    → Fewer false positives
    → But REAL failures take longer to detect
    → Users suffer while you wait to be "sure"
  
  THE SOLUTION: PHI ACCRUAL FAILURE DETECTOR
  
  Instead of binary alive/dead:
    → Track historical heartbeat arrival times
    → Calculate statistical distribution of intervals
    → Compute "suspicion level" (phi value)
    → phi < 1: probably alive
    → phi > 3: probably dead
    → phi > 8: almost certainly dead
  
  Used by: Cassandra, Akka

  SIMPLER APPROACH: CONSECUTIVE FAILURE THRESHOLD
  
    healthy_threshold = 1   (1 success → mark healthy)
    unhealthy_threshold = 3 (3 consecutive failures → mark unhealthy)
    interval = 10 seconds
    timeout = 3 seconds
    
    This means:
      → Best case detection time: 10 seconds (first check fails)
      → Worst case detection time: 30 seconds (3 checks × 10s)
      → Confirmation time: 30 seconds
      → Recovery detection: 10 seconds (first success)

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: The Health Check That Lies
  → Health endpoint returns 200 OK
  → But the service is returning errors on ALL business requests
  → CAUSE: Health check only checks "am I running"
  → FIX: Include error rate in health calculation
         If error_rate > 50% → report DEGRADED even if running

PITFALL 2: The Thundering Herd on Recovery
  → Service goes down → removed from LB rotation
  → Service comes back → LB sends ALL pending traffic at once
  → Service immediately crashes again under the flood
  → FIX: Gradual traffic ramp-up after recovery
         (send 10% → 25% → 50% → 100%)

PITFALL 3: Health Check Dependency Loop
  → Service A health check calls Service B
  → Service B health check calls Service C
  → Service C health check calls Service A
  → One goes down → all report unhealthy
  → FIX: Health checks should only check DIRECT dependencies
         Use shallow checks for liveness, deep for readiness

PITFALL 4: The Monitor Itself Dies
  → Your monitoring system crashes
  → Nobody monitors the monitor
  → All services could be down and nobody knows
  → FIX: "Who watches the watchmen?"
         → Run monitor in HA (multiple instances)
         → External uptime monitoring (Pingdom, UptimeRobot)
         → Monitor the monitor with a DIFFERENT system

PITFALL 5: Alert Fatigue
  → 500 alerts per day for minor issues
  → Team starts ignoring ALL alerts
  → Critical alert gets buried among noise
  → FIX: Severity levels (INFO/WARN/CRITICAL)
         Only page humans for CRITICAL
         Auto-resolve transient issues
         Deduplicate related alerts

PITFALL 6: Health Check Caching Gone Wrong
  → Cache health result for 60 seconds to reduce load
  → Service dies at second 1
  → Health still reports "healthy" for 59 more seconds
  → FIX: Short cache TTL for health (5-10 seconds max)
         Never cache UNHEALTHY results

PITFALL 7: Clock Skew in Heartbeats
  → Service says "heartbeat at T=100"
  → Monitor's clock says T=95
  → Heartbeat appears to be FROM THE FUTURE
  → Calculations break
  → FIX: Use relative durations, not absolute timestamps
         Or use NTP to synchronize clocks across machines

PITFALL 8: Network Partition False Positive
  → Monitor can't reach Service B (network issue)
  → But Service B is perfectly healthy
  → Clients can reach Service B just fine
  → Monitor marks it DEAD and removes from rotation
  → FIX: Multi-path checking (check from multiple monitors)
         Peer-to-peer health confirmation
         Quorum-based failure detection

📈 SCALABILITY CONSIDERATIONS

SCALING THE MONITORING ITSELF:

Small Scale (< 20 services):
  → Single monitor process doing pull-based checks
  → Check every 10 seconds
  → Simple in-memory state tracking
  → THIS is what we build in code below

Medium Scale (20-500 services):
  → Push-based heartbeats (services report to monitor)
  → Monitor becomes a receiver, not a poller
  → Store health history in time-series database (InfluxDB)
  → Alert evaluation on aggregated data

Large Scale (500-5000 services):
  → Distributed monitoring (multiple monitor instances)
  → Each monitor responsible for subset of services
  → Consistent hashing (Project 14) to assign responsibility
  → Gossip protocol for monitors to share health state
  → Hierarchical health: team → service → instance

Planet Scale (5000+ services):
  → Push-based metrics pipeline
  → Services → Metrics Agents → Aggregation Layer → Query Engine
  → Prometheus federation or Thanos architecture
  → Automated anomaly detection (ML-based)
  → Self-healing systems that auto-restart/replace

NUMBERS TO KNOW:
  → 1 health check HTTP request ≈ 1-5ms server-side
  → 1000 services × every 10s = 100 checks/second (manageable)
  → 10,000 services × every 5s = 2000 checks/second (need distribution)
  → Heartbeat UDP packet ≈ 100-200 bytes
  → Storing 1 metric per service per second for 30 days:

class ServiceHealthChecker:
    """
    Encapsulates ALL health checking logic for a service.
    Every microservice would embed an instance of this.
    """
    
    # Health status constants
    HEALTHY = "healthy"
    DEGRADED = "degraded"
    UNHEALTHY = "unhealthy"
    
    def __init__(self, service_name, service_version):
        self.service_name = service_name
        self.service_version = service_version
        self.start_time = time.time()
        self.is_ready = False
        
        # Track request metrics
        self.total_requests = 0
        self.error_count = 0
        self.request_lock = threading.Lock()
        
        # Dependencies to check
        self.dependencies = {}
        
        # Health thresholds
        self.thresholds = {
            "max_error_rate_pct": 10.0,
            "max_memory_usage_pct": 90.0,
            "max_cpu_usage_pct": 85.0,
            "max_dependency_latency_ms": 200,
            "max_db_pool_usage_pct": 80.0
        }
    
    def register_dependency(self, name, checker_func):
        """Register an external dependency with its health check function"""
        self.dependencies[name] = checker_func
    
    def record_request(self, is_error=False):
        """Called by the service on every request to track metrics"""
        with self.request_lock:
            self.total_requests += 1
            if is_error:
                self.error_count += 1
    
    def get_error_rate(self):
        if self.total_requests == 0:
            return 0.0
        return round((self.error_count / self.total_requests) * 100, 2)
    
    def mark_ready(self):
        """Called when service finishes initialization"""
        self.is_ready = True
    
    def mark_not_ready(self):
        """Called during graceful shutdown"""
        self.is_ready = False
    
    # ---- LIVENESS CHECK (Am I alive at all?) ----
    def check_liveness(self):
        """
        Minimal check — can the process respond?
        If this fails → restart the process
        """
        return {
            "status": "alive",
            "service": self.service_name,
            "timestamp": datetime.now().isoformat()
        }
    
    # ---- READINESS CHECK (Can I serve traffic?) ----
    def check_readiness(self):
        """
        Can this service handle requests RIGHT NOW?
        If this fails → stop sending traffic but don't restart
        """
        if not self.is_ready:
            return {
                "ready": False,
                "reason": "Service is still initializing"
            }
        
        # Check all dependencies
        all_deps_ok = True
        dep_results = {}
        
        for dep_name, checker_func in self.dependencies.items():
            try:
                dep_results[dep_name] = checker_func()
            except Exception as e:
                dep_results[dep_name] = {
                    "connected": False,
                    "error": str(e)
                }
                all_deps_ok = False
        
        return {
            "ready": all_deps_ok,
            "dependencies": dep_results
        }
    
    # ---- DEEP HEALTH CHECK (Full status report) ----
    def check_health(self):
        """
        Comprehensive health check with all details.
        This is what monitoring dashboards consume.
        """
        health_report = {
            "service": self.service_name,
            "version": self.service_version,
            "timestamp": datetime.now().isoformat(),
            "uptime_seconds": round(time.time() - self.start_time, 1),
            "is_ready": self.is_ready
        }
        
        # ---- System Metrics ----
        try:
            cpu_pct = psutil.cpu_percent(interval=0.1)
            memory = psutil.virtual_memory()
            disk = psutil.disk_usage('/')
            
            system_metrics = {
                "cpu_usage_pct": cpu_pct,
                "memory_usage_pct": memory.percent,
                "memory_available_mb": round(memory.available / 1024 / 1024, 1),
                "disk_usage_pct": disk.percent
            }
        except Exception:
            system_metrics = {"error": "Could not collect system metrics"}
        
        health_report["system"] = system_metrics
        
        # ---- Application Metrics ----
        error_rate = self.get_error_rate()
        app_metrics = {
            "total_requests": self.total_requests,
            "error_count": self.error_count,
            "error_rate_pct": error_rate
        }
        health_report["application"] = app_metrics
        
        # ---- Dependency Checks ----
        dependency_results = {}
        dependency_issues = []
        
        for dep_name, checker_func in self.dependencies.items():
            try:
                result = checker_func()
                result["status"] = "healthy"
                
                # Check for degradation signals
                if result.get("latency_ms", 0) > self.thresholds["max_dependency_latency_ms"]:
                    result["status"] = "degraded"
                    dependency_issues.append(
                        f"{dep_name}: high latency ({result['latency_ms']}ms)"
                    )
                
                if result.get("pool_usage_pct", 0) > self.thresholds["max_db_pool_usage_pct"]:
                    result["status"] = "degraded"
                    dependency_issues.append(
                        f"{dep_name}: pool nearly exhausted ({result['pool_usage_pct']}%)"
                    )
                
                dependency_results[dep_name] = result
                
            except Exception as e:
                dependency_results[dep_name] = {
                    "status": "unhealthy",
                    "connected": False,
                    "error": str(e)
                }
                dependency_issues.append(f"{dep_name}: {str(e)}")
        
        health_report["dependencies"] = dependency_results
        
        # ---- Overall Status Determination ----
        overall_status = self.HEALTHY
        issues = []
        
        # Check system thresholds
        cpu = system_metrics.get("cpu_usage_pct", 0)
        mem = system_metrics.get("memory_usage_pct", 0)
        
        if cpu > self.thresholds["max_cpu_usage_pct"]:
            overall_status = self.DEGRADED
            issues.append(f"High CPU: {cpu}%")
        
        if mem > self.thresholds["max_memory_usage_pct"]:
            overall_status = self.DEGRADED
            issues.append(f"High Memory: {mem}%")
        
        # Check error rate
        if error_rate > self.thresholds["max_error_rate_pct"]:
            overall_status = self.UNHEALTHY
            issues.append(f"High error rate: {error_rate}%")
        
        # Check dependencies
        for dep_name, dep_result in dependency_results.items():
            if dep_result["status"] == "unhealthy":
                overall_status = self.UNHEALTHY
            elif dep_result["status"] == "degraded" and overall_status == self.HEALTHY:
                overall_status = self.DEGRADED
        
        issues.extend(dependency_issues)
        
        # Not ready overrides everything
        if not self.is_ready:
            overall_status = self.UNHEALTHY
            issues.append("Service not ready")
        
        health_report["overall_status"] = overall_status
        health_report["issues"] = issues if issues else ["None"]
        
        return health_report

class AlertState:
    OK = "OK"
    PENDING = "PENDING"
    ALERTING = "ALERTING"


# ================================================================
# SERVICE MONITOR — Tracks one service's health over time
# ================================================================

class ServiceMonitor:
    
    def __init__(self, service_name, health_url, config):
        self.service_name = service_name
        self.health_url = health_url
        
        # Configuration
        self.check_timeout_sec = config.get("check_timeout_sec", 3)
        self.unhealthy_threshold = config.get("unhealthy_threshold", 3)
        self.healthy_threshold = config.get("healthy_threshold", 1)
        
        # Current state
        self.current_status = "unknown"
        self.alert_state = AlertState.OK
        self.consecutive_failures = 0
        self.consecutive_successes = 0
        self.last_check_time = None
        self.last_healthy_time = None
        self.last_response = None
        
        # History (last 100 checks)
        self.history = deque(maxlen=100)
        
        # Uptime tracking
        self.total_checks = 0
        self.healthy_checks = 0
    
    def perform_check(self):
        """Execute one health check against the service"""
        check_result = {
            "timestamp": datetime.now().isoformat(),
            "service": self.service_name,
            "url": self.health_url
        }
        
        try:
            req = urllib.request.Request(self.health_url)
            req.add_header('User-Agent', 'HealthMonitor/1.0')
            
            start = time.time()
            with urllib.request.urlopen(req, timeout=self.check_timeout_sec) as resp:
                elapsed_ms = round((time.time() - start) * 1000, 1)
                body = json.loads(resp.read().decode('utf-8'))
                
                check_result["response_code"] = resp.status
                check_result["latency_ms"] = elapsed_ms
                check_result["response_body"] = body
                check_result["overall_status"] = body.get("overall_status", "healthy")
                
                # Determine if this check PASSED
                if resp.status == 200 and body.get("overall_status") != "unhealthy":
                    check_result["check_passed"] = True
                else:
                    check_result["check_passed"] = False
                    
        except urllib.error.HTTPError as e:
            elapsed_ms = round((time.time() - start) * 1000, 1) if 'start' in dir() else 0
            try:
                body = json.loads(e.read().decode('utf-8'))
            except Exception:
                body = {}
            
            check_result["response_code"] = e.code
            check_result["latency_ms"] = elapsed_ms
            check_result["response_body"] = body
            check_result["overall_status"] = body.get("overall_status", "unhealthy")
            check_result["check_passed"] = False
            
        except urllib.error.URLError as e:
            check_result["response_code"] = None
            check_result["latency_ms"] = None
            check_result["error"] = f"Connection failed: {e.reason}"
            check_result["overall_status"] = "dead"
            check_result["check_passed"] = False
            
        except Exception as e:
            check_result["response_code"] = None
            check_result["latency_ms"] = None
            check_result["error"] = f"Unexpected: {str(e)}"
            check_result["overall_status"] = "dead"
            check_result["check_passed"] = False
        
        # Update state based on result
        self._update_state(check_result)
        
        return check_result
    
    def _update_state(self, check_result):
        """Update internal state machine based on check result"""
        self.total_checks += 1
        self.last_check_time = check_result["timestamp"]
        self.last_response = check_result
        self.history.append(check_result)
        
        passed = check_result["check_passed"]
        
        if passed:
            self.consecutive_successes += 1
            self.consecutive_failures = 0
            self.healthy_checks += 1
            self.last_healthy_time = check_result["timestamp"]
            
            # State transition: anything → OK
            if self.consecutive_successes >= self.healthy_threshold:
                old_status = self.current_status
                self.current_status = check_result.get("overall_status", "healthy")
                
                if self.alert_state == AlertState.ALERTING:
                    self.alert_state = AlertState.OK
                    return self._create_alert("RECOVERY", old_status, self.current_status)
                
                self.alert_state = AlertState.OK
        
        else:
            self.consecutive_failures += 1
            self.consecutive_successes = 0
            
            if self.consecutive_failures == 1:
                # First failure → move to PENDING
                self.alert_state = AlertState.PENDING
            
            if self.consecutive_failures >= self.unhealthy_threshold:
                old_status = self.current_status
                self.current_status = check_result.get("overall_status", "unhealthy")
                
                if self.alert_state != AlertState.ALERTING:
                    # Transition to ALERTING (only alert ONCE)
                    self.alert_state = AlertState.ALERTING
                    return self._create_alert("FAILURE", old_status, self.current_status)
        
        return None  # No alert to send
    
    def _create_alert(self, alert_type, old_status, new_status):
        """Create a structured alert"""
        return {
            "alert_type": alert_type,
            "service": self.service_name,
            "old_status": old_status,
            "new_status": new_status,
            "consecutive_failures": self.consecutive_failures,
            "timestamp": datetime.now().isoformat(),
            "last_response": self.last_response,
            "message": self._format_alert_message(alert_type, new_status)
        }
    
    def _format_alert_message(self, alert_type, status):
        if alert_type == "FAILURE":
            return (
                f"🔴 ALERT: {self.service_name} is {status.upper()}\n"
                f"   Failed {self.consecutive_failures} consecutive checks\n"
                f"   Last healthy: {self.last_healthy_time or 'never'}\n"
                f"   URL: {self.health_url}"
            )
        elif alert_type == "RECOVERY":
            return (
                f"🟢 RECOVERED: {self.service_name} is {status.upper()}\n"
                f"   Service is back online\n"
                f"   URL: {self.health_url}"
            )
    
    def get_uptime_percentage(self):
        if self.total_checks == 0:
            return 0.0
        return round((self.healthy_checks / self.total_checks) * 100, 2)
    
    def get_summary(self):
        return {
            "service": self.service_name,
            "current_status": self.current_status,
            "alert_state": self.alert_state,
            "consecutive_failures": self.consecutive_failures,
            "consecutive_successes": self.consecutive_successes,
            "total_checks": self.total_checks,
            "uptime_pct": self.get_uptime_percentage(),
            "last_check": self.last_check_time,
            "last_healthy": self.last_healthy_time
        }

class CentralHealthMonitor:
    
    def __init__(self, check_interval_sec=10):
        self.check_interval = check_interval_sec
        self.monitors = {}
        self.alerts_log = deque(maxlen=500)
        self.is_running = False
        self.lock = threading.Lock()
    
    def register_service(self, service_name, health_url, config=None):
        """Register a service to be monitored"""
        default_config = {
            "check_timeout_sec": 3,
            "unhealthy_threshold": 3,
            "healthy_threshold": 1
        }
        if config:
            default_config.update(config)
        
        self.monitors[service_name] = ServiceMonitor(
            service_name=service_name,
            health_url=health_url,
            config=default_config
        )
        print(f"  [REGISTERED] {service_name} → {health_url}")
    
    def unregister_service(self, service_name):
        if service_name in self.monitors:
            del self.monitors[service_name]
            print(f"  [UNREGISTERED] {service_name}")
    
    def _check_all_services(self):
        """Run health checks on all registered services"""
        results = {}
        
        for name, monitor in self.monitors.items():
            alert = monitor.perform_check()
            results[name] = monitor.get_summary()
            
            if alert:
                self.alerts_log.append(alert)
                self._dispatch_alert(alert)
        
        return results
    
    def _dispatch_alert(self, alert):
        """
        Send alert to appropriate channels
        In production: email, Slack, PagerDuty, etc.
        Here: print to console
        """
        print(f"\n{'!' * 60}")
        print(f"  {alert['message']}")
        print(f"{'!' * 60}\n")
    
    def run_once(self):
        """Run a single check cycle (useful for testing)"""
        return self._check_all_services()
    
    def start(self):
        """Start the continuous monitoring loop"""
        self.is_running = True
        print(f"\n{'=' * 60}")
        print(f"  CENTRAL HEALTH MONITOR STARTED")
        print(f"  Check interval: {self.check_interval} seconds")
        print(f"  Monitoring {len(self.monitors)} services")
        print(f"{'=' * 60}\n")
        
        cycle = 0
        while self.is_running:
            cycle += 1
            timestamp = datetime.now().strftime("%H:%M:%S")
            print(f"── Check Cycle #{cycle} at {timestamp} ──")
            
            results = self._check_all_services()
            
            # Print status summary
            for name, summary in results.items():
                status = summary["current_status"]
                alert = summary["alert_state"]
                uptime = summary["uptime_pct"]
                
                # Status emoji
                emoji = {
                    "healthy": "🟢",
                    "degraded": "🟡",
                    "unhealthy": "🔴",
                    "dead": "⚫",
                    "unknown": "⚪"
                }.get(status, "❓")
                
                print(
                    f"  {emoji} {name:<25} "
                    f"Status: {status:<12} "
                    f"Alert: {alert:<10} "
                    f"Uptime: {uptime}%"
                )
            
            print()
            time.sleep(self.check_interval)

# PROJECT 15C: PUSH-BASED HEARTBEAT SYSTEM
# Services actively PUSH heartbeats to a central collector
# This is what distributed systems like ZooKeeper use
# Builds on: Queue patterns from Project 11

import time
import json
import socket
import threading
from datetime import datetime
from collections import defaultdict, deque


# ================================================================
# HEARTBEAT RECEIVER (Central Collector)
# ================================================================

class HeartbeatReceiver:
    """
    Receives heartbeat signals from services via UDP
    UDP chosen because:
      → Fire-and-forget (service doesn't wait for ACK)
      → Low overhead (no TCP handshake per heartbeat)
      → Missing one heartbeat is OK (we tolerate N misses)
      → Thousands of heartbeats per second with minimal resources
    """
    
    # Service lifecycle states
    ALIVE = "alive"
    SUSPECT = "suspect"
    DEAD = "dead"
    NEW = "new"
    
    def __init__(self, host='127.0.0.1', port=7777, config=None):
        self.host = host
        self.port = port
        
        default_config = {
            "heartbeat_interval_sec": 5,
            "suspect_after_missed": 2,    # 2 missed → suspect
            "dead_after_missed": 4,       # 4 missed → dead
            "cleanup_after_sec": 300,     # Remove from registry after 5 min dead
            "evaluation_interval_sec": 2  # How often to evaluate states
        }
        self.config = default_config
        if config:
            self.config.update(config)
        
        # Registry of known services
        # key: service_id → value: service state dict
        self.registry = {}
        self.registry_lock = threading.Lock()
        
        # Alert callbacks
        self.alert_callbacks = []
        
        # History
        self.event_log = deque(maxlen=1000)
        
        # UDP socket
        self.udp_socket = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
        self.udp_socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
        
        self.is_running = False
    
    def register_alert_callback(self, callback_func):
        """Register a function to be called when alerts fire"""
        self.alert_callbacks.append(callback_func)
    
    def start(self):
        """Start the heartbeat receiver and evaluator"""
        self.is_running = True
        
        # Bind UDP socket
        self.udp_socket.bind((self.host, self.port))
        print(f"[HEARTBEAT RECEIVER] Listening on UDP {self.host}:{self.port}")
        
        # Start receiver thread (listens for incoming heartbeats)
        receiver_thread = threading.Thread(
            target=self._receive_loop,
            daemon=True
        )
        receiver_thread.start()
        
        # Start evaluator thread (checks for missed heartbeats)
        evaluator_thread = threading.Thread(
            target=self._evaluation_loop,
            daemon=True
        )
        evaluator_thread.start()
        
        # Start display thread
        display_thread = threading.Thread(
            target=self._display_loop,
            daemon=True
        )
        display_thread.start()
        
        return receiver_thread, evaluator_thread, display_thread
    
    def stop(self):
        self.is_running = False
        self.udp_socket.close()
    
    def _receive_loop(self):
        """Continuously receive heartbeat UDP packets"""
        while self.is_running:
            try:
                # UDP receive — blocks until packet arrives
                # Max packet size 4096 bytes (plenty for heartbeat)
                data, addr = self.udp_socket.recvfrom(4096)
                heartbeat = json.loads(data.decode('utf-8'))
                self._process_heartbeat(heartbeat, addr)
                
            except socket.error:
                if self.is_running:
                    time.sleep(0.1)
            except json.JSONDecodeError:
                print(f"[WARN] Received malformed heartbeat from {addr}")
            except Exception as e:
                if self.is_running:
                    print(f"[ERROR] Receiver: {e}")
    
    def _process_heartbeat(self, heartbeat, addr):
        """Process a single heartbeat signal"""
        service_id = heartbeat.get("service_id")
        if not service_id:
            return
        
        now = time.time()
        
        with self.registry_lock:
            if service_id not in self.registry:
                # New service just appeared
                self.registry[service_id] = {
                    "service_id": service_id,
                    "service_type": heartbeat.get("service_type", "unknown"),
                    "host": heartbeat.get("host", addr[0]),
                    "port": heartbeat.get("port", "unknown"),
                    "first_seen": now,
                    "last_heartbeat": now,
                    "heartbeat_count": 0,
                    "state": self.NEW,
                    "previous_state": None,
                    "metrics": {},
                    "missed_count": 0
                }
                self._log_event("SERVICE_REGISTERED", service_id,
                              f"New service detected from {addr}")
                self._fire_alert("NEW_SERVICE", service_id,
                               f"🆕 New service registered: {service_id}")
            
            entry = self.registry[service_id]
            entry["last_heartbeat"] = now
            entry["heartbeat_count"] += 1
            entry["metrics"] = heartbeat.get("metrics", {})
            entry["missed_count"] = 0
            
            # State transition: suspect/dead → alive
            if entry["state"] in (self.SUSPECT, self.DEAD):
                old_state = entry["state"]
                entry["previous_state"] = old_state
                entry["state"] = self.ALIVE
                self._log_event("SERVICE_RECOVERED", service_id,
                              f"Recovered from {old_state}")
                self._fire_alert("RECOVERY", service_id,
                               f"🟢 RECOVERED: {service_id} is back alive "
                               f"(was {old_state})")
            elif entry["state"] == self.NEW:
                entry["state"] = self.ALIVE
    
    def _evaluation_loop(self):
        """Periodically evaluate all services for missed heartbeats"""
        interval = self.config["heartbeat_interval_sec"]
        suspect_threshold = self.config["suspect_after_missed"]
        dead_threshold = self.config["dead_after_missed"]
        cleanup_threshold = self.config["cleanup_after_sec"]
        eval_interval = self.config["evaluation_interval_sec"]
        
        while self.is_running:
            now = time.time()
            services_to_remove = []
            
            with self.registry_lock:
                for service_id, entry in self.registry.items():
                    time_since_last = now - entry["last_heartbeat"]
                    missed = int(time_since_last / interval)
                    entry["missed_count"] = missed
                    
                    old_state = entry["state"]
                    
                    # Determine new state based on missed heartbeats
                    if missed < suspect_threshold:
                        new_state = self.ALIVE
                    elif missed < dead_threshold:
                        new_state = self.SUSPECT
                    else:
                        new_state = self.DEAD
                    
                    # Only act on state TRANSITIONS
                    if new_state != old_state:
                        entry["previous_state"] = old_state
                        entry["state"] = new_state
                        
                        if new_state == self.SUSPECT:
                            self._log_event(
                                "SERVICE_SUSPECT", service_id,
                                f"Missed {missed} heartbeats"
                            )
                            self._fire_alert(
                                "WARNING", service_id,
                                f"🟡 SUSPECT: {service_id} missed "
                                f"{missed} heartbeats "
                                f"(last seen {time_since_last:.1f}s ago)"
                            )
                        
                        elif new_state == self.DEAD:
                            self._log_event(
                                "SERVICE_DEAD", service_id,
                                f"Missed {missed} heartbeats — declared dead"
                            )
                            self._fire_alert(
                                "CRITICAL", service_id,
                                f"🔴 DEAD: {service_id} has not sent a "
                                f"heartbeat for {time_since_last:.1f}s"
                            )
                    
                    # Cleanup very old dead services
                    if (entry["state"] == self.DEAD and 
                            time_since_last > cleanup_threshold):
                        services_to_remove.append(service_id)
                
                for sid in services_to_remove:
                    del self.registry[sid]
                    self._log_event("SERVICE_REMOVED", sid,
                                  "Removed after extended death")
            
            time.sleep(eval_interval)
    
    def _display_loop(self):
        """Periodically display registry status"""
        while self.is_running:
            time.sleep(10)
            self._print_registry()
    
    def _print_registry(self):
        """Print current registry status"""
        now = time.time()
        
        print(f"\n{'─' * 75}")
        print(f"  HEARTBEAT REGISTRY — {datetime.now().strftime('%H:%M:%S')}")
        print(f"{'─' * 75}")
        print(f"  {'Service ID':<25} {'State':<10} {'Missed':<8} "
              f"{'Last Seen':<12} {'Heartbeats':<12}")
        print(f"  {'─' * 25} {'─' * 10} {'─' * 8} {'─' * 12} {'─' * 12}")
        
        with self.registry_lock:
            if not self.registry:
                print(f"  (no services registered)")
            
            for sid, entry in sorted(self.registry.items()):
                state = entry["state"]
                emoji = {
                    self.ALIVE: "🟢",
                    self.SUSPECT: "🟡",
                    self.DEAD: "🔴",
                    self.NEW: "🔵"
                }.get(state, "⚪")
                
                time_since = now - entry["last_heartbeat"]
                
                print(
                    f"  {emoji} {sid:<23} {state:<10} "
                    f"{entry['missed_count']:<8} "
                    f"{time_since:>8.1f}s ago "
                    f"{entry['heartbeat_count']:<12}"
                )
                
                # Show metrics if available
                metrics = entry.get("metrics", {})
                if metrics:
                    cpu = metrics.get("cpu_pct", "?")
                    mem = metrics.get("mem_pct", "?")
                    reqs = metrics.get("active_requests", "?")
                    print(
                        f"    └─ CPU: {cpu}%  "
                        f"MEM: {mem}%  "
                        f"Active Requests: {reqs}"
                    )
        
        print(f"{'─' * 75}\n")
    
    def _log_event(self, event_type, service_id, message):
        """Log an event to the event history"""
        self.event_log.append({
            "timestamp": datetime.now().isoformat(),
            "event_type": event_type,
            "service_id": service_id,
            "message": message
        })
    
    def _fire_alert(self, severity, service_id, message):
        """Dispatch alert to all registered callbacks"""
        alert = {
            "severity": severity,
            "service_id": service_id,
            "message": message,
            "timestamp": datetime.now().isoformat()
        }
        
        # Print to console
        print(f"\n  {'!' * 50}")
        print(f"  [{severity}] {message}")
        print(f"  {'!' * 50}\n")
        
        # Call registered callbacks
        for callback in self.alert_callbacks:
            try:
                callback(alert)
            except Exception as e:
                print(f"  [ALERT CALLBACK ERROR] {e}")
    
    def get_alive_services(self):
        """Get list of services currently alive (for service discovery)"""
        with self.registry_lock:
            return {
                sid: {
                    "host": entry["host"],
                    "port": entry["port"],
                    "service_type": entry["service_type"],
                    "state": entry["state"]
                }
                for sid, entry in self.registry.items()
                if entry["state"] == self.ALIVE
            }
    
    def get_all_services(self):
        """Get full registry snapshot"""
        with self.registry_lock:
            return {
                sid: dict(entry) 
                for sid, entry in self.registry.items()
            }

class HeartbeatSender:
    """
    Each service creates one of these to send heartbeats
    to the central receiver. Runs in a background thread.
    """
    
    def __init__(self, service_id, service_type, service_port,
                 receiver_host='127.0.0.1', receiver_port=7777,
                 heartbeat_interval=5):
        self.service_id = service_id
        self.service_type = service_type
        self.service_port = service_port
        self.receiver_host = receiver_host
        self.receiver_port = receiver_port
        self.heartbeat_interval = heartbeat_interval
        
        # UDP socket for sending
        self.udp_socket = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
        
        # Metrics collector function (can be overridden)
        self.metrics_collector = None
        
        self.is_running = False
        self.heartbeats_sent = 0
    
    def set_metrics_collector(self, collector_func):
        """
        Set a function that returns current metrics dict
        Example: lambda: {"cpu_pct": 45, "mem_pct": 60}
        """
        self.metrics_collector = collector_func
    
    def start(self):
        """Start sending heartbeats in background thread"""
        self.is_running = True
        thread = threading.Thread(target=self._send_loop, daemon=True)
        thread.start()
        print(f"  [HEARTBEAT] {self.service_id} sending to "
              f"{self.receiver_host}:{self.receiver_port} "
              f"every {self.heartbeat_interval}s")
        return thread
    
    def stop(self):
        """Stop sending heartbeats"""
        self.is_running = False
        self.udp_socket.close()
    
    def _send_loop(self):
        """Continuously send heartbeats"""
        while self.is_running:
            try:
                self._send_one_heartbeat()
                self.heartbeats_sent += 1
            except Exception as e:
                print(f"  [HEARTBEAT ERROR] {self.service_id}: {e}")
            
            time.sleep(self.heartbeat_interval)
    
    def _send_one_heartbeat(self):
        """Construct and send a single heartbeat packet"""
        # Collect current metrics
        metrics = {}
        if self.metrics_collector:
            try:
                metrics = self.metrics_collector()
            except Exception:
                metrics = {"error": "metrics collection failed"}
        
        heartbeat = {
            "service_id": self.service_id,
            "service_type": self.service_type,
            "host": socket.gethostname(),
            "port": self.service_port,
            "timestamp": time.time(),
            "metrics": metrics
        }
        
        payload = json.dumps(heartbeat).encode('utf-8')
        self.udp_socket.sendto(
            payload,
            (self.receiver_host, self.receiver_port)
        )
    
    def send_final_heartbeat(self, reason="shutdown"):
        """Send a graceful shutdown notification"""
        heartbeat = {
            "service_id": self.service_id,
            "service_type": self.service_type,
            "host": socket.gethostname(),
            "port": self.service_port,
            "timestamp": time.time(),
            "shutting_down": True,
            "reason": reason
        }
        payload = json.dumps(heartbeat).encode('utf-8')
        try:
            self.udp_socket.sendto(
                payload,
                (self.receiver_host, self.receiver_port)
            )
        except Exception:
            pass

    Aggregates health data from multiple sources:
    - Pull-based HTTP health checks
    - Push-based heartbeat signals
    - Manual status overrides (maintenance mode)
    
    Provides a single API for dashboard consumption
    """
    
    def __init__(self):
        self.services = {}
        self.incidents = deque(maxlen=200)
        self.maintenance_windows = {}
        self.lock = threading.Lock()
        
        # SLA tracking
        self.sla_targets = {}
        self.uptime_records = {}
    
    def register_service(self, service_id, service_type,
                         health_url=None, sla_target=99.9):
        """Register a service with optional SLA target"""
        with self.lock:
            self.services[service_id] = {
                "service_id": service_id,
                "service_type": service_type,
                "health_url": health_url,
                "current_status": "unknown",
                "last_check": None,
                "status_history": deque(maxlen=1440),  # 24 hours at 1/min
                "total_checks": 0,
                "healthy_checks": 0,
                "sla_target": sla_target,
                "metadata": {},
                "in_maintenance": False
            }
    
    def update_service_status(self, service_id, status,
                              details=None, source="pull"):
        """Update a service's status from any source"""
        with self.lock:
            if service_id not in self.services:
                self.register_service(service_id, "auto-detected")
            
            svc = self.services[service_id]
            old_status = svc["current_status"]
            now = datetime.now().isoformat()
            
            svc["current_status"] = status
            svc["last_check"] = now
            svc["total_checks"] += 1
            svc["metadata"] = details or {}
            
            if status in ("healthy", "alive"):
                svc["healthy_checks"] += 1
            
            # Record in history
            svc["status_history"].append({
                "timestamp": now,
                "status": status,
                "source": source
            })
            
            # Detect state transitions → create incidents
            if old_status != status and old_status != "unknown":
                if status in ("unhealthy", "dead"):
                    incident = {
                        "id": f"INC-{len(self.incidents)+1:04d}",
                        "service_id": service_id,
                        "type": "outage",
                        "started_at": now,
                        "resolved_at": None,
                        "old_status": old_status,
                        "new_status": status,
                        "details": details
                    }
                    self.incidents.append(incident)
                
                elif old_status in ("unhealthy", "dead") and status in ("healthy", "alive"):
                    # Resolve the most recent incident for this service
                    for inc in reversed(self.incidents):
                        if (inc["service_id"] == service_id and 
                                inc["resolved_at"] is None):
                            inc["resolved_at"] = now
                            break
    
    def set_maintenance(self, service_id, is_maintenance, reason=""):
        """Put a service in/out of maintenance mode"""
        with self.lock:
            if service_id in self.services:
                self.services[service_id]["in_maintenance"] = is_maintenance
                self.maintenance_windows[service_id] = {
                    "active": is_maintenance,
                    "reason": reason,
                    "started_at": datetime.now().isoformat() if is_maintenance else None
                }
    
    def get_system_overview(self):
        """Get high-level system health overview"""
        with self.lock:
            total = len(self.services)
            healthy = sum(
                1 for s in self.services.values()
                if s["current_status"] in ("healthy", "alive")
            )
            degraded = sum(
                1 for s in self.services.values()
                if s["current_status"] == "degraded"
            )
            unhealthy = sum(
                1 for s in self.services.values()
                if s["current_status"] in ("unhealthy", "dead")
            )
            maintenance = sum(
                1 for s in self.services.values()
                if s["in_maintenance"]
            )
            unknown = total - healthy - degraded - unhealthy
            
            # Overall system status
            if unhealthy > 0:
                system_status = "unhealthy"
            elif degraded > 0:
                system_status = "degraded"
            elif healthy == total:
                system_status = "healthy"
            else:
                system_status = "unknown"
            
            # Active incidents
            active_incidents = [
                dict(inc) for inc in self.incidents
                if inc["resolved_at"] is None
            ]
            
            return {
                "system_status": system_status,
                "timestamp": datetime.now().isoformat(),
                "summary": {
                    "total_services": total,
                    "healthy": healthy,
                    "degraded": degraded,
                    "unhealthy": unhealthy,
                    "unknown": unknown,
                    "in_maintenance": maintenance
                },
                "active_incidents": active_incidents,
                "active_incident_count": len(active_incidents)
            }
    
    def get_service_detail(self, service_id):
        """Get detailed status for one service"""
        with self.lock:
            if service_id not in self.services:
                return None
            
            svc = self.services[service_id]
            
            # Calculate uptime
            uptime_pct = 0.0
            if svc["total_checks"] > 0:
                uptime_pct = round(
                    (svc["healthy_checks"] / svc["total_checks"]) * 100, 3
                )
            
            # SLA compliance
            sla_target = svc["sla_target"]
            sla_met = uptime_pct >= sla_target
            
            # Recent history (last 20 checks)
            recent = list(svc["status_history"])[-20:]
            
            # Related incidents
            related_incidents = [
                dict(inc) for inc in self.incidents
                if inc["service_id"] == service_id
            ][-10:]  # Last 10
            
            return {
                "service_id": service_id,
                "service_type": svc["service_type"],
                "current_status": svc["current_status"],
                "in_maintenance": svc["in_maintenance"],
                "last_check": svc["last_check"],
                "uptime": {
                    "percentage": uptime_pct,
                    "sla_target": sla_target,
                    "sla_met": sla_met,
                    "total_checks": svc["total_checks"],
                    "healthy_checks": svc["healthy_checks"]
                },
                "recent_history": recent,
                "recent_incidents": related_incidents,
                "metadata": svc["metadata"]
            }
    
    def get_all_services_status(self):
        """Get status of all services"""
        with self.lock:
            result = []
            for sid, svc in sorted(self.services.items()):
                uptime = 0.0
                if svc["total_checks"] > 0:
                    uptime = round(
                        (svc["healthy_checks"] / svc["total_checks"]) * 100, 2
                    )
                
                result.append({
                    "service_id": sid,
                    "service_type": svc["service_type"],
                    "status": svc["current_status"],
                    "in_maintenance": svc["in_maintenance"],
                    "uptime_pct": uptime,
                    "last_check": svc["last_check"]
                })
            
            return result

# Connects Project 6 (Load Balancer) + Project 15 (Health Checks)
# The load balancer ONLY routes to healthy backends

import time
import json
import threading
import urllib.request
import urllib.error
import random
from datetime import datetime
from collections import deque


class HealthAwareLoadBalancer:
    """
    Load balancer that uses health checks to decide routing.
    
    From Project 6: We learned Round Robin and Least Connections
    NOW: We add health awareness on top of those algorithms
    
    Flow:
      1. Periodically health-check all backends
      2. Maintain a list of HEALTHY backends only
      3. Route traffic only to healthy backends
      4. When backend recovers → gradually add back
    """
    
    def __init__(self, health_check_interval=10, health_check_timeout=3,
                 unhealthy_threshold=3, healthy_threshold=2):
        # Backend registry
        self.backends = {}
        self.lock = threading.Lock()
        
        # Health check config
        self.hc_interval = health_check_interval
        self.hc_timeout = health_check_timeout
        self.unhealthy_threshold = unhealthy_threshold
        self.healthy_threshold = healthy_threshold
        
        # Routing state
        self.round_robin_index = 0
        
        # Metrics
        self.total_routed = 0
        self.total_rejected = 0  # Rejected because no healthy backend
        self.routing_log = deque(maxlen=200)
    
    def add_backend(self, backend_id, host, port, health_path="/health",
                    weight=1):
        """Register a backend server"""
        with self.lock:
            self.backends[backend_id] = {
                "backend_id": backend_id,
                "host": host,
                "port": port,
                "health_url": f"http://{host}:{port}{health_path}",
                "weight": weight,
                "is_healthy": False,     # Start as unhealthy until first check
                "is_draining": False,    # Graceful removal
                "consecutive_failures": 0,
                "consecutive_successes": 0,
                "last_check_time": None,
                "last_check_result": None,
                "total_requests_routed": 0,
                "active_connections": 0
            }
        print(f"  [BACKEND ADDED] {backend_id} → {host}:{port}")
    
    def remove_backend(self, backend_id, graceful=True):
        """Remove a backend (optionally drain first)"""
        with self.lock:
            if backend_id in self.backends:
                if graceful:
                    self.backends[backend_id]["is_draining"] = True
                    self.backends[backend_id]["is_healthy"] = False
                    print(f"  [DRAINING] {backend_id}")
                else:
                    del self.backends[backend_id]
                    print(f"  [REMOVED] {backend_id}")
    
    def _perform_health_check(self, backend):
        """Check one backend's health"""
        try:
            req = urllib.request.Request(backend["health_url"])
            req.add_header('User-Agent', 'HealthAwareLB/1.0')
            
            start = time.time()
            with urllib.request.urlopen(req, timeout=self.hc_timeout) as resp:
                latency = round((time.time() - start) * 1000, 1)
                body = json.loads(resp.read().decode('utf-8'))
                
                overall = body.get("overall_status", "healthy")
                
                return {
                    "reachable": True,
                    "status_code": resp.status,
                    "overall_status": overall,
                    "latency_ms": latency,
                    "passed": resp.status == 200 and overall != "unhealthy"
                }
                
        except Exception as e:
            return {
                "reachable": False,
                "error": str(e),
                "passed": False
            }
    
    def _health_check_loop(self):
        """Continuously check all backends"""
        while True:
            with self.lock:
                backends_to_check = list(self.backends.values())
            
            for backend in backends_to_check:
                if backend["is_draining"]:
                    continue
                
                result = self._perform_health_check(backend)
                
                with self.lock:
                    bid = backend["backend_id"]
                    if bid not in self.backends:
                        continue
                    
                    b = self.backends[bid]
                    b["last_check_time"] = datetime.now().isoformat()
                    b["last_check_result"] = result
                    
                    if result["passed"]:
                        b["consecutive_failures"] = 0
                        b["consecutive_successes"] += 1
                        
                        if (not b["is_healthy"] and 
                                b["consecutive_successes"] >= self.healthy_threshold):
                            b["is_healthy"] = True
                            print(f"  [HEALTHY] ✅ {bid} is now in rotation")
                    
                    else:
                        b["consecutive_successes"] = 0
                        b["consecutive_failures"] += 1
                        
                        if (b["is_healthy"] and 
                                b["consecutive_failures"] >= self.unhealthy_threshold):
                            b["is_healthy"] = False
                            print(f"  [REMOVED] ❌ {bid} removed from rotation "
                                  f"(failed {b['consecutive_failures']} checks)")
            
            time.sleep(self.hc_interval)
    
    def start_health_checks(self):
        """Start background health checking"""
        thread = threading.Thread(target=self._health_check_loop, daemon=True)
        thread.start()
        print(f"  [HEALTH CHECKS] Started (interval: {self.hc_interval}s)")
        return thread
    
    def get_healthy_backends(self):
        """Return list of healthy, non-draining backends"""
        with self.lock:
            return [
                b for b in self.backends.values()
                if b["is_healthy"] and not b["is_draining"]
            ]
    
    def route_request(self, request_info=None):
        """
        Select a healthy backend to route a request to.
        Uses weighted round-robin among healthy backends.
        
        Returns: selected backend dict or None if no healthy backend
        """
        healthy = self.get_healthy_backends()
        
        if not healthy:
            self.total_rejected += 1
            self.routing_log.append({
                "timestamp": datetime.now().isoformat(),
                "action": "rejected",
                "reason": "no healthy backends",
                "request": request_info
            })
            return None
        
        # Weighted round-robin
        # Build weighted list
        weighted_list = []
        for b in healthy:
            weighted_list.extend([b] * b["weight"])
        
        with self.lock:
            selected = weighted_list[self.round_robin_index % len(weighted_list)]
            self.round_robin_index += 1
            selected["total_requests_routed"] += 1
            selected["active_connections"] += 1
            self.total_routed += 1
        
        self.routing_log.append({
            "timestamp": datetime.now().isoformat(),
            "action": "routed",
            "backend": selected["backend_id"],
            "request": request_info
        })
        
        return {
            "backend_id": selected["backend_id"],
            "host": selected["host"],
            "port": selected["port"],
            "url": f"http://{selected['host']}:{selected['port']}"
        }
    
    def release_connection(self, backend_id):
        """Called when a request completes"""
        with self.lock:
            if backend_id in self.backends:
                self.backends[backend_id]["active_connections"] = max(
                    0, self.backends[backend_id]["active_connections"] - 1
                )
    
    def get_status(self):
        """Get full LB status for monitoring"""
        with self.lock:
            backends_status = []
            for bid, b in sorted(self.backends.items()):
                emoji = "🟢" if b["is_healthy"] else "🔴"
                if b["is_draining"]:
                    emoji = "🟠"
                
                backends_status.append({
                    "id": bid,
                    "emoji": emoji,
                    "healthy": b["is_healthy"],
                    "draining": b["is_draining"],
                    "consecutive_failures": b["consecutive_failures"],
                    "consecutive_successes": b["consecutive_successes"],
                    "requests_routed": b["total_requests_routed"],
                    "active_connections": b["active_connections"],
                    "last_check": b["last_check_time"]
                })
            
            return {
                "total_backends": len(self.backends),
                "healthy_backends": sum(
                    1 for b in self.backends.values() if b["is_healthy"]
                ),
                "total_routed": self.total_routed,
                "total_rejected": self.total_rejected,
                "backends": backends_status
            }
