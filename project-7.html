THE PHONE BOOK ANALOGY:
  → You want to call "Mario's Pizza"
  → You don't memorize their phone number
  → You look it up in the PHONE BOOK
  → Phone book: "Mario's Pizza" → 555-0123
  → You dial 555-0123
  → Mario's changes location → new number 555-0456
  → Phone book updated → you look up again → get new number
  → You never hardcode the number

DNS IS THE INTERNET'S PHONE BOOK:
  → You type: api.restaurant.com
  → DNS looks up: api.restaurant.com → 203.0.113.10
  → Browser/app connects to 203.0.113.10
  → IP changes? → Update DNS record → everyone gets new IP
  → No app updates needed
  → No downtime
  → Automatic

NOW WE ADD THE NAMING LAYER (before the connection even happens):

Step 1: UNDERSTAND the DNS hierarchy
         → Root servers → TLD servers → Authoritative servers
         → Like: Library → Section → Shelf → Book

Step 2: UNDERSTAND record types
         → A record: name → IPv4 address
         → AAAA record: name → IPv6 address
         → CNAME record: name → another name (alias)
         → MX record: domain → mail server
         → TXT record: domain → text data (verification, SPF)
         → NS record: domain → nameserver responsible

Step 3: UNDERSTAND caching and TTL
         → DNS responses are CACHED (like Project 5!)
         → TTL = how long to cache before re-querying
         → Short TTL (60s) = more DNS queries, faster changes
         → Long TTL (86400s) = fewer queries, slower changes

Step 4: UNDERSTAND the resolution flow
         → Browser → Local cache → OS cache → Router cache →
           ISP Resolver → Root → TLD → Authoritative → Response

Step 5: APPLY DNS for system design
         → Subdomain routing (api. vs www. vs cdn.)
         → Geographic routing (US users → US IP, EU → EU IP)
         → Failover (primary down → switch to backup IP)
         → Load balancing (return different IPs round-robin)

THE DNS HIERARCHY — HOW THE PHONE BOOK IS ORGANIZED:

  The internet's naming system is a TREE:

  ┌─────────────────────────────────────────────────────────────┐
  │                        . (ROOT)                             │
  │                      /    |    \                            │
  │                   .com   .org   .io   .net   .uk           │
  │                  /    \                                     │
  │          google.com  restaurant.com                         │
  │                      /      |      \                       │
  │               api.restaurant.com                            │
  │               www.restaurant.com                            │
  │               cdn.restaurant.com                            │
  │               mail.restaurant.com                           │
  └─────────────────────────────────────────────────────────────┘

  Each level has DIFFERENT servers responsible:

  ROOT SERVERS (13 clusters worldwide, labeled a-m):
    → Know where to find .com, .org, .io, etc.
    → Don't know specific domains
    → "restaurant.com? Ask the .com servers"

  TLD SERVERS (Top Level Domain):
    → .com server knows all .com domains
    → "restaurant.com? Ask nameserver ns1.restaurant.com"
    → Operated by registries (Verisign for .com)

  AUTHORITATIVE SERVERS (your DNS servers):
    → Know YOUR specific records
    → "api.restaurant.com → 203.0.113.10"
    → You control these (or your DNS provider does)
    → Cloudflare, Route53, Google Cloud DNS

  ┌─────────────────────────────────────────────────────┐

  ┌─────────────────────────────────────────────────────┐
  │ Primary:  api.restaurant.com → 203.0.113.10        │

✅ WHAT SHOULD DO

1. ALWAYS use a domain name, never hardcode IPs
   → App calls api.restaurant.com, not 203.0.113.10
   → IP changes → update DNS → zero app changes
   → This is THE reason DNS exists

2. ALWAYS set appropriate TTLs
   → Long TTL (3600s = 1 hour): stable records, less DNS traffic
   → Short TTL (60s): records that might change soon
   → Before migration: lower TTL days in advance
     → Was 3600s → change to 60s → wait 24 hours
     → Now change IP → propagates in 60 seconds
     → After migration → raise TTL back to 3600s

3. ALWAYS have at least 2 nameservers
   → NS1 goes down → NS2 still answers queries
   → Most providers give you 2-4 nameservers automatically
   → Geographically distributed for resilience

4. ALWAYS use subdomains for different services
   ✅ api.restaurant.com → API servers
   ✅ www.restaurant.com → Web frontend
   ✅ admin.restaurant.com → Admin panel
   ❌ restaurant.com/api → can't route at DNS level

5. ALWAYS monitor DNS resolution
   → Check that your domain resolves correctly
   → Monitor from multiple locations worldwide
   → DNS failure = invisible to your application logs
   → Your servers are fine but nobody can reach them

6. ALWAYS configure DNS health checks for critical services
   → DNS provider checks your endpoints
   → Automatically removes unhealthy IPs from responses
   → Second layer of failover (beyond LB health checks — Project 6)

7. ALWAYS secure your DNS
   → DNSSEC: cryptographic signatures prevent spoofing
   → Two-factor auth on DNS provider account
   → DNS hijacking = attacker redirects your traffic
   → One of the most devastating attacks possible

❌ WHAT SHOULD NOT DO

1. NEVER hardcode IP addresses in application code
   ❌ API_URL = "http://203.0.113.10:8080"
   ✅ API_URL = "https://api.restaurant.com"
   → IP changes = code change + redeploy + app store review

2. NEVER set TTL to 0
   ❌ TTL=0 means "never cache"
   → Every single request requires full DNS resolution
   → Adds 20-200ms latency to EVERY request
   → Massive load on DNS servers
   → Minimum practical TTL: 30-60 seconds

3. NEVER forget to lower TTL before migrations
   ❌ TTL=86400 (24 hours) → change IP → users cached old IP for 24h
   → Some users can't reach you for an entire DAY
   → Lower TTL days BEFORE the change

4. NEVER use CNAME at zone apex (root domain)
   ❌ restaurant.com → CNAME → something.cloudfront.net
   → CNAME at root violates DNS RFC
   → Breaks MX records and other records
   → Use ALIAS/ANAME (provider-specific) or A record instead
   → api.restaurant.com → CNAME is fine (subdomain)

5. NEVER ignore DNS propagation time
   → Changed DNS record → not instant worldwide
   → Old records cached everywhere with old TTL
   → Can take minutes to hours for full propagation
   → Plan for this during migrations

6. NEVER have a single point of failure in DNS
   ❌ One nameserver → it goes down → entire domain unreachable
   → Always at least 2 nameservers on different networks
   → DNS failure = complete outage for your entire system

7. NEVER expose internal service names via public DNS
   ❌ db-primary.restaurant.com → 10.0.1.5 (internal IP)
   → Reveals internal architecture to attackers
   → Use private/internal DNS for internal services

🚫 WHEN NOT TO USE

1. DNS load balancing is NOT suitable for:
   → Fine-grained load distribution (use LB from Project 6)
   → Real-time health awareness (DNS TTL caching delays failover)
   → Session affinity (DNS caching makes sticky sessions unreliable)
   → Weighted distribution (limited support, client ignores weights)
   
   DNS load balancing = COARSE (region-level, LB-level)
   Application LB = FINE (request-level, server-level)

2. Public DNS is NOT suitable for:
   → Internal service-to-service communication
     → Use private DNS or service discovery (Project 20)
   → Frequently changing records (sub-minute changes)
     → Caching makes rapid changes unreliable
   → Security-sensitive internal routing
     → Don't expose internal IPs to the world

3. Low TTL is NOT suitable when:
   → You want to minimize DNS resolution latency
   → You don't plan to change records
   → Your DNS provider charges per query (cost adds up)

4. GeoDNS is NOT suitable when:
   → You have servers in only one region
   → All users are in one geographic area
   → Accuracy isn't critical (GeoDNS uses IP geolocation — not perfect)

═══════════════════════════════════════════════════════

  Record: cdn.restaurant.com → CNAME → d1234.cloudfront.net

  1. Query: cdn.restaurant.com

  4. Query: d1234.cloudfront.net
  5. Answer: A → 54.230.105.42
  6. Return 54.230.105.42 to client

  CNAME adds one extra lookup step
  → Slight latency increase
  → But enables powerful indirection (CDN, load balancer services)

  Resolver at 8.8.8.8 (Google US) queries:
  → api.restaurant.com → 203.0.113.10 (US server)

  Resolver at 1.0.0.1 (Cloudflare Asia) queries:
  → api.restaurant.com → 192.0.2.20 (Asia server)

  Same domain, different answers based on WHO is asking

  Limitation: Uses RESOLVER location, not CLIENT location
  → User in Japan using US VPN → gets US IP
  → Corporate users with central DNS → all get same IP

PART 4: HOW DNS FAILOVER WORKS
═══════════════════════════════════════════════════════

  Configuration (e.g., Route53):
    Primary:   api.restaurant.com → 203.0.113.10 (health-checked)
    Secondary: api.restaurant.com → 198.51.100.5 (health-checked)

  Normal state:
    Health check: GET https://203.0.113.10/health → 200 OK ✅
    DNS response: 203.0.113.10

  Primary fails:
    Health check: GET https://203.0.113.10/health → timeout ❌
    (3 consecutive failures)
    DNS response: 198.51.100.5 (automatic failover)

  Primary recovers:
    Health check: GET https://203.0.113.10/health → 200 OK ✅
    (2 consecutive successes)
    DNS response: 203.0.113.10 (automatic failback)

  Failover speed depends on:
    → Health check interval (10-30 seconds)
    → Failure threshold (2-3 checks)
    → DNS TTL (clients cache old IP for TTL duration)
    → Total failover time: 30s-5min typically

PART 5: DNS PACKET STRUCTURE
═══════════════════════════════════════════════════════

  DNS uses UDP port 53 (TCP for large responses)

  Query packet:
  ┌──────────────────────────────────────────┐
  │ Header:                                   │
  │   ID: 0x1234 (match request to response) │
  │   Flags: QR=0 (query), RD=1 (recursive)  │
  │   Questions: 1                            │
  │                                           │
  │ Question:                                 │
  │   Name: api.restaurant.com                │
  │   Type: A (IPv4 address)                  │
  │   Class: IN (Internet)                    │
  └──────────────────────────────────────────┘

  Response packet:
  ┌──────────────────────────────────────────┐
  │ Header:                                   │
  │   ID: 0x1234 (matches query)              │
  │   Flags: QR=1 (response), AA=1 (auth)    │
  │   Answers: 1                              │
  │                                           │
  │ Answer:                                   │
  │   Name: api.restaurant.com                │
  │   Type: A                                 │
  │   Class: IN                               │
  │   TTL: 300                                │
  │   Data: 203.0.113.10                      │
  └──────────────────────────────────────────┘

  WHY UDP (not TCP):
  → DNS queries are small (< 512 bytes typically)
  → UDP = no handshake overhead = faster
  → One packet out, one packet back
  → TCP used only for: zone transfers, large responses (> 512 bytes)

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: DNS Propagation Delay
  → Changed DNS record from IP-A to IP-B
  → Some users still reach IP-A for hours
  → Old TTL still cached in resolvers worldwide
  → FIX: Lower TTL BEFORE the change, not after

PITFALL 2: DNS Provider Outage
  → 2016: Dyn DNS attacked (DDoS)
  → Twitter, GitHub, Netflix, Reddit all went down
  → Their servers were fine — DNS was unreachable
  → FIX: Use multiple DNS providers (multi-DNS)
  → FIX: Keep TTLs reasonable (clients use cached records during outage)

PITFALL 3: CNAME at Zone Apex
  → restaurant.com → CNAME → something.elb.amazonaws.com
  → VIOLATES DNS standard
  → Breaks MX records, NS records at same level
  → FIX: Use A record or provider-specific ALIAS record

PITFALL 4: Forgetting to Renew Domain
  → Domain registration expires
  → DNS stops working
  → ENTIRE system unreachable
  → Someone else can register your domain
  → FIX: Enable auto-renewal, monitor expiration dates

PITFALL 5: DNS Cache Poisoning
  → Attacker injects false DNS response
  → Users directed to malicious server
  → Can steal credentials, serve malware
  → FIX: Enable DNSSEC (cryptographic verification)
  → FIX: Use DoH (DNS over HTTPS) or DoT (DNS over TLS)

PITFALL 6: Negative Caching
  → Query for nonexistent subdomain → NXDOMAIN response
  → This negative result is ALSO cached (SOA minimum TTL)
  → You create the record → but clients still see "not found"
  → FIX: Be aware of negative TTL settings

PITFALL 7: Split-Horizon DNS Confusion
  → Internal DNS: api.restaurant.com → 10.0.1.5 (private)
  → External DNS: api.restaurant.com → 203.0.113.10 (public)
  → Developer on VPN gets internal IP → works
  → Developer off VPN gets external IP → works
  → Mixed environment → confusion and debugging nightmares
  → FIX: Document clearly, use different subdomains if possible

            result["priority"] = self.priority
        if self.record_type == RecordType.SRV:
            result["priority"] = self.priority
            result["weight"] = self.weight
            result["port"] = self.port
        return result

class DNSZone:
    """
    Represents an authoritative DNS zone.
    Contains all records for a domain.
    Like a page in the phone book for one organization.
    """
    
    def __init__(self, domain, admin_email="admin", serial=1):
        self.domain = domain
        self.admin_email = admin_email
        self.serial = serial
        self.records = defaultdict(list)  # (name, type) → [records]
        
        # SOA record (auto-created)
        self.soa = {
            "primary_ns": f"ns1.{domain}",
            "admin_email": admin_email,
            "serial": serial,
            "refresh": 3600,
            "retry": 600,
            "expire": 86400,
            "minimum_ttl": 60
        }
    
    def add_record(self, record):
        """Add a DNS record to the zone."""
        key = (record.name.lower(), record.record_type)
        self.records[key].append(record)
        self.serial += 1
        return record
    
    def remove_record(self, name, record_type, value=None):
        """Remove DNS record(s)."""
        key = (name.lower(), record_type)
        if value:
            self.records[key] = [
                r for r in self.records[key] if r.value != value
            ]
        else:
            self.records[key] = []
        self.serial += 1
    
    def query(self, name, record_type):
        """Query for records matching name and type."""
        key = (name.lower(), record_type)
        records = self.records.get(key, [])
        return records
    
    def get_all_records(self):
        """Get all records in the zone."""
        all_records = []
        for key, records in self.records.items():
            for record in records:
                all_records.append(record.to_dict())
        return all_records

class DNSCache:
    """
    Caches DNS responses with TTL expiration.
    Same concept as Project 5 LRU cache but DNS-specific.
    """
    
    def __init__(self, max_size=10000, name="DNS-Cache"):
        self.max_size = max_size
        self.name = name
        self.store = OrderedDict()
        self.lock = threading.Lock()
        
        # Metrics
        self.hits = 0
        self.misses = 0
        self.expirations = 0
    
    def get(self, name, record_type):
        """Look up cached DNS response."""
        key = (name.lower(), record_type.value)
        
        with self.lock:
            if key not in self.store:
                self.misses += 1
                return None
            
            entry = self.store[key]
            
            # Check TTL expiration
            if time.time() > entry["expires_at"]:
                del self.store[key]
                self.expirations += 1
                self.misses += 1
                return None
            
            self.store.move_to_end(key)
            self.hits += 1
            
            return entry["records"]
    
    def put(self, name, record_type, records, ttl):
        """Cache DNS response."""
        key = (name.lower(), record_type.value)
        
        with self.lock:
            if len(self.store) >= self.max_size:
                self.store.popitem(last=False)
            
            self.store[key] = {
                "records": records,
                "cached_at": time.time(),
                "expires_at": time.time() + ttl,
                "ttl": ttl
            }
    
    def flush(self):
        """Clear entire cache."""
        with self.lock:
            self.store.clear()
    
    def flush_domain(self, domain):
        """Flush all cached records for a domain."""
        with self.lock:
            keys_to_remove = [
                k for k in self.store.keys()
                if domain.lower() in k[0]
            ]
            for key in keys_to_remove:
                del self.store[key]
            return len(keys_to_remove)
    
    def get_metrics(self):
        total = self.hits + self.misses
        return {
            "name": self.name,
            "entries": len(self.store),
            "max_size": self.max_size,
            "hits": self.hits,
            "misses": self.misses,
            "hit_rate_pct": round(
                self.hits / max(1, total) * 100, 2
            ),
            "expirations": self.expirations
        }
    
    def inspect(self, limit=10):
        """View cache contents."""
        entries = []
        now = time.time()
        with self.lock:
            for (name, rtype), entry in list(self.store.items())[-limit:]:
                entries.append({
                    "name": name,
                    "type": rtype,
                    "ttl_remaining": round(
                        max(0, entry["expires_at"] - now), 1
                    ),
                    "original_ttl": entry["ttl"],
                    "records": len(entry["records"]),
                    "expired": now > entry["expires_at"]
                })
        return entries

class RootDNSServer:
    """Simulates root DNS server — knows TLD server locations."""
    
    def __init__(self):
        self.name = "root-server"
        self.tld_map = {}  # .com → TLD server reference
        self.query_count = 0
    
    def register_tld(self, tld, tld_server):
        self.tld_map[tld.lower()] = tld_server
    
    def query(self, name, record_type):
        """Root server: return referral to TLD server."""
        self.query_count += 1
        
        parts = name.lower().rstrip('.').split('.')
        if len(parts) < 2:
            return {"type": "NXDOMAIN", "message": "Invalid domain"}
        
        tld = parts[-1]  # .com, .org, etc.
        
        if tld in self.tld_map:
            return {
                "type": "REFERRAL",
                "message": f"Ask the .{tld} TLD server",
                "tld": tld,
                "tld_server": self.tld_map[tld]
            }
        
        return {"type": "NXDOMAIN", "message": f"Unknown TLD: .{tld}"}


class TLDServer:
    """Simulates TLD server (.com, .org) — knows authoritative servers."""

                "auth_server": self.domain_map[domain]

class AuthoritativeDNSServer:
    """
    Simulates authoritative DNS server — has the actual records.
    Supports: GeoDNS, health-checked failover, round-robin.
    """
    
    def __init__(self, name="auth-server"):
        self.name = name
        self.zones = {}  # domain → DNSZone
        self.query_count = 0
        
        # Health check state for failover records
        self.health_state = {}  # (name, value) → healthy boolean
        
        # Geo routing rules
        self.geo_rules = {}  # name → {region: [records]}
    
    def add_zone(self, zone):
        self.zones[zone.domain.lower()] = zone
    
    def add_geo_rule(self, name, region, records):
        """Add geographic routing rule."""
        if name not in self.geo_rules:
            self.geo_rules[name] = {}
        self.geo_rules[name][region.lower()] = records
    
    def set_health(self, name, value, healthy):
        """Update health status for failover records."""
        self.health_state[(name.lower(), value)] = healthy
    
    def query(self, name, record_type, client_region=None):
        """Answer DNS query with the actual records."""
        self.query_count += 1
        name_lower = name.lower()
        
        # Check geo rules first
        if client_region and name_lower in self.geo_rules:
            geo_records = self.geo_rules[name_lower].get(
                client_region.lower()
            )
            if geo_records:
                healthy_records = [
                    r for r in geo_records
                    if self.health_state.get(
                        (name_lower, r.value), True
                    )
                ]
                if healthy_records:
                    return {
                        "type": "ANSWER",
                        "records": [r.to_dict() for r in healthy_records],
                        "geo_routed": True,
                        "region": client_region
                    }
        
        # Find the zone
        parts = name_lower.rstrip('.').split('.')
        domain = '.'.join(parts[-2:]) if len(parts) >= 2 else name_lower
        
        zone = self.zones.get(domain)
        if not zone:
            return {"type": "NXDOMAIN", "message": f"No zone for {domain}"}
        
        # Query the zone
        records = zone.query(name_lower, record_type)
        
        if not records:
            # Check for CNAME
            cname_records = zone.query(name_lower, RecordType.CNAME)
            if cname_records:
                return {
                    "type": "CNAME",
                    "records": [r.to_dict() for r in cname_records],
                    "message": f"CNAME: resolve {cname_records[0].value} instead"
                }
            
            return {
                "type": "NXDOMAIN",
                "message": f"No {record_type.value} record for {name}"
            }
        
        # Filter by health for failover
        healthy_records = [
            r for r in records
            if self.health_state.get((name_lower, r.value), True)
        ]
        
        if not healthy_records:
            # All records unhealthy — return all (best effort)
            healthy_records = records
        
        return {
            "type": "ANSWER",
            "records": [r.to_dict() for r in healthy_records],
            "geo_routed": False
        }

class RecursiveResolver:
    """
    Simulates a recursive DNS resolver (like 8.8.8.8).
    Walks the DNS hierarchy: Root → TLD → Authoritative.
    Caches results with TTL.
    """
    
    def __init__(self, root_server, name="Resolver"):
        self.root_server = root_server
        self.name = name
        self.cache = DNSCache(name=f"{name}-Cache")
        self.total_queries = 0
        self.resolution_log = []
    
    def resolve(self, name, record_type=RecordType.A,
                client_region=None, trace=False):
        """
        Resolve a DNS name to records.
        Implements full recursive resolution with caching.
        """
        self.total_queries += 1
        steps = []
        
        def log_step(step_num, action, detail):
            entry = {
                "step": step_num,
                "action": action,
                "detail": detail
            }
            steps.append(entry)
            if trace:
                print(f"      Step {step_num}: {action} → {detail}")
        
        # Step 1: Check cache
        cached = self.cache.get(name, record_type)
        if cached:
            log_step(1, "CACHE HIT", f"Found {name} in cache")
            return {
                "status": "SUCCESS",
                "records": cached,
                "source": "cache",
                "steps": steps
            }
        
        log_step(1, "CACHE MISS", f"{name} not in cache")
        
        # Step 2: Ask root server
        log_step(2, "QUERY ROOT", f"Asking root for {name}")
        root_response = self.root_server.query(name, record_type)
        
        if root_response["type"] == "NXDOMAIN":
            log_step(2, "ROOT NXDOMAIN", root_response["message"])
            return {
                "status": "NXDOMAIN",
                "message": root_response["message"],
                "steps": steps
            }
        
        log_step(2, "ROOT REFERRAL",
                 f"Referred to .{root_response['tld']} TLD server")
        
        # Step 3: Ask TLD server
        tld_server = root_response["tld_server"]
        log_step(3, "QUERY TLD", f"Asking .{root_response['tld']} for {name}")
        tld_response = tld_server.query(name, record_type)
        
        if tld_response["type"] == "NXDOMAIN":
            log_step(3, "TLD NXDOMAIN", tld_response["message"])
            return {
                "status": "NXDOMAIN",
                "message": tld_response["message"],
                "steps": steps
            }
        
        log_step(3, "TLD REFERRAL",
                 f"Referred to authoritative server for {tld_response['domain']}")
        
        # Step 4: Ask authoritative server
        auth_server = tld_response["auth_server"]
        log_step(4, "QUERY AUTH",
                 f"Asking authoritative server for {name}")
        auth_response = auth_server.query(
            name, record_type, client_region=client_region
        )
        
        if auth_response["type"] == "NXDOMAIN":
            log_step(4, "AUTH NXDOMAIN", auth_response["message"])
            # Cache negative result (NXDOMAIN)
            self.cache.put(name, record_type, [], 60)
            return {
                "status": "NXDOMAIN",
                "message": auth_response["message"],
                "steps": steps
            }
        
        # Handle CNAME
        if auth_response["type"] == "CNAME":
            cname_target = auth_response["records"][0]["value"]
            log_step(4, "CNAME",
                     f"{name} → CNAME → {cname_target}")
            log_step(5, "RESOLVE CNAME",
                     f"Now resolving {cname_target}")
            
            # Recursively resolve the CNAME target
            cname_result = self.resolve(
                cname_target, record_type,
                client_region=client_region, trace=trace
            )
            
            if cname_result["status"] == "SUCCESS":
                cname_result["cname_chain"] = [name, cname_target]
                cname_result["steps"] = steps + cname_result.get("steps", [])
            
            return cname_result
        
        # Step 5: Cache and return answer
        records = auth_response["records"]
        ttl = records[0]["ttl"] if records else 300
        
        geo_note = ""
        if auth_response.get("geo_routed"):
            geo_note = f" (geo-routed: {auth_response.get('region')})"
        
        log_step(5 if auth_response["type"] != "CNAME" else 6,
                 "ANSWER",
                 f"{name} → {[r['value'] for r in records]}"
                 f" TTL={ttl}{geo_note}")
        
        # Cache the result
        self.cache.put(name, record_type, records, ttl)
        
        return {
            "status": "SUCCESS",
            "records": records,
            "source": "authoritative",
            "geo_routed": auth_response.get("geo_routed", False),
            "steps": steps
        }
    
    def get_stats(self):
        return {
            "resolver": self.name,
            "total_queries": self.total_queries,
            "cache": self.cache.get_metrics()
        }

class DNSHealthChecker:
    """Monitors endpoints and updates DNS failover state."""
    
    def __init__(self, auth_server, check_interval=2.0):
        self.auth_server = auth_server
        self.check_interval = check_interval
        self.monitored = []  # List of (name, value, check_fn)
        self.running = False
        self.thread = None
        self.check_results = defaultdict(list)
    
    def add_monitor(self, dns_name, ip_value, check_fn):
        """Add an endpoint to monitor."""
        self.monitored.append({
            "name": dns_name,
            "value": ip_value,
            "check_fn": check_fn,
            "consecutive_failures": 0,
            "consecutive_successes": 0
        })
        self.auth_server.set_health(dns_name, ip_value, True)
    
    def start(self):
        self.running = True
        self.thread = threading.Thread(target=self._check_loop, daemon=True)
        self.thread.start()
    
    def stop(self):
        self.running = False
        if self.thread:
            self.thread.join(timeout=5)
    
    def _check_loop(self):
        while self.running:
            for monitor in self.monitored:
                try:
                    is_healthy = monitor["check_fn"]()
                except Exception:
                    is_healthy = False
                
                if is_healthy:

THE SESSION PROBLEM:
━━━━━━━━━━━━━━━━━━━

FROM PROJECT 6 WE KNOW:
  → Load balancer distributes requests across servers
  → Request 1 might go to Server 1
  → Request 2 might go to Server 3
  → Servers are identical and interchangeable

THE PROBLEM:
  → User logs in on Server 1
  → Server 1 stores: sessions["abc123"] = {user: "Alice", role: "admin"}
  → Next request → Load Balancer sends to Server 3
  → Server 3 checks: sessions["abc123"] → NOT FOUND
  → Server 3 says: "Who are you? Please log in again"
  → User: "I JUST logged in!" → frustration

  ┌────────┐     ┌────┐     ┌──────────┐
  │ Alice  │────→│ LB │────→│ Server 1 │  Login → stores session
  └────────┘     └────┘     └──────────┘
       │                         ✅ "Welcome, Alice!"
       │
       │ (next request)
       │         ┌────┐     ┌──────────┐
       └────────→│ LB │────→│ Server 3 │  No session → "Who are you?"
                 └────┘     └──────────┘
                                 ❌ "Please log in"

TWO SOLUTIONS:

  Bad: Sticky Sessions (IP Hash — Project 6)
    → Same user always goes to same server
    → Defeats load balancing
    → Server dies → user loses session

  Good: Stateless Servers + External State (Project 8)
    → Servers store NO session data
    → Session stored in: Redis cache, database, or JWT token
    → ANY server can serve ANY user
    → Servers are truly disposable
