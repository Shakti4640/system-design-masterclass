PILLAR 3: CONTROL PLANE vs DATA PLANE SEPARATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  THIS is the single most important architectural decision

  ┌──────────────────┐          ┌──────────────────┐
  │  CONTROL PLANE   │          │   DATA PLANE     │
  │                  │          │                  │
  │  "Decides WHAT   │ ──────→  │  "Does the WORK" │
  │   should happen" │          │                  │
  │                  │ ←──────  │  "Reports status"│
  └──────────────────┘          └──────────────────┘

  CONTROL PLANE characteristics:
    → Low throughput, high importance
    → Manages state and configuration
    → Must be highly available (Raft consensus — Project 45)
    → Can be slower (milliseconds to seconds OK)
    → Small number of nodes (3, 5, 7 — odd numbers for quorum)
    → Example: "Place container X on node Y"

  DATA PLANE characteristics:
    → High throughput, high volume
    → Processes actual customer workloads
    → Scales horizontally (hundreds to thousands of nodes)
    → Must be fast (microseconds to milliseconds)
    → Stateless where possible (Project 8)
    → Example: "Route this HTTP request to container X"

  WHY SEPARATE?
    → Control plane failure = can't make NEW changes
      but existing workloads KEEP RUNNING
    → Data plane failure = specific workloads affected
      but platform management KEEPS WORKING
    → They scale INDEPENDENTLY
    → They have DIFFERENT availability requirements

📌 WHAT COULD BE DONE

APPROACH A: MONOLITHIC PLATFORM (Don't do this)
  → Single giant application managing everything
  → Easier to build initially
  → Impossible to scale, upgrade, or debug at cloud scale
  → One bug takes down the entire platform

APPROACH B: MICROSERVICES WITH SHARED INFRASTRUCTURE (Common)
  → Each platform capability is a separate service
  → Compute Service, Storage Service, Network Service, etc.
  → Shared message bus, shared monitoring
  → This is what most cloud providers actually do
  → ✅ WE CHOOSE THIS

APPROACH C: FULLY DECENTRALIZED / PEER-TO-PEER (Experimental)
  → No central control plane
  → Nodes self-organize using gossip protocols (Project 33)
  → Very resilient but very hard to guarantee consistency
  → Used in some edge computing scenarios

APPROACH D: LAYERED PLATFORM (Hybrid — What we implement)
  → Layer 1: Infrastructure (raw compute, storage, network)
  → Layer 2: Platform Services (managed DB, cache, queues)
  → Layer 3: Application Services (serverless functions, ML serving)
  → Each layer uses services from the layer below
  → ✅ COMBINED WITH B — THIS IS OUR FINAL ARCHITECTURE

✅ WHAT SHOULD DO

1. ALWAYS separate control plane from data plane
   → Control plane crash must NOT stop running workloads
   → Data plane crash must NOT stop platform management

2. ALWAYS enforce tenant isolation at the API layer FIRST
   → Every request must carry tenant_id
   → Every query must filter by tenant_id
   → Defense in depth: isolate at network, compute, storage too

3. ALWAYS use quotas and admission control
   → Prevent any single tenant from exhausting shared resources
   → Hard limits on CPU, memory, storage, API calls
   → Soft limits with alerting before hard cutoff

4. ALWAYS design for partial failure
   → One node dying is NORMAL, not exceptional
   → One zone going offline is a Tuesday
   → Auto-healing must be the default, not manual intervention

5. ALWAYS meter before you bill
   → Emit usage events for EVERY resource action
   → Metering is the source of truth
   → If you can't meter it, you can't bill it, and you can't manage it

6. ALWAYS version your APIs
   → Tenants build on your API — you CANNOT break them
   → /v1/instances, /v2/instances
   → Deprecation policy with migration timeline

7. ALWAYS implement idempotency at the API level
   → "Create VM" might be retried due to network issues
   → Same request twice must NOT create two VMs
   → Use idempotency keys (Project 37)

8. ALWAYS deploy control plane changes with canary rollouts
   → ML model serving pattern (Project 49) applies here too
   → Roll out to 1% of traffic → monitor → expand

❌ WHAT SHOULD NOT DO

1. NEVER let the control plane be a single point of failure
   → Minimum 3 nodes with Raft consensus (Project 45)
   → Spread across availability zones
   → If control plane dies, NOTHING can be managed

2. NEVER trust tenant input
   → They will request 10,000 CPUs if you let them
   → They will try to access other tenants' data
   → Validate, sanitize, authorize EVERYTHING

3. NEVER share encryption keys between tenants
   → Each tenant gets their own key hierarchy
   → Key rotation must be per-tenant
   → Compromise of one tenant must not affect others

4. NEVER couple billing to the request path
   → Billing aggregation is async (Project 11, 43)
   → If billing system is slow, it must NOT slow down VM creation
   → Eventual consistency is fine for billing

5. NEVER do synchronous cross-region calls in the data path
   → User in Tokyo hitting an API that synchronously calls Virginia
   → Latency will destroy the experience
   → Use async replication (Project 46), local reads

6. NEVER skip graceful shutdown
   → When a node is being decommissioned
   → Drain workloads first → migrate → then terminate
   → Abrupt termination = customer data loss = lawsuits

7. NEVER assume clock synchronization across nodes
   → Clocks drift (Project 46 covered this)
   → Use logical clocks or hybrid clocks for ordering
   → Physical timestamps are for humans, not for consensus

8. NEVER design without capacity planning
   → Know your limits BEFORE tenants discover them
   → Load test at 2x expected peak
   → Have headroom for burst traffic

🚫 WHEN NOT TO USE

1. DON'T build a cloud platform when:
   → You have < 10 internal services
   → A managed cloud (AWS/GCP) solves your problem
   → You don't have a team of 50+ engineers to maintain it
   → Your scale doesn't justify the infrastructure cost

2. DON'T use Raft consensus for data plane routing
   → Too slow for per-request decisions
   → Use eventually consistent caches (Project 33)
   → Consensus is for CONTROL PLANE state only

3. DON'T use synchronous API calls for billing
   → Billing can be minutes behind — that's fine
   → Real-time billing at scale is impossibly expensive
   → Batch + stream hybrid (Project 43 Lambda architecture)

4. DON'T use a single database for the entire platform
   → Each service owns its own data (Project 30 CQRS)
   → Shared database = shared failure domain
   → Shared database = schema coupling nightmare

5. DON'T implement your own consensus protocol in production
   → Use proven implementations (etcd, ZooKeeper)
   → Project 45 teaches you HOW it works
   → Knowing internals ≠ building your own for production

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: CONTROL PLANE OVERLOAD
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Symptom: Tenants can't create/modify resources
  Cause: Too many API calls hitting control plane
  Why it happens: No rate limiting per tenant on internal paths
  Fix:
    → Rate limit at API Gateway (Project 9)
    → Queue non-urgent requests (Project 11)
    → Cache frequently-read state (Project 33)
    → Separate read path from write path (Project 30 CQRS)

PITFALL 2: NOISY NEIGHBOR
━━━━━━━━━━━━━━━━━━━━━━━━━
  Symptom: Tenant A's workload slows down Tenant B
  Cause: Shared physical resources without proper isolation
  Why it happens: CPU/IO/Network not properly limited per tenant
  Fix:
    → cgroup enforcement for CPU and memory
    → IO bandwidth limits per container
    → Network bandwidth throttling per tenant
    → Consider dedicated hosts for premium tenants

PITFALL 3: CASCADING FAILURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Symptom: One service failure brings down the entire platform
  Cause: Tight coupling between services
  Why it happens: Synchronous call chains without circuit breakers
  Fix:
    → Circuit breaker on every inter-service call (Project 21)
    → Bulkhead pattern: isolate failure domains
    → Async communication where possible (Project 29)
    → Graceful degradation: return cached/stale data instead of error

PITFALL 4: SPLIT BRAIN IN CONTROL PLANE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Symptom: Two nodes think they're the leader
  Cause: Network partition between control plane nodes
  Why it happens: Inadequate consensus protocol implementation
  Fix:
    → Raft/Paxos with proper quorum (Project 45)
    → Fencing tokens for leader actions (Project 26)
    → NEVER use simple leader election without consensus

PITFALL 5: BILLING DATA LOSS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Symptom: Tenants charged incorrectly (too little or too much)
  Cause: Usage events dropped or duplicated
  Why it happens: At-most-once delivery in metering pipeline
  Fix:
    → At-least-once delivery + idempotent aggregation
    → Write-ahead log for metering events (Project 45 principle)
    → Reconciliation jobs that cross-check usage vs billing

PITFALL 6: THUNDERING HERD ON RECOVERY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Symptom: Platform crashes again right after recovering
  Cause: All retrying clients hit the server simultaneously
  Why it happens: No exponential backoff, no jitter
  Fix:
    → Client-side exponential backoff with jitter
    → Server-side admission control during recovery
    → Circuit breaker prevents retry storms (Project 21)
    → Queue-based load leveling (Project 11)

PITFALL 7: CONFIGURATION DRIFT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Symptom: Nodes behave differently from each other
  Cause: Configuration pushed to some nodes but not others
  Why it happens: Manual config changes, partial rollouts
  Fix:
    → Single source of truth in consensus store
    → Nodes PULL config from central store
    → Version all configurations
    → Automated rollback on anomaly detection

PITFALL 8: ZOMBIE RESOURCES
━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Symptom: Resources running that nobody is using or paying for
  Cause: Deletion failed midway, or orphaned by failed scheduling
  Why it happens: No reconciliation between desired state and actual state
  Fix:
    → Reconciliation loop: continuously compare
      "what SHOULD exist" vs "what DOES exist"
    → Garbage collection for orphaned resources
    → Billing alerts for idle resources

# ---- Clock (Project 46: Logical Clocks) ----

class HybridClock:
    """
    Hybrid Logical Clock for ordering events across the platform.
    Physical time for humans, logical counter for ordering.
    Referenced: Project 46 (GlobeDB)
    """
    def __init__(self):
        self._lock = threading.Lock()
        self._logical_counter = 0
        self._last_physical = 0
    
    def now(self):
        with self._lock:
            physical = int(time.time() * 1000)
            if physical > self._last_physical:
                self._last_physical = physical
                self._logical_counter = 0
            else:
                self._logical_counter += 1
            return (self._last_physical, self._logical_counter)
    
    def to_string(self, ts=None):
        if ts is None:
            ts = self.now()
        return f"{ts[0]}.{ts[1]:04d}"


PLATFORM_CLOCK = HybridClock()

class EventBus:
    """
    In-memory event bus for decoupled communication.
    Every state change emits an event that multiple consumers handle.
    Referenced: Project 29 (EventFlow)
    """
    def __init__(self):
        self._subscribers = defaultdict(list)
        self._lock = threading.Lock()
        self._event_log = []
    
    def subscribe(self, event_type, handler):
        with self._lock:
            self._subscribers[event_type].append(handler)
    
    def publish(self, event_type, data):
        event = {
            "event_id": str(uuid.uuid4())[:8],
            "event_type": event_type,
            "data": data,
            "timestamp": PLATFORM_CLOCK.to_string()
        }
        with self._lock:
            self._event_log.append(event)
            handlers = list(self._subscribers.get(event_type, []))
        
        for handler in handlers:
            try:
                threading.Thread(target=handler, args=(event,), daemon=True).start()
            except Exception as e:
                print(f"[EVENT-BUS ERROR] {event_type}: {e}")
        
        return event
    
    def get_events(self, event_type=None, limit=50):
        with self._lock:
            if event_type:
                return [e for e in self._event_log if e["event_type"] == event_type][-limit:]
            return self._event_log[-limit:]

class MessageQueue:
    """
    In-memory message queue with consumer groups.
    Decouples producers from consumers.
    Referenced: Project 11 (The Post Office)
    """
    def __init__(self):
        self._queues = defaultdict(deque)
        self._lock = threading.Lock()
        self._processors = {}
        self._running = False
    
    def enqueue(self, queue_name, message):
        with self._lock:
            self._queues[queue_name].append({
                "id": str(uuid.uuid4())[:8],
                "payload": message,
                "enqueued_at": PLATFORM_CLOCK.to_string(),
                "attempts": 0
            })
    
    def dequeue(self, queue_name):
        with self._lock:
            if self._queues[queue_name]:
                msg = self._queues[queue_name].popleft()
                msg["attempts"] += 1
                return msg
        return None
    
    def register_processor(self, queue_name, processor_fn):
        self._processors[queue_name] = processor_fn
    
    def start_processing(self):
        self._running = True
        for queue_name in self._processors:
            t = threading.Thread(
                target=self._process_loop,
                args=(queue_name,),
                daemon=True
            )
            t.start()
    
    def _process_loop(self, queue_name):
        while self._running:
            msg = self.dequeue(queue_name)
            if msg:
                try:
                    self._processors[queue_name](msg["payload"])
                except Exception as e:
                    if msg["attempts"] < 3:
                        self.enqueue(queue_name, msg["payload"])
            else:
                time.sleep(0.1)
    
    def queue_depth(self, queue_name):
        with self._lock:
            return len(self._queues[queue_name])

# ---- Distributed Lock (Project 27: Shared Lock) ----

class DistributedLockManager:
    """
    Lease-based distributed locking.
    Prevents double-booking of resources during scheduling.
    Referenced: Project 27 (The Shared Lock)
    """
    def __init__(self):
        self._locks = {}
        self._lock = threading.Lock()
    
    def acquire(self, resource_id, holder_id, ttl_seconds=10):
        with self._lock:
            now = time.time()
            if resource_id in self._locks:
                existing = self._locks[resource_id]
                if existing["expires_at"] > now and existing["holder"] != holder_id:
                    return False
            
            self._locks[resource_id] = {
                "holder": holder_id,
                "acquired_at": now,
                "expires_at": now + ttl_seconds,
                "fencing_token": int(now * 1000)
            }
            return True
    
    def release(self, resource_id, holder_id):
        with self._lock:
            if resource_id in self._locks:
                if self._locks[resource_id]["holder"] == holder_id:
                    del self._locks[resource_id]
                    return True
            return False
    
    def is_locked(self, resource_id):
        with self._lock:
            if resource_id in self._locks:
                if self._locks[resource_id]["expires_at"] > time.time():
                    return True
                else:
                    del self._locks[resource_id]
            return False

# ---- Circuit Breaker (Project 21: Resilience) ----

class CircuitBreaker:
    """
    Prevents cascading failures by stopping calls to unhealthy services.
    States: CLOSED (normal) → OPEN (failing) → HALF_OPEN (testing)
    Referenced: Project 21 (The Circuit Breaker)
    """
    CLOSED = "CLOSED"
    OPEN = "OPEN"
    HALF_OPEN = "HALF_OPEN"
    
    def __init__(self, name, failure_threshold=5, recovery_timeout=30):
        self.name = name
        self.failure_threshold = failure_threshold
        self.recovery_timeout = recovery_timeout
        self.state = self.CLOSED
        self.failure_count = 0
        self.last_failure_time = 0
        self.success_count = 0
        self._lock = threading.Lock()
    
    def call(self, func, *args, **kwargs):
        with self._lock:
            if self.state == self.OPEN:
                if time.time() - self.last_failure_time > self.recovery_timeout:
                    self.state = self.HALF_OPEN
                    self.success_count = 0
                else:
                    raise Exception(f"Circuit {self.name} is OPEN — request rejected")
        
        try:
            result = func(*args, **kwargs)
            with self._lock:
                if self.state == self.HALF_OPEN:
                    self.success_count += 1
                    if self.success_count >= 3:
                        self.state = self.CLOSED
                        self.failure_count = 0
                self.failure_count = max(0, self.failure_count - 1)
            return result
        except Exception as e:
            with self._lock:
                self.failure_count += 1
                self.last_failure_time = time.time()
                if self.failure_count >= self.failure_threshold:
                    self.state = self.OPEN
            raise e
    
    def get_status(self):
        return {
            "name": self.name,
            "state": self.state,
            "failure_count": self.failure_count,
            "threshold": self.failure_threshold
        }

class CacheLayer:
    """
    LRU cache with TTL support.
    Reduces load on metadata store and speeds up reads.
    Referenced: Project 5 (Librarian's Memory), Project 33 (CacheGrid)
    """
    def __init__(self, max_size=1000, default_ttl=60):
        self.max_size = max_size
        self.default_ttl = default_ttl
        self._cache = OrderedDict()
        self._expiry = {}
        self._lock = threading.Lock()
        self._hits = 0
        self._misses = 0
    
    def get(self, key):
        with self._lock:
            if key in self._cache:
                if self._expiry[key] > time.time():
                    self._cache.move_to_end(key)
                    self._hits += 1
                    return copy.deepcopy(self._cache[key])
                else:
                    del self._cache[key]
                    del self._expiry[key]
            self._misses += 1
            return None
    
    def set(self, key, value, ttl=None):
        ttl = ttl or self.default_ttl
        with self._lock:
            if key in self._cache:
                self._cache.move_to_end(key)
            self._cache[key] = copy.deepcopy(value)
            self._expiry[key] = time.time() + ttl
            while len(self._cache) > self.max_size:
                oldest_key, _ = self._cache.popitem(last=False)
                self._expiry.pop(oldest_key, None)
    
    def invalidate(self, key):
        with self._lock:
            self._cache.pop(key, None)
            self._expiry.pop(key, None)
    
    def invalidate_prefix(self, prefix):
        with self._lock:
            keys_to_remove = [k for k in self._cache if k.startswith(prefix)]
            for k in keys_to_remove:
                del self._cache[k]
                self._expiry.pop(k, None)
    
    def stats(self):
        total = self._hits + self._misses
        return {
            "size": len(self._cache),
            "max_size": self.max_size,
            "hits": self._hits,
            "misses": self._misses,
            "hit_rate": round(self._hits / total, 4) if total > 0 else 0
        }

class RateLimiter:
    """
    Sliding window rate limiter per tenant.

        self._lock = threading.Lock()

        now = time.time()
        window_start = now - 60
        
        with self._lock:
            self._windows[tenant_id] = [
                t for t in self._windows[tenant_id] if t > window_start
            ]
            
            if len(self._windows[tenant_id]) >= limit_per_minute:
                return False
            
            self._windows[tenant_id].append(now)
            return True
    
    def get_usage(self, tenant_id):
        now = time.time()
        window_start = now - 60
        with self._lock:
            current = [t for t in self._windows[tenant_id] if t > window_start]
            return len(current)

class NodeRegistry:
    """
    Tracks all compute nodes in the data plane.
    Nodes send heartbeats; missing heartbeat = unhealthy.
    Referenced: Project 15 (Pulse Check), Project 20 (Service Directory)
    """
    def __init__(self):
        self._nodes = {}
        self._lock = threading.Lock()
        self._heartbeat_timeout = 30
    
    def register_node(self, node_id, region, zone, vcpus, memory_gb):
        node = ComputeNode(
            node_id=node_id,
            region=region,
            zone=zone,
            total_vcpus=vcpus,
            total_memory_gb=memory_gb,
            last_heartbeat=time.time()
        )
        with self._lock:
            self._nodes[node_id] = node
        
        EVENT_BUS.publish("node.registered", {
            "node_id": node_id,
            "region": region,
            "zone": zone
        })
        return node
    
    def heartbeat(self, node_id):
        with self._lock:
            if node_id in self._nodes:
                self._nodes[node_id].last_heartbeat = time.time()
                if self._nodes[node_id].status == NodeStatus.UNHEALTHY:
                    self._nodes[node_id].status = NodeStatus.HEALTHY
                return True
        return False
    
    def check_health(self):
        """Called periodically to detect failed nodes"""
        now = time.time()
        failed_nodes = []
        with self._lock:
            for node_id, node in self._nodes.items():
                if node.status == NodeStatus.OFFLINE:
                    continue
                if now - node.last_heartbeat > self._heartbeat_timeout:
                    if node.status != NodeStatus.UNHEALTHY:
                        node.status = NodeStatus.UNHEALTHY
                        failed_nodes.append(node_id)
        
        for node_id in failed_nodes:
            EVENT_BUS.publish("node.unhealthy", {"node_id": node_id})
        
        return failed_nodes
    
    def get_healthy_nodes(self, region=None, zone=None):
        with self._lock:
            nodes = []
            for node in self._nodes.values():
                if node.status != NodeStatus.HEALTHY:
                    continue
                if region and node.region != region:
                    continue
                if zone and node.zone != zone:
                    continue
                nodes.append(node)
            return nodes
    
    def get_node(self, node_id):
        with self._lock:
            return self._nodes.get(node_id)
    
    def list_nodes(self):
        with self._lock:
            return [n.to_dict() for n in self._nodes.values()]
    
    def allocate_resources(self, node_id, vcpus, memory_gb, instance_id):
        with self._lock:
            node = self._nodes.get(node_id)
            if not node:
                return False
            if node.free_vcpus < vcpus or node.free_memory_gb < memory_gb:
                return False
            node.used_vcpus += vcpus
            node.used_memory_gb += memory_gb
            node.instances.append(instance_id)
            return True
    
    def release_resources(self, node_id, vcpus, memory_gb, instance_id):
        with self._lock:
            node = self._nodes.get(node_id)
            if not node:
                return False
            node.used_vcpus = max(0, node.used_vcpus - vcpus)
            node.used_memory_gb = max(0, node.used_memory_gb - memory_gb)
            if instance_id in node.instances:
                node.instances.remove(instance_id)
            return True

class ResourceScheduler:
    """
    Two-phase scheduler: Filter → Score → Bind
    Finds the best node for a workload request.
    Referenced: Project 36 (TaskMaster)
    """
    def __init__(self, node_registry, lock_manager):
        self._node_registry = node_registry
        self._lock_manager = lock_manager
        self._scheduling_circuit = CircuitBreaker("scheduler", failure_threshold=3, recovery_timeout=15)
    
    def schedule(self, instance):
        """
        Main scheduling pipeline:
          1. Filter: Which nodes CAN run this?
          2. Score: Which node is BEST?
          3. Bind: Reserve resources on chosen node
        """
        def _do_schedule():
            # PHASE 1: FILTER
            candidates = self._filter_nodes(instance)
            if not candidates:
                raise Exception(
                    f"No nodes available in {instance.region}/{instance.zone} "
                    f"with {instance.vcpus} vCPUs and {instance.memory_gb} GB RAM"
                )
            
            # PHASE 2: SCORE
            scored = self._score_nodes(candidates, instance)
            scored.sort(key=lambda x: x[1], reverse=True)
            
            # PHASE 3: BIND (try top candidates)
            for node, score in scored[:3]:
                lock_key = f"node-schedule:{node.node_id}"
                holder = f"inst:{instance.instance_id}"
                
                if self._lock_manager.acquire(lock_key, holder, ttl_seconds=5):
                    try:
                        success = self._node_registry.allocate_resources(
                            node.node_id,
                            instance.vcpus,
                            instance.memory_gb,
                            instance.instance_id
                        )
                        if success:
                            return node
                    finally:
                        self._lock_manager.release(lock_key, holder)
            
            raise Exception("Failed to bind to any candidate node (resource contention)")
        
        return self._scheduling_circuit.call(_do_schedule)
    
    def _filter_nodes(self, instance):
        """Remove nodes that cannot run this instance"""
        healthy = self._node_registry.get_healthy_nodes(
            region=instance.region,
            zone=instance.zone
        )
        
        candidates = []
        for node in healthy:
            if node.free_vcpus >= instance.vcpus and node.free_memory_gb >= instance.memory_gb:
                candidates.append(node)
        
        return candidates
    
    def _score_nodes(self, candidates, instance):
        """
        Score each candidate node.
        Higher score = better placement.
        
        Scoring factors:
          1. Balance: prefer nodes that result in even utilization
          2. Spread: prefer spreading tenant instances across nodes
          3. Fit: prefer nodes where the instance fits well (not too much waste)
        """
        scored = []
        for node in candidates:
            # Factor 1: Balance score (prefer ~60-70% utilization after placement)
            future_cpu_util = (node.used_vcpus + instance.vcpus) / node.total_vcpus
            future_mem_util = (node.used_memory_gb + instance.memory_gb) / node.total_memory_gb
            avg_util = (future_cpu_util + future_mem_util) / 2
            balance_score = 1.0 - abs(avg_util - 0.65)
            
            # Factor 2: Spread score (fewer tenant instances on this node = better)
            tenant_instances_on_node = sum(
                1 for iid in node.instances
                if INSTANCE_STORE.get(iid) and 
                   INSTANCE_STORE.get(iid).tenant_id == instance.tenant_id
            )
            spread_score = 1.0 / (1 + tenant_instances_on_node)
            
            # Factor 3: Fit score (less wasted resources = better)
            cpu_fit = instance.vcpus / node.free_vcpus if node.free_vcpus > 0 else 0
            mem_fit = instance.memory_gb / node.free_memory_gb if node.free_memory_gb > 0 else 0
            fit_score = (cpu_fit + mem_fit) / 2
            
            total_score = (balance_score * 0.4) + (spread_score * 0.3) + (fit_score * 0.3)
            scored.append((node, round(total_score, 4)))
        
        return scored

# The main service that handles instance lifecycle
# ================================================================

class ComputeService:
    """
    Manages the full lifecycle of compute instances.
    Create → Schedule → Provision → Monitor → Terminate
    This is the primary control plane service.
    Referenced: Project 2 (REST API), Project 36 (TaskMaster)
    """
    def __init__(self):
        self._idempotency_store = {}
        self._lock = threading.Lock()
    
    def create_instance(self, tenant, request_body, idempotency_key=None):
        """Full instance creation pipeline"""
        
        # Step 1: Idempotency check (Project 37)
        if idempotency_key:
            with self._lock:
                if idempotency_key in self._idempotency_store:
                    return self._idempotency_store[idempotency_key]
        
        # Step 2: Validate request
        name = request_body.get("name", "")
        instance_type = request_body.get("type", "")
        region = request_body.get("region", tenant.region)
        zone = request_body.get("zone", f"{region}a")
        
        if not name:
            return {"status": 400, "error": "Missing 'name' field"}
        
        if instance_type not in INSTANCE_TYPES:
            return {
                "status": 400,
                "error": f"Unknown instance type: '{instance_type}'",
                "available_types": list(INSTANCE_TYPES.keys())
            }
        
        type_spec = INSTANCE_TYPES[instance_type]
        
        # Step 3: Quota check (Admission Control)
        usage = INSTANCE_STORE.get_tenant_usage(tenant.tenant_id)
        quota = tenant.quota
        
        if usage["active_instances"] >= quota.max_instances:
            return {
                "status": 429,
                "error": f"Instance limit reached ({quota.max_instances})",
                "current_usage": usage
            }
        
        if usage["running_vcpus"] + type_spec["vcpus"] > quota.max_vcpus:
            return {
                "status": 429,
                "error": f"vCPU quota exceeded (limit: {quota.max_vcpus})",
                "current_usage": usage
            }
        
        if usage["running_memory_gb"] + type_spec["memory_gb"] > quota.max_memory_gb:
            return {
                "status": 429,
                "error": f"Memory quota exceeded (limit: {quota.max_memory_gb} GB)",
                "current_usage": usage
            }
        
        # Step 4: Create instance record
        instance_id = f"i-{str(uuid.uuid4())[:8]}"
        instance = Instance(
            instance_id=instance_id,
            tenant_id=tenant.tenant_id,
            name=name,
            instance_type=instance_type,
            vcpus=type_spec["vcpus"],
            memory_gb=type_spec["memory_gb"],
            region=region,
            zone=zone,
            status=InstanceStatus.SCHEDULING,
            price_per_hour=type_spec["price_per_hour"],
            created_at=PLATFORM_CLOCK.to_string()
        )
        INSTANCE_STORE.create(instance)
        
        # Step 5: Schedule (find a node)
        try:
            node = SCHEDULER.schedule(instance)
            
            # Step 6: Provision (simulate starting on node)
            private_ip = f"10.{random.randint(0,255)}.{random.randint(0,255)}.{random.randint(1,254)}"
            
            INSTANCE_STORE.update_status(
                instance_id,
                InstanceStatus.RUNNING,
                node_id=node.node_id,
                private_ip=private_ip,
                started_at=PLATFORM_CLOCK.to_string()
            )
            
            # Step 7: Emit events
            EVENT_BUS.publish("instance.created", {
                "instance_id": instance_id,
                "tenant_id": tenant.tenant_id,
                "instance_type": instance_type,
                "node_id": node.node_id,
                "region": region,
                "zone": zone,
                "vcpus": type_spec["vcpus"],
                "memory_gb": type_spec["memory_gb"],
                "price_per_hour": type_spec["price_per_hour"]
            })
            
            # Enqueue metering start
            MESSAGE_QUEUE.enqueue("metering", {
                "action": "start_metering",
                "instance_id": instance_id,
                "tenant_id": tenant.tenant_id,
                "resource_spec": type_spec
            })
            
            result = {
                "status": 201,
                "message": "Instance created successfully",
                "instance": INSTANCE_STORE.get(instance_id).to_dict()
            }
            
        except Exception as e:
            INSTANCE_STORE.update_status(instance_id, InstanceStatus.FAILED)
            result = {
                "status": 503,
                "error": f"Scheduling failed: {str(e)}",
                "instance_id": instance_id
            }
        
        # Store for idempotency
        if idempotency_key:
            with self._lock:
                self._idempotency_store[idempotency_key] = result
        
        return result
    
    def terminate_instance(self, tenant, instance_id):
        """Graceful instance termination"""
        instance = INSTANCE_STORE.get(instance_id)
        
        if not instance:
            return {"status": 404, "error": f"Instance {instance_id} not found"}
        
        if instance.tenant_id != tenant.tenant_id:
            return {"status": 403, "error": "Access denied — not your instance"}
        
        if instance.status == InstanceStatus.TERMINATED:
            return {"status": 400, "error": "Instance already terminated"}
        
        # Release resources on node
        if instance.node_id:
            NODE_REGISTRY.release_resources(
                instance.node_id,
                instance.vcpus,
                instance.memory_gb,
                instance.instance_id
            )
        
        INSTANCE_STORE.update_status(
            instance_id,
            InstanceStatus.TERMINATED,
            terminated_at=PLATFORM_CLOCK.to_string()
        )
        
        EVENT_BUS.publish("instance.terminated", {
            "instance_id": instance_id,
            "tenant_id": tenant.tenant_id,
            "node_id": instance.node_id
        })
        
        MESSAGE_QUEUE.enqueue("metering", {
            "action": "stop_metering",
            "instance_id": instance_id,
            "tenant_id": tenant.tenant_id
        })
        
        return {
            "status": 200,
            "message": f"Instance {instance_id} terminated",
            "instance": INSTANCE_STORE.get(instance_id).to_dict()
        }
    
    def get_instance(self, tenant, instance_id):
        instance = INSTANCE_STORE.get(instance_id)
        if not instance:
            return {"status": 404, "error": f"Instance {instance_id} not found"}
        if instance.tenant_id != tenant.tenant_id:
            return {"status": 403, "error": "Access denied"}
        return {"status": 200, "instance": instance.to_dict()}
    
    def list_instances(self, tenant):
        cached = PLATFORM_CACHE.get(f"instances:{tenant.tenant_id}")
        if cached:
            return cached
        
        instances = INSTANCE_STORE.get_by_tenant(tenant.tenant_id)
        result = {
            "status": 200,
            "count": len(instances),
            "instances": [i.to_dict() for i in instances]
        }
        PLATFORM_CACHE.set(f"instances:{tenant.tenant_id}", result, ttl=30)
        return result

class ObjectStorageService:
    """
    Key-value object storage per tenant with namespace isolation.
    Referenced: Project 42 (InfiniteStore), Project 17 (PasteDrop)
    """
    def __init__(self):
        self._buckets = defaultdict(dict)
        self._metadata = defaultdict(dict)
        self._lock = threading.Lock()
    
    def put_object(self, tenant_id, bucket, key, data):
        namespace = f"{tenant_id}/{bucket}"
        content_hash = hashlib.sha256(data.encode()).hexdigest()[:16]
        
        with self._lock:
            self._buckets[namespace][key] = data
            self._metadata[namespace][key] = {
                "key": key,
                "bucket": bucket,
                "size_bytes": len(data.encode('utf-8')),
                "content_hash": content_hash,
                "created_at": PLATFORM_CLOCK.to_string(),
                "tenant_id": tenant_id
            }
        
        EVENT_BUS.publish("storage.object_created", {
            "tenant_id": tenant_id,
            "bucket": bucket,
            "key": key,
            "size_bytes": len(data.encode('utf-8'))
        })
        
        return {
            "status": 201,
            "message": "Object stored",
            "metadata": self._metadata[namespace][key]
        }
    
    def get_object(self, tenant_id, bucket, key):
        namespace = f"{tenant_id}/{bucket}"
        with self._lock:
            if namespace in self._buckets and key in self._buckets[namespace]:
                return {
                    "status": 200,
                    "data": self._buckets[namespace][key],
                    "metadata": self._metadata[namespace][key]
                }
        return {"status": 404, "error": f"Object {bucket}/{key} not found"}
    
    def delete_object(self, tenant_id, bucket, key):
        namespace = f"{tenant_id}/{bucket}"
        with self._lock:
            if namespace in self._buckets and key in self._buckets[namespace]:
                del self._buckets[namespace][key]
                meta = self._metadata[namespace].pop(key, {})
                
                EVENT_BUS.publish("storage.object_deleted", {
                    "tenant_id": tenant_id,
                    "bucket": bucket,
                    "key": key
                })
                
                return {"status": 200, "message": f"Object {bucket}/{key} deleted"}
        return {"status": 404, "error": f"Object {bucket}/{key} not found"}
    
    def list_objects(self, tenant_id, bucket):
        namespace = f"{tenant_id}/{bucket}"
        with self._lock:
            objects = list(self._metadata.get(namespace, {}).values())
        return {
            "status": 200,
            "bucket": bucket,
            "count": len(objects),
            "objects": objects
        }
    
    def get_tenant_storage_usage(self, tenant_id):
        total_bytes = 0
        total_objects = 0
        with self._lock:
            for namespace, objects in self._metadata.items():
                if namespace.startswith(f"{tenant_id}/"):
                    for meta in objects.values():
                        total_bytes += meta["size_bytes"]
                        total_objects += 1
        return {
            "total_bytes": total_bytes,
            "total_objects": total_objects,
            "total_gb": round(total_bytes / (1024 ** 3), 6)
        }

class MeteringService:
    """
    Tracks per-tenant resource usage for billing.
    Consumes events from the message queue.
    Referenced: Project 37 (PayFlow), Project 43 (PulseStream)
    """
    def __init__(self):
        self._active_meters = {}
        self._usage_records = defaultdict(list)
        self._invoices = defaultdict(list)
        self._lock = threading.Lock()
    
    def start_metering(self, instance_id, tenant_id, resource_spec):
        with self._lock:
            self._active_meters[instance_id] = {
                "tenant_id": tenant_id,
                "started_at": time.time(),
                "resource_spec": resource_spec,
                "instance_id": instance_id
            }
    
    def stop_metering(self, instance_id, tenant_id):
        with self._lock:
            meter = self._active_meters.pop(instance_id, None)
            if meter:
                duration_hours = (time.time() - meter["started_at"]) / 3600
                cost = duration_hours * meter["resource_spec"]["price_per_hour"]
                
                record = {
                    "instance_id": instance_id,
                    "tenant_id": tenant_id,
                    "duration_hours": round(duration_hours, 6),
                    "price_per_hour": meter["resource_spec"]["price_per_hour"],
                    "cost": round(cost, 6),
                    "vcpus": meter["resource_spec"]["vcpus"],
                    "memory_gb": meter["resource_spec"]["memory_gb"],
                    "recorded_at": PLATFORM_CLOCK.to_string()
                }
                self._usage_records[tenant_id].append(record)
                
                EVENT_BUS.publish("billing.usage_recorded", {
                    "tenant_id": tenant_id,
                    "instance_id": instance_id,
                    "cost": record["cost"]
                })
                
                return record
        return None
    
    def process_metering_message(self, payload):
        """Queue processor for metering messages"""
        action = payload.get("action")
        if action == "start_metering":
            self.start_metering(
                payload["instance_id"],
                payload["tenant_id"],
                payload["resource_spec"]
            )
        elif action == "stop_metering":
            self.stop_metering(
                payload["instance_id"],
                payload["tenant_id"]
            )
    
    def get_current_costs(self, tenant_id):
        """Calculate real-time running costs"""
        with self._lock:
            running_cost = 0
            active_resources = []
            now = time.time()
            
            for instance_id, meter in self._active_meters.items():
                if meter["tenant_id"] == tenant_id:
                    duration_hours = (now - meter["started_at"]) / 3600
                    cost = duration_hours * meter["resource_spec"]["price_per_hour"]
                    active_resources.append({
                        "instance_id": instance_id,
                        "running_hours": round(duration_hours, 4),
                        "current_cost": round(cost, 6)
                    })
                    running_cost += cost
            
            historical = self._usage_records.get(tenant_id, [])
            historical_cost = sum(r["cost"] for r in historical)
            
            return {
                "status": 200,
                "tenant_id": tenant_id,
                "active_resources": active_resources,
                "running_cost": round(running_cost, 6),
                "historical_cost": round(historical_cost, 6),
                "total_cost": round(running_cost + historical_cost, 6),
                "currency": "USD"
            }
    
    def generate_invoice(self, tenant_id):
        costs = self.get_current_costs(tenant_id)
        storage = STORAGE_SERVICE.get_tenant_storage_usage(tenant_id)
        storage_cost = storage["total_gb"] * 0.023
        
        invoice = {
            "invoice_id": f"INV-{str(uuid.uuid4())[:8]}",
            "tenant_id": tenant_id,
            "generated_at": PLATFORM_CLOCK.to_string(),
            "line_items": {
                "compute": costs["total_cost"],
                "storage": round(storage_cost, 6),
            },
            "total": round(costs["total_cost"] + storage_cost, 6),
            "currency": "USD"
        }
        
        with self._lock:
            self._invoices[tenant_id].append(invoice)
        
        return {"status": 200, "invoice": invoice}

# Continuously ensures desired state matches actual state

class PlatformMonitor:
    """
    Background loops that keep the platform healthy:
      1. Node health checking (heartbeat monitoring)
      2. Reconciliation (desired state vs actual state)
      3. Zombie resource cleanup
    Referenced: Project 15 (Pulse Check), Project 22 (Detective Trail)
    """
    def __init__(self):
        self._running = False
        self._health_check_interval = 10
        self._reconcile_interval = 30
        self._stats = {
            "health_checks_run": 0,
            "reconciliations_run": 0,
            "instances_rescheduled": 0,
            "zombies_cleaned": 0
        }
    
    def start(self):
        self._running = True
        threading.Thread(target=self._health_check_loop, daemon=True).start()
        threading.Thread(target=self._reconciliation_loop, daemon=True).start()
        threading.Thread(target=self._simulate_heartbeats, daemon=True).start()
    
    def stop(self):
        self._running = False
    
    def _simulate_heartbeats(self):
        """Simulate node heartbeats (in production, nodes send these)"""
        while self._running:
            for node_data in NODE_REGISTRY.list_nodes():
                if node_data["status"] != "offline":
                    NODE_REGISTRY.heartbeat(node_data["node_id"])
            time.sleep(5)
    
    def _health_check_loop(self):
        """Detect failed nodes"""
        while self._running:
            time.sleep(self._health_check_interval)
            failed = NODE_REGISTRY.check_health()
            self._stats["health_checks_run"] += 1
            
            if failed:
                for node_id in failed:
                    self._handle_node_failure(node_id)
    
    def _handle_node_failure(self, node_id):
        """Reschedule instances from a failed node"""
        affected = INSTANCE_STORE.get_instances_on_node(node_id)
        for instance in affected:
            try:
                INSTANCE_STORE.update_status(
                    instance.instance_id,
                    InstanceStatus.SCHEDULING
                )
                new_node = SCHEDULER.schedule(instance)
                
                private_ip = f"10.{random.randint(0,255)}.{random.randint(0,255)}.{random.randint(1,254)}"
                INSTANCE_STORE.update_status(
                    instance.instance_id,
                    InstanceStatus.RUNNING,
                    node_id=new_node.node_id,
                    private_ip=private_ip
                )
                
                self._stats["instances_rescheduled"] += 1
                
                EVENT_BUS.publish("instance.rescheduled", {
                    "instance_id": instance.instance_id,
                    "from_node": node_id,
                    "to_node": new_node.node_id,
                    "tenant_id": instance.tenant_id
                })
                
            except Exception as e:
                INSTANCE_STORE.update_status(
                    instance.instance_id,
                    InstanceStatus.FAILED
                )
                EVENT_BUS.publish("instance.reschedule_failed", {
                    "instance_id": instance.instance_id,
                    "error": str(e)
                })
    
    def _reconciliation_loop(self):
        """Ensure desired state matches actual state"""
        while self._running:
            time.sleep(self._reconcile_interval)
            self._stats["reconciliations_run"] += 1
            self._clean_zombies()
    
    def _clean_zombies(self):
        """Find and clean up orphaned resources"""
        all_instances = INSTANCE_STORE.list_all()
        for inst_data in all_instances:
            if inst_data["status"] == "running" and inst_data["node_id"]:
                node = NODE_REGISTRY.get_node(inst_data["node_id"])
                if not node or node.status == NodeStatus.OFFLINE:
                    INSTANCE_STORE.update_status(
                        inst_data["instance_id"],
                        InstanceStatus.FAILED
                    )
                    self._stats["zombies_cleaned"] += 1
    
    def get_stats(self):
        return self._stats

# Tracks platform availability and SLA compliance

class SLAEngine:
    """
    Monitors and reports on SLA compliance per tenant tier.
    Referenced: Pillar 4 in the design section above
    """
    SLA_TARGETS = {
        TenantTier.FREE: 0.99,
        TenantTier.STANDARD: 0.999,
        TenantTier.PREMIUM: 0.9999,
        TenantTier.ENTERPRISE: 0.9999,
    }
    
    def __init__(self):
        self._uptime_records = defaultdict(lambda: {"up": 0, "total": 0})
        self._incidents = []
        self._lock = threading.Lock()
    
    def record_check(self, tenant_id, is_up):
        with self._lock:
            self._uptime_records[tenant_id]["total"] += 1
            if is_up:
                self._uptime_records[tenant_id]["up"] += 1
    
    def record_incident(self, tenant_id, description, duration_minutes):
        with self._lock:
            self._incidents.append({
                "tenant_id": tenant_id,
                "description": description,
                "duration_minutes": duration_minutes,
                "recorded_at": PLATFORM_CLOCK.to_string()
            })
    
    def get_sla_report(self, tenant_id):
        tenant = TENANT_REGISTRY.get_tenant(tenant_id)
        if not tenant:
            return {"status": 404, "error": "Tenant not found"}
        
        with self._lock:
            record = self._uptime_records.get(tenant_id, {"up": 100, "total": 100})
            availability = record["up"] / record["total"] if record["total"] > 0 else 1.0
            target = self.SLA_TARGETS.get(tenant.tier, 0.99)
            
            return {
                "status": 200,
                "tenant_id": tenant_id,
                "tier": tenant.tier.value,
                "sla_target": target,
                "current_availability": round(availability, 6),
                "compliant": availability >= target,
                "total_checks": record["total"],
                "incidents": [
                    i for i in self._incidents if i["tenant_id"] == tenant_id
                ]
            }
    
    def get_platform_report(self):
        with self._lock:
            total_checks = sum(r["total"] for r in self._uptime_records.values())
            total_up = sum(r["up"] for r in self._uptime_records.values())
            availability = total_up / total_checks if total_checks > 0 else 1.0
            
            return {
                "platform_availability": round(availability, 6),
                "total_tenants_monitored": len(self._uptime_records),
                "total_incidents": len(self._incidents),
                "total_checks": total_checks
            }


SLA_ENGINE = SLAEngine()

    def read_body(self):
        length = int(self.headers.get('Content-Length', 0))
        if length > 0:
            return json.loads(self.rfile.read(length).decode('utf-8'))
        return {}

    # ---- MAIN REQUEST ROUTER ----
    
    def route_request(self, method):
        path, query = self.parse_path()
        correlation_id = str(uuid.uuid4())[:8]
        
        # Step 1: Try public endpoints (no auth needed)
        if self.handle_public(method, path):
            return
        
        # Step 2: Try admin endpoints (no auth for demo — in production, admin auth)
        if path.startswith("/admin"):
            if self.handle_admin(method, path):
                return
        
        # Step 3: Authenticate
        tenant = self.authenticate()
        if not tenant:
            self.send_json(401, {
                "status": 401,
                "error": "Unauthorized — provide valid API key",
                "hint": "Set header: Authorization: Bearer <your-api-key>",
                "correlation_id": correlation_id
            })
            return
        
        # Step 4: Rate limit check (Project 9)
        if not self.rate_limit_check(tenant):
            self.send_json(429, {
                "status": 429,
                "error": "Rate limit exceeded",
                "limit": tenant.quota.max_api_calls_per_minute,
                "tier": tenant.tier.value,
                "retry_after_seconds": 60,
                "correlation_id": correlation_id
            })
            return
        
        # Step 5: Billing status check
        if tenant.billing_status != "active":
            self.send_json(403, {
                "status": 403,
                "error": "Account suspended — billing issue",
                "billing_status": tenant.billing_status,
                "correlation_id": correlation_id
            })
            return
        
        # Step 6: Route to tenant-scoped endpoints
        if self.handle_tenant_request(method, path, tenant):
            return
        
        # Step 7: Not found
        self.send_json(404, {
            "status": 404,
            "error": f"Endpoint not found: {method} {path}",
            "correlation_id": correlation_id,
            "available_endpoints": {
                "public": {
                    "GET /health": "Platform health check",
                    "GET /v1/instance-types": "List instance types",
                    "POST /v1/tenants": "Create new tenant"
                },
                "authenticated": {
                    "GET /v1/instances": "List your instances",
                    "POST /v1/instances": "Create an instance",
                    "GET /v1/instances/{id}": "Get instance details",
                    "DELETE /v1/instances/{id}": "Terminate instance",
                    "PUT /v1/storage/{bucket}/{key}": "Store object",
                    "GET /v1/storage/{bucket}/{key}": "Get object",
                    "GET /v1/storage/{bucket}": "List objects in bucket",
                    "DELETE /v1/storage/{bucket}/{key}": "Delete object",
                    "GET /v1/billing/costs": "Current costs",
                    "GET /v1/billing/invoice": "Generate invoice",
                    "GET /v1/usage": "Resource usage & quota",
                    "GET /v1/sla": "SLA report"
                },
                "admin": {
                    "GET /admin/nodes": "All compute nodes",
                    "GET /admin/tenants": "All tenants",
                    "GET /admin/instances": "All instances",
                    "GET /admin/events": "Event log",
                    "GET /admin/monitor": "Platform monitoring"
                }
            }
        })
    
    # ---- HTTP Method Handlers ----
    
    def do_GET(self):
        self.route_request("GET")
    
    def do_POST(self):
        self.route_request("POST")
    
    def do_PUT(self):
        self.route_request("PUT")
    
    def do_DELETE(self):
        self.route_request("DELETE")

THE 5 UNIVERSAL TRUTHS OF SYSTEM DESIGN:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. EVERYTHING IS A TRADEOFF
     → Consistency vs Availability (CAP)
     → Latency vs Throughput
     → Simplicity vs Scalability
     → Cost vs Performance
     → There are NO perfect systems — only appropriate ones

  2. FAILURE IS NORMAL, NOT EXCEPTIONAL
     → Disks fail, networks partition, processes crash
     → Design for failure from Day 1
     → Redundancy, retries, circuit breakers, graceful degradation

  3. DATA IS THE HARDEST PROBLEM
     → Moving data is expensive
     → Keeping data consistent is hard
     → Losing data is unforgivable
     → Every architectural decision is ultimately about data

  4. COMPLEXITY IS THE ENEMY
     → The best system is the SIMPLEST one that works
     → Don't use microservices when a monolith suffices
     → Don't shard when a single database handles the load
     → Add complexity only when the PROBLEM demands it

  5. UNDERSTAND THE REQUIREMENTS BEFORE THE ARCHITECTURE
     → How many users? How much data? What latency?
     → What's the read:write ratio?
     → What consistency model do users need?
     → What's the budget?
     → THESE answers determine the design, not the other way around
