🎯 HYPOTHETICAL PROBLEM

You are building a real-time analytics platform like
Apache Kafka + Flink, Google Cloud Dataflow, or Amazon Kinesis

The situation:
→ 10 MILLION events per second flowing in from:
    • 500M mobile app users generating click events
    • 100K web servers emitting access logs
    • 50K IoT sensors sending temperature readings
    • 10K payment terminals processing transactions
→ Each event: 200-500 bytes of structured data
→ Total throughput: 2-5 GB/sec ingestion
→ Must answer questions IN REAL TIME:

  "How many users are active RIGHT NOW?"
    → Must update every 1 second
    → Cannot wait for end-of-day batch job

  "What is the 95th percentile page load time in the last 5 minutes?"
    → Sliding window aggregation
    → Must be available within 3 seconds of data arriving

  "Alert me if error rate exceeds 5% in any 1-minute window"
    → Continuous monitoring with threshold detection
    → Alert must fire within 10 seconds of threshold breach

  "What are the top-10 trending products in the last hour?"
    → Rolling aggregation with ranking
    → Must update every minute

  "Show me revenue by region for the last 24 hours, updating live"
    → Dashboard that refreshes every 5 seconds
    → Must handle late-arriving events gracefully

WHAT GOES WRONG without proper design:

  Scenario A — BATCH PROCESSING ONLY:
    → Collect all events into files
    → Run MapReduce/Spark job every hour
    → "Active users right now" → answer is 1 hour stale
    → Error rate alert → fires 1 hour after the outage started
    → By then, millions of users experienced errors
    → UNACCEPTABLE for real-time operations

  Scenario B — SIMPLE DATABASE INSERT:
    → INSERT every event into PostgreSQL
    → 10M events/sec × 200 bytes = 2 GB/sec writes
    → PostgreSQL max: ~50K inserts/sec with indexing
    → 10M / 50K = 200× overloaded
    → Database crashes within seconds

  Scenario C — IN-MEMORY AGGREGATION ON ONE MACHINE:
    → Keep all aggregations in memory on one server
    → Server has 256 GB RAM
    → 10M events/sec × 200 bytes = 2 GB/sec
    → Fill memory in 128 seconds
    → Even with windowing: tracking 500M unique users = ~50GB
    → Single point of failure — server dies, all state lost

  Scenario D — EVENTS ARRIVE OUT OF ORDER:
    → Event happened at 12:00:00 on user's phone
    → Network delay: arrives at server at 12:00:15 (15 sec late)
    → The 12:00:00 window already closed and was reported
    → Do we update the old window? Ignore the event?
    → "Our analytics show 0 errors at noon, but users reported outage at noon"
    → Late events make analytics WRONG

  Scenario E — EXACTLY-ONCE PROCESSING FAILURE:
    → Event processed, counter incremented
    → Server crashes before acknowledging the event
    → Event redelivered → counter incremented AGAIN
    → "Revenue shows $200M but actual is $100M" (double-counted)
    → OR: Event acknowledged but not processed → LOST
    → "Revenue shows $80M but actual is $100M" (under-counted)

THE REAL CHALLENGE:
→ Process 10 MILLION events per second
→ Aggregate with sub-second latency
→ Handle late-arriving and out-of-order events correctly
→ Guarantee exactly-once processing semantics
→ Maintain state across windows spanning hours
→ Scale horizontally as load increases
→ Never lose events, never double-count

🔍 HOW TO APPROACH IT

Decompose into 7 DISTINCT sub-problems:

SUB-PROBLEM 1: EVENT INGESTION
  → Accept 10M events/sec without dropping any
  → Buffer in durable queue before processing
  → Partitioned for parallel consumption
  → Already learned queuing in Project 11 — apply at massive scale

SUB-PROBLEM 2: STREAM PROCESSING
  → Read events from queue
  → Apply transformations (filter, map, enrich)
  → Route to appropriate aggregation pipelines
  → Parallel processing across many workers

SUB-PROBLEM 3: WINDOWING
  → Group events into time-based windows
  → Tumbling window: fixed non-overlapping (every 1 minute)
  → Sliding window: overlapping (last 5 min, sliding every 10 sec)
  → Session window: per-user activity periods
  → Emit results when window closes

SUB-PROBLEM 4: AGGREGATION
  → Count, sum, average, percentile, top-K, distinct count
  → Must be INCREMENTAL (update as each event arrives)
  → Cannot re-scan all events in window (too slow)
  → Approximate algorithms for massive cardinality (HyperLogLog)

SUB-PROBLEM 5: LATE EVENT HANDLING
  → Events can arrive seconds to minutes late
  → Watermarks: "I believe all events before time T have arrived"
  → Allowed lateness: accept events up to X seconds late
  → Retraction: update previously emitted results

SUB-PROBLEM 6: STATE MANAGEMENT
  → Aggregations require state (running counts, sums)
  → State must survive worker failures
  → Checkpointing: periodically save state
  → Recovery: restore from checkpoint on failure

SUB-PROBLEM 7: OUTPUT & SERVING
  → Write aggregated results to dashboards
  → Trigger alerts when thresholds breached
  → Store historical aggregations for querying
  → CQRS: separate write path (events) from read path (dashboards)
  → Project 30 CQRS pattern applied at stream scale

SOLUTION CORE 1: PARTITIONED EVENT STREAM
═══════════════════════════════════════════════════════════

  WHY PARTITIONING:
    → 10M events/sec cannot be handled by one consumer
    → Split into N partitions (e.g., 100)
    → Each partition: 100K events/sec (manageable per worker)
    → Partition key determines which partition an event goes to

  PARTITION KEY SELECTION:
    → user_id: all events from same user go to same partition
      → Enables per-user session windows
      → Enables per-user aggregations
    → region: all events from same region together
      → Enables per-region aggregations
    → event_type: all clicks together, all purchases together
      → Enables per-type processing pipelines

  ORDERING GUARANTEE:
    → Events WITHIN a partition: strictly ordered
    → Events ACROSS partitions: no ordering guarantee
    → This is sufficient because:
      → Per-user analytics: user's events are in same partition
      → Global analytics: aggregated from all partitions

  DURABILITY:
    → Events written to disk before acknowledgment
    → Replicated across 3 brokers (Project 13 replication)
    → Event log retained for 7 days (replay capability)
    → Consumer tracks "offset" (how far it has read)

  This is exactly the Kafka model:
    Topic → Partitions → Consumer Groups

SOLUTION CORE 2: WINDOWING — THE KEY ABSTRACTION
═══════════════════════════════════════════════════════════

  THREE TYPES OF WINDOWS:

  TYPE 1: TUMBLING WINDOW (fixed, non-overlapping)
  ┌────────┐┌────────┐┌────────┐┌────────┐
  │ 0:00-  ││ 1:00-  ││ 2:00-  ││ 3:00-  │
  │ 0:59   ││ 1:59   ││ 2:59   ││ 3:59   │
  └────────┘└────────┘└────────┘└────────┘
  
  Use case: "Events per minute", "Revenue per hour"
  Each event belongs to EXACTLY ONE window
  Simple, non-overlapping, efficient

  TYPE 2: SLIDING WINDOW (overlapping)
  ┌──────────────────┐
  │    Window A: 0:00 - 0:04    │
  └──────────────────┘
     ┌──────────────────┐
     │    Window B: 0:01 - 0:05    │
     └──────────────────┘
        ┌──────────────────┐
        │    Window C: 0:02 - 0:06    │
        └──────────────────┘
  
  Use case: "Error rate in last 5 minutes, updated every minute"
  Each event belongs to MULTIPLE windows
  More compute, but provides rolling metrics

  TYPE 3: SESSION WINDOW (gap-based, per-key)
  User Alice:
  ┌──click──click──click──────┐  ┌──click──click──┐
  │     Session 1 (active)    │  │   Session 2    │
  └───────────────────────────┘  └────────────────┘
            30min gap →
  
  Use case: "Average session duration", "Actions per session"
  Window closes after N minutes of inactivity
  Each user has independent session windows

SOLUTION CORE 3: INCREMENTAL AGGREGATION
═══════════════════════════════════════════════════════════

  PROBLEM: Window contains 600M events (1 minute at 10M/sec)
  Cannot re-scan 600M events every time we need the count

  SOLUTION: Incremental aggregators — update state per event

  COUNT:
    state = 0
    on_event: state += 1
    on_window_close: emit state; reset to 0

  SUM:
    state = 0
    on_event: state += event.value
    on_window_close: emit state; reset to 0

  AVERAGE:
    state = {sum: 0, count: 0}
    on_event: sum += event.value; count += 1
    on_window_close: emit sum/count; reset

  MIN/MAX:
    state = {min: ∞, max: -∞}
    on_event: min = min(min, event.value); max = max(max, event.value)
    on_window_close: emit; reset

  PERCENTILE (approximate — T-Digest or DDSketch):
    → Cannot sort 600M values incrementally
    → Use approximate data structure
    → T-Digest: O(1) per insert, O(1) per query
    → Accuracy: within 1% for p50, p95, p99
    → Memory: ~10KB per digest

  DISTINCT COUNT (approximate — HyperLogLog):
    → "How many unique users in last hour?"
    → Cannot store 500M user IDs in memory
    → HyperLogLog: O(1) per insert, O(1) per query
    → Memory: ~12KB for <2% error on billions of uniques
    → Already seen in many production systems (Redis HLL)

  TOP-K (approximate — Count-Min Sketch + Heap):
    → "Top 10 trending products"
    → Count-Min Sketch: approximate frequency counter
    → Maintain min-heap of top-K items
    → Memory: O(K + sketch_size) regardless of input size

SOLUTION CORE 4: WATERMARKS & LATE EVENTS
═══════════════════════════════════════════════════════════

  THE PROBLEM:
    Event timestamp: 12:00:00 (when it happened)
    Processing time: 12:00:15 (when server received it)
    Difference: 15 seconds of "event time skew"

  WATERMARK:
    → A timestamp that says: "I believe all events with
      event_time ≤ watermark have been received"
    → As watermark advances, windows can be closed

    Time →
    Events:   e1(12:00) e2(12:01) e3(12:00) e4(12:02) e5(12:01)
    Watermark:  12:00     12:00     12:00     12:01     12:01
    
    Watermark = min(event_times in recent window) - allowed_lateness

  WHAT HAPPENS WHEN WATERMARK PASSES WINDOW END:
    → Window [12:00 - 12:01) closes
    → Emit aggregation result
    → Any event arriving after with timestamp in [12:00-12:01)
      is a "late event"

  HANDLING LATE EVENTS (3 strategies):
    Strategy A: DROP — ignore late events
      → Simple but loses data
      → OK if <0.1% of events are late

    Strategy B: UPDATE — recompute and re-emit
      → Late event arrives → update the closed window
      → Emit a "retraction" (previous result was wrong)
      → Then emit corrected result
      → Complex but accurate

    Strategy C: ALLOWED LATENESS — keep window open longer
      → Window end = 12:01, but allow lateness = 30 sec
      → Window actually closes at 12:01:30
      → Events arriving before 12:01:30 are included
      → After 12:01:30: treated as truly late (drop or update)

SOLUTION CORE 5: EXACTLY-ONCE PROCESSING
═══════════════════════════════════════════════════════════

  THREE SEMANTICS:
    AT-MOST-ONCE: Event might be lost (unacceptable for analytics)
    AT-LEAST-ONCE: Event processed 1+ times (double-counting!)
    EXACTLY-ONCE: Event processed exactly 1 time (what we want)

  HOW TO ACHIEVE EXACTLY-ONCE:
    → Checkpointing + Idempotent writes + Atomic commit

  CHECKPOINTING:
    → Periodically save: (partition_offset, aggregation_state)
    → "I've processed up to offset 42,891 and my count is 7,234"
    → On failure: restore from checkpoint, replay from offset 42,891
    → Events 0-42,891 are NOT reprocessed (already in state)

  THE CHECKPOINT DANCE:
    Normal operation:
      Read event → Process → Update state → Advance offset
      Read event → Process → Update state → Advance offset
      ...
    
    Checkpoint:
      PAUSE processing
      Save (offset=42891, state={count: 7234, sum: 892341})
      RESUME processing
    
    Crash + Recovery:
      Load checkpoint: offset=42891, state={count: 7234}
      Seek to offset 42892 in partition
      Resume processing from there
      State is exactly as it was at checkpoint ✅

  PROBLEM: What if crash happens between state update and offset advance?
    → State says count=7235 (processed event 42892)
    → But offset still says 42891 (didn't advance yet)
    → On recovery: replay event 42892 → count becomes 7236 (WRONG!)

  SOLUTION: Atomic checkpoint
    → Save state AND offset in ONE atomic transaction
    → Either both are saved, or neither
    → On recovery: state and offset are always consistent

SOLUTION CORE 6: LAMBDA vs KAPPA ARCHITECTURE
═══════════════════════════════════════════════════════════

  LAMBDA ARCHITECTURE (Batch + Speed layers):
  ┌─────────────────────────────────────────┐
  │  Events ──→ Batch Layer (hourly jobs)   │ → Accurate but delayed
  │         └──→ Speed Layer (stream)       │ → Fast but approximate
  │         Merge both for complete view     │
  └─────────────────────────────────────────┘
  
  PROS: Batch corrects stream errors, handles reprocessing
  CONS: Two separate systems to maintain, complex merge logic

  KAPPA ARCHITECTURE (Stream only):
  ┌─────────────────────────────────────────┐
  │  Events ──→ Stream Layer (only)         │ → Both fast AND accurate
  │  Reprocessing: replay from event log    │ → One system to maintain
  └─────────────────────────────────────────┘
  
  PROS: Single system, simpler, event log enables replay
  CONS: Stream must be as accurate as batch

  OUR DESIGN: KAPPA (Stream-first)
    → Single processing pipeline
    → Event log retained for replay
    → Reprocessing = replay from beginning with new logic
    → Simpler to build, maintain, and reason about

📌 WHAT COULD BE DONE

OPTION A: BATCH ONLY (MapReduce/Spark)
  → Collect events into files hourly
  → Process with Spark batch jobs
  → Latency: 1-24 hours
  → Simple but too slow for real-time

OPTION B: MICRO-BATCH (Spark Streaming)
  → Process events in small batches (0.5-2 seconds)
  → Good balance of throughput and latency
  → Latency: 0.5-5 seconds
  → Limited window semantics

OPTION C: TRUE STREAM (Flink / our approach)
  → Process each event individually
  → Rich windowing and state management
  → Latency: milliseconds to seconds
  → Most complex but most powerful

OPTION D: DATABASE-CENTRIC (TimescaleDB, ClickHouse)
  → Insert all events into analytics database
  → Query in real-time with SQL
  → Works up to ~1M events/sec
  → Limited window/aggregation flexibility

OUR DESIGN: OPTION C (True Stream) + Event Log for replay
  → Event-by-event processing
  → Rich windowing (tumbling, sliding, session)
  → Incremental aggregation
  → Checkpointed state
  → Exactly-once semantics
  → Retained event log for reprocessing

✅ WHAT SHOULD DO

1. USE EVENT TIME, NOT PROCESSING TIME
   → Event carries its own timestamp (when it happened)
   → Processing time varies due to network, queuing, load
   → Aggregating by processing time = inaccurate windows
   → Event time = what actually happened when

2. PARTITION BY KEY FOR PARALLELISM
   → Same key → same partition → same worker
   → Enables per-key state (session windows, per-user counts)
   → Scale by adding more partitions and workers

3. CHECKPOINT STATE REGULARLY
   → Every 30-60 seconds: save (offset, state)
   → On failure: lose at most 60 seconds of reprocessing
   → Checkpoint to durable storage (S3 — Project 42, or DB)

4. USE APPROXIMATE DATA STRUCTURES FOR HIGH CARDINALITY
   → Distinct count of 500M users: HyperLogLog (~12KB)
   → Top-K trending: Count-Min Sketch + Heap
   → Percentiles: T-Digest or DDSketch
   → Exact computation at this scale = impossible in real-time

5. IMPLEMENT WATERMARK-BASED WINDOW CLOSING
   → Don't close windows by wall clock
   → Close by watermark (tracks event time progress)
   → Handles out-of-order events naturally
   → Allowed lateness for stragglers

6. SEPARATE INGESTION FROM PROCESSING
   → Queue buffers bursts (10x spike absorbed)
   → Processors consume at their own pace
   → If processors fall behind: events wait in queue (not lost)
   → Back-pressure propagation when queue is full

7. EMIT INTERMEDIATE RESULTS FOR DASHBOARDS
   → Don't wait for window to close
   → Emit partial aggregation every N seconds
   → Dashboard shows "current window so far"
   → Mark as "partial" vs "final" result

8. RETAIN EVENT LOG FOR REPROCESSING
   → Keep raw events for 7-30 days
   → Bug in aggregation logic? Replay with fixed code
   → New metric needed? Replay historical events
   → This is the KAPPA architecture advantage

❌ WHAT SHOULD NOT DO

1. NEVER USE PROCESSING TIME FOR BUSINESS METRICS
   → "Revenue at 12:00" should mean "purchases that happened at 12:00"
   → NOT "purchases that our server processed at 12:00"
   → Processing time metrics are only for system health monitoring

2. NEVER KEEP UNBOUNDED STATE
   → "Count of all events ever" grows forever
   → State must be bounded by windows
   → Old window state must be evicted
   → Memory leak = eventual OOM crash

3. NEVER ASSUME EVENTS ARRIVE IN ORDER
   → Network reordering, retries, multi-path delivery
   → ALWAYS sort by event time within processing
   → ALWAYS handle late arrivals gracefully

4. NEVER SKIP CHECKPOINTING TO IMPROVE PERFORMANCE
   → "Checkpointing slows us down by 5%"
   → Crash without checkpoint → lose hours of state
   → 5% performance cost is NOTHING compared to data loss

5. NEVER PROCESS THE SAME EVENT TWICE WITHOUT IDEMPOTENCY
   → At-least-once delivery means duplicates WILL arrive
   → Aggregations must handle duplicates
   → Use dedup based on event_id
   → Or use idempotent operations (max, min are naturally idempotent)

6. NEVER BLOCK ON SLOW DOWNSTREAM
   → Dashboard database is slow → don't block processing
   → Buffer output writes
   → Processing must continue regardless of output speed
   → Otherwise: back-pressure cascades to ingestion → events dropped

7. NEVER MIX WINDOW TYPES WITHOUT UNDERSTANDING COST
   → Tumbling window: 1 copy of each event
   → Sliding window (5 min, slide 10 sec): 30 copies of each event!
     (5min / 10sec = 30 overlapping windows)
   → Memory and CPU cost: 30× for sliding vs tumbling
   → Use tumbling where possible, sliding only when needed

8. NEVER EMIT FINAL RESULTS BEFORE WATERMARK PASSES
   → "The minute is over" by wall clock ≠ "all events arrived"
   → Wait for watermark to pass window end
   → Then wait for allowed lateness
   → THEN emit final result

MECHANICS 1: EVENT FLOW — END TO END
═══════════════════════════════════════════════════════════

  EVENT LIFECYCLE:

  T+0ms: EVENT CREATED
    → User clicks "Buy" button on phone
    → App creates event:
      {
        "event_id": "evt_abc123",
        "event_type": "purchase",
        "user_id": "user_42",
        "product_id": "prod_789",
        "amount": 49.99,
        "region": "us-west",
        "event_time": "2025-01-20T12:00:00.123Z",
        "device": "mobile"
      }

  T+50ms: EVENT SENT TO INGESTION
    → App sends event over HTTPS/gRPC to ingestion endpoint
    → Load balanced across ingestion servers

  T+55ms: INGESTION WRITES TO PARTITION
    → Partition key = hash(user_id) % num_partitions
    → Event appended to partition log (durable write)
    → Offset assigned: partition_7, offset_42891

  T+60ms: CONSUMER READS EVENT
    → Stream processor worker assigned to partition_7
    → Reads event at offset 42891
    → Deserializes event

  T+61ms: EVENT PROCESSING
    → Parse event_time (12:00:00.123)
    → Determine which windows this event belongs to:
      Tumbling 1-min: [12:00 - 12:01)
      Sliding 5-min: [11:56-12:01), [11:57-12:02), ... 5 windows
    → For each window: update aggregation state

  T+62ms: STATE UPDATE
    → Window [12:00 - 12:01) state:
      revenue_counter: 49.99 added → new total: $12,482.31
      purchase_count: 1 added → new total: 247
      unique_users: HLL.add("user_42")
      top_products: CMS.add("prod_789")

  T+65ms: WATERMARK ADVANCE
    → Watermark for this partition: max(event_times) - lateness
    → New watermark: 12:00:00.123 - 10sec = 11:59:50.123
    → Window [11:59 - 12:00) not yet closeable (watermark < 12:00)

  T+70ms: EMIT PARTIAL RESULT (for dashboard)
    → Window [12:00 - 12:01) partial:

MECHANICS 3: HYPERLOGLOG — COUNT DISTINCT MAGIC
═══════════════════════════════════════════════════════════

  PROBLEM: Count unique users in 1-minute window
    → 500M potential users
    → Exact: HashSet of user_ids → 500M × 20 bytes = 10GB RAM!
    → Per window! Multiple windows = 100s of GB

  HYPERLOGLOG SOLUTION:
    → Uses only 12 KB of memory
    → Estimates cardinality within ~2% error
    → How?

  THE INSIGHT:
    → Hash each user_id to a uniform random number
    → Count the longest run of leading zeros in the hash
    → If you see a run of N leading zeros:
      → Probability of that happening: 1/2^N
      → So you've probably seen ~2^N unique items

  SIMPLIFIED EXAMPLE:
    user_42 → hash → 0001011010... (3 leading zeros)
    user_99 → hash → 0000001101... (6 leading zeros)
    user_7  → hash → 0100110100... (1 leading zero)

    Max leading zeros seen: 6
    Estimated unique count: 2^6 = 64

    (Actual HLL uses 2^14 = 16384 registers for better accuracy)

  OPERATIONS:
    HLL.add(user_id):
      h = hash(user_id)
      register_index = first 14 bits of h
      leading_zeros = count_leading_zeros(remaining bits)
      registers[register_index] = max(registers[register_index], leading_zeros)

    HLL.count():
      harmonic_mean = 1 / (Σ 2^(-registers[i]))
      return alpha × m² × harmonic_mean
      (alpha is a bias correction constant, m = number of registers)

  MERGE (combine two HLLs):
    For each register: merged[i] = max(hll_a[i], hll_b[i])
    → Can merge HLLs from different partitions!
    → Global unique count = merge all partition HLLs

class Partition:
    """Single ordered partition of the event stream"""

    def __init__(self, partition_id: int):
        self.partition_id = partition_id
        self.events: List[Event] = []
        self.next_offset = 0
        self._lock = threading.Lock()

    def append(self, event: Event) -> int:
        with self._lock:
            offset = self.next_offset
            event.partition = self.partition_id
            self.events.append(event)
            self.next_offset += 1
            return offset

    def read(self, offset: int, max_events: int = 100) -> List[Tuple[int, Event]]:
        with self._lock:
            end = min(offset + max_events, len(self.events))
            return [
                (i, self.events[i])
                for i in range(offset, end)
            ]

    def latest_offset(self) -> int:
        return self.next_offset


class EventStream:
    """Partitioned event stream (simplified Kafka topic)"""

    def __init__(self, num_partitions: int = 10):
        self.num_partitions = num_partitions
        self.partitions = [
            Partition(i) for i in range(num_partitions)
        ]
        self.total_events = 0

    def publish(self, event: Event, partition_key: str = None) -> Tuple[int, int]:
        """Publish event to appropriate partition"""
        if partition_key:
            part_id = hash(partition_key) % self.num_partitions
        else:
            part_id = random.randint(0, self.num_partitions - 1)

        event.processing_time = time.time()
        offset = self.partitions[part_id].append(event)
        self.total_events += 1
        return part_id, offset

    def consume(
        self, partition_id: int, offset: int, max_events: int = 100
    ) -> List[Tuple[int, Event]]:
        return self.partitions[partition_id].read(offset, max_events)

class HyperLogLog:
    """
    Probabilistic cardinality estimator.
    
    Memory: ~12KB (2^14 = 16384 registers × 6 bits each)
    Accuracy: standard error ~1.6%
    
    Can count billions of unique items with 12KB of memory.
    """

    def __init__(self, precision: int = 14):
        self.p = precision
        self.m = 1 << precision  # Number of registers (2^p)
        self.registers = [0] * self.m
        self.alpha = self._get_alpha(self.m)

    @staticmethod
    def _get_alpha(m: int) -> float:
        if m == 16:
            return 0.673
        elif m == 32:
            return 0.697
        elif m == 64:
            return 0.709
        else:
            return 0.7213 / (1 + 1.079 / m)

    @staticmethod
    def _hash(value: str) -> int:
        return int(hashlib.md5(value.encode()).hexdigest(), 16)

    def add(self, value: str):
        h = self._hash(value)
        # First p bits determine register index
        register_idx = h & (self.m - 1)
        # Remaining bits: count leading zeros + 1
        remaining = h >> self.p
        run_length = 1
        while remaining & 1 == 0 and run_length <= 64 - self.p:
            run_length += 1
            remaining >>= 1
        self.registers[register_idx] = max(
            self.registers[register_idx], run_length
        )

    def count(self) -> int:
        # Harmonic mean estimator
        indicator = sum(2.0 ** (-r) for r in self.registers)
        estimate = self.alpha * self.m * self.m / indicator

        # Small range correction
        if estimate <= 2.5 * self.m:
            zeros = self.registers.count(0)
            if zeros > 0:
                estimate = self.m * math.log(self.m / zeros)

        return int(estimate)

    def merge(self, other: 'HyperLogLog'):
        """Merge another HLL into this one (for combining partitions)"""
        for i in range(self.m):
            self.registers[i] = max(self.registers[i], other.registers[i])

    def memory_bytes(self) -> int:
        return self.m  # 1 byte per register (could be 6 bits)

class AggregationState:
    """
    Maintains running aggregations for a window.
    All updates are O(1) — incremental, not re-scan.
    """

    def __init__(self):
        self.count = 0
        self.sum_value = 0.0
        self.min_value = float('inf')
        self.max_value = float('-inf')
        self.hll = HyperLogLog(precision=10)  # Smaller for demo
        self.event_type_counts = defaultdict(int)
        self.region_sums = defaultdict(float)
        self.first_event_time = None
        self.last_event_time = None

    def update(self, event: Event):
        """O(1) incremental update per event"""
        self.count += 1
        self.sum_value += event.value
        self.min_value = min(self.min_value, event.value)
        self.max_value = max(self.max_value, event.value)

        if event.user_id:
            self.hll.add(event.user_id)

        self.event_type_counts[event.event_type] += 1
        if event.region:
            self.region_sums[event.region] += event.value

        if self.first_event_time is None:
            self.first_event_time = event.event_time
        self.last_event_time = event.event_time

    @property
    def average(self) -> float:
        return self.sum_value / max(self.count, 1)

    @property
    def unique_users(self) -> int:
        return self.hll.count()

    def to_dict(self) -> dict:
        return {
            "count": self.count,
            "sum": round(self.sum_value, 2),
            "average": round(self.average, 2),
            "min": round(self.min_value, 2) if self.min_value != float('inf') else 0,
            "max": round(self.max_value, 2) if self.max_value != float('-inf') else 0,
            "unique_users": self.unique_users,
            "event_types": dict(self.event_type_counts),
            "revenue_by_region": {
                k: round(v, 2) for k, v in self.region_sums.items()
            },
        }

    def serialize(self) -> dict:
        """For checkpointing"""
        return {
            "count": self.count,
            "sum_value": self.sum_value,
            "min_value": self.min_value,
            "max_value": self.max_value,
            "hll_registers": list(self.hll.registers),
            "event_type_counts": dict(self.event_type_counts),
            "region_sums": dict(self.region_sums),
        }

# Manages window lifecycle: creation, assignment, closing

class WindowManager:
    """
    Manages windows for a specific aggregation pipeline.
    Handles tumbling and sliding windows.
    """

    def __init__(self, spec: WindowSpec):
        self.spec = spec
        self.windows: Dict[str, Window] = {}
        self.states: Dict[str, AggregationState] = {}
        self.closed_results: List[dict] = []
        self.late_events = 0

    def assign_windows(self, event_time: float) -> List[Window]:
        """Determine which windows an event belongs to"""
        assigned = []

        if self.spec.window_type == WindowType.TUMBLING:
            window_start = (
                event_time // self.spec.size_seconds
            ) * self.spec.size_seconds
            window_end = window_start + self.spec.size_seconds
            window = Window(start=window_start, end=window_end)
            assigned.append(window)

        elif self.spec.window_type == WindowType.SLIDING:
            slide = self.spec.slide_seconds
            size = self.spec.size_seconds
            earliest_start = (
                (event_time - size) // slide + 1
            ) * slide
            earliest_start = max(0, earliest_start)

            current_start = earliest_start
            while current_start <= event_time:
                window_end = current_start + size
                if current_start <= event_time < window_end:
                    window = Window(start=current_start, end=window_end)
                    assigned.append(window)
                current_start += slide

        return assigned

    def process_event(self, event: Event, watermark: float) -> List[dict]:
        """
        Process an event: assign to windows, update state.
        Returns any results emitted (closed windows).
        """
        emitted = []

        windows = self.assign_windows(event.event_time)

        for window in windows:
            key = window.key

            if key in self.windows and self.windows[key].status == "FINAL":
                self.late_events += 1
                continue

            if key not in self.windows:
                self.windows[key] = window
                self.states[key] = AggregationState()

            self.states[key].update(event)

        # Check if any windows should close based on watermark
        for key in list(self.windows.keys()):
            window = self.windows[key]
            if window.status == "OPEN" and watermark >= window.end:
                window.status = "FINAL"
                state = self.states[key]
                result = {
                    "window": key,
                    "window_start": window.start,
                    "window_end": window.end,
                    "status": "FINAL",
                    "aggregation": state.to_dict(),
                }
                self.closed_results.append(result)
                emitted.append(result)

        return emitted

    def get_partial_results(self) -> List[dict]:
        """Get current state of open windows (for live dashboards)"""
        results = []
        for key, window in self.windows.items():
            if window.status == "OPEN":
                state = self.states[key]
                results.append({
                    "window": key,
                    "window_start": window.start,
                    "window_end": window.end,
                    "status": "PARTIAL",
                    "aggregation": state.to_dict(),
                })
        return results

    def evict_old_windows(self, watermark: float, keep_seconds: float = 300):
        """Remove state for windows that are far behind watermark"""
        evict_threshold = watermark - keep_seconds
        evicted = 0
        for key in list(self.windows.keys()):
            window = self.windows[key]
            if window.end < evict_threshold and window.status == "FINAL":
                del self.windows[key]
                del self.states[key]
                evicted += 1
        return evicted

    def get_stats(self) -> dict:
        open_windows = sum(
            1 for w in self.windows.values() if w.status == "OPEN"
        )
        closed_windows = sum(
            1 for w in self.windows.values() if w.status == "FINAL"
        )
        return {
            "window_spec": str(self.spec),
            "open_windows": open_windows,
            "closed_windows": closed_windows,
            "total_results_emitted": len(self.closed_results),
            "late_events": self.late_events,
            "state_memory_entries": len(self.states),
        }

class SessionWindowManager:
    """
    Session windows are per-key (per-user) and gap-based.
    
    A session starts when a user's first event arrives.
    It stays open as long as events keep arriving within the gap.
    When no event arrives for gap_seconds, the session closes.
    """

    def __init__(self, gap_seconds: float = 30.0):
        self.gap_seconds = gap_seconds
        # Per-user session state
        self.sessions: Dict[str, dict] = {}
        self.closed_sessions: List[dict] = []

    def process_event(self, event: Event, current_time: float) -> List[dict]:
        """Process event for session window tracking"""
        user_id = event.user_id
        emitted = []

        if user_id in self.sessions:
            session = self.sessions[user_id]
            # Check if session expired (gap exceeded)
            if event.event_time - session["last_event_time"] > self.gap_seconds:
                # Close old session
                closed = self._close_session(user_id)
                emitted.append(closed)
                # Start new session
                self._start_session(user_id, event)
            else:
                # Continue existing session
                session["last_event_time"] = event.event_time
                session["event_count"] += 1
                session["total_value"] += event.value
                session["state"].update(event)
        else:
            # New session
            self._start_session(user_id, event)

        return emitted

    def _start_session(self, user_id: str, event: Event):
        state = AggregationState()
        state.update(event)
        self.sessions[user_id] = {
            "user_id": user_id,
            "start_time": event.event_time,
            "last_event_time": event.event_time,
            "event_count": 1,
            "total_value": event.value,
            "state": state,
        }

    def _close_session(self, user_id: str) -> dict:
        session = self.sessions.pop(user_id)
        duration = session["last_event_time"] - session["start_time"]
        result = {
            "type": "session_closed",
            "user_id": user_id,
            "start_time": session["start_time"],
            "end_time": session["last_event_time"],
            "duration_seconds": round(duration, 2),
            "event_count": session["event_count"],
            "total_value": round(session["total_value"], 2),
            "aggregation": session["state"].to_dict(),
        }
        self.closed_sessions.append(result)
        return result

    def expire_idle_sessions(self, current_time: float) -> List[dict]:
        """Close sessions that have been idle beyond the gap"""
        expired = []
        for user_id in list(self.sessions.keys()):
            session = self.sessions[user_id]
            if current_time - session["last_event_time"] > self.gap_seconds:
                closed = self._close_session(user_id)
                expired.append(closed)
        return expired

    def get_stats(self) -> dict:
        return {
            "active_sessions": len(self.sessions),
            "closed_sessions": len(self.closed_sessions),
            "gap_seconds": self.gap_seconds,
        }

class AlertSeverity(Enum):
    INFO = "INFO"
    WARNING = "WARNING"
    CRITICAL = "CRITICAL"


@dataclass
class AlertRule:
    rule_id: str
    name: str
    metric: str                # "count", "average", "error_rate", etc.
    operator: str              # "gt", "lt", "gte", "lte", "eq"
    threshold: float
    severity: AlertSeverity
    event_type_filter: str = ""  # Only check specific event types
    suppress_seconds: float = 60  # Don't re-alert within this period


@dataclass
class Alert:
    alert_id: str
    rule_id: str
    rule_name: str
    severity: str
    metric_value: float
    threshold: float
    window: str
    triggered_at: float
    message: str


class AlertEngine:
    """
    Evaluates alert rules against window aggregation results.
    Supports suppression to avoid alert storms.
    """

    def __init__(self):
        self.rules: List[AlertRule] = []
        self.fired_alerts: List[Alert] = []
        self.last_alert_time: Dict[str, float] = {}

    def add_rule(self, rule: AlertRule):
        self.rules.append(rule)

    def evaluate(self, window_result: dict) -> List[Alert]:
        """Evaluate all rules against a window result"""
        new_alerts = []
        aggregation = window_result.get("aggregation", {})
        now = time.time()

        for rule in self.rules:
            # Check suppression
            last_fired = self.last_alert_time.get(rule.rule_id, 0)
            if now - last_fired < rule.suppress_seconds:
                continue

            # Get metric value
            metric_value = self._extract_metric(aggregation, rule)
            if metric_value is None:
                continue

            # Check threshold
            triggered = self._check_threshold(
                metric_value, rule.operator, rule.threshold
            )

            if triggered:
                alert = Alert(
                    alert_id=f"alert_{uuid.uuid4().hex[:8]}",
                    rule_id=rule.rule_id,
                    rule_name=rule.name,
                    severity=rule.severity.value,
                    metric_value=round(metric_value, 4),
                    threshold=rule.threshold,
                    window=window_result.get("window", ""),
                    triggered_at=now,
                    message=(
                        f"{rule.name}: {rule.metric}={metric_value:.2f} "
                        f"{rule.operator} {rule.threshold}"
                    ),
                )
                new_alerts.append(alert)
                self.fired_alerts.append(alert)
                self.last_alert_time[rule.rule_id] = now

        return new_alerts

    def _extract_metric(self, aggregation: dict, rule: AlertRule) -> Optional[float]:
        """Extract metric value from aggregation result"""
        if rule.metric == "count":
            return aggregation.get("count", 0)
        elif rule.metric == "average":
            return aggregation.get("average", 0)
        elif rule.metric == "sum":
            return aggregation.get("sum", 0)
        elif rule.metric == "max":
            return aggregation.get("max", 0)
        elif rule.metric == "unique_users":
            return aggregation.get("unique_users", 0)
        elif rule.metric == "error_rate":
            event_types = aggregation.get("event_types", {})
            total = aggregation.get("count", 0)
            errors = event_types.get("error", 0)
            return (errors / max(total, 1)) * 100
        return None

    @staticmethod
    def _check_threshold(value: float, operator: str, threshold: float) -> bool:
        if operator == "gt":
            return value > threshold
        elif operator == "lt":
            return value < threshold
        elif operator == "gte":
            return value >= threshold
        elif operator == "lte":
            return value <= threshold
        elif operator == "eq":
            return abs(value - threshold) < 0.001
        return False

    def get_stats(self) -> dict:
        return {
            "rules_configured": len(self.rules),
            "total_alerts_fired": len(self.fired_alerts),
            "alerts_by_severity": {
                severity.value: sum(
                    1 for a in self.fired_alerts
                    if a.severity == severity.value
                )
                for severity in AlertSeverity
            },
        }

class CheckpointManager:
    """
    Manages checkpointing of stream processing state.
    
    Saves: partition offsets + window states
    Restores: on worker failure, load latest checkpoint
    """

    def __init__(self):
        self.checkpoints: List[dict] = []
        self.checkpoint_count = 0

    def create_checkpoint(
        self,
        partition_offsets: Dict[int, int],
        window_manager: WindowManager,
        watermark: float,
    ) -> dict:
        """Save current state as a checkpoint"""
        # Serialize window states
        window_states = {}
        for key, state in window_manager.states.items():
            window_states[key] = state.serialize()

        checkpoint = {
            "checkpoint_id": f"chk_{self.checkpoint_count:06d}",
            "timestamp": time.time(),
            "partition_offsets": dict(partition_offsets),
            "watermark": watermark,
            "window_states": window_states,
            "open_window_keys": [
                key for key, w in window_manager.windows.items()
                if w.status == "OPEN"
            ],
            "stats": {
                "open_windows": sum(
                    1 for w in window_manager.windows.values()
                    if w.status == "OPEN"
                ),
                "state_entries": len(window_states),
            },
        }

        self.checkpoints.append(checkpoint)
        self.checkpoint_count += 1

        # Keep only last 5 checkpoints
        if len(self.checkpoints) > 5:
            self.checkpoints = self.checkpoints[-5:]

        return checkpoint

    def get_latest_checkpoint(self) -> Optional[dict]:
        if self.checkpoints:
            return self.checkpoints[-1]
        return None

    def get_stats(self) -> dict:
        latest = self.get_latest_checkpoint()
        return {
            "total_checkpoints": self.checkpoint_count,
            "stored_checkpoints": len(self.checkpoints),
            "latest_checkpoint_id": latest["checkpoint_id"] if latest else None,
            "latest_checkpoint_age_sec": round(
                time.time() - latest["timestamp"], 1
            ) if latest else None,
        }

class StreamProcessor:
    """
    Complete stream processing engine.
    
    Reads from partitioned event stream,
    processes through windows,
    manages watermarks,
    evaluates alerts,
    emits results to output.
    """

    def __init__(
        self,
        event_stream: EventStream,
        partition_ids: List[int],
        window_specs: List[WindowSpec] = None,
        allowed_lateness_seconds: float = 10.0,
        checkpoint_interval_events: int = 1000,
    ):
        self.stream = event_stream
        self.partition_ids = partition_ids
        self.allowed_lateness = allowed_lateness_seconds
        self.checkpoint_interval = checkpoint_interval_events

        # Window managers (one per window spec)
        self.window_managers: Dict[str, WindowManager] = {}
        if window_specs:
            for i, spec in enumerate(window_specs):
                name = f"pipeline_{i}_{spec.window_type.value.lower()}"
                self.window_managers[name] = WindowManager(spec)

        # Session window manager
        self.session_manager = SessionWindowManager(gap_seconds=30.0)

        # Alert engine
        self.alert_engine = AlertEngine()

        # Checkpoint manager
        self.checkpoint_mgr = CheckpointManager()

        # Partition offsets (how far we've read)
        self.offsets: Dict[int, int] = {pid: 0 for pid in partition_ids}

        # Watermark tracking
        self.watermark = 0.0
        self.partition_watermarks: Dict[int, float] = {
            pid: 0.0 for pid in partition_ids
        }

        # Metrics
        self.events_processed = 0
        self.events_late = 0
        self.results_emitted = 0
        self.alerts_fired = 0
        self.output_buffer: List[dict] = []

    def process_batch(self, max_events_per_partition: int = 100) -> dict:
        """
        Process a batch of events from all assigned partitions.
        Returns processing summary.
        """
        batch_start = time.time()
        batch_events = 0
        batch_results = []
        batch_alerts = []

        for pid in self.partition_ids:
            events = self.stream.consume(
                pid, self.offsets[pid], max_events_per_partition
            )

            for offset, event in events:
                # Update partition watermark
                self.partition_watermarks[pid] = max(
                    self.partition_watermarks[pid],
                    event.event_time
                )

                # Process through all window managers
                for name, wm in self.window_managers.items():
                    emitted = wm.process_event(event, self.watermark)
                    for result in emitted:
                        result["pipeline"] = name
                        batch_results.append(result)

                        # Evaluate alerts on closed windows
                        alerts = self.alert_engine.evaluate(result)
                        batch_alerts.extend(alerts)

                # Process session windows
                session_results = self.session_manager.process_event(
                    event, event.event_time
                )
                batch_results.extend(session_results)

                batch_events += 1
                self.events_processed += 1
                self.offsets[pid] = offset + 1

        # Update global watermark (minimum across all partitions)
        if self.partition_watermarks:
            active_watermarks = [
                wm for wm in self.partition_watermarks.values() if wm > 0
            ]
            if active_watermarks:
                self.watermark = min(active_watermarks) - self.allowed_lateness

        # Close windows based on updated watermark
        for name, wm in self.window_managers.items():
            emitted = wm.process_event(
                Event(
                    event_id="watermark_advance",
                    event_type="system",
                    event_time=self.watermark,
                ),
                self.watermark,
            )
            for result in emitted:
                result["pipeline"] = name
                batch_results.append(result)
                alerts = self.alert_engine.evaluate(result)
                batch_alerts.extend(alerts)

        # Expire idle sessions
        expired_sessions = self.session_manager.expire_idle_sessions(
            self.watermark
        )
        batch_results.extend(expired_sessions)

        # Evict old window state
        for wm in self.window_managers.values():
            wm.evict_old_windows(self.watermark)

        # Store results in output buffer
        self.output_buffer.extend(batch_results)
        self.results_emitted += len(batch_results)
        self.alerts_fired += len(batch_alerts)

        # Checkpoint if needed
        checkpoint = None
        if self.events_processed % self.checkpoint_interval == 0 and self.events_processed > 0:
            # Use first window manager for checkpointing demo
            if self.window_managers:
                first_wm = list(self.window_managers.values())[0]
                checkpoint = self.checkpoint_mgr.create_checkpoint(
                    self.offsets, first_wm, self.watermark
                )

        elapsed = (time.time() - batch_start) * 1000

        return {
            "events_processed": batch_events,
            "results_emitted": len(batch_results),
            "alerts_fired": len(batch_alerts),
            "alerts": batch_alerts,
            "watermark": self.watermark,
            "checkpoint": checkpoint,
            "elapsed_ms": round(elapsed, 3),
        }

    def get_live_dashboard(self) -> dict:
        """Get current partial results for live dashboard"""
        dashboard = {
            "timestamp": time.time(),
            "watermark": self.watermark,
            "events_processed_total": self.events_processed,
            "pipelines": {},
        }

        for name, wm in self.window_managers.items():
            partial = wm.get_partial_results()
            if partial:
                # Show the most recent open window
                latest = max(partial, key=lambda p: p["window_start"])
                dashboard["pipelines"][name] = latest

        # Session stats
        dashboard["sessions"] = self.session_manager.get_stats()

        return dashboard

    def get_stats(self) -> dict:
        pipeline_stats = {}
        for name, wm in self.window_managers.items():
            pipeline_stats[name] = wm.get_stats()

        return {
            "events_processed": self.events_processed,
            "results_emitted": self.results_emitted,
            "alerts_fired": self.alerts_fired,
            "events_late": sum(
                wm.late_events for wm in self.window_managers.values()
            ),
            "watermark": self.watermark,
            "partition_offsets": dict(self.offsets),
            "pipelines": pipeline_stats,
            "sessions": self.session_manager.get_stats(),
            "alerts": self.alert_engine.get_stats(),
            "checkpoints": self.checkpoint_mgr.get_stats(),
        }

class EventGenerator:
    """Generates realistic event streams for testing"""

    REGIONS = ["us-east", "us-west", "eu-west", "eu-east", "ap-south", "ap-east"]
    EVENT_TYPES = ["page_view", "click", "purchase", "error", "signup", "search"]

    def __init__(self, num_users: int = 1000, base_time: float = None):
        self.num_users = num_users
        self.base_time = base_time or time.time()
        self.event_counter = 0

    def generate_event(self, time_offset: float = 0, jitter: float = 2.0) -> Event:
        """Generate a single realistic event"""
        self.event_counter += 1

        # Event time with jitter (simulates real-world clock differences)
        event_time = self.base_time + time_offset + random.uniform(-jitter, jitter)

        # Random event type (weighted: more page_views than errors)
        weights = [40, 25, 10, 5, 5, 15]
        event_type = random.choices(self.EVENT_TYPES, weights=weights, k=1)[0]

        # Value depends on event type
        if event_type == "purchase":
            value = random.uniform(5.0, 500.0)
        elif event_type == "page_view":
            value = random.uniform(0.1, 5.0)  # Page load time in seconds
        elif event_type == "error":
            value = 1.0  # Error count
        else:
            value = random.uniform(0.0, 1.0)

        return Event(
            event_id=f"evt_{self.event_counter:08d}",
            event_type=event_type,
            event_time=event_time,
            user_id=f"user_{random.randint(1, self.num_users):05d}",
            region=random.choice(self.REGIONS),
            value=value,
        )

    def generate_batch(
        self, count: int, time_offset: float = 0, time_spread: float = 60
    ) -> List[Event]:
        """Generate a batch of events spread over a time period"""
        events = []
        for i in range(count):
            offset = time_offset + (time_spread * i / count)
            event = self.generate_event(time_offset=offset, jitter=1.0)
            events.append(event)
        return events

    def generate_late_events(
        self, count: int, time_offset: float, delay_seconds: float = 15
    ) -> List[Event]:
        """Generate events that arrive late (event_time is old)"""
        events = []
        for _ in range(count):
            event = self.generate_event(
                time_offset=time_offset - delay_seconds, jitter=0.5
            )
            events.append(event)
        return events
