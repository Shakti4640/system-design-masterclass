🎯 HYPOTHETICAL PROBLEM

You are building a social network like LinkedIn, Facebook, or Twitter

The situation:
→ 500 million users
→ Average user has 300 connections (friends/followers)
→ Total edges (relationships) in the graph: ~75 BILLION
→ Every user who opens the app expects to see:
    "People You May Know" — 5 suggestions in <200ms
    "Mutual Friends" with any profile — count in <100ms
    "Recommended Content" — based on what their network liked
    "Shortest Path" — "How are you connected to Elon Musk?"
    "Trending in Your Network" — what your 2nd-degree connections share

WHAT GOES WRONG without proper design:

  Scenario A — RELATIONAL DATABASE MELTDOWN:
    → Store friendships in SQL: CREATE TABLE friends (user_a, user_b)
    → "Friends of friends" query:
        SELECT DISTINCT f2.user_b 
        FROM friends f1 
        JOIN friends f2 ON f1.user_b = f2.user_a
        WHERE f1.user_a = 'alice'
          AND f2.user_b NOT IN (SELECT user_b FROM friends WHERE user_a = 'alice')
    → Alice has 500 friends, each has 500 friends
    → Join produces 250,000 rows
    → Filter existing friends: another 500-row subquery
    → Index scan on each: ~1ms per lookup
    → Total: 250ms+ for a single "People You May Know"
    → With 10,000 concurrent users: DATABASE DIES

  Scenario B — MEMORY EXPLOSION:
    → Load entire graph into memory for fast traversal
    → 500M users × 300 edges × 16 bytes per edge = ~2.4 TB
    → Won't fit in a single machine's RAM
    → Need distributed graph processing

  Scenario C — FAN-OUT EXPLOSION:
    → "What are friends of friends of friends sharing?"
    → 3-hop traversal: 300 × 300 × 300 = 27 MILLION nodes
    → Even at 1μs per node = 27 seconds
    → Users won't wait 27 seconds

  Scenario D — STALE RECOMMENDATIONS:
    → Precompute "People You May Know" once per day
    → User adds 10 new friends during the day
    → Recommendations still suggest those 10 people
    → "Why is it recommending someone I already connected with?"
    → Need near-real-time updates

THE REAL CHALLENGE:
→ How do you traverse a 75-billion-edge graph
→ In under 200ms per query
→ For 10,000 concurrent users
→ While the graph changes constantly (new follows, unfollows)
→ And produce RELEVANT recommendations, not random connections

🔍 HOW TO APPROACH IT

Decompose into 6 DISTINCT sub-problems:

SUB-PROBLEM 1: GRAPH STORAGE
  → How to store billions of nodes and edges efficiently
  → Adjacency list vs adjacency matrix vs graph database
  → Partitioning the graph across machines (graph sharding)

SUB-PROBLEM 2: GRAPH TRAVERSAL
  → BFS (Breadth-First Search) for shortest path, mutual friends
  → DFS (Depth-First Search) for connected components
  → Bidirectional BFS for "degrees of separation"
  → Limiting traversal depth to control explosion

SUB-PROBLEM 3: FRIEND RECOMMENDATIONS ("People You May Know")
  → Friends-of-friends with ranking
  → Common neighbors scoring
  → Jaccard similarity between connection sets
  → Adamic-Adar index (weight rare mutual friends higher)

SUB-PROBLEM 4: CONTENT RECOMMENDATIONS
  → Collaborative filtering: "Users like you liked X"
  → Bipartite graph: Users ↔ Content
  → Random walk with restart (PersonalizedPageRank)
  → Combine social graph signals with content signals

SUB-PROBLEM 5: REAL-TIME GRAPH UPDATES
  → New follow/unfollow: update adjacency list
  → Invalidate cached recommendations
  → Incremental re-computation vs full recompute
  → Event-driven updates (Project 29 concepts)

SUB-PROBLEM 6: DISTRIBUTED GRAPH PROCESSING
  → Graph partitioning: minimize cross-partition edges
  → Scatter-gather query pattern
  → Precomputation for common queries
  → Caching hot subgraphs (Project 5 concepts)

SOLUTION LAYER 1: GRAPH STORAGE — ADJACENCY LIST
═══════════════════════════════════════════════════════

WHY NOT adjacency matrix:
  → 500M × 500M matrix = 250 PETABYTES (even as bitmap)
  → 99.99994% of entries are 0 (sparse graph)
  → Waste of space

WHY adjacency list:
  → Store only actual connections
  → 500M users × 300 avg connections × 8 bytes = ~1.2 TB
  → Fits across a cluster of machines
  → O(1) to check if edge exists (with hash set per node)
  → O(degree) to get all neighbors

STORAGE FORMAT PER USER:
  user_id: "alice"
  neighbors: {"bob", "charlie", "dave", ...}  ← HashSet
  metadata: {
    "follower_count": 542,

    "created_at": "2020-01-15",
    "last_active": "2025-01-20"
  }
  edge_metadata: {
    "bob": {"since": "2021-03-10", "type": "mutual"},
    "charlie": {"since": "2022-07-22", "type": "following"}
  }

SOLUTION LAYER 2: FRIEND RECOMMENDATIONS (PYMK)
═══════════════════════════════════════════════════════

ALGORITHM: FRIENDS-OF-FRIENDS WITH SCORING

  For user Alice:
  1. Get Alice's friends: {Bob, Charlie, Dave}
  2. For each friend, get THEIR friends:
     Bob's friends:     {Alice, Eve, Frank, Grace}
     Charlie's friends: {Alice, Eve, Hank}
     Dave's friends:    {Alice, Frank, Iris}
  3. Count how many times each non-friend appears:
     Eve:   2 (through Bob AND Charlie)
     Frank: 2 (through Bob AND Dave)
     Grace: 1 (through Bob only)
     Hank:  1 (through Charlie only)
     Iris:  1 (through Dave only)
  4. Score by mutual friend count:
     Eve: 2, Frank: 2, Grace: 1, Hank: 1, Iris: 1
  5. Return top-K by score: [Eve, Frank, Grace, ...]

  WHY MUTUAL COUNT WORKS:
    → More mutual friends = higher likelihood of knowing each other
    → LinkedIn data: 30% acceptance rate for 2+ mutual connections
    → vs 5% for 0 mutual connections

ADVANCED SCORING — ADAMIC-ADAR INDEX:
  → Mutual friend who has 5 connections is MORE informative
    than mutual friend who has 5,000 connections
  → Score contribution of mutual friend M:
    1 / log(degree(M))
  → Person with 10 friends connecting you: weight = 1/log(10) = 0.43
  → Celebrity with 1M followers connecting you: weight = 1/log(1M) = 0.07
  → This downweights "celebrity" connections

SOLUTION LAYER 3: CONTENT RECOMMENDATIONS
═══════════════════════════════════════════════════════

COLLABORATIVE FILTERING ON BIPARTITE GRAPH:

  Users ←→ Items (posts, products, articles)

  User-Item interaction graph:
    Alice → {Post1, Post3, Post7}
    Bob   → {Post1, Post2, Post3}
    Carol → {Post2, Post4, Post7}

  For Alice, recommend items she hasn't seen:
  1. Find users similar to Alice (shared item interactions)
     Alice & Bob share: {Post1, Post3} → similarity = 2
     Alice & Carol share: {Post7} → similarity = 1
  2. Items those similar users liked that Alice hasn't seen:
     Bob liked Post2 (Alice hasn't seen) → score = 2 (Bob's similarity)
     Carol liked Post4 (Alice hasn't seen) → score = 1
     Carol liked Post2 → score += 1 → Post2 total = 3
  3. Recommend: Post2 (score 3), Post4 (score 1)

PERSONALIZED PAGERANK (Random Walk):
  → Start at Alice's node
  → At each step: 85% chance follow a random edge
  → 15% chance "teleport" back to Alice
  → Run for 1000 iterations
  → Count how often each node is visited
  → Nodes visited most = most relevant to Alice
  → Captures graph structure beyond direct connections

SOLUTION LAYER 4: SHORTEST PATH (Degrees of Separation)
═══════════════════════════════════════════════════════

NAIVE BFS: O(V + E)
  → Start from source, expand level by level
  → 300 friends → 90,000 at depth 2 → 27M at depth 3
  → Too slow for depth > 2

BIDIRECTIONAL BFS: O(√(V + E))
  → Start BFS from BOTH source AND target simultaneously
  → When the two frontiers MEET → path found
  → Instead of exploring 27M nodes from one side
  → Explore ~600 from each side until they meet
  → Dramatically faster for finding paths in dense graphs

  Source: Alice ───BFS→    ←BFS─── Target: Zack
         depth 1: 300       300 :depth 1
         depth 2: 90K       90K :depth 2
         MEET at some node in the middle
         Total explored: ~180K instead of ~27M

📌 WHAT COULD BE DONE

OPTION A: RELATIONAL DATABASE (PostgreSQL)
  → Store edges in a table: (user_a, user_b)
  → JOIN queries for traversal
  → Works up to ~1M users
  → Dies at scale due to JOIN explosion
  → Throughput: ~100 PYMK queries/sec

OPTION B: DEDICATED GRAPH DATABASE (Neo4j, JanusGraph)
  → Native graph storage: index-free adjacency
  → Traversal is pointer-following (O(1) per hop)
  → Cypher query: MATCH (a)-[:FRIEND*2]-(b) RETURN b
  → Works well up to ~100M edges
  → Throughput: ~5,000 traversals/sec

OPTION C: IN-MEMORY ADJACENCY LIST (Our approach)
  → Store graph as hash maps in memory
  → Partition across machines by user_id hash
  → BFS/DFS operates on in-memory data
  → Fastest traversal possible
  → Requires ~1.2 TB RAM across cluster
  → Throughput: ~50,000 PYMK queries/sec

OPTION D: PRECOMPUTED + CACHE
  → Batch compute PYMK every few hours
  → Store results in cache (Project 5)
  → Serve from cache, not live graph
  → Stale but fast
  → Throughput: ~500,000 reads/sec from cache

OPTION E: HYBRID (Our design)
  → In-memory adjacency list for hot graph operations
  → Precomputed PYMK refreshed periodically
  → Real-time incremental updates on new connections
  → Cache for read-heavy paths
  → Persistent store (DB) as source of truth for recovery
  → Combines speed of C + freshness of realtime updates

✅ WHAT SHOULD DO

1. USE ADJACENCY LIST WITH HASH SET PER NODE
   → O(1) edge existence check
   → O(degree) neighbor enumeration
   → Memory efficient for sparse graphs

2. LIMIT TRAVERSAL DEPTH TO 2-3 HOPS
   → Depth 1: 300 nodes (trivial)
   → Depth 2: ~90,000 nodes (manageable)
   → Depth 3: ~27,000,000 nodes (too expensive for real-time)
   → PYMK only needs depth 2
   → Shortest path uses bidirectional BFS (avoids depth 3+ from one side)

3. PRECOMPUTE EXPENSIVE QUERIES OFFLINE
   → PYMK: batch compute every 4-6 hours
   → PageRank: batch compute daily
   → Serve from cache, invalidate on significant changes
   → Real-time for: mutual friends (cheap), edge checks (cheap)

4. USE BLOOM FILTERS FOR "ALREADY FRIENDS" CHECK
   → PYMK generates candidates, must filter existing friends
   → Checking 90,000 candidates against 300 friends
   → Bloom filter: O(1) per check, ~1 byte per friend
   → Project 10 indexing concept applied to sets

5. SHARD GRAPH BY USER_ID HASH
   → Each partition owns a subset of users + their adjacency lists
   → Local queries (neighbors of user X) are single-partition
   → Cross-partition queries (mutual friends) need scatter-gather
   → Consistent hashing (Project 14) for rebalancing

6. CACHE HOT SUBGRAPHS
   → Celebrity nodes (1M+ followers) are queried constantly
   → Cache their neighbor lists and precomputed metrics
   → TTL: 5-15 minutes for celebrity data
   → Project 5 caching applied to graph data

7. USE EDGE WEIGHTS FOR RANKING
   → Not all friends are equal
   → Weight by: recency of interaction, message frequency, profile views
   → "Close friends" have higher edge weight
   → Recommendations from close friends rank higher

8. TRACK BIDIRECTIONAL RELATIONSHIPS SEPARATELY
   → Follow: A → B (directed)
   → Mutual follow: A ↔ B (bidirectional)
   → Store both directions for O(1) lookup either way
   → "Does Alice follow Bob?" and "Does Bob follow Alice?" both O(1)

❌ WHAT SHOULD NOT DO

1. NEVER STORE GRAPH AS ADJACENCY MATRIX
   → 500M × 500M = absurd memory
   → Even sparse matrix formats waste space
   → Adjacency list is ALWAYS better for social graphs (sparse)

2. NEVER DO UNBOUNDED GRAPH TRAVERSAL
   → "Find ALL people connected to Alice" on a connected graph
   → That's the ENTIRE graph (500M users)
   → ALWAYS set max_depth and max_nodes limits
   → Return partial results with "more available" indicator

3. NEVER JOIN TABLES FOR MULTI-HOP QUERIES IN SQL
   → 3-hop JOIN on 75B edges = query optimizer gives up
   → Even 2-hop JOINs are slow at scale
   → Use dedicated graph traversal, not SQL JOINs

4. NEVER RECOMPUTE PYMK ON EVERY REQUEST
   → User opens app → compute PYMK from scratch?
   → 90,000 candidate evaluations × scoring = 200ms+
   → Multiply by 10,000 concurrent users = impossible
   → Precompute + cache + incremental updates

5. NEVER RECOMMEND ALREADY-CONNECTED USERS
   → Obvious but commonly missed
   → Must filter candidates against existing connections
   → Must also filter: blocked users, pending requests, self

6. NEVER TREAT ALL EDGES AS EQUAL
   → Friend from 10 years ago with 0 interactions ≠ daily chat partner
   → Edge weight matters for recommendation quality
   → Recency bias: recent connections are more relevant

7. NEVER IGNORE GRAPH PARTITIONING QUALITY
   → Bad partition: all of Alice's friends on different machines
   → Every PYMK query requires N network round-trips
   → Good partition: community-aware (friends tend to be co-located)
   → Use METIS or social community detection for partitioning

8. NEVER SKIP CYCLE DETECTION IN TRAVERSAL
   → Social graphs have cycles (A→B→C→A)
   → BFS without visited set = infinite loop
   → ALWAYS maintain visited set during traversal

🚫 WHEN NOT TO USE

1. GRAPH DATABASE IS NOT FOR:
   → Simple key-value lookups (use Redis — Project 23)
   → Tabular data with fixed schema (use PostgreSQL — Project 3)
   → Time-series data (use specialized time-series DB)
   → Full-text search (use inverted index — covered in Project 41)

2. IN-MEMORY GRAPH IS NOT FOR:
   → Graphs too large for available RAM cluster
   → Infrequently accessed graphs (waste of expensive RAM)
   → Write-heavy workloads (disk-based is more durable)

3. COLLABORATIVE FILTERING IS NOT FOR:
   → New users with no interactions (cold start problem)
   → Items with no interaction history
   → Need content-based recommendations for cold start
   → Hybrid: collaborative + content-based

4. BFS TRAVERSAL IS NOT FOR:
   → Weighted shortest path (use Dijkstra instead)
   → Finding ALL paths (exponential, use DFS with backtracking)
   → Very deep graphs (>6 hops — use approximate methods)

5. PRECOMPUTED PYMK IS NOT FOR:
   → Users who just joined (no graph data yet)
   → Users who just made many new connections (stale precompute)
   → Rapidly changing graphs (invalidation too frequent)

MECHANICS 2: BFS — LEVEL-BY-LEVEL TRAVERSAL
═══════════════════════════════════════════════════════════

  BFS for "friends of friends" of Alice:

  LEVEL 0: {Alice}                          ← start
  LEVEL 1: {Bob, Charlie, Dave}             ← Alice's friends
  LEVEL 2: {Eve, Frank, Grace, Hank, Iris}  ← friends of friends
            (excluding Alice, Bob, Charlie, Dave — already seen)

  INTERNAL EXECUTION:
    queue = deque([("alice", 0)])
    visited = {"alice"}
    levels = defaultdict(set)

    while queue:
        node, depth = queue.popleft()
        if depth > max_depth:
            break
        levels[depth].add(node)
        
        for neighbor in graph[node].neighbors:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, depth + 1))

  COMPLEXITY:
    → Time: O(V + E) within traversal boundary
    → Space: O(V) for visited set
    → For depth-2 with 300 avg degree:
      ~1 + 300 + 90,000 = ~90,301 nodes explored
      ~90,301 × (hash lookup + queue ops) ≈ 5-10ms


═══════════════════════════════════════════════════════════
MECHANICS 3: BIDIRECTIONAL BFS — MEETING IN THE MIDDLE
═══════════════════════════════════════════════════════════

  Find shortest path from Alice to Zack:

  FORWARD frontier (from Alice):
    F0 = {Alice}
    F1 = {Bob, Charlie, Dave}          ← 300 nodes
    F2 = {Eve, Frank, ...}             ← 90,000 nodes

  BACKWARD frontier (from Zack):
    B0 = {Zack}
    B1 = {Yolanda, Xavier, Wendy}      ← 300 nodes
    B2 = {Victor, Uma, ...}            ← 90,000 nodes

  At each step:
    Expand the SMALLER frontier
    Check if any node in new frontier is in OTHER frontier's visited set
    If YES → path found! Total distance = forward_depth + backward_depth

  WHY FASTER:
    Unidirectional BFS depth 4: 300^4 = 8.1 BILLION nodes
    Bidirectional BFS depth 2+2: 2 × 300^2 = 180,000 nodes
    That's 45,000x fewer nodes explored!

  RECONSTRUCTING THE PATH:
    Forward tracks: parent_forward[node] = predecessor
    Backward tracks: parent_backward[node] = predecessor
    Meeting node found → trace back in both directions → combine

MECHANICS 4: PYMK SCORING — RANKING CANDIDATES
═══════════════════════════════════════════════════════════

  STEP 1: Get candidates (depth-2 BFS)
    candidates = {Eve: 2, Frank: 2, Grace: 1, Hank: 1, Iris: 1}

    Return top 5-10

  Step 3: Recommend
    Post2 (score 0.67) ← recommend this!

═══════════════════════════════════════════════════════════
MECHANICS 6: GRAPH PARTITIONING
═══════════════════════════════════════════════════════════

  HASH PARTITIONING (simple):
    partition = hash(user_id) % num_partitions
    
    Problem: Alice on partition 1, all her friends on partition 2
    → Every neighbor lookup = network call
    → "Chatty" queries kill performance

  COMMUNITY-AWARE PARTITIONING:
    → People who are friends tend to be in same community
    → Place communities together on same partition
    → Minimize "edge cuts" (edges crossing partitions)
    → Algorithms: METIS, Kernighan-Lin, label propagation

    RESULT:
      Partition 1: {Alice, Bob, Charlie, Dave}  ← friend group
      Partition 2: {Eve, Frank, Grace}          ← another group
      Cross-partition edges: only 3 (instead of 15 with random)

    BENEFIT:
      BFS depth-1 for Alice: all neighbors on same partition
      → 0 network calls (was 4 with hash partitioning)
      → 10x faster for local traversals

@dataclass
class UserNode:
    user_id: str
    name: str
    metadata: dict = field(default_factory=dict)
    neighbors: set = field(default_factory=set)
    edge_weights: dict = field(default_factory=dict)
    created_at: float = field(default_factory=time.time)

    @property
    def degree(self) -> int:
        return len(self.neighbors)


class SocialGraph:
    """
    In-memory social graph using adjacency lists.
    
    Each node: user with neighbor hash set (O(1) edge check)
    Bidirectional: adding edge A→B also adds B→A
    Edge weights: interaction strength between users
    """

    def __init__(self):
        self.nodes: Dict[str, UserNode] = {}
        self.edge_count = 0
        self._lock = threading.Lock()

    def add_user(self, user_id: str, name: str, metadata: dict = None):
        with self._lock:
            if user_id not in self.nodes:
                self.nodes[user_id] = UserNode(
                    user_id=user_id,
                    name=name,
                    metadata=metadata or {},
                )

    def add_edge(self, user_a: str, user_b: str, weight: float = 1.0):
        """Add bidirectional friendship edge"""
        with self._lock:
            if user_a not in self.nodes or user_b not in self.nodes:
                return False
            if user_b in self.nodes[user_a].neighbors:
                return False  # Already connected

            self.nodes[user_a].neighbors.add(user_b)
            self.nodes[user_b].neighbors.add(user_a)
            self.nodes[user_a].edge_weights[user_b] = weight
            self.nodes[user_b].edge_weights[user_a] = weight
            self.edge_count += 1
            return True

    def remove_edge(self, user_a: str, user_b: str):
        """Remove bidirectional friendship edge"""
        with self._lock:
            if user_a in self.nodes and user_b in self.nodes:
                self.nodes[user_a].neighbors.discard(user_b)
                self.nodes[user_b].neighbors.discard(user_a)
                self.nodes[user_a].edge_weights.pop(user_b, None)
                self.nodes[user_b].edge_weights.pop(user_a, None)
                self.edge_count -= 1

    def has_edge(self, user_a: str, user_b: str) -> bool:
        """O(1) edge existence check"""
        if user_a not in self.nodes:
            return False
        return user_b in self.nodes[user_a].neighbors

    def get_neighbors(self, user_id: str) -> Set[str]:
        """O(1) to get set, O(degree) to iterate"""
        if user_id not in self.nodes:
            return set()
        return set(self.nodes[user_id].neighbors)

    def get_degree(self, user_id: str) -> int:
        if user_id not in self.nodes:
            return 0
        return self.nodes[user_id].degree

    def get_stats(self) -> dict:
        degrees = [n.degree for n in self.nodes.values()]
        return {
            "total_nodes": len(self.nodes),
            "total_edges": self.edge_count,
            "avg_degree": sum(degrees) / max(len(degrees), 1),
            "max_degree": max(degrees) if degrees else 0,
            "min_degree": min(degrees) if degrees else 0,
        }

class GraphTraversal:

    @staticmethod
    def bfs(
        graph: SocialGraph,
        start: str,
        max_depth: int = 3,
        max_nodes: int = 100000,
    ) -> Dict[str, int]:
        """
        Breadth-First Search from start node.
        Returns dict of {node: depth} for all reachable nodes within limits.
        """
        if start not in graph.nodes:
            return {}

        visited = {start: 0}
        queue = deque([(start, 0)])
        nodes_explored = 0

        while queue and nodes_explored < max_nodes:
            node, depth = queue.popleft()

            if depth >= max_depth:
                continue

            for neighbor in graph.get_neighbors(node):
                if neighbor not in visited:
                    visited[neighbor] = depth + 1
                    queue.append((neighbor, depth + 1))
                    nodes_explored += 1

                    if nodes_explored >= max_nodes:
                        break

        return visited

    @staticmethod
    def get_nodes_at_depth(
        graph: SocialGraph, start: str, depth: int
    ) -> Set[str]:
        """Get all nodes exactly at given depth from start"""
        bfs_result = GraphTraversal.bfs(graph, start, max_depth=depth)
        return {node for node, d in bfs_result.items() if d == depth}

    @staticmethod
    def bidirectional_bfs(
        graph: SocialGraph, source: str, target: str, max_depth: int = 6
    ) -> Optional[dict]:
        """
        Bidirectional BFS for shortest path.
        
        Expand from BOTH source and target simultaneously.
        When frontiers meet → path found.
        
        Much faster than unidirectional for dense graphs:
        Unidirectional depth-4: 300^4 = 8.1B nodes
        Bidirectional depth-2+2: 2×300^2 = 180K nodes
        """
        if source not in graph.nodes or target not in graph.nodes:
            return None

        if source == target:
            return {"path": [source], "distance": 0, "nodes_explored": 1}

        # Forward BFS state (from source)
        forward_visited = {source: None}   # node → parent
        forward_frontier = deque([source])

        # Backward BFS state (from target)
        backward_visited = {target: None}
        backward_frontier = deque([target])

        nodes_explored = 2
        meeting_node = None

        for depth in range(max_depth):
            # Expand the SMALLER frontier (optimization)
            if len(forward_frontier) <= len(backward_frontier):
                # Expand forward one level
                next_frontier = deque()
                while forward_frontier:
                    node = forward_frontier.popleft()
                    for neighbor in graph.get_neighbors(node):
                        if neighbor not in forward_visited:
                            forward_visited[neighbor] = node
                            next_frontier.append(neighbor)
                            nodes_explored += 1

                            # Check if backward has seen this node
                            if neighbor in backward_visited:
                                meeting_node = neighbor
                                break
                    if meeting_node:
                        break
                forward_frontier = next_frontier
            else:
                # Expand backward one level
                next_frontier = deque()
                while backward_frontier:
                    node = backward_frontier.popleft()
                    for neighbor in graph.get_neighbors(node):
                        if neighbor not in backward_visited:
                            backward_visited[neighbor] = node
                            next_frontier.append(neighbor)
                            nodes_explored += 1

                            if neighbor in forward_visited:
                                meeting_node = neighbor
                                break
                    if meeting_node:
                        break
                backward_frontier = next_frontier

            if meeting_node:
                # Reconstruct path
                # Forward path: source → ... → meeting
                forward_path = []
                node = meeting_node
                while node is not None:
                    forward_path.append(node)
                    node = forward_visited[node]
                forward_path.reverse()

                # Backward path: meeting → ... → target
                backward_path = []
                node = backward_visited[meeting_node]
                while node is not None:
                    backward_path.append(node)
                    node = backward_visited[node]

                full_path = forward_path + backward_path

                return {
                    "path": full_path,
                    "distance": len(full_path) - 1,
                    "nodes_explored": nodes_explored,
                    "meeting_node": meeting_node,
                }

            if not forward_frontier and not backward_frontier:
                break

        return None  # No path found within max_depth

    @staticmethod
    def mutual_friends(
        graph: SocialGraph, user_a: str, user_b: str
    ) -> Set[str]:
        """Set intersection of neighbor lists — O(min(degree_a, degree_b))"""
        neighbors_a = graph.get_neighbors(user_a)
        neighbors_b = graph.get_neighbors(user_b)
        return neighbors_a & neighbors_b

    @staticmethod
    def connected_components(graph: SocialGraph) -> List[Set[str]]:
        """Find all connected components using BFS"""
        visited = set()
        components = []

        for node_id in graph.nodes:
            if node_id not in visited:
                component = set()
                queue = deque([node_id])

                while queue:
                    node = queue.popleft()
                    if node in visited:
                        continue
                    visited.add(node)
                    component.add(node)
                    for neighbor in graph.get_neighbors(node):
                        if neighbor not in visited:
                            queue.append(neighbor)

                components.append(component)

        return sorted(components, key=len, reverse=True)

class PYMKEngine:
    """
    People You May Know recommendation engine.
    
    Scoring signals:
    1. Mutual friend count (strongest signal)
    2. Adamic-Adar index (weighted mutual friends)
    3. Jaccard similarity (overlap of friend sets)
    4. Community overlap (same metadata attributes)
    """

    def __init__(self, graph: SocialGraph):
        self.graph = graph

    def get_recommendations(
        self,
        user_id: str,
        top_k: int = 10,
        max_candidates: int = 10000,
    ) -> List[dict]:
        """
        Full PYMK pipeline:
        1. Get friends-of-friends (BFS depth 2)
        2. Filter out existing friends
        3. Score each candidate
        4. Return top-K
        """
        if user_id not in self.graph.nodes:
            return []

        start_time = time.time()

        # Step 1: Get existing friends (for filtering)
        existing_friends = self.graph.get_neighbors(user_id)
        exclude_set = existing_friends | {user_id}

        # Step 2: Get friends-of-friends with mutual friend tracking
        mutual_friend_count = defaultdict(int)
        mutual_friend_list = defaultdict(list)

        for friend in existing_friends:
            for fof in self.graph.get_neighbors(friend):
                if fof not in exclude_set:
                    mutual_friend_count[fof] += 1
                    mutual_friend_list[fof].append(friend)

                    if len(mutual_friend_count) >= max_candidates:
                        break
            if len(mutual_friend_count) >= max_candidates:
                break

        # Step 3: Score each candidate
        scored_candidates = []
        user_degree = self.graph.get_degree(user_id)
        user_metadata = self.graph.nodes[user_id].metadata

        for candidate_id, mutual_count in mutual_friend_count.items():
            score = self._compute_score(
                user_id, candidate_id, mutual_count,
                mutual_friend_list[candidate_id],
                user_degree, user_metadata,
            )
            scored_candidates.append({
                "user_id": candidate_id,
                "name": self.graph.nodes[candidate_id].name,
                "score": score["total"],
                "score_breakdown": score,
                "mutual_friends": mutual_friend_list[candidate_id][:5],
                "mutual_count": mutual_count,
            })

        # Step 4: Sort by score, return top-K
        scored_candidates.sort(key=lambda x: x["score"], reverse=True)
        top_results = scored_candidates[:top_k]

        elapsed = time.time() - start_time

        return {
            "user_id": user_id,
            "recommendations": top_results,
            "candidates_evaluated": len(scored_candidates),
            "computation_time_ms": round(elapsed * 1000, 2),
        }

    def _compute_score(
        self, user_id, candidate_id, mutual_count,
        mutual_friends, user_degree, user_metadata,
    ) -> dict:
        """Multi-signal scoring"""
        # Signal 1: Mutual friend count (normalized)
        max_possible_mutual = user_degree
        mutual_score = mutual_count / max(max_possible_mutual, 1)

        # Signal 2: Adamic-Adar index
        adamic_adar = 0.0
        for mutual in mutual_friends:
            degree_m = self.graph.get_degree(mutual)
            if degree_m > 1:
                adamic_adar += 1.0 / math.log(degree_m)
        # Normalize
        max_aa = len(mutual_friends) * (1.0 / math.log(2)) if mutual_friends else 1
        aa_score = adamic_adar / max(max_aa, 0.001)

        # Signal 3: Jaccard similarity
        candidate_neighbors = self.graph.get_neighbors(candidate_id)
        user_neighbors = self.graph.get_neighbors(user_id)
        intersection = len(user_neighbors & candidate_neighbors)
        union = len(user_neighbors | candidate_neighbors)
        jaccard = intersection / max(union, 1)

        # Signal 4: Metadata/community overlap
        candidate_metadata = self.graph.nodes[candidate_id].metadata
        community_score = self._community_overlap(user_metadata, candidate_metadata)

        # Weighted combination
        total = (
            0.35 * mutual_score
            + 0.25 * aa_score
            + 0.20 * jaccard
            + 0.20 * community_score
        )

        return {
            "total": round(total, 4),
            "mutual_score": round(mutual_score, 4),
            "adamic_adar": round(aa_score, 4),
            "jaccard": round(jaccard, 4),
            "community": round(community_score, 4),
        }

    def _community_overlap(self, meta_a: dict, meta_b: dict) -> float:
        """Score based on shared attributes (company, school, city)"""
        overlap = 0
        total_keys = 0
        for key in ["company", "school", "city", "industry"]:
            if key in meta_a and key in meta_b:
                total_keys += 1
                if meta_a[key] == meta_b[key]:
                    overlap += 1
        return overlap / max(total_keys, 1)

    Scoring: weighted by user similarity
    """

    def __init__(self):
        self.user_items: Dict[str, Set[str]] = defaultdict(set)
        self.item_users: Dict[str, Set[str]] = defaultdict(set)

    def add_interaction(self, user_id: str, item_id: str):
        self.user_items[user_id].add(item_id)
        self.item_users[item_id].add(user_id)

    def get_similar_users(
        self, user_id: str, top_k: int = 10
    ) -> List[Tuple[str, float]]:
        """Find users with most similar interaction patterns"""
        if user_id not in self.user_items:
            return []

        my_items = self.user_items[user_id]
        candidate_scores = defaultdict(float)

        # Find users who interacted with same items
        for item in my_items:
            for other_user in self.item_users[item]:
                if other_user != user_id:
                    candidate_scores[other_user] += 1

        similarities = []
        for other_user, shared_count in candidate_scores.items():
            other_items = self.user_items[other_user]
            union_size = len(my_items | other_items)
            jaccard = shared_count / max(union_size, 1)
            similarities.append((other_user, jaccard))

                ],
            })

        return results

# SECTION 5: PERSONALIZED PAGERANK
# Random walk with restart for personalized recommendations
# ================================================================

class PersonalizedPageRank:
    """
    Personalized PageRank (PPR) computes relevance of every node
    to a specific user by simulating random walks.
    
    At each step:
    → 85% probability: follow a random neighbor
    → 15% probability: "teleport" back to the start user
    
    Nodes visited most often = most relevant to the user.
    Captures GLOBAL graph structure, not just local neighborhood.
    """

    @staticmethod
    def compute(
        graph: SocialGraph,
        start_user: str,
        damping: float = 0.85,
        iterations: int = 20,
        top_k: int = 10,
    ) -> List[dict]:
        """Power iteration method for Personalized PageRank."""
        if start_user not in graph.nodes:
            return []

        # Initialize: all probability mass on start_user
        scores = defaultdict(float)
        scores[start_user] = 1.0

        for iteration in range(iterations):
            new_scores = defaultdict(float)

            for node_id, score in scores.items():
                if score < 1e-8:
                    continue

                neighbors = graph.get_neighbors(node_id)
                degree = len(neighbors)

                if degree == 0:
                    # Dead end: teleport back to start
                    new_scores[start_user] += score
                    continue

                # Distribute score to neighbors (damping fraction)
                distributed = damping * score / degree
                for neighbor in neighbors:
                    new_scores[neighbor] += distributed

            # Teleport: (1 - damping) goes back to start user
            total_distributed = sum(new_scores.values())
            teleport_mass = 1.0 - total_distributed
            new_scores[start_user] += teleport_mass

            scores = new_scores

        # Remove start user and existing friends from results
        existing_friends = graph.get_neighbors(start_user)
        exclude = existing_friends | {start_user}

        ranked = [
            {"user_id": uid, "ppr_score": round(score, 6)}
            for uid, score in sorted(scores.items(), key=lambda x: x[1], reverse=True)
            if uid not in exclude
        ]

        return ranked[:top_k]

    @staticmethod
    def random_walk_simulation(
        graph: SocialGraph,
        start_user: str,
        num_walks: int = 1000,
        walk_length: int = 10,
        restart_prob: float = 0.15,
    ) -> List[dict]:
        """
        Monte Carlo approximation of Personalized PageRank.
        Actually simulate random walks instead of power iteration.
        More intuitive, slightly less precise, easily parallelizable.
        """
        if start_user not in graph.nodes:
            return []

        visit_count = defaultdict(int)

        for _ in range(num_walks):
            current = start_user

            for _ in range(walk_length):
                visit_count[current] += 1

                # Restart with probability restart_prob
                if random.random() < restart_prob:
                    current = start_user
                    continue

                # Follow random neighbor
                neighbors = list(graph.get_neighbors(current))
                if not neighbors:
                    current = start_user
                    continue

                current = random.choice(neighbors)

        # Normalize
        total_visits = sum(visit_count.values())
        existing_friends = graph.get_neighbors(start_user)
        exclude = existing_friends | {start_user}

        ranked = [
            {
                "user_id": uid,
                "visit_count": count,
                "ppr_score": round(count / total_visits, 6),
            }
            for uid, count in sorted(
                visit_count.items(), key=lambda x: x[1], reverse=True
            )
            if uid not in exclude
        ]

        return ranked[:10]

# SECTION 6: BLOOM FILTER FOR FAST MEMBERSHIP CHECK
# Used to quickly check "is this person already a friend?"
# Reference: Project 10 (Indexing concepts)
# ================================================================

class BloomFilter:
    """
    Probabilistic set membership check.
    
    Properties:
    → NO false negatives: if it says "not in set", definitely not
    → POSSIBLE false positives: if it says "in set", might be wrong
    → O(1) check, O(k) hash computations where k = number of hash functions
    → Memory: ~10 bits per element for 1% false positive rate
    
    Used in PYMK to quickly filter out existing friends from candidates.
    Instead of checking 90,000 candidates against 300-element set
    with hash set (fine at small scale), Bloom filter is more
    cache-friendly and uses less memory at massive scale.
    """

    def __init__(self, expected_elements: int, false_positive_rate: float = 0.01):
        # Calculate optimal size and hash count
        self.size = self._optimal_size(expected_elements, false_positive_rate)
        self.hash_count = self._optimal_hash_count(self.size, expected_elements)
        self.bit_array = [False] * self.size
        self.element_count = 0

    def _optimal_size(self, n: int, p: float) -> int:
        """m = -(n * ln(p)) / (ln(2))^2"""
        m = -(n * math.log(p)) / (math.log(2) ** 2)
        return max(int(m), 64)

    def _optimal_hash_count(self, m: int, n: int) -> int:
        """k = (m/n) * ln(2)"""
        k = (m / max(n, 1)) * math.log(2)
        return max(int(k), 1)

    def _get_hash_positions(self, item: str) -> List[int]:
        """Generate k hash positions using double hashing"""
        positions = []
        h1 = int(hashlib.md5(item.encode()).hexdigest(), 16)
        h2 = int(hashlib.sha1(item.encode()).hexdigest(), 16)
        for i in range(self.hash_count):
            pos = (h1 + i * h2) % self.size
            positions.append(pos)
        return positions

    def add(self, item: str):
        for pos in self._get_hash_positions(item):
            self.bit_array[pos] = True
        self.element_count += 1

    def might_contain(self, item: str) -> bool:
        """Returns True if item MIGHT be in set, False if DEFINITELY not"""
        return all(self.bit_array[pos] for pos in self._get_hash_positions(item))

    def get_stats(self) -> dict:
        bits_set = sum(self.bit_array)
        return {
            "size_bits": self.size,
            "size_bytes": self.size // 8,
            "hash_functions": self.hash_count,
            "elements_added": self.element_count,
            "bits_set": bits_set,
            "fill_ratio": round(bits_set / self.size, 4),
        }

# SECTION 7: COMMUNITY DETECTION
# Label Propagation algorithm — find clusters in the graph
# ================================================================

class CommunityDetector:
    """
    Label Propagation Algorithm for community detection.
    
    Simple and efficient:
    1. Each node starts with unique label
    2. Each iteration: node adopts the most common label among neighbors
    3. Repeat until convergence (labels stop changing)
    4. Nodes with same label = same community
    
    Time complexity: O(iterations × edges)
    """

    @staticmethod
    def detect_communities(
        graph: SocialGraph, max_iterations: int = 20
    ) -> dict:
        """Run label propagation and return communities"""
        # Step 1: Initialize each node with its own label
        labels = {uid: uid for uid in graph.nodes}

        for iteration in range(max_iterations):
            changed = 0
            # Process nodes in random order (important for convergence)
            node_order = list(graph.nodes.keys())
            random.shuffle(node_order)

            for node_id in node_order:
                neighbors = graph.get_neighbors(node_id)
                if not neighbors:
                    continue

                # Count neighbor labels
                label_counts = defaultdict(int)
                for neighbor in neighbors:
                    label_counts[labels[neighbor]] += 1

                # Adopt most common label
                max_count = max(label_counts.values())
                most_common = [
                    label for label, count in label_counts.items()
                    if count == max_count
                ]

                # Break ties deterministically
                new_label = min(most_common)

                if labels[node_id] != new_label:
                    labels[node_id] = new_label
                    changed += 1

            if changed == 0:
                break

        # Group nodes by label
        communities = defaultdict(set)
        for node_id, label in labels.items():
            communities[label].add(node_id)

        # Sort by community size (largest first)
        sorted_communities = sorted(
            communities.values(), key=len, reverse=True
        )

        return {
            "num_communities": len(sorted_communities),
            "communities": sorted_communities,
            "largest_community_size": len(sorted_communities[0]) if sorted_communities else 0,
            "iterations_used": iteration + 1,
        }

# SECTION 8: GRAPH PARTITIONER
# Distribute graph across machines for distributed processing
# Reference: Project 14 (Consistent Hashing)
# ================================================================

class GraphPartitioner:
    """
    Partition graph across N shards.
    
    Two strategies:
    1. Hash partitioning (simple, random distribution)
    2. Community-aware partitioning (minimize cross-partition edges)
    """

    @staticmethod
    def hash_partition(
        graph: SocialGraph, num_partitions: int
    ) -> dict:
        """Simple hash-based partitioning"""
        partitions = defaultdict(set)

        for user_id in graph.nodes:
            partition_id = hash(user_id) % num_partitions
            partitions[partition_id].add(user_id)

        # Count cross-partition edges
        cross_edges = 0
        local_edges = 0
        user_partition = {}
        for pid, users in partitions.items():
            for uid in users:
                user_partition[uid] = pid

        for uid in graph.nodes:
            for neighbor in graph.get_neighbors(uid):
                if user_partition.get(uid) != user_partition.get(neighbor):
                    cross_edges += 1
                else:
                    local_edges += 1

        # Each edge counted twice (bidirectional)
        cross_edges //= 2
        local_edges //= 2

        return {
            "strategy": "hash",
            "num_partitions": num_partitions,
            "partition_sizes": {
                pid: len(users) for pid, users in partitions.items()
            },
            "cross_partition_edges": cross_edges,
            "local_edges": local_edges,
            "cross_edge_ratio": round(
                cross_edges / max(cross_edges + local_edges, 1), 4
            ),
            "partitions": dict(partitions),
        }

    @staticmethod
    def community_aware_partition(
        graph: SocialGraph, num_partitions: int
    ) -> dict:
        """
        Use community detection to place friends together.
        Communities are assigned to partitions round-robin by size.
        """
        # Detect communities first
        community_result = CommunityDetector.detect_communities(graph)
        communities = community_result["communities"]

        # Assign communities to partitions (largest first, round-robin)
        partitions = defaultdict(set)
        partition_sizes = defaultdict(int)

        for i, community in enumerate(communities):
            # Assign to least-loaded partition
            target = min(range(num_partitions), key=lambda p: partition_sizes[p])
            partitions[target].update(community)
            partition_sizes[target] += len(community)

        # Count cross-partition edges
        user_partition = {}
        for pid, users in partitions.items():
            for uid in users:
                user_partition[uid] = pid

        cross_edges = 0
        local_edges = 0
        for uid in graph.nodes:
            for neighbor in graph.get_neighbors(uid):
                if user_partition.get(uid) != user_partition.get(neighbor):
                    cross_edges += 1
                else:
                    local_edges += 1

        cross_edges //= 2
        local_edges //= 2

        return {
            "strategy": "community_aware",
            "num_partitions": num_partitions,
            "communities_detected": len(communities),
            "partition_sizes": {
                pid: len(users) for pid, users in partitions.items()
            },
            "cross_partition_edges": cross_edges,
            "local_edges": local_edges,
            "cross_edge_ratio": round(
                cross_edges / max(cross_edges + local_edges, 1), 4
            ),
            "partitions": dict(partitions),
        }

# Real-time updates when graph changes

class IncrementalPYMKUpdater:
    """
    When a new edge is added, incrementally update PYMK caches
    instead of full recomputation.
    
    New edge: Alice → Bob
    Impact:
    → Alice's PYMK: add Bob's friends as new candidates
    → Bob's PYMK: add Alice's friends as new candidates
    → Friends of Alice: Bob is now a stronger candidate for them
    → Friends of Bob: Alice is now a stronger candidate for them
    """

    def __init__(self, graph: SocialGraph, pymk_cache: dict = None):
        self.graph = graph
        self.pymk_cache = pymk_cache or {}
        self.update_log = []

    def on_new_edge(self, user_a: str, user_b: str) -> dict:
        """Process a new friendship and update affected PYMK caches"""
        updates = {
            "edge": (user_a, user_b),
            "affected_users": set(),
            "cache_invalidations": 0,
            "new_candidates_added": 0,
        }

        # 1. Remove each other from PYMK caches (they're now friends)
        if user_a in self.pymk_cache:
            self.pymk_cache[user_a] = [
                r for r in self.pymk_cache.get(user_a, [])
                if r.get("user_id") != user_b
            ]
            updates["cache_invalidations"] += 1

        if user_b in self.pymk_cache:
            self.pymk_cache[user_b] = [
                r for r in self.pymk_cache.get(user_b, [])
                if r.get("user_id") != user_a
            ]
            updates["cache_invalidations"] += 1

        # 2. Bob's friends become new PYMK candidates for Alice
        for bob_friend in self.graph.get_neighbors(user_b):
            if (bob_friend != user_a and
                    not self.graph.has_edge(user_a, bob_friend)):
                updates["new_candidates_added"] += 1
                updates["affected_users"].add(user_a)

        # 3. Alice's friends become new PYMK candidates for Bob
        for alice_friend in self.graph.get_neighbors(user_a):
            if (alice_friend != user_b and
                    not self.graph.has_edge(user_b, alice_friend)):
                updates["new_candidates_added"] += 1
                updates["affected_users"].add(user_b)

        # 4. Friends of both: mutual friend count increased
        friends_of_a = self.graph.get_neighbors(user_a)
        friends_of_b = self.graph.get_neighbors(user_b)

        for friend_of_a in friends_of_a:
            if friend_of_a != user_b:
                # friend_of_a now has one more mutual friend with Bob
                updates["affected_users"].add(friend_of_a)

        for friend_of_b in friends_of_b:
            if friend_of_b != user_a:
                updates["affected_users"].add(friend_of_b)

        updates["affected_users"] = len(updates["affected_users"])

        self.update_log.append(updates)
        return updates

class SocialGraphGenerator:
    """
    Generate realistic social graphs with community structure.
    
    Uses preferential attachment + community clustering:
    → Popular nodes get more connections (power law degree distribution)
    → Nodes in same community have higher connection probability
    → Mimics real-world social network properties
    """

    @staticmethod
    def generate(
        num_users: int = 100,
        num_communities: int = 5,
        intra_community_prob: float = 0.3,
        inter_community_prob: float = 0.02,
        companies: list = None,
        schools: list = None,
        cities: list = None,
    ) -> SocialGraph:
        """Generate a realistic social graph"""
        graph = SocialGraph()

        companies = companies or ["TechCorp", "DataInc", "CloudSoft", "AILabs", "NetBase"]
        schools = schools or ["MIT", "Stanford", "Harvard", "Berkeley", "CMU"]
        cities = cities or ["NYC", "SF", "London", "Tokyo", "Berlin"]

        first_names = [
            "Alice", "Bob", "Charlie", "Diana", "Eve", "Frank",
            "Grace", "Hank", "Iris", "Jack", "Kate", "Leo",
            "Mia", "Noah", "Olivia", "Pete", "Quinn", "Rosa",
            "Sam", "Tara", "Uma", "Vic", "Wendy", "Xander",
            "Yolanda", "Zack",
        ]

        # Assign users to communities
        community_assignments = {}
        for i in range(num_users):
            community_assignments[i] = i % num_communities

        # Create users
        for i in range(num_users):
            community_id = community_assignments[i]
            user_id = f"user_{i:04d}"
            name = f"{random.choice(first_names)}_{i}"

            graph.add_user(user_id, name, metadata={
                "community": community_id,
                "company": companies[community_id % len(companies)],
                "school": schools[random.randint(0, len(schools) - 1)],
                "city": cities[community_id % len(cities)],
            })

        # Create edges based on community structure
        user_ids = list(graph.nodes.keys())
        for i in range(num_users):
            for j in range(i + 1, num_users):
                same_community = (
                    community_assignments[i] == community_assignments[j]
                )

                prob = (
                    intra_community_prob if same_community
                    else inter_community_prob
                )

                if random.random() < prob:
                    weight = random.uniform(0.5, 1.0) if same_community else random.uniform(0.1, 0.5)
                    graph.add_edge(user_ids[i], user_ids[j], weight=weight)

        return graph

        print("  📈 STEP 7: PERSONALIZED PAGERANK")
        print("=" * 70)

        print(f"\n  Power Iteration PPR for: {test_user}")
        start_time = time.time()
        ppr_results = PersonalizedPageRank.compute(
            graph, test_user, iterations=20, top_k=8
        )
        ppr_elapsed = (time.time() - start_time) * 1000

        print(f"  Computation time: {ppr_elapsed:.2f}ms")
        print(f"\n  {'Rank':<6}{'User':<18}{'PPR Score':<15}")
        print(f"  {'─' * 40}")
        for i, result in enumerate(ppr_results, 1):
            print(f"  {i:<6}{result['user_id']:<18}{result['ppr_score']:<15.6f}")

        print(f"\n  Random Walk Simulation PPR for: {test_user}")
        start_time = time.time()
        rw_results = PersonalizedPageRank.random_walk_simulation(
            graph, test_user, num_walks=500, walk_length=15
        )
        rw_elapsed = (time.time() - start_time) * 1000

        print(f"  Computation time: {rw_elapsed:.2f}ms (500 walks × 15 steps)")
        print(f"\n  {'Rank':<6}{'User':<18}{'Visits':<10}{'PPR Score':<15}")
        print(f"  {'─' * 50}")
        for i, result in enumerate(rw_results[:8], 1):
            print(
                f"  {i:<6}{result['user_id']:<18}"
                f"{result['visit_count']:<10}{result['ppr_score']:<15.6f}"
            )

        # ── Step 8: Community Detection ──
        print("\n" + "=" * 70)
        print("  🏘️  STEP 8: COMMUNITY DETECTION (Label Propagation)")
        print("=" * 70)

        start_time = time.time()
        community_result = CommunityDetector.detect_communities(graph)
        cd_elapsed = (time.time() - start_time) * 1000

        print(f"\n  Communities detected: {community_result['num_communities']}")
        print(f"  Iterations used: {community_result['iterations_used']}")
        print(f"  Computation time: {cd_elapsed:.2f}ms")

        print(f"\n  Community sizes (top 10):")
        for i, community in enumerate(community_result['communities'][:10], 1):
            sample = list(community)[:4]
            more = f" +{len(community) - 4} more" if len(community) > 4 else ""
            print(f"    Community {i}: {len(community)} members — {sample}{more}")

        # ── Step 9: Graph Partitioning Comparison ──
        print("\n" + "=" * 70)
        print("  🗄️  STEP 9: GRAPH PARTITIONING (Hash vs Community-Aware)")
        print("=" * 70)

        num_partitions = 4

        hash_result = GraphPartitioner.hash_partition(graph, num_partitions)
        community_result_part = GraphPartitioner.community_aware_partition(
            graph, num_partitions
        )

        print(f"\n  Partitioning {stats['total_nodes']} users into {num_partitions} partitions:")

        print(f"\n  ┌{'─' * 25}┬{'─' * 22}┬{'─' * 22}┐")
        print(f"  │{'Metric':<25}│{'Hash Partition':^22}│{'Community-Aware':^22}│")
        print(f"  ├{'─' * 25}┼{'─' * 22}┼{'─' * 22}┤")

        metrics = [
            ("Cross-partition edges",
             str(hash_result['cross_partition_edges']),
             str(community_result_part['cross_partition_edges'])),
            ("Local edges",
             str(hash_result['local_edges']),
             str(community_result_part['local_edges'])),
            ("Cross-edge ratio",
             f"{hash_result['cross_edge_ratio']:.1%}",
             f"{community_result_part['cross_edge_ratio']:.1%}"),
        ]

        for metric, hash_val, comm_val in metrics:
            print(f"  │{metric:<25}│{hash_val:^22}│{comm_val:^22}│")

        print(f"  └{'─' * 25}┴{'─' * 22}┴{'─' * 22}┘")

        hash_sizes = list(hash_result['partition_sizes'].values())
        comm_sizes = list(community_result_part['partition_sizes'].values())

        print(f"\n  Partition sizes:")
        print(f"    Hash:      {hash_sizes}")
        print(f"    Community: {comm_sizes}")

        if community_result_part['cross_edge_ratio'] < hash_result['cross_edge_ratio']:
            improvement = (
                (hash_result['cross_edge_ratio'] - community_result_part['cross_edge_ratio'])
                / max(hash_result['cross_edge_ratio'], 0.001)
            ) * 100
            print(f"\n  ✅ Community-aware partitioning reduced cross-edges by {improvement:.0f}%")
            print(f"     This means {improvement:.0f}% fewer network calls during traversal!")

        # ── Step 10: Bloom Filter Demo ──
        print("\n" + "=" * 70)
        print("  🌸 STEP 10: BLOOM FILTER FOR FAST FRIEND CHECK")
        print("=" * 70)

        friends_list = list(graph.get_neighbors(test_user))
        bf = BloomFilter(expected_elements=len(friends_list), false_positive_rate=0.01)

        for friend in friends_list:
            bf.add(friend)

        bf_stats = bf.get_stats()
        print(f"\n  Bloom filter for {test_user}'s {len(friends_list)} friends:")
        print(f"    Size: {bf_stats['size_bits']} bits ({bf_stats['size_bytes']} bytes)")
        print(f"    Hash functions: {bf_stats['hash_functions']}")
        print(f"    Fill ratio: {bf_stats['fill_ratio']:.2%}")

        # Test accuracy
        true_positives = sum(1 for f in friends_list if bf.might_contain(f))
        non_friends = [
            uid for uid in list(graph.nodes.keys())[:100]
            if uid not in graph.get_neighbors(test_user) and uid != test_user
        ]
        false_positives = sum(1 for nf in non_friends if bf.might_contain(nf))

        print(f"\n  Accuracy test:")
        print(f"    True positives:  {true_positives}/{len(friends_list)} (should be 100%)")
        print(f"    False positives: {false_positives}/{len(non_friends)} "
              f"({false_positives / max(len(non_friends), 1):.1%}, target: <1%)")

        # Memory comparison
        hashset_memory = len(friends_list) * 50  # ~50 bytes per string in Python set
        bloom_memory = bf_stats['size_bytes']
        print(f"\n  Memory comparison:")
        print(f"    HashSet: ~{hashset_memory} bytes")
        print(f"    Bloom:   ~{bloom_memory} bytes")
        print(f"    Savings: {(1 - bloom_memory / max(hashset_memory, 1)) * 100:.0f}% less memory")

        # ── Step 11: Incremental PYMK Update ──
        print("\n" + "=" * 70)
        print("  ⚡ STEP 11: INCREMENTAL PYMK UPDATE (Real-Time)")
        print("=" * 70)

        updater = IncrementalPYMKUpdater(graph)

        # Simulate a new connection
        non_friend = None
        for uid in graph.nodes:
            if uid != test_user and not graph.has_edge(test_user, uid):
                non_friend = uid
                break

        if non_friend:
            print(f"\n  New connection: {test_user} ↔ {non_friend}")
            print(f"  {test_user} friends before: {graph.get_degree(test_user)}")

            # Add the edge
            graph.add_edge(test_user, non_friend)
            update_result = updater.on_new_edge(test_user, non_friend)

            print(f"  {test_user} friends after:  {graph.get_degree(test_user)}")
            print(f"\n  Incremental update impact:")
            print(f"    Affected users:        {update_result['affected_users']}")
            print(f"    Cache invalidations:   {update_result['cache_invalidations']}")
            print(f"    New PYMK candidates:   {update_result['new_candidates_added']}")
            print(f"\n  💡 Instead of recomputing PYMK for ALL {stats['total_nodes']} users,")
            print(f"     we only need to update {update_result['affected_users']} users' caches")

        # ── Step 12: Connected Components ──
        print("\n" + "=" * 70)
        print("  🔗 STEP 12: CONNECTED COMPONENTS")
        print("=" * 70)

        components = GraphTraversal.connected_components(graph)
        print(f"\n  Total components: {len(components)}")
        print(f"  Largest component: {len(components[0])} users "
              f"({len(components[0]) / stats['total_nodes'] * 100:.1f}% of graph)")
        if len(components) > 1:
            print(f"  Isolated components: {len(components) - 1}")
            for i, comp in enumerate(components[1:6], 2):
                print(f"    Component {i}: {len(comp)} users — {list(comp)[:3]}")

        # ── Final Summary ──
        print("\n" + "=" * 70)
        print("  📊 PERFORMANCE SUMMARY")
        print("=" * 70)

        print(f"\n  ┌{'─' * 40}┬{'─' * 15}┐")
        print(f"  │{'Operation':<40}│{'Typical Time':>15}│")
        print(f"  ├{'─' * 40}┼{'─' * 15}┤")
        operations = [
            ("Edge existence check", "O(1) ~0.001ms"),
            ("Get all neighbors", "O(degree) ~0.01ms"),
            ("Mutual friends (set intersection)", "~0.05ms"),
            ("BFS depth-2 traversal", "~1-5ms"),
            ("PYMK full computation", f"{pymk_result['computation_time_ms']}ms"),
            ("Bidirectional shortest path", f"{elapsed:.2f}ms"),
            ("Personalized PageRank (power)", f"{ppr_elapsed:.2f}ms"),
            ("Personalized PageRank (walk)", f"{rw_elapsed:.2f}ms"),
            ("Community detection", f"{cd_elapsed:.2f}ms"),
            ("Bloom filter check", "O(k) ~0.001ms"),
        ]
        for op, timing in operations:
            print(f"  │{op:<40}│{timing:>15}│")
        print(f"  └{'─' * 40}┴{'─' * 15}┘")

        # ── Concepts Summary ──
        print("\n")
        print("╔" + "═" * 68 + "╗")
        print("║" + "  CONCEPTS DEMONSTRATED IN THIS PROJECT".center(68) + "║")
        print("╠" + "═" * 68 + "╣")
        concepts = [
