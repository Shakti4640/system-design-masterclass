🎯 HYPOTHETICAL PROBLEM

You run an online food delivery platform (like DoorDash)
→ 50,000 orders placed per minute at peak (lunch rush)
→ Each order triggers 5-8 database writes:
   - Create order record
   - Reserve restaurant capacity
   - Assign delivery driver
   - Process payment
   - Update inventory
   - Create tracking record
   - Send notifications

→ SIMULTANEOUSLY, hundreds of thousands of reads:
   - Customer app: "Where is my food?" (polling every 5 seconds)
   - Restaurant dashboard: "Show pending orders with details"
   - Driver app: "Show my active deliveries with route info"
   - Admin panel: "Revenue by restaurant, hour, region"
   - Analytics: "Average delivery time by zone"
   - Customer support: "Look up order with full history"
   - Marketing: "Top 100 restaurants by order count this week"

READ:WRITE RATIO = approximately 50:1

WHY READ REPLICAS (Project 13) ARE NOT ENOUGH:

  Read replicas help with THROUGHPUT but NOT with:
  → Schema mismatch: writes need normalized tables,
    reads need denormalized views
  → Query complexity: 7-table JOIN is slow even on replica
  → Index conflict: indexes that help reads hurt writes
  → Model mismatch: write model ≠ what UI needs to display
  → Replication lag: replica might show stale data

  Read replicas give you MORE COPIES of the SAME schema
  CQRS gives you DIFFERENT SCHEMAS optimized for each purpose

THE CQRS SOLUTION:

  COMMAND SIDE (Write)              QUERY SIDE (Read)
  ┌─────────────────┐              ┌─────────────────────┐
  │ Normalized DB    │   Events    │ Denormalized Views   │
  │ orders           │ ─────────→ │                      │
  │ order_items      │ (Proj 29)  │ customer_orders_view │
  │ restaurants      │             │ restaurant_dashboard │
  │ drivers          │             │ driver_active_view   │
  │ payments         │             │ admin_analytics      │
  │ tracking         │             │ support_lookup       │
  └─────────────────┘              └─────────────────────┘
  Optimized for:                   Optimized for:
  → INSERT/UPDATE                  → SELECT (no JOINs)
  → Data integrity                 → Pre-computed aggregates
  → Transactions                   → Instant responses
  → 1ms writes                     → 2ms reads

🔍 HOW TO APPROACH IT

Step 1: Identify WRITE operations (Commands)
         → "Place Order" — creates records across 5 tables
         → "Update Order Status" — single column update
         → "Assign Driver" — update driver + order
         → These are COMMANDS: "Make this change happen"

Step 2: Identify READ operations (Queries)
         → "Show customer's order history with details"
         → "Show restaurant's pending orders"
         → "Show driver's current deliveries"
         → These are QUERIES: "Tell me what the state is"

Step 3: Design WRITE model (Command side)
         → Normalized tables (3NF)
         → Minimal indexes (fast writes)
         → Strong consistency (ACID transactions)
         → Schema driven by business rules, not UI needs

Step 4: Design READ models (Query side)
         → One denormalized view PER read use case
         → Pre-joined, pre-aggregated data
         → Heavy indexing (fast reads, no write penalty)
         → Schema driven by UI/API response shape
         → Eventual consistency acceptable (seconds of lag)

Step 5: Connect with EVENTS (Project 29)
         → Write side publishes events on every change
         → Read side consumes events and updates its views
         → Events are the BRIDGE between the two worlds
         → Same event bus architecture from Project 29

Step 6: Handle CONSISTENCY gap
         → Write happens → event published → read model updated
         → Between write and read update: INCONSISTENCY WINDOW
         → Typically 10-500ms
         → For most UIs: completely invisible to user
         → For critical reads: query write DB directly (escape hatch)

📌 WHAT COULD BE DONE

Option A: Same DB, Different Tables (Simplest CQRS)
  → Write tables: normalized orders, items, etc.
  → Read tables: denormalized views in SAME database
  → Triggers or application logic syncs them
  → Pro: Simple, single DB transaction possible
  → Con: Still shares DB resources, limited scaling

Option B: Same DB, Materialized Views (Database-Level)
  → Use DB-native materialized views
  → PostgreSQL: CREATE MATERIALIZED VIEW ... 
  → Auto-refreshed on schedule or trigger
  → Pro: DB handles sync logic
  → Con: Refresh can be expensive, limited flexibility

Option C: Separate DBs, Event-Synced (CHOSEN — Full CQRS)
  → Write DB: PostgreSQL (normalized, ACID)
  → Read DB: Different store optimized per use case
     - Redis for real-time dashboards
     - Elasticsearch for search/filtering
     - MongoDB for document-style queries
  → Events (Project 29) bridge the two
  → Pro: Full independence, optimal per use case
  → Con: Eventual consistency, operational complexity

Option D: Event Sourcing + Projections (Strongest)
  → NO separate write DB
  → Event Store IS the write side (Project 29 Event Sourcing)
  → Read models built entirely from event projections
  → Pro: Single source of truth (events), unlimited read models
  → Con: More complex, event store must be reliable

FOR THIS PROJECT:
  → Option C as primary architecture
  → Option D concepts for analytics use case
  → Both demonstrated with working code

✅ WHAT SHOULD DO

1. DESIGN read models from the CONSUMER'S perspective
   → Start with the API response shape / UI wireframe
   → Work BACKWARD to determine what data the view needs
   → NOT: "What tables do I have?" → "How do I join them?"
   → YES: "What does the customer screen show?" → "Store exactly that"

2. CREATE one read model PER use case
   → Customer dashboard ≠ Restaurant dashboard ≠ Admin report
   → Each has different data, different access patterns
   → One view per screen/API endpoint
   → Denormalization is INTENTIONAL, not a mistake

3. MAKE read model updates IDEMPOTENT (Project 29 lesson)
   → Same event processed twice → same read model state
   → Use SET operations, not INCREMENT
   → Or track processed event IDs per view

4. HANDLE the consistency gap explicitly
   → After command: return command result (order_id, status)
   → Client can immediately query write side for confirmation
   → Subsequent reads go to read side (eventually consistent)
   → UI pattern: "Your order has been placed" (from command)
     then dashboard shows it (from read model, ~100ms later)

5. VERSION your read models
   → Read model schema WILL change as UI evolves
   → Version: customer_orders_v1, customer_orders_v2
   → Build v2 from event replay while v1 still serves
   → Atomic switch when v2 is caught up
   → Same as trie swap in Project 28

6. MONITOR read model lag
   → Track: latest event timestamp vs read model timestamp
   → Alert if lag > threshold (e.g., 5 seconds)
   → Lag indicates: projector is slow or crashed
   → Same consumer lag concept from Project 29

7. PROVIDE escape hatch for critical reads
   → Some reads MUST be strongly consistent
   → "Is this order REALLY cancelled?" (before refund)
   → Route these directly to write DB
   → Flag in API: ?consistency=strong → read from write DB
   → ?consistency=eventual (default) → read from read model

8. SEPARATE command validation from query
   → Commands: validate business rules, reject if invalid
   → Queries: just return data, no validation needed
   → Different error handling, different response shapes

❌ WHAT SHOULD NOT DO

1. NEVER use CQRS for simple CRUD applications
   → Blog with 100 users → single DB is perfect
   → CQRS adds complexity that must be JUSTIFIED
   → Rule: if read:write ratio < 10:1 → probably overkill
   → Rule: if < 3 different read patterns → single DB is fine

2. NEVER expect immediate consistency from read models
   → Command succeeds → read model NOT yet updated
   → Designing UI that immediately reads back from read model = bug
   → Pattern: command returns confirmation → UI shows it
   → Background: read model catches up within milliseconds

3. NEVER put business logic in the read model updater
   → Projector should be a DUMB data transformer
   → Event data → read model row (simple mapping)
   → Business rules belong in the COMMAND handler
   → If projector needs business logic → your events are too raw

4. NEVER share the write database directly with read consumers
   → Defeats the entire purpose of CQRS
   → Read queries will contend with writes
   → Schema changes affect both sides
   → The ONLY shared thing is EVENTS

5. NEVER build ONE mega read model for all use cases
   → "Universal view with all columns" = worst of both worlds
   → Each use case needs exactly its own data shape
   → Multiple focused views > one bloated view
   → Storage is cheap, developer time is expensive

6. NEVER skip event replay capability
   → Bug in projector → read model is wrong
   → Must be able to: delete read model → replay all events → rebuild
   → If you can't replay → you can't fix read model bugs
   → Events are the source of truth, read models are disposable

7. NEVER use CQRS without monitoring the consistency gap
   → Invisible lag is the most dangerous lag
   → Customer sees stale data → files support ticket
   → Support sees different data (from write DB) → confusion
   → ALWAYS show "last updated" timestamp on read model data

🚫 WHEN NOT TO USE

1. WHEN YOUR SYSTEM IS SIMPLE
   → 1 service, 1 database, 5 API endpoints
   → Adding CQRS = 2 databases, event bus, projectors, monitoring
   → Complexity cost FAR exceeds benefit
   → Start simple, refactor to CQRS when pain points emerge

2. WHEN STRONG CONSISTENCY IS NON-NEGOTIABLE
   → Banking: "Show balance" must reflect LATEST transaction
   → CQRS read model might be 100ms behind
   → Workaround: read from write DB for critical queries
   → But if MOST reads need strong consistency → CQRS adds complexity
     without solving your actual problem

3. WHEN READ AND WRITE PATTERNS ARE SIMILAR
   → API returns the same shape that was written
   → No complex joins, no aggregations
   → Single table CRUD → CQRS adds zero value

4. WHEN YOU DON'T HAVE EVENT INFRASTRUCTURE
   → CQRS without events = manual sync nightmare
   → Need: event bus (Project 29), idempotent consumers
   → If you don't have this → build it first, then CQRS
   → Or use simple DB triggers (Option A from above)

5. WHEN YOUR TEAM HASN'T BUILT DISTRIBUTED SYSTEMS
   → CQRS introduces: eventual consistency, event ordering,
     projector failures, read model rebuilds, schema evolution
   → Team must understand: "read model might be stale"
   → Debugging is harder: "which side has the bug?"
   → Train team on Projects 11-29 first

MECHANISM 2: PROJECTION (READ MODEL UPDATE)
═══════════════════════════════════════════════════════

PROJECTION FLOW (Event → Read Model):

  EventBus → Projector → ReadStore
  
  The PROJECTOR is a specialized event consumer that:
  1. Subscribes to relevant event types
  2. Transforms event data into read model format
  3. Writes/updates the denormalized read store
  4. Tracks its position (offset) for replay capability

  Example: CustomerOrderView Projector

  Event: OrderPlaced
  {
    order_id: "ORD-123",
    customer_id: "cust-alice",
    items: [{name: "Burger", qty: 2, price: 12.99}],
    restaurant: {id: "rest-001", name: "Bob's Burgers"},
    total: 25.98
  }
  
  Projector transforms to:
  
  READ MODEL ROW:
  {
    order_id: "ORD-123",
    customer_id: "cust-alice",
    restaurant_name: "Bob's Burgers",      ← denormalized
    items_summary: "2x Burger",            ← pre-formatted
    total: "$25.98",                       ← pre-formatted
    status: "Placed",
    status_emoji: "📋",                    ← UI-ready
    driver_name: null,
    eta_minutes: null,
    tracking_url: null,
    placed_at: "12:30 PM",                ← pre-formatted
    last_updated: "12:30 PM"
  }

  Later Event: DriverAssigned
  → Projector UPDATES the existing row:
  → Sets driver_name, eta_minutes

  Later Event: OrderDelivered  
  → Projector UPDATES the row:
  → Sets status="Delivered", delivered_at, removes from active list

  KEY INSIGHT:
  → Read model row looks EXACTLY like the API response
  → Query handler just reads the row and returns it
  → ZERO computation at query time
  → All the "work" happens in the projector (async, background)

MECHANISM 4: READ MODEL REBUILD (Replay)
═══════════════════════════════════════════════════════

  WHEN YOU NEED TO REBUILD:
  → Bug in projector logic → read model has wrong data
  → New read model added → needs historical data
  → Schema change → need to re-project all events

  REBUILD PROCESS:
  
  1. Create new read model (v2) alongside existing (v1)
  2. Start projector for v2 from offset 0 (beginning of event log)
  3. v2 projector replays ALL historical events
  4. v1 continues serving live traffic
  5. When v2 catches up to current offset → switch
  6. Route queries from v1 to v2
  7. Delete v1

  Timeline:
  ─────────────────────────────────────────────────
  T0: v1 serving | v2 starts replay from offset 0
  T1: v1 serving | v2 at offset 50,000 (catching up)
  T2: v1 serving | v2 at offset 99,000 (almost there)
  T3: v1 serving | v2 at offset 100,000 (caught up!)
  T4: SWITCH → v2 now serving
  T5: v1 deleted
  ─────────────────────────────────────────────────

  ZERO DOWNTIME — same principle as trie swap (Project 28)

MECHANISM 5: CONSISTENCY STRATEGIES
═══════════════════════════════════════════════════════

  STRATEGY 1: ACCEPT EVENTUAL CONSISTENCY (Default)
  → Most reads are fine with 100-500ms lag
  → "Show me my orders" — 200ms stale is invisible
  → Simplest, highest throughput

  STRATEGY 2: READ-YOUR-OWN-WRITES
  → After command: client includes "last-write-version" token
  → Query handler checks: is read model at this version?
  → If yes: return from read model
  → If no: wait briefly or fall back to write DB
  → Guarantees: user sees their OWN changes immediately
  → Other users may still see stale data (acceptable)

  STRATEGY 3: CAUSAL CONSISTENCY
  → Extends read-your-own-writes to related operations
  → If Alice placed order → Bob (support agent) should see it
  → Use correlation_id to track causal chain
  → Query includes "must-have-processed-event: EVT-123"

  STRATEGY 4: STRONG CONSISTENCY (Escape Hatch)
  → Route query directly to write database
  → Bypass read model entirely
  → Use sparingly: defeats purpose of CQRS
  → Only for: payment confirmation, cancellation checks
  → API flag: GET /orders/123?consistency=strong

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: READ MODEL PROJECTOR CRASHES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → Projector dies → read model stops updating
  → Users see stale data → "my order still shows preparing"
  → But order was actually delivered 10 minutes ago
  → FIX: Monitor projector health (heartbeat — Project 15)
  → FIX: Monitor lag (offset difference)
  → FIX: Alert if lag > 5 seconds
  → FIX: Auto-restart projector on failure

PITFALL 2: EVENT ORDERING BUG IN PROJECTOR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → Events arrive: OrderPlaced, then OrderCancelled
  → Projector processes OrderCancelled first (out of order)
  → Read model: order doesn't exist yet → error
  → FIX: Same partition key ensures ordering (Project 29)
  → FIX: Projector handles "entity not found" gracefully
  → FIX: Upsert instead of update (create if missing)

PITFALL 3: SCHEMA DRIFT BETWEEN WRITE AND READ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → Write model adds "delivery_instructions" field
  → Event carries this field
  → Projector doesn't know about it → field lost in read model
  → FIX: Projector must evolve with event schema
  → FIX: Versioned events (Project 29)
  → FIX: Automated testing: "all event fields projected?"

PITFALL 4: READ MODEL STORAGE FILLS UP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → Every order creates a read model row
  → 50K orders/minute × 30 days = 2.16 BILLION rows
  → Read model DB runs out of disk
  → FIX: Archival policy (move completed orders to cold storage)
  → FIX: TTL on read model entries
  → FIX: Separate "active" and "historical" read models

PITFALL 5: QUERY THAT NEEDS DATA FROM MULTIPLE READ MODELS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → "Show order with customer loyalty points AND restaurant rating"
  → Loyalty is in loyalty_read_model
  → Rating is in restaurant_read_model
  → Need to JOIN across read models → defeats purpose
  → FIX: Create a COMPOSITE read model for this specific use case
  → FIX: API composition layer (fetch from multiple read models)

PITFALL 6: PROJECTOR BUGS CORRUPT READ MODEL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → Bug: projector calculates wrong total for analytics
  → 3 days of wrong revenue numbers
  → FIX: Read models are DISPOSABLE — rebuild from events
  → FIX: Fix projector bug → replay all events → correct data

class SimpleEventBus:
    """Simplified event bus — focus is on CQRS pattern, not bus internals"""

    def __init__(self):
        self.subscribers: Dict[str, List[Callable]] = defaultdict(list)
        self.published: List[Event] = []
        self._lock = threading.Lock()

    def subscribe(self, event_type: str, handler: Callable):
        self.subscribers[event_type].append(handler)

    def subscribe_all(self, handler: Callable):
        """Subscribe to ALL event types"""
        self.subscribers["*"].append(handler)

    def publish(self, event: Event):
        with self._lock:
            self.published.append(event)

        # Deliver to type-specific subscribers
        for handler in self.subscribers.get(event.event_type, []):
            try:
                handler(event)
            except Exception as e:
                pass  # In production: DLQ (Project 29)

        # Deliver to wildcard subscribers
        for handler in self.subscribers.get("*", []):
            try:
                handler(event)
            except Exception:
                pass

    @property
    def total_published(self) -> int:
        return len(self.published)

class WriteDatabase:
    """
    Simulates a normalized relational database.
    
    Tables:
    → orders: order_id, customer_id, status, total, created_at
    → order_items: item_id, order_id, product_id, name, qty, price
    → customers: customer_id, name, email, phone
    → restaurants: restaurant_id, name, address, cuisine
    → drivers: driver_id, name, status, current_location
    → payments: payment_id, order_id, amount, status, method
    
    Normalized = no redundant data, foreign key relationships.
    Optimized for WRITE operations (INSERT, UPDATE).
    """

    def __init__(self):
        self.tables: Dict[str, Dict[str, Dict]] = {
            "orders": {},
            "order_items": {},
            "customers": {},
            "restaurants": {},
            "drivers": {},
            "payments": {},
            "tracking": {}
        }
        self._lock = threading.Lock()
        self.write_count = 0
        self.write_log: List[Dict] = []

    def insert(self, table: str, record_id: str, data: Dict) -> bool:
        with self._lock:
            if table not in self.tables:
                return False
            self.tables[table][record_id] = {
                **data,
                "_created_at": time.time(),
                "_updated_at": time.time()
            }
            self.write_count += 1
            self.write_log.append({
                "op": "INSERT",
                "table": table,
                "id": record_id,
                "time": time.strftime("%H:%M:%S")
            })
            return True

    def update(self, table: str, record_id: str, updates: Dict) -> bool:
        with self._lock:
            if table not in self.tables:
                return False
            if record_id not in self.tables[table]:
                return False
            self.tables[table][record_id].update(updates)
            self.tables[table][record_id]["_updated_at"] = time.time()
            self.write_count += 1
            self.write_log.append({
                "op": "UPDATE",
                "table": table,
                "id": record_id,
                "fields": list(updates.keys()),
                "time": time.strftime("%H:%M:%S")
            })
            return True

    def get(self, table: str, record_id: str) -> Optional[Dict]:
        with self._lock:
            return deepcopy(
                self.tables.get(table, {}).get(record_id)
            )

    def query(self, table: str, **filters) -> List[Dict]:
        """Simple query with field matching"""
        with self._lock:
            results = []
            for rid, record in self.tables.get(table, {}).items():
                match = all(
                    record.get(k) == v for k, v in filters.items()
                )
                if match:
                    results.append({**record, "_id": rid})
            return deepcopy(results)

    def get_stats(self) -> Dict:
        with self._lock:
            return {
                "total_writes": self.write_count,
                "table_sizes": {
                    t: len(records)
                    for t, records in self.tables.items()
                }
            }

    def print_table(self, table: str, max_rows: int = 10):
        with self._lock:
            records = self.tables.get(table, {})
            print(f"\n  TABLE: {table} ({len(records)} rows)")
            for i, (rid, data) in enumerate(
                list(records.items())[:max_rows]
            ):
                clean = {
                    k: v for k, v in data.items()
                    if not k.startswith("_")
                }
                print(f"  [{rid}] {json.dumps(clean)}")

class CommandResult:
    """Standard result from command execution"""

    def __init__(self, success: bool, data: Dict = None,
                 error: str = ""):

        self.data = data or {}

class OrderCommandHandler:
    """
    Handles all order-related COMMANDS (write operations).
    
    Each command:
    1. VALIDATES input and business rules
    2. WRITES to normalized write database
    3. PUBLISHES event(s) to event bus
    4. RETURNS result to caller
    
    Command handler is the ONLY component that writes to the DB.
    It is the SINGLE SOURCE OF TRUTH for business rules.
    """

    def __init__(self, write_db: WriteDatabase,
                 event_bus: SimpleEventBus):
        self.db = write_db
        self.bus = event_bus
        self._lock = threading.Lock()

    def place_order(self, customer_id: str, customer_name: str,
                    customer_email: str, restaurant_id: str,
                    restaurant_name: str,
                    items: List[Dict]) -> CommandResult:
        """
        Command: Place a new order
        
        Validation:
        → Customer exists (or create)
        → Items are not empty
        → Total is positive
        
        Writes:
        → orders table (1 row)
        → order_items table (N rows)
        → customers table (upsert)
        
        Events:
        → order.placed
        """
        # Validate
        if not items:
            return CommandResult(False, error="No items in order")

        if not customer_id or not restaurant_id:
            return CommandResult(
                False, error="Customer and restaurant required"
            )

        # Calculate total
        total = sum(
            item.get("price", 0) * item.get("quantity", 1)
            for item in items
        )
        if total <= 0:
            return CommandResult(False, error="Invalid order total")

        # Generate IDs
        order_id = f"ORD-{uuid.uuid4().hex[:8].upper()}"
        correlation_id = str(uuid.uuid4())

        # Write to normalized tables (simulated transaction)
        with self._lock:
            # Upsert customer
            self.db.insert("customers", customer_id, {
                "name": customer_name,
                "email": customer_email
            })

            # Insert order
            self.db.insert("orders", order_id, {
                "customer_id": customer_id,
                "restaurant_id": restaurant_id,
                "status": "placed",
                "total": total,
                "item_count": len(items)
            })

            # Insert order items
            for i, item in enumerate(items):
                item_id = f"{order_id}-item-{i}"
                self.db.insert("order_items", item_id, {
                    "order_id": order_id,
                    "product_id": item.get("product_id", ""),
                    "product_name": item.get("name", ""),
                    "quantity": item.get("quantity", 1),
                    "unit_price": item.get("price", 0)
                })

        # Publish event (carries ALL data consumers need)
        event = Event(
            event_type="order.placed",
            aggregate_id=order_id,
            source="order-command-handler",
            correlation_id=correlation_id,
            payload={
                "order_id": order_id,
                "customer_id": customer_id,
                "customer_name": customer_name,
                "customer_email": customer_email,
                "restaurant_id": restaurant_id,
                "restaurant_name": restaurant_name,
                "items": items,
                "total": total,
                "item_count": len(items)
            }
        )
        self.bus.publish(event)

        return CommandResult(True, {
            "order_id": order_id,
            "total": total,
            "status": "placed",
            "correlation_id": correlation_id
        })

    def assign_driver(self, order_id: str, driver_id: str,
                      driver_name: str,
                      eta_minutes: int) -> CommandResult:
        """Command: Assign a driver to an order"""
        order = self.db.get("orders", order_id)
        if not order:
            return CommandResult(False, error="Order not found")

        if order["status"] != "placed":
            return CommandResult(
                False,
                error=f"Cannot assign driver: order is '{order['status']}'"
            )

        # Update order
        self.db.update("orders", order_id, {
            "status": "driver_assigned",
            "driver_id": driver_id
        })

        # Update driver
        self.db.update("drivers", driver_id, {
            "status": "on_delivery",
            "current_order": order_id
        })

        # Publish event
        event = Event(
            event_type="order.driver_assigned",
            aggregate_id=order_id,
            source="order-command-handler",
            payload={
                "order_id": order_id,
                "driver_id": driver_id,
                "driver_name": driver_name,
                "eta_minutes": eta_minutes,
                "customer_id": order["customer_id"]
            }
        )
        self.bus.publish(event)

        return CommandResult(True, {
            "order_id": order_id,
            "driver_id": driver_id,
            "eta_minutes": eta_minutes
        })

    def update_status(self, order_id: str,
                      new_status: str,
                      extra_data: Dict = None) -> CommandResult:
        """Command: Update order status"""
        order = self.db.get("orders", order_id)
        if not order:
            return CommandResult(False, error="Order not found")

        valid_transitions = {
            "placed": ["driver_assigned", "cancelled"],
            "driver_assigned": ["preparing", "cancelled"],
            "preparing": ["ready_for_pickup", "cancelled"],
            "ready_for_pickup": ["picked_up"],
            "picked_up": ["in_transit"],
            "in_transit": ["delivered"],
            "delivered": ["rated"],
        }

        allowed = valid_transitions.get(order["status"], [])
        if new_status not in allowed:
            return CommandResult(
                False,
                error=(f"Invalid transition: "
                       f"'{order['status']}' → '{new_status}'. "
                       f"Allowed: {allowed}")
            )

        self.db.update("orders", order_id, {"status": new_status})

        event = Event(
            event_type=f"order.{new_status}",
            aggregate_id=order_id,
            source="order-command-handler",
            payload={
                "order_id": order_id,
                "old_status": order["status"],
                "new_status": new_status,
                "customer_id": order["customer_id"],
                **(extra_data or {})
            }
        )
        self.bus.publish(event)

        return CommandResult(True, {
            "order_id": order_id,
            "old_status": order["status"],
            "new_status": new_status
        })

    def cancel_order(self, order_id: str,
                     reason: str = "") -> CommandResult:
        """Command: Cancel an order"""
        order = self.db.get("orders", order_id)
        if not order:
            return CommandResult(False, error="Order not found")

        non_cancellable = ["delivered", "rated", "cancelled"]
        if order["status"] in non_cancellable:
            return CommandResult(
                False,
                error=f"Cannot cancel: order is '{order['status']}'"
            )

        self.db.update("orders", order_id, {"status": "cancelled"})

        event = Event(
            event_type="order.cancelled",
            aggregate_id=order_id,
            source="order-command-handler",
            payload={
                "order_id": order_id,
                "reason": reason,
                "total": order["total"],
                "customer_id": order["customer_id"],
                "restaurant_id": order["restaurant_id"]
            }
        )
        self.bus.publish(event)

        return CommandResult(True, {
            "order_id": order_id,
            "status": "cancelled",
            "reason": reason
        })

    def process_payment(self, order_id: str,
                        payment_method: str) -> CommandResult:
        """Command: Process payment for an order"""
        order = self.db.get("orders", order_id)
        if not order:
            return CommandResult(False, error="Order not found")

        payment_id = f"PAY-{uuid.uuid4().hex[:8].upper()}"

        # Simulate payment processing
        payment_success = random.random() > 0.05  # 95% success

        self.db.insert("payments", payment_id, {
            "order_id": order_id,
            "amount": order["total"],
            "method": payment_method,
            "status": "completed" if payment_success else "failed"
        })

        event_type = ("payment.completed" if payment_success
                       else "payment.failed")
        event = Event(
            event_type=event_type,
            aggregate_id=order_id,
            source="payment-handler",
            payload={
                "payment_id": payment_id,
                "order_id": order_id,
                "amount": order["total"],
                "method": payment_method,
                "customer_id": order["customer_id"]
            }
        )
        self.bus.publish(event)

        return CommandResult(payment_success, {
            "payment_id": payment_id,
            "amount": order["total"],
            "status": "completed" if payment_success else "failed"
        })

            self.tables[table][record_id].update(updates)
            self.tables[table][record_id]["_updated_at"] = time.time()
            self.write_count += 1
            return True

    def get(self, table, record_id):
        with self._lock:
            return deepcopy(self.tables.get(table, {}).get(record_id))

    def query(self, table, **filters):
        with self._lock:
            results = []
            for rid, rec in self.tables.get(table, {}).items():
                if all(rec.get(k) == v for k, v in filters.items()):
                    results.append({**rec, "_id": rid})
            return deepcopy(results)

    def get_stats(self):
        with self._lock:
            return {
                "writes": self.write_count,
                "tables": {t: len(r) for t, r in self.tables.items()}
            }

class OrderCommandHandler:
    def __init__(self, write_db: WriteDatabase,
                 event_bus: SimpleEventBus):
        self.db = write_db
        self.bus = event_bus
        self._lock = threading.Lock()

    def place_order(self, customer_id, customer_name,
                    customer_email, restaurant_id,
                    restaurant_name, items):
        if not items:
            return CommandResult(False, error="No items")
        total = sum(i.get("price", 0) * i.get("quantity", 1)
                    for i in items)
        order_id = f"ORD-{uuid.uuid4().hex[:8].upper()}"
        corr_id = str(uuid.uuid4())
        with self._lock:
            self.db.insert("customers", customer_id, {
                "name": customer_name, "email": customer_email
            })
            self.db.insert("orders", order_id, {
                "customer_id": customer_id,
                "restaurant_id": restaurant_id,
                "status": "placed", "total": total,
                "item_count": len(items)
            })
            for i, item in enumerate(items):
                self.db.insert("order_items", f"{order_id}-{i}", {
                    "order_id": order_id,
                    "product_name": item.get("name", ""),
                    "quantity": item.get("quantity", 1),
                    "unit_price": item.get("price", 0)
                })
        self.bus.publish(Event(
            event_type="order.placed",
            aggregate_id=order_id,
            source="order-cmd",
            correlation_id=corr_id,
            payload={
                "order_id": order_id,
                "customer_id": customer_id,
                "customer_name": customer_name,
                "customer_email": customer_email,
                "restaurant_id": restaurant_id,
                "restaurant_name": restaurant_name,
                "items": items, "total": total
            }
        ))
        return CommandResult(True, {
            "order_id": order_id, "total": total,
            "status": "placed", "correlation_id": corr_id
        })

    def assign_driver(self, order_id, driver_id,
                      driver_name, eta_minutes):
        order = self.db.get("orders", order_id)
        if not order:
            return CommandResult(False, error="Order not found")
        self.db.update("orders", order_id, {
            "status": "driver_assigned",
            "driver_id": driver_id
        })
        self.bus.publish(Event(
            event_type="order.driver_assigned",
            aggregate_id=order_id,
            source="order-cmd",
            payload={
                "order_id": order_id,
                "driver_id": driver_id,
                "driver_name": driver_name,
                "eta_minutes": eta_minutes,
                "customer_id": order["customer_id"]
            }
        ))
        return CommandResult(True, {
            "order_id": order_id,
            "driver": driver_name,
            "eta": eta_minutes
        })

    def update_status(self, order_id, new_status, extra_data=None):
        order = self.db.get("orders", order_id)
        if not order:
            return CommandResult(False, error="Order not found")
        valid = {
            "placed": ["driver_assigned", "cancelled"],
            "driver_assigned": ["preparing", "cancelled"],
            "preparing": ["ready_for_pickup"],
            "ready_for_pickup": ["picked_up"],
            "picked_up": ["in_transit"],
            "in_transit": ["delivered"],
            "delivered": ["rated"]
        }
        if new_status not in valid.get(order["status"], []):
            return CommandResult(
                False,
                error=f"Invalid: '{order['status']}' → '{new_status}'"
            )
        self.db.update("orders", order_id, {"status": new_status})
        self.bus.publish(Event(
            event_type=f"order.{new_status}",
            aggregate_id=order_id,
            source="order-cmd",
            payload={
                "order_id": order_id,
                "old_status": order["status"],
                "new_status": new_status,
                "customer_id": order["customer_id"],
                **(extra_data or {})
            }
        ))
        return CommandResult(True, {
            "order_id": order_id,
            "status": new_status
        })

    def cancel_order(self, order_id, reason=""):
        order = self.db.get("orders", order_id)
        if not order:
            return CommandResult(False, error="Order not found")
        if order["status"] in ["delivered", "rated", "cancelled"]:
            return CommandResult(
                False,
                error=f"Cannot cancel: '{order['status']}'"
            )
        self.db.update("orders", order_id, {"status": "cancelled"})
        self.bus.publish(Event(
            event_type="order.cancelled",
            aggregate_id=order_id,
            source="order-cmd",
            payload={
                "order_id": order_id,
                "reason": reason,
                "total": order["total"],
                "customer_id": order["customer_id"],
                "restaurant_id": order["restaurant_id"]
            }
        ))
        return CommandResult(True, {
            "order_id": order_id, "status": "cancelled"
        })

class ReadModelStore:
    """
    Stores denormalized read models.
    
    Unlike WriteDatabase (normalized, many tables):
    → Each "view" is a flat, pre-joined structure
    → Designed to match EXACTLY what the API returns
    → No JOINs needed at query time
    → Different views for different use cases
    
    Simulates Redis/MongoDB/Elasticsearch depending on use case.
    """

    def __init__(self, name: str):
        self.name = name
        self.data: Dict[str, Dict] = {}
        self._lock = threading.Lock()
        self.read_count = 0
        self.write_count = 0
        self.last_updated = 0.0
        self.last_event_id = ""

    def upsert(self, key: str, record: Dict):
        """Insert or update a record (idempotent by design)"""
        with self._lock:
            self.data[key] = {
                **record,
                "_last_updated": time.time(),
                "_last_updated_str": time.strftime("%H:%M:%S")
            }
            self.write_count += 1
            self.last_updated = time.time()

    def update_fields(self, key: str, updates: Dict) -> bool:
        """Update specific fields of an existing record"""
        with self._lock:
            if key not in self.data:
                return False
            self.data[key].update(updates)
            self.data[key]["_last_updated"] = time.time()
            self.data[key]["_last_updated_str"] = time.strftime("%H:%M:%S")
            self.write_count += 1
            self.last_updated = time.time()
            return True

    def delete(self, key: str) -> bool:
        with self._lock:
            if key in self.data:
                del self.data[key]
                self.write_count += 1
                return True
            return False

    def get(self, key: str) -> Optional[Dict]:
        """Get single record — O(1) lookup"""
        with self._lock:
            self.read_count += 1
            record = self.data.get(key)
            return deepcopy(record) if record else None

    def query(self, **filters) -> List[Dict]:
        """Query with field matching — scans all records"""
        with self._lock:
            self.read_count += 1
            results = []
            for key, record in self.data.items():
                match = all(
                    record.get(k) == v for k, v in filters.items()
                )
                if match:
                    results.append({**record, "_key": key})
            return deepcopy(results)

    def get_all(self) -> List[Dict]:
        with self._lock:
            self.read_count += 1
            return [
                {**v, "_key": k}
                for k, v in self.data.items()
            ]

    def aggregate(self, group_by: str, sum_field: str = None,
                  count: bool = True) -> Dict:
        """Simple aggregation — pre-computed in production"""
        with self._lock:
            self.read_count += 1
            groups = defaultdict(lambda: {"count": 0, "sum": 0})
            for record in self.data.values():
                group_val = record.get(group_by, "unknown")
                groups[group_val]["count"] += 1
                if sum_field:
                    groups[group_val]["sum"] += record.get(sum_field, 0)
            return dict(groups)

    def get_stats(self) -> Dict:
        with self._lock:
            lag = time.time() - self.last_updated if self.last_updated else 0
            return {
                "name": self.name,
                "records": len(self.data),
                "reads": self.read_count,
                "writes": self.write_count,
                "lag_seconds": round(lag, 3) if self.last_updated else "N/A"
            }

class IdempotencyTracker:
    """Prevents duplicate event processing (Project 29 concept)"""

    def __init__(self):
        self.processed: OrderedDict = OrderedDict()
        self.duplicates = 0

    def is_duplicate(self, event_id: str) -> bool:
        if event_id in self.processed:
            self.duplicates += 1
            return True
        return False

    def mark_processed(self, event_id: str):
        self.processed[event_id] = time.time()
        if len(self.processed) > 50000:
            self.processed.popitem(last=False)


class CustomerOrderViewProjector:
    """
    Builds the CUSTOMER ORDER DASHBOARD read model.
    
    This is what the customer sees in their app:
    → Order ID, status with emoji
    → Restaurant name
    → Items summary (pre-formatted)
    → Total (pre-formatted)
    → Driver name and ETA
    → Placed/delivered timestamps
    
    ALL of this pre-computed from events.
    Query handler just returns the row — ZERO computation.
    
    Listens to:
    → order.placed → create new row
    → order.driver_assigned → update driver info
    → order.preparing → update status
    → order.in_transit → update status
    → order.delivered → update status + delivery time
    → order.cancelled → update status
    """

    STATUS_EMOJIS = {
        "placed": "📋",
        "driver_assigned": "🚗",
        "preparing": "👨‍🍳",
        "ready_for_pickup": "📦",
        "picked_up": "🏃",
        "in_transit": "🚚",
        "delivered": "✅",
        "cancelled": "❌",
        "rated": "⭐"
    }

    def __init__(self, read_store: ReadModelStore):
        self.store = read_store
        self.idempotency = IdempotencyTracker()
        self.events_processed = 0

    def handle_event(self, event: Event):
        """Route event to appropriate handler"""
        if self.idempotency.is_duplicate(event.event_id):
            return

        handler_map = {
            "order.placed": self._on_order_placed,
            "order.driver_assigned": self._on_driver_assigned,
            "order.preparing": self._on_status_change,
            "order.ready_for_pickup": self._on_status_change,
            "order.picked_up": self._on_status_change,
            "order.in_transit": self._on_status_change,
            "order.delivered": self._on_delivered,
            "order.cancelled": self._on_cancelled,
            "order.rated": self._on_status_change,
        }

        handler = handler_map.get(event.event_type)
        if handler:
            handler(event)
            self.events_processed += 1

        self.idempotency.mark_processed(event.event_id)

    def _on_order_placed(self, event: Event):
        """Create new customer order view row"""
        p = event.payload
        items = p.get("items", [])

        # Pre-format items summary for UI
        items_summary = ", ".join(
            f"{item.get('quantity', 1)}x {item.get('name', '?')}"
            for item in items
        )

        self.store.upsert(event.aggregate_id, {
            "order_id": event.aggregate_id,
            "customer_id": p.get("customer_id"),
            "customer_name": p.get("customer_name"),
            "restaurant_name": p.get("restaurant_name"),
            "items_summary": items_summary,
            "item_count": len(items),

            "status": "placed",

            "driver_name": None,
            "eta_minutes": None,
            "placed_at": event.time_str,
            "delivered_at": None,
            "cancelled_at": None,
            "cancel_reason": None,
            "correlation_id": event.correlation_id
        })

    def _on_driver_assigned(self, event: Event):
        p = event.payload
        status = "driver_assigned"
        self.store.update_fields(event.aggregate_id, {
            "status": status,
            "status_display": f"{self.STATUS_EMOJIS[status]} Driver Assigned",
            "driver_name": p.get("driver_name"),
            "eta_minutes": p.get("eta_minutes")
        })

    def _on_status_change(self, event: Event):
        new_status = event.payload.get("new_status", "unknown")
        emoji = self.STATUS_EMOJIS.get(new_status, "❓")
        display = new_status.replace("_", " ").title()
        self.store.update_fields(event.aggregate_id, {
            "status": new_status,
            "status_display": f"{emoji} {display}"
        })

    def _on_delivered(self, event: Event):
        self.store.update_fields(event.aggregate_id, {
            "status": "delivered",
            "status_display": f"{self.STATUS_EMOJIS['delivered']} Delivered",
            "delivered_at": event.time_str,
            "eta_minutes": 0
        })

    def _on_cancelled(self, event: Event):

            "cancelled_at": event.time_str,

        })


class RestaurantDashboardProjector:
    """
    Builds the RESTAURANT PENDING ORDERS read model.
    
    Restaurant sees:
    → Only ACTIVE orders (not delivered/cancelled)
    → Order details, customer info, special instructions
    → Ordered by time (oldest first = prepare first)
    
    Listens to:
    → order.placed → add to pending
    → order.delivered → remove from pending
    → order.cancelled → remove from pending
    """

    def __init__(self, read_store: ReadModelStore):
        self.store = read_store
        self.idempotency = IdempotencyTracker()
        self.events_processed = 0

    def handle_event(self, event: Event):
        if self.idempotency.is_duplicate(event.event_id):
            return

        if event.event_type == "order.placed":
            self._on_order_placed(event)
        elif event.event_type in ["order.delivered", "order.cancelled"]:
            self._on_order_completed(event)
        elif event.event_type in ["order.preparing",
                                   "order.ready_for_pickup",
                                   "order.driver_assigned"]:
            self._on_status_update(event)

        self.idempotency.mark_processed(event.event_id)
        self.events_processed += 1

    def _on_order_placed(self, event: Event):
        p = event.payload
        items = p.get("items", [])
        items_detail = [
            {"name": i.get("name"), "qty": i.get("quantity", 1),
             "price": i.get("price", 0)}
            for i in items
        ]
        # Key includes restaurant_id for efficient restaurant-specific queries
        key = f"{p.get('restaurant_id')}:{event.aggregate_id}"
        self.store.upsert(key, {
            "order_id": event.aggregate_id,
            "restaurant_id": p.get("restaurant_id"),
            "restaurant_name": p.get("restaurant_name"),
            "customer_name": p.get("customer_name"),
            "items_detail": items_detail,
            "items_summary": ", ".join(
                f"{i['qty']}x {i['name']}" for i in items_detail
            ),
            "total": p.get("total", 0),
            "status": "placed",
            "ordered_at": event.time_str,
            "priority": "normal",
            "is_active": True
        })

    def _on_order_completed(self, event: Event):
        p = event.payload
        restaurant_id = p.get("restaurant_id", "")
        key = f"{restaurant_id}:{event.aggregate_id}"
        # Mark as inactive (or delete)
        self.store.update_fields(key, {
            "is_active": False,
            "status": event.payload.get("new_status", "completed"),
            "completed_at": event.time_str
        })

    def _on_status_update(self, event: Event):
        p = event.payload
        # Try to find the key (restaurant_id might be in event or not)
        for key in list(self.store.data.keys()):
            if key.endswith(f":{event.aggregate_id}"):
                new_status = p.get("new_status", event.event_type.split(".")[-1])
                self.store.update_fields(key, {
                    "status": new_status
                })
                break


class AnalyticsDashboardProjector:
    """
    Builds ADMIN ANALYTICS read model.
    
    Pre-aggregated metrics:
    → Total orders, revenue by status
    → Orders per restaurant
    → Revenue per hour
    → Cancellation rate
    
    Updated INCREMENTALLY per event.
    No need to scan millions of rows for reports.
    """

    def __init__(self, read_store: ReadModelStore):
        self.store = read_store
        self.idempotency = IdempotencyTracker()
        self.events_processed = 0
        self._lock = threading.Lock()

        # Initialize aggregate counters
        self.store.upsert("global_metrics", {
            "total_orders": 0,
            "total_revenue": 0.0,
            "total_cancelled": 0,
            "total_delivered": 0,
            "cancellation_rate": "0.0%",
            "avg_order_value": 0.0,
        })

    def handle_event(self, event: Event):
        if self.idempotency.is_duplicate(event.event_id):
            return

        with self._lock:
            if event.event_type == "order.placed":
                self._on_order_placed(event)
            elif event.event_type == "order.cancelled":
                self._on_cancelled(event)
            elif event.event_type == "order.delivered":
                self._on_delivered(event)

        self.idempotency.mark_processed(event.event_id)
        self.events_processed += 1

    def _on_order_placed(self, event: Event):
        p = event.payload
        total = p.get("total", 0)
        restaurant = p.get("restaurant_name", "unknown")

        # Update global metrics
        metrics = self.store.get("global_metrics") or {}
        total_orders = metrics.get("total_orders", 0) + 1
        total_revenue = metrics.get("total_revenue", 0) + total
        total_cancelled = metrics.get("total_cancelled", 0)
        total_delivered = metrics.get("total_delivered", 0)

        self.store.upsert("global_metrics", {
            "total_orders": total_orders,
            "total_revenue": total_revenue,
            "total_cancelled": total_cancelled,
            "total_delivered": total_delivered,
            "cancellation_rate": (
                f"{total_cancelled / max(1, total_orders):.1%}"
            ),
            "avg_order_value": total_revenue / max(1, total_orders)
        })

        # Update per-restaurant metrics
        rest_key = f"restaurant:{p.get('restaurant_id', 'unknown')}"
        rest_metrics = self.store.get(rest_key) or {
            "restaurant_name": restaurant,
            "order_count": 0,
            "revenue": 0.0
        }
        rest_metrics["order_count"] = rest_metrics.get("order_count", 0) + 1
        rest_metrics["revenue"] = rest_metrics.get("revenue", 0) + total
        self.store.upsert(rest_key, rest_metrics)

        # Update hourly metrics
        hour_key = f"hour:{time.strftime('%Y-%m-%d-%H')}"
        hour_metrics = self.store.get(hour_key) or {
            "hour": time.strftime("%Y-%m-%d %H:00"),
            "order_count": 0,
            "revenue": 0.0
        }
        hour_metrics["order_count"] = hour_metrics.get("order_count", 0) + 1
        hour_metrics["revenue"] = hour_metrics.get("revenue", 0) + total
        self.store.upsert(hour_key, hour_metrics)

    def _on_cancelled(self, event: Event):
        p = event.payload
        total = p.get("total", 0)
        metrics = self.store.get("global_metrics") or {}
        metrics["total_cancelled"] = metrics.get("total_cancelled", 0) + 1
        metrics["total_revenue"] = metrics.get("total_revenue", 0) - total
        total_orders = metrics.get("total_orders", 1)
        metrics["cancellation_rate"] = (
            f"{metrics['total_cancelled'] / max(1, total_orders):.1%}"
        )
        self.store.upsert("global_metrics", metrics)

    def _on_delivered(self, event: Event):
        metrics = self.store.get("global_metrics") or {}
        metrics["total_delivered"] = metrics.get("total_delivered", 0) + 1
        self.store.upsert("global_metrics", metrics)

class CustomerQueryHandler:
    """
    Handles all CUSTOMER read queries.
    
    Reads ONLY from denormalized read model.
    ZERO JOINs. ZERO computation. Just return the row.
    
    Compare to write-side approach:
      Write DB: SELECT o.*, c.*, oi.*, d.* FROM orders o
                JOIN customers c ON ... JOIN order_items oi ON ...
                JOIN drivers d ON ... WHERE o.customer_id = ?
                → 5 table JOIN, 50-200ms
    
      Read Model: GET customer_orders[order_id]
                → Single key lookup, < 1ms
    """

    def __init__(self, customer_view: ReadModelStore,
                 write_db: WriteDatabase = None):
        self.view = customer_view
        self.write_db = write_db  # Escape hatch for strong consistency

    def get_order(self, order_id: str,
                  consistency: str = "eventual") -> Optional[Dict]:
        """
        Get single order details.
        
        consistency="eventual" → read from read model (fast, maybe stale)
        consistency="strong" → read from write DB (slower, always fresh)
        """
        if consistency == "strong" and self.write_db:
            return self._get_from_write_db(order_id)

        record = self.view.get(order_id)
        if record:
            # Remove internal fields
            return {
                k: v for k, v in record.items()
                if not k.startswith("_")
            }
        return None

    def get_customer_orders(self, customer_id: str) -> List[Dict]:
        """Get all orders for a customer"""
        results = self.view.query(customer_id=customer_id)
        # Sort by placed_at descending (newest first)
        results.sort(
            key=lambda x: x.get("placed_at", ""),
            reverse=True
        )
        return [
            {k: v for k, v in r.items() if not k.startswith("_")}
            for r in results
        ]

    def get_active_orders(self, customer_id: str) -> List[Dict]:
        """Get only active (non-delivered, non-cancelled) orders"""
        all_orders = self.get_customer_orders(customer_id)
        active_statuses = [
            "placed", "driver_assigned", "preparing",
            "ready_for_pickup", "picked_up", "in_transit"
        ]
        return [
            o for o in all_orders
            if o.get("status") in active_statuses
        ]

    def _get_from_write_db(self, order_id: str) -> Optional[Dict]:
        """Escape hatch: read directly from write DB"""
        order = self.write_db.get("orders", order_id)
        if not order:
            return None
        # Must manually join data (this is WHY we have read models)
        customer = self.write_db.get(
            "customers", order.get("customer_id", "")
        )
        return {
            "order_id": order_id,
            "status": order.get("status"),
            "total": order.get("total"),
            "customer_name": customer.get("name") if customer else "?",
            "_source": "write_db (strong consistency)"
        }


class RestaurantQueryHandler:
    """Handles restaurant dashboard queries"""

    def __init__(self, restaurant_view: ReadModelStore):

    def get_pending_orders(self, restaurant_id: str) -> List[Dict]:

        results = self.view.query(
            restaurant_id=restaurant_id,

class ConsistencyLagMonitor:
    """
    Tracks the gap between write side and read side.
    
    Measures:
    → Last event published timestamp
    → Last event processed by each projector
    → Difference = consistency lag
    
    Alerts if lag exceeds threshold.
    """

    def __init__(self, event_bus: SimpleEventBus,
                 read_stores: Dict[str, ReadModelStore],
                 alert_threshold_seconds: float = 5.0):
        self.event_bus = event_bus
        self.read_stores = read_stores
        self.alert_threshold = alert_threshold_seconds
        self.alerts: List[Dict] = []

    def check_lag(self) -> Dict:
        """Check current lag for all read models"""
        latest_event_time = 0
        with self.event_bus._lock:
            if self.event_bus.published:
                latest_event_time = self.event_bus.published[-1].timestamp

        if latest_event_time == 0:
            return {"status": "no_events"}

        results = {}
        for name, store in self.read_stores.items():
            store_updated = store.last_updated
            if store_updated == 0:
                lag = float('inf')
            else:
                lag = latest_event_time - store_updated

            lag_ms = lag * 1000 if lag != float('inf') else -1
            is_healthy = lag < self.alert_threshold

            results[name] = {
                "lag_ms": round(lag_ms, 1) if lag_ms >= 0 else "N/A",
                "healthy": is_healthy,
                "last_updated": (
                    time.strftime("%H:%M:%S",
                                  time.localtime(store_updated))
                    if store_updated else "never"
                )
            }

            if not is_healthy:
                self.alerts.append({
                    "time": time.strftime("%H:%M:%S"),
                    "store": name,
                    "lag_ms": lag_ms,
                    "message": f"Lag exceeds {self.alert_threshold}s"
                })

        return results

    def print_lag_report(self):
        results = self.check_lag()
        print(f"\n  {'='*60}")
        print(f"  CONSISTENCY LAG MONITOR")
        print(f"  {'='*60}")
        for name, info in results.items():
            status = "✅" if info.get("healthy", False) else "🔴"
            print(f"  {status} {name:<30} "
                  f"lag={info.get('lag_ms', 'N/A')}ms "
                  f"updated={info.get('last_updated', 'never')}")
        if self.alerts:
            print(f"\n  ⚠️  ALERTS: {len(self.alerts)}")
            for alert in self.alerts[-5:]:
                print(f"  [{alert['time']}] {alert['store']}: "
                      f"{alert['message']}")
        print(f"  {'='*60}")

class CQRSSystem:
    """
    Orchestrates the complete CQRS architecture:
    
    COMMAND SIDE:
    → OrderCommandHandler → WriteDatabase → EventBus
    
    QUERY SIDE:
    → EventBus → Projectors → ReadModelStores → QueryHandlers
    
    MONITORING:
    → ConsistencyLagMonitor tracks read model freshness
    """

    def __init__(self):
        # Infrastructure
        self.event_bus = SimpleEventBus()
        self.write_db = WriteDatabase()

        # Read model stores (one per use case)
        self.customer_view_store = ReadModelStore("customer_orders")
        self.restaurant_view_store = ReadModelStore("restaurant_pending")
        self.analytics_view_store = ReadModelStore("admin_analytics")

        # Projectors (event → read model)
        self.customer_projector = CustomerOrderViewProjector(
            self.customer_view_store
        )
        self.restaurant_projector = RestaurantDashboardProjector(
            self.restaurant_view_store
        )
        self.analytics_projector = AnalyticsDashboardProjector(
            self.analytics_view_store
        )

        # Wire projectors to event bus
        self.event_bus.subscribe_all(self.customer_projector.handle_event)
        self.event_bus.subscribe_all(self.restaurant_projector.handle_event)
        self.event_bus.subscribe_all(self.analytics_projector.handle_event)

        # Command handler
        self.commands = OrderCommandHandler(self.write_db, self.event_bus)

        # Query handlers
        self.customer_queries = CustomerQueryHandler(
            self.customer_view_store, self.write_db
        )
        self.restaurant_queries = RestaurantQueryHandler(
            self.restaurant_view_store
        )
        self.admin_queries = AdminQueryHandler(self.analytics_view_store)

        # Monitoring
        self.lag_monitor = ConsistencyLagMonitor(
            self.event_bus,
            {
                "customer_orders": self.customer_view_store,
                "restaurant_pending": self.restaurant_view_store,
                "admin_analytics": self.analytics_view_store
            }
        )

    def print_full_dashboard(self):
        print(f"\n  {'#'*65}")
        print(f"  #  CQRS SYSTEM DASHBOARD")
        print(f"  {'#'*65}")

        # Write side stats
        w_stats = self.write_db.get_stats()
        print(f"\n  WRITE SIDE:")
        print(f"    Total writes: {w_stats['writes']}")
        print(f"    Tables: {w_stats['tables']}")

        # Event bus stats
        print(f"\n  EVENT BUS:")
        print(f"    Events published: {self.event_bus.total_published}")

        # Read side stats
        print(f"\n  READ SIDE:")
        for store in [self.customer_view_store,
                      self.restaurant_view_store,
                      self.analytics_view_store]:
            s = store.get_stats()
            print(f"    {s['name']}: {s['records']} records | "
                  f"reads={s['reads']} writes={s['writes']} | "
                  f"lag={s['lag_seconds']}s")

        # Projector stats
        print(f"\n  PROJECTORS:")
        print(f"    Customer: {self.customer_projector.events_processed} events")
        print(f"    Restaurant: {self.restaurant_projector.events_processed} events")
        print(f"    Analytics: {self.analytics_projector.events_processed} events")

        # Lag
        self.lag_monitor.print_lag_report()

        print(f"  {'#'*65}")
