PROJECT 32: FEEDENGINE — DESIGN A NEWS FEED / TIMELINE SYSTEM

📋 PHASE 4 (Hard) — Advanced Patterns

🎯 HYPOTHETICAL PROBLEM

You are building a social platform — like Twitter, Instagram, or LinkedIn

→ 500 million users
→ Each user follows anywhere from 10 to 5 million other users
→ Users post text, images, links — call these "posts"
→ When a user opens their app, they expect to see a FEED
   — a chronological or ranked list of posts from people they follow

THE REAL CHALLENGE:

User Alice follows 500 people
→ She opens the app
→ She expects to see the LATEST posts from those 500 people
→ Mixed together, sorted, maybe ranked by relevance
→ In under 200 milliseconds
→ While 100 million OTHER users are doing the EXACT same thing simultaneously

Now consider:
→ Celebrity Beyoncé has 80 million followers
→ She posts one photo
→ That ONE post must appear in 80 MILLION feeds
→ Within seconds

And simultaneously:
→ Regular user Bob has 47 followers
→ He posts a thought
→ That post must appear in 47 feeds
→ Also within seconds

HOW do you architect a system that handles BOTH cases
without melting your servers, bankrupting your budget,
or making users wait 10 seconds for their feed?

This is the News Feed problem — one of the hardest
and most asked system design challenges in the industry.

🔍 HOW TO APPROACH IT

DECOMPOSE THE PROBLEM INTO SUB-PROBLEMS:

Sub-Problem 1: POST CREATION
  → A user creates a post
  → Store it durably
  → Trigger feed distribution

Sub-Problem 2: FEED DISTRIBUTION (the HARD part)
  → How does that post reach every follower's feed?
  → Two fundamental strategies exist (covered below)

Sub-Problem 3: FEED RETRIEVAL
  → User opens app → fetch their personalized feed
  → Must be fast (<200ms)
  → Must be paginated (infinite scroll)

Sub-Problem 4: FEED RANKING
  → Chronological alone is boring
  → Relevance, engagement, recency all matter
  → Ranking model scores each candidate post

Sub-Problem 5: THE CELEBRITY PROBLEM
  → Users with millions of followers break naive approaches
  → Needs special treatment

🛠️ HOW CAN WE SOLVE IT

THE TWO FUNDAMENTAL STRATEGIES:

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STRATEGY A: FAN-OUT-ON-WRITE (Push Model)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When a user POSTS:
  → Immediately look up ALL their followers
  → Write that post ID into EACH follower's pre-computed feed cache
  → When follower opens app → feed is ALREADY ready

  Alice posts ─────→ Find Alice's 500 followers
                     │
                     ├→ Write post_id into Bob's feed cache
                     ├→ Write post_id into Carol's feed cache
                     ├→ Write post_id into Dave's feed cache
                     └→ ... (500 writes total)

  When Bob opens app:
    → Read Bob's feed cache → INSTANT ✅
    → No computation at read time

  PROS:
    ✅ Read is EXTREMELY fast (pre-computed)
    ✅ Feed is ready before user even opens app
    ✅ Simple read path

  CONS:
    ❌ Write amplification — 1 post = N writes (N = follower count)
    ❌ Celebrity problem — Beyoncé posts = 80M writes 💀
    ❌ Wasted work if followers never open their app
    ❌ Feed cache storage grows with every post from every followed user
    ❌ Delayed delivery for high-follower users


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STRATEGY B: FAN-OUT-ON-READ (Pull Model)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When a user POSTS:
  → Just store the post in the posts table
  → Do NOTHING else

When a user READS their feed:
  → Look up everyone they follow
  → Fetch recent posts from EACH followed user
  → Merge + Sort + Rank in real-time
  → Return the feed

  Bob opens app ──→ Find Bob's 500 followees
                    │
                    ├→ Fetch latest posts from Alice
                    ├→ Fetch latest posts from Carol
                    ├→ Fetch latest posts from Dave
                    └→ ... (500 fetches)
                    │
                    ▼
                  Merge + Sort + Rank → Return Feed

  PROS:
    ✅ Write is simple — just store the post
    ✅ No celebrity problem at write time
    ✅ No wasted work for inactive users
    ✅ Always fresh data

  CONS:
    ❌ Read is SLOW — must aggregate from many sources
    ❌ High latency at read time (unacceptable for feeds)
    ❌ Heavy computation for users following many accounts
    ❌ Every feed open = expensive query


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STRATEGY C: HYBRID (What Real Systems Use)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  → For NORMAL users (< 10K followers): Fan-Out-On-Write
      → Their posts get pushed to all followers' caches
      → Fast for both writer and readers

  → For CELEBRITIES (> 10K followers): Fan-Out-On-Read
      → Their posts are NOT pushed
      → When a reader opens feed:
          1. Read their pre-computed cache (normal user posts)
          2. ALSO fetch latest from celebrities they follow
          3. Merge both sets together
          4. Rank and return

  This is EXACTLY what Twitter does.

  ┌──────────────────────────────────────────────────┐
  │                HYBRID MODEL                       │
  │                                                   │
  │  User opens feed:                                 │
  │    1. Read pre-computed cache        → O(1)       │
  │    2. Fetch from N celebrities       → O(N) small │
  │    3. Merge + Rank                   → O(K log K) │
  │    4. Return top K posts             → Fast ✅    │
  │                                                   │
  │  Normal user posts:                               │
  │    → Fan-out to followers' caches    → Background │
  │                                                   │
  │  Celebrity posts:                                 │
  │    → Store only, no fan-out          → Instant    │
  └──────────────────────────────────────────────────┘

📌 WHAT COULD BE DONE

ARCHITECTURE OPTIONS:

Option 1: Monolithic Feed Service
  → One big service handles everything
  → Works for < 1M users
  → Cannot scale independently

Option 2: Microservice Decomposition (OUR APPROACH)
  → Post Service: Handles post creation + storage
  → Social Graph Service: Who follows whom
  → Fan-Out Service: Distributes posts to feeds
  → Feed Cache Service: Stores pre-computed feeds
  → Feed Retrieval Service: Reads + merges + ranks
  → Ranking Service: Scores posts by relevance
  → Each scales independently

Option 3: Event-Driven Pipeline
  → Post creation emits event (from Project 29)
  → Fan-out workers consume events asynchronously
  → Feed cache updated via event handlers
  → Fully decoupled

WE COMBINE Options 2 + 3:
  → Microservices connected via message queue (Project 11)
  → Event-driven fan-out (Project 29)
  → CQRS separation of write/read (Project 30)
  → Caching layer for feeds (Project 5)
  → Consistent hashing for feed cache distribution (Project 14)

✅ WHAT SHOULD DO

1. USE REDIS SORTED SETS FOR FEED CACHE
   → Score = timestamp (or ranking score)
   → Members = post IDs
   → ZREVRANGE gives top-N posts instantly
   → ZADD inserts with O(log N)
   → Trim old entries with ZREMRANGEBYRANK

2. LIMIT FEED CACHE SIZE PER USER
   → Store only latest 500-1000 post IDs per user
   → Nobody scrolls beyond 500 posts
   → Saves enormous amounts of memory

3. STORE ONLY POST IDs IN FEED CACHE
   → NOT full post content
   → When rendering, fetch full post data separately
   → Deduplication, updates, and deletions become easy
   → Post content cached separately (Project 5 pattern)

4. USE ASYNC FAN-OUT WITH MESSAGE QUEUES
   → Post creation returns immediately to author
   → Fan-out happens in background workers
   → Users don't wait for distribution to complete
   → From Project 11: decouple producer from consumer

5. CLASSIFY USERS BY FOLLOWER COUNT
   → Threshold (e.g., 10K followers) determines strategy
   → Below threshold: fan-out-on-write
   → Above threshold: fan-out-on-read
   → Threshold is tunable based on system capacity

6. IMPLEMENT PAGINATION WITH CURSORS
   → NOT offset-based (breaks with new posts arriving)
   → Use cursor = last_seen_post_id or last_seen_score
   → Stable pagination even as feed changes

7. PRE-WARM FEEDS FOR ACTIVE USERS
   → If user is online (has active session from Project 8)
   → Push new posts to their feed cache immediately
   → Inactive users' caches can be built lazily on next login

❌ WHAT SHOULD NOT DO

1. NEVER FAN-OUT SYNCHRONOUSLY IN THE REQUEST PATH
   → Author posts → waits until all 50K followers updated → timeout
   → ALWAYS async via queue
   → Return 201 to author immediately

2. NEVER STORE FULL POST CONTENT IN EVERY FEED CACHE
   → 1 post × 1M followers = 1M copies of same content
   → Store post IDs only → fetch content separately
   → Content is shared, references are per-user

3. NEVER USE FAN-OUT-ON-WRITE FOR ALL USERS
   → Celebrity with 80M followers posts 5 times/day
   → = 400M cache writes/day from ONE user
   → System collapses under write amplification

4. NEVER USE FAN-OUT-ON-READ FOR ALL USERS
   → User follows 500 people, opens app
   → 500 DB queries to build one feed view
   → Latency explodes, DB melts

5. NEVER IGNORE INACTIVE USERS IN FAN-OUT
   → 60% of users may be inactive in any given month
   → Pushing posts to their caches wastes resources
   → Track last_active timestamp
   → Only fan-out to recently active users
   → Build feed on-demand when inactive user returns

6. NEVER PAGINATE WITH OFFSET
   → "Give me posts 20-40" breaks when new posts arrive
   → Post at position 20 shifts to position 21
   → User sees duplicate posts
   → Use cursor-based pagination always

7. NEVER RANK WITHOUT A FALLBACK
   → If ranking service is down, fall back to chronological
   → A slightly-less-relevant feed is better than NO feed
   → Circuit breaker pattern from Project 21

🚫 WHEN NOT TO USE

1. DO NOT BUILD A FEED SYSTEM WHEN:
   → Your platform has < 10K users
   → Simple DB query (SELECT posts WHERE author IN followed_ids ORDER BY created_at)
   → Works perfectly fine up to moderate scale
   → Over-engineering kills small products

2. DO NOT USE FAN-OUT-ON-WRITE WHEN:
   → Average follower count is very high (influencer platform)
   → Write budget is limited
   → Most users are inactive (high waste ratio)

3. DO NOT USE FAN-OUT-ON-READ WHEN:
   → Users follow thousands of accounts
   → Read latency SLA is < 100ms
   → Database cannot handle burst read loads

4. DO NOT USE REAL-TIME PUSH FOR FEEDS WHEN:
   → Feed is not time-critical (LinkedIn vs Twitter)
   → Eventual consistency of 5-30 seconds is acceptable
   → Push infrastructure cost outweighs benefit

5. DO NOT RANK FEEDS WHEN:
   → Your product value IS chronological order
   → Users explicitly chose "Latest" mode
   → You don't have enough engagement data to rank well

  WHY REDIS SORTED SETS:
    → ZADD: Insert with score (timestamp) — O(log N)
    → ZREVRANGE: Get top-N by score (newest first) — O(log N + M)
    → ZREMRANGEBYRANK: Trim oldest entries — O(log N + M)
    → All operations are atomic per key
    → Perfect for ordered, bounded, fast-access lists

STEP 5: FEED RANKING INTERNALS

  For each candidate post, compute a SCORE:

  score = w1 × recency_score
        + w2 × engagement_score
        + w3 × affinity_score
        + w4 × content_type_boost
        + w5 × author_credibility

  WHERE:
    recency_score = decay_function(now - post.created_at)
                  = 1 / (1 + hours_since_posted)

    engagement_score = normalize(likes + 2×comments + 3×shares)
                     = (weighted_engagement) / max_engagement

    affinity_score = how often reader interacts with author
                   = (reader_interactions_with_author) / total_interactions

    content_type_boost = {image: 1.2, video: 1.5, text: 1.0, link: 0.9}

    author_credibility = account_age × verified_bonus × spam_penalty


STEP 6: PAGINATION WITH CURSORS

  First request:
    GET /feed?limit=50
    → Returns posts + cursor = "score:1706140800.123:postid:abc123"

  Next request:
    GET /feed?limit=50&cursor=score:1706140800.123:postid:abc123
    → Server decodes cursor
    → ZREVRANGEBYSCORE feed:{uid} (cursor_score) -inf LIMIT 0 50
    → Returns next page + new cursor

  WHY NOT OFFSET:
    Page 1: posts 0-49
    User scrolls slowly, 10 new posts arrive
    Page 2 with offset 50: now shows posts 50-99
    But posts 40-49 shifted to 50-59 → USER SEES DUPLICATES
    Cursor-based: "give me everything AFTER this specific post" → no dupes


STEP 7: POST DELETION FLOW

  Author deletes post_id = "XYZ":
    1. Mark as deleted in Posts DB (soft delete)
    2. Emit "post_deleted" event to queue
    3. Fan-out workers do NOT remove from every feed cache
       (too expensive — same write amplification problem)
    4. Instead: at read time, hydration step checks post status
       → If deleted → skip it → fetch one more to fill the page
    5. Optionally: lazy cleanup — background job periodically
       scans feed caches and removes deleted post IDs

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: THUNDERING HERD ON CELEBRITY POST
  → Celebrity posts → millions of followers check feed simultaneously
  → All cache misses → all hit DB at once
  → DB overwhelmed
  → FIX: Celebrity posts fetched from separate cache (not per-user)
         Pre-cache celebrity posts with high TTL
         Rate-limit fan-out-on-read queries per celebrity

PITFALL 2: HOT PARTITION IN FEED CACHE
  → Consistent hashing (Project 14) distributes feed caches
  → One Redis node holds feeds for a cluster of active users
  → That node becomes hot
  → FIX: Virtual nodes (from Project 14) spread load
         Replicate hot user feeds across multiple nodes

PITFALL 3: FAN-OUT QUEUE BACKLOG
  → Viral post + many followers = millions of queue messages
  → Workers can't keep up → queue grows → delay increases
  → Users don't see post for minutes/hours
  → FIX: Priority queues (from Project 24)
         High-priority: posts from close friends
         Low-priority: posts from accounts rarely interacted with
         Auto-scale workers based on queue depth

PITFALL 4: STALE FEED AFTER UNFOLLOW
  → User A unfollows User B
  → User B's old posts still in User A's feed cache
  → User A sees posts from someone they unfollowed
  → FIX: On unfollow, remove author's post_ids from follower's feed
         OR: at hydration time, check if reader still follows author
         Lazy approach is cheaper, strict approach is better UX

PITFALL 5: FEED CACHE COLD START
  → New user signs up → follows 200 accounts → opens feed
  → Feed cache is EMPTY (nothing was ever pushed)
  → FIX: On first feed request with empty cache:
         Trigger on-demand fan-out-on-read
         Build initial feed from followed users' recent posts
         Cache the result for future reads
         Same approach for returning inactive users

PITFALL 6: DUPLICATE POSTS IN FEED
  → Network retry causes fan-out worker to process same event twice
  → Same post_id added to feed cache twice
  → FIX: Redis ZADD is idempotent — adding same member just updates score
         Sorted sets naturally deduplicate by member value ✅
         Still need idempotency at the event processing level (Project 29)

PITFALL 7: RANKING LATENCY KILLS FEED SPEED
  → Complex ML ranking model takes 500ms per feed
  → Total feed latency = cache read + celebrity fetch + ranking
  → Exceeds 200ms SLA

class SocialGraphService:

    def __init__(self):
        # user_id → set of user_ids they FOLLOW
        self.following: Dict[str, Set[str]] = defaultdict(set)
        # user_id → set of user_ids who follow THEM
        self.followers: Dict[str, Set[str]] = defaultdict(set)
        self.lock = threading.Lock()

    def follow(self, follower_id: str, followee_id: str):
        with self.lock:
            self.following[follower_id].add(followee_id)
            self.followers[followee_id].add(follower_id)

    def unfollow(self, follower_id: str, followee_id: str):
        with self.lock:
            self.following[follower_id].discard(followee_id)
            self.followers[followee_id].discard(follower_id)

    def get_followers(self, user_id: str) -> Set[str]:
        return self.followers.get(user_id, set()).copy()

    def get_following(self, user_id: str) -> Set[str]:
        return self.following.get(user_id, set()).copy()

    def get_follower_count(self, user_id: str) -> int:
        return len(self.followers.get(user_id, set()))

    def is_following(self, follower_id: str, followee_id: str) -> bool:
        return followee_id in self.following.get(follower_id, set())

class PostStorageService:

        self.lock = threading.Lock()

                    media_type: str = "text") -> Post:
        post_id = f"post_{uuid.uuid4().hex[:12]}"
        post = Post(
            post_id=post_id,
            author_id=author_id,
            content=content,
            media_type=media_type,
            created_at=time.time()
        )
        with self.lock:
            self.posts[post_id] = post
            self.author_posts[author_id].append(post_id)
        return post

    def get_post(self, post_id: str) -> Optional[Post]:
        return self.posts.get(post_id)

    def get_posts_batch(self, post_ids: List[str]) -> List[Post]:
        """Batch fetch — simulates SELECT WHERE post_id IN (...)"""
        results = []
        for pid in post_ids:
            post = self.posts.get(pid)
            if post and not post.is_deleted:
                results.append(post)
        return results

    def get_recent_posts_by_author(self, author_id: str,
                                    limit: int = 10) -> List[Post]:
        """Fetch latest N posts by an author"""
        post_ids = self.author_posts.get(author_id, [])
        recent_ids = post_ids[-limit:][::-1]  # newest first
        return [self.posts[pid] for pid in recent_ids
                if pid in self.posts and not self.posts[pid].is_deleted]

    def delete_post(self, post_id: str) -> bool:
        if post_id in self.posts:
            self.posts[post_id].is_deleted = True
            return True
        return False

    def add_engagement(self, post_id: str, likes: int = 0,
                       comments: int = 0, shares: int = 0):
        if post_id in self.posts:
            self.posts[post_id].likes += likes
            self.posts[post_id].comments += comments
            self.posts[post_id].shares += shares

# SECTION 4: FEED CACHE SERVICE
# Simulates Redis Sorted Sets for per-user feed storage
# ============================================================

class FeedCacheService:
    """
    Simulates Redis Sorted Sets
    Each user has a sorted set: member=post_id, score=timestamp
    Operations mirror ZADD, ZREVRANGE, ZREMRANGEBYRANK
    """

    MAX_FEED_SIZE = 500  # Keep only latest 500 posts per user

    def __init__(self):
        # user_id → list of (score, post_id) — maintained as sorted list
        self.feeds: Dict[str, List[Tuple[float, str]]] = defaultdict(list)
        self.lock = threading.Lock()
        # Metrics
        self.cache_hits = 0
        self.cache_misses = 0

    def add_to_feed(self, user_id: str, post_id: str, score: float):
        """ZADD equivalent — add post to user's feed with score"""
        with self.lock:
            feed = self.feeds[user_id]
            # Check for duplicate (ZADD idempotency)
            for i, (s, pid) in enumerate(feed):
                if pid == post_id:
                    feed[i] = (score, post_id)  # Update score
                    feed.sort(key=lambda x: x[0], reverse=True)
                    return
            # Insert and maintain sorted order (descending by score)
            feed.append((score, post_id))
            feed.sort(key=lambda x: x[0], reverse=True)
            # Trim to max size (ZREMRANGEBYRANK equivalent)
            if len(feed) > self.MAX_FEED_SIZE:
                self.feeds[user_id] = feed[:self.MAX_FEED_SIZE]

    def add_to_feed_batch(self, user_id: str,
                          items: List[Tuple[float, str]]):
        """Batch ZADD — used during fan-out and cold start"""
        with self.lock:
            feed = self.feeds[user_id]
            existing_pids = {pid for _, pid in feed}
            for score, post_id in items:
                if post_id not in existing_pids:
                    feed.append((score, post_id))
                    existing_pids.add(post_id)
            feed.sort(key=lambda x: x[0], reverse=True)
            if len(feed) > self.MAX_FEED_SIZE:
                self.feeds[user_id] = feed[:self.MAX_FEED_SIZE]

    def get_feed(self, user_id: str, offset: int = 0,
                 limit: int = 50) -> List[Tuple[float, str]]:
        """ZREVRANGE equivalent — get top-N posts by score"""
        feed = self.feeds.get(user_id, [])
        if feed:
            self.cache_hits += 1
        else:
            self.cache_misses += 1
        return feed[offset:offset + limit]

    def get_feed_cursor(self, user_id: str, cursor_score: float,
                        limit: int = 50) -> List[Tuple[float, str]]:
        """Cursor-based pagination — get posts with score < cursor"""
        feed = self.feeds.get(user_id, [])
        result = []
        for score, post_id in feed:
            if score < cursor_score:
                result.append((score, post_id))
                if len(result) >= limit:
                    break
        if result:
            self.cache_hits += 1
        else:
            self.cache_misses += 1
        return result

    def remove_from_feed(self, user_id: str, post_id: str):
        """Remove specific post from user's feed"""
        with self.lock:
            if user_id in self.feeds:
                self.feeds[user_id] = [
                    (s, pid) for s, pid in self.feeds[user_id]
                    if pid != post_id
                ]

    def remove_author_from_feed(self, user_id: str,
                                 author_post_ids: Set[str]):
        """Remove all posts by an author from feed (on unfollow)"""
        with self.lock:
            if user_id in self.feeds:
                self.feeds[user_id] = [
                    (s, pid) for s, pid in self.feeds[user_id]
                    if pid not in author_post_ids
                ]

    def feed_exists(self, user_id: str) -> bool:
        return user_id in self.feeds and len(self.feeds[user_id]) > 0

    def get_cache_stats(self) -> dict:
        total = self.cache_hits + self.cache_misses
        hit_rate = (self.cache_hits / total * 100) if total > 0 else 0
        return {
            "cache_hits": self.cache_hits,
            "cache_misses": self.cache_misses,
            "hit_rate_pct": round(hit_rate, 2),
            "total_feeds_cached": len(self.feeds),
            "total_entries": sum(len(f) for f in self.feeds.values())
        }

class UserClassificationService:

    CELEBRITY_THRESHOLD = 100  # Low for demo; real = 10,000+
    INACTIVE_DAYS = 30

    def __init__(self, social_graph: SocialGraphService,
                 users: Dict[str, User]):
        self.social_graph = social_graph
        self.users = users

    def get_fan_out_strategy(self, author_id: str) -> FanOutStrategy:
        follower_count = self.social_graph.get_follower_count(author_id)
        if follower_count > self.CELEBRITY_THRESHOLD:
            return FanOutStrategy.PULL
        return FanOutStrategy.PUSH

    def is_user_active(self, user_id: str) -> bool:
        user = self.users.get(user_id)
        if not user:
            return False
        days_inactive = (time.time() - user.last_active) / 86400
        return days_inactive < self.INACTIVE_DAYS

    def get_celebrity_followees(self, user_id: str) -> List[str]:
        """Get list of celebrities that this user follows"""
        following = self.social_graph.get_following(user_id)
        celebrities = []
        for followee_id in following:
            if self.get_fan_out_strategy(followee_id) == FanOutStrategy.PULL:
                celebrities.append(followee_id)
        return celebrities

# SECTION 6: FEED RANKING SERVICE
# Scores posts by relevance
# ============================================================

class FeedRankingService:

    # Weights for ranking formula
    W_RECENCY = 0.35
    W_ENGAGEMENT = 0.25
    W_AFFINITY = 0.20
    W_CONTENT_TYPE = 0.10
    W_CREDIBILITY = 0.10

    CONTENT_BOOST = {
        "video": 1.5,
        "image": 1.3,
        "text": 1.0,
        "link": 0.9
    }

    def __init__(self, users: Dict[str, User]):
        self.users = users
        # user_id → {interacted_user_id: count}
        self.interaction_history: Dict[str, Dict[str, int]] = defaultdict(
            lambda: defaultdict(int)
        )

    def record_interaction(self, user_id: str, author_id: str):
        """Track when user interacts with an author's content"""
        self.interaction_history[user_id][author_id] += 1

    def compute_score(self, post: Post, reader_id: str) -> float:
        """Compute ranking score for a post relative to a reader"""

        # 1. Recency Score: exponential decay
        hours_ago = (time.time() - post.created_at) / 3600
        recency = 1.0 / (1.0 + hours_ago)

        # 2. Engagement Score: weighted and normalized
        raw_engagement = (
            post.likes +
            2 * post.comments +
            3 * post.shares
        )
        engagement = math.log1p(raw_engagement) / 10.0  # Normalize
        engagement = min(engagement, 1.0)

        # 3. Affinity Score: how much reader interacts with author
        interactions = self.interaction_history.get(reader_id, {})
        author_interactions = interactions.get(post.author_id, 0)
        total_interactions = sum(interactions.values()) if interactions else 1
        affinity = author_interactions / max(total_interactions, 1)

        # 4. Content Type Boost
        content_boost = self.CONTENT_BOOST.get(post.media_type, 1.0)
        content_score = content_boost / 1.5  # Normalize to 0-1

        # 5. Author Credibility
        author = self.users.get(post.author_id)
        credibility = 0.5  # Default
        if author:
            age_factor = min(author.account_age_days / 365, 1.0)
            verified_bonus = 1.3 if author.is_verified else 1.0
            credibility = age_factor * verified_bonus
            credibility = min(credibility, 1.0)

        # Weighted sum
        score = (
            self.W_RECENCY * recency +
            self.W_ENGAGEMENT * engagement +
            self.W_AFFINITY * affinity +
            self.W_CONTENT_TYPE * content_score +
            self.W_CREDIBILITY * credibility
        )

        return round(score, 6)

    def rank_posts(self, posts: List[Post],
                   reader_id: str) -> List[Tuple[float, Post]]:
        """Score and rank a list of posts for a specific reader"""
        scored = []
        for post in posts:
            score = self.compute_score(post, reader_id)
            scored.append((score, post))
        scored.sort(key=lambda x: x[0], reverse=True)
        return scored

class MessageQueue:
    """Simplified async queue — represents Kafka/RabbitMQ"""

    def __init__(self):
        self.queue: deque = deque()
        self.lock = threading.Lock()
        self.processed_count = 0

    def publish(self, event: FanOutEvent):
        with self.lock:
            self.queue.append(event)

    def consume(self) -> Optional[FanOutEvent]:
        with self.lock:
            if self.queue:
                self.processed_count += 1
                return self.queue.popleft()
            return None

    def size(self) -> int:
        return len(self.queue)

    def stats(self) -> dict:
        return {
            "pending": len(self.queue),
            "processed": self.processed_count
        }

# SECTION 8: FAN-OUT SERVICE
# The core distribution engine
# ============================================================

class FanOutService:

    def __init__(
        self,
        social_graph: SocialGraphService,
        feed_cache: FeedCacheService,
        classifier: UserClassificationService,
        queue: MessageQueue
    ):
        self.social_graph = social_graph
        self.feed_cache = feed_cache
        self.classifier = classifier
        self.queue = queue
        # Metrics
        self.fan_out_count = 0
        self.fan_out_writes = 0
        self.skipped_inactive = 0
        self.celebrity_skips = 0

    def handle_new_post(self, post: Post):
        """Called when a new post is created — decides strategy"""
        strategy = self.classifier.get_fan_out_strategy(post.author_id)
        event = FanOutEvent(
            event_id=f"evt_{uuid.uuid4().hex[:8]}",
            post_id=post.post_id,
            author_id=post.author_id,
            timestamp=post.created_at,
            strategy=strategy
        )
        self.queue.publish(event)

    def process_events(self, max_events: int = 1000):
        """Worker loop — processes fan-out events from queue"""
        processed = 0
        while processed < max_events:
            event = self.queue.consume()
            if event is None:
                break
            self._execute_fan_out(event)
            processed += 1
        return processed

    def _execute_fan_out(self, event: FanOutEvent):
        """Execute fan-out for a single event"""
        self.fan_out_count += 1

        if event.strategy == FanOutStrategy.PULL:
            # Celebrity — do NOT fan out
            # Post is stored, followers will pull at read time
            self.celebrity_skips += 1
            return

        # Normal user — fan out on write
        followers = self.social_graph.get_followers(event.author_id)

        for follower_id in followers:
            # Skip inactive users
            if not self.classifier.is_user_active(follower_id):
                self.skipped_inactive += 1
                continue

            # Write post_id to follower's feed cache
            self.feed_cache.add_to_feed(
                user_id=follower_id,
                post_id=event.post_id,
                score=event.timestamp
            )
            self.fan_out_writes += 1

    def get_metrics(self) -> dict:
        return {
            "total_fan_outs": self.fan_out_count,
            "total_cache_writes": self.fan_out_writes,
            "skipped_inactive": self.skipped_inactive,
            "celebrity_skips": self.celebrity_skips,
            "queue_stats": self.queue.stats()
        }

class FeedRetrievalService:

    def __init__(
        self,
        feed_cache: FeedCacheService,
        post_storage: PostStorageService,
        social_graph: SocialGraphService,
        classifier: UserClassificationService,
        ranking_service: FeedRankingService
    ):
        self.feed_cache = feed_cache
        self.post_storage = post_storage
        self.social_graph = social_graph
        self.classifier = classifier
        self.ranking_service = ranking_service
        # Metrics
        self.feed_reads = 0
        self.cold_starts = 0
        self.avg_latency_ms = 0.0

    def get_feed(self, user_id: str, limit: int = 20,
                 cursor_score: Optional[float] = None,
                 use_ranking: bool = True) -> dict:
        """
        Main feed retrieval — the function users call when opening the app
        Returns ranked, paginated feed with metadata
        """
        start_time = time.time()
        self.feed_reads += 1

        # ---- STEP 1: Read from pre-computed cache ----
        if cursor_score:
            cached_items = self.feed_cache.get_feed_cursor(
                user_id, cursor_score, limit=limit * 2
            )
        else:
            cached_items = self.feed_cache.get_feed(
                user_id, offset=0, limit=limit * 2
            )

        # ---- STEP 2: Cold start handling ----
        if not cached_items and not cursor_score:
            self.cold_starts += 1
            cached_items = self._build_cold_start_feed(user_id, limit * 2)

        # ---- STEP 3: Fetch celebrity posts (fan-out-on-read) ----
        celebrity_items = self._fetch_celebrity_posts(user_id, limit)

        # ---- STEP 4: Merge cached + celebrity items ----
        all_post_ids = set()
        merged_items = []

        for score, post_id in cached_items:
            if post_id not in all_post_ids:
                merged_items.append(FeedItem(post_id, score, "cache"))
                all_post_ids.add(post_id)

        for score, post_id in celebrity_items:
            if post_id not in all_post_ids:
                merged_items.append(FeedItem(post_id, score, "celebrity_pull"))
                all_post_ids.add(post_id)

        # ---- STEP 5: Hydrate — fetch full post data ----
        post_ids_to_fetch = [item.post_id for item in merged_items]
        posts = self.post_storage.get_posts_batch(post_ids_to_fetch)
        post_map = {p.post_id: p for p in posts}

        # Filter out deleted posts
        valid_items = [
            item for item in merged_items
            if item.post_id in post_map
        ]

        # ---- STEP 6: Rank (or sort chronologically) ----
        if use_ranking and valid_items:
            valid_posts = [post_map[item.post_id] for item in valid_items]
            ranked = self.ranking_service.rank_posts(valid_posts, user_id)
            final_posts = [post for score, post in ranked[:limit]]
        else:
            valid_items.sort(key=lambda x: x.score, reverse=True)
            final_posts = [
                post_map[item.post_id]
                for item in valid_items[:limit]
                if item.post_id in post_map
            ]

        # ---- STEP 7: Build response with cursor ----
        next_cursor = None
        if final_posts:
            last_post = final_posts[-1]
            next_cursor = last_post.created_at

        latency_ms = (time.time() - start_time) * 1000
        self._update_avg_latency(latency_ms)

        return {
            "user_id": user_id,
            "posts": final_posts,
            "count": len(final_posts),
            "next_cursor": next_cursor,
            "has_more": len(valid_items) > limit,
            "sources": {
                "from_cache": sum(
                    1 for i in valid_items if i.source == "cache"
                ),
                "from_celebrity_pull": sum(
                    1 for i in valid_items if i.source == "celebrity_pull"
                )
            },
            "latency_ms": round(latency_ms, 2),
            "was_cold_start": self.cold_starts == self.feed_reads
        }

    def _fetch_celebrity_posts(self, user_id: str,
                                limit: int) -> List[Tuple[float, str]]:
        """Fan-out-on-read for celebrities the user follows"""
        celebrity_ids = self.classifier.get_celebrity_followees(user_id)
        results = []

        for celeb_id in celebrity_ids:
            recent = self.post_storage.get_recent_posts_by_author(
                celeb_id, limit=5
            )
            for post in recent:
                results.append((post.created_at, post.post_id))

        # Sort by timestamp descending, take top N
        results.sort(key=lambda x: x[0], reverse=True)
        return results[:limit]

    def _build_cold_start_feed(self, user_id: str,
                                limit: int) -> List[Tuple[float, str]]:
        """Build feed from scratch for new/returning users"""
        following = self.social_graph.get_following(user_id)
        all_items = []

        for followee_id in following:
            recent = self.post_storage.get_recent_posts_by_author(
                followee_id, limit=5
            )
            for post in recent:
                all_items.append((post.created_at, post.post_id))

        all_items.sort(key=lambda x: x[0], reverse=True)
        top_items = all_items[:limit]

        # Cache this for future reads
        if top_items:
            self.feed_cache.add_to_feed_batch(user_id, top_items)

        return top_items

    def _update_avg_latency(self, new_latency_ms: float):
        alpha = 0.1  # Exponential moving average
        self.avg_latency_ms = (
            alpha * new_latency_ms +
            (1 - alpha) * self.avg_latency_ms
        )

class FeedEngine:
    """
    The main entry point — orchestrates all services
    This is what the API Gateway (Project 19) would call
    """

    def __init__(self):
        # ---- Initialize all services ----
        self.users: Dict[str, User] = {}
        self.social_graph = SocialGraphService()
        self.post_storage = PostStorageService()
        self.feed_cache = FeedCacheService()
        self.message_queue = MessageQueue()

        self.classifier = UserClassificationService(
            social_graph=self.social_graph,
            users=self.users
        )

        self.ranking_service = FeedRankingService(users=self.users)

        self.fan_out_service = FanOutService(
            social_graph=self.social_graph,
            feed_cache=self.feed_cache,
            classifier=self.classifier,
            queue=self.message_queue
        )

        self.feed_retrieval = FeedRetrievalService(
            feed_cache=self.feed_cache,
            post_storage=self.post_storage,
            social_graph=self.social_graph,
            classifier=self.classifier,
            ranking_service=self.ranking_service
        )

        # System metrics
        self.total_posts_created = 0
        self.total_feeds_served = 0

    # ---- USER MANAGEMENT ----

    def register_user(self, user_id: str, username: str,
                      is_verified: bool = False,
                      account_age_days: int = 365) -> User:
        user = User(
            user_id=user_id,
            username=username,
            is_verified=is_verified,
            account_age_days=account_age_days,
            last_active=time.time()
        )
        self.users[user_id] = user
        return user

    def follow_user(self, follower_id: str, followee_id: str):
        if follower_id not in self.users or followee_id not in self.users:
            return {"error": "User not found"}
        if follower_id == followee_id:
            return {"error": "Cannot follow yourself"}

        self.social_graph.follow(follower_id, followee_id)

        # Update cached counts
        self.users[followee_id].follower_count = (
            self.social_graph.get_follower_count(followee_id)
        )
        self.users[follower_id].following_count = (
            len(self.social_graph.get_following(follower_id))
        )

        return {
            "status": "followed",
            "follower": follower_id,
            "followee": followee_id
        }

    def unfollow_user(self, follower_id: str, followee_id: str):
        self.social_graph.unfollow(follower_id, followee_id)

        # Update counts
        self.users[followee_id].follower_count = (
            self.social_graph.get_follower_count(followee_id)
        )
        self.users[follower_id].following_count = (
            len(self.social_graph.get_following(follower_id))
        )

        # Remove unfollowed user's posts from feed cache
        author_posts = self.post_storage.author_posts.get(followee_id, [])
        if author_posts:
            self.feed_cache.remove_author_from_feed(
                follower_id, set(author_posts)
            )

        return {
            "status": "unfollowed",
            "follower": follower_id,
            "followee": followee_id
        }

    # ---- POST CREATION ----

    def create_post(self, author_id: str, content: str,
                    media_type: str = "text") -> dict:
        if author_id not in self.users:
            return {"error": "User not found"}

        # Mark user as active
        self.users[author_id].last_active = time.time()

        # Store the post
        post = self.post_storage.create_post(author_id, content, media_type)
        self.total_posts_created += 1

        # Determine fan-out strategy
        strategy = self.classifier.get_fan_out_strategy(author_id)

        # Emit fan-out event to queue (async — author doesn't wait)
        self.fan_out_service.handle_new_post(post)

        return {
            "status": "created",
            "post_id": post.post_id,
            "author_id": author_id,
            "strategy": strategy.value,
            "follower_count": self.social_graph.get_follower_count(author_id)
        }

    # ---- PROCESS FAN-OUT QUEUE (simulates background workers) ----

    def process_fan_out_queue(self):
        """
        In production: separate worker processes consume the queue
        Here we call it explicitly to simulate async processing
        """
        processed = self.fan_out_service.process_events(max_events=10000)
        return {"events_processed": processed}

    # ---- FEED RETRIEVAL ----

    def get_user_feed(self, user_id: str, limit: int = 10,
                      cursor: float = None,
                      use_ranking: bool = True) -> dict:
        if user_id not in self.users:
            return {"error": "User not found"}

        # Mark user as active
        self.users[user_id].last_active = time.time()
        self.total_feeds_served += 1

        result = self.feed_retrieval.get_feed(
            user_id=user_id,
            limit=limit,
            cursor_score=cursor,
            use_ranking=use_ranking
        )

        return result

    # ---- ENGAGEMENT ----

    def like_post(self, user_id: str, post_id: str):
        post = self.post_storage.get_post(post_id)
        if not post:
            return {"error": "Post not found"}
        self.post_storage.add_engagement(post_id, likes=1)
        # Track interaction for affinity ranking
        self.ranking_service.record_interaction(user_id, post.author_id)
        return {"status": "liked", "post_id": post_id}

    def delete_post(self, post_id: str) -> dict:
        success = self.post_storage.delete_post(post_id)
        if success:
            return {"status": "deleted", "post_id": post_id}
        return {"error": "Post not found"}

    # ---- SYSTEM METRICS ----

    def get_system_metrics(self) -> dict:
        return {
            "users": {
                "total_registered": len(self.users),
                "verified": sum(
                    1 for u in self.users.values() if u.is_verified
                ),
                "celebrities": sum(
                    1 for u in self.users.values()
                    if u.follower_count > UserClassificationService.CELEBRITY_THRESHOLD
                )
            },
            "posts": {
                "total_created": self.total_posts_created,
                "total_stored": len(self.post_storage.posts)
            },
            "feeds": {
                "total_served": self.total_feeds_served,
                "avg_latency_ms": round(
                    self.feed_retrieval.avg_latency_ms, 2
                ),
                "cold_starts": self.feed_retrieval.cold_starts
            },
            "cache": self.feed_cache.get_cache_stats(),
            "fan_out": self.fan_out_service.get_metrics(),
            "social_graph": {
                "total_follow_relationships": sum(
                    len(f) for f in self.social_graph.followers.values()
                )
            }
        }

class FeedDisplayFormatter:

    @staticmethod
    def format_feed(feed_result: dict, engine: FeedEngine) -> str:
        lines = []
        lines.append("")
        lines.append("=" * 65)

        user = engine.users.get(feed_result["user_id"])
        username = user.username if user else "Unknown"
        lines.append(f"  📰 FEED FOR: @{username} ({feed_result['user_id']})")
        lines.append(f"  Posts: {feed_result['count']} | "
                     f"Latency: {feed_result['latency_ms']}ms")
        lines.append(f"  Sources → Cache: {feed_result['sources']['from_cache']}"
                     f" | Celebrity Pull: "
                     f"{feed_result['sources']['from_celebrity_pull']}")
        lines.append("=" * 65)

        if not feed_result["posts"]:
            lines.append("  (empty feed — follow some users!)")
        else:
            for i, post in enumerate(feed_result["posts"], 1):
                author = engine.users.get(post.author_id)
                author_name = author.username if author else "?"
                verified = " ✓" if (author and author.is_verified) else ""
                media_icon = {
                    "text": "📝", "image": "🖼️",
                    "video": "🎬", "link": "🔗"
                }.get(post.media_type, "📝")

                age_secs = time.time() - post.created_at
                if age_secs < 60:
                    age_str = f"{int(age_secs)}s ago"
                elif age_secs < 3600:
                    age_str = f"{int(age_secs/60)}m ago"
                else:
                    age_str = f"{int(age_secs/3600)}h ago"

                lines.append(
                    f"  {i:2d}. {media_icon} @{author_name}{verified} "
                    f"({age_str})"
                )
                lines.append(
                    f"      \"{post.content}\""
                )
                lines.append(
                    f"      ❤️ {post.likes}  "
                    f"💬 {post.comments}  "
                    f"🔄 {post.shares}"
                )
                lines.append(f"      [ID: {post.post_id}]")
                if i < len(feed_result["posts"]):
                    lines.append("      " + "-" * 45)

        lines.append("=" * 65)

        if feed_result.get("next_cursor"):
            lines.append(
                f"  📄 Next page cursor: {feed_result['next_cursor']}"
            )

        lines.append("")
        return "\n".join(lines)

    @staticmethod
    def format_metrics(metrics: dict) -> str:
        lines = []
        lines.append("")
        lines.append("=" * 65)
        lines.append("  📊 SYSTEM METRICS")
        lines.append("=" * 65)

        for category, values in metrics.items():
            lines.append(f"\n  ┌─ {category.upper()}")
            if isinstance(values, dict):
                for key, val in values.items():
                    lines.append(f"  │  {key}: {val}")
            else:
                lines.append(f"  │  {values}")
            lines.append(f"  └{'─' * 40}")

        lines.append("")
        return "\n".join(lines)
