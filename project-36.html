HOW do you build a scheduler that handles millions of tasks,

🛠️ HOW CAN WE SOLVE IT

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PART A: TIMER DATA STRUCTURES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

OPTION 1: MIN-HEAP (Priority Queue)
  → Store all tasks in a min-heap ordered by fire_time
  → Peek: O(1) — see the soonest task
  → Pop: O(log N) — remove the soonest task
  → Insert: O(log N) — add new task
  → Cancel: O(N) — find and remove specific task (expensive!)
  
  GOOD FOR: Small to medium task counts (< 1M)
  BAD FOR: Frequent cancellations, very large task counts


OPTION 2: SORTED SET (Redis ZSET)
  → Score = fire_time (Unix timestamp)
  → Member = task_id
  → ZRANGEBYSCORE 0 {now}: get all tasks due
  → ZADD: O(log N) insert
  → ZREM: O(log N) cancel
  → Range query: O(log N + K) where K = results
  
  GOOD FOR: Distributed systems, moderate scale
  BAD FOR: Very high throughput (Redis single-threaded)


OPTION 3: HIERARCHICAL TIME WHEEL (Our Primary Choice)

  Inspired by Linux kernel timer implementation

  Imagine a CLOCK with 60 slots (like seconds on a clock face):

  WHEEL LEVEL 1 (seconds): 60 slots, each = 1 second
  ┌──┬──┬──┬──┬──┬──┬──┬──┬──┬──┐
  │s0│s1│s2│s3│s4│s5│..│..│s58│s59│
  └──┴──┴──┴──┴──┴──┴──┴──┴──┴──┘
       ↑ current = s1

  WHEEL LEVEL 2 (minutes): 60 slots, each = 1 minute
  ┌──┬──┬──┬──┬──┬──┬──┬──┬──┬──┐
  │m0│m1│m2│m3│m4│..│..│..│m58│m59│
  └──┴──┴──┴──┴──┴──┴──┴──┴──┴──┘

  WHEEL LEVEL 3 (hours): 24 slots, each = 1 hour
  ┌──┬──┬──┬──┬──┬──┬──┬──┬──┐
  │h0│h1│h2│h3│..│..│..│h22│h23│
  └──┴──┴──┴──┴──┴──┴──┴──┴──┘

  WHEEL LEVEL 4 (days): 30 slots, each = 1 day

  HOW IT WORKS:
    Task fires in 3 minutes 45 seconds:
      → Goes into minute wheel, slot m3
      → When minute hand reaches m3:
          → Move all tasks from m3 into second wheel
          → Task now in second wheel slot s45
      → When second hand reaches s45:
          → FIRE the task!

    Task fires in 2 hours 15 minutes:
      → Goes into hour wheel, slot h2
      → When hour hand reaches h2:
          → Cascade into minute wheel
      → Then cascade into second wheel
      → Then fire

  COMPLEXITY:
    Insert: O(1) — just put in correct slot
    Fire: O(1) per task — just pop from current slot
    Cascade: O(K) where K = tasks in the cascading slot
    Cancel: O(1) with task-to-slot lookup map
    Memory: O(N) for N active tasks + O(S) for S total slots

  WHY IT'S BRILLIANT:
    → Most operations are O(1)
    → Near-term tasks (most common) are in the fastest wheel
    → Far-future tasks cascade down as time approaches
    → Memory efficient: only store tasks, not empty time slots

PART B: EXACTLY-ONCE EXECUTION VIA LEASING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  THE PROBLEM:
    Task T is due → Scheduler fires it → Worker W1 picks it up
    → W1 starts executing → W1 crashes at 50% complete
    → How does the system know to retry?
    → How to prevent W2 from running it simultaneously?

  THE SOLUTION: LEASE-BASED EXECUTION

    STEP 1: Task becomes due → Scheduler marks it as "LEASED"
      → lease_owner = worker_id
      → lease_expiry = now + 30 seconds
      → lease_token = unique random token

    STEP 2: Worker executes the task
      → Must complete before lease expires
      → On success: mark task as COMPLETED with lease_token
      → Lease_token ensures only the lease holder can complete it

    STEP 3: If worker crashes:
      → Lease expires after 30 seconds
      → Scheduler sees: task LEASED but lease expired
      → Scheduler marks it as PENDING again
      → Another worker picks it up

    STEP 4: Idempotency guard
      → Task handler must be IDEMPOTENT (safe to run twice)
      → Use task_id as deduplication key
      → Check: "has this task_id already been processed?"
      → From Project 29: idempotent event processing

  LEASE STATE MACHINE:
    PENDING → LEASED → COMPLETED
                ↓ (lease expired)
              PENDING (retry)
                ↓ (max retries)
              DEAD_LETTERED

PART C: TASK SHARDING FOR DISTRIBUTION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  50M tasks across 20 scheduler nodes:
  → Each node manages ~2.5M tasks

  SHARDING STRATEGIES:

  Strategy 1: SHARD BY TIME BUCKET
    → Divide time into 1-minute buckets
    → Bucket "2025-01-15T14:30" → assigned to Node 7
    → All tasks firing in that minute are on Node 7
    → PRO: Efficient — node only processes its time range
    → CON: Uneven distribution if some minutes are busier

  Strategy 2: SHARD BY TASK_ID HASH
    → hash(task_id) % num_nodes → assigned node
    → Even distribution guaranteed
    → PRO: Balanced load
    → CON: Every node must check all time ranges
    → From Project 14: consistent hashing for node assignment

  Strategy 3: SHARD BY TIME BUCKET + CONSISTENT HASHING (Our Approach)
    → Time divided into 1-second buckets
    → Each bucket assigned to a node via consistent hashing
    → Node manages all tasks in its assigned time buckets
    → On node failure: buckets reassigned to other nodes
    → Rebalancing moves only ~1/N of buckets

  IMPLEMENTATION:
    → Each scheduler node runs a timer engine
    → Timer engine only processes tasks in its assigned buckets
    → New task: compute bucket from fire_time → route to owning node
    → Node failure: consistent hashing redistributes buckets

PART D: CRON SCHEDULING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Cron expression: "*/5 * * * *" = every 5 minutes
  
  Format: minute hour day_of_month month day_of_week

  Common patterns:
    "0 * * * *"     → Every hour at minute 0
    "*/5 * * * *"   → Every 5 minutes
    "0 0 * * *"     → Daily at midnight
    "0 9 * * 1"     → Every Monday at 9 AM
    "0 0 1 * *"     → First day of each month at midnight

  HOW RECURRING TASKS WORK:
    1. Parse cron expression
    2. Compute NEXT fire time from NOW
    3. Schedule one-time task at that time
    4. When task fires:
       a. Execute the task
       b. Compute NEXT fire time from NOW
       c. Schedule another one-time task
       d. Repeat forever (until cancelled)

  This converts recurring tasks into a CHAIN of one-time tasks
  → Only one instance exists in the timer at any time
  → No need for special recurring timer logic
  → Clean, simple, same execution path

📌 WHAT COULD BE DONE

ARCHITECTURE OPTIONS:

Option 1: Database Polling
  → Store tasks in SQL: SELECT * WHERE fire_time <= NOW() AND status = PENDING
  → Poll every second
  → Simple but: DB load scales with task count
  → Works for < 100K tasks

Option 2: Redis Sorted Sets
  → ZRANGEBYSCORE tasks 0 {now} LIMIT 0 100
  → Fast, O(log N + K) per poll
  → Works for < 10M tasks on one Redis
  → Distribute with CacheGrid (Project 33) for more

Option 3: Time Wheel + Persistent Backing Store (Our Approach)
  → Time wheel in memory for near-term tasks (< 1 hour)
  → Redis sorted set for medium-term (1 hour - 1 day)
  → Database for long-term (> 1 day)
  → Three-tier approach like StreamBox storage tiers (Project 34)

Option 4: Message Queue with Delayed Delivery
  → Publish message with delay: "deliver after 15 seconds"
  → RabbitMQ, SQS, Kafka with delay support
  → PRO: Built-in, battle-tested
  → CON: Limited control over scheduling logic, no cron

WE IMPLEMENT: Option 3 (time wheel) + Option 2 (Redis sorted set backup)
  → Time wheel for precision and throughput
  → Sorted set for persistence and recovery
  → Database for long-term and recurring task definitions

✅ WHAT SHOULD DO

1. MAKE ALL TASK HANDLERS IDEMPOTENT
   → Task may execute more than once (lease expired, retry)
   → Handler must produce same result on re-execution
   → Use task_id as deduplication key in downstream systems
   → From Project 29: idempotent event processing

2. USE EXPONENTIAL BACKOFF FOR RETRIES
   → First retry: 1 minute
   → Second retry: 5 minutes
   → Third retry: 30 minutes
   → Fourth retry: 2 hours
   → Max retries: 5 → then dead letter queue
   → Prevents hammering a failing downstream service

3. SEPARATE SCHEDULING FROM EXECUTION
   → Scheduler decides WHEN tasks fire
   → Executor decides HOW tasks run
   → Different scaling needs: few schedulers, many executors
   → From Project 30 (CQRS): separate write path from read path

4. PERSIST TASKS BEFORE ACKNOWLEDGING
   → Client submits task → write to durable storage FIRST
   → Then add to in-memory timer
   → If scheduler crashes before timer: recovery loads from storage
   → Never acknowledge a task that isn't durably stored

5. USE FENCING TOKENS FOR LEASE SAFETY
   → Lease token = monotonically increasing number
   → Worker must present correct token to complete/cancel task
   → Prevents stale worker from completing a re-assigned task
   → From Project 27: fencing tokens in distributed locks

6. MONITOR TASK LAG
   → Task lag = actual_fire_time - scheduled_fire_time
   → Should be < 1 second for 99% of tasks
   → If lag grows → scheduler overloaded → add nodes
   → Critical metric for system health

7. IMPLEMENT GRACEFUL SHUTDOWN
   → Scheduler node shutting down:
     a. Stop accepting new tasks
     b. Finish executing in-progress tasks (drain)
     c. Transfer remaining timers to other nodes
     d. Only then shut down
   → Prevents task loss during deployments

❌ WHAT SHOULD NOT DO

1. NEVER USE sleep() FOR SCHEDULING
   → Thread.sleep(15000) to wait 15 seconds
   → Blocks a thread, wastes resources
   → Doesn't survive process restart
   → Use timer data structures instead

2. NEVER STORE TIMERS ONLY IN MEMORY
   → Process crashes → all timers lost
   → All scheduled tasks vanish
   → ALWAYS back with persistent storage
   → In-memory is optimization, not source of truth

3. NEVER POLL DATABASE EVERY SECOND FOR ALL TASKS
   → 50M tasks → full table scan every second
   → Database melts
   → Use indexed queries + time-bucketed partitioning
   → Or use in-memory timer (time wheel) for near-term

4. NEVER EXECUTE TASKS IN THE SCHEDULER THREAD
   → Scheduler thread fires task → executes it → takes 5 seconds
   → During those 5 seconds → no other tasks fire → cascade of delays
   → ALWAYS dispatch to worker pool
   → Scheduler only DECIDES, workers EXECUTE

5. NEVER RETRY INFINITELY
   → Task keeps failing → retry forever
   → Consumes resources, never succeeds
   → Max retries (3-5) → then dead letter queue
   → Alert operations team for manual investigation

6. NEVER IGNORE TIMEZONE IN CRON SCHEDULES
   → "Run at midnight" → which midnight? UTC? Local?
   → Daylight saving time: 2 AM happens twice or not at all
   → ALWAYS store and compute in UTC
   → Convert to local time only for display

7. NEVER SCHEDULE MILLIONS OF TASKS AT THE SAME SECOND
   → "All subscriptions renew at midnight" → spike at 00:00:00
   → System overwhelmed
   → Add JITTER: spread tasks within a window
   → "Renew between 00:00 and 00:59" → smooth distribution

8. NEVER ALLOW UNBOUNDED TASK QUEUES
   → If executor is slow → tasks accumulate
   → Memory grows → OOM crash → more tasks lost
   → Set max queue size → reject or back-pressure when full

HIERARCHICAL TIME WHEEL — DETAILED MECHANICS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  STRUCTURE:
    Level 1 (seconds):  60 slots, tick every 1 second
    Level 2 (minutes):  60 slots, tick every 1 minute
    Level 3 (hours):    24 slots, tick every 1 hour
    Level 4 (days):     30 slots, tick every 1 day

    Each slot = linked list of tasks scheduled for that time

  EXAMPLE: Current time = 14:30:15 (2:30:15 PM)

    Task A fires at 14:30:20 (in 5 seconds):
      → Level 1, slot 20
      → Will fire when second hand reaches slot 20

    Task B fires at 14:35:00 (in 4 min 45 sec):
      → Level 2, slot 35
      → When minute hand reaches 35 → cascade to Level 1
      → Tasks in minute slot 35 move to second slots 0-59

    Task C fires at 17:00:00 (in 2.5 hours):
      → Level 3, slot 17
      → When hour hand reaches 17 → cascade to Level 2
      → Then cascade to Level 1 when minute hand arrives

    Task D fires in 5 days:
      → Level 4, slot (current_day + 5) % 30
      → Cascades down through hours → minutes → seconds

  TICK OPERATION (called every second):

    tick():
      1. Advance Level 1 pointer by 1
      2. Fire ALL tasks in current Level 1 slot
      3. If Level 1 wrapped around (hit 0):
         → Advance Level 2 pointer by 1
         → CASCADE: move tasks from current Level 2 slot → Level 1
      4. If Level 2 wrapped around:
         → Advance Level 3 pointer
         → CASCADE: Level 3 → Level 2
      5. If Level 3 wrapped around:
         → Advance Level 4 pointer
         → CASCADE: Level 4 → Level 3

  CASCADE OPERATION:
    When minute slot M is reached:
      → Take ALL tasks from minute slot M
      → For each task:
          → Compute which SECOND it should fire
          → Place it in the correct Level 1 (second) slot
      → Minute slot M is now empty

  WHY O(1) INSERT:
    → Compute: which level? which slot?
    → Append to that slot's linked list
    → No sorting, no rebalancing, no heap operations
    → Constant time regardless of total task count

  WHY O(1) FIRE:
    → Current second slot → iterate linked list → fire all
    → No searching, no scanning
    → Cascade is amortized: happens once per minute/hour/day

    a. Write task to persistent storage (DB/Redis)
       Status: PENDING, fire_at: 14:30:20
    b. Add to time wheel in correct slot
    c. Respond to client: "Task scheduled, id=task_abc123"

  STEP 3: TIME WHEEL TICK (at 14:30:20)
    → Second wheel reaches slot 20
    → Task task_abc123 is in this slot
    → Move to execution queue

  STEP 4: LEASE ACQUISITION
    → Worker W1 takes task from execution queue
    → Atomic operation:
        UPDATE tasks SET
          status = 'LEASED',
          lease_owner = 'W1',
          lease_token = 'tok_random123',
          lease_expiry = NOW() + 30s
        WHERE task_id = 'task_abc123'
          AND status = 'PENDING'
    → If successful → W1 has the lease
    → If failed → another worker got it first → skip

  STEP 5: EXECUTION
    W1 executes the callback:
      POST https://ridenow.com/api/reservations/expire
      Body: {"reservation_id": "res_xyz"}
      Headers: {
        "X-Task-Id": "task_abc123",
        "X-Idempotency-Key": "expire_res_xyz"
      }

    Possible outcomes:
      a. HTTP 200 → Success
      b. HTTP 4xx → Client error → don't retry (permanent failure)
      c. HTTP 5xx → Server error → retry
      d. Timeout → retry
      e. Worker crash → lease expires → retry

  STEP 6a: SUCCESS
    → Mark task as COMPLETED
    → Record completion time and result
    → If recurring (cron) → compute next fire time → schedule next

  STEP 6b: FAILURE + RETRY
    → Increment retry_count
    → Compute next retry time (exponential backoff):
        delay = initial_delay × 2^(retry_count - 1)
        next_fire = now + delay
    → Schedule retry at next_fire
    → Status: PENDING (with updated fire_at)

  STEP 6c: MAX RETRIES EXCEEDED
    → Move to dead letter queue
    → Status: DEAD_LETTERED
    → Alert operations team
    → Log: task_id, all attempts, last error

  STEP 6d: LEASE EXPIRED (worker crash)
    → Lease monitor detects: lease_expiry < NOW() AND status = LEASED
    → Reset: status = PENDING, clear lease fields
    → Re-add to time wheel at fire_at = NOW (immediate retry)
    → Retry count incremented

    initial_delay_sec: float = 60.0
    backoff_multiplier: float = 2.0

            delay *= random.uniform(0.8, 1.2)

class CronParser:
    """
    Parses cron expressions and computes next fire time
    Format: minute hour day_of_month month day_of_week
    Supports: *, */N, N, N-M, N,M
    """

    FIELD_RANGES = {
        'minute': (0, 59),
        'hour': (0, 23),
        'day_of_month': (1, 31),
        'month': (1, 12),
        'day_of_week': (0, 6)  # 0=Sunday
    }

    FIELD_ORDER = ['minute', 'hour', 'day_of_month', 'month', 'day_of_week']

    @classmethod
    def parse(cls, expression: str) -> Dict[str, List[int]]:
        """Parse cron expression into valid values per field"""
        parts = expression.strip().split()
        if len(parts) != 5:
            raise ValueError(
                f"Cron expression must have 5 fields, got {len(parts)}"
            )

        result = {}
        for i, field_name in enumerate(cls.FIELD_ORDER):
            min_val, max_val = cls.FIELD_RANGES[field_name]
            result[field_name] = cls._parse_field(
                parts[i], min_val, max_val
            )

        return result

    @classmethod
    def _parse_field(cls, field_str: str,
                     min_val: int, max_val: int) -> List[int]:
        """Parse a single cron field"""
        values = set()

        for part in field_str.split(','):
            if '/' in part:
                # Step: */5 or 1-30/5
                range_part, step = part.split('/')
                step = int(step)
                if range_part == '*':
                    start, end = min_val, max_val
                elif '-' in range_part:
                    start, end = map(int, range_part.split('-'))
                else:
                    start = int(range_part)
                    end = max_val
                for v in range(start, end + 1, step):
                    values.add(v)

            elif part == '*':
                values.update(range(min_val, max_val + 1))

            elif '-' in part:
                start, end = map(int, part.split('-'))
                values.update(range(start, end + 1))

            else:
                values.add(int(part))

        return sorted(v for v in values if min_val <= v <= max_val)

    @classmethod
    def next_fire_time(cls, expression: str,
                       after: float = None) -> float:
        """Compute the next fire time after the given timestamp"""
        if after is None:
            after = time.time()

        parsed = cls.parse(expression)
        t = time.gmtime(after + 60)  # Start from next minute

        # Search forward up to 1 year
        for _ in range(525960):  # ~1 year in minutes
            minute = t.tm_min
            hour = t.tm_hour
            day = t.tm_mday
            month = t.tm_mon
            weekday = (t.tm_wday + 1) % 7  # Convert: Mon=0 → Sun=0

            if (minute in parsed['minute'] and
                    hour in parsed['hour'] and
                    day in parsed['day_of_month'] and
                    month in parsed['month'] and
                    weekday in parsed['day_of_week']):
                import calendar
                result = calendar.timegm(t)
                if result > after:
                    return float(result)

            # Advance by 1 minute
            next_ts = calendar.timegm(t) + 60
            t = time.gmtime(next_ts)

        raise ValueError(f"Could not find next fire time for: {expression}")

    @classmethod
    def describe(cls, expression: str) -> str:
        """Human-readable description of cron expression"""
        parts = expression.strip().split()
        descriptions = {
            "* * * * *": "Every minute",
            "*/5 * * * *": "Every 5 minutes",
            "*/15 * * * *": "Every 15 minutes",
            "0 * * * *": "Every hour",
            "0 */2 * * *": "Every 2 hours",
            "0 0 * * *": "Daily at midnight",
            "0 9 * * *": "Daily at 9:00 AM",
            "0 9 * * 1": "Every Monday at 9:00 AM",
            "0 0 1 * *": "First day of month at midnight",
        }
        return descriptions.get(expression, f"Custom: {expression}")

class TimeWheelLevel:
    """A single level of the hierarchical time wheel"""

    def __init__(self, slots: int, resolution_sec: float, name: str):
        self.slots = slots
        self.resolution_sec = resolution_sec
        self.name = name
        self.buckets: List[List[str]] = [[] for _ in range(slots)]
        self.current_slot = 0
        self.total_tasks = 0

    def add_task(self, task_id: str, slot: int):
        """Add task to specific slot"""
        slot = slot % self.slots
        self.buckets[slot].append(task_id)
        self.total_tasks += 1

    def remove_task(self, task_id: str) -> bool:
        """Remove task from any slot"""
        for bucket in self.buckets:
            if task_id in bucket:
                bucket.remove(task_id)
                self.total_tasks -= 1
                return True
        return False

    def advance(self) -> Tuple[List[str], bool]:
        """
        Advance to next slot
        Returns: (tasks in current slot, whether we wrapped around)
        """
        tasks = self.buckets[self.current_slot]
        self.buckets[self.current_slot] = []
        self.total_tasks -= len(tasks)

        self.current_slot = (self.current_slot + 1) % self.slots
        wrapped = self.current_slot == 0

        return tasks, wrapped

    def get_occupancy(self) -> Dict[int, int]:
        """Return non-empty slot counts"""
        return {
            i: len(b) for i, b in enumerate(self.buckets) if b
        }


class HierarchicalTimeWheel:
    """
    4-level hierarchical time wheel
    Level 0: seconds (60 slots, 1s resolution)
    Level 1: minutes (60 slots, 60s resolution)
    Level 2: hours (24 slots, 3600s resolution)
    Level 3: days (30 slots, 86400s resolution)
    """

    def __init__(self):
        self.levels = [
            TimeWheelLevel(60, 1, "seconds"),
            TimeWheelLevel(60, 60, "minutes"),
            TimeWheelLevel(24, 3600, "hours"),
            TimeWheelLevel(30, 86400, "days"),
        ]
        self.base_time = time.time()
        self.current_tick = 0
        self.task_locations: Dict[str, Tuple[int, int]] = {}
        self.lock = threading.Lock()
        # Metrics
        self.total_inserts = 0
        self.total_fires = 0
        self.total_cascades = 0

    def add_task(self, task_id: str, fire_at: float) -> bool:
        """Add a task to the appropriate wheel level"""
        with self.lock:
            delay_sec = fire_at - time.time()
            if delay_sec < 0:
                delay_sec = 0  # Fire immediately

            level, slot = self._compute_slot(delay_sec)

            if level < 0 or level >= len(self.levels):
                return False  # Too far in the future

            self.levels[level].add_task(task_id, slot)
            self.task_locations[task_id] = (level, slot)
            self.total_inserts += 1
            return True

    def cancel_task(self, task_id: str) -> bool:
        """Cancel a scheduled task"""
        with self.lock:
            location = self.task_locations.pop(task_id, None)
            if location:
                level, slot = location
                return self.levels[level].remove_task(task_id)
            return False

    def tick(self) -> List[str]:
        """
        Advance time by 1 second
        Returns list of task_ids that should fire NOW
        """
        with self.lock:
            self.current_tick += 1
            fired_tasks = []

            # Advance level 0 (seconds)
            tasks, wrapped = self.levels[0].advance()
            fired_tasks.extend(tasks)
            self.total_fires += len(tasks)

            # Clean up locations for fired tasks
            for tid in tasks:
                self.task_locations.pop(tid, None)

            # Cascade through higher levels if wrapped
            level = 1
            while wrapped and level < len(self.levels):
                cascade_tasks, wrapped = self.levels[level].advance()
                self.total_cascades += len(cascade_tasks)

                # Re-distribute cascaded tasks into lower level
                for tid in cascade_tasks:
                    old_loc = self.task_locations.pop(tid, None)
                    # Compute new slot in the level below
                    # Place evenly across the lower level's slots
                    target_level = level - 1
                    target_slot = random.randint(
                        0, self.levels[target_level].slots - 1
                    )
                    self.levels[target_level].add_task(tid, target_slot)
                    self.task_locations[tid] = (target_level, target_slot)

                level += 1

            return fired_tasks

    def _compute_slot(self, delay_sec: float) -> Tuple[int, int]:
        """Determine which level and slot a task should go into"""
        if delay_sec < 60:
            # Level 0: seconds
            slot = (self.levels[0].current_slot + int(delay_sec)) % 60
            return 0, slot
        elif delay_sec < 3600:
            # Level 1: minutes
            minutes = int(delay_sec / 60)
            slot = (self.levels[1].current_slot + minutes) % 60
            return 1, slot
        elif delay_sec < 86400:
            # Level 2: hours
            hours = int(delay_sec / 3600)
            slot = (self.levels[2].current_slot + hours) % 24
            return 2, slot
        elif delay_sec < 86400 * 30:
            # Level 3: days
            days = int(delay_sec / 86400)
            slot = (self.levels[3].current_slot + days) % 30
            return 3, slot
        else:
            return -1, -1  # Too far in future

    def get_stats(self) -> dict:
        return {
            "current_tick": self.current_tick,
            "total_inserts": self.total_inserts,
            "total_fires": self.total_fires,
            "total_cascades": self.total_cascades,
            "active_tasks": len(self.task_locations),
            "levels": {
                level.name: {
                    "slots": level.slots,
                    "current_slot": level.current_slot,
                    "total_tasks": level.total_tasks,
                    "non_empty_slots": len(level.get_occupancy())
                }
                for level in self.levels
            }
        }

        """Save or update a task"""
        with self.lock:
            # Remove from old status set if updating
            if task.task_id in self.tasks:
                old_status = self.tasks[task.task_id].status
                self.by_status[old_status].discard(task.task_id)

            self.tasks[task.task_id] = task
            self.by_status[task.status].add(task.task_id)
            self.by_group[task.task_group].add(task.task_id)

    def get_task(self, task_id: str) -> Optional[Task]:
        return self.tasks.get(task_id)

    def get_tasks_by_status(self, status: TaskStatus) -> List[Task]:
        with self.lock:
            task_ids = self.by_status.get(status, set())
            return [self.tasks[tid] for tid in task_ids if tid in self.tasks]

    def get_expired_leases(self) -> List[Task]:
        """Find tasks with expired leases — for recovery"""
        now = time.time()
        with self.lock:
            leased_ids = self.by_status.get(TaskStatus.LEASED, set())
            expired = []
            for tid in leased_ids:
                task = self.tasks.get(tid)
                if task and task.lease_expiry and task.lease_expiry < now:
                    expired.append(task)
            return expired

    def move_to_dead_letter(self, task_id: str):
        """Move a failed task to dead letter queue"""
        with self.lock:
            task = self.tasks.get(task_id)
            if task:
                old_status = task.status
                self.by_status[old_status].discard(task_id)
                task.status = TaskStatus.DEAD_LETTERED
                self.by_status[TaskStatus.DEAD_LETTERED].add(task_id)
                self.dead_letter_queue.append(task_id)

    def get_stats(self) -> dict:
        with self.lock:
            status_counts = {
                status.value: len(ids)
                for status, ids in self.by_status.items()
                if ids
            }
            return {
                "total_tasks": len(self.tasks),
                "by_status": status_counts,
                "groups": len(self.by_group),
                "dead_letter_queue_size": len(self.dead_letter_queue)
            }

class LeaseManager:
    """
    Manages task leases for exactly-once execution
    Implements compare-and-swap for lease acquisition
    From Project 27: distributed locking concepts
    """

    DEFAULT_LEASE_DURATION_SEC = 30.0

    def __init__(self, task_store: TaskStore):
        self.task_store = task_store
        self.lock = threading.Lock()
        self.leases_granted = 0
        self.leases_denied = 0
        self.leases_expired = 0
        self.leases_completed = 0

    def acquire_lease(self, task_id: str, worker_id: str,
                      duration_sec: float = None) -> Optional[str]:
        """
        Try to acquire a lease on a task
        Returns lease_token if successful, None if failed
        """
        if duration_sec is None:
            duration_sec = self.DEFAULT_LEASE_DURATION_SEC

        with self.lock:
            task = self.task_store.get_task(task_id)
            if not task:
                self.leases_denied += 1
                return None

            if task.status not in (TaskStatus.PENDING, TaskStatus.SCHEDULED):
                self.leases_denied += 1
                return None

            # Generate unique lease token (fencing token)
            lease_token = f"lease_{uuid.uuid4().hex[:12]}"

            # Atomic status change
            task.status = TaskStatus.LEASED
            task.lease_owner = worker_id
            task.lease_token = lease_token
            task.lease_expiry = time.time() + duration_sec
            task.history.append({
                "event": "lease_acquired",
                "worker": worker_id,
                "token": lease_token,
                "expiry": task.lease_expiry,
                "timestamp": time.time()
            })

            self.task_store.save_task(task)
            self.leases_granted += 1
            return lease_token

    def complete_lease(self, task_id: str, lease_token: str,
                       success: bool, result: Any = None,
                       error: str = None) -> bool:
        """
        Complete a lease — only the lease holder can do this
        Fencing token prevents stale workers from completing
        """
        with self.lock:
            task = self.task_store.get_task(task_id)
            if not task:
                return False

            # Verify lease token (fencing)
            if task.lease_token != lease_token:
                return False

            if task.status != TaskStatus.LEASED:
                return False

            if success:
                task.status = TaskStatus.COMPLETED
                task.completed_at = time.time()
                task.history.append({
                    "event": "completed",
                    "result": str(result)[:100] if result else None,
                    "timestamp": time.time()
                })
                self.leases_completed += 1
            else:
                task.status = TaskStatus.FAILED
                task.last_error = error
                task.history.append({
                    "event": "failed",
                    "error": error,
                    "retry_count": task.retry_count,
                    "timestamp": time.time()
                })

            # Clear lease info
            task.lease_owner = None
            task.lease_token = None
            task.lease_expiry = None

            self.task_store.save_task(task)
            return True

    def recover_expired_leases(self) -> List[str]:
        """
        Find and recover tasks with expired leases
        Called periodically by scheduler
        """
        recovered = []
        expired_tasks = self.task_store.get_expired_leases()

        with self.lock:
            for task in expired_tasks:
                task.status = TaskStatus.PENDING
                task.lease_owner = None
                task.lease_token = None
                task.lease_expiry = None
                task.retry_count += 1
                task.history.append({
                    "event": "lease_expired_recovery",
                    "retry_count": task.retry_count,
                    "timestamp": time.time()
                })
                self.task_store.save_task(task)
                recovered.append(task.task_id)
                self.leases_expired += 1

        return recovered

    def get_stats(self) -> dict:
        return {
            "leases_granted": self.leases_granted,
            "leases_denied": self.leases_denied,
            "leases_expired": self.leases_expired,
            "leases_completed": self.leases_completed
        }

class TaskExecutor:
    """
    Executes task callbacks
    Handles function calls, simulated HTTP callbacks
    Reports results back through lease manager
    """

    def __init__(self, worker_id: str, lease_manager: LeaseManager,
                 task_store: TaskStore):
        self.worker_id = worker_id
        self.lease_manager = lease_manager
        self.task_store = task_store
        # Idempotency tracking
        self.processed_keys: Set[str] = set()
        # Metrics
        self.tasks_executed = 0
        self.tasks_succeeded = 0
        self.tasks_failed = 0
        self.total_execution_ms = 0.0

    def execute_task(self, task_id: str) -> TaskResult:
        """
        Full execution flow:
        1. Acquire lease
        2. Check idempotency
        3. Execute callback
        4. Report result
        """
        start_time = time.time()
        self.tasks_executed += 1

        # Step 1: Acquire lease
        lease_token = self.lease_manager.acquire_lease(
            task_id, self.worker_id
        )
        if not lease_token:
            return TaskResult(
                task_id=task_id, success=False,
                error="Could not acquire lease",
                worker_id=self.worker_id
            )

        task = self.task_store.get_task(task_id)
        if not task:
            return TaskResult(
                task_id=task_id, success=False,
                error="Task not found",
                worker_id=self.worker_id
            )

        # Step 2: Idempotency check
        idem_key = task.idempotency_key or task.task_id
        if idem_key in self.processed_keys:
            # Already processed — complete as success (idempotent)
            self.lease_manager.complete_lease(
                task_id, lease_token, success=True,
                result="Idempotent skip"
            )
            return TaskResult(
                task_id=task_id, success=True,
                result="Idempotent skip — already processed",
                worker_id=self.worker_id
            )

        # Step 3: Execute callback
        task.status = TaskStatus.EXECUTING
        task.fired_at = time.time()

        try:
            result = self._execute_callback(task.callback)
            execution_ms = (time.time() - start_time) * 1000
            task.execution_duration_ms = execution_ms
            self.total_execution_ms += execution_ms

            # Step 4a: Report success
            self.lease_manager.complete_lease(
                task_id, lease_token, success=True, result=result
            )
            self.processed_keys.add(idem_key)
            self.tasks_succeeded += 1

            return TaskResult(
                task_id=task_id, success=True,
                result=result,
                execution_time_ms=execution_ms,
                worker_id=self.worker_id
            )

        except Exception as e:
            execution_ms = (time.time() - start_time) * 1000
            error_msg = str(e)

            # Step 4b: Report failure
            self.lease_manager.complete_lease(
                task_id, lease_token, success=False, error=error_msg
            )
            self.tasks_failed += 1

            return TaskResult(
                task_id=task_id, success=False,
                error=error_msg,
                execution_time_ms=execution_ms,
                worker_id=self.worker_id
            )

    def _execute_callback(self, callback: TaskCallback) -> Any:
        """Execute the task's callback"""
        if callback.callback_type == CallbackType.FUNCTION:
            if callback.function:
                args = callback.args or ()
                kwargs = callback.kwargs or {}
                return callback.function(*args, **kwargs)
            raise ValueError("No function provided for FUNCTION callback")

        elif callback.callback_type == CallbackType.HTTP:
            # Simulate HTTP callback
            return self._simulate_http_callback(callback)

        elif callback.callback_type == CallbackType.QUEUE:
            # Simulate queue publish
            return {"published": True, "body": callback.body}

        raise ValueError(f"Unknown callback type: {callback.callback_type}")

    def _simulate_http_callback(self, callback: TaskCallback) -> dict:
        """Simulate an HTTP callback execution"""
        # Simulate network latency
        latency_ms = random.uniform(5, 50)
        time.sleep(latency_ms / 1000)

        # Simulate occasional failures (10% failure rate)
        if random.random() < 0.10:
            raise Exception(
                f"HTTP {callback.method} {callback.url} "
                f"returned 500 Internal Server Error"
            )

        return {
            "status": 200,
            "url": callback.url,
            "method": callback.method,
            "latency_ms": round(latency_ms, 1)
        }

    def get_stats(self) -> dict:
        avg_ms = (
            self.total_execution_ms / self.tasks_executed
            if self.tasks_executed > 0 else 0
        )
        return {
            "worker_id": self.worker_id,
            "tasks_executed": self.tasks_executed,
            "tasks_succeeded": self.tasks_succeeded,
            "tasks_failed": self.tasks_failed,
            "success_rate_pct": round(
                self.tasks_succeeded / max(self.tasks_executed, 1) * 100, 1
            ),
            "avg_execution_ms": round(avg_ms, 2),
            "idempotency_cache_size": len(self.processed_keys)
        }

class WorkerPool:
    """
    Pool of task executors with concurrent execution
    Dispatches tasks from execution queue to workers
    """

    def __init__(self, num_workers: int, lease_manager: LeaseManager,
                 task_store: TaskStore):
        self.num_workers = num_workers
        self.workers: Dict[str, TaskExecutor] = {}
        self.execution_queue: deque = deque()
        self.lock = threading.Lock()

        for i in range(num_workers):
            worker_id = f"worker_{i:03d}"
            self.workers[worker_id] = TaskExecutor(
                worker_id, lease_manager, task_store
            )

        # Metrics
        self.total_dispatched = 0
        self.queue_max_depth = 0

    def enqueue(self, task_id: str):
        """Add task to execution queue"""
        with self.lock:
            self.execution_queue.append(task_id)
            self.queue_max_depth = max(
                self.queue_max_depth, len(self.execution_queue)
            )

    def enqueue_batch(self, task_ids: List[str]):
        """Add multiple tasks to execution queue"""
        with self.lock:
            self.execution_queue.extend(task_ids)
            self.queue_max_depth = max(
                self.queue_max_depth, len(self.execution_queue)
            )

    def process_queue(self, max_tasks: int = None) -> List[TaskResult]:
        """Process tasks from the execution queue"""
        results = []
        limit = max_tasks or len(self.execution_queue)
        processed = 0

        while processed < limit:
            with self.lock:
                if not self.execution_queue:
                    break
                task_id = self.execution_queue.popleft()

            # Round-robin worker selection
            worker_id = f"worker_{processed % self.num_workers:03d}"
            worker = self.workers[worker_id]

            result = worker.execute_task(task_id)
            results.append(result)
            self.total_dispatched += 1
            processed += 1

        return results

    def get_queue_depth(self) -> int:
        return len(self.execution_queue)

    def get_stats(self) -> dict:
        worker_stats = {}
        total_succeeded = 0
        total_failed = 0

        for wid, worker in self.workers.items():
            stats = worker.get_stats()
            worker_stats[wid] = stats
            total_succeeded += stats["tasks_succeeded"]
            total_failed += stats["tasks_failed"]

        return {
            "num_workers": self.num_workers,
            "queue_depth": len(self.execution_queue),
            "queue_max_depth": self.queue_max_depth,
            "total_dispatched": self.total_dispatched,
            "total_succeeded": total_succeeded,
            "total_failed": total_failed,
            "workers": worker_stats
        }

class RetryManager:
    """
    Handles retry logic for failed tasks
    Computes backoff delays and manages retry limits
    """

    def __init__(self, task_store: TaskStore,
                 time_wheel: HierarchicalTimeWheel):
        self.task_store = task_store
        self.time_wheel = time_wheel
        self.retries_scheduled = 0
        self.tasks_dead_lettered = 0

    def handle_failure(self, task_id: str) -> dict:
        """Process a failed task — retry or dead letter"""
        task = self.task_store.get_task(task_id)
        if not task:
            return {"error": "Task not found"}

        policy = task.retry_policy

        if task.retry_count >= policy.max_retries:
            # Max retries exceeded → dead letter queue
            self.task_store.move_to_dead_letter(task_id)
            self.tasks_dead_lettered += 1
            task.history.append({
                "event": "dead_lettered",
                "retry_count": task.retry_count,
                "max_retries": policy.max_retries,
                "last_error": task.last_error,
                "timestamp": time.time()
            })
            return {
                "action": "dead_lettered",
                "task_id": task_id,
                "retry_count": task.retry_count,
                "reason": "Max retries exceeded"
            }

        # Compute retry delay
        delay = policy.get_delay(task.retry_count)
        next_fire = time.time() + delay

        # Schedule retry
        task.status = TaskStatus.RETRYING
        task.fire_at = next_fire
        task.retry_count += 1
        task.history.append({
            "event": "retry_scheduled",
            "retry_count": task.retry_count,
            "delay_sec": round(delay, 1),
            "next_fire": next_fire,
            "timestamp": time.time()
        })

        self.task_store.save_task(task)

        # Add back to time wheel
        task.status = TaskStatus.PENDING
        self.task_store.save_task(task)
        self.time_wheel.add_task(task_id, next_fire)

        self.retries_scheduled += 1

        return {
            "action": "retry_scheduled",
            "task_id": task_id,
            "retry_count": task.retry_count,
            "delay_sec": round(delay, 1),
            "next_fire_in_sec": round(delay, 1)
        }

    def get_stats(self) -> dict:
        return {
            "retries_scheduled": self.retries_scheduled,
            "tasks_dead_lettered": self.tasks_dead_lettered
        }

class TaskScheduler:
    """
    Main scheduler — ties time wheel, task store, lease manager,
    worker pool, retry manager, and cron together
    """

    def __init__(self, num_workers: int = 4,
                 scheduler_id: str = "scheduler_001"):
        self.scheduler_id = scheduler_id

        # Core components
        self.task_store = TaskStore()
        self.time_wheel = HierarchicalTimeWheel()
        self.lease_manager = LeaseManager(self.task_store)
        self.worker_pool = WorkerPool(
            num_workers, self.lease_manager, self.task_store
        )
        self.retry_manager = RetryManager(
            self.task_store, self.time_wheel
        )

        # Cron task registry
        self.cron_tasks: Dict[str, dict] = {}

        # Metrics
        self.tasks_submitted = 0
        self.tasks_fired = 0
        self.ticks_processed = 0
        self.task_lag_samples: List[float] = []

    # ============================================================
    # TASK SUBMISSION
    # ============================================================

    def submit_delayed_task(self, delay_sec: float,
                            callback: TaskCallback,
                            task_group: str = "default",
                            retry_policy: RetryPolicy = None,
                            idempotency_key: str = None,
                            tags: Dict[str, str] = None) -> str:
        """Submit a one-time task to fire after a delay"""
        task_id = f"task_{uuid.uuid4().hex[:10]}"
        fire_at = time.time() + delay_sec

        task = Task(
            task_id=task_id,
            task_type=TaskType.DELAYED,
            callback=callback,
            fire_at=fire_at,
            retry_policy=retry_policy or RetryPolicy(),
            created_at=time.time(),
            task_group=task_group,
            idempotency_key=idempotency_key,
            tags=tags or {}
        )
        task.history.append({
            "event": "submitted",
            "delay_sec": delay_sec,
            "fire_at": fire_at,
            "timestamp": time.time()
        })

        # Persist first (durability)
        self.task_store.save_task(task)

        # Then add to time wheel (optimization)
        self.time_wheel.add_task(task_id, fire_at)
        task.status = TaskStatus.SCHEDULED
        self.task_store.save_task(task)

        self.tasks_submitted += 1
        return task_id

    def submit_scheduled_task(self, fire_at: float,
                               callback: TaskCallback,
                               task_group: str = "default",
                               retry_policy: RetryPolicy = None,
                               idempotency_key: str = None,
                               tags: Dict[str, str] = None) -> str:
        """Submit a one-time task to fire at a specific time"""
        delay = max(0, fire_at - time.time())
        task_id = f"task_{uuid.uuid4().hex[:10]}"

        task = Task(
            task_id=task_id,
            task_type=TaskType.ONE_TIME,
            callback=callback,
            fire_at=fire_at,
            retry_policy=retry_policy or RetryPolicy(),
            created_at=time.time(),
            task_group=task_group,
            idempotency_key=idempotency_key,
            tags=tags or {}
        )
        task.history.append({
            "event": "submitted",
            "fire_at": fire_at,
            "delay_sec": round(delay, 1),
            "timestamp": time.time()
        })

        self.task_store.save_task(task)
        self.time_wheel.add_task(task_id, fire_at)
        task.status = TaskStatus.SCHEDULED
        self.task_store.save_task(task)

        self.tasks_submitted += 1
        return task_id

    def submit_recurring_task(self, cron_expression: str,
                               callback: TaskCallback,
                               task_group: str = "cron",
                               retry_policy: RetryPolicy = None,
                               tags: Dict[str, str] = None) -> str:
        """Submit a recurring task with cron schedule"""
        cron_id = f"cron_{uuid.uuid4().hex[:8]}"

        # Compute first fire time
        next_fire = CronParser.next_fire_time(cron_expression)

        # Store cron definition
        self.cron_tasks[cron_id] = {
            "cron_id": cron_id,
            "expression": cron_expression,
            "description": CronParser.describe(cron_expression),
            "callback": callback,
            "task_group": task_group,
            "retry_policy": retry_policy or RetryPolicy(),
            "tags": tags or {},
            "active": True,
            "executions": 0,
            "created_at": time.time()
        }

        # Schedule first occurrence as a one-time task
        task_id = self._schedule_cron_occurrence(cron_id, next_fire)

        return cron_id

    def _schedule_cron_occurrence(self, cron_id: str,
                                   fire_at: float) -> str:
        """Schedule one occurrence of a recurring task"""
        cron_def = self.cron_tasks.get(cron_id)
        if not cron_def or not cron_def["active"]:
            return ""

        task_id = f"cron_exec_{cron_id}_{uuid.uuid4().hex[:6]}"

        task = Task(
            task_id=task_id,
            task_type=TaskType.RECURRING,
            callback=cron_def["callback"],
            fire_at=fire_at,
            cron_expression=cron_def["expression"],
            retry_policy=cron_def["retry_policy"],
            created_at=time.time(),
            task_group=cron_def["task_group"],
            tags={**cron_def["tags"], "cron_id": cron_id}
        )
        task.history.append({
            "event": "cron_occurrence_scheduled",
            "cron_id": cron_id,
            "fire_at": fire_at,
            "timestamp": time.time()
        })

        self.task_store.save_task(task)
        self.time_wheel.add_task(task_id, fire_at)
        task.status = TaskStatus.SCHEDULED
        self.task_store.save_task(task)

        self.tasks_submitted += 1
        return task_id

    def cancel_cron_task(self, cron_id: str) -> dict:
        """Cancel a recurring task"""
        if cron_id not in self.cron_tasks:
            return {"error": "Cron task not found"}

        self.cron_tasks[cron_id]["active"] = False
        return {"status": "cancelled", "cron_id": cron_id}

    def cancel_task(self, task_id: str) -> dict:
        """Cancel a specific task"""
        task = self.task_store.get_task(task_id)
        if not task:
            return {"error": "Task not found"}

        if task.status in (TaskStatus.COMPLETED, TaskStatus.DEAD_LETTERED):
            return {"error": "Cannot cancel completed/dead-lettered task"}

        self.time_wheel.cancel_task(task_id)
        task.status = TaskStatus.CANCELLED
        task.history.append({
            "event": "cancelled",
            "timestamp": time.time()
        })
        self.task_store.save_task(task)

        return {"status": "cancelled", "task_id": task_id}

    # ============================================================
    # SCHEDULER TICK — CALLED EVERY SECOND
    # ============================================================

    def tick(self) -> dict:
        """
        Main scheduler loop iteration — call every second
        1. Advance time wheel → get due tasks
        2. Dispatch to worker pool
        3. Process execution results
        4. Handle retries
        5. Recover expired leases
        6. Schedule next cron occurrences
        """
        self.ticks_processed += 1
        tick_start = time.time()

        # Step 1: Advance time wheel
        fired_task_ids = self.time_wheel.tick()
        self.tasks_fired += len(fired_task_ids)

        # Record task lag
        for tid in fired_task_ids:
            task = self.task_store.get_task(tid)
            if task:
                lag = time.time() - task.fire_at
                self.task_lag_samples.append(lag)
                # Keep only last 1000 samples
                if len(self.task_lag_samples) > 1000:
                    self.task_lag_samples = self.task_lag_samples[-1000:]

        # Step 2: Dispatch to worker pool
        if fired_task_ids:
            self.worker_pool.enqueue_batch(fired_task_ids)

        # Step 3: Process execution queue
        results = self.worker_pool.process_queue(
            max_tasks=len(fired_task_ids) + 10
        )

        # Step 4: Handle failures and retries
        retries = []
        cron_reschedules = []
        for result in results:
            if not result.success and result.error != "Could not acquire lease":
                retry_result = self.retry_manager.handle_failure(
                    result.task_id
                )
                retries.append(retry_result)

            elif result.success:
                # Check if this was a cron task → schedule next occurrence
                task = self.task_store.get_task(result.task_id)
                if task and task.task_type == TaskType.RECURRING:
                    cron_id = task.tags.get("cron_id")
                    if cron_id and cron_id in self.cron_tasks:
                        cron_def = self.cron_tasks[cron_id]
                        if cron_def["active"]:
                            next_fire = CronParser.next_fire_time(
                                task.cron_expression
                            )
                            self._schedule_cron_occurrence(cron_id, next_fire)
                            cron_def["executions"] += 1
                            cron_reschedules.append(cron_id)

        # Step 5: Recover expired leases (every 10 ticks)
        recovered = []
        if self.ticks_processed % 10 == 0:
            recovered = self.lease_manager.recover_expired_leases()
            for tid in recovered:
                self.time_wheel.add_task(tid, time.time())

        tick_duration_ms = (time.time() - tick_start) * 1000

        return {
            "tick": self.ticks_processed,
            "fired": len(fired_task_ids),
            "executed": len(results),
            "succeeded": sum(1 for r in results if r.success),
            "failed": sum(1 for r in results if not r.success),
            "retries": len(retries),
            "cron_rescheduled": len(cron_reschedules),
            "leases_recovered": len(recovered),
            "tick_duration_ms": round(tick_duration_ms, 2)
        }

class SchedulerDisplayFormatter:

    @staticmethod
    def format_tick_result(result: dict) -> str:
        parts = []
        parts.append(f"  Tick {result['tick']:>4}: ")
        parts.append(f"Fired={result['fired']:>3} ")
        parts.append(f"OK={result['succeeded']:>3} ")
        parts.append(f"Fail={result['failed']:>2} ")
        parts.append(f"Retry={result['retries']:>2} ")
        parts.append(f"Cron={result['cron_rescheduled']:>2} ")
        parts.append(f"Recovered={result['leases_recovered']:>2} ")
        parts.append(f"({result['tick_duration_ms']:.1f}ms)")
        return "".join(parts)

    @staticmethod
    def format_task_inspection(info: dict) -> str:
        lines = []
        lines.append(f"\n  🔍 TASK: {info['task_id']}")
        lines.append(f"     Type: {info['type']} | Status: {info['status']}")
        lines.append(f"     Group: {info['group']}")
        lines.append(f"     Retries: {info['retry_count']}/{info['max_retries']}")
        if info['last_error']:
            lines.append(f"     Last Error: {info['last_error'][:60]}")
        if info['execution_ms']:
            lines.append(f"     Execution: {info['execution_ms']:.1f}ms")
        if info['tags']:
            lines.append(f"     Tags: {info['tags']}")
        lines.append(f"     History ({info['history_count']} events):")
        for event in info['history']:
            lines.append(f"       → {event['event']} "
                        f"(t={event.get('timestamp', 0):.0f})")
        return "\n".join(lines)

    @staticmethod
    def format_metrics(metrics: dict) -> str:
        lines = []
        lines.append("")
        lines.append("=" * 70)
        lines.append("  📊 SCHEDULER METRICS")
        lines.append("=" * 70)

        for section, data in metrics.items():
            if isinstance(data, dict):
                lines.append(f"\n  ┌─ {section.upper()}")
                for k, v in data.items():
                    if isinstance(v, dict):
                        lines.append(f"  │  {k}:")
                        for k2, v2 in v.items():
                            if isinstance(v2, dict):
                                lines.append(f"  │    {k2}: ...")
                            else:
                                lines.append(f"  │    {k2}: {v2}")
                    else:
                        lines.append(f"  │  {k}: {v}")
                lines.append(f"  └{'─' * 55}")
            else:
                lines.append(f"  {section}: {data}")

        lines.append("")
        return "\n".join(lines)

   → Lease has expiry → crash recovery automatic
   → Only lease holder can complete the task
   → Stale workers rejected by fencing token

✅ Exponential backoff prevents thundering retry storms
   → Each retry waits longer: 1m → 5m → 30m → 2h
   → Jitter prevents synchronized retries
   → Max retries → dead letter queue → human intervention

✅ Cron tasks are chains of one-time tasks
   → Parse expression → compute next fire time
   → Schedule one-time → execute → compute next → schedule next
   → Same execution path as delayed tasks (simpler system)

✅ Persist before acknowledge
   → Write to durable storage FIRST
   → Then add to in-memory timer
   → Crash between the two → recovery loads from storage

✅ Separate scheduling from execution
   → Scheduler: WHEN (timer management, few instances)
   → Executor: HOW (callback execution, many workers)
   → Different scaling needs, independent failure domains

THE GAP WE JUST EXPOSED:

TaskMaster handles the TIMING and EXECUTION of tasks
But what happens when the task itself is a FINANCIAL TRANSACTION?

Consider this scenario:
  → TaskMaster fires: "Charge user $29.99 for subscription renewal"
  → Worker calls payment gateway → gateway processes charge
  → Worker crashes AFTER charge succeeded but BEFORE marking complete
  → Lease expires → TaskMaster retries
  → Worker calls payment gateway AGAIN → user charged TWICE → $59.98

OUR IDEMPOTENCY GUARD HELPS:
  → Check idempotency_key before executing
  → But what if the PAYMENT GATEWAY doesn't support idempotency?
  → What if the charge succeeded at the gateway but we don't know?
  → What if the charge is part of a MULTI-STEP transaction?
    (reserve inventory → charge card → confirm order → ship)
  → If step 2 fails, step 1 must be REVERSED (compensation)

THIS REQUIRES A NEW PATTERN: THE SAGA

  A Saga is a sequence of local transactions:
    Step 1: Reserve inventory       ← If this fails → stop
    Step 2: Charge payment          ← If this fails → unreserve inventory
    Step 3: Confirm order           ← If this fails → refund payment + unreserve
    Step 4: Notify shipping         ← If this fails → cancel order + refund + unreserve

  Each step has a COMPENSATING ACTION (undo):
    reserve → unreserve
    charge → refund
    confirm → cancel
    notify → cancel shipment

  If ANY step fails → execute compensations in REVERSE ORDER
  This maintains consistency WITHOUT distributed transactions
