PROJECT 31: CHATSPHERE — DESIGN A REAL-TIME CHAT SYSTEM

📋 PHASE 4 (Hard) — Advanced Patterns

🔗 Builds on: Project 8 (Stateless/Stateful) + Project 11 (Message Queues) + Project 24 (Notifications) + Project 29 (Event-Driven)

🎯 HYPOTHETICAL PROBLEM

You are building a messaging platform (like WhatsApp/Slack)
→ 500 million registered users
→ 50 million online simultaneously at peak
→ 100 billion messages sent per day (~1.2 million messages/second)
→ Average user is in 20 group chats
→ Largest group chat: 1000 members

REQUIREMENTS:

  1. ONE-TO-ONE CHAT:
     → Alice sends "Hello" to Bob
     → Bob sees it within 200ms (if online)
     → Alice sees "✓ Sent" → "✓✓ Delivered" → "✓✓ Read"
     → Messages arrive IN ORDER (no scrambling)
     → If Bob is offline → message stored → delivered when he reconnects

  2. GROUP CHAT:
     → Alice sends "Meeting at 3pm" to a group of 50 people
     → ALL 50 members see it within 500ms
     → Each member sees their own read receipt
     → Members can join/leave groups

  3. PRESENCE:
     → Green dot = online, Grey dot = offline
     → "Last seen 5 minutes ago"
     → "Alice is typing..." indicator
     → Must update within 5 seconds of status change

  4. MESSAGE HISTORY:
     → Scroll up to load older messages
     → Search within chat
     → Media messages (images, files — reference only)

  5. RELIABILITY:
     → Messages NEVER lost (even if servers crash)
     → Messages NEVER delivered out of order
     → Messages delivered EXACTLY ONCE to each recipient
     → System works even if some servers are down

WHY THIS IS THE HARDEST TYPE OF SYSTEM:

  Challenge 1: BIDIRECTIONAL REAL-TIME
  → HTTP is request-response (client asks, server answers)
  → Chat needs: server PUSHES to client without being asked
  → Solution: WebSocket — persistent bidirectional connection

  Challenge 2: STATEFUL CONNECTIONS
  → Each user has a persistent WebSocket to ONE specific server
  → If Alice is connected to Server A, Bob to Server B
  → Alice's message must route: Alice → Server A → ??? → Server B → Bob
  → HOW does Server A know Bob is on Server B?
  → This is the CONNECTION ROUTING problem

  Challenge 3: FAN-OUT AT SCALE
  → Group message to 1000 members
  → Each member on a DIFFERENT server
  → 1 message → 1000 deliveries across 100+ servers
  → Must happen in < 500ms

  Challenge 4: ORDERING GUARANTEES
  → Alice sends "I love" then "pizza"
  → Bob must see "I love" BEFORE "pizza"
  → Network can reorder packets
  → Multiple servers can process at different speeds
  → Must enforce per-conversation ordering

  Challenge 5: OFFLINE + ONLINE CONVERGENCE
  → Bob was offline for 2 hours
  → 500 messages waiting for him
  → Bob comes online → all 500 delivered in correct order
  → Plus new real-time messages flowing in
  → Must merge seamlessly

🔍 HOW TO APPROACH IT

Step 1: Identify the CONNECTION model
         → WebSocket for persistent bidirectional communication
         → Each client maintains ONE connection to ONE server
         → Server tracks: which users are connected to me?
         → Global registry: which user is on which server?

Step 2: Identify the MESSAGE FLOW
         → Sender → WebSocket → Chat Server → Message Store → 
           Route to recipient's server → WebSocket → Recipient
         → For groups: fan-out to all members' servers

Step 3: Identify the STORAGE model
         → Messages: append-only log per conversation (Project 29)
         → Ordered by sequence number within conversation
         → Sharded by conversation_id (Project 18)
         → Recent messages cached (Project 5)

Step 4: Identify the PRESENCE model
         → Heartbeat-based (Project 15)
         → User sends heartbeat every 30 seconds
         → No heartbeat for 60 seconds → mark offline
         → Store in fast cache with TTL (Project 5/23)

Step 5: Identify the DELIVERY GUARANTEE model
         → At-least-once delivery (retry until ack)
         → Idempotent on client side (deduplicate by message_id)
         → Exactly-once semantics via Project 29 patterns

Step 6: Identify the SCALING model
         → Consistent hashing (Project 14) for user→server mapping
         → Message queue (Project 11) for async fan-out
         → Read replicas (Project 13) for message history
         → CQRS (Project 30) for chat list vs message stream

MESSAGE FLOW — Alice sends "Hello" to Bob:

  1. Alice's phone → WebSocket → Chat Server A
     Message: {from: alice, to: bob, text: "Hello", msg_id: "M1"}

  2. Chat Server A:
     a. Validate message
     b. Assign sequence number (per conversation)
     c. Store in Message Store (persistent)
     d. Send "✓ Sent" ack to Alice
     e. Look up: "Where is Bob connected?"
        → Presence Service says: "Bob is on Chat Server B"

  3. Chat Server A → Message Router → Chat Server B
     Route message to Bob's server

  4. Chat Server B:
     a. Find Bob's WebSocket connection
     b. Push message to Bob via WebSocket
     c. Wait for delivery ack from Bob's client
     d. Send "✓✓ Delivered" back through router to Server A
     e. Server A pushes "✓✓ Delivered" to Alice

  5. Bob reads the message:
     a. Bob's client sends "read" event
     b. Chat Server B → Router → Server A → Alice
     c. Alice sees "✓✓ Read" (blue ticks)


GROUP MESSAGE FLOW — Alice sends to group of 50:

  1. Alice → Chat Server A → Store message
  2. Look up group members: [bob, charlie, ... 48 others]
  3. For each member:
     a. Look up which server they're on
     b. Route message to that server
  4. Each server pushes to its connected members
  5. Offline members: message queued for later delivery

  Optimization: Don't route individually to 50 servers
  → Batch by server: "Server B has [bob, dave, eve]"
  → Send ONE message to Server B with recipient list
  → Server B delivers to all three locally

📌 WHAT COULD BE DONE

TRANSPORT OPTIONS:

  Option A: HTTP Long Polling
  → Client sends request, server holds until new message
  → Simple but high overhead per "connection"
  → Used by: early Facebook Messenger

  Option B: Server-Sent Events (SSE)
  → Server pushes to client (one-directional)
  → Client still uses HTTP POST to send
  → Simpler than WebSocket but not bidirectional

  Option C: WebSocket (CHOSEN)
  → Full bidirectional persistent connection
  → Low overhead after handshake
  → Used by: WhatsApp, Slack, Discord
  → Best for real-time chat

  Option D: WebRTC (Project 47)
  → Peer-to-peer, no server relay
  → Great for voice/video, overkill for text
  → Used by: peer-to-peer calls

MESSAGE ROUTING OPTIONS:

  Option 1: Direct server-to-server
  → Each chat server connects to every other
  → O(N²) connections for N servers
  → Doesn't scale past ~20 servers

  Option 2: Central message router (CHOSEN)
  → All servers publish to internal message bus
  → Router delivers to target server
  → Scales to thousands of servers
  → Same concept as Event Bus (Project 29)

  Option 3: Pub/Sub per user
  → Each user has a "channel" in Redis pub/sub
  → Server subscribes to channels of connected users
  → Redis handles routing
  → Simple but Redis becomes bottleneck at scale

FOR THIS PROJECT:
  → WebSocket for client-server
  → Internal message bus for server-to-server
  → Redis for presence + routing table
  → Sharded database for message persistence

✅ WHAT SHOULD DO

1. ASSIGN monotonically increasing sequence numbers PER CONVERSATION
   → Message 1, 2, 3, 4... within chat between Alice and Bob
   → Client displays in sequence order (not arrival order)
   → Handles network reordering gracefully
   → Same concept as event ordering (Project 29)

2. STORE messages BEFORE delivering
   → Write to persistent store FIRST
   → Then attempt delivery
   → If delivery fails → message is safe in store
   → Recipient gets it when they come online

3. USE connection-level heartbeats
   → WebSocket ping/pong every 30 seconds
   → Detect dead connections quickly
   → Clean up resources for disconnected clients
   → Same heartbeat concept as Project 15

4. IMPLEMENT client-side message deduplication
   → Server may retry delivery (at-least-once)
   → Client tracks received message IDs
   → Duplicate message_id → skip display
   → Same idempotency pattern as Project 29

5. BATCH group message fan-out by server
   → Group of 500: don't send 500 individual messages
   → Group recipients by their connected server
   → Send ONE message per server with recipient list
   → Each server does local fan-out

6. SEPARATE hot data from cold data
   → Last 50 messages per chat: in Redis cache (Project 5)
   → Older messages: in database (loaded on scroll)
   → Very old messages: archived to cold storage
   → CQRS pattern (Project 30): recent view vs history view

7. USE consistent hashing for user-to-server assignment
   → hash(user_id) % N → assigned chat server
   → When user connects → route to assigned server
   → Other servers know WHERE to find each user
   → Same concept as Project 14

8. IMPLEMENT graceful reconnection
   → Client disconnects (network switch, sleep)
   → Client reconnects with last_received_sequence
   → Server sends all messages after that sequence
   → Seamless experience for user

❌ WHAT SHOULD NOT DO

1. NEVER store messages only in memory
   → Server crash = ALL messages lost
   → ALWAYS persist before acknowledging to sender
   → Memory cache is for SPEED, database is for SAFETY

2. NEVER use global sequence numbers
   → One counter for ALL messages across ALL chats
   → Becomes bottleneck at 1.2M messages/second
   → Use PER-CONVERSATION sequence numbers instead
   → Each conversation has independent ordering

3. NEVER fan-out synchronously for groups
   → Alice sends to group of 500
   → Synchronously delivering to 500 → 500 × 50ms = 25 seconds
   → Alice waits 25 seconds for "Sent" → terrible UX
   → DO: Persist message → ack to Alice → async fan-out

4. NEVER keep presence state in database
   → "Is Alice online?" queried millions of times per second
   → Database query per presence check = too slow
   → Use in-memory cache (Redis) with TTL-based expiry
   → Heartbeat refreshes TTL, no heartbeat = auto-offline

5. NEVER route messages through a single server
   → Single point of failure AND bottleneck
   → Use distributed message bus (Project 29)
   → Any chat server can route to any other

6. NEVER trust client-provided timestamps for ordering
   → Client clocks are unreliable (wrong timezone, skewed)
   → Server assigns sequence numbers and timestamps
   → Client uses server-assigned order for display

7. NEVER load entire chat history on connection
   → Chat with 50,000 messages → loading all = slow + wasteful
   → Load last 20 messages initially
   → Load more on scroll (pagination)
   → Same as infinite scroll pattern

🚫 WHEN NOT TO USE

1. WEBSOCKET IS WRONG WHEN:
   → Updates are infrequent (once per hour)
   → Maintaining persistent connection wastes resources
   → Use HTTP polling or SSE instead

2. PER-CONVERSATION SHARDING IS WRONG WHEN:
   → One chat has millions of messages (celebrity group)
   → Single shard overloaded
   → Use time-based sub-sharding within conversation

3. SYNCHRONOUS FAN-OUT IS WRONG WHEN:
   → Group size > 50 members
   → Sender would wait too long for ack
   → Use async queue-based fan-out

4. REDIS PRESENCE IS WRONG WHEN:
   → Need historical presence data (was Alice online yesterday?)
   → Redis is for CURRENT state only
   → Log presence changes to database for history

5. REAL-TIME CHAT ARCHITECTURE IS WRONG WHEN:
   → Your app is email-like (async, hours between responses)
   → Simple notification system suffices
   → Overkill for low-frequency communication

MECHANISM 2: MESSAGE ROUTING BETWEEN SERVERS
═══════════════════════════════════════════════════════

THE PROBLEM:
  → Alice on Server A sends message to Bob on Server B
  → Server A doesn't have Bob's WebSocket connection
  → HOW does the message get to Bob?

SOLUTION: Presence Registry + Internal Message Bus

  Step 1: PRESENCE REGISTRY (Redis)
    When user connects:
      SET user:alice:server "chat-server-A" EX 60
      SET user:bob:server "chat-server-B" EX 60
    Heartbeat refreshes TTL every 30 seconds

  Step 2: ROUTING LOOKUP
    Server A: GET user:bob:server → "chat-server-B"
    Now Server A knows: send to Server B

  Step 3: INTERNAL MESSAGE BUS (Project 29 concepts)
  
    ┌────────────┐                          ┌────────────┐
    │ Server A   │    Internal Bus          │ Server B   │
    │ (Alice)    │──→ channel:server-B ──→ │ (Bob)      │
    │            │                          │            │
    │ Publish:   │                          │ Subscribe: │
    │ {to: bob,  │                          │ Receive    │
    │  msg: ...} │                          │ Push to Bob│
    └────────────┘                          └────────────┘

    Each server subscribes to its OWN channel
    When message arrives for a user on this server → deliver locally

  Step 4: OFFLINE HANDLING
    Server A: GET user:charlie:server → (nil)
    Charlie is OFFLINE
    → Store message in "undelivered" queue for Charlie
    → When Charlie connects → flush queue to Charlie

MECHANISM 3: PER-CONVERSATION ORDERING
═══════════════════════════════════════════════════════

THE PROBLEM:
  Alice sends: "I" then "love" then "pizza"
  Network delivers: "love" then "I" then "pizza"
  Bob sees: "love I pizza" → WRONG

SOLUTION: Server-assigned sequence numbers

  Conversation "chat-alice-bob":
    Sequence counter: atomic integer per conversation
    
    Alice sends "I"     → server assigns seq=1
    Alice sends "love"  → server assigns seq=2
    Alice sends "pizza" → server assigns seq=3

  Client-side rendering:
    Messages arrive: seq=2, seq=1, seq=3
    Client sorts by sequence: seq=1, seq=2, seq=3
    Display: "I" "love" "pizza" ✅

  IMPLEMENTATION:
    → Redis INCR for atomic sequence: INCR chat:alice-bob:seq
    → Or database auto-increment within conversation partition
    → Sequence is per-conversation, NOT global
    → No global bottleneck (millions of conversations independently)

  GAP DETECTION:
    Client has: seq 1, 2, 3, 5 (missing 4)
    Client requests: "Give me message seq=4 for this chat"
    Server retrieves from store → delivers
    Client now has complete sequence

MECHANISM 4: GROUP MESSAGE FAN-OUT
═══════════════════════════════════════════════════════

NAIVE APPROACH (slow):
  Message to group of 500 members
  For each member:
    1. Look up their server
    2. Send individually
  = 500 lookups + 500 sends = slow

OPTIMIZED APPROACH:

  Step 1: Get all group members and their servers
    Members: {alice: server-A, bob: server-B, charlie: server-B,
              dave: server-A, eve: server-C, ...}

  Step 2: Group by server
    server-A: [alice, dave, ...]      (150 members)
    server-B: [bob, charlie, ...]     (180 members)
    server-C: [eve, ...]              (170 members)

  Step 3: Send ONE message per server with recipient list
    → server-A: {msg: "Hello", recipients: [alice, dave, ...]}
    → server-B: {msg: "Hello", recipients: [bob, charlie, ...]}
    → server-C: {msg: "Hello", recipients: [eve, ...]}

  Step 4: Each server does LOCAL fan-out
    server-A loops through its 150 connected users → push via WebSocket

  Result: 3 cross-server messages instead of 500 ✅

  For very large groups (> 1000):
    → Use message queue (Project 11) for async fan-out
    → Don't block sender's server
    → Workers process fan-out in background

MECHANISM 5: PRESENCE SERVICE
═══════════════════════════════════════════════════════

ONLINE/OFFLINE STATUS:

  On connect:
    SET presence:alice "online" EX 60
    SET presence:alice:server "server-A" EX 60
    SET presence:alice:last_seen {timestamp} EX 86400

  Heartbeat (every 30s):
    EXPIRE presence:alice 60      (refresh TTL)
    EXPIRE presence:alice:server 60

  On disconnect:
    DEL presence:alice
    DEL presence:alice:server
    SET presence:alice:last_seen {timestamp}

  Auto-offline (no heartbeat for 60s):
    Redis TTL expires → keys deleted automatically
    → User appears offline without explicit disconnect
    → Handles: app crash, network loss, phone sleep

  Query "Is Alice online?":
    EXISTS presence:alice → 1 (online) or 0 (offline)
    GET presence:alice:last_seen → "5 minutes ago"

TYPING INDICATOR:

  Alice starts typing in chat with Bob:
    → Alice's client sends: {type: "typing", chat_id: "chat-001"}
    → Server routes to Bob's server (same routing as messages)
    → Bob's client shows "Alice is typing..."
    → Auto-expires after 3 seconds (no explicit "stopped typing")
    → NOT persisted (ephemeral, fire-and-forget)

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: THUNDERING HERD ON RECONNECT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → Server crashes → 100,000 users disconnect
  → All 100,000 reconnect simultaneously
  → New server overwhelmed by connection storm
  → FIX: Client-side exponential backoff with jitter
  → FIX: Connection rate limiting at load balancer (Project 6)
  → FIX: Gradual reconnect window (random delay 0-30 seconds)

PITFALL 2: MESSAGE ORDERING ACROSS MULTIPLE SENDERS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → Group chat: Alice and Bob both send at "same time"
  → Alice's message hits Server A, Bob's hits Server B
  → Different servers assign sequence numbers independently?
  → FIX: Single sequence generator per conversation
  → FIX: Redis INCR on conversation key (atomic)
  → FIX: Or single "owner server" per conversation (Project 14)

PITFALL 3: PRESENCE FLAPPING
━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → User on mobile with bad network
  → Connection drops for 2 seconds → reconnects
  → All contacts see: "offline" then "online" repeatedly
  → FIX: Grace period before showing "offline" (30-60 seconds)
  → FIX: Client-side debounce of presence display
  → FIX: Only show "offline" if gone for > 2 minutes

PITFALL 4: UNDELIVERED MESSAGE QUEUE GROWS FOREVER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → User uninstalls app → never comes back
  → Their undelivered queue grows daily
  → Millions of abandoned queues consuming storage
  → FIX: TTL on undelivered messages (30 days)
  → FIX: Max queue size (10,000 messages)
  → FIX: Archive old undelivered to cold storage

PITFALL 5: HOT GROUP CHATS
━━━━━━━━━━━━━━━━━━━━━━━━━
  → Celebrity group chat with 100,000 members
  → Every message = 100,000 fan-out operations
  → 10 messages/second = 1,000,000 operations/second
  → FIX: Rate limit posting in large groups
  → FIX: Async fan-out with message queue workers
  → FIX: "Channel" mode (push notification, not real-time)

PITFALL 6: SPLIT BRAIN IN PRESENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  → Redis primary fails, replica promoted (Project 13)
  → Presence data might be stale on new primary
  → Users appear online when actually offline
  → FIX: Short TTL ensures self-correction within 60 seconds
  → FIX: Heartbeat renewal reestablishes truth quickly
  → FIX: Accept presence as "best effort" (not critical path)

class WebSocketConnection:
    def __init__(self, user_id: str, server_id: str):
        self.connection_id = f"ws-{uuid.uuid4().hex[:8]}"
        self.user_id = user_id
        self.server_id = server_id
        self.connected_at = time.time()
        self.last_activity = time.time()
        self.is_open = True
        self.received_messages: List[Dict] = []
        self._lock = threading.Lock()

    def send(self, data: Dict):
        if not self.is_open:
            raise ConnectionError(f"WebSocket closed")
        with self._lock:
            self.received_messages.append({
                "data": data, "received_at": time.time()
            })
            self.last_activity = time.time()

    def close(self):
        self.is_open = False

    def get_received(self, msg_type: str = None) -> List[Dict]:
        with self._lock:
            if msg_type:
                return [m["data"] for m in self.received_messages
                        if m["data"].get("type") == msg_type]
            return [m["data"] for m in self.received_messages]

class PresenceService:
    def __init__(self, offline_threshold: float = 60.0):
        self.offline_threshold = offline_threshold
        self.presence: Dict[str, Dict] = {}
        self.typing_indicators: Dict[str, Dict[str, float]] = {}
        self._lock = threading.Lock()
        self.status_changes: List[Dict] = []

    def user_connected(self, user_id: str, server_id: str):
        with self._lock:
            self.presence[user_id] = {
                "server_id": server_id,
                "last_heartbeat": time.time(),
                "connected_at": time.time(),
                "status": "online"
            }
            self.status_changes.append({
                "user_id": user_id, "status": "online",
                "time": time.strftime("%H:%M:%S")
            })

    def user_disconnected(self, user_id: str):
        with self._lock:
            if user_id in self.presence:
                self.presence[user_id]["status"] = "offline"
                self.presence[user_id]["last_seen"] = time.time()
                self.status_changes.append({
                    "user_id": user_id, "status": "offline",
                    "time": time.strftime("%H:%M:%S")
                })

    def heartbeat(self, user_id: str):
        with self._lock:
            if user_id in self.presence:
                self.presence[user_id]["last_heartbeat"] = time.time()
                self.presence[user_id]["status"] = "online"

    def is_online(self, user_id: str) -> bool:
        with self._lock:
            return self._is_online(user_id)

    def _is_online(self, user_id: str) -> bool:
        if user_id not in self.presence:
            return False
        info = self.presence[user_id]
        if info["status"] == "offline":
            return False
        if time.time() - info["last_heartbeat"] > self.offline_threshold:
            info["status"] = "offline"
            return False
        return True

    def get_server(self, user_id: str) -> Optional[str]:
        with self._lock:
            if self._is_online(user_id):
                return self.presence[user_id]["server_id"]
            return None

    def get_last_seen(self, user_id: str) -> str:
        with self._lock:
            info = self.presence.get(user_id)
            if not info:
                return "never"
            if self._is_online(user_id):
                return "online"
            last = info.get("last_seen", info.get("last_heartbeat", 0))
            elapsed = time.time() - last
            if elapsed < 60:
                return f"{int(elapsed)}s ago"
            elif elapsed < 3600:
                return f"{int(elapsed/60)}m ago"
            return f"{int(elapsed/3600)}h ago"

    def set_typing(self, user_id: str, conversation_id: str):
        with self._lock:
            if user_id not in self.typing_indicators:
                self.typing_indicators[user_id] = {}
            self.typing_indicators[user_id][conversation_id] = time.time()

    def get_typing_users(self, conversation_id: str,
                         exclude_user: str = "") -> List[str]:
        with self._lock:
            typing = []
            now = time.time()
            for uid, convos in list(self.typing_indicators.items()):
                if uid == exclude_user:
                    continue
                if conversation_id in convos:
                    if now - convos[conversation_id] < 3.0:
                        typing.append(uid)
            return typing

    def get_online_users(self, user_ids: List[str]) -> Dict[str, bool]:
        with self._lock:
            return {uid: self._is_online(uid) for uid in user_ids}

    def print_status(self):
        print(f"\n  PRESENCE SERVICE:")
        with self._lock:
            for uid, info in self.presence.items():
                online = self._is_online(uid)
                status = "🟢" if online else "⚪"
                server = info.get("server_id", "?")
                print(f"    {status} {uid:<15} server={server}")

class MessageStore:
    def __init__(self):
        self.messages: Dict[str, List[Message]] = defaultdict(list)
        self.sequences: Dict[str, int] = defaultdict(int)
        self.conversations: Dict[str, Conversation] = {}
        self._lock = threading.Lock()
        self.total_stored = 0

    def create_conversation(self, conv: Conversation):
        with self._lock:
            self.conversations[conv.conversation_id] = conv

    def get_conversation(self, conv_id: str) -> Optional[Conversation]:
        with self._lock:
            return self.conversations.get(conv_id)

    def get_user_conversations(self, user_id: str) -> List[Conversation]:
        with self._lock:
            result = [c for c in self.conversations.values()
                      if user_id in c.members]
            result.sort(key=lambda c: c.last_message_time, reverse=True)
            return result

    def store_message(self, message: Message) -> int:
        with self._lock:
            cid = message.conversation_id
            self.sequences[cid] += 1
            message.sequence = self.sequences[cid]
            message.delivery_status = DeliveryStatus.SENT.value
            self.messages[cid].append(message)
            self.total_stored += 1
            if cid in self.conversations:
                c = self.conversations[cid]
                c.last_message_seq = message.sequence
                c.last_message_preview = message.content[:50]
                c.last_message_time = message.timestamp
            return message.sequence

    def get_messages(self, cid: str, after_seq: int = 0,
                     limit: int = 50) -> List[Message]:
        with self._lock:
            msgs = self.messages.get(cid, [])
            return [m for m in msgs if m.sequence > after_seq][:limit]

    def get_recent_messages(self, cid: str,
                            limit: int = 20) -> List[Message]:
        with self._lock:
            return self.messages.get(cid, [])[-limit:]

    def update_delivery_status(self, cid: str, msg_id: str,
                               status: str):
        with self._lock:
            for msg in self.messages.get(cid, []):
                if msg.message_id == msg_id:
                    msg.delivery_status = status
                    break

    def get_stats(self) -> Dict:
        with self._lock:
            return {
                "total_messages": self.total_stored,
                "conversations": len(self.conversations),
                "per_conversation": {
                    cid: len(msgs)
                    for cid, msgs in self.messages.items()
                }
            }

class MessageRouter:
    def __init__(self):
        self.servers: Dict[str, 'ChatServer'] = {}
        self._lock = threading.Lock()
        self.routed_count = 0

    def register_server(self, server_id: str, server: 'ChatServer'):
        with self._lock:
            self.servers[server_id] = server

    def route_message(self, message: Message,
                      target_server_id: str,
                      recipient_ids: List[str]):
        with self._lock:
            server = self.servers.get(target_server_id)
            if server:
                server.receive_routed_message(message, recipient_ids)
                self.routed_count += 1

    def route_delivery_ack(self, sender_id: str, msg_id: str,
                           conv_id: str, delivered_to: str):
        for server in self.servers.values():
            with server._lock:
                if sender_id in server.connections:
                    server._send_to_user(sender_id, {
                        "type": "delivery_ack",
                        "message_id": msg_id,
                        "status": "delivered",
                        "delivered_to": delivered_to
                    })
                    server.store.update_delivery_status(
                        conv_id, msg_id, DeliveryStatus.DELIVERED.value
                    )
                    break

class ChatServer:
    def __init__(self, server_id: str, store: MessageStore,
                 presence: PresenceService, router: MessageRouter):
        self.server_id = server_id
        self.store = store
        self.presence = presence
        self.router = router
        self.connections: Dict[str, WebSocketConnection] = {}
        self._lock = threading.Lock()
        self.undelivered: Dict[str, List[Message]] = defaultdict(list)
        self.messages_received = 0
        self.messages_delivered = 0
        self.messages_routed = 0

    def connect_user(self, user_id: str) -> WebSocketConnection:
        ws = WebSocketConnection(user_id, self.server_id)
        with self._lock:
            if user_id in self.connections:
                self.connections[user_id].close()
            self.connections[user_id] = ws
        self.presence.user_connected(user_id, self.server_id)
        self.router.register_server(self.server_id, self)
        self._deliver_queued(user_id)
        return ws

    def disconnect_user(self, user_id: str):
        with self._lock:
            if user_id in self.connections:
                self.connections[user_id].close()
                del self.connections[user_id]
        self.presence.user_disconnected(user_id)

    def handle_send_message(self, sender_id: str, conv_id: str,
                            content: str,
                            msg_type: str = "text") -> Message:
        message = Message(
            conversation_id=conv_id, sender_id=sender_id,
            message_type=msg_type, content=content
        )
        seq = self.store.store_message(message)
        self.messages_received += 1

        self._send_to_user(sender_id, {
            "type": "delivery_ack",
            "message_id": message.message_id,
            "status": "sent", "sequence": seq
        })

        conv = self.store.get_conversation(conv_id)
        if conv:
            recipients = [m for m in conv.members if m != sender_id]
            self._route_message(message, recipients)
        return message

    def handle_typing(self, user_id: str, conv_id: str):
        self.presence.set_typing(user_id, conv_id)
        conv = self.store.get_conversation(conv_id)
        if conv:
            for member in conv.members:
                if member != user_id:
                    self._send_to_user(member, {
                        "type": "typing",
                        "user_id": user_id,
                        "conversation_id": conv_id
                    })

    def handle_read_receipt(self, user_id: str, conv_id: str,
                            up_to_seq: int):
        conv = self.store.get_conversation(conv_id)
        if not conv:
            return
        recent = self.store.get_recent_messages(conv_id, 50)
        for msg in recent:
            if msg.sequence <= up_to_seq and msg.sender_id != user_id:
                self.store.update_delivery_status(
                    conv_id, msg.message_id, DeliveryStatus.READ.value
                )
                self._send_to_user(msg.sender_id, {
                    "type": "read_receipt",
                    "message_id": msg.message_id,
                    "read_by": user_id, "status": "read"
                })

    def receive_routed_message(self, message: Message,
                               recipient_ids: List[str]):
        for rid in recipient_ids:
            delivered = self._send_to_user(rid, {
                "type": "new_message",
                "message": message.to_dict()
            })
            if delivered:
                self.messages_delivered += 1
                self.router.route_delivery_ack(
                    message.sender_id, message.message_id,
                    message.conversation_id, rid
                )
            else:
                self.undelivered[rid].append(message)

    def _route_message(self, message: Message, recipients: List[str]):
        local = []
        remote: Dict[str, List[str]] = defaultdict(list)
        for r in recipients:
            server = self.presence.get_server(r)
            if server == self.server_id:
                local.append(r)
            elif server:
                remote[server].append(r)
            else:
                self.undelivered[r].append(message)

        for r in local:
            if self._send_to_user(r, {
                "type": "new_message",
                "message": message.to_dict()
            }):
                self.messages_delivered += 1

        for target, rlist in remote.items():
            self.router.route_message(message, target, rlist)
            self.messages_routed += len(rlist)

    def _send_to_user(self, user_id: str, data: Dict) -> bool:
        with self._lock:
            ws = self.connections.get(user_id)
            if ws and ws.is_open:
                try:
                    ws.send(data)
                    return True
                except ConnectionError:
                    return False
        return False

    def _deliver_queued(self, user_id: str):
        queued = self.undelivered.pop(user_id, [])
        queued.sort(key=lambda m: m.sequence)
        for msg in queued:
            self._send_to_user(user_id, {
                "type": "new_message",
                "message": msg.to_dict(), "queued": True
            })
            self.messages_delivered += 1

    def get_stats(self) -> Dict:
        with self._lock:
            return {
                "server_id": self.server_id,
                "connections": len(self.connections),
                "users": list(self.connections.keys()),
                "received": self.messages_received,
                "delivered": self.messages_delivered,
                "routed": self.messages_routed,
                "queued": sum(len(q) for q in self.undelivered.values())
            }

class ChatSystem:
    def __init__(self, num_servers: int = 3):
        self.message_store = MessageStore()
        self.presence = PresenceService(offline_threshold=60)
        self.router = MessageRouter()
        self.servers: Dict[str, ChatServer] = {}
        for i in range(num_servers):
            sid = f"chat-server-{i+1}"
            server = ChatServer(sid, self.message_store,
                                self.presence, self.router)
            self.servers[sid] = server
            self.router.register_server(sid, server)
        self.user_server_map: Dict[str, str] = {}
        self.server_ids = list(self.servers.keys())
        self.user_connections: Dict[str, WebSocketConnection] = {}

    def _assign_server(self, user_id: str) -> str:
        if user_id not in self.user_server_map:
            idx = hash(user_id) % len(self.server_ids)
            self.user_server_map[user_id] = self.server_ids[idx]
        return self.user_server_map[user_id]

    def user_connect(self, user_id: str) -> WebSocketConnection:
        sid = self._assign_server(user_id)
        server = self.servers[sid]
        ws = server.connect_user(user_id)
        self.user_connections[user_id] = ws
        return ws

    def user_disconnect(self, user_id: str):
        sid = self._assign_server(user_id)
        self.servers[sid].disconnect_user(user_id)
        self.user_connections.pop(user_id, None)

    def create_direct_chat(self, user_a: str,
                           user_b: str) -> Conversation:
        members = sorted([user_a, user_b])
        conv_id = f"dm-{members[0]}-{members[1]}"
        existing = self.message_store.get_conversation(conv_id)
        if existing:
            return existing
        conv = Conversation(
            conversation_id=conv_id, chat_type="direct",
            members=members, name=f"{user_a} & {user_b}"
        )
        self.message_store.create_conversation(conv)
        return conv

    def create_group_chat(self, name: str,
                          members: List[str]) -> Conversation:
        conv = Conversation(
            chat_type="group", members=members, name=name
        )
        self.message_store.create_conversation(conv)
        return conv

    def send_message(self, sender_id: str, conv_id: str,
                     content: str) -> Optional[Message]:
        sid = self._assign_server(sender_id)
        return self.servers[sid].handle_send_message(
            sender_id, conv_id, content
        )

    def send_typing(self, user_id: str, conv_id: str):
        sid = self._assign_server(user_id)
        self.servers[sid].handle_typing(user_id, conv_id)

    def send_read_receipt(self, user_id: str, conv_id: str,
                          up_to_seq: int):
        sid = self._assign_server(user_id)
        self.servers[sid].handle_read_receipt(
            user_id, conv_id, up_to_seq
        )

    def get_chat_history(self, conv_id: str,
                         limit: int = 20) -> List[Dict]:
        msgs = self.message_store.get_recent_messages(conv_id, limit)
        return [m.to_dict() for m in msgs]

    def get_user_chats(self, user_id: str) -> List[Dict]:
        convs = self.message_store.get_user_conversations(user_id)
        result = []
        for c in convs:
            online_status = self.presence.get_online_users(c.members)
            result.append({
                "conversation_id": c.conversation_id,
                "name": c.name,
                "type": c.chat_type,
                "members": c.members,
                "members_online": sum(1 for v in online_status.values() if v),
                "last_message": c.last_message_preview,
                "last_message_time": (
                    time.strftime("%H:%M:%S",
                                  time.localtime(c.last_message_time))
                    if c.last_message_time else "never"
                )
            })
        return result

    def print_dashboard(self):
        print(f"\n  {'#'*65}")
        print(f"  #  CHATSPHERE SYSTEM DASHBOARD")
        print(f"  {'#'*65}")
        store_stats = self.message_store.get_stats()
        print(f"\n  MESSAGES: {store_stats['total_messages']} total | "
              f"{store_stats['conversations']} conversations")
        print(f"\n  SERVERS:")
        for sid, server in self.servers.items():
            s = server.get_stats()
            print(f"    {sid}: {s['connections']} users "
                  f"[{', '.join(s['users'][:5])}{'...' if len(s['users'])>5 else ''}] "
                  f"| recv={s['received']} del={s['delivered']} "
                  f"route={s['routed']} queue={s['queued']}")
        self.presence.print_status()
        print(f"  {'#'*65}")

def demo_concurrent_load():
    print("\n" + "=" * 70)
    print("  DEMO 5: CONCURRENT LOAD — Multiple Chats Simultaneously")
    print("=" * 70)

    system = ChatSystem(num_servers=3)

    # Create 10 users
    num_users = 10
    users = [f"user-{i:02d}" for i in range(num_users)]

    print(f"\n  --- Connecting {num_users} users ---")
    for user in users:
        system.user_connect(user)

    # Create multiple conversations
    conversations = []

    # 5 direct chats
    for i in range(5):
        a, b = random.sample(users, 2)
        conv = system.create_direct_chat(a, b)
        conversations.append(conv)

    # 2 group chats
    group1 = system.create_group_chat(
        "Engineering", random.sample(users, 6)
    )
    group2 = system.create_group_chat(
        "All Hands", users  # Everyone
    )
    conversations.extend([group1, group2])

    print(f"  Created {len(conversations)} conversations "
          f"({5} DM + {2} groups)")

    # Simulate concurrent messaging
    print(f"\n  --- Sending messages concurrently ---")
    total_messages = 0
    threads = []
    msg_lock = threading.Lock()

    def send_batch(sender, conv_id, count):
        nonlocal total_messages
        for i in range(count):
            system.send_message(
                sender, conv_id,
                f"Message {i+1} from {sender}"
            )
            time.sleep(random.uniform(0.001, 0.01))
        with msg_lock:
            total_messages += count

    start_time = time.time()

    for conv in conversations:
        for member in conv.members[:3]:  # Max 3 senders per conv
            t = threading.Thread(
                target=send_batch,
                args=(member, conv.conversation_id,
                      random.randint(3, 8))
            )
            threads.append(t)
            t.start()

    for t in threads:
        t.join()

    elapsed = time.time() - start_time

    print(f"  Total messages sent: {total_messages}")
    print(f"  Time elapsed: {elapsed:.3f}s")
    print(f"  Throughput: {total_messages/elapsed:.0f} messages/sec")

    # Verify message ordering per conversation
    print(f"\n  --- Ordering verification ---")
    all_ordered = True
    for conv in conversations:
        history = system.get_chat_history(
            conv.conversation_id, limit=100
        )
        sequences = [m["sequence"] for m in history]
        ordered = all(sequences[i] < sequences[i+1]
                      for i in range(len(sequences)-1))
        if not ordered:
            all_ordered = False
            print(f"  ❌ {conv.conversation_id}: OUT OF ORDER!")
        else:
            print(f"  ✅ {conv.name or conv.conversation_id}: "
                  f"{len(history)} messages, correctly ordered")

    print(f"\n  Global ordering: "
          f"{'✅ ALL CORRECT' if all_ordered else '❌ ISSUES FOUND'}")

    # System dashboard
    system.print_dashboard()

    # Store stats
    stats = system.message_store.get_stats()
    print(f"\n  Store: {stats['total_messages']} messages across "
          f"{stats['conversations']} conversations")

  → Alice posts: "Beautiful sunset! 🌅"
  → All 500 followers should see this in their news feed
  → But WHEN does the fan-out happen?

  OPTION A: FAN-OUT ON WRITE (push model)
  → When Alice posts → immediately write to all 500 followers' feeds
  → Fast reads (feed is pre-built) but slow writes for popular users
  → Celebrity with 50M followers → 50M writes per post = slow

  OPTION B: FAN-OUT ON READ (pull model)
  → When Bob opens his feed → query all people he follows
  → Fast writes but slow reads (must aggregate at query time)
  → Following 1000 people → merge 1000 timelines = slow

  OPTION C: HYBRID (what Twitter/Instagram actually do)
  → Regular users: fan-out on write (push to followers' feeds)
  → Celebrities: fan-out on read (merge at query time)
  → Best of both worlds
