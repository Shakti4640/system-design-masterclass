📌 WHAT COULD BE DONE

Option A: Flat File Storage (CSV/JSON files)
  → Write JSON to a file after every change
  → Read entire file on startup
  → No query capability — must scan entire file
  → No relationships — just raw data
  → When: Tiny config files, logs
  → NOT for transactional data

Option B: Key-Value Store (like a dictionary on disk)
  → Store by key, retrieve by key
  → Very fast for exact lookups
  → Cannot search "all items where price > 10"
  → When: Caching, session storage (Project 5, 23)

Option C: Relational Database (SQL)
  → Structured tables with enforced schemas
  → Powerful query language (SQL)
  → Relationships with foreign keys
  → ACID guarantees
  → When: Business data, transactions, reporting
  → THIS IS WHAT WE USE FOR THIS PROJECT

Option D: Document Database (NoSQL — MongoDB style)
  → Store JSON documents directly
  → Flexible schema (no fixed columns)
  → Good for varied/nested data structures
  → When: Content management, catalogs, prototyping
  → Tradeoff: Weaker consistency guarantees

Option E: Graph Database (Neo4j style)
  → Nodes and edges as first-class citizens
  → When: Social networks, recommendations (Project 40)

For THIS project → Option C (SQLite — file-based relational DB)
→ Zero setup, included with Python
→ Same SQL concepts apply to PostgreSQL/MySQL
→ Perfect for learning, prototyping, and small-medium apps

✅ WHAT SHOULD DO

1. ALWAYS design schema BEFORE writing code
   → Draw out tables, columns, relationships on paper
   → Identify primary keys, foreign keys, constraints
   → Think about what queries you'll need
   → Schema changes later are PAINFUL

2. ALWAYS use PRIMARY KEYS
   → Every table needs a unique identifier
   → Usually auto-incrementing integer or UUID
   → This is HOW you reference specific rows

3. ALWAYS use FOREIGN KEYS
   → If order_items references an order → make it a foreign key
   → Database ENFORCES the relationship
   → Cannot create order_item for non-existent order
   → Cannot delete order that has order_items (unless cascading)

4. ALWAYS add CONSTRAINTS
   → NOT NULL: Field must have a value
   → UNIQUE: No duplicates allowed
   → CHECK: Value must satisfy a condition (price > 0)
   → DEFAULT: Automatic value if not provided

5. ALWAYS use TRANSACTIONS for multi-step operations
   → Creating an order = insert order + insert order_items + update totals
   → If ANY step fails → ALL steps must roll back
   → Never leave data in a half-finished state

6. ALWAYS add TIMESTAMPS
   → created_at: When the record was first created
   → updated_at: When it was last modified
   → Essential for debugging, auditing, sorting

7. ALWAYS use PARAMETERIZED QUERIES
   → NEVER build SQL by string concatenation
   → SQL injection: "'; DROP TABLE orders; --"
   → Use ? placeholders and let the database handle escaping

8. ALWAYS normalize to appropriate level
   → Don't store "Margherita Pizza, $12.99" as text in orders
   → Store item_id and look it up → single source of truth
   → If price changes, you don't update 10,000 order records

❌ WHAT SHOULD NOT DO

1. NEVER use string concatenation for SQL
   ❌ f"SELECT * FROM items WHERE name = '{user_input}'"
   ✅ "SELECT * FROM items WHERE name = ?", (user_input,)
   → SQL injection is the #1 web vulnerability for decades

2. NEVER store everything in one giant table
   ❌ orders table with: order_id, item1_name, item1_price,
      item2_name, item2_price, item3_name, item3_price...
   → What if order has 20 items? 100 columns?
   → Use proper normalization with junction tables

3. NEVER ignore database errors
   → IntegrityError: constraint violated (duplicate, null, etc.)
   → OperationalError: database locked, disk full
   → Always catch, log, and return appropriate HTTP status

4. NEVER leave transactions open
   → BEGIN transaction → do work → COMMIT or ROLLBACK
   → Open transaction = locked rows = blocked other users
   → Always commit or rollback, even in error paths

5. NEVER store derived data without reason
   → Total price = sum of line items
   → If you store it separately, it can get out of sync
   → UNLESS you need it for performance (denormalization — Project 30)
   → When you DO store it, update it in the SAME transaction

6. NEVER use the database as a message queue
   → Polling a table for "new" rows = inefficient
   → Use actual message queues (Project 11)

7. NEVER skip schema migrations planning
   → Version 1: items table has 5 columns
   → Version 2: need to add "allergens" column
   → Need a migration strategy from day one
   → ALTER TABLE is your friend, but plan ahead

🚫 WHEN NOT TO USE

1. Relational DB is NOT ideal when:
   → Data has no fixed structure
      Example: Each product has completely different attributes
      → Document DB might be better
   
   → You need sub-millisecond lookups by key
      Example: Session tokens, cache entries
      → Key-Value store is faster (Project 5, 23)
   
   → Data is heavily interconnected (graph-like)
      Example: "Friends of friends who also like X"
      → Graph DB handles this natively (Project 40)
   
   → You're storing billions of time-series data points
      Example: IoT sensor readings every second
      → Time-series DB is optimized for this

2. SQLite specifically is NOT ideal when:
   → Multiple processes write simultaneously
      → SQLite uses file-level locking
      → PostgreSQL handles concurrent writes much better
   
   → Database is larger than ~1TB
      → SQLite has practical size limits
   
   → You need advanced features
      → Stored procedures, materialized views, partitioning
      → Use PostgreSQL

3. ANY database is NOT ideal when:
   → Data is temporary and disposable
      → Use in-memory cache instead (Project 5)
   
   → Data is a stream of events being processed
      → Use message queue (Project 11)

  Each ROW is stored as a sequence of bytes:
  [row_header][col1_data][col2_data][col3_data]...
  
  To find a specific row without an index:
  → Must scan EVERY page → EVERY row → check condition
  → This is a FULL TABLE SCAN → O(n) → slow for large tables
  → Solution: Indexes (Project 10)

PART 2: HOW SQL QUERIES EXECUTE
═══════════════════════════════════════════════════════

  Your SQL: SELECT * FROM menu_items WHERE price > 10

  Step 1: PARSER
    → Reads the SQL text
    → Checks syntax (is it valid SQL?)
    → Produces an Abstract Syntax Tree (AST)
    
    AST:
    └── SELECT
        ├── columns: *
        ├── FROM: menu_items
        └── WHERE: price > 10

  Step 2: QUERY PLANNER / OPTIMIZER
    → Decides HOW to execute the query
    → Options:
      a) Full table scan (read every row)
      b) Use index on price column (if it exists)
    → Picks the cheapest option based on statistics
    → Outputs: Query Execution Plan

  Step 3: EXECUTOR
    → Follows the plan
    → Reads pages from disk (or cache)
    → Filters rows matching condition
    → Returns result set

  Step 4: RESULT
    → Rows matching the query are returned
    → Formatted as specified (columns requested)

PART 3: ACID PROPERTIES — THE DATABASE GUARANTEE
═══════════════════════════════════════════════════════

  A — ATOMICITY (All or Nothing)
  ────────────────────────────────
  Transaction: Transfer $100 from Account A to Account B
    Step 1: Subtract $100 from A
    Step 2: Add $100 to B
  
  If Step 2 fails → Step 1 is UNDONE (rolled back)
  Money doesn't vanish into thin air
  
  In our restaurant:
    Create order:
      Step 1: INSERT into orders
      Step 2: INSERT into order_items (line 1)
      Step 3: INSERT into order_items (line 2)
      Step 4: UPDATE order total
    
    If Step 3 fails → Steps 1, 2 are undone
    No half-created orders exist

  C — CONSISTENCY (Rules Always Honored)
  ────────────────────────────────────────
  If constraint says price > 0:
    → Database REJECTS INSERT with price = -5
    → Even inside a transaction
    → Data ALWAYS satisfies all defined rules
  
  If foreign key says order_items.order_id → orders.id:
    → Cannot create order_item for order that doesn't exist
    → Cannot delete order that has order_items (unless CASCADE)

  I — ISOLATION (Transactions Don't Interfere)
  ──────────────────────────────────────────────
  Transaction A: Reading total revenue
  Transaction B: Creating a new order
  
  Isolation ensures:
    → A sees a CONSISTENT snapshot
    → Either sees the new order fully, or not at all
    → Never sees a half-inserted order
  
  Isolation Levels (from weakest to strongest):
    1. Read Uncommitted → can see uncommitted changes (dirty reads)
    2. Read Committed   → only sees committed changes
    3. Repeatable Read  → same query returns same results within txn
    4. Serializable     → transactions execute as if one at a time
    
    Higher isolation = safer but slower
    Default in most DBs: Read Committed

  D — DURABILITY (Survives Crashes)
  ──────────────────────────────────
  Once COMMIT returns successfully:
    → Data is written to disk
    → Even if power goes out 1ms later → data is safe
    → Even if OS crashes → data is safe
  
  HOW: Write-Ahead Log (WAL)
    → Before changing actual data pages
    → Write the CHANGE to a log file first
    → Log is flushed to disk (fsync)
    → Then apply change to data pages
    → If crash before apply → replay log on recovery
  
  ┌────────────────────────────────────────────────┐
  │  Transaction: INSERT order #500                 │
  │                                                 │
  │  1. Write to WAL: "INSERT order #500 data..."  │
  │  2. Flush WAL to disk (fsync) ✅               │
  │  3. Return COMMIT SUCCESS to application       │
  │  4. Later: Apply change to actual data pages   │
  │                                                 │
  │  If crash between step 3 and 4:                │
  │  → On restart, replay WAL → order #500 exists  │
  └────────────────────────────────────────────────┘

PART 4: HOW FOREIGN KEYS WORK INTERNALLY
═══════════════════════════════════════════════════════

  CREATE TABLE order_items (
      id INTEGER PRIMARY KEY,
      order_id INTEGER REFERENCES orders(id),
      item_id INTEGER REFERENCES menu_items(id)
  );

  When you INSERT into order_items:
    → DB checks: Does orders.id = {order_id} exist?
    → DB checks: Does menu_items.id = {item_id} exist?
    → If either doesn't exist → INSERT REJECTED
    → Error: FOREIGN KEY constraint failed

  When you DELETE from orders:
    → DB checks: Do any order_items reference this order?
    → If yes → DELETE REJECTED (default behavior)
    → Unless ON DELETE CASCADE → auto-delete referencing rows
    → Or ON DELETE SET NULL → set foreign key to NULL

  CASCADE Example:
    DELETE FROM orders WHERE id = 5;
    → Automatically deletes all order_items WHERE order_id = 5
    → One command cleans up everything

═══════════════════════════════════════════════════════
PART 5: HOW TRANSACTIONS WORK INTERNALLY
═══════════════════════════════════════════════════════

  BEGIN TRANSACTION;
    INSERT INTO orders (...) VALUES (...);
    INSERT INTO order_items (...) VALUES (...);
    INSERT INTO order_items (...) VALUES (...);
    UPDATE orders SET total_price = 45.97 WHERE id = 5;
  COMMIT;

  Internally:
  1. BEGIN → Database creates a savepoint
  2. Each statement executes but changes are "tentative"
  3. Other transactions may not see these changes (isolation)
  4. COMMIT → All changes become permanent atomically
  5. If any statement FAILS between BEGIN and COMMIT:
     → ROLLBACK → All tentative changes are undone
     → Database returns to state before BEGIN

  ┌─────────────────────────────────────────────────┐
  │  State Before: orders = [1, 2, 3, 4]           │
  │                                                  │
  │  BEGIN                                           │
  │    INSERT order 5    ← tentative                │
  │    INSERT item 5a    ← tentative                │
  │    INSERT item 5b    ← FAILS (invalid item_id) │
  │  ROLLBACK (automatic)                           │
  │                                                  │
  │  State After: orders = [1, 2, 3, 4]            │
  │  Order 5 was NEVER created                      │
  └─────────────────────────────────────────────────┘

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: Forgetting to Enable Foreign Keys in SQLite
  → SQLite has foreign keys DISABLED by default
  → Must run: PRAGMA foreign_keys = ON;
  → Without this, foreign key constraints are silently ignored
  → Your data integrity is an illusion

PITFALL 2: SQL Injection
  → User input: "'; DROP TABLE orders; --"
  ❌ f"SELECT * FROM items WHERE name = '{user_input}'"
     Becomes: SELECT * FROM items WHERE name = ''; DROP TABLE orders; --'
     → Your orders table is GONE
  ✅ cursor.execute("SELECT * FROM items WHERE name = ?", (user_input,))
     → Database treats entire input as a literal string value

PITFALL 3: Not Using Transactions for Multi-Table Operations
  → Insert order → Insert order_items → server crashes
  → Order exists but has no items
  → Ghost order confuses the entire system
  → FIX: Wrap in transaction (BEGIN → work → COMMIT/ROLLBACK)

PITFALL 4: N+1 Query Problem
  → Get 100 orders: SELECT * FROM orders LIMIT 100
  → For each order, get items: SELECT * FROM order_items WHERE order_id = ?
  → 1 + 100 = 101 queries for one page
  → FIX: Use JOIN to get everything in one query
  → SELECT * FROM orders JOIN order_items ON orders.id = order_items.order_id

PITFALL 5: Storing Money as FLOAT
  ❌ price REAL → 0.1 + 0.2 = 0.30000000000000004
  ✅ price stored as INTEGER in cents → 1299 means $12.99
  ✅ Or use DECIMAL(10,2) in PostgreSQL/MySQL
  → For SQLite: we use REAL but round carefully

PITFALL 6: No Indexes on Frequently Queried Columns
  → WHERE status = 'preparing' on 10 million rows → full table scan
  → Solution: CREATE INDEX (Project 10 deep dive)
  → For now: know that indexes exist and matter

PITFALL 7: Forgetting updated_at Timestamps
  → Record modified but updated_at still shows creation time
  → Cannot track when changes happened
  → FIX: Always update updated_at in every UPDATE query

PITFALL 8: Connection Leaks
  → Open database connection → do work → forget to close
  → Connections pile up → database rejects new ones
  → FIX: Always use try/finally or context managers (with statement)

PITFALL 9: Mixing Business Logic and SQL
  → Complex calculations done partly in Python, partly in SQL
  → Leads to inconsistencies
  → Rule: Let the database handle data operations (filtering, summing)
  → Let the application handle business decisions (can this order be cancelled?)

📈 SCALABILITY CONSIDERATIONS

FROM PROJECT 1: Single server limits (not repeated)
FROM PROJECT 2: Payload size, pagination (not repeated)

NEW FOR DATABASES:

1. CONNECTION POOLING
   → Creating a new DB connection: ~5-50ms overhead
   → If every request creates/closes a connection → slow
   → Solution: Pool of pre-created connections, reuse them
   → For SQLite: less relevant (file-based, in-process)
   → For PostgreSQL/MySQL: CRITICAL (Project 13)

2. READ vs WRITE PATTERNS
   → Most apps: 90% reads, 10% writes
   → One database handles both → write lock blocks reads
   → Solution: Read replicas (Project 13)
     → Primary handles writes
     → Replicas handle reads
     → Replication keeps them in sync

3. TABLE SIZE LIMITS
   → 1,000 rows: Any query is fast
   → 1,000,000 rows: Need indexes (Project 10)
   → 100,000,000 rows: Need partitioning or sharding (Project 18)
   → 1,000,000,000+ rows: Need distributed database (Project 46)

4. QUERY COMPLEXITY
   → Simple SELECT by ID: < 1ms
   → JOIN across 3 tables with filters: 10-100ms
   → Aggregation across millions of rows: 1-10 seconds
   → Complex reporting queries: seconds to minutes
   → Solution: Pre-compute, materialized views (Project 30)

5. WRITE THROUGHPUT
   → SQLite: ~50,000 inserts/sec (single writer)
   → PostgreSQL: ~10,000-50,000 inserts/sec (concurrent writers)
   → Need more? → Sharding (Project 18) or async writes (Project 11)

6. BACKUP AND RECOVERY
   → What if the disk dies?
   → Regular backups (full + incremental)
   → Point-in-time recovery from WAL
   → Replication to another machine (Project 13)

class DatabaseManager:
    """
    Manages SQLite database lifecycle:
    - Schema creation with migrations
    - Connection management (thread-safe)
    - All CRUD operations
    - Transaction handling
    """
    
    def __init__(self, db_path="restaurant.db"):
        self.db_path = db_path
        self.local = threading.local()
        self._initialize_schema()
    
    # ---- Connection Management ----
    
    def get_connection(self):
        """Thread-local connection (each thread gets its own)"""
        if not hasattr(self.local, 'connection') or self.local.connection is None:
            conn = sqlite3.connect(self.db_path)
            # Return rows as dictionaries instead of tuples
            conn.row_factory = sqlite3.Row
            # CRITICAL: Enable foreign key enforcement
            conn.execute("PRAGMA foreign_keys = ON")
            # Enable WAL mode for better concurrent read performance
            conn.execute("PRAGMA journal_mode = WAL")
            self.local.connection = conn
        return self.local.connection
    
    def close_connection(self):
        """Close thread-local connection"""
        if hasattr(self.local, 'connection') and self.local.connection:
            self.local.connection.close()
            self.local.connection = None
    
    # ---- Schema Definition ----
    
    def _initialize_schema(self):
        """Create tables if they don't exist"""
        conn = self.get_connection()
        cursor = conn.cursor()
        
        # ---- TABLE: categories ----
        cursor.execute("""
            CREATE TABLE IF NOT EXISTS categories (
                id          INTEGER PRIMARY KEY AUTOINCREMENT,
                name        TEXT NOT NULL UNIQUE,
                description TEXT DEFAULT '',
                created_at  TEXT NOT NULL DEFAULT (datetime('now'))
            )
        """)
        
        # ---- TABLE: menu_items ----
        cursor.execute("""
            CREATE TABLE IF NOT EXISTS menu_items (
                id             INTEGER PRIMARY KEY AUTOINCREMENT,
                name           TEXT NOT NULL,
                category_id    INTEGER NOT NULL,
                price          REAL NOT NULL CHECK (price >= 0),
                available      INTEGER NOT NULL DEFAULT 1 CHECK (available IN (0, 1)),
                prep_time_mins INTEGER NOT NULL DEFAULT 10 CHECK (prep_time_mins >= 0),
                description    TEXT DEFAULT '',
                created_at     TEXT NOT NULL DEFAULT (datetime('now')),
                updated_at     TEXT NOT NULL DEFAULT (datetime('now')),
                
                FOREIGN KEY (category_id) REFERENCES categories(id)
                    ON DELETE RESTRICT
            )
        """)
        
        # ---- TABLE: orders ----
        cursor.execute("""
            CREATE TABLE IF NOT EXISTS orders (
                id            INTEGER PRIMARY KEY AUTOINCREMENT,
                customer_name TEXT NOT NULL DEFAULT 'Anonymous',
                status        TEXT NOT NULL DEFAULT 'received' 
                              CHECK (status IN (
                                  'received', 'preparing', 'ready',
                                  'delivered', 'cancelled'
                              )),
                total_price   REAL NOT NULL DEFAULT 0 CHECK (total_price >= 0),
                notes         TEXT DEFAULT '',
                created_at    TEXT NOT NULL DEFAULT (datetime('now')),
                updated_at    TEXT NOT NULL DEFAULT (datetime('now'))
            )
        """)
        
        # ---- TABLE: order_items (junction table) ----
        cursor.execute("""
            CREATE TABLE IF NOT EXISTS order_items (
                id         INTEGER PRIMARY KEY AUTOINCREMENT,
                order_id   INTEGER NOT NULL,
                item_id    INTEGER NOT NULL,
                quantity   INTEGER NOT NULL DEFAULT 1 CHECK (quantity > 0 AND quantity <= 100),
                unit_price REAL NOT NULL CHECK (unit_price >= 0),
                line_total REAL NOT NULL CHECK (line_total >= 0),
                
                FOREIGN KEY (order_id) REFERENCES orders(id)
                    ON DELETE CASCADE,
                FOREIGN KEY (item_id) REFERENCES menu_items(id)
                    ON DELETE RESTRICT
            )
        """)
        
        # ---- INDEXES for common queries ----
        cursor.execute("""
            CREATE INDEX IF NOT EXISTS idx_menu_items_category
            ON menu_items(category_id)
        """)
        cursor.execute("""
            CREATE INDEX IF NOT EXISTS idx_menu_items_available
            ON menu_items(available)
        """)
        cursor.execute("""
            CREATE INDEX IF NOT EXISTS idx_orders_status
            ON orders(status)
        """)
        cursor.execute("""
            CREATE INDEX IF NOT EXISTS idx_orders_created
            ON orders(created_at)
        """)
        cursor.execute("""
            CREATE INDEX IF NOT EXISTS idx_order_items_order
            ON order_items(order_id)
        """)
        cursor.execute("""
            CREATE INDEX IF NOT EXISTS idx_order_items_item
            ON order_items(item_id)
        """)
        
        conn.commit()
        
        # Seed data if tables are empty
        cursor.execute("SELECT COUNT(*) as cnt FROM categories")
        if cursor.fetchone()["cnt"] == 0:
            self._seed_data(cursor, conn)

    # ---- Helper: Row to Dict ----
    
    def row_to_dict(self, row):
        """Convert sqlite3.Row to plain dictionary"""
        if row is None:
            return None
        return dict(row)
    
    def rows_to_list(self, rows):
        """Convert list of sqlite3.Row to list of dicts"""
        return [dict(row) for row in rows]

    def list_items(self, filters=None, page=1, limit=10,
                   sort_by="id", sort_order="asc"):
        conn = self.get_connection()
        
        # Build dynamic query with parameterized conditions
        where_clauses = []
        params = []
        
        if filters:
            if "category" in filters:
                where_clauses.append("""
                    mi.category_id = (
                        SELECT id FROM categories WHERE name = ?
                    )
                """)
                params.append(filters["category"])
            
            if "available" in filters:
                where_clauses.append("mi.available = ?")
                params.append(1 if filters["available"].lower() == "true" else 0)
            
            if "min_price" in filters:
                where_clauses.append("mi.price >= ?")
                params.append(float(filters["min_price"]))
            
            if "max_price" in filters:
                where_clauses.append("mi.price <= ?")
                params.append(float(filters["max_price"]))
            
            if "search" in filters:
                where_clauses.append("mi.name LIKE ?")
                params.append(f"%{filters['search']}%")
        
        where_sql = ""
        if where_clauses:
            where_sql = "WHERE " + " AND ".join(where_clauses)
        
        # Whitelist sort columns to prevent SQL injection
        allowed_sorts = {
            "id": "mi.id",
            "name": "mi.name",
            "price": "mi.price",
            "category": "c.name",
            "created_at": "mi.created_at"
        }
        sort_column = allowed_sorts.get(sort_by, "mi.id")
        sort_dir = "DESC" if sort_order.lower() == "desc" else "ASC"
        
        # Count total matching rows (for pagination metadata)
        count_sql = f"""
            SELECT COUNT(*) as total
            FROM menu_items mi
            JOIN categories c ON c.id = mi.category_id
            {where_sql}
        """
        total = conn.execute(count_sql, params).fetchone()["total"]
        
        # Fetch paginated results with JOIN to get category name
        offset = (page - 1) * limit
        data_sql = f"""
            SELECT 
                mi.id,
                mi.name,
                c.name as category,
                mi.category_id,
                mi.price,
                mi.available,
                mi.prep_time_mins,
                mi.description,
                mi.created_at,
                mi.updated_at
            FROM menu_items mi
            JOIN categories c ON c.id = mi.category_id
            {where_sql}
            ORDER BY {sort_column} {sort_dir}
            LIMIT ? OFFSET ?
        """
        params.extend([limit, offset])
        
        rows = self.rows_to_list(conn.execute(data_sql, params).fetchall())
        
        # Convert available from 0/1 to boolean for API response
        for row in rows:
            row["available"] = bool(row["available"])
        
        total_pages = max(1, -(-total // limit))
        
        return {
            "items": rows,
            "pagination": {
                "total": total,
                "page": page,
                "limit": limit,
                "total_pages": total_pages,
                "has_next": page < total_pages,
                "has_prev": page > 1
            }
        }
    
    def get_item(self, item_id):
        conn = self.get_connection()
        cursor = conn.execute("""
            SELECT 
                mi.id, mi.name, c.name as category, mi.category_id,
                mi.price, mi.available, mi.prep_time_mins,
                mi.description, mi.created_at, mi.updated_at
            FROM menu_items mi
            JOIN categories c ON c.id = mi.category_id
            WHERE mi.id = ?
        """, (item_id,))
        row = self.row_to_dict(cursor.fetchone())
        if row:
            row["available"] = bool(row["available"])
        return row
    
    def create_item(self, data):
        conn = self.get_connection()
        now = datetime.datetime.now().isoformat()
        
        # Resolve category name to ID
        category_id = self._resolve_category(data.get("category", ""), conn)
        if category_id is None:
            return {"error": f"Category '{data.get('category')}' not found"}
        
        try:
            cursor = conn.execute("""
                INSERT INTO menu_items 
                    (name, category_id, price, available, prep_time_mins,
                     description, created_at, updated_at)
                VALUES (?, ?, ?, ?, ?, ?, ?, ?)
            """, (
                data["name"],
                category_id,
                data["price"],
                1 if data.get("available", True) else 0,
                data.get("prep_time_mins", 10),
                data.get("description", ""),
                now, now
            ))
            conn.commit()
            return self.get_item(cursor.lastrowid)
        
        except sqlite3.IntegrityError as e:
            conn.rollback()
            return {"error": str(e)}
    
    def update_item(self, item_id, data, partial=True):
        conn = self.get_connection()
        now = datetime.datetime.now().isoformat()
        
        existing = self.get_item(item_id)
        if not existing:
            return None
        
        if partial:
            # PATCH: merge with existing
            updates = {}
            if "name" in data:
                updates["name"] = data["name"]
            if "price" in data:
                updates["price"] = data["price"]
            if "available" in data:
                updates["available"] = 1 if data["available"] else 0
            if "prep_time_mins" in data:
                updates["prep_time_mins"] = data["prep_time_mins"]
            if "description" in data:
                updates["description"] = data["description"]
            if "category" in data:
                cat_id = self._resolve_category(data["category"], conn)
                if cat_id is None:
                    return {"error": f"Category '{data['category']}' not found"}
                updates["category_id"] = cat_id
            
            if not updates:
                return {"error": "No valid fields to update"}
            
            updates["updated_at"] = now
            set_clause = ", ".join(f"{k} = ?" for k in updates.keys())
            values = list(updates.values()) + [item_id]
            
            try:
                conn.execute(
                    f"UPDATE menu_items SET {set_clause} WHERE id = ?",
                    values
                )
                conn.commit()
            except sqlite3.IntegrityError as e:
                conn.rollback()
                return {"error": str(e)}
        
        else:
            # PUT: replace all fields
            category_id = self._resolve_category(
                data.get("category", ""), conn
            )
            if category_id is None:
                return {"error": f"Category '{data.get('category')}' not found"}
            
            try:
                conn.execute("""
                    UPDATE menu_items 
                    SET name = ?, category_id = ?, price = ?, available = ?,
                        prep_time_mins = ?, description = ?, updated_at = ?
                    WHERE id = ?
                """, (
                    data["name"], category_id, data["price"],
                    1 if data.get("available", True) else 0,
                    data.get("prep_time_mins", 10),
                    data.get("description", ""),
                    now, item_id
                ))
                conn.commit()
            except sqlite3.IntegrityError as e:
                conn.rollback()
                return {"error": str(e)}
        
        return self.get_item(item_id)
    
    def delete_item(self, item_id):
        conn = self.get_connection()
        
        # Check for active orders referencing this item
        cursor = conn.execute("""
            SELECT o.id as order_id, o.status
            FROM order_items oi
            JOIN orders o ON o.id = oi.order_id
            WHERE oi.item_id = ?
            AND o.status IN ('received', 'preparing')
        """, (item_id,))
        active_orders = self.rows_to_list(cursor.fetchall())
        
        if active_orders:
            return {
                "conflict": True,
                "message": "Item is in active orders",
                "active_orders": [o["order_id"] for o in active_orders]
            }
        
        try:
            cursor = conn.execute(
                "DELETE FROM menu_items WHERE id = ?", (item_id,)
            )
            conn.commit()
            return {"deleted": cursor.rowcount > 0}
        except sqlite3.IntegrityError as e:
            conn.rollback()
            return {"error": str(e)}
    
    def _resolve_category(self, category_name, conn):
        """Get category ID from name"""
        cursor = conn.execute(
            "SELECT id FROM categories WHERE name = ?",
            (category_name.lower(),)
        )
        row = cursor.fetchone()
        return row["id"] if row else None

    def create_order(self, data):
        """
        Creates an order with multiple items in a SINGLE TRANSACTION.
        If ANY step fails, EVERYTHING rolls back.
        """
        conn = self.get_connection()
        now = datetime.datetime.now().isoformat()
        
        try:
            # ---- BEGIN TRANSACTION ----
            conn.execute("BEGIN")
            
            # Step 1: Insert the order (with placeholder total)
            cursor = conn.execute("""
                INSERT INTO orders (customer_name, status, total_price,
                                    notes, created_at, updated_at)
                VALUES (?, 'received', 0, ?, ?, ?)
            """, (
                data.get("customer_name", "Anonymous"),
                data.get("notes", ""),
                now, now
            ))
            order_id = cursor.lastrowid
            
            # Step 2: Insert each order item
            total_price = 0
            order_items = []
            
            for entry in data.get("items", []):
                item_id = entry["item_id"]
                quantity = entry.get("quantity", 1)
                
                # Fetch current price from menu (not from client!)
                item_cursor = conn.execute(
                    "SELECT id, name, price, available FROM menu_items WHERE id = ?",
                    (item_id,)
                )
                menu_item = item_cursor.fetchone()
                
                if not menu_item:
                    # ROLLBACK: Item doesn't exist
                    conn.execute("ROLLBACK")
                    return {
                        "error": f"Menu item {item_id} does not exist"
                    }
                
                if not menu_item["available"]:
                    conn.execute("ROLLBACK")
                    return {
                        "error": f"'{menu_item['name']}' is currently unavailable"
                    }
                
                unit_price = menu_item["price"]
                line_total = round(unit_price * quantity, 2)
                total_price += line_total
                
                conn.execute("""
                    INSERT INTO order_items 
                        (order_id, item_id, quantity, unit_price, line_total)
                    VALUES (?, ?, ?, ?, ?)
                """, (order_id, item_id, quantity, unit_price, line_total))
                
                order_items.append({
                    "item_id": item_id,
                    "name": menu_item["name"],
                    "quantity": quantity,
                    "unit_price": unit_price,
                    "line_total": line_total
                })
            
            # Step 3: Update order with calculated total
            total_price = round(total_price, 2)
            conn.execute(
                "UPDATE orders SET total_price = ? WHERE id = ?",
                (total_price, order_id)
            )
            
            # ---- COMMIT TRANSACTION ----
            conn.execute("COMMIT")
            
            return {
                "id": order_id,
                "customer_name": data.get("customer_name", "Anonymous"),
                "items": order_items,
                "total_price": total_price,
                "status": "received",
                "notes": data.get("notes", ""),
                "created_at": now,
                "updated_at": now
            }
        
        except Exception as e:
            # ---- ROLLBACK ON ANY ERROR ----
            try:
                conn.execute("ROLLBACK")
            except Exception:
                pass
            return {"error": f"Failed to create order: {str(e)}"}
    
    def get_order(self, order_id):
        conn = self.get_connection()
        
        # Get order header
        cursor = conn.execute(
            "SELECT * FROM orders WHERE id = ?", (order_id,)
        )
        order = self.row_to_dict(cursor.fetchone())
        if not order:
            return None
        
        # Get order items with JOIN to menu_items for names
        items_cursor = conn.execute("""
            SELECT 
                oi.id,
                oi.item_id,
                mi.name as item_name,
                oi.quantity,
                oi.unit_price,
                oi.line_total
            FROM order_items oi
            JOIN menu_items mi ON mi.id = oi.item_id
            WHERE oi.order_id = ?
            ORDER BY oi.id
        """, (order_id,))
        
        order["items"] = self.rows_to_list(items_cursor.fetchall())
        return order
    
    def list_orders(self, filters=None, page=1, limit=10):
        conn = self.get_connection()
        
        where_clauses = []
        params = []
        
        if filters:
            if "status" in filters:
                where_clauses.append("o.status = ?")
                params.append(filters["status"])
            if "customer" in filters:
                where_clauses.append("o.customer_name LIKE ?")
                params.append(f"%{filters['customer']}%")
            if "min_total" in filters:
                where_clauses.append("o.total_price >= ?")
                params.append(float(filters["min_total"]))
            if "date_from" in filters:
                where_clauses.append("o.created_at >= ?")
                params.append(filters["date_from"])
            if "date_to" in filters:
                where_clauses.append("o.created_at <= ?")
                params.append(filters["date_to"])
        
        where_sql = ""
        if where_clauses:
            where_sql = "WHERE " + " AND ".join(where_clauses)
        
        # Count
        count_params = list(params)
        total = conn.execute(
            f"SELECT COUNT(*) as total FROM orders o {where_sql}",
            count_params
        ).fetchone()["total"]
        
        # Fetch with pagination
        offset = (page - 1) * limit
        data_params = params + [limit, offset]
        rows = self.rows_to_list(conn.execute(f"""
            SELECT * FROM orders o
            {where_sql}
            ORDER BY o.created_at DESC
            LIMIT ? OFFSET ?
        """, data_params).fetchall())
        
        # Attach item count to each order (avoid N+1 with subquery)
        order_ids = [r["id"] for r in rows]
        if order_ids:
            placeholders = ",".join("?" * len(order_ids))
            item_counts = conn.execute(f"""
                SELECT order_id, COUNT(*) as item_count,
                       SUM(quantity) as total_quantity
                FROM order_items
                WHERE order_id IN ({placeholders})
                GROUP BY order_id
            """, order_ids).fetchall()
            
            count_map = {
                r["order_id"]: {
                    "item_count": r["item_count"],
                    "total_quantity": r["total_quantity"]
                }
                for r in item_counts
            }
            
            for row in rows:
                counts = count_map.get(row["id"], {
                    "item_count": 0, "total_quantity": 0
                })
                row["item_count"] = counts["item_count"]
                row["total_quantity"] = counts["total_quantity"]
        
        total_pages = max(1, -(-total // limit))
        
        return {
            "orders": rows,
            "pagination": {
                "total": total,
                "page": page,
                "limit": limit,
                "total_pages": total_pages,
                "has_next": page < total_pages,
                "has_prev": page > 1
            }
        }
    
    def update_order_status(self, order_id, new_status):
        conn = self.get_connection()
        
        order = conn.execute(
            "SELECT * FROM orders WHERE id = ?", (order_id,)
        ).fetchone()
        
        if not order:
            return None
        
        valid_transitions = {
            "received": ["preparing", "cancelled"],
            "preparing": ["ready", "cancelled"],
            "ready": ["delivered"],
            "delivered": [],
            "cancelled": []
        }
        
        current = order["status"]
        if new_status not in valid_transitions.get(current, []):
            return {
                "transition_error": True,
                "message": f"Cannot go from '{current}' to '{new_status}'",
                "valid_next": valid_transitions.get(current, [])
            }
        
        now = datetime.datetime.now().isoformat()
        conn.execute(
            "UPDATE orders SET status = ?, updated_at = ? WHERE id = ?",
            (new_status, now, order_id)
        )
        conn.commit()
        
        return self.get_order(order_id)

            ORDER BY revenue DESC
        """).fetchall())
        
        # Overall stats
        overall = self.row_to_dict(conn.execute("""
            SELECT
                COUNT(*) as total_orders,
                ROUND(SUM(total_price), 2) as total_revenue,
                ROUND(AVG(total_price), 2) as avg_order_value,
                MAX(total_price) as highest_order,
                MIN(CASE WHEN total_price > 0 THEN total_price END) 
                    as lowest_order
            FROM orders
            WHERE status != 'cancelled'
        """).fetchone())
        
        return {
            "overall": overall,
            "revenue_by_status": revenue_by_status,
            "top_items": popular_items,
            "revenue_by_category": revenue_by_category
        }

    # DATABASE HEALTH & SCHEMA INFO
    # ================================================================
    
    def get_db_info(self):
        conn = self.get_connection()
        
        tables = self.rows_to_list(conn.execute("""
            SELECT name FROM sqlite_master 
            WHERE type='table' AND name NOT LIKE 'sqlite_%'
            ORDER BY name
        """).fetchall())
        
        table_info = {}
        for table in tables:
            name = table["name"]
            count = conn.execute(
                f"SELECT COUNT(*) as cnt FROM {name}"
            ).fetchone()["cnt"]
            
            columns = self.rows_to_list(conn.execute(
                f"PRAGMA table_info({name})"
            ).fetchall())
            
            table_info[name] = {
                "row_count": count,
                "columns": [
                    {
                        "name": col["name"],
                        "type": col["type"],
                        "not_null": bool(col["notnull"]),
                        "primary_key": bool(col["pk"])
                    }
                    for col in columns
                ]
            }
        
        fk_info = {}
        for table in tables:
            name = table["name"]
            fks = self.rows_to_list(conn.execute(
                f"PRAGMA foreign_key_list({name})"
            ).fetchall())
            if fks:
                fk_info[name] = [
                    {
                        "column": fk["from"],
                        "references": f"{fk['table']}.{fk['to']}",
                        "on_delete": fk["on_delete"]
                    }
                    for fk in fks
                ]
        
        indexes = self.rows_to_list(conn.execute("""
            SELECT name, tbl_name as table_name
            FROM sqlite_master
            WHERE type='index' AND name NOT LIKE 'sqlite_%'
            ORDER BY tbl_name, name
        """).fetchall())
        
        db_size = os.path.getsize(self.db_path) if os.path.exists(self.db_path) else 0
        
        return {
            "database_file": self.db_path,
            "file_size_bytes": db_size,
            "file_size_kb": round(db_size / 1024, 2),
            "tables": table_info,
            "foreign_keys": fk_info,
            "indexes": [
                {"name": idx["name"], "table": idx["table_name"]}
                for idx in indexes
            ],
            "pragma": {
                "foreign_keys": conn.execute(
                    "PRAGMA foreign_keys"
                ).fetchone()[0],
                "journal_mode": conn.execute(
                    "PRAGMA journal_mode"
                ).fetchone()[0]
            }
        }

class Validator:
    
    VALID_CATEGORIES = [
        "pizza", "burger", "salad", "appetizer",
        "dessert", "pasta", "beverage"
    ]
    
    @staticmethod
    def validate_menu_item(data, partial=False):
        errors = []
        
        if not partial or "name" in data:
            name = data.get("name", "")
            if not name or not isinstance(name, str):
                errors.append({
                    "field": "name",
                    "issue": "required, must be non-empty string"
                })
            elif len(name) > 100:
                errors.append({
                    "field": "name",
                    "issue": "must be 100 characters or fewer"
                })
        
        if not partial or "price" in data:
            price = data.get("price")
            if price is None and not partial:
                errors.append({"field": "price", "issue": "required"})
            elif price is not None:
                if not isinstance(price, (int, float)) or price < 0:
                    errors.append({
                        "field": "price",
                        "issue": "must be non-negative number"
                    })
                elif price > 9999.99:
                    errors.append({
                        "field": "price",
                        "issue": "must be less than 10000"
                    })
        
        if not partial or "category" in data:
            cat = data.get("category", "")
            if not partial and not cat:
                errors.append({"field": "category", "issue": "required"})
            elif cat and cat.lower() not in Validator.VALID_CATEGORIES:
                errors.append({
                    "field": "category",
                    "issue": f"must be one of: {Validator.VALID_CATEGORIES}"
                })
        
        if "available" in data:
            if not isinstance(data["available"], bool):
                errors.append({
                    "field": "available",
                    "issue": "must be true or false"
                })
        
        if "prep_time_mins" in data:
            pt = data["prep_time_mins"]
            if not isinstance(pt, int) or pt < 0:
                errors.append({
                    "field": "prep_time_mins",
                    "issue": "must be non-negative integer"
                })
        
        return errors
    
    @staticmethod
    def validate_order(data, db):
        errors = []
        
        items = data.get("items")
        if not items or not isinstance(items, list):
            errors.append({
                "field": "items",
                "issue": "required, must be non-empty list"
            })
            return errors
        
        if len(items) > 50:
            errors.append({
                "field": "items",
                "issue": "maximum 50 line items per order"
            })
            return errors
        
        for idx, entry in enumerate(items):
            item_id = entry.get("item_id")
            if not item_id or not isinstance(item_id, int):
                errors.append({
                    "field": f"items[{idx}].item_id",
                    "issue": "required, must be integer"
                })
                continue
            
            menu_item = db.get_item(item_id)
            if not menu_item:
                errors.append({
                    "field": f"items[{idx}].item_id",
                    "issue": f"item {item_id} does not exist"
                })
            elif not menu_item.get("available"):
                errors.append({
                    "field": f"items[{idx}].item_id",
                    "issue": f"'{menu_item['name']}' is unavailable"
                })
            
            qty = entry.get("quantity", 1)
            if not isinstance(qty, int) or qty < 1 or qty > 100:
                errors.append({
                    "field": f"items[{idx}].quantity",
                    "issue": "must be integer between 1 and 100"
                })
        
        return errors

class Router:
    def __init__(self):
        self.routes = []
    
    def add_route(self, method, pattern, handler):
        regex_pattern = re.sub(
            r'\{(\w+)\}',
            r'(?P<\1>[^/]+)',
            pattern
        )
        regex_pattern = f'^{regex_pattern}$'
        self.routes.append({
            "method": method.upper(),
            "regex": re.compile(regex_pattern),
            "handler": handler
        })
    
    def resolve(self, method, path):
        for route in self.routes:
            if route["method"] != method.upper():
                continue
            match = route["regex"].match(path)
            if match:
                params = match.groupdict()
                for key, value in params.items():
                    if value.isdigit():
                        params[key] = int(value)
                return route["handler"], params
        return None, {}

class RestaurantHandler(BaseHTTPRequestHandler):
    
    router = Router()
    
    # ---- Response Helpers (same pattern as Project 2) ----
    
    def send_json(self, status_code, data):
        self.send_response(status_code)
        self.send_header('Content-Type', 'application/json; charset=utf-8')
        self.send_header('Access-Control-Allow-Origin', '*')
        body = json.dumps(data, indent=2, default=str)
        self.send_header('Content-Length', str(len(body.encode('utf-8'))))
        self.end_headers()
        self.wfile.write(body.encode('utf-8'))
    
    def send_success(self, data, status_code=200, meta=None):
        response = {"success": True, "data": data}
        if meta:
            response["meta"] = meta
        self.send_json(status_code, response)
    
    def send_error(self, status_code, code, message, details=None):
        response = {
            "success": False,
            "error": {"code": code, "message": message}
        }
        if details:
            response["error"]["details"] = details
        self.send_json(status_code, response)
    
    def read_body(self):
        content_length = int(self.headers.get('Content-Length', 0))
        if content_length > 0:
            body = self.rfile.read(content_length)
            try:
                return json.loads(body.decode('utf-8')), None
            except json.JSONDecodeError as e:
                return None, str(e)
        return {}, None
    
    def get_query_params(self):
        parsed = urllib.parse.urlparse(self.path)
        return dict(urllib.parse.parse_qsl(parsed.query))
    
    def get_clean_path(self):
        parsed = urllib.parse.urlparse(self.path)
        return parsed.path.rstrip('/')
    
    # ---- Routing ----
    
    def route_request(self, method):
        path = self.get_clean_path()
        ts = datetime.datetime.now().strftime('%H:%M:%S')
        print(f"  [{ts}] {method:6s} {self.path}")
        
        handler, params = self.router.resolve(method, path)
        if handler:
            try:
                handler(self, **params)
            except Exception as e:
                print(f"  [ERROR] {e}")
                self.send_error(500, "INTERNAL_ERROR",
                                "An unexpected error occurred")
        else:
            self.send_error(404, "NOT_FOUND",
                            f"Endpoint {method} {path} not found")
    
    def do_GET(self):
        self.route_request("GET")
    
    def do_POST(self):
        self.route_request("POST")
    
    def do_PUT(self):
        self.route_request("PUT")
    
    def do_PATCH(self):
        self.route_request("PATCH")
    
    def do_DELETE(self):
        self.route_request("DELETE")
    
    def do_OPTIONS(self):
        self.send_json(200, {"methods": ["GET","POST","PUT","PATCH","DELETE"]})
    
    def log_message(self, format, *args):
        pass
